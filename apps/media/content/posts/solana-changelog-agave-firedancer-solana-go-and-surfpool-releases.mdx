**Agave, Firedancer, Solana Go, and Surfpool all published new releases this week**, while client teams advanced validator, wallet, and program-framework work across the Solana stack.

The June 10, 2026 updates also included a new Solana Improvement Documents (SIMDs) discussion for an upcoming-leader SysVar account and work across Agave, Mithril, kit-plugins, wallet-standard, and Pinocchio.

## New releases

Several Solana engineering projects published new versions, including validator clients and developer tooling.

- Agave published [v4.1.0-beta.3 release notes](https://github.com/anza-xyz/agave/releases/tag/v4.1.0-beta.3) and [v4.0.2 release notes](https://github.com/anza-xyz/agave/releases/tag/v4.0.2).
- Firedancer published [Testnet v0.1002.40103 release notes](https://github.com/firedancer-io/firedancer/releases/tag/v0.1002.40103) and [Mainnet v0.911.40002 release notes](https://github.com/firedancer-io/firedancer/releases/tag/v0.911.40002).
- Solana Go published [v1.21.0 release notes](https://github.com/solana-foundation/solana-go/releases/tag/v1.21.0).
- Surfpool published [v1.3.1 release notes](https://github.com/solana-foundation/surfpool/releases/tag/v1.3.1).

## SIMD and validator client work

Solana Improvement Documents (SIMDs) moved forward with a proposal related to leader scheduling information. Validator client work focused on Agave changes and Mithril replay work.

- A SIMD discussion proposes creating a new SysVar account with details about the upcoming leader: [SIMD discussion #558](https://github.com/solana-foundation/solana-improvement-documents/discussions/558).
- Agave added Shred Tracer work in [Agave PR #12928](https://github.com/anza-xyz/agave/pull/12928).
- Agave worked on XdpSender in [Agave PR #12897](https://github.com/anza-xyz/agave/pull/12897).
- Agave worked on Repair Signing Pool in [Agave PR #12987](https://github.com/anza-xyz/agave/pull/12987).
- Agave added `Bank: add_root` work in [Agave PR #12913](https://github.com/anza-xyz/agave/pull/12913).
- Agave worked on Dynamic VAT in [Agave PR #12990](https://github.com/anza-xyz/agave/pull/12990).
- Agave added 802.1Q VLAN encapsulation to the XDP transmit loop in [Agave PR #12920](https://github.com/anza-xyz/agave/pull/12920).
- Agave is reconfiguring its multicast implementation in [Agave PR #12923](https://github.com/anza-xyz/agave/pull/12923).
- Mithril is working to implement Replay from Solana Turbine in [Mithril PR #243](https://github.com/Overclock-Validator/mithril/pull/243).

## Web and wallet client work

Client work included kit-plugins changes and an offchain message signing specification spanning wallet-standard, Agave, and a specification page.

- An alternative to wallet adapter is tracked in [kit-plugins PR #173](https://github.com/anza-xyz/kit-plugins/pull/173).
- Offchain message signing work is tracked across [wallet-standard PR #92](https://github.com/anza-xyz/wallet-standard/pull/92), [Agave PR #12965](https://github.com/anza-xyz/agave/pull/12965), the [Solana Off-Chain Message Signing (OCMS) specification](https://app.notion.com/p/solanafoundation/Solana-Off-Chain-Message-Signing-OCMS-374d36dad52d80bf981ddbcebfeca5f0), and [wallet-standard PR #93](https://github.com/anza-xyz/wallet-standard/pull/93).

## Program frameworks and developer tooling

Program-framework work included a Pinocchio change, and developer tooling updates included a way to deploy on Devnet without SOL.

- A Pinocchio program-framework change is tracked in [Pinocchio PR #421](https://github.com/anza-xyz/pinocchio/pull/421).
- Kora Deploy allows builders to deploy a Solana program on Devnet without SOL, according to this [Kora Deploy post](https://x.com/dev_jodee/status/2064439356626964752).
