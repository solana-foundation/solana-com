---
title: How to Create a Token-2022 Mint with Extensions
description:
  Create a Token-2022 mint with the metadata, pausable, permanent delegate, and
  default account state extensions in a single transaction.
---

Token-2022 [extensions](/docs/tokens/extensions) must be enabled when the mint
account is created - they cannot be added later. This recipe creates a
compliance-ready mint of the kind used for
[stablecoins and tokenized assets](/docs/tokenization): onchain metadata,
pausable transfers, a permanent delegate, and token accounts that start frozen
until the issuer thaws them.

Most extensions are initialized before `InitializeMint`; token metadata is
written after it, so the account is allocated without the metadata bytes but
funded with rent for the full size.

Drop any extension you don't need from the `extensions` array - the sizing,
pre-initialize, and post-initialize helpers all derive from it. Each extension's
own reference page under [Token Extensions](/docs/tokens/extensions) covers its
runtime behavior.
