Um Cross-Program Invocation (CPI) ocorre quando um
[programa](/docs/core/programs) chama uma [instrução](/docs/core/instructions)
em outro programa durante a execução. CPIs permitem composabilidade: as
instruções de qualquer programa podem ser invocadas por qualquer outro programa
na rede.

## Factos principais

- **Duas funções**: _rs`invoke`_ (sem assinatura PDA) e _rs`invoke_signed`_ (com
  assinatura PDA).
- **Extensão de privilégios**: Os privilégios de [conta](/docs/core/accounts)
  (signatário, gravável) estendem-se do chamador para o chamado. Um chamado não
  pode escalar privilégios além do que o chamador passou.
- **Orçamento de computação partilhado**: O consumo de CU do chamado reduz o
  orçamento restante do chamador.
- **Reentrância**: A auto-recursão direta é permitida (A->A->A). A reentrância
  indireta não é permitida (A->B->A retorna _rs`ReentrancyNotAllowed`_).

## `invoke` vs `invoke_signed`

Solana fornece duas funções para fazer CPIs:

| Função          | Caso de uso                                                                  | Assinatura PDA                  |
| --------------- | ---------------------------------------------------------------------------- | ------------------------------- |
| `invoke`        | CPIs onde todos os signatários necessários já assinaram a transação original | Não                             |
| `invoke_signed` | CPIs onde o programa chamador precisa assinar em nome de um PDA que possui   | Sim, via sementes de signatário |

Internamente, _rs`invoke`_ simplesmente chama _rs`invoke_signed`_ com um array
vazio de sementes de signatário. Use _rs`invoke`_ quando não precisar de
assinatura PDA, e _rs`invoke_signed`_ quando o programa precisar autorizar uma
ação em nome de um PDA.

Ambas as funções acabam por acionar a mesma syscall
([`sol_invoke_signed_rust`](https://github.com/anza-xyz/agave/blob/v3.1.8/syscalls/src/cpi.rs#L11-L33))
e fluem pelo mesmo caminho de runtime
([`cpi_common`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/cpi.rs#L802-L925)).
A única diferença é se as sementes de signatário são fornecidas. Quando as
sementes são fornecidas, o runtime deriva chaves públicas PDA e adiciona-as ao
conjunto de signatários válidos antes da verificação de privilégios.
