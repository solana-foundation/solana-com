<Callout type="info" title="Resumo">
  Verificações de tempo de execução após cada instrução: apenas o proprietário
  pode debitar lamports ou modificar dados, os dados podem crescer no máximo 10
  KiB por instrução, mudanças de proprietário requerem dados inicializados a
  zero, e a flag executável é irreversível.
</Callout>

O tempo de execução Solana aplica estas regras após a execução de cada instrução
através dos métodos
[`BorrowedInstructionAccount`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L722-L727).
Cada regra é verificada no momento da modificação, e a transação é revertida se
alguma verificação falhar.

## Regras de lamports

| Regra                                                         | Aplicação                                                                                                                                                                                                                                          | Erro                          |
| ------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Apenas o proprietário pode debitar lamports                   | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L785): verifica `is_owned_by_current_program()` quando `lamports < current`                                                                   | `ExternalAccountLamportSpend` |
| Contas somente leitura não podem ter lamports alterados       | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L788-L789): verifica `is_writable()`                                                                                                               | `ReadonlyLamportChange`       |
| Qualquer programa pode creditar lamports a uma conta gravável | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L789): a verificação de propriedade aplica-se apenas quando o novo saldo é inferior ao saldo atual; a verificação de gravável ainda se aplica | `ReadonlyLamportChange`       |
| Lamports devem estar equilibrados numa instrução              | [`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L366-L398): verifica `get_lamports_delta() == 0`                                                                                        | `UnbalancedInstruction`       |

## Transições de estado de rent

As contas existem num de três valores
[`RentState`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L19-L30):
_rs`Uninitialized`_ (0 lamports), _rs`RentPaying`_ (acima de 0 mas abaixo do
mínimo de isenção de rent), e _rs`RentExempt`_ (no mínimo ou acima). Transições
não permitidas produzem _rs`TransactionError::InsufficientFundsForRent`_.

O runtime aplica estas regras através de
[`transition_allowed()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L107-L126):

- Qualquer conta pode transitar para _rs`Uninitialized`_ (fechar) ou
  _rs`RentExempt`_.
- Nenhuma conta pode entrar em _rs`RentPaying`_ a partir de _rs`Uninitialized`_
  ou _rs`RentExempt`_. Todas as novas contas devem estar isentas de rent.

## Regras de empréstimo de contas

Durante a execução de instruções, o runtime aplica semântica de empréstimo de
escritor único as contas. Um programa pode obter uma referência mutável ou
múltiplas referências imutáveis a uma conta, mas não ambas simultaneamente. Se
um programa tentar emprestar uma conta que já está mutavelmente emprestada (ou
emprestar mutavelmente uma conta que já está imutuavelmente emprestada), a
instrução falha com _rs`AccountBorrowFailed`_ através de
[`try_borrow()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L498-L508)
e
[`try_borrow_mut()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L512-L520).
Se uma instrução for concluída enquanto um empréstimo ainda estiver pendente, o
runtime retorna _rs`AccountBorrowOutstanding`_ em
[`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L374-L385).
