# Transações

Métodos para enviar, simular e gerenciar transações.

## sendTransaction

```typescript
sendTransaction(tx: Transaction): TransactionMetadata | FailedTransactionMetadata
```

Executa uma transação. Retorna metadados em caso de sucesso ou detalhes do erro
em caso de falha.

```typescript
const result = client.svm.sendTransaction(signedTx);

// Check for errors
if ("err" in result && result.err()) {
  console.error("Transaction failed:", result.err());
  return;
}

// Access result properties (note: these are getter functions)
console.log("Signature:", result.signature());
console.log("Compute units:", result.computeUnitsConsumed());
console.log("Logs:", result.logs());
```

<Callout type="warn">
  Propriedades do resultado da transação como `err()`, `computeUnitsConsumed()`,
  `logs()` e `signature()` são **funções getter**, não propriedades. Sempre as
  invoque com parênteses.
</Callout>

### Métodos de TransactionMetadata

| Método                   | Retorna                 | Descrição                         |
| ------------------------ | ----------------------- | --------------------------------- |
| `signature()`            | `Uint8Array`            | Bytes da assinatura da transação  |
| `computeUnitsConsumed()` | `bigint`                | Unidades de computação utilizadas |
| `logs()`                 | `string[]`              | Logs do programa                  |
| `innerInstructions()`    | `InnerInstruction[]`    | Instruções internas (CPI)         |
| `returnData()`           | `TransactionReturnData` | Dados de retorno dos programas    |
| `prettyLogs()`           | `string`                | Logs formatados com cores         |

## simulateTransaction

```typescript
simulateTransaction(tx: Transaction): SimulatedTransactionInfo | FailedTransactionMetadata
```

Simula uma transação sem modificar o estado.

```typescript
const simResult = client.svm.simulateTransaction(signedTx);

if ("err" in simResult && simResult.err()) {
  console.error("Simulation failed:", simResult.err());
  return;
}

// Use .meta() to get TransactionMetadata
const meta = simResult.meta();
console.log("Would use", meta.computeUnitsConsumed(), "compute units");
console.log("Logs:", meta.logs());

// Get post-execution account states
const postAccounts = simResult.postAccounts();
console.log("Post-state accounts:", postAccounts);
```

<Callout type="info">
  A simulação não altera o estado. Use-a para verificar as transações antes de
  enviá-las.
</Callout>

## Gerenciamento de Blockhash

### latestBlockhash

```typescript
latestBlockhash(): Blockhash
```

Obtém o blockhash atual.

```typescript
const blockhash = client.svm.latestBlockhash();
console.log("Blockhash:", blockhash);
```

### getLatestBlockhash (via client.rpc)

Obtém o blockhash com informações de tempo de vida para construir transações
manualmente. Use o helper RPC, pois ele retorna tanto `blockhash` quanto
`lastValidBlockHeight`.

```typescript
import {
  createTransactionMessage,
  setTransactionMessageLifetimeUsingBlockhash
} from "@solana/kit";

const { value: blockhashLifetime } = await client.rpc
  .getLatestBlockhash()
  .send();

const tx = setTransactionMessageLifetimeUsingBlockhash(
  blockhashLifetime,
  createTransactionMessage({ version: 0 })
);
```

<Callout type="info">
  Ao usar plugins de programa como `systemProgram()` ou `tokenProgram()`, você
  nunca precisa buscar o blockhash manualmente — `client.sendTransaction()` (e o
  helper `.sendTransaction()` em cadeias de instruções) cuida disso para você.
</Callout>

### expireBlockhash

```typescript
expireBlockhash(): LiteSVM
```

Avança para um novo blockhash. Útil para testar cenários de expiração de
blockhash.

```typescript
const oldBlockhash = client.svm.latestBlockhash();
client.svm.expireBlockhash();
const newBlockhash = client.svm.latestBlockhash();

console.log("Old:", oldBlockhash);
console.log("New:", newBlockhash);
// Blockhashes will be different
```

## Histórico de Transações

Ative o histórico de transações para recuperar transações após o envio:

```typescript
// Enable history storage (stores last N transactions)
client.svm.withTransactionHistory(100n);

// Send a transaction
const result = client.svm.sendTransaction(signedTx);

// Later, retrieve it by signature (base58 string)
// Note: getTransaction expects a base58-encoded signature string
```
