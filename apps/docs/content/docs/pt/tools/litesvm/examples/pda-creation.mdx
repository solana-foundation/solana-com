## Teste Completo

```rust title="tests/pda_test.rs"
use litesvm::LiteSVM;
use solana_account::Account;
use solana_sdk::pubkey::Pubkey;

#[test]
fn test_pda_creation() {
    let mut svm = LiteSVM::new();

    // Program that owns the PDA
    let program_id = Pubkey::new_unique();

    // Derive PDA address
    let seed = b"my_pda";
    let (pda, bump) = Pubkey::find_program_address(
        &[seed],
        &program_id
    );

    println!("PDA: {}", pda);
    println!("Bump: {}", bump);

    // Create PDA account manually (for testing)
    svm.set_account(pda, Account {
        lamports: 1_000_000,
        data: vec![bump], // Store bump seed in data
        owner: program_id,
        executable: false,
        rent_epoch: 0,
    }).unwrap();

    // Verify PDA was created
    let account = svm.get_account(&pda).unwrap();
    assert_eq!(account.owner, program_id);
    assert_eq!(account.data[0], bump);

    println!("PDA created successfully!");
}
```

## Pontos Principais

1. **Derivação de PDA**: Use `Pubkey::find_program_address()` para derivar
   endereços PDA
2. **Seeds**: Os PDAs são derivados de seeds e do ID do programa
3. **Bump Seed**: O bump seed garante que o endereço está fora da curva ed25519
4. **Criação de Conta**: Use `set_account()` para criar contas manualmente para
   testes
5. **Propriedade**: Os PDAs devem ser de propriedade do programa que os derivou
