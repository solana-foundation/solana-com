title: Signatários SVM
description: Signatários para Solana e Blockchains Compatíveis com SVM

Esses signatários estão disponíveis ao usar o addon SVM para assinatura de
transações.

## secret_key

O signatário `svm::secret_key` pode ser usado para assinar uma transação de
forma síncrona utilizando uma chave secreta, mnemônico ou arquivo keypair.

### Entradas

| Name              | Required | Type   | Description                                                      |
| ----------------- | -------- | ------ | ---------------------------------------------------------------- |
| `secret_key`      | optional | string | A chave secreta usada para assinar mensagens e transações        |
| `mnemonic`        | optional | string | A frase mnemônica usada para gerar a chave secreta               |
| `derivation_path` | optional | string | O caminho de derivação usado para gerar a chave secreta          |
| `keypair_json`    | optional | string | Um caminho para um arquivo keypair.json contendo a chave secreta |

### Saídas

| Name         | Type   | Description                            |
| ------------ | ------ | -------------------------------------- |
| `public_key` | string | A chave pública da conta               |
| `address`    | string | O endereço SVM (alias para public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Você também pode usar um mnemônico:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Ou um arquivo keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

O signatário `svm::web_wallet` permitirá que um operador de Runbook assine a
transação com o signatário de navegador de sua escolha.

### Entradas

| Name               | Required | Type   | Description                                                            |
| ------------------ | -------- | ------ | ---------------------------------------------------------------------- |
| `expected_address` | optional | string | O endereço SVM esperado para conexão (omita para permitir qualquer um) |

### Saídas

| Name         | Type   | Description                        |
| ------------ | ------ | ---------------------------------- |
| `address`    | string | O endereço da conta conectada      |
| `public_key` | string | A chave pública da conta conectada |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Este assinante solicitará ao utilizador na interface Web que:

1. Conecte a sua carteira
2. Forneça uma pubkey através de assinatura de mensagem
3. Assine todas as transações que requeiram este assinante

## squads

O assinante `svm::squads` pode ser utilizado para assinar transações com um
cofre multisig Squads.

### Entradas

| Nome      | Obrigatório | Tipo   | Descrição                 |
| --------- | ----------- | ------ | ------------------------- |
| `address` | opcional    | string | O endereço do cofre squad |

### Saídas

| Nome         | Tipo   | Descrição                              |
| ------------ | ------ | -------------------------------------- |
| `public_key` | string | A pubkey do cofre squad                |
| `address`    | string | O endereço SVM (alias para public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

A utilização do multisig Squads permite a assinatura segura de transações por
múltiplas partes sem expor chaves privadas individuais.
