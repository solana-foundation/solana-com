  Construa um cliente Solana a partir de plugins do Kit para RPC, assinatura e

`@solana/kit` fornece `createClient`, um host simples que você estende com
plugins. Cada plugin adiciona uma capacidade — uma conexão RPC, um assinador,
planejamento de transações — e o cliente composto suporta scripts, rotas de
servidor e interfaces igualmente.

<Callout type="info">
  Construindo um app React? O [guia React](/docs/frontend/react-hooks) encapsula
  este mesmo cliente com `@solana/react` para que os componentes leiam o estado
  da carteira e os dados RPC através de hooks. Comece por lá e volte aqui quando
  precisar de maior controle.
</Callout>

## Instalar

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

Use qualquer gerenciador de pacotes. O Kit é v7+ e os pacotes
`@solana/kit-plugin-*` são 0.13+.

## Criar um cliente

Componha plugins com `.use(...)`. Defina primeiro um assinador, depois a conexão
RPC. `solanaDevnetRpc()` adiciona `client.rpc`, assinaturas, o
planejador/executor de transações e o helper `client.sendTransaction` — assim um
app raramente precisa conectar essas partes manualmente.

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

O plugin `signer` define tanto o pagador de taxas quanto a identidade do app. O
pacote RPC exige que um pagador esteja configurado primeiro, por isso a ordem
importa.

### Variantes de assinador

`@solana/kit-plugin-signer` expõe três funções para que você conceda apenas o
que um cliente precisa:

- `signer(x)` — define tanto `client.payer` quanto `client.identity`.
- `payer(x)` — somente pagador de taxas.
- `identity(x)` — somente autoridade do app.

Cada um possui variantes `*FromFile(path)` (Node), `generated*()`,
`generated*WithSol(amount)` e `airdrop*()` para testes e scripts.

## Enviar uma transação

Construa instruções com um cliente de programa gerado e, em seguida, passe-as
para `client.sendTransaction`. O planejador resolve o blockhash, estima o
processamento, assina com o pagador do cliente e envia.

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

`sendTransaction` resolve para um objeto de resultado; a assinatura da transação
está em `result.context.signature`. Ele verifica se o plano cabe em uma única
transação e lança um erro caso contrário — use `sendTransactions` quando um lote
pode abranger múltiplas transações.

## Escolha um pacote RPC

`@solana/kit-plugin-rpc` disponibiliza presets de cluster:

- `solanaDevnetRpc()` — configurações padrão de devnet, airdrop ativado
  (`client.airdrop`).
- `solanaMainnetRpc()` — mainnet; airdrop gera erro em tempo de compilação.
- `solanaLocalRpc()` — `http://127.0.0.1:8899` para um validator local.
- `solanaRpc({ rpcUrl })` — qualquer endpoint.

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### Composição granular

Os presets agrupam a conexão RPC, o planejador, o executor e os auxiliares de
envio, portanto a maioria das aplicações nunca precisará das peças
individualmente. Recorra aos plugins granulares apenas para personalizar um
deles — por exemplo, um planejador personalizado ou um canal de subscrição
diferente. Esse caminho também requer `@solana/kit-plugin-instruction-plan`, que
os presets incluem automaticamente:

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## Adicionar clientes de programa

Os pacotes `@solana-program/*` gerados expõem plugins de cliente para que as
chamadas de programa fiquem vinculadas ao cliente. Instale o programa necessário
e adicione seu plugin com `.use()`:

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="Intervalos de peer dependency">
  Algumas versões do `@solana-program/*` ainda declaram um peer `@solana/kit
  ^6.x` enquanto o Kit está na versão 7.x. Eles funcionam com o Kit 7 —
  substitua o intervalo de peer no seu gerenciador de pacotes caso ele reclame.
</Callout>

## Assinatura sem uma carteira de navegador

Executando em modo headless em uma rota de API, worker ou script? Assine com um
serviço de gerenciamento de chaves backend em vez de um keypair diretamente.
Consulte [Keychain](/docs/tools/keychain) e
[Assinatura em Produção](/docs/core/transactions/signing-in-production).

## Padrões comuns para desenvolvedores Solana

- **Configuração compartilhada**: Construa o cliente uma vez em um módulo
  central e importe-o em scripts, rotas de servidor e na sua interface.
- **Leituras do lado do servidor**: Um cliente com apenas `signer` + `solanaRpc`
  (sem plugin de carteira) lê contas e simula transações a partir do servidor.
- **Testes locais**: Substitua o pacote RPC por `.use(litesvm())` do
  `@solana/kit-plugin-litesvm` para executar contra um validator em memória
  (somente Node).
- **Testabilidade**: O cliente composto é fácil de simular — substitua respostas
  RPC ou signatários sem a necessidade de uma carteira de navegador.
