`@solana/react` publica um cliente Kit na sua árvore de componentes através de
um provedor, e então os hooks leem dados RPC, assinaturas e estado da carteira a
partir dele. Os hooks de carteira do navegador vivem em
`@solana/kit-plugin-wallet/react` e leem do mesmo provedor — não há um provedor
de carteira separado.

## Instalar

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Construa o cliente

Construa o cliente uma vez, no escopo do módulo, fora da árvore React. No
navegador, `walletSigner` torna a carteira conectada tanto o pagador de taxas
quanto a identidade.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` adiciona `client.wallet` (descoberta e conexão do Wallet
Standard), e `solanaRpc` adiciona `client.rpc` mais o auxiliar
`client.sendTransaction`. Exporte o tipo do cliente encapsulado em `Awaited<>`
para que permaneça correto caso você adicione posteriormente um plugin que
resolva de forma assíncrona, e passe esse `AppClient` para cada chamada
`useClient`.

## Envolva sua árvore uma vez

`ClientProvider` publica o cliente. Cada hook abaixo lê a partir dele.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Hooks de carteira

Os hooks de carteira leem a capacidade `wallet` do cliente, portanto só
funcionam dentro de um `ClientProvider` cujo cliente instalou um plugin de
carteira.

- `useWallets()` — Carteiras Wallet Standard descobertas para a chain do
  cliente.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` ou `null`. `signer` é
  `null` para carteiras somente leitura.
- `useConnect()` / `useDisconnect()` — retornam um `ActionResult` cujo
  `dispatch` (e `dispatchAsync`) executa a ação e cujos `isRunning`, `status`,
  `data`, `error` e `reset` rastreiam seu resultado.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Enviar uma transação

`useClient` retorna o mesmo cliente que você publicou. Construa instruções com
um cliente de programa gerado e envie-as através de `client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Passar o tipo `AppClient` exportado para `useClient` fornece ao hook o cliente
totalmente estendido pelo plugin, de modo que `client.rpc` e
`client.sendTransaction` ficam tipados.

## Ler dados

`@solana/react` fornece hooks que leem através do cliente:

- `useRequest(source)` — leitura RPC pontual; retorna
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — uma assinatura em tempo real; retorna
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — um valor inicializado via RPC e mantido atualizado
  por uma assinatura.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Memorize a fonte (com `useMemo`, ou `useCallback` para uma função) para que o
hook recarregue apenas quando suas entradas mudarem, e não a cada renderização.

Para cache e revalidação, os subcaminhos `@solana/react/swr` e
`@solana/react/query` envolvem os mesmos hooks para
[SWR](https://swr.vercel.app) e [TanStack Query](https://tanstack.com/query).

## Padrões comuns para devs Solana

- **Um único provider**: `ClientProvider` é o único provider necessário; os
  hooks de carteira o leem através da capacidade `wallet` do cliente.
- **Compatível com Server Components**: Marque apenas os componentes folha que
  chamam hooks com `"use client"`; leituras no servidor podem usar um cliente
  RPC Kit simples sem carteira.
- **Testes**: Publique um cliente simulado através de `ClientProvider` para
  simular carteiras e respostas RPC.

<Callout type="warn" title="Duas famílias de hooks, mesmos nomes">
  `useSignIn` e `useSignMessage` existem tanto em `@solana/react` (recebem um
  argumento `UiWalletAccount`) quanto em `@solana/kit-plugin-wallet/react`
  (baseados no cliente, sem argumento de conta, retornam um `ActionResult`). Use
  as versões de `@solana/kit-plugin-wallet/react` com o padrão de cliente acima;
  os hooks de carteira mais antigos de `@solana/react` estão sendo substituídos.
</Callout>

Combine este guia com a [visão geral do cliente Kit](/docs/frontend/client) para
compreender o cliente que cada hook utiliza.
