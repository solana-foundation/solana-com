Se você vem do `@solana/web3.js` (v1), `@solana/wallet-adapter-*`, ou dos
pacotes framework-kit anteriores (`@solana/client`, `@solana/react-hooks`,
`@solana/web3-compat`), essas abordagens foram substituídas. Novos aplicativos
são construídos diretamente sobre o `@solana/kit` e os plugins do Kit. Esta
página mapeia os conceitos que você já conhece para seus equivalentes no Kit.

## Mapeamento de conceitos

| web3.js v1 / wallet-adapter            | Kit                                                                            |
| -------------------------------------- | ------------------------------------------------------------------------------ |
| `new Connection(url)`                  | `createSolanaRpc(url)`, ou `.use(solanaRpc({ rpcUrl }))` em um cliente         |
| `PublicKey`                            | `Address` — converta uma string com `address("...")`                           |
| `Keypair`                              | `generateKeyPairSigner()` ou os plugins `@solana/kit-plugin-signer`            |
| `SystemProgram.transfer(...)`          | `getTransferSolInstruction(...)` de `@solana-program/system`                   |
| `LAMPORTS_PER_SOL` math                | `lamports(...)` de `@solana/kit`                                               |
| `Transaction` / `VersionedTransaction` | Mensagens de transação construídas e empacotadas pelo planejador de transações |
| `sendAndConfirmTransaction(...)`       | `client.sendTransaction([...])` (o planejador assina, envia e confirma)        |
| `@solana/wallet-adapter-*`             | `@solana/kit-plugin-wallet` + descoberta via Wallet Standard                   |

## Vindo do web3.js?

Se você mantém uma grande base de código `@solana/web3.js` v1 e uma reescrita
completa para o Kit ainda não é viável, o web3.js v3 é a ponte. Ele é
distribuído atualmente como `@solana/web3.js@rc` (a linha `3.0.0-rc`) e
reconstrói a API clássica baseada em classes — `Connection`, `Keypair`,
`Transaction` — sobre os internos do `@solana/kit`. Como ele se baseia no Kit, a
interoperabilidade é estreita: `PublicKey` é um alias obsoleto de `Address`, e
um `Keypair` do v3 satisfaz estruturalmente o `KeyPairSigner` do Kit, portanto
você pode passá-lo diretamente para APIs do Kit, plugins e clientes gerados pelo
Codama.

O v3 ainda é um **release candidate** — fixe versões exatas e espere alguma
instabilidade de API entre os RCs. É um caminho de interoperabilidade legada,
não o padrão para novos projetos; novos aplicativos devem ser construídos
diretamente no Kit. Para a migração em si, consulte o
[guia oficial de migração web3.js v1 → v3](https://github.com/solana-foundation/solana-web3.js/blob/v3.x/docs/web3js-v1-to-v3-migration.md).

## Carteiras

Kit não utiliza wallet-adapter. `@solana/kit-plugin-wallet` descobre carteiras
compatíveis com o Wallet Standard e as expõe através de `client.wallet` e dos
[React hooks](/docs/frontend/react-hooks). Carteiras modernas (Phantom,
Solflare, Backpack, entre outras) se anunciam pelo Wallet Standard, portanto
nenhum pacote adaptador por carteira é necessário.

Você também pode utilizar ferramentas pré-construídas como
[@solana/connector](https://www.npmjs.com/package/@solana/connector), que usa
primitivos do Kit diretamente para se conectar às carteiras.

<Callout type="info" title="Ainda usando wallet-adapter?">
  Se você mantém um aplicativo desenvolvido com `@solana/wallet-adapter-*` e não
  está pronto para migrar, o [repositório wallet-adapter
  ](https://github.com/anza-xyz/wallet-adapter) continua sendo a referência para
  essa stack. Para qualquer coisa nova, prefira o plugin de carteira do Kit.
</Callout>
