Um guia para configurar o seu ambiente de desenvolvimento local Solana. Aprenda
a instalar Rust, a CLI Solana e o Anchor Framework no Windows (WSL), Linux e
Mac. Use este guia se preferir instalar cada dependência individualmente, ou se
a [instalação rápida](/docs/intro/installation/) falhar por qualquer motivo.

### Pré-requisitos

<Accordions>
<Accordion title="Windows">

Para desenvolver programas Solana no Windows, **deve** usar o
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Após instalar o WSL, todas as outras dependências serão instaladas
através do terminal Linux.

Para instalar o WSL, execute o seguinte comando no Windows PowerShell:

```terminal
$ wsl --install
```

O processo de instalação irá solicitar que crie uma conta de utilizador
predefinida.

![Instalação do WSL](/assets/docs/intro/installation/wsl-install.png)

Por predefinição, o WSL instala o Ubuntu. Pode abrir um terminal Linux
procurando "Ubuntu" na barra de pesquisa.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Se o seu terminal Ubuntu se parecer com a imagem abaixo, poderá encontrar um
problema em que `ctrl + v` (atalho de teclado para colar) não funciona no
terminal.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Se encontrar este problema, abra o Windows Terminal procurando "Terminal" na
barra de pesquisa.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Em seguida, feche o Windows Terminal e reabra um terminal Linux procurando
Ubuntu novamente. O terminal deverá agora parecer-se com a imagem abaixo, onde
`ctrl + v` (atalho de teclado para colar) funciona.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Se usar o VS Code, a
[extensão WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
permite-lhe usar o WSL e o VS Code em conjunto.

![Configuração do WSL no VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Você verá o seguinte na barra de status do VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Depois de configurar o WSL, você está pronto para:

- Concluir a [instalação rápida](/docs/intro/installation) usando o terminal
  Ubuntu (Linux)
- Instalar o [Rust](#install-rust), a [CLI do Solana](#install-solana-cli) e a
  [CLI do Anchor](#install-anchor-cli) individualmente, conforme descrito abaixo

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Se estiver no Debian ou em alternativas derivadas do Debian como Ubuntu ou Mint, siga os passos abaixo:

1. Atualize as listas de pacotes para garantir que todos os pacotes estejam
   atualizados:

```terminal
$ sudo apt-get update
```

2. Instale as seguintes dependências:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Se você encontrar o seguinte erro ao instalar `protobuf-compiler`, certifique-se
de executar primeiro _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Para derivados do RHEL como Fedora ou Red Hat Linux, siga os seguintes passos.

Atualize as listas de pacotes para obter as versões mais recentes dos pacotes
executando o comando.

```terminal
$ sudo dnf upgrade --refresh
```

Em seguida, instale as seguintes dependências:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

  </Tab>
</Tabs>

### Instalar o Rust

Os desenvolvedores criam programas Solana usando a linguagem de programação
[Rust](https://www.rust-lang.org/).

1. Instale o Rust usando [rustup](https://www.rust-lang.org/tools/install)
   digitando o seguinte comando no seu terminal:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Após uma instalação bem-sucedida, você verá a seguinte mensagem:

<Accordions>
<Accordion title="Mensagem de instalação bem-sucedida do Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Recarregue a sua variável de ambiente PATH para incluir o diretório bin do
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Verifique se a instalação foi bem-sucedida.

```terminal
$ rustc --version
```

Você verá uma saída como a seguinte:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Instalar Solana CLI

A Solana CLI fornece todas as ferramentas necessárias para construir e
implementar programas Solana.

1. Instale o conjunto de ferramentas Solana CLI usando o comando de instalação
   oficial:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Você pode substituir `stable` pela etiqueta de lançamento correspondente à
  versão do software do seu lançamento desejado (por exemplo, `v2.0.3`), ou usar
  um dos três nomes de canal simbólicos: `stable`, `beta`, ou `edge`.
</Callout>

2. Adicione uma variável de ambiente PATH

Uma instalação inicial da Solana CLI pode solicitar que você adicione uma
variável de ambiente `PATH`. Para fazer isso, feche e reabra o seu terminal ou
execute o seguinte no seu shell:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Atualize a sua variável de ambiente PATH

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Se você estiver a usar **Linux** ou **WSL**, deve adicionar o binário da Solana
CLI ao seu PATH para que o comando esteja disponível no seu terminal. Para fazer
isso, siga os passos abaixo:

**a.** Verifique qual shell você está a usar:

```terminal
$ echo $SHELL
```

- Se a saída contiver `/bash`, use `.bashrc`.
- Se a saída contiver `/zsh`, use `.zshrc`.

**b.** Execute o comando apropriado, com base no seu shell.

Para Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Para Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Se você usa um Mac com `zsh`, executar o comando `export PATH` padrão registado
na instalação não persiste depois de fechar o terminal. Em vez disso, adicione o
PATH ao seu ficheiro de configuração do shell executando o seguinte comando:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Os passos acima não se aplicam aos utilizadores de `bash`, pois as alterações
  à variável PATH persistirão por padrão.
</Callout>

</Tab>
</Tabs>

4. Reinicie o seu terminal ou execute o seguinte comando para atualizar a sessão
   do terminal:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Verifique se a instalação foi bem-sucedida verificando a versão do Solana
   CLI:

```terminal
$ solana --version
```

Você verá uma saída como a seguinte:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Você pode visualizar todas as versões disponíveis no
[repositório Github do Agave](https://github.com/anza-xyz/agave/releases). Agave
é o cliente validator da [Anza](https://www.anza.xyz/), anteriormente conhecido
como cliente validator do Solana Labs.

</Callout>

5. Atualize o Solana CLI para a versão mais recente, conforme necessário
   _(Opcional)_

```terminal
$ agave-install update
```

</Step>

### Instalar Anchor CLI

[Anchor](https://www.anchor-lang.com/) é uma framework para desenvolver
programas Solana. A framework Anchor utiliza macros Rust para simplificar o
processo de escrever programas Solana.

#### Pré-requisitos

O ficheiro de teste padrão do projeto Anchor (TypeScript) criado com o comando
_shell`anchor init`_ requer Node.js e Yarn. (O modelo de teste Rust está
disponível usando _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Instalação do Node">

1. Instale o Node usando o
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) executando o
   seguinte comando:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Reinicie o seu terminal e confirme que o comando `nvm` é executado com
   sucesso:

```terminal
$ command -v nvm
```

3. Use `nvm` para instalar o Node:

```terminal
$ nvm install node
```

4. Verifique se a instalação foi bem-sucedida, verificando a versão do Node:

```terminal
$ node --version
```

Você verá uma saída como a seguinte:

```
v23.7.0
```

</Accordion>
<Accordion title="Instalação do Yarn">

1. Instale o Yarn:

```terminal
$ npm install --global yarn
```

2. Verifique se a instalação foi bem-sucedida, verificando a versão do Yarn:

```terminal
$ yarn --version
```

Você verá a seguinte saída:

```
1.22.1
```

</Accordion>
</Accordions>

#### Instalação

Você pode instalar a CLI e as ferramentas do Anchor de duas formas:

- Anchor Version Manager (AVM) — Método de instalação recomendado
- Sem AVM — Instalar diretamente do GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

O Anchor Version Manager (AVM) permite instalar e gerir diferentes versões do
Anchor no seu sistema e atualizar facilmente as versões do Anchor no futuro.
Para instalar o Anchor usando o AVM, siga os passos abaixo:

1. Instale o AVM com o seguinte comando:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Confirme que o AVM foi instalado com sucesso:

```terminal
$ avm --version
```

3. Instale o Anchor CLI usando o AVM:

Para instalar a versão mais recente:

```terminal
$ avm install latest
$ avm use latest
```

Para instalar uma versão específica, especifique o número da versão:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Ao instalar o Anchor CLI no Linux ou WSL, poderá encontrar este erro:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Se vir esta mensagem de erro, siga estes passos:

1. Instale as dependências listadas na secção
   [Linux](/docs/intro/installation#install-dependencies) no topo desta página.
2. Tente novamente instalar o Anchor CLI.

</Callout>

4. Verifique se a instalação foi bem-sucedida, verificando a versão do Anchor
   CLI:

```terminal
$ anchor --version
```

Você verá uma saída como a seguinte:

```
anchor-cli 0.31.1
```

<Callout type="info">

Não se esqueça de executar o comando _shell`avm use`_ para declarar a versão do
Anchor CLI a executar no seu sistema.

- Se instalou a versão `latest`, execute _shell`avm use latest`_.
- Se instalou a versão `0.30.1`, execute _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Instale uma versão específica do Anchor CLI com o seguinte comando:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Verifique se a instalação foi bem-sucedida, verificando a versão do Anchor
   CLI:

```terminal
$ anchor --version
```

Você verá uma saída como a seguinte:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>

### Instalar Surfpool CLI

[Surfpool](https://surfpool.run/) é uma ferramenta para desenvolvimento local e
uma substituição aprimorada para `solana-test-validator`. Saiba mais sobre os
recursos do Surfpool na [documentação do Surfpool](https://docs.surfpool.run/).

1. Instale o Surfpool com o seguinte comando:

```terminal
$ curl -sL https://run.surfpool.run/ | bash
```

2. Verifique se a instalação foi bem-sucedida verificando a versão do Surfpool
   CLI:

```terminal
$ surfpool --version
```

Você verá uma saída como a seguinte:

```
surfpool 0.12.0
```

</Step>
