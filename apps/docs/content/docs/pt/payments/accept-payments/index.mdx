Aceite pagamentos na Solana usando componentes prontos, o protocolo Solana Pay
ou integração direta via RPC.

## Convenção de endereço de recebimento

Na Solana, **partilhe o seu endereço de carteira, não a sua token account**. É
típico o remetente derivar a
[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)
apropriada para o token que está a enviar (consulte
[Enviar pagamentos](/docs/payments/send-payments) para mais detalhes).

**Exemplo:** Partilhe `7EcDhS...` (a sua carteira), não `9xKpQ...` (a sua token
account de USDC).

<Callout>
  Para ponto de venda e comércio eletrónico, consulte [Solana
  Pay](/docs/payments/accept-payments/solana-pay) para códigos QR e URLs de
  pagamento, ou [Payment Button](/docs/payments/accept-payments/payment-button)
  para componentes React prontos a usar.
</Callout>

## Opções de integração

| Abordagem                                                       | Ideal para                            | Complexidade |
| --------------------------------------------------------------- | ------------------------------------- | ------------ |
| [Payment Button](/docs/payments/accept-payments/payment-button) | Aplicações React, integração rápida   | Baixa        |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | Códigos QR, links de pagamento, PDV   | Média        |
| [RPC direto](/docs/payments/interacting-with-solana)            | Fluxos personalizados, controlo total | Mais elevada |

## Verificação

Após receber um pagamento, verifique se foi concluído usando
[Ferramentas de verificação](/docs/payments/accept-payments/verification-tools).
Para aplicações de alto volume, consulte
[Indexação](/docs/payments/accept-payments/indexing) para monitorização de
transações de nível de produção.
