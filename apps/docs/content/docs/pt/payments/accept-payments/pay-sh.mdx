---
title: Com Pay.sh
description: Aceite pagamentos com Pay.sh.
---

Pode aceitar pagamentos em pedidos de API através do Pay.sh configurando um
servidor Pay. Instale o Pay com:

```bash
brew install pay
```

ou

```bash
npm install -g @solana/pay
```

Verifique a instalação com:

```bash
pay --version
```

O toolchain irá configurar o seu endereço automaticamente. Execute o seguinte
para configurar e recarregar a sua carteira:

```bash
pay setup
```

## Configurar o servidor de demonstração

Pode começar a aceitar pagamentos iniciando o seu servidor. Pode ver um servidor
de demonstração em funcionamento com:

```bash
pay server demo
```

Isto faz duas coisas. Inicia um servidor Pay em tempo real que pode ser
consultado via pedido HTTP, e gera um ficheiro `pay-demo.yml` no seu diretório
atual.

Todas as rotas da API são configuradas através do ficheiro de configuração YAML
`endpoints`. Para aceitar pagamentos, tem de adicionar um atributo `metering:`.
A rota `/api/v1/reports/usage` foi configurada como exemplo:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Pode testar se os pagamentos funcionam executando o seguinte comando como
cliente noutro terminal:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Pode saber mais sobre como definir preços na
[documentação Pay.sh Building with Pay > Pricing](https://pay.sh/docs/building-with-pay/pricing)

Saiba mais sobre como configurar o Pay.sh em produção na
[documentação de Configuração do Pay.sh](https://pay.sh/docs/toolchain/configuration).
