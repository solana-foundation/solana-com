# LiteSVM cho TypeScript

Các bộ kiểm thử Solana khởi động `solana-test-validator` mất vài giây mỗi lần
kiểm thử — bạn đang chờ khởi động tiến trình, kết nối RPC và các vòng đi-về qua
mạng. LiteSVM chạy các kiểm thử tương tự trong mili giây. Không cần tiến trình
bên ngoài, không cần mạng, tất cả đều trong bộ nhớ bên trong trình chạy kiểm thử
của bạn.

<Callout type="info">
  **Đã kiểm thử với:** `@solana/kit@6.8.0`, `@solana/kit-plugin-litesvm@0.10.0`,
  `@solana/kit-plugin-signer@0.10.0`, `@solana-program/system@0.12.0`,
  `@solana-program/token@0.13.0`. Các gói plugin này còn mới và đang phát triển
  nhanh chóng — nếu bạn gặp sự không tương thích, hãy kiểm tra
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  để biết chính xác các phiên bản mà tài liệu và bộ kiểm thử đang sử dụng.
</Callout>

Bạn lắp ráp một Kit client từ `@solana/kit` bằng cách cài đặt các plugin với
`.use()`: một plugin ký
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
plugin LiteSVM
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm)),
và bất kỳ plugin chương trình nào bạn cần. Kết quả sẽ hiển thị
`client.sendTransaction()`, `client.rpc`, và truy cập trực tiếp vào trạng thái
tài khoản, đồng hồ và sysvars — tất cả mà không cần rời khỏi tiến trình của bạn.

### Tạo Client Của Bạn

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` trả về một shell rỗng — mỗi `.use(plugin)` bổ sung các khả
  năng vào đó. Plugin `signer()` phải đứng **trước** `litesvm()` vì plugin
  transport cần một payer để gắn vào. `signer(mySigner)` là cách viết tắt để cài
  đặt cùng một keypair vừa làm `payer` vừa làm `identity` — dùng `payer()` hoặc
  `identity()` từ cùng gói đó nếu bạn muốn tách biệt hai vai trò này. Thứ tự này
  cũng áp dụng tương tự cho `solanaRpc()` khi bạn đang dùng mạng thực.
</Callout>

<Callout type="info">
  Muốn thiết lập nhanh gọn trong một dòng cho các thử nghiệm nhanh? Bundle
  `litesvm()` đã bao gồm airdrop, vì vậy bạn có thể nối chuỗi
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` từ
  `@solana/kit-plugin-signer` để tự động tạo và nạp tiền cho payer.
</Callout>

</Step>

</Steps>

## Tìm Hiểu Về Client

Sau `.use(signer(...)).use(litesvm())`, client cung cấp:

| Thuộc tính                  | Mô tả                                                                                                                                                                                                          |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | `TransactionSigner` mà bạn đã cài đặt với plugin `signer()` (cũng được đặt là `client.identity`)                                                                                                               |
| `client.identity`           | Ví ký (wallet signer) sở hữu các tài khoản, token và quyền hạn                                                                                                                                                 |
| `client.svm`                | Instance [`LiteSVM`](https://www.npmjs.com/package/litesvm) trực tiếp để quản lý tài khoản, giao dịch và cấu hình                                                                                              |
| `client.rpc`                | Giao diện RPC tương thích với Kit (tập con: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Lên kế hoạch, ký và gửi các instruction dưới dạng giao dịch                                                                                                                                                    |
| `client.airdrop`            | Yêu cầu SOL từ faucet của LiteSVM                                                                                                                                                                              |
| `client.getMinimumBalance`  | Tính toán số lamport tối thiểu để miễn rent                                                                                                                                                                    |

### Sử Dụng client.svm

Truy cập trực tiếp vào LiteSVM để:

- Quản lý tài khoản (`setAccount`, `getAccount`, `airdrop`)
- Thực thi giao dịch (`sendTransaction`, `simulateTransaction`)
- Cấu hình (`withSigverify`, `withBlockhashCheck`, v.v.)
- Thao tác thời gian (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### Sử Dụng client.rpc

Giao diện RPC tương thích với Kit cho các thao tác tiêu chuẩn:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  Lớp RPC chỉ hỗ trợ **mã hóa base64**. Yêu cầu các mã hóa khác sẽ gây ra lỗi.
</Callout>

## Cấu Hình

Cấu hình LiteSVM bằng các phương thức builder trên `client.svm`:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## Gửi Giao Dịch

Thêm các plugin chương trình sau transport — chúng gắn các instruction builder
vào client để bạn không bao giờ phải thao tác trực tiếp với `TransactionMessage`
thô:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## Luồng SVM → RPC

Bạn có thể thiết lập trạng thái qua SVM và đọc nó qua RPC:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```

## Tiếp Theo

Phần này đề cập đến client cốt lõi — tạo tài khoản, gửi instruction và đọc trạng
thái. Nếu các bài kiểm thử của bạn liên quan đến SPL token (mint, ATA, chuyển
token), [hướng dẫn kiểm thử token](/docs/tools/litesvm/typescript/tokens) sẽ chỉ
ra cách plugin `tokenProgram()` rút gọn ~40 dòng thiết lập xuống còn 3 lần gọi
hàm. Để xem tài liệu API đầy đủ, hãy tham khảo
[tài liệu API](/docs/tools/litesvm/typescript/api-reference).
