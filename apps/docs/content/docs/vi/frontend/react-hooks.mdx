`@solana/react` xuất bản một Kit client lên cây component của bạn thông qua một
provider duy nhất, sau đó các hook đọc dữ liệu RPC, các subscription và trạng
thái ví từ đó. Các hook ví trình duyệt nằm trong
`@solana/kit-plugin-wallet/react` và đọc từ cùng provider đó — không có provider
ví riêng biệt nào.

## Cài đặt

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Xây dựng client

Xây dựng client một lần, ở phạm vi module, bên ngoài cây React. Trong trình
duyệt, `walletSigner` đặt ví đã kết nối làm cả người trả phí lẫn danh tính.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` thêm `client.wallet` (khám phá và kết nối Wallet Standard), và
`solanaRpc` thêm `client.rpc` cùng helper `client.sendTransaction`. Xuất kiểu
client được bọc trong `Awaited<>` để nó vẫn chính xác nếu bạn sau này thêm một
plugin phân giải bất đồng bộ, và truyền `AppClient` đó vào mọi lời gọi
`useClient`.

## Bọc cây của bạn một lần

`ClientProvider` xuất bản client. Mọi hook bên dưới đều đọc từ nó.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Các hook ví

Các hook ví đọc khả năng `wallet` của client, vì vậy chúng chỉ hoạt động bên
trong một `ClientProvider` mà client đã cài đặt plugin ví.

- `useWallets()` — Các ví Wallet Standard được khám phá cho chain của client.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` hoặc `null`. `signer`
  là `null` đối với ví chỉ đọc.
- `useConnect()` / `useDisconnect()` — trả về một `ActionResult` mà `dispatch`
  (và `dispatchAsync`) của nó thực thi hành động và `isRunning`, `status`,
  `data`, `error`, và `reset` theo dõi kết quả của nó.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Gửi một giao dịch

`useClient` trả về cùng client mà bạn đã xuất bản. Xây dựng các instruction với
một program client được tạo sẵn và gửi chúng thông qua `client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Truyền kiểu `AppClient` đã xuất vào `useClient` giúp hook nhận được client đầy
đủ với các plugin mở rộng, do đó `client.rpc` và `client.sendTransaction` đều
được định kiểu.

## Đọc dữ liệu

`@solana/react` cung cấp các hook để đọc dữ liệu thông qua client:

- `useRequest(source)` — đọc RPC một lần; trả về
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — đăng ký theo dõi trực tiếp; trả về
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — giá trị được khởi tạo từ RPC và được cập nhật liên
  tục thông qua subscription.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Hãy ghi nhớ (memoize) nguồn dữ liệu (dùng `useMemo`, hoặc `useCallback` cho hàm)
để hook chỉ tải lại khi đầu vào thay đổi thay vì mỗi lần render.

Để hỗ trợ caching và revalidation, các đường dẫn con `@solana/react/swr` và
`@solana/react/query` bao bọc các hook tương tự cho
[SWR](https://swr.vercel.app) và [TanStack Query](https://tanstack.com/query).

## Các mẫu phổ biến cho nhà phát triển Solana

- **Một provider duy nhất**: `ClientProvider` là provider duy nhất bạn cần; các
  wallet hook đọc thông qua khả năng `wallet` của client.
- **Hỗ trợ Server Components**: Chỉ đánh dấu các component lá gọi hook với
  `"use client"`; các thao tác đọc phía server có thể dùng client RPC Kit thông
  thường mà không cần wallet.
- **Kiểm thử**: Truyền một client giả lập qua `ClientProvider` để mô phỏng
  wallet và các phản hồi RPC.

<Callout type="warn" title="Hai họ hook trùng tên">
  `useSignIn` và `useSignMessage` tồn tại trong cả `@solana/react` (nhận tham số
  `UiWalletAccount`) lẫn `@solana/kit-plugin-wallet/react` (dựa trên client,
  không có tham số account, trả về `ActionResult`). Hãy sử dụng các phiên bản
  `@solana/kit-plugin-wallet/react` với mẫu client ở trên; các wallet hook cũ từ
  `@solana/react` đang dần được thay thế.
</Callout>

Hãy kết hợp hướng dẫn này với [tổng quan về Kit client](/docs/frontend/client)
để hiểu rõ client mà mỗi hook đọc dữ liệu từ đó.
