  Xây dựng một Solana client từ các plugin Kit cho RPC, ký và gửi giao dịch.
---

`@solana/kit` cung cấp `createClient`, một host nhỏ mà bạn mở rộng bằng các
plugin. Mỗi plugin bổ sung một khả năng — kết nối RPC, trình ký, lập kế hoạch
giao dịch — và client được tổng hợp hỗ trợ các script, route server và giao diện
người dùng như nhau.

<Callout type="info">
  Đang xây dựng ứng dụng React? [Hướng dẫn React](/docs/frontend/react-hooks)
  bọc cùng client này với `@solana/react` để các component đọc trạng thái ví và
  dữ liệu RPC thông qua hook. Bắt đầu từ đó và quay lại đây khi bạn cần kiểm
  soát chi tiết hơn.
</Callout>

## Cài đặt

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

Sử dụng bất kỳ trình quản lý gói nào. Kit là v7+ và các gói
`@solana/kit-plugin-*` là 0.13+.

## Tạo một client

Kết hợp các plugin với `.use(...)`. Đặt trình ký trước, sau đó là kết nối RPC.
`solanaDevnetRpc()` thêm `client.rpc`, các subscription, trình lập kế hoạch/thực
thi giao dịch và helper `client.sendTransaction` — vì vậy một ứng dụng hiếm khi
cần tự kết nối các thành phần đó.

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

Plugin `signer` đặt cả người trả phí lẫn danh tính ứng dụng. Gói RPC yêu cầu
người trả phí phải được thiết lập trước, đó là lý do thứ tự quan trọng.

### Các biến thể của trình ký

`@solana/kit-plugin-signer` cung cấp ba vai trò để bạn chỉ cấp quyền những gì
client cần:

- `signer(x)` — đặt cả `client.payer` và `client.identity`.
- `payer(x)` — chỉ người trả phí.
- `identity(x)` — chỉ quyền ứng dụng.

Mỗi cái có các biến thể `*FromFile(path)` (Node), `generated*()`,
`generated*WithSol(amount)` và `airdrop*()` cho các bài kiểm tra và script.

## Gửi một giao dịch

Xây dựng các lệnh với một program client được tạo sẵn, sau đó chuyển chúng cho
`client.sendTransaction`. Trình lập kế hoạch phân giải blockhash, ước tính
compute, ký bằng người trả phí của client và gửi đi.

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

`sendTransaction` phân giải thành một đối tượng kết quả; chữ ký giao dịch nằm
tại `result.context.signature`. Nó khẳng định kế hoạch phù hợp với một giao dịch
duy nhất và ném lỗi nếu không — sử dụng `sendTransactions` khi một lô có thể
trải rộng trên nhiều giao dịch.

## Chọn một gói RPC

`@solana/kit-plugin-rpc` cung cấp các cài đặt sẵn cho cluster:

- `solanaDevnetRpc()` — mặc định devnet, bật airdrop (`client.airdrop`).
- `solanaMainnetRpc()` — mainnet; airdrop là lỗi biên dịch.
- `solanaLocalRpc()` — `http://127.0.0.1:8899` cho validator cục bộ.
- `solanaRpc({ rpcUrl })` — bất kỳ endpoint nào.

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### Kết hợp theo từng phần

Các gói cài đặt sẵn đã bao gồm kết nối RPC, planner, executor và các trình trợ
giúp gửi, vì vậy hầu hết các ứng dụng không cần dùng riêng từng thành phần. Chỉ
chuyển sang các plugin chi tiết khi bạn cần tùy chỉnh một thành phần — ví dụ như
một planner tùy chỉnh hoặc một kênh đăng ký khác. Cách này cũng cần
`@solana/kit-plugin-instruction-plan`, vốn đã được các gói cài đặt sẵn tích hợp
sẵn cho bạn:

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## Thêm các client chương trình

Các gói `@solana-program/*` được tạo tự động cung cấp các plugin client để các
lệnh gọi chương trình được gắn vào client. Cài đặt chương trình bạn cần và
`.use()` plugin của nó:

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="Phạm vi phụ thuộc ngang hàng">
  Một số bản phát hành `@solana-program/*` vẫn khai báo peer `@solana/kit ^6.x`
  trong khi Kit đang ở phiên bản 7.x. Chúng hoạt động với Kit 7 — hãy ghi đè
  phạm vi peer trong trình quản lý gói của bạn nếu có cảnh báo.
</Callout>

## Ký giao dịch không cần ví trình duyệt

Đang chạy headless trong một API route, worker hoặc script? Hãy ký bằng dịch vụ
quản lý khóa backend thay vì dùng keypair thô. Xem
[Keychain](/docs/tools/keychain) và
[Ký trong Môi trường Production](/docs/core/transactions/signing-in-production).

## Các mẫu phổ biến dành cho lập trình viên Solana

- **Thiết lập dùng chung**: Xây dựng client một lần trong một module cốt lõi và
  nhập nó vào các script, server route và giao diện người dùng.
- **Đọc phía server**: Một client chỉ với `signer` + `solanaRpc` (không có
  plugin ví) có thể đọc tài khoản và mô phỏng giao dịch từ phía server.
- **Kiểm thử cục bộ**: Thay gói RPC bằng `.use(litesvm())` từ
  `@solana/kit-plugin-litesvm` để chạy với validator trong bộ nhớ (chỉ dành cho
  Node).
- **Khả năng kiểm thử**: Client được kết hợp rất dễ mock — stub các phản hồi RPC
  hoặc signer mà không cần ví trình duyệt.
