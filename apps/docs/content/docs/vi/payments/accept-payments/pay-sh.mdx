---
title: Với Pay.sh
description: Chấp nhận thanh toán với Pay.sh.
---

Bạn có thể chấp nhận thanh toán cho các yêu cầu API thông qua Pay.sh bằng cách
thiết lập một máy chủ Pay. Cài đặt Pay với:

```bash
brew install pay
```

hoặc

```bash
npm install -g @solana/pay
```

Xác minh cài đặt với:

```bash
pay --version
```

Bộ công cụ sẽ thiết lập địa chỉ của bạn. Chạy lệnh sau để thiết lập và nạp tiền
vào ví của bạn:

```bash
pay setup
```

## Thiết lập máy chủ demo

Bạn có thể bắt đầu chấp nhận thanh toán bằng cách khởi động máy chủ của mình.
Bạn có thể xem máy chủ demo hoạt động với:

```bash
pay server demo
```

Thao tác này thực hiện hai việc. Nó khởi động một máy chủ Pay trực tiếp mà bạn
có thể truy vấn qua yêu cầu HTTP, và tạo ra một tệp `pay-demo.yml` trong thư mục
hiện tại của bạn.

Tất cả các tuyến API được cấu hình thông qua tệp cấu hình YAML `endpoints`. Để
chấp nhận thanh toán, bạn phải thêm thuộc tính `metering:`. Tuyến
`/api/v1/reports/usage` đã được thực hiện như một ví dụ cho bạn:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Bạn có thể kiểm tra xem việc thanh toán có hoạt động không bằng cách chạy lệnh
sau trên một terminal khác với tư cách là client:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Bạn có thể tìm hiểu thêm về cách xác định giá trong
[Tài liệu Pay.sh Building with Pay > Pricing](https://pay.sh/docs/building-with-pay/pricing)

Tìm hiểu thêm về cách thiết lập Pay.sh trong môi trường production trong
[Tài liệu cấu hình Pay.sh](https://pay.sh/docs/toolchain/configuration).
