Cette section fournit quelques commandes et exemples courants pour vous aider à
démarrer avec
l'[interface en ligne de commande Anchor](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Initialiser le projet

Créez un nouveau projet Anchor en exécutant la commande indiquée ci-dessous.
Elle créera un nouveau répertoire avec le nom du projet et l'utilisera pour
initialiser un nouveau projet Anchor.

```terminal
$ anchor init <project-name>
```

Par exemple, la commande ci-dessous créera un projet appelé `my-project`. Le
répertoire `my-project` contiendra un programme Rust de base et un modèle de
test TypeScript.

```terminal
$ anchor init my-project
```

Ensuite, naviguez vers le répertoire du projet :

```terminal
$ cd <project-name>
```

Consultez la
[structure des fichiers du projet](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
d'Anchor.

</Step>
<Step>

### Compiler le programme

Pour compiler votre projet, exécutez la commande suivante :

```terminal
$ anchor build
```

Vous trouverez le programme compilé dans le répertoire `/target/deploy`.

<Callout type="warn">

Lors de l'exécution de `anchor build`, si vous rencontrez les erreurs suivantes
:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Essayez ces solutions :

1. Forcer l'installation en utilisant la commande suivante :

```terminal
$ cargo build-sbf --force-tools-install
```

2. Si la solution précédente ne fonctionne pas, effacez le cache Solana :

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Vous pouvez corriger cela en modifiant le champ version du fichier `Cargo.lock` de 4 à 3 :

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Consultez [ce problème](https://github.com/solana-foundation/anchor/issues/3392)
pour plus d'informations.

</Accordion>

</Accordions>

Après avoir appliqué la solution précédente, essayez d'exécuter à nouveau
_shell`anchor build`_.

</Callout>

</Step>
<Step>

### Déployer le programme

Pour déployer votre projet, exécutez la commande suivante :

```terminal
$ anchor deploy
```

Cette commande déploie votre programme sur le `cluster` spécifié dans le fichier
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Tester le programme

Pour tester votre projet, exécutez la commande suivante :

```terminal
$ anchor test
```

Cette commande compile, déploie et exécute les tests de votre projet.

<Callout>
  Lors de l'utilisation du cluster `localnet`, Anchor démarre automatiquement un
  validateur local, déploie le programme, exécute les tests, puis arrête le
  validateur.
</Callout>

<Callout type="warn">

L'une ou l'autre des erreurs suivantes peut indiquer que Node.js ou Yarn n'est
pas installé :

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
