_Disponible sur GitHub :
[`v2.2.0-beta.7`](https://github.com/solana-foundation/kora/releases/tag/v2.2.0-beta.7)_

Kora v2.2.0-beta introduit la prise en charge des bundles Jito, de nouvelles
méthodes RPC, des images Docker, des fonctionnalités de sécurité améliorées et
des améliorations du SDK.

> **Ceci est une pré-version.** Les fonctionnalités documentées ici sont
> disponibles dans `v2.2.0-beta.7` et peuvent changer avant la version stable.
> Pour la documentation stable, consultez la
> [documentation principale de Kora](/docs/tools/kora).

## Points clés

### Prise en charge des bundles Jito

Soumettez jusqu'à 5 transactions de manière atomique via le moteur de blocs de
Jito. Nouvelles méthodes RPC :

- [`signBundle`](/docs/tools/kora/beta/bundle-methods#signbundle) — Signer un
  bundle sans le diffuser
- [`signAndSendBundle`](/docs/tools/kora/beta/bundle-methods#signandsendbundle)
  — Signer et soumettre à Jito
- [`estimateBundleFee`](/docs/tools/kora/beta/bundle-methods#estimatebundlefee)
  — Estimer les frais totaux du bundle

Consultez le [Guide des bundles Jito](/docs/tools/kora/beta/jito-guide) pour une
procédure complète.

### Images Docker

Images officielles multi-plateformes (amd64/arm64) publiées sur GitHub Container
Registry. Voir [Installation](/docs/tools/kora/beta/installation).

### Plugins de transaction

Un nouveau système de plugins permet à Kora de valider les structures de
transaction pendant les flux de signature. Le premier plugin intégré,
`gas_swap`, applique des règles strictes pour les transactions d'échange de
jetons contre SOL sans frais de gas. Les plugins sont configurés dans
`kora.toml` et s'exécutent automatiquement pendant `signTransaction`,
`signAndSendTransaction`, `signBundle`, et `signAndSendBundle`.

Voir
[Configuration — Plugins de transaction](/docs/tools/kora/beta/configuration#transaction-plugins).

### Nouvelles options de configuration

- **Plugins de transaction** pour la validation des structures de transaction
  (`gas_swap`)
- **reCAPTCHA v3** protection anti-bots pour les points de terminaison de
  signature
- **Lighthouse** protection contre le drainage du payeur de frais
- **Limites d'utilisation** limitation des transactions par portefeuille
- **Configuration des bundles** pour l'intégration Jito

Voir [Configuration](/docs/tools/kora/beta/configuration).

### Modifications du SDK

- Prise en charge des méthodes de bundle (`signBundle`, `signAndSendBundle`,
  `estimateBundleFee`)

Voir [Modifications du SDK](/docs/tools/kora/beta/sdk-changes).

### Méthodes RPC supplémentaires

- [`getVersion`](/docs/tools/kora/beta/bundle-methods#getversion) — Récupérer
  les informations de version du serveur

## Installation

```bash
cargo install kora-cli@2.2.0-beta.7
```

Ou via Docker :

```bash
docker pull ghcr.io/solana-foundation/kora:v2.2.0-beta.7
```
