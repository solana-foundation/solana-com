**Solana Keychain fournit une interface unifiée pour signer des transactions
Solana sur plusieurs backends de gestion de clés.** Développez localement avec
une clé en mémoire, puis basculez vers une gestion de clés d'entreprise, des HSM
cloud ou des services de portefeuille gérés en production — sans réécrire votre
code de signature.

### Pourquoi Keychain ?

- **Interface unique** : Un trait `SolanaSigner` fonctionne sur tous les
  backends
- **Changement de backend** : Changez de fournisseur de gestion de clés via la
  configuration, pas le code
- **Abstraction sans coût** : Les indicateurs de fonctionnalités n'incluent que
  ce dont vous avez besoin
- **Audité** : Revue de sécurité par [Accretion](https://accretion.xyz)

### Architecture

- **Langages** : Rust + TypeScript, à parité complète
- **Trait** : Interface `SolanaSigner` unifiée
- **Backends** : Memory, Vault, AWS KMS, GCP KMS, Privy, Turnkey, Fireblocks,
  CDP, Crossmint, Dfns, Openfort, Para, Utila
- **Compatibilité** : Compatible `@solana/kit` et `@solana/signers` (TypeScript)
  | Compatible `solana-sdk` et `solana-sdk-v3` (Rust)

### Backends pris en charge

Les backends couvrent le développement local, la gestion de clés auto-hébergée
et cloud, ainsi que les services de portefeuille gérés ou MPC. Consultez
[Choisir un backend](/docs/tools/keychain/choosing-a-backend) pour comparer les
modèles de garde et choisir le plus adapté.

| Backend         | Cas d'usage                         | Rust | TypeScript |
| --------------- | ----------------------------------- | ---- | ---------- |
| Memory          | Développement, tests                | ✓    | ✓          |
| HashiCorp Vault | Gestion de clés auto-hébergée       | ✓    | ✓          |
| AWS KMS         | Cloud natif (AWS)                   | ✓    | ✓          |
| GCP KMS         | Cloud natif (GCP)                   | ✓    | ✓          |
| Privy           | Portefeuilles intégrés              | ✓    | ✓          |
| Turnkey         | Gestion de clés non-custodiale      | ✓    | ✓          |
| Fireblocks      | Garde institutionnelle              | ✓    | ✓          |
| CDP             | Coinbase Developer Platform         | ✓    | ✓          |

| Dfns            | Infrastructure de portefeuille Dfns | ✓    | ✓          |

| Para            | Portefeuilles MPC Para              | ✓    | ✓          |

## Démarrage rapide

### Rust

Installez le crate Rust :

```bash
cargo add solana-keychain
```

Utilisation de base :

```rust
use solana_keychain::{Signer, SolanaSigner};

// Create a signer from any backend
let signer = Signer::from_memory("base58_private_key")?;

// All signers share the same interface
let pubkey = signer.pubkey();
let signature = signer.sign_transaction(&mut tx).await?;
```

- [Guide Rust](/docs/tools/keychain/getting-started/rust) - Installation
  complète et configuration du backend
- [Crates.io](https://crates.io/crates/solana-keychain) - Crate Rust

### TypeScript

Installez le package TypeScript :

```bash
pnpm add @solana/keychain
```

Utilisation de base :

```typescript
import { createKeychainSigner } from "@solana/keychain";
import { signTransactionWithSigners } from "@solana/signers";

// Create any signer via the unified factory
const signer = await createKeychainSigner({
  backend: "vault",
  vaultAddr: "https://vault.example.com:8200",
  vaultToken: "hvs.xxxxx",
  keyName: "my-solana-key",
  publicKey: "base58_public_key"
});

// Sign an already-compiled transaction
const signedTx = await signTransactionWithSigners(
  [signer],
  compiledTransaction
);
```

- [Guide TypeScript](/docs/tools/keychain/getting-started/typescript) - Packages
  `@solana/keychain`
- [Package npm](https://www.npmjs.com/package/@solana/keychain) - Package npm
