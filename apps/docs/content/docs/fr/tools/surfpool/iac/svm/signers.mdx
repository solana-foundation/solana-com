---
title: Signataires SVM
description: Signataires pour Solana et les blockchains compatibles SVM
---

Ces signataires sont disponibles lors de l'utilisation de l'addon SVM pour la
signature de transactions.

## secret_key

Le signataire `svm::secret_key` peut être utilisé pour signer une transaction de
manière synchrone à l'aide d'une clé secrète, d'un mnémonique ou d'un fichier
keypair.

### Entrées

| Name              | Required | Type   | Description                                                          |
| ----------------- | -------- | ------ | -------------------------------------------------------------------- |
| `secret_key`      | optional | string | La clé secrète utilisée pour signer les messages et les transactions |
| `mnemonic`        | optional | string | La phrase mnémonique utilisée pour générer la clé secrète            |
| `derivation_path` | optional | string | Le chemin de dérivation utilisé pour générer la clé secrète          |
| `keypair_json`    | optional | string | Un chemin vers un fichier keypair.json contenant la clé secrète      |

### Sorties

| Name         | Type   | Description                           |
| ------------ | ------ | ------------------------------------- |
| `public_key` | string | La clé publique du compte             |
| `address`    | string | L'adresse SVM (alias pour public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Vous pouvez également utiliser un mnémonique :

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Ou un fichier keypair :

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

Le signataire `svm::web_wallet` permettra à un opérateur de Runbook de signer la
transaction avec le signataire de navigateur de son choix.

### Entrées

| Name               | Required | Type   | Description                                                                              |
| ------------------ | -------- | ------ | ---------------------------------------------------------------------------------------- |
| `expected_address` | optional | string | L'adresse SVM dont la connexion est attendue (omettre pour autoriser n'importe laquelle) |

### Sorties

| Name         | Type   | Description                        |
| ------------ | ------ | ---------------------------------- |
| `address`    | string | L'adresse du compte connecté       |
| `public_key` | string | La clé publique du compte connecté |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Ce signataire invitera l'utilisateur dans l'interface Web à :

1. Connecter son portefeuille
2. Fournir une pubkey via la signature d'un message
3. Signer toutes les transactions nécessitant ce signataire

## squads

Le signataire `svm::squads` peut être utilisé pour signer des transactions avec
un coffre-fort multisig Squads.

### Entrées

| Nom       | Requis    | Type   | Description                    |
| --------- | --------- | ------ | ------------------------------ |
| `address` | optionnel | string | L'adresse du coffre-fort Squad |

### Sorties

| Nom          | Type   | Description                           |
| ------------ | ------ | ------------------------------------- |
| `public_key` | string | La pubkey du coffre-fort Squad        |
| `address`    | string | L'adresse SVM (alias pour public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

L'utilisation du multisig Squads permet une signature de transactions sécurisée
et multi-parties sans exposer les clés privées individuelles.
