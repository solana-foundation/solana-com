## Test Complet

```rust title="tests/pda_test.rs"
use litesvm::LiteSVM;
use solana_account::Account;
use solana_sdk::pubkey::Pubkey;

#[test]
fn test_pda_creation() {
    let mut svm = LiteSVM::new();

    // Program that owns the PDA
    let program_id = Pubkey::new_unique();

    // Derive PDA address
    let seed = b"my_pda";
    let (pda, bump) = Pubkey::find_program_address(
        &[seed],
        &program_id
    );

    println!("PDA: {}", pda);
    println!("Bump: {}", bump);

    // Create PDA account manually (for testing)
    svm.set_account(pda, Account {
        lamports: 1_000_000,
        data: vec![bump], // Store bump seed in data
        owner: program_id,
        executable: false,
        rent_epoch: 0,
    }).unwrap();

    // Verify PDA was created
    let account = svm.get_account(&pda).unwrap();
    assert_eq!(account.owner, program_id);
    assert_eq!(account.data[0], bump);

    println!("PDA created successfully!");
}
```

## Points Clés

1. **Dérivation de PDA** : Utilisez `Pubkey::find_program_address()` pour
   dériver les adresses PDA
2. **Seeds** : Les PDAs sont dérivés à partir de seeds et de l'ID du programme
3. **Bump Seed** : Le bump seed garantit que l'adresse est en dehors de la
   courbe ed25519
4. **Création de compte** : Utilisez `set_account()` pour créer manuellement des
   comptes lors des tests
5. **Propriété** : Les PDAs doivent appartenir au programme qui les a dérivés
