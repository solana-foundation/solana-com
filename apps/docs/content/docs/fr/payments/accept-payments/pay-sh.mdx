---

Vous pouvez accepter des paiements sur des requêtes API via Pay.sh en
configurant un serveur Pay. Installez Pay avec :

```bash
brew install pay
```

ou

```bash
npm install -g @solana/pay
```

Vérifiez l'installation avec :

```bash
pay --version
```

La chaîne d'outils configurera votre adresse automatiquement. Exécutez la
commande suivante pour configurer et recharger votre portefeuille :

```bash
pay setup
```

## Configuration du serveur de démonstration

Vous pouvez commencer à accepter des paiements en démarrant votre serveur. Vous
pouvez voir un serveur de démonstration en action avec :

```bash
pay server demo
```

Cela fait deux choses. Il démarre un serveur Pay en direct que vous pouvez
interroger via des requêtes HTTP, et il génère un fichier `pay-demo.yml` dans
votre répertoire courant.

Toutes les routes API sont configurées via la configuration YAML `endpoints`.
Pour accepter des paiements, vous devez ajouter un attribut `metering:`. La
route `/api/v1/reports/usage` a été réalisée comme exemple pour vous :

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Vous pouvez tester le bon fonctionnement des paiements en exécutant la commande
suivante en tant que client dans un autre terminal :

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Vous pouvez en savoir plus sur la définition des tarifs dans la
[documentation Pay.sh - Construire avec Pay > Tarification](https://pay.sh/docs/building-with-pay/pricing)

Apprenez-en davantage sur la configuration de Pay.sh en production dans la
[documentation de configuration Pay.sh](https://pay.sh/docs/toolchain/configuration).
