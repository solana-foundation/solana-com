Acceptez des paiements sur Solana en utilisant des composants prêts à l'emploi,
le protocole Solana Pay, ou une intégration RPC directe.

## Convention d'adresse de réception

Sur Solana, **partagez votre adresse de portefeuille, pas votre token account**.
Il est courant que l'expéditeur dérive le
[associated token account (ATA)](/docs/payments/how-payments-work#token-accounts)
approprié pour le token qu'il envoie (voir
[Envoyer des paiements](/docs/payments/send-payments) pour plus de détails).

**Exemple :** Partagez `7EcDhS...` (votre portefeuille), pas `9xKpQ...` (votre
token account USDC).

<Callout>
  Pour les points de vente et le commerce électronique, consultez [Solana
  Pay](/docs/payments/accept-payments/solana-pay) pour les codes QR et les URL
  de paiement, ou [Payment
  Button](/docs/payments/accept-payments/payment-button) pour des composants
  React prêts à l'emploi.
</Callout>

## Options d'intégration

| Approche                                                        | Idéal pour                             | Complexité |
| --------------------------------------------------------------- | -------------------------------------- | ---------- |
| [Payment Button](/docs/payments/accept-payments/payment-button) | Applications React, intégration rapide | Faible     |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | Codes QR, liens de paiement, PDV       | Moyenne    |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Flux personnalisés, contrôle total     | Élevée     |

## Vérification

Après avoir reçu un paiement, vérifiez qu'il est bien arrivé en utilisant les
[outils de vérification](/docs/payments/accept-payments/verification-tools).
Pour les applications à fort volume, consultez
[Indexation](/docs/payments/accept-payments/indexing) pour une surveillance des
transactions de niveau production.
