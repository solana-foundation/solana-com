Sept concepts constituent les fondements du développement Solana. Lisez Comptes,
Programmes, Instructions, Transactions et Frais dans cet ordre, car chacun
s'appuie sur le précédent. Lisez Comptes et Programmes avant les PDA. Lisez
Programmes, Instructions et PDA avant les CPI.

<Cards>
  <Card title="Comptes" href="/docs/core/accounts">
    Solana stocke tous les états dans des comptes, un magasin clé-valeur
    (key-value) indexé par des adresses de 32 octets. Chaque compte possède cinq
    champs : lamports, data, owner, executable et rent_epoch.
  </Card>
  <Card title="Programmes" href="/docs/core/programs">
    Les smart contracts de Solana, compilés en sBPF. Les programmes sont sans
    état et tous les états mutables résident dans des comptes de données séparés
    transmis via des instructions.
  </Card>
  <Card title="Instructions" href="/docs/core/instructions">
    Une requête pour exécuter un programme, contenant un ID de programme, une
    liste de comptes et des octets de données opaques. Plusieurs instructions se
    composent en une seule transaction.
  </Card>
  <Card title="Transactions" href="/docs/core/transactions">
    L'unité atomique d'exécution sur Solana. Contient des signatures et un
    message (adresses de comptes, blockhash récent et instructions compilées).
    Maximum 1232 octets.
  </Card>
  <Card title="Frais" href="/docs/core/fees">
    Frais de base de 5 000 lamports par signature, plus des frais de priorité
    optionnels basés sur les unités de calcul demandées. Maximum 1,4 M d'unités
    de calcul par transaction.
  </Card>
  <Card title="Adresses dérivées de programmes (PDA)" href="/docs/core/pda">
    Adresses déterministes dérivées d'un ID de programme et de seeds. Seul le
    programme propriétaire peut signer au nom d'un PDA via invoke_signed.
  </Card>
  <Card title="Invocations inter-programmes (CPI)" href="/docs/core/cpi">
    Un programme appelant un autre via invoke ou invoke_signed. Les privilèges
    de signataire et d'écriture s'étendent à l'appelé. Profondeur maximale de
    pile d'instructions de 5 (niveau supérieur plus CPI).
  </Card>
</Cards>
