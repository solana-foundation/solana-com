<Callout type="info" title="Résumé">
  Vérifications du runtime après chaque instruction : seul le propriétaire peut
  débiter des lamports ou modifier les données, les données peuvent croître de
  10 Kio maximum par instruction, les changements de propriétaire nécessitent
  des données initialisées à zéro, et le flag exécutable est irréversible.
</Callout>

Le runtime Solana applique ces règles après l'exécution de chaque instruction
via les méthodes
[`BorrowedInstructionAccount`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L722-L727).
Chaque règle est vérifiée au moment de la modification, et la transaction est
annulée si une vérification échoue.

## Règles des lamports

| Règle                                                                    | Application                                                                                                                                                                                                                                                                  | Erreur                        |
| ------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Seul le propriétaire peut débiter des lamports                           | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L785) : vérifie `is_owned_by_current_program()` quand `lamports < current`                                                                                              | `ExternalAccountLamportSpend` |
| Les comptes en lecture seule ne peuvent pas voir leurs lamports modifiés | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L788-L789) : vérifie `is_writable()`                                                                                                                                         | `ReadonlyLamportChange`       |
| Tout programme peut créditer des lamports sur un compte modifiable       | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L789) : la vérification de propriété s'applique uniquement lorsque le nouveau solde est inférieur au solde actuel ; la vérification de modification s'applique toujours | `ReadonlyLamportChange`       |
| Les lamports doivent être équilibrés dans une instruction                | [`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L366-L398) : vérifie `get_lamports_delta() == 0`                                                                                                                  | `UnbalancedInstruction`       |

## Transitions d'état de rent

Les comptes existent dans l'une des trois valeurs
[`RentState`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L19-L30)
: _rs`Uninitialized`_ (0 lamport), _rs`RentPaying`_ (au-dessus de 0 mais en
dessous du minimum d'exemption de rent), et _rs`RentExempt`_ (au minimum ou
au-dessus). Les transitions interdites produisent
_rs`TransactionError::InsufficientFundsForRent`_.

Le runtime applique ces règles via
[`transition_allowed()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L107-L126)
:

- Tout compte peut passer à _rs`Uninitialized`_ (fermer) ou _rs`RentExempt`_.
- Aucun compte ne peut entrer dans _rs`RentPaying`_ depuis _rs`Uninitialized`_
  ou _rs`RentExempt`_. Tous les nouveaux comptes doivent être exemptés de rent.

## Règles d'emprunt de compte

Pendant l'exécution des instructions, le runtime applique une sémantique
d'emprunt à écrivain unique sur les comptes. Un programme peut obtenir soit une
référence mutable, soit plusieurs références immuables à un compte, mais pas les
deux simultanément. Si un programme tente d'emprunter un compte qui est déjà
emprunté de manière mutable (ou d'emprunter de manière mutable un compte qui est
déjà emprunté de manière immuable), l'instruction échoue avec
_rs`AccountBorrowFailed`_ via
[`try_borrow()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L498-L508)
et
[`try_borrow_mut()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L512-L520).
Si une instruction se termine alors qu'un emprunt est toujours en cours, le
runtime renvoie _rs`AccountBorrowOutstanding`_ dans
[`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L374-L385).
