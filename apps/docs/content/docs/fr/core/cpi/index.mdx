Un Cross Program Invocation (CPI) se produit lorsqu'un
[programme](/docs/core/programs) appelle une
[instruction](/docs/core/instructions) sur un autre programme pendant
l'exécution. Les CPI permettent la composabilité : les instructions de n'importe
quel programme peuvent être invoquées par n'importe quel autre programme sur le
réseau.

## Faits clés

- **Deux fonctions** : _rs`invoke`_ (sans signature PDA) et _rs`invoke_signed`_
  (avec signature PDA).
- **Extension de privilèges** : les privilèges de [compte](/docs/core/accounts)
  (signataire, modifiable) s'étendent de l'appelant à l'appelé. Un appelé ne
  peut pas augmenter les privilèges au-delà de ce que l'appelant a transmis.
- **Budget de calcul partagé** : la consommation de CU d'un appelé réduit le
  budget restant de l'appelant.
- **Réentrance** : l'auto-récursion directe est autorisée (A->A->A). La
  réentrance indirecte ne l'est pas (A->B->A retourne
  _rs`ReentrancyNotAllowed`_).

## `invoke` vs `invoke_signed`

Solana fournit deux fonctions pour effectuer des CPI :

| Fonction        | Cas d'usage                                                                | Signature PDA                      |
| --------------- | -------------------------------------------------------------------------- | ---------------------------------- |
| `invoke`        | CPI où tous les signataires requis ont déjà signé la transaction d'origine | Non                                |
| `invoke_signed` | CPI où le programme appelant doit signer au nom d'un PDA qu'il possède     | Oui, via les graines de signataire |

En coulisses, _rs`invoke`_ appelle simplement _rs`invoke_signed`_ avec un
tableau de graines de signataire vide. Utilisez _rs`invoke`_ lorsque vous n'avez
pas besoin de signature PDA, et _rs`invoke_signed`_ lorsque le programme doit
autoriser une action au nom d'un PDA.

Les deux fonctions déclenchent finalement le même appel système
([`sol_invoke_signed_rust`](https://github.com/anza-xyz/agave/blob/v3.1.8/syscalls/src/cpi.rs#L11-L33))
et suivent le même chemin d'exécution
([`cpi_common`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/cpi.rs#L802-L925)).
La seule différence est de savoir si des graines de signataire sont fournies.
Lorsque des graines sont fournies, le runtime dérive les clés publiques PDA et
les ajoute à l'ensemble des signataires valides avant la vérification des
privilèges.
