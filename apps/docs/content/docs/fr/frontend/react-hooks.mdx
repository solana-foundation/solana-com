  Partagez un client Kit dans votre application React avec @solana/react et les

`@solana/react` publie un client Kit dans votre arbre de composants via un seul
fournisseur, puis les hooks lisent les données RPC, les abonnements et l'état du
wallet depuis celui-ci. Les hooks de wallet navigateur se trouvent dans
`@solana/kit-plugin-wallet/react` et lisent le même fournisseur — il n'y a pas
de fournisseur de wallet séparé.

## Installation

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Construire le client

Construisez le client une seule fois, au niveau du module, en dehors de l'arbre
React. Dans le navigateur, `walletSigner` définit le wallet connecté à la fois
comme payeur de frais et comme identité.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` ajoute `client.wallet` (découverte et connexion Wallet Standard),
et `solanaRpc` ajoute `client.rpc` ainsi que le helper `client.sendTransaction`.
Exportez le type du client encapsulé dans `Awaited<>` afin qu'il reste correct
si vous ajoutez ultérieurement un plugin qui se résout de manière asynchrone, et
passez ce `AppClient` à chaque appel `useClient`.

## Envelopper votre arbre une seule fois

`ClientProvider` publie le client. Chaque hook ci-dessous le lit depuis
celui-ci.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Hooks de wallet

Les hooks de wallet lisent la capacité `wallet` du client, ils ne fonctionnent
donc qu'à l'intérieur d'un `ClientProvider` dont le client a installé un plugin
wallet.

- `useWallets()` — Wallets Wallet Standard découverts pour la chaîne du client.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` ou `null`. `signer` est
  `null` pour les wallets en lecture seule.
- `useConnect()` / `useDisconnect()` — retournent un `ActionResult` dont
  `dispatch` (et `dispatchAsync`) exécute l'action et dont `isRunning`,
  `status`, `data`, `error` et `reset` suivent son résultat.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Envoyer une transaction

`useClient` retourne le même client que vous avez publié. Construisez des
instructions avec un client de programme généré et envoyez-les via
`client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Passer le type `AppClient` exporté à `useClient` donne au hook le client complet
étendu par les plugins, de sorte que `client.rpc` et `client.sendTransaction`
sont typés.

## Lire les données

`@solana/react` fournit des hooks pour lire via le client :

- `useRequest(source)` — lecture RPC ponctuelle ; retourne
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — un abonnement en direct ; retourne
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — une valeur initialisée par RPC maintenue à jour par
  un abonnement.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Mémoïsez la source (avec `useMemo`, ou `useCallback` pour une fonction) afin que
le hook ne récupère les données que lorsque ses entrées changent, plutôt qu'à
chaque rendu.

Pour la mise en cache et la revalidation, les sous-chemins `@solana/react/swr`
et `@solana/react/query` enveloppent les mêmes hooks pour
[SWR](https://swr.vercel.app) et [TanStack Query](https://tanstack.com/query).

## Cas d'usage courants pour les développeurs Solana

- **Un seul provider** : `ClientProvider` est le seul provider dont vous avez
  besoin ; les hooks de wallet y accèdent via la capacité `wallet` du client.
- **Compatible avec les composants serveur** : Marquez uniquement les composants
  feuilles qui appellent des hooks avec `"use client"` ; les lectures côté
  serveur peuvent utiliser un client RPC Kit standard sans wallet.
- **Tests** : Publiez un client mocké via `ClientProvider` pour simuler des
  wallets et des réponses RPC.

<Callout type="warn" title="Deux familles de hooks, mêmes noms">
  `useSignIn` et `useSignMessage` existent dans `@solana/react` (ils prennent un
  argument `UiWalletAccount`) et dans `@solana/kit-plugin-wallet/react` (basé
  sur le client, sans argument de compte, retourne un `ActionResult`). Utilisez
  les versions `@solana/kit-plugin-wallet/react` avec le pattern client décrit
  ci-dessus ; les anciens hooks wallet `@solana/react` sont en cours de
  remplacement.
</Callout>

Associez ce guide à la [vue d'ensemble du client Kit](/docs/frontend/client)
pour comprendre le client que chaque hook utilise.
