Le développement frontend sur Solana implique de travailler avec des programmes,
des portefeuilles et des frameworks JavaScript populaires comme React. Interagir
avec ces composants nécessite de gérer la connexion, la création de transactions
et la lecture des comptes Solana.

[Solana Kit](https://github.com/anza-xyz/kit) (`@solana/kit`) est le SDK
JavaScript standard pour ce type de travail. Vous assemblez un client à partir
de petits plugins composables, ajoutez des liaisons React si nécessaire, et
intégrez les clients de programmes générés pour les programmes on-chain que vous
appelez.

## Bibliothèques principales

<Cards>
  <Card title="@solana/kit" href="https://github.com/anza-xyz/kit">
    - SDK principal : RPC, codecs, signataires et transactions - Fournit
    `createClient`, l'hôte de plugins qu'étendent toutes les autres
    bibliothèques
  </Card>
  <Card title="Kit plugins" href="https://github.com/anza-xyz/kit-plugins">
    - Composez un client avec `.use(...)` : connexion RPC, portefeuille,
    signataire et planification de transactions - Publiés en tant que packages
    `@solana/kit-plugin-*`
  </Card>
  <Card
    title="@solana/react"
    href="https://github.com/anza-xyz/kit/tree/main/packages/react"
  >
    - Liaisons React pour un client Kit - `ClientProvider` ainsi que des hooks
    pour les lectures, les abonnements et l'état du portefeuille
  </Card>
  <Card title="@solana-program/*" href="https://github.com/solana-program">
    - Clients générés pour les programmes on-chain (System, Token, et plus) -
    Fournissent des constructeurs d'instructions et des plugins client comme
    `tokenProgram()`
  </Card>
</Cards>

<Callout type="warn" title="web3.js v1 and wallet-adapter are legacy">
  De nombreux projets Solana reposent encore sur l'ancien `@solana/web3.js` (v1)
  et `@solana/wallet-adapter-*`. Pour les nouveaux développements, privilégiez
  `@solana/kit` avec les plugins kit ci-dessous ; la découverte Wallet Standard
  (via `@solana/kit-plugin-wallet`) remplace wallet-adapter pour les
  portefeuilles modernes.
</Callout>

- [Guide du client Kit](/docs/frontend/client) : construisez un client à partir
  de plugins pour le RPC, la signature et l'envoi de transactions.
- [Guide React](/docs/frontend/react-hooks) : liaisons `@solana/react` et hooks
  de portefeuille superposés sur le même client.
- [Tutoriel Next.js](/docs/frontend/nextjs-solana) : une application complète de
  connexion de portefeuille et de transfert de SOL avec l'App Router.
- [Migration vers Kit](/docs/frontend/web3-compat) : correspondance entre les
  concepts web3.js et wallet-adapter et leurs équivalents dans Kit.
