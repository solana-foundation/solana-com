  Bangun klien Solana dari plugin Kit untuk RPC, penandatanganan, dan pengiriman
  transaksi.
---

`@solana/kit` dilengkapi dengan `createClient`, sebuah host kecil yang dapat
diperluas dengan plugin. Setiap plugin menambahkan kemampuan — koneksi RPC,
penanda tangan, perencanaan transaksi — dan klien yang tersusun mendukung skrip,
rute server, maupun antarmuka pengguna.

<Callout type="info">
  Membangun aplikasi React? [Panduan React](/docs/frontend/react-hooks)
  membungkus klien yang sama ini dengan `@solana/react` agar komponen dapat
  membaca status dompet dan data RPC melalui hook. Mulailah dari sana dan
  kembali ke sini ketika Anda membutuhkan kendali yang lebih detail.
</Callout>

## Install

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

Gunakan manajer paket apa pun. Kit versi 7+ dan paket `@solana/kit-plugin-*`
adalah 0.13+.

## Buat klien

Susun plugin dengan `.use(...)`. Tetapkan penanda tangan terlebih dahulu, lalu
koneksi RPC. `solanaDevnetRpc()` menambahkan `client.rpc`, langganan,
perencana/eksekutor transaksi, dan pembantu `client.sendTransaction` — sehingga
sebuah aplikasi jarang perlu menghubungkan komponen-komponen tersebut secara
manual.

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

Plugin `signer` menetapkan pembayar biaya sekaligus identitas aplikasi. Bundel
RPC mensyaratkan pembayar sudah ada terlebih dahulu, itulah mengapa urutan ini
penting.

### Varian penanda tangan

`@solana/kit-plugin-signer` menyediakan tiga peran agar Anda hanya memberikan
izin sesuai kebutuhan klien:

- `signer(x)` — menetapkan `client.payer` dan `client.identity` sekaligus.
- `payer(x)` — pembayar biaya saja.
- `identity(x)` — otoritas aplikasi saja.

Masing-masing memiliki varian `*FromFile(path)` (Node), `generated*()`,
`generated*WithSol(amount)`, dan `airdrop*()` untuk pengujian dan skrip.

## Kirim transaksi

Bangun instruksi dengan klien program yang telah dibuat, lalu serahkan ke
`client.sendTransaction`. Perencana akan mengambil blockhash, memperkirakan
komputasi, menandatangani dengan pembayar klien, dan mengirimkan transaksi.

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

`sendTransaction` menghasilkan objek hasil; tanda tangan transaksi ada di
`result.context.signature`. Fungsi ini memastikan rencana tersebut memenuhi satu
transaksi dan melempar error jika tidak — gunakan `sendTransactions` ketika
sebuah batch dapat mencakup beberapa transaksi.

## Pilih paket RPC

`@solana/kit-plugin-rpc` menyediakan preset kluster:

- `solanaDevnetRpc()` — default devnet, airdrop diaktifkan (`client.airdrop`).
- `solanaMainnetRpc()` — mainnet; airdrop adalah error saat kompilasi.
- `solanaLocalRpc()` — `http://127.0.0.1:8899` untuk validator lokal.
- `solanaRpc({ rpcUrl })` — endpoint apa pun.

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### Susun secara granular

Preset menggabungkan koneksi RPC, perencana, eksekutor, dan helper pengiriman,
sehingga sebagian besar aplikasi tidak perlu menggunakan komponen-komponen
tersebut secara individual. Gunakan plugin granular hanya untuk menyesuaikan
salah satunya — misalnya perencana kustom atau kanal langganan yang berbeda.
Jalur ini juga membutuhkan `@solana/kit-plugin-instruction-plan`, yang sudah
disertakan oleh preset secara otomatis:

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## Tambahkan klien program

Paket `@solana-program/*` yang dihasilkan mengekspos plugin klien sehingga
pemanggilan program terhubung ke klien. Instal program yang Anda butuhkan dan
`.use()` plugin-nya:

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="Peer dependency ranges">
  Beberapa rilis `@solana-program/*` masih mendeklarasikan peer `@solana/kit
  ^6.x` sementara Kit berada di versi 7.x. Paket ini tetap kompatibel dengan Kit
  7 — timpa rentang peer di package manager Anda jika ada peringatan.
</Callout>

## Penandatanganan tanpa dompet browser

Menjalankan proses tanpa antarmuka di rute API, worker, atau skrip? Tandatangani
menggunakan layanan manajemen kunci backend, bukan keypair mentah. Lihat
[Keychain](/docs/tools/keychain) dan
[Penandatanganan di Produksi](/docs/core/transactions/signing-in-production).

## Pola umum untuk pengembang Solana

- **Pengaturan bersama**: Bangun klien sekali di modul inti dan impor ke dalam
  skrip, rute server, dan UI Anda.
- **Pembacaan sisi server**: Klien hanya dengan `signer` + `solanaRpc` (tanpa
  plugin dompet) membaca akun dan mensimulasikan transaksi dari server.
- **Pengujian lokal**: Ganti paket RPC dengan `.use(litesvm())` dari
  `@solana/kit-plugin-litesvm` untuk menjalankan terhadap validator dalam memori
  (khusus Node).
- **Kemudahan pengujian**: Klien yang disusun mudah di-mock — stub respons RPC
  atau penanda tangan tanpa dompet browser yang aktif.
