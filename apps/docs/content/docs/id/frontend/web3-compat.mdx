---
title: Migrasi ke Kit
description:
  Bagaimana konsep @solana/web3.js dan wallet-adapter dipetakan ke @solana/kit.
---

Jika Anda berasal dari `@solana/web3.js` (v1), `@solana/wallet-adapter-*`, atau
paket framework-kit sebelumnya (`@solana/client`, `@solana/react-hooks`,
`@solana/web3-compat`), pendekatan-pendekatan tersebut telah digantikan.
Aplikasi baru dibangun langsung di atas `@solana/kit` dan plugin kit. Halaman
ini memetakan konsep yang sudah Anda ketahui ke padanannya di Kit.

## Pemetaan konsep

| web3.js v1 / wallet-adapter            | Kit                                                                                     |
| -------------------------------------- | --------------------------------------------------------------------------------------- |
| `new Connection(url)`                  | `createSolanaRpc(url)`, atau `.use(solanaRpc({ rpcUrl }))` pada klien                   |
| `PublicKey`                            | `Address` — konversi string dengan `address("...")`                                     |
| `Keypair`                              | `generateKeyPairSigner()` atau plugin `@solana/kit-plugin-signer`                       |
| `SystemProgram.transfer(...)`          | `getTransferSolInstruction(...)` dari `@solana-program/system`                          |
| `LAMPORTS_PER_SOL` math                | `lamports(...)` dari `@solana/kit`                                                      |
| `Transaction` / `VersionedTransaction` | Pesan transaksi yang dibangun dan dikemas oleh perencana transaksi                      |
| `sendAndConfirmTransaction(...)`       | `client.sendTransaction([...])` (perencana menandatangani, mengirim, dan mengonfirmasi) |
| `@solana/wallet-adapter-*`             | `@solana/kit-plugin-wallet` + penemuan Wallet Standard                                  |

## Berasal dari web3.js?

Jika Anda memelihara basis kode `@solana/web3.js` v1 yang besar dan penulisan
ulang penuh ke Kit belum praktis saat ini, web3.js v3 adalah jembatannya. Saat
ini dikirimkan sebagai `@solana/web3.js@rc` (lini `3.0.0-rc`) dan membangun
ulang API berbasis kelas klasik — `Connection`, `Keypair`, `Transaction` — di
atas internal `@solana/kit`. Karena berada di atas Kit, interop sangat erat:
`PublicKey` adalah alias yang sudah usang dari `Address`, dan `Keypair` v3
secara struktural memenuhi `KeyPairSigner` Kit, sehingga Anda dapat
meneruskannya langsung ke API Kit, plugin, dan klien yang dihasilkan oleh
Codama.

v3 masih dalam tahap **release candidate** — pin versi yang tepat dan harapkan
beberapa perubahan API di antara RC. Ini adalah jalur interop warisan, bukan
standar untuk proyek baru; aplikasi baru harus dibangun langsung di atas Kit.
Untuk proses migrasi itu sendiri, lihat panduan resmi
[migrasi web3.js v1 → v3](https://github.com/solana-foundation/solana-web3.js/blob/v3.x/docs/web3js-v1-to-v3-migration.md).

## Dompet

Kit tidak menggunakan wallet-adapter. `@solana/kit-plugin-wallet` menemukan
dompet Wallet Standard dan mengeksposnya melalui `client.wallet` dan
[React hooks](/docs/frontend/react-hooks). Dompet modern (Phantom, Solflare,
Backpack, dan lainnya) mengiklankan diri mereka melalui Wallet Standard,
sehingga tidak diperlukan paket adapter per-dompet.

Anda juga dapat menggunakan alat bawaan seperti
[@solana/connector](https://www.npmjs.com/package/@solana/connector), yang
menggunakan primitif Kit secara langsung untuk terhubung ke dompet.

<Callout type="info" title="Masih menggunakan wallet-adapter?">
  Jika Anda mengelola aplikasi yang dibangun di atas `@solana/wallet-adapter-*`
  dan belum siap untuk bermigrasi, [repositori wallet-adapter
  ](https://github.com/anza-xyz/wallet-adapter) tetap menjadi referensi untuk
  stack tersebut. Untuk proyek baru, gunakan plugin dompet Kit.
</Callout>
