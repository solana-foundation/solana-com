`@solana/react` mempublikasikan klien Kit ke pohon komponen Anda melalui satu
provider, lalu hook membaca data RPC, langganan, dan status wallet darinya. Hook
wallet browser berada di `@solana/kit-plugin-wallet/react` dan membaca dari
provider yang sama — tidak ada provider wallet terpisah.

## Instalasi

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Bangun klien

Bangun klien sekali saja, di lingkup modul, di luar pohon React. Di browser,
`walletSigner` menjadikan wallet yang terhubung sebagai pembayar biaya sekaligus
identitas.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` menambahkan `client.wallet` (penemuan dan koneksi Wallet
Standard), dan `solanaRpc` menambahkan `client.rpc` beserta helper
`client.sendTransaction`. Ekspor tipe klien yang dibungkus dalam `Awaited<>`
agar tetap benar jika Anda nanti menambahkan plugin yang diselesaikan secara
asinkron, dan teruskan `AppClient` tersebut ke setiap panggilan `useClient`.

## Bungkus pohon Anda sekali saja

`ClientProvider` mempublikasikan klien. Setiap hook di bawah membaca darinya.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Hook wallet

Hook wallet membaca kapabilitas `wallet` dari klien, sehingga hanya berfungsi di
dalam `ClientProvider` yang kliennya telah memasang plugin wallet.

- `useWallets()` — Wallet Wallet Standard yang ditemukan untuk chain klien.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` atau `null`. `signer`
  adalah `null` untuk wallet hanya-baca.
- `useConnect()` / `useDisconnect()` — mengembalikan `ActionResult` yang
  `dispatch`-nya (dan `dispatchAsync`) menjalankan aksi dan `isRunning`,
  `status`, `data`, `error`, serta `reset`-nya melacak hasilnya.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Kirim transaksi

`useClient` mengembalikan klien yang sama yang Anda publikasikan. Bangun
instruksi dengan klien program yang dihasilkan dan kirimkan melalui
`client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Meneruskan tipe `AppClient` yang diekspor ke `useClient` memberikan hook klien
yang telah diperluas penuh oleh plugin sehingga `client.rpc` dan
`client.sendTransaction` memiliki tipe yang tepat.

## Baca data

`@solana/react` menyediakan hook yang membaca melalui klien:

- `useRequest(source)` — pembacaan RPC satu kali; mengembalikan
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — langganan langsung; mengembalikan
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — nilai yang diinisialisasi dari RPC dan tetap
  diperbarui melalui langganan.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Lakukan memoize pada sumber (dengan `useMemo`, atau `useCallback` untuk fungsi)
agar hook hanya mengambil ulang data ketika inputnya berubah, bukan pada setiap
render.

Untuk keperluan caching dan revalidasi, subpath `@solana/react/swr` dan
`@solana/react/query` membungkus hook yang sama untuk
[SWR](https://swr.vercel.app) dan [TanStack Query](https://tanstack.com/query).

## Pola umum untuk developer Solana

- **Satu provider**: `ClientProvider` adalah satu-satunya provider yang Anda
  butuhkan; hook wallet membacanya melalui kapabilitas `wallet` milik klien.
- **Mendukung server component**: Hanya tandai komponen leaf yang memanggil hook
  dengan `"use client"`; pembacaan di sisi server dapat menggunakan klien RPC
  Kit biasa tanpa wallet.
- **Pengujian**: Publikasikan klien tiruan melalui `ClientProvider` untuk
  mensimulasikan wallet dan respons RPC.

<Callout type="warn" title="Dua keluarga hook, nama yang sama">
  `useSignIn` dan `useSignMessage` tersedia di `@solana/react` (menggunakan
  argumen `UiWalletAccount`) maupun di `@solana/kit-plugin-wallet/react`
  (berbasis klien, tanpa argumen akun, mengembalikan `ActionResult`). Gunakan
  versi `@solana/kit-plugin-wallet/react` dengan pola klien di atas; hook wallet
  `@solana/react` yang lebih lama sedang digantikan.
</Callout>

Padukan panduan ini dengan [gambaran umum klien Kit](/docs/frontend/client)
untuk memahami klien yang dibaca oleh setiap hook.
