---
title: Pemrograman Staking
---

Untuk memaksimalkan distribusi stake, desentralisasi, dan ketahanan terhadap
sensor di jaringan Solana, staking dapat dilakukan secara terprogram. Tim dan
komunitas telah mengembangkan beberapa program onchain dan offchain untuk
membuat pengelolaan stake menjadi lebih mudah.

### Program Delegasi Solana Foundation

[Program Delegasi Solana Foundation](https://solana.org/delegation-program)
dirancang untuk melayani kebutuhan jaringan dengan menargetkan tujuan-tujuan
berikut:

1. Memaksimalkan desentralisasi, keandalan, dan performa Solana
2. Memaksimalkan jumlah validator yang memiliki sumber stake yang beragam
3. Mempertahankan testnet yang besar dan representatif

### Stake Pools

Program onchain ini mengumpulkan SOL untuk di-stake oleh seorang manajer,
memungkinkan pemegang SOL untuk melakukan staking dan mendapatkan reward tanpa
harus mengelola stake secara langsung. Pengguna mendepositkan SOL dan
mendapatkan token SPL (derivatif staking) yang mewakili kepemilikan mereka dalam
stake pool. Manajer pool melakukan staking terhadap SOL yang didepositkan sesuai
dengan strategi mereka, mungkin menggunakan varian bot delegasi otomatis seperti
yang dijelaskan di atas. Seiring stake mendapatkan reward, pool dan token pool
bertumbuh secara proporsional dalam nilai. Terakhir, pemegang token pool dapat
mengirimkan token SPL kembali ke stake pool untuk menukarkan SOL, sehingga
berpartisipasi dalam desentralisasi dengan upaya yang jauh lebih sedikit.
Informasi lebih lanjut dapat ditemukan di
[dokumentasi SPL stake pool](https://spl.solana.com/stake-pool).
