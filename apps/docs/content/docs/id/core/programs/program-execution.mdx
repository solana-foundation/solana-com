<Callout type="info" title="Ringkasan">
  Program dikompilasi ke sBPF melalui LLVM dan berjalan dalam VM sandbox dengan
  anggaran 1,4 juta CU per transaksi. Runtime menyimpan cache hingga 512 program
  yang dikompilasi, menyediakan syscall untuk logging, CPI, kripto, dan memori,
  serta menunda deployment baru sebanyak 1 slot.
</Callout>

## Kompilasi

Solana menggunakan [LLVM](https://llvm.org/) untuk mengompilasi program ke dalam
binari [ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format) yang
berisi Solana Bytecode Format (sBPF). Binari ELF disimpan on-chain dalam akun
yang dapat dieksekusi.

<Callout>
  sBPF adalah varian khusus Solana dari bytecode
  [eBPF](https://en.wikipedia.org/wiki/EBPF), yang disesuaikan untuk runtime
  Solana. Ini bukan eBPF standar dan memiliki modifikasi khusus Solana.
</Callout>

## Menulis program

Program Solana terutama ditulis dalam
[Rust](https://rust-book.cs.brown.edu/title-page.html) menggunakan salah satu
dari dua pendekatan:

<Cards>
  <Card title="Anchor" href="https://www.anchor-lang.com/docs">
    Framework yang menggunakan makro Rust untuk mengurangi boilerplate.
    Direkomendasikan untuk sebagian besar developer.
  </Card>
  <Card title="Native Rust" href="/docs/programs/rust">
    Rust langsung tanpa framework. Menawarkan kontrol penuh tetapi memerlukan
    lebih banyak implementasi manual.
  </Card>
</Cards>

## Model eksekusi program

Ketika transaksi diproses, runtime mengeksekusi setiap instruksi secara
berurutan melalui
[`process_message()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/message_processor.rs#L15).
Untuk setiap instruksi, runtime:

1. **Menyiapkan konteks instruksi.** Memanggil
   [`prepare_next_top_level_instruction()`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L436)
   untuk memetakan indeks akun instruksi, mengatur flag signer dan writable, dan
   mengonfigurasi
   [`TransactionContext`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L467).

2. **Memeriksa precompile.** Jika program adalah
   [precompile](/docs/core/programs/precompiles), runtime memanggil
   [`process_precompile()`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L491),
   yang masih melakukan push dan pop stack frame (melalui _rs`push()`_ dan
   _rs`pop()`_) tetapi melewati sBPF VM dan pencarian cache program,
   mengeksekusi kode native secara langsung.

3. **Mendorong stack frame.** (Langkah 3-6 terjadi di dalam
   [`InvokeContext::process_instruction()`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L481)
   dan
   [`process_executable_chain()`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L507),
   dipanggil dari _rs`process_message()`_.) Memanggil
   [`push()`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L233)
   pada
   [`InvokeContext`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L184),
   yang menambah tinggi stack instruksi dan menerapkan aturan reentrancy: sebuah
   program hanya dapat masuk kembali ke dirinya sendiri jika pemanggil
   **langsung** (program di posisi teratas stack instruksi saat ini) adalah
   program yang sama. Rekursi diri yang dalam (A -> A -> A) diperbolehkan,
   tergantung pada batas kedalaman stack. Pola reentrancy lainnya (misalnya, A
   memanggil B memanggil A) mengembalikan
   _rs`InstructionError::ReentrancyNotAllowed`_.

4. **Menyelesaikan program.** Runtime memanggil
   [`process_executable_chain()`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L507)
   yang menentukan loader. Jika pemilik program account adalah
   [native loader](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L517),
   program tersebut adalah builtin dan fungsi entrypoint-nya dicari langsung
   dari
   [`ProgramCacheForTxBatch`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L706).
   Jika pemiliknya adalah salah satu BPF loader (`bpf_loader_deprecated`,
   `bpf_loader`, `bpf_loader_upgradeable`, atau `loader_v4`), entrypoint builtin
   loader itu sendiri yang dipanggil.

5. **Mengeksekusi program BPF.** Untuk program BPF,
   [entrypoint loader](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L385)
   mencari executable yang telah dikompilasi dari cache program.
   [`execute()`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L1449)
   kemudian:
   - Menserialisasi data account ke dalam buffer parameter datar
   - Membuat VM sBPF dengan stack, heap, dan region memori
   - Menjalankan kode yang telah dikompilasi, mengonsumsi unit komputasi selama
     eksekusi. Mengembalikan _rs`ComputationalBudgetExceeded`_ jika anggaran
     terlampaui.
   - Mendeserialisasi data account dari buffer kembali ke state account

6. **Mengeluarkan stack frame.** Memanggil
   [`pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/invoke_context.rs#L268)
   yang memverifikasi bahwa instruksi tidak melanggar aturan akuntansi runtime
   (saldo lamport seimbang, account readonly tidak dimodifikasi, ukuran data
   account berada dalam batas).

7. **Mengakumulasi unit komputasi.** Unit komputasi yang dikonsumsi oleh
   instruksi ditambahkan ke total transaksi melalui
   [`saturating_add`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/message_processor.rs#L53).

## Cache program

Runtime memelihara
[`ProgramCache`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L677)
global yang menyimpan program yang telah diverifikasi dan dikompilasi. Cache ini
menyadari fork-graph dan menangani aturan visibilitas deployment, eviction, dan
kompilasi ulang pada batas epoch.

### Jenis entri cache

Setiap program yang di-cache memiliki
[`ProgramCacheEntryType`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L132)
yang menentukan perilaku runtime-nya:

| Jenis                                                                                                             | Deskripsi                                                                                                                                                                                                                                                                                             |
| ----------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`Loaded`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L147)             | Program yang telah diverifikasi dan dikompilasi, siap untuk dieksekusi.                                                                                                                                                                                                                               |
| [`Builtin`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L149)            | Program native yang dikompilasi ke dalam binary validator (System, Stake, Vote, dll.). Tidak disimpan on-chain.                                                                                                                                                                                       |
| [`Unloaded`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L145)           | Program yang sebelumnya telah diverifikasi namun executable yang dikompilasi telah dievict dari memori untuk mengosongkan ruang. Masih melacak statistik penggunaan. Dapat dimuat ulang tanpa verifikasi ulang.                                                                                       |
| [`FailedVerification`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L134) | Tombstone untuk program yang tidak lolos verifier sBPF di bawah feature set saat ini. Dapat menjadi `Loaded` jika aktivasi fitur mengubah aturan verifikasi.                                                                                                                                          |
| [`Closed`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L139)             | Tombstone untuk program yang secara eksplisit ditutup atau tidak pernah di-deploy. Juga digunakan untuk akun (seperti akun buffer) yang termasuk dalam loader tetapi tidak berisi kode yang dapat dieksekusi.                                                                                         |
| [`DelayVisibility`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L141)    | Tombstone sintetis yang dikembalikan oleh [`ProgramCacheForTxBatch::find()`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L755) ketika entri `Loaded` ada tetapi belum efektif (`effective_slot`-nya di masa depan). Tidak pernah disimpan langsung di cache. |

### Penundaan visibilitas

Program yang baru di-deploy atau di-upgrade tidak langsung efektif. Konstanta
[`DELAY_VISIBILITY_SLOT_OFFSET`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L32)
adalah `1`, yang berarti program yang di-deploy di slot N menjadi efektif di
slot N+1. Selama slot deployment, setiap upaya untuk memanggil versi baru
mengembalikan _rs`DelayVisibility`_, menyebabkan runtime melaporkan "Program is
not deployed."

### Kebijakan eviksi

Cache menyimpan hingga
[`MAX_LOADED_ENTRY_COUNT`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L31)
(512) entri program yang dikompilasi. Ketika batas tercapai, program yang paling
jarang digunakan akan dieviksi ke status _rs`Unloaded`_. Penggunaan dilacak oleh
[`tx_usage_counter`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L197)
(ditambahkan setiap kali transaksi mereferensikan program) dan
[`latest_access_slot`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L199).

### Rekompilasi batas epoch

Jika aktivasi fitur mengubah
[`ProgramRuntimeEnvironments`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/loaded_programs.rs#L529)
pada batas epoch, semua program yang di-cache akan
[dikompilasi ulang](https://github.com/anza-xyz/agave/blob/v3.1.8/runtime/src/bank.rs#L1494)
terhadap lingkungan yang baru.

## Data pengembalian

Program dapat mengatur data pengembalian melalui syscall `sol_set_return_data`.
Data disimpan dalam
[`TransactionReturnData`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L499)
struct tingkat transaksi yang menyimpan byte data dan `program_id` dari program
yang instruksinya memanggil syscall. Ukuran maksimum adalah 1.024 byte
(_rs`MAX_RETURN_DATA`_).
