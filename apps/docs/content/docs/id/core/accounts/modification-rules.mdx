<Callout type="info" title="Ringkasan">
  Pemeriksaan runtime setelah setiap instruksi: hanya pemilik yang dapat
  mendebit lamport atau memodifikasi data, data dapat bertambah maksimal 10 KiB
  per instruksi, perubahan pemilik memerlukan data yang diinisialisasi nol, dan
  flag executable bersifat ireversibel.
</Callout>

Runtime Solana menerapkan aturan-aturan ini setelah setiap instruksi dieksekusi
melalui metode
[`BorrowedInstructionAccount`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L722-L727).
Setiap aturan diperiksa pada titik modifikasi, dan transaksi akan dibatalkan
jika ada pemeriksaan yang gagal.

## Aturan lamport

| Aturan                                                              | Penerapan                                                                                                                                                                                                                          | Error                         |
| ------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Hanya pemilik yang dapat mendebit lamport                           | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L785): memeriksa `is_owned_by_current_program()` ketika `lamports < current`                                                  | `ExternalAccountLamportSpend` |
| Akun read-only tidak dapat mengubah lamport                         | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L788-L789): memeriksa `is_writable()`                                                                                              | `ReadonlyLamportChange`       |
| Program apa pun dapat mengkredit lamport ke akun yang dapat ditulis | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L789): pemeriksaan kepemilikan hanya berlaku ketika saldo baru kurang dari saldo saat ini; pemeriksaan writable tetap berlaku | `ReadonlyLamportChange`       |
| Lamport harus seimbang di seluruh instruksi                         | [`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L366-L398): memverifikasi `get_lamports_delta() == 0`                                                                   | `UnbalancedInstruction`       |

## Transisi status rent

Akun berada dalam salah satu dari tiga nilai
[`RentState`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L19-L30):
_rs`Uninitialized`_ (0 lamport), _rs`RentPaying`_ (di atas 0 tetapi di bawah
minimum bebas rent), dan _rs`RentExempt`_ (pada atau di atas minimum). Transisi
yang tidak diizinkan menghasilkan
_rs`TransactionError::InsufficientFundsForRent`_.

Runtime menegakkan aturan ini melalui
[`transition_allowed()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L107-L126):

- Akun apa pun dapat bertransisi ke _rs`Uninitialized`_ (tutup) atau
  _rs`RentExempt`_.
- Tidak ada akun yang dapat memasuki _rs`RentPaying`_ dari _rs`Uninitialized`_
  atau _rs`RentExempt`_. Semua akun baru harus bebas rent.

## Aturan peminjaman akun

Selama eksekusi instruksi, runtime menegakkan semantik peminjaman single-writer
pada akun. Sebuah program dapat memperoleh satu referensi mutable atau beberapa
referensi immutable ke sebuah akun, tetapi tidak keduanya secara bersamaan. Jika
sebuah program mencoba meminjam akun yang sudah dipinjam secara mutable (atau
meminjam secara mutable akun yang sudah dipinjam secara immutable), instruksi
gagal dengan _rs`AccountBorrowFailed`_ melalui
[`try_borrow()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L498-L508)
dan
[`try_borrow_mut()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L512-L520).
Jika sebuah instruksi selesai sementara peminjaman masih berlangsung, runtime
mengembalikan _rs`AccountBorrowOutstanding`_ di
[`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L374-L385).
