---
title: Permintaan Transfer
description: Buat URL pembayaran sederhana untuk transfer SOL dan token SPL
---

Permintaan transfer adalah bentuk integrasi Solana Pay yang paling sederhana.
Mereka membuat URL pembayaran non-interaktif yang dapat digunakan dompet untuk
langsung menyusun dan mengirim transaksi tanpa memerlukan server.

## Ringkasan

URL permintaan transfer mengikuti format ini:

```
solana:<recipient>
      ?amount=<amount>
      &spl-token=<spl-token>
      &reference=<reference>
      &label=<label>
      &message=<message>
      &memo=<memo>
```

## Transfer SOL Dasar

Buat permintaan pembayaran SOL sederhana:

```typescript
import { address } from "@solana/kit";
import { encodeURL } from "@solana/pay";

// Recipient's Solana wallet address
const recipient = address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB");

// Create the payment URL
const url = encodeURL({
  recipient,
  amount: 0.01, // 0.01 SOL
  label: "My Store",
  message: "Thanks for your purchase!"
});

console.log(url.toString());
// Output: solana:FvJ8k8Hh...?amount=0.01&label=My%20Store&message=Thanks%20for%20your%20purchase!
```

## Transfer Token SPL

Buat permintaan pembayaran USDC:

```typescript
import { address } from "@solana/kit";
import { encodeURL } from "@solana/pay";

// USDC mint address on mainnet
const usdcMint = address("EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v");

const url = encodeURL({
  recipient: address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB"),
  amount: 10.5, // $10.50 USDC
  splToken: usdcMint,
  label: "Coffee Shop",
  message: "Grande Latte + Tip"
});

console.log(url.toString());
```

## Parameter URL

### Parameter Wajib

#### Penerima

Alamat dompet Solana penerima (dikodekan base58):

```typescript
const recipient = address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB");
```

### Parameter Opsional

#### Jumlah

Jumlah pembayaran dalam satuan yang mudah dipahami pengguna (satuan SOL atau
token):

```typescript
// SOL amounts
const solAmount = 1.5; // 1.5 SOL
const smallAmount = 0.001; // 0.001 SOL

// Token amounts (USDC example)
const usdcAmount = 25.99; // $25.99 USDC
```

Jika dihilangkan, dompet akan meminta pengguna untuk memasukkan jumlahnya.

#### Token SPL

Untuk transfer token, tentukan alamat mint:

```typescript
// Common SPL tokens
const USDC = address("EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v");
const USDT = address("Es9vMFrzaCERmJfrF4H2FYD4KCoNkY11McCe8BenwNYB");
// Omit splToken for native SOL transfers
```

#### Referensi

Pengenal unik untuk melacak pembayaran. Buat alamat untuk digunakan sebagai
kunci referensi:

```typescript
import { generateKeyPair, getAddressFromPublicKey } from "@solana/kit";

// Generate a unique reference
const keypair = await generateKeyPair();
const reference = await getAddressFromPublicKey(keypair.publicKey);
```

#### Label

Nama merchant atau aplikasi yang ditampilkan kepada pengguna:

```typescript
const label = "Acme Coffee Shop";
```

#### Pesan

Deskripsi pembayaran:

```typescript
const message = "Order #12345 - 2x Americano";
```

#### Memo

Memo on-chain (terlihat secara publik):

```typescript
const memo = "OrderId:12345";
```

## Integrasi Kode QR

Buat kode QR menggunakan fungsi `createQR` bawaan:

```typescript
import { address } from "@solana/kit";
import { encodeURL, createQR } from "@solana/pay";

const url = encodeURL({
  recipient: address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB"),
  amount: 0.05,
  label: "Donation",
  message: "Support our project"
});

// Generate a QR code element
const qr = createQR(url);

// Append to DOM
const element = document.getElementById("qr-code");
qr.append(element);
```

## Validasi Pembayaran

### Menggunakan Merchant Client

Cara paling sederhana untuk menemukan dan memvalidasi pembayaran:

```typescript
import { address } from "@solana/kit";
import { createMerchantClient } from "@solana/pay";

const merchant = createMerchantClient({
  rpcUrl: "https://api.mainnet-beta.solana.com"
});

const recipient = address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB");
const reference = address("YOUR_REFERENCE_ADDRESS");

// Find the transaction by reference
const found = await merchant.pay.findReference(reference);

// Validate the transfer matches expected fields
await merchant.pay.validateTransfer(found.signature, {
  recipient,
  amount: 0.1
});

console.log("Payment confirmed!", found.signature);
```

### Menggunakan Fungsi Standalone

```typescript
import { createSolanaRpc, address } from "@solana/kit";
import { findReference, validateTransfer } from "@solana/pay";

const rpc = createSolanaRpc("https://api.mainnet-beta.solana.com");
const reference = address("YOUR_REFERENCE_ADDRESS");

// Find transaction by reference
const found = await findReference(rpc, reference);

// Validate transaction details
await validateTransfer(rpc, found.signature, {
  recipient: address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB"),
  amount: 0.1
});

console.log("Payment confirmed!", found.signature);
```

### Memantau Pembayaran (WebSocket)

Pendekatan yang direkomendasikan — `watchReference` berlangganan melalui
WebSocket dan menyelesaikan segera setelah transaksi yang sesuai masuk. Tidak
perlu polling.

```typescript
async function waitForPayment(reference, recipient, amount) {
  const merchant = createMerchantClient({
    rpcUrl: "https://api.mainnet-beta.solana.com"
  });

  // Resolves on the first transaction mentioning this reference
  const result = await merchant.pay.watchReference(reference);

  if (result.err) {
    throw new Error(`Transaction failed: ${JSON.stringify(result.err)}`);
  }

  // Validate that the transaction matches the expected payment
  await merchant.pay.validateTransfer(result.signature, { recipient, amount });

  return { signature: result.signature };
}
```

### Polling untuk Pembayaran (fallback HTTP)

Jika langganan WebSocket tidak tersedia (misalnya beberapa penyedia RPC),
gunakan polling sebagai alternatif dengan `findReference`:

```typescript
async function pollForPayment(reference, recipient, amount) {
  const merchant = createMerchantClient({
    rpcUrl: "https://api.mainnet-beta.solana.com"
  });

  const checkPayment = async () => {
    try {
      const found = await merchant.pay.findReference(reference);
      await merchant.pay.validateTransfer(found.signature, {
        recipient,
        amount
      });
      return { success: true, signature: found.signature };
    } catch {
      return { success: false };
    }
  };

  // Check every 2 seconds
  return new Promise((resolve) => {
    const interval = setInterval(async () => {
      const result = await checkPayment();
      if (result.success) {
        clearInterval(interval);
        resolve(result);
      }
    }, 2000);
  });
}
```

## Contoh Lengkap

Berikut adalah contoh lengkap dengan pembuatan kode QR dan pemantauan
pembayaran:

```typescript
import { address, generateKeyPair, getAddressFromPublicKey } from "@solana/kit";
import { encodeURL, createQR, createMerchantClient } from "@solana/pay";

const recipient = address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB");
const amount = 0.1;

// Generate unique reference for this payment
const keypair = await generateKeyPair();
const reference = await getAddressFromPublicKey(keypair.publicKey);

// Create payment URL and QR code
const url = encodeURL({
  recipient,
  amount,
  reference,
  label: "Demo Store",
  message: "Test Purchase"
});
const qr = createQR(url);
qr.append(document.getElementById("qr-code"));

// Monitor for payment
const merchant = createMerchantClient({
  rpcUrl: "https://api.mainnet-beta.solana.com"
});

const interval = setInterval(async () => {
  try {
    const found = await merchant.pay.findReference(reference);
    await merchant.pay.validateTransfer(found.signature, {
      recipient,
      amount,
      reference
    });
    clearInterval(interval);
    console.log("Payment confirmed!", found.signature);
  } catch {
    // Payment not found yet, keep polling
  }
}, 2000);
```

## Praktik Terbaik

### Keamanan

- Selalu validasi pembayaran di sisi server
- Gunakan referensi unik untuk setiap pembayaran
- Verifikasi jumlah transaksi dan penerima
- Periksa status keberhasilan transaksi

### Pengalaman Pengguna

- Tampilkan jumlah pembayaran dan deskripsi yang jelas
- Berikan pembaruan status pembayaran
- Tangani koneksi dompet dengan baik
- Dukung kode QR dan tautan langsung

### Penanganan Kesalahan

- Tangani kegagalan koneksi dompet
- Sediakan alternatif untuk dompet yang tidak didukung
- Tampilkan pesan kesalahan yang membantu
- Terapkan batas waktu pembayaran
