---
title: Konfigurasi Lingkungan
description:
  Metode pola builder untuk mengonfigurasi lingkungan pengujian LiteSVM
---

Semua metode konfigurasi menggunakan pola builder dengan `LiteSVM::default()`.

## with_compute_budget

```rust
pub fn with_compute_budget(self, compute_budget: ComputeBudget) -> Self
```

Tetapkan batas unit komputasi dan ukuran heap.

```rust
use solana_compute_budget::compute_budget::ComputeBudget;

let mut svm = LiteSVM::default()
    .with_compute_budget(ComputeBudget {
        compute_unit_limit: 1_400_000,
        heap_size: 256 * 1024,
        ..ComputeBudget::default()
    });
```

## with_sigverify

```rust
pub fn with_sigverify(self, sigverify: bool) -> Self
```

Aktifkan atau nonaktifkan verifikasi tanda tangan. Default: `true`.

```rust
// Disable for faster tests
let mut svm = LiteSVM::default()
    .with_sigverify(false);
```

## with_blockhash_check

```rust
pub fn with_blockhash_check(self, check: bool) -> Self
```

Aktifkan atau nonaktifkan validasi blockhash. Default: `true`.

```rust
// Disable to use any blockhash
let mut svm = LiteSVM::default()
    .with_blockhash_check(false);
```

## with_transaction_history

```rust
pub fn with_transaction_history(self, capacity: usize) -> Self
```

Tetapkan jumlah transaksi yang disimpan dalam riwayat. Default: `0`.

```rust
// Keep last 100 transactions
let mut svm = LiteSVM::default()
    .with_transaction_history(100);
```

## with_log_bytes_limit

```rust
pub fn with_log_bytes_limit(self, limit: Option<usize>) -> Self
```

Tetapkan jumlah maksimum byte untuk log transaksi. Default: `Some(10_000)`.

```rust
// Unlimited logs
let mut svm = LiteSVM::default()
    .with_log_bytes_limit(None);

// Custom limit (50KB)
let mut svm = LiteSVM::default()
    .with_log_bytes_limit(Some(50_000));
```

## with_lamports

```rust
pub fn with_lamports(self, lamports: u64) -> Self
```

Tetapkan lamport awal untuk pengumpulan biaya. Default: `1_000_000_000`.

```rust
let mut svm = LiteSVM::default()
    .with_lamports(10_000_000_000);
```

## with_default_programs

```rust
pub fn with_default_programs(self) -> Self
```

Sertakan program SPL default (Token, Associated Token, dll.).

```rust
let mut svm = LiteSVM::default()
    .with_default_programs();
```

## with_sysvars

```rust
pub fn with_sysvars(self) -> Self
```

Menyertakan sysvar default (Clock, Rent, EpochSchedule, SlotHashes,
StakeHistory, dll.).

```rust
let mut svm = LiteSVM::default()
    .with_sysvars();
```

## with_feature_set

```rust
pub fn with_feature_set(self, feature_set: FeatureSet) -> Self
```

Konfigurasikan feature set Solana untuk menguji fitur runtime tertentu.

```rust
use agave_feature_set::FeatureSet;

let mut svm = LiteSVM::default()
    .with_feature_set(FeatureSet::all_enabled());
```

## with_precompiles

```rust
#[cfg(feature = "precompiles")]
pub fn with_precompiles(self) -> Self
```

Menyertakan program precompile standar (ed25519, secp256k1, secp256r1).

```rust
let mut svm = LiteSVM::default()
    .with_precompiles();
```

<Callout type="info">
  Memerlukan feature flag `precompiles`:

```toml
[dev-dependencies]
litesvm = { version = "0.11", features = ["precompiles"] }
```

</Callout>

## with_feature_accounts

```rust
pub fn with_feature_accounts(self) -> Self
```

Sertakan akun feature gate di lingkungan pengujian. Ini memastikan perilaku
runtime Solana yang dibatasi fitur tercermin secara akurat dalam pengujian Anda.

```rust
let mut svm = LiteSVM::default()
    .with_feature_accounts();
```

## Fitur register-tracing

Fitur `register-tracing` memungkinkan pelacakan register SBF secara terperinci
selama exekusi transaksi. Fitur ini mengimplikasikan fitur
`invocation-inspect-callback`.

```toml
[dev-dependencies]
litesvm = { version = "0.11", features = ["register-tracing"] }
```

Gunakan `LiteSVM::new_debuggable(true)` untuk membuat instance dengan pelacakan
register yang diaktifkan:

```rust
#[cfg(feature = "register-tracing")]
let mut svm = LiteSVM::new_debuggable(true);
```

Jika diaktifkan, atur variabel lingkungan `SBF_TRACE_DISASSEMBLE` untuk juga
menampilkan output disassembly bersama dengan jejak register:

```bash
SBF_TRACE_DISASSEMBLE=1 cargo test
```

## set_invocation_inspect_callback

```rust
pub fn set_invocation_inspect_callback<C: InvocationInspectCallback + 'static>(&mut self, callback: C)
```

Daftarkan callback yang dipicu sebelum dan sesudah setiap pemanggilan program.
Membutuhkan fitur `invocation-inspect-callback`.

```toml
[dev-dependencies]
litesvm = { version = "0.11", features = ["invocation-inspect-callback"] }
```

```rust
use litesvm::{LiteSVM, types::InvocationInspectCallback};
use solana_svm::invoke_context::InvokeContext;
use solana_svm_transaction::svm_message::SVMMessage;

struct MyInspector;

impl InvocationInspectCallback for MyInspector {
    fn before_invocation(
        &self,
        _svm: &LiteSVM,
        _tx: &impl SVMMessage,
        _program_indices: &[u16],
        _invoke_context: &InvokeContext,
    ) {
        println!("Before program invocation");
    }

    fn after_invocation(
        &self,
        _svm: &LiteSVM,
        _invoke_context: &InvokeContext,
        _enable_register_tracing: bool,
    ) {
        println!("After program invocation");
    }
}

let mut svm = LiteSVM::new();
svm.set_invocation_inspect_callback(MyInspector);
```
