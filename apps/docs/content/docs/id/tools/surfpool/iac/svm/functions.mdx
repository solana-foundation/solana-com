Fungsi-fungsi ini tersedia saat menggunakan addon SVM untuk Solana dan
blockchain yang kompatibel dengan SVM.

## system_program_id

`svm::system_program_id` mengembalikan id dari System Program,
`11111111111111111111111111111111`.

```hcl
output "system_program_id" {
    value = svm::system_program_id()
}
// > 11111111111111111111111111111111
```

## get_instruction_data_from_idl_path

`svm::get_instruction_data_from_idl_path` membuat instruction data yang telah
dienkode untuk pemanggilan program, menyediakan pemeriksaan tipe dan serialisasi
berdasarkan file IDL yang diberikan.

### Input

| Name               | Required | Type          | Description                                           |
| ------------------ | -------- | ------------- | ----------------------------------------------------- |
| `idl_path`         | required | string        | Jalur, relatif terhadap txtx.yml, ke file IDL `.json` |
| `instruction_name` | required | string        | Nama instruksi untuk menghasilkan data                |
| `arguments`        | optional | array[string] | Argumen instruksi untuk menghasilkan data             |

```hcl
output "data" {
    value = svm::get_instruction_data_from_idl("/path/to/idl.json", "my_instruction", ["arg1", "arg2"])
}
// > data: 0x95763bdcc47fa1b305000000776f726c64
```

## get_instruction_data_from_idl

`svm::get_instruction_data_from_idl` membuat instruction data yang telah
dienkode untuk pemanggilan program, menyediakan pemeriksaan tipe dan serialisasi
berdasarkan data IDL yang diberikan.

### Input

| Name               | Required | Type              | Description                                                           |
| ------------------ | -------- | ----------------- | --------------------------------------------------------------------- |
| `idl`              | required | string \| SVM_IDL | IDL program                                                           |
| `instruction_name` | required | string            | Nama instruksi untuk menghasilkan data, sebagaimana diindeks oleh IDL |
| `arguments`        | optional | array[string]     | Argumen instruksi untuk menghasilkan data                             |

```hcl
output "data" {
    value = svm::get_instruction_data_from_idl(variable.contract.idl, "my_instruction", ["arg1", "arg2"])
}
// > data: 0x95763bdcc47fa1b305000000776f726c64
```

## get_program_from_anchor_project

`svm::get_program_from_anchor_project` mengambil artefak deployment program
untuk sebuah program dalam proyek Anchor.

### Input

| Nama           | Wajib    | Tipe   | Deskripsi                   |
| -------------- | -------- | ------ | --------------------------- |
| `program_name` | wajib    | string | Nama program yang di-deploy |
| `keypair_path` | opsional | string | Lokasi file keypair program |
| `idl_path`     | opsional | string | Lokasi file IDL program     |
| `bin_path`     | opsional | string | Lokasi file biner program   |

```hcl
variable "contract" {
    value = svm::get_program_from_anchor_project("my_program")
}
output "idl" {
    value = variable.contract.idl
}
```

`svm::sol_to_lamports` mengonversi jumlah SOL yang diberikan menjadi lamport.

## find_pda

`svm::find_pda` menemukan PDA yang valid menggunakan program id dan seed yang
diberikan.

### Input

| Nama         | Wajib    | Tipe          | Deskripsi                                            |
| ------------ | -------- | ------------- | ---------------------------------------------------- |
| `program_id` | wajib    | string        | Alamat program asal PDA diturunkan                   |
| `seeds`      | opsional | array[string] | Array seed opsional (maks 16, masing-masing 32 byte) |

```hcl
variable "pda" {
    value = svm::find_pda("3bv3j4GvMPjvvBX9QdoX27pVoWhDSXpwKZipFF1QiVr6", ["data"])
}
output "pda" {
    value = std::encode_base58(variable.pda.pda)
}
output "bump" {
    value = variable.pda.bump_seed
}
// > pda: 4amHoWMBgLkPfM8Nq9ZP33Liq9FCuqrLoU1feejkdsUJ
// > bump: 252
```

```hcl
    value = svm::get_associated_token_account(signer.caller.address, "So11111111111111111111111111111111111111112")
}
```

## create_token_account_instruction

`svm::create_token_account_instruction` membuat byte instruksi mentah untuk
membuat associated token account.

### Input

| Name                 | Required | Type   | Description          |
| -------------------- | -------- | ------ | -------------------- |
| `funding_address`    | required | string | Alamat akun pendana  |
| `wallet_address`     | required | string | Alamat wallet        |
| `token_mint_address` | optional | string | Alamat token mint    |
| `token_program_id`   | optional | string | Alamat token program |

```hcl
action "call" "svm::process_instructions" {
    signers = [signer.caller]

    instruction {
        raw_bytes = svm::create_token_account_instruction(
            signer.caller.address, // funding address
            signer.caller.address, // wallet address
            variable.token_mint, // token mint address
            variable.token_program // token program id
        )
    }
}
```

## u64

`svm::u64` membuat representasi array byte dari integer u64, yang cocok
digunakan sebagai seed dalam derivasi PDA.

### Input

| Name    | Required | Type    | Description                                         |
| ------- | -------- | ------- | --------------------------------------------------- |
| `value` | required | integer | Integer u64 yang akan dikonversi menjadi array byte |

```hcl
variable "pda" {
    value = svm::find_pda("3bv3j4GvMPjvvBX9QdoX27pVoWhDSXpwKZipFF1QiVr6", [svm::u64(1000000000)])
}
```
