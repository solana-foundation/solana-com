Penanda tangan ini tersedia saat menggunakan addon SVM untuk penandatanganan
transaksi.

## secret_key

Penanda tangan `svm::secret_key` dapat digunakan untuk menandatangani transaksi
secara sinkron menggunakan secret key, mnemonic, atau file keypair.

### Input

| Name              | Required | Type   | Description                                                        |
| ----------------- | -------- | ------ | ------------------------------------------------------------------ |
| `secret_key`      | optional | string | Secret key yang digunakan untuk menandatangani pesan dan transaksi |
| `mnemonic`        | optional | string | Frasa mnemonic yang digunakan untuk menghasilkan secret key        |
| `derivation_path` | optional | string | Jalur derivasi yang digunakan untuk menghasilkan secret key        |
| `keypair_json`    | optional | string | Jalur ke file keypair.json yang berisi secret key                  |

### Output

| Name         | Type   | Description                         |
| ------------ | ------ | ----------------------------------- |
| `public_key` | string | pubkey dari akun                    |
| `address`    | string | Alamat SVM (alias untuk public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Anda juga dapat menggunakan mnemonic:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Atau file keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

Penanda tangan `svm::web_wallet` memungkinkan operator Runbook untuk
menandatangani transaksi dengan penanda tangan browser pilihan mereka.

### Input

| Name               | Required | Type   | Description                                                                    |
| ------------------ | -------- | ------ | ------------------------------------------------------------------------------ |
| `expected_address` | optional | string | Alamat SVM yang diharapkan untuk terhubung (kosongkan untuk mengizinkan semua) |

### Output

| Name         | Type   | Description                     |
| ------------ | ------ | ------------------------------- |
| `address`    | string | Alamat akun yang terhubung      |
| `public_key` | string | pubkey dari akun yang terhubung |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Penanda tangan ini akan meminta pengguna di Web UI untuk:

1. Menghubungkan dompet mereka
2. Memberikan pubkey melalui tanda tangan pesan
3. Menandatangani semua transaksi yang memerlukan penanda tangan ini

## squads

Penanda tangan `svm::squads` dapat digunakan untuk menandatangani transaksi
dengan vault multisig Squads.

### Input

| Nama      | Wajib    | Tipe   | Deskripsi          |
| --------- | -------- | ------ | ------------------ |
| `address` | opsional | string | Alamat vault squad |

### Output

| Nama         | Tipe   | Deskripsi                           |
| ------------ | ------ | ----------------------------------- |
| `public_key` | string | Kunci publik dari vault squad       |
| `address`    | string | Alamat SVM (alias untuk public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Menggunakan multisig Squads memungkinkan penandatanganan transaksi yang aman
oleh beberapa pihak tanpa mengekspos kunci privat masing-masing individu.
