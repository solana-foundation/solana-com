---
title: Dengan Pay.sh
description: Terima pembayaran dengan Pay.sh.
---

Anda dapat menerima pembayaran pada permintaan API melalui Pay.sh dengan
menyiapkan server Pay. Instal Pay dengan:

```bash
brew install pay
```

atau

```bash
npm install -g @solana/pay
```

Verifikasi instalasi dengan:

```bash
pay --version
```

Toolchain akan menyiapkan alamat Anda secara otomatis. Jalankan perintah berikut
untuk menyiapkan dan mengisi ulang dompet Anda:

```bash
pay setup
```

## Menyiapkan server demo

Anda dapat mulai menerima pembayaran dengan menjalankan server Anda. Lihat
server demo yang berjalan dengan:

```bash
pay server demo
```

Ini melakukan dua hal. Ini menjalankan server Pay secara langsung yang dapat
Anda query melalui permintaan HTTP, dan menghasilkan file `pay-demo.yml` di
direktori Anda saat ini.

Semua rute API dikonfigurasi melalui konfigurasi YAML `endpoints`. Untuk
menerima pembayaran, Anda harus menambahkan atribut `metering:`. Rute
`/api/v1/reports/usage` telah dibuat sebagai contoh untuk Anda:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Anda dapat menguji apakah pembayaran berfungsi dengan menjalankan perintah
berikut sebagai klien di terminal lain:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Anda dapat mempelajari lebih lanjut tentang cara menentukan harga di
[Dokumentasi Pay.sh Building with Pay > Pricing](https://pay.sh/docs/building-with-pay/pricing)

Pelajari lebih lanjut tentang menyiapkan Pay.sh di lingkungan produksi di
[Dokumentasi Konfigurasi Pay.sh](https://pay.sh/docs/toolchain/configuration).
