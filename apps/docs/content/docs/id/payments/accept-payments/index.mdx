Terima pembayaran di Solana menggunakan komponen drop-in, protokol Solana Pay,
atau integrasi RPC langsung.

## Konvensi alamat penerima

Di Solana, **bagikan alamat wallet Anda, bukan token account Anda**. Biasanya
pengirim akan menurunkan
[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)
yang sesuai untuk token apa pun yang mereka kirim (lihat
[Mengirim pembayaran](/docs/payments/send-payments) untuk detail lebih lanjut).

**Contoh:** Bagikan `7EcDhS...` (wallet Anda), bukan `9xKpQ...` (token account
USDC Anda).

<Callout>
  Untuk point-of-sale dan e-commerce, lihat [Solana
  Pay](/docs/payments/accept-payments/solana-pay) untuk kode QR dan URL
  pembayaran, atau [Payment
  Button](/docs/payments/accept-payments/payment-button) untuk komponen React
  drop-in.
</Callout>

## Opsi integrasi

| Pendekatan                                                      | Terbaik untuk                   | Kompleksitas |
| --------------------------------------------------------------- | ------------------------------- | ------------ |
| [Payment Button](/docs/payments/accept-payments/payment-button) | Aplikasi React, integrasi cepat | Rendah       |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | Kode QR, link pembayaran, POS   | Sedang       |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Alur kustom, kontrol penuh      | Lebih tinggi |

## Verifikasi

Setelah menerima pembayaran, verifikasi bahwa pembayaran telah masuk menggunakan
[Verification Tools](/docs/payments/accept-payments/verification-tools). Untuk
aplikasi dengan volume tinggi, lihat
[Indexing](/docs/payments/accept-payments/indexing) untuk pemantauan transaksi
tingkat produksi.
