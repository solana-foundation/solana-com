---
title: Cabut Delegasi dan Langganan yang Ditinggalkan
description:
  Panduan untuk mengklaim kembali rent dari delegasi atau langganan yang
  Otoritas Langganannya sudah tidak ada.
---

Delegasi dan langganan menyimpan rent yang biasanya dikembalikan ke pembayar
yang tercatat saat dicabut. Jika sponsor mendanai akun dan pengguna kemudian
menutup atau mengganti Otoritas Langganan mereka, akun tersebut menjadi
terbengkalai dan rent-nya tertahan. `RevokeAbandonedDelegation` dan
`RevokeAbandonedSubscription` memungkinkan pembayar yang tercatat untuk
mengklaimnya kembali.

## Kapan Ini Berlaku

Hanya diizinkan ketika Otoritas Langganan akun terbukti sudah tidak aktif —
ditutup, atau ditutup dan dibuat ulang (sehingga `init_id` tidak lagi cocok).
Delegasi atau langganan yang masih aktif tidak dapat pernah ditutup dengan cara
ini.

|                                                  | `RevokeAbandonedDelegation`                                                                                                | `RevokeAbandonedSubscription`                                                                                                                  |
| ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Menutup                                          | PDA delegasi tetap atau berulang                                                                                           | PDA Langganan                                                                                                                                  |
| Penanda tangan                                   | Pembayar yang tercatat                                                                                                     | Pembayar yang tercatat                                                                                                                         |
| Akun                                             | Delegasi + otoritas yang tercatat                                                                                          | Langganan + otoritas yang tercatat + paket                                                                                                     |
| Memenuhi syarat ketika                           | Otoritas ditutup, atau dibuat ulang (ketidakcocokan `init_id`)                                                             | Sama, untuk mint paket — mint dibaca dari paket yang terikat, sehingga otoritas yang tidak terkait tidak dapat diganti untuk memaksa penutupan |
| Gunakan pencabutan biasa sebagai gantinya ketika | Delegasi [sudah kedaluwarsa atau habis digunakan](/docs/payments/subscriptions/recurring-delegation#revoke-the-delegation) | Langganan [dibatalkan dan kedaluwarsa](/docs/payments/subscriptions/subscription-plan#cancel-and-revoke)                                       |

`RevokeAbandonedDelegation` juga merupakan satu-satunya jalur pemulihan bagi
pembayar yang tercatat untuk delegasi yang tidak pernah kedaluwarsa
(`expiry_ts == 0`) yang belum sepenuhnya digunakan: pencabutan biasa berlaku
untuk pembayar hanya saat kedaluwarsa atau, untuk delegasi tetap, saat habis
digunakan sepenuhnya.

## Cabut Delegasi yang Ditinggalkan

Pembayar yang tercatat menandatangani. Masukkan PDA delegasi yang tidak aktif
dan Otoritas Langganan yang tercatat (yang mungkin sudah ditutup). Rent
dikembalikan ke pembayar.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedDelegationInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedDelegationInstruction({
      payer: payerSigner,
      delegationAccount: delegationPda,
      subscriptionAuthority: subscriptionAuthorityPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let delegation_pda: Address = address!("DELEGATION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");

    let revoke_abandoned_ix = RevokeAbandonedDelegationBuilder::new()
        .payer(payer)
        .delegation_account(delegation_pda)
        .subscription_authority(subscription_authority)
        .instruction();
    ```

  </Tab>
</Tabs>

## Cabut Langganan yang Ditinggalkan

Pembayar yang tercatat menandatangani. Berikan PDA langganan yang ditinggalkan,
Otoritas Langganan yang tercatat (yang mungkin sudah ditutup), dan paket yang
menjadi bagian dari langganan tersebut. rent dikembalikan kepada pembayar.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedSubscriptionInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedSubscriptionInstruction({
      payer: payerSigner,
      subscriptionAccount: subscriptionPda,
      subscriptionAuthority: subscriptionAuthorityPda,
      planPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let subscription_pda: Address = address!("SUBSCRIPTION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");
    let plan_pda: Address = address!("PLAN_PDA_ADDRESS_HERE");

    let revoke_abandoned_ix = RevokeAbandonedSubscriptionBuilder::new()
        .payer(payer)
        .subscription_account(subscription_pda)
        .subscription_authority(subscription_authority)
        .plan_pda(plan_pda)
        .instruction();
    ```

  </Tab>
</Tabs>

## Catatan

- Pembayar yang tercatat yang menandatangani, bukan delegator, delegatee, atau
  pelanggan.
- Kedua instruksi akan gagal jika Otoritas Langganan masih aktif untuk akun
  tersebut — gunakan cabut biasa sebagai gantinya.
- `RevokeAbandonedDelegation` berfungsi untuk delegasi tetap maupun berulang.
