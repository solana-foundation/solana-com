Panduan untuk menyiapkan lingkungan pengembangan Solana lokal Anda. Pelajari
cara menginstal Rust, Solana CLI, dan Anchor Framework di Windows (WSL), Linux,
dan Mac. Gunakan panduan ini jika Anda lebih suka menginstal setiap dependensi
secara individual, atau jika [instalasi cepat](/docs/intro/installation/) gagal
karena alasan apa pun.

### Prasyarat

<Accordions>
<Accordion title="Windows">

Untuk mengembangkan program Solana di Windows, Anda **harus** menggunakan
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Setelah menginstal WSL, semua dependensi lainnya akan diinstal melalui
terminal Linux.

Untuk menginstal WSL, jalankan perintah berikut di Windows PowerShell:

```terminal
$ wsl --install
```

Proses instalasi akan meminta Anda untuk membuat akun pengguna default.

![WSL Install](/assets/docs/intro/installation/wsl-install.png)

Secara default, WSL menginstal Ubuntu. Anda dapat membuka terminal Linux dengan
mencari "Ubuntu" di bilah pencarian.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Jika terminal Ubuntu Anda terlihat seperti gambar di bawah ini, Anda mungkin
mengalami masalah di mana `ctrl + v` (pintasan keyboard paste) tidak berfungsi
di terminal.

![Ubuntu Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Jika Anda mengalami masalah ini, buka Windows Terminal dengan mencari "Terminal"
di bilah pencarian.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Selanjutnya, tutup Windows Terminal dan buka kembali terminal Linux dengan
mencari Ubuntu lagi. Terminal sekarang seharusnya terlihat seperti gambar di
bawah ini, di mana `ctrl + v` (pintasan keyboard paste) berfungsi.

![Ubuntu Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Jika Anda menggunakan VS Code,
[ekstensi WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
memungkinkan Anda untuk menggunakan WSL dan VS Code bersama-sama.

![Pengaturan WSL di VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Anda akan melihat hal berikut di status bar VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Setelah Anda menyiapkan WSL, Anda siap untuk:

- Menyelesaikan [instalasi cepat](/docs/intro/installation) menggunakan terminal
  Ubuntu (Linux)
- Menginstal [Rust](#install-rust), [Solana CLI](#install-solana-cli), dan
  [Anchor CLI](#install-anchor-cli) secara individual, seperti yang dijelaskan
  di bawah ini

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Jika menggunakan Debian atau alternatif turunan Debian seperti Ubuntu atau Mint, ikuti langkah-langkah berikut:

1. Perbarui daftar paket untuk memastikan semua paket sudah terbaru:

```terminal
$ sudo apt-get update
```

2. Instal dependensi berikut:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Jika Anda mengalami error berikut saat menginstal `protobuf-compiler`, pastikan
Anda menjalankan terlebih dahulu `sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Untuk turunan RHEL seperti Fedora atau Red Hat Linux, ikuti langkah-langkah berikut.

Perbarui daftar paket untuk mendapatkan versi paket yang lebih baru dengan
menjalankan perintah.

```terminal
$ sudo dnf upgrade --refresh
```

Selanjutnya, instal dependensi berikut:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Instal Rust

Developer membangun program Solana menggunakan bahasa pemrograman
[Rust](https://www.rust-lang.org/).

1. Instal Rust menggunakan [rustup](https://www.rust-lang.org/tools/install)
   dengan memasukkan perintah berikut di terminal Anda:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Setelah instalasi berhasil, Anda akan melihat pesan berikut:

<Accordions>
<Accordion title="Pesan Instalasi Rust Berhasil">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Muat ulang variabel environment PATH Anda untuk menyertakan direktori bin
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Verifikasi bahwa instalasi berhasil.

```terminal
$ rustc --version
```

Anda akan melihat output seperti berikut:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Instal Solana CLI

Solana CLI menyediakan semua alat yang diperlukan untuk membangun dan men-deploy
program Solana.

1. Instal Solana CLI tool suite dengan menggunakan perintah instalasi resmi:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Anda dapat mengganti `stable` dengan tag rilis yang sesuai dengan versi
  software dari rilis yang Anda inginkan (misalnya `v2.0.3`), atau gunakan salah
  satu dari tiga nama channel simbolik: `stable`, `beta`, atau `edge`.
</Callout>

2. Tambahkan variabel environment PATH

Instalasi pertama kali Solana CLI mungkin akan meminta Anda untuk menambahkan
variabel environment `PATH`. Untuk melakukannya, tutup dan buka kembali terminal
Anda atau jalankan perintah berikut di shell Anda:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Perbarui variabel environment PATH Anda

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Jika Anda menggunakan **Linux** atau **WSL**, Anda harus menambahkan binary
Solana CLI ke PATH Anda agar perintah tersebut tersedia di terminal Anda. Untuk
melakukannya, ikuti langkah-langkah berikut:

**a.** Periksa shell mana yang Anda gunakan:

```terminal
$ echo $SHELL
```

- Jika output berisi `/bash`, gunakan `.bashrc`.
- Jika output berisi `/zsh`, gunakan `.zshrc`.

**b.** Jalankan perintah yang sesuai, berdasarkan shell Anda.

Untuk Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Untuk Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Jika Anda menggunakan Mac dengan `zsh`, menjalankan perintah `export PATH`
default yang dicatat dari instalasi tidak akan bertahan setelah Anda menutup
terminal. Sebagai gantinya, tambahkan PATH ke file konfigurasi shell Anda dengan
menjalankan perintah berikut:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Langkah-langkah di atas tidak berlaku untuk pengguna `bash`, karena perubahan
  pada variabel PATH akan bertahan secara default.
</Callout>

</Tab>
</Tabs>

4. Restart terminal Anda atau jalankan perintah berikut untuk menyegarkan sesi
   terminal:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Verifikasi bahwa instalasi berhasil dengan memeriksa versi Solana CLI:

```terminal
$ solana --version
```

Anda akan melihat output seperti berikut:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Anda dapat melihat semua versi yang tersedia di
[repo Github Agave](https://github.com/anza-xyz/agave/releases). Agave adalah
klien validator dari [Anza](https://www.anza.xyz/), yang sebelumnya dikenal
sebagai klien validator Solana Labs.

</Callout>

5. Perbarui Solana CLI ke versi terbaru, jika diperlukan _(Opsional)_

```terminal
$ agave-install update
```

</Step>

### Instal Anchor CLI

[Anchor](https://www.anchor-lang.com/) adalah framework untuk mengembangkan
program Solana. Framework Anchor memanfaatkan makro Rust untuk menyederhanakan
proses penulisan program Solana.

#### Prasyarat

File tes proyek Anchor default (TypeScript) yang dibuat dengan perintah
_shell`anchor init`_ memerlukan Node.js dan Yarn. (Template tes Rust tersedia
menggunakan _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Instalasi Node">

1. Instal Node menggunakan
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) dengan
   menjalankan perintah berikut:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Restart terminal Anda dan konfirmasi bahwa perintah `nvm` berjalan dengan
   sukses:

```terminal
$ command -v nvm
```

3. Gunakan `nvm` untuk menginstal Node:

```terminal
$ nvm install node
```

4. Verifikasi bahwa instalasi berhasil dengan memeriksa versi Node:

```terminal
$ node --version
```

Anda akan melihat output seperti berikut:

```
v23.7.0
```

</Accordion>
<Accordion title="Instalasi Yarn">

1. Instal Yarn:

```terminal
$ npm install --global yarn
```

2. Verifikasi bahwa instalasi berhasil dengan memeriksa versi Yarn:

```terminal
$ yarn --version
```

Anda akan melihat output berikut:

```
1.22.1
```

</Accordion>
</Accordions>

#### Instalasi

Anda dapat menginstal Anchor CLI dan tooling dengan dua cara:

- Anchor Version Manager (AVM) — Metode instalasi yang direkomendasikan
- Tanpa AVM — Instal langsung dari GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Anchor Version Manager (AVM) memungkinkan Anda untuk menginstal dan mengelola
berbagai versi Anchor pada sistem Anda dan dengan mudah memperbarui versi Anchor
di masa mendatang. Untuk menginstal Anchor menggunakan AVM, ikuti
langkah-langkah berikut:

1. Instal AVM dengan perintah berikut:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Konfirmasi bahwa AVM berhasil diinstal:

```terminal
$ avm --version
```

3. Instal Anchor CLI menggunakan AVM:

Untuk menginstal versi terbaru:

```terminal
$ avm install latest
$ avm use latest
```

Untuk menginstal versi tertentu, tentukan nomor versinya:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Saat menginstal Anchor CLI pada Linux atau WSL, Anda mungkin mengalami error
berikut:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Jika Anda melihat pesan error ini, ikuti langkah-langkah berikut:

1. Instal dependensi yang tercantum di bagian
   [Linux](/docs/intro/installation#install-dependencies) di bagian atas halaman
   ini.
2. Coba lagi menginstal Anchor CLI.

</Callout>

4. Verifikasi bahwa instalasi berhasil, dengan memeriksa versi Anchor CLI:

```terminal
$ anchor --version
```

Anda akan melihat output seperti berikut:

```
anchor-cli 0.31.1
```

<Callout type="info">

Jangan lupa untuk menjalankan perintah _shell`avm use`_ untuk mendeklarasikan
versi Anchor CLI yang akan dijalankan pada sistem Anda.

- Jika Anda menginstal versi `latest`, jalankan _shell`avm use latest`_.
- Jika Anda menginstal versi `0.30.1`, jalankan _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Instal versi spesifik dari Anchor CLI dengan perintah berikut:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Verifikasi bahwa instalasi berhasil, dengan memeriksa versi Anchor CLI:

```terminal
$ anchor --version
```

Anda akan melihat output seperti berikut:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>

### Instal Surfpool CLI

[Surfpool](https://surfpool.run/) adalah alat untuk pengembangan lokal dan
pengganti yang lebih baik untuk `solana-test-validator`. Pelajari lebih lanjut
tentang fitur Surfpool di [dokumentasi Surfpool](https://docs.surfpool.run/).

1. Instal Surfpool dengan perintah berikut:

```terminal
$ curl -sL https://run.surfpool.run/ | bash
```

2. Verifikasi bahwa instalasi berhasil dengan memeriksa versi Surfpool CLI:

```terminal
$ surfpool --version
```

Anda akan melihat output seperti berikut:

```
surfpool 0.12.0
```

</Step>
