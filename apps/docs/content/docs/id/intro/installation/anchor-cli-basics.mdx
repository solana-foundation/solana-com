Bagian ini menyediakan beberapa perintah dan contoh umum untuk membantu Anda
memulai menggunakan [Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Inisialisasi proyek

Buat proyek Anchor baru dengan menjalankan perintah yang ditunjukkan di bawah
ini. Ini akan membuat direktori baru dengan nama proyek dan menggunakannya untuk
menginisialisasi proyek Anchor baru.

```terminal
$ anchor init <project-name>
```

Misalnya, perintah di bawah ini akan membuat proyek bernama `my-project`.
Direktori `my-project` akan berisi program Rust dasar dan template test
TypeScript.

```terminal
$ anchor init my-project
```

Kemudian navigasi ke direktori proyek:

```terminal
$ cd <project-name>
```

Lihat
[struktur file proyek](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
Anchor.

</Step>
<Step>

### Build program

Untuk mem-build proyek Anda, jalankan perintah berikut:

```terminal
$ anchor build
```

Anda dapat menemukan program yang telah dikompilasi di direktori
`/target/deploy`.

<Callout type="warn">

Saat menjalankan `anchor build`, jika Anda mengalami error berikut:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Coba solusi berikut:

1. Force install menggunakan perintah berikut:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Jika solusi sebelumnya tidak berhasil, hapus cache Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Anda dapat memperbaiki ini dengan mengubah field version pada file `Cargo.lock` dari 4 menjadi 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Lihat [issue ini](https://github.com/solana-foundation/anchor/issues/3392) untuk
informasi lebih lanjut.

</Accordion>

</Accordions>

Setelah menerapkan solusi sebelumnya, coba jalankan _shell`anchor build`_ lagi.

</Callout>

</Step>
<Step>

### Deploy program

Untuk men-deploy proyek Anda, jalankan perintah berikut:

```terminal
$ anchor deploy
```

Perintah ini men-deploy program Anda ke `cluster` yang ditentukan dalam file
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Test program

Untuk menguji proyek Anda, jalankan perintah berikut:

```terminal
$ anchor test
```

Perintah ini mem-build, men-deploy, dan menjalankan test untuk proyek Anda.

<Callout>
  Saat menggunakan cluster `localnet`, Anchor secara otomatis memulai validator
  lokal, men-deploy program, menjalankan test, kemudian menghentikan validator.
</Callout>

<Callout type="warn">

Salah satu dari error berikut dapat mengindikasikan bahwa Anda belum menginstal
Node.js atau Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
