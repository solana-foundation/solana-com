---

您可以通过设置 Pay 服务器，经由 Pay.sh 接受 API 请求的付款。使用以下命令安装 Pay：

```bash
brew install pay
```

或

```bash
npm install -g @solana/pay
```

使用以下命令验证安装：

```bash
pay --version
```

工具链将自动为您配置地址。运行以下命令以设置并为您的钱包充值：

```bash
pay setup
```

## 设置演示服务器

您可以通过启动服务器开始接受付款。使用以下命令查看演示服务器的实际效果：

```bash
pay server demo
```

此操作将完成两件事：启动一个可通过 HTTP 请求查询的实时 Pay 服务器，并在当前目录中生成一个
`pay-demo.yml` 文件。

所有 API 路由均通过 `endpoints` YAML 配置文件进行配置。若要接受付款，您需要添加
`metering:` 属性。其中 `/api/v1/reports/usage` 路由已作为示例为您配置好：

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

您可以在另一个终端中以客户端身份运行以下命令，以测试付款功能是否正常：

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

[Pay.sh 使用 Pay 构建 > 定价文档](https://pay.sh/docs/building-with-pay/pricing)
中了解更多关于如何定义定价的信息。

在 [Pay.sh 配置文档](https://pay.sh/docs/toolchain/configuration)
中了解更多关于在生产环境中设置 Pay.sh 的信息。
