AI
agent 在工作流程中变得越来越自主：编写代码、获取数据、配置资源等。要实现完全独立运行，它们需要一种能够以极低成本、实时、自动化地为服务进行“微支付”的方式。传统支付渠道并不适用于微支付。Solana 的亚分币手续费和亚秒级最终性，使这一点在技术和经济上都成为可能。

## 为什么这很重要？

举例来说，假设开发者在用 LLM 构建应用时需要从付费 API 获取数据。为了获取重要数据，开发者需要中断工作流程，查找 API、选择服务商、注册账号、添加支付方式、生成 API 密钥、将密钥复制到环境中，然后才能继续工作。这种繁琐的流程会打断专注状态。对于 AI 辅助的工作流来说，这就是一个阻碍。

而通过 agentic
payments，agent 可以直接发起请求、完成支付，并继续工作流程。无需账号，无需 API 密钥，无需人工干预。

### 使用 pay.sh 支付 API 费用

[pay.sh](https://pay.sh) 是一个面向 HTTP 代理和命令行工具的支付层。请参阅
[pay.sh 文档](https://pay.sh/docs)以安装 `pay`。`pay` CLI 封装了 `curl`、`codex`
和 `claude`
等工具,使它们能够在无需提供商账户或 API 密钥的情况下调用付费 API。当 API 返回 MPP 或 x402
`402 Payment Required` 质询时,`pay` 会请求您的本地钱包批准签名。

您可以通过以下方式安装 Pay：

```bash
brew install pay
```

或

```bash
npm install -g @solana/pay
```

使用以下命令验证安装：

```bash
pay --version
```

工具链将自动为您配置地址。运行以下命令以设置并充值您的钱包：

```bash
pay setup
```

## x402 协议

自主代理支付需要一种方式，让客户端和服务器能够通过标准 Web 基础设施协商支付条款，同时不影响传统 Web 界面的正常使用。小额支付领域目前仍处于起步阶段，随着该领域的发展，新的工具和标准将不断涌现。然而，[x402 协议](https://www.x402.org/)已作为早期标准崭露头角，并获得了强大的生态系统支持。x402 采用
[HTTP 的 402「需要付款」](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status/402)状态码——这一标准自 HTTP/1.1 起便已存在，但直到区块链结算出现后才真正具备实用价值。

### 工作原理

客户端发起请求，收到携带支付条款的 402 响应后，再附带签名支付重新发起请求。服务器将验证和结算委托给促进者（Facilitator）——一个负责处理链上交易提交的可选中间方。一旦促进者确认结算完成，服务器便返回所请求的内容。这种分离设计使 API 提供商无需直接管理任何链上基础设施即可接受支付。

![x402 流程图](/assets/guides/x402/x402-flow-solana.png)

该协议是无状态的（无需会话或 API 密钥），可与标准 HTTP 基础设施协同工作，并支持服务器所接受的任意 SPL 代币。以下是使用 x402 中间件搭建 Express 服务器的示例代码：

```ts
app.use(
  paymentMiddleware(
    {
      "GET /costly-data": {
        accepts: [
          {
            scheme: "exact",
            price: "$0.001",
            network: "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
            payTo: svmAddress
          }
        ],
        description: "Costly data",
        mimeType: "application/json"
      }
    },
    new x402ResourceServer(facilitatorClient).register(
      "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
      new ExactSvmScheme()
    )
  )
);

app.get("/costly-data", (req, res) => {
  res.send({
    report: {
      data: "costly data"
    }
  });
});
```
