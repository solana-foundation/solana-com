Surfpool 将基础设施即代码（IaC）引入 Solana 生态系统，使开发者能够以极低的开销实现可复现、安全且自动化的部署。虽然模拟是 Surfpool 的核心功能，但其底层 IaC 系统在支持现代 Solana 开发工作流方面同样至关重要。

## 概述

加密基础设施即代码（Crypto
IaC）的理念将传统 DevOps 所熟悉的优势——自动化、可审计性、模块化——带入了 Web3 领域。该模型旨在确保在复杂的加密环境中实现一致且可扩展的部署实践。

Surfpool 的 Crypto IaC 方案源于多年来与高级协议的实践经验，包括 Pyth、Wormhole
Core、Circle CCTP、Bitcoin Ordinals 等。

一个健全的 Crypto
IaC 系统通常涵盖三类组件：链上基础设施、签名基础设施和链下基础设施。

## 链上基础设施

该类别包括：

- Solana 程序部署
- 升级
- 状态迁移

结构完善的 IaC 实现可确保这些操作具有可复现性、版本可控性和可审计性。开发者可以追踪每次部署，并在执行前了解其影响。

## 签名基础设施

安全的交易签名在生产环境中至关重要。现代 IaC 系统支持以下方式，而非依赖本地 keypair（存在重大安全风险）：

- 硬件钱包
- 门限密码学
- 多签钱包

Surfpool
IaC 中的签名模块具有模块化和可配置性。从硬编码私钥迁移到复杂的签名方案（如 Squads 多签）只需更新配置即可完成，无需重写代码。

## 链下基础设施

包括：

- 索引器
- 状态监视器
- 钱包哨兵
- 自动化脚本

这些组件通常响应链上事件，应被视为部署流水线的一部分——像任何其他代码制品一样，具备可测试性、可移植性和版本管理能力。

> **注意：**
> Surfpool 的 IaC 刻意排除了 RPC 和节点供应。这些底层关注点已由 Terraform、Ansible 及云原生服务等工具提供了良好支持，超出了应用级基础设施管理的范畴。

## 语言设计原则

静态分析是一项关键设计目标。开发者无需运行代码，即可生成执行计划——包括涉及的程序、预期签名者、关联账户及预估费用。

核心原则：

- 声明式语法（无 JS DSL，无命令式 Shell 脚本）
- 可组合结构
- 极低学习曲线
- 基础设施与逻辑的关注点分离

通过简单替换签名提供商或网络端点等组件，即可实现从本地开发到主网部署的无缝过渡。

## Web3 Runbook

Surfpool 的 IaC 引擎由 Web3
Runbooks 驱动，这是一个专为区块链部署构建的领域特定框架。该系统历经数月开发，旨在确保：

- 易用性
- 例程可组合性
- 安全的运行时执行

它同时支持 Solana 和以太坊兼容链。

在以太坊环境中，Runbooks 已被高性能团队广泛使用——例如 Consensys 的 Infura 团队——用于涉及 Eigenlayer 和多个二层网络的复杂协议部署。

## Surfpool 集成

借助 Surfpool，我们将 Runbooks 提升到了新的高度——优化了开发者体验，并针对 Solana 使用场景进行了深度定制。目标是让 Solana 开发从第一天起就具备模块化、可测试性和生产就绪能力。

通过将 Surfnets 与 Crypto IaC 相结合，Surfpool 提供：

- 本地优先的开发者技术栈
- 通过简单的配置更新实现向主网的平滑过渡
- 完全定义且可复现的网络环境
