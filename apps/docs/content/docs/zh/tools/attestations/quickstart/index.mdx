只需几个步骤即可开始使用 Solana 认证。本指南将引导您了解核心概念并帮助您创建第一个认证。

## 您将学到什么

- 安装和设置认证库
- 创建您的第一个凭证（认证机构）
- 为您的认证定义模式
- 发布您的第一个认证
- 获取和验证认证

## 前置要求

- Solana 开发基础知识
- Node.js 16+ 或 Rust 开发环境
- 一个包含一些 SOL 用于交易费用的 Solana 钱包

## 快速入门步骤

### 1. 安装

安装必要的软件包并设置您的开发环境。

**[→ 安装指南](/docs/tools/attestations/quickstart/installation)**

### 2. 创建凭证

设置一个可以发布认证的认证机构。

**[→ 创建凭证指南](/docs/tools/attestations/instructions/create-credential)**

### 3. 定义模式

创建一个定义您的认证结构的模式。

**[→ 创建模式指南](/docs/tools/attestations/instructions/create-schema)**

### 4. 发布认证

使用凭证和模式创建您的第一个可验证声明。

**[→ 创建认证指南](/docs/tools/attestations/instructions/create-attestation)**

## 核心概念

### 凭证

凭证代表一个认证机构。它定义了谁可以发布认证，并包含授权签名者列表。

### 模式

模式定义了认证的结构和验证规则。它们指定了认证包含哪些字段及其数据类型。

### 证明

证明是由凭证下的授权签名者创建的可验证声明或陈述，遵循特定的架构结构。

## 使用场景

证明可用于：

- **身份验证**：验证用户身份和合规性
- **信誉系统**：建立信任评分和信誉指标
- **认证**：颁发专业证书和资质
- **合规**：确保金融服务的监管合规
- **供应链**：追踪商品的真实性和来源
