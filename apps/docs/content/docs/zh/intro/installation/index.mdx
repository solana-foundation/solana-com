通过一条命令快速搭建本地 Solana 开发环境。可在 Windows（WSL）、Linux 和 Mac 上安装 Rust、Solana
CLI 及 Anchor
Framework。（如有需要，你也可以[单独安装每个依赖](/docs/intro/installation/dependencies)。）

## 前置条件

<Accordions>
<Accordion title="Windows">

在 Windows 上开发 Solana 程序时，**必须**使用
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)（WSL）。安装 WSL 后，其他依赖项都将在 Linux 终端中安装。

要安装 WSL，请在 Windows PowerShell 中运行以下命令：

```terminal
$ wsl --install
```

安装过程中会提示你创建一个默认用户账户。

![WSL 安装](/assets/docs/intro/installation/wsl-install.png)

WSL 默认安装 Ubuntu。你可以在搜索栏中输入 "Ubuntu" 来打开 Linux 终端。

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

如果你的 Ubuntu 终端界面如下图所示，可能会遇到
`ctrl + v`（粘贴快捷键）在终端中无法使用的问题。

![Ubuntu 终端](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

如果遇到此问题，请在搜索栏中输入 "Terminal" 打开 Windows Terminal。

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

接下来，关闭 Windows
Terminal，再次通过搜索 Ubuntu 打开 Linux 终端。此时终端应如下图所示，`ctrl + v`（粘贴快捷键）可以正常使用。

![Ubuntu 终端](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

如果你使用 VS
Code，[WSL 扩展](https://code.visualstudio.com/docs/remote/wsl-tutorial)可以让你在 WSL 和 VS
Code 间协同开发。

![VS Code 中的 WSL 设置](/assets/docs/intro/installation/wsl-vscode.png)

您将在 VS Code 状态栏中看到如下内容：

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

一旦您完成 WSL 设置，您就可以：

- 使用 Ubuntu（Linux）终端完成[快速安装](#installation)
- 按下方说明，分别安装
  [Rust](/docs/intro/installation/dependencies#install-rust)、[Solana CLI](/docs/intro/installation/dependencies#install-solana-cli)
  和 [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli)

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
如果您使用 Debian 或基于 Debian 的发行版（如 Ubuntu 或 Mint），请按照以下步骤操作：

1. 更新软件包列表，确保所有软件包都是最新的：

```terminal
$ sudo apt-get update
```

2. 安装以下依赖项：

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
如果您在安装 `protobuf-compiler` 时遇到如下错误，请确保您已先运行 _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    如果您使用 RHEL 衍生版（如 Fedora 或 Red Hat Linux），请按照以下步骤操作。

通过运行命令更新软件包列表，以获取最新版本的软件包。

```terminal
$ sudo dnf upgrade --refresh
```

接下来，安装以下依赖项：

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## 安装

1. 在终端中输入以下命令，以安装所有必需的依赖项：

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

安装成功后，会返回类似如下的输出：

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. 通过检查每个已安装依赖项的版本，验证安装是否成功。

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  如果快速安装命令失败，请参考下方的[安装依赖](/docs/intro/installation/dependencies)部分，按照说明单独安装每个依赖。
</Callout>
