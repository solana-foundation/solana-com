h1: 部署你的第一个 Solana 程序
---

使用 Anchor 框架，将你的第一个 [程序](/docs/core/programs)
构建、部署并测试到 Solana 区块链。本指南将帮助你熟悉 Solana
Playground，并为你后续使用程序派生地址（PDA）和跨程序调用（CPI）做好准备。

<Steps>

<Step>

### 创建 Anchor 项目

1. 在网页浏览器中打开 [Solana Playground](https://beta.solpg.io)
2. 在左侧面板点击 **+ 创建新项目**
3. 输入 **项目名称**
4. 选择 **Anchor (Rust)** 作为框架
5. 点击 **创建**

![新建项目](/assets/docs/intro/quickstart/pg-new-project.gif)

`src/lib.rs` 源文件将会打开。它包含一个名为 `Initialize` 的 struct，其功能如下：

- 接收一个名为 _rs`data: u64`_ 的必需参数
- 创建一个新账户
- 将 _rs`data: u64`_ 的值保存到新账户中
- 向交易的程序日志添加一条消息

<WithNotes>

```rust title="lib.rs"
use anchor_lang::prelude::*;

// This is your program's public key and it will update
// automatically when you build the project.
// !tooltip[/declare_id/] declare_id
declare_id!("11111111111111111111111111111111");

// !tooltip[/program/] program
#[program]
mod hello_anchor {
    use super::*;
    // !tooltip[/data: u64/] parameter
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !tooltip[/ctx.accounts.new_account.data = data/] initialize
        ctx.accounts.new_account.data = data;
        // !tooltip[/msg!/] msg
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}

// !tooltip[/Accounts/] accounts
#[derive(Accounts)]
pub struct Initialize<'info> {
    // We must specify the space in order to initialize an account.
    // First 8 bytes are default account discriminator,
    // next 8 bytes come from NewAccount.data being type u64.
    // (u64 = 64 bits unsigned integer = 8 bytes)
    // !tooltip[/account/] constraints
    #[account(
      // !tooltip[/init/] init
      init,
      // !tooltip[/payer/] payer
      payer = signer,
      // !tooltip[/space/] space
      space = 8 + 8
    )]
    pub new_account: Account<'info, NewAccount>,
    // !tooltip[/account/] constraints
    #[account(mut)]
    pub signer: Signer<'info>,
    pub system_program: Program<'info, System>,
}

// !tooltip[/account/] account
#[account]
pub struct NewAccount {
    data: u64
}
```

### !declare_id

你的程序链上地址。每次构建项目时都会更新

### !program

包含程序指令的模块

### !accounts

用于标注包含程序账户的结构体

### !account

表示本程序可创建和拥有的账户数据类型的结构体

### !constraints

账户约束。例如，常见的检查或增强程序安全性的功能

### !parameter

调用指令时传递的值

### !initialize

访问 `data` 字段，并将 `new_account` 账户的该字段设置为 `data: u64` 的值

### !msg

_rs`msg!`_ 宏会将一条消息记录到交易的程序日志中

### !init

调用 System
Program 创建新账户，并将本程序设为该账户的所有者。作为账户所有者，本程序随后可以写入该账户数据

### !payer

为新账户支付费用的账户

### !space

为账户分配的字节数

</WithNotes>

<Accordion title="说明">

<WithMentions>

- [`declare_id!`](mention:one) 宏指定了你程序的链上地址。Solana
  Playground 在你构建程序时会自动更新该地址。

```rs
// !mention one
declare_id!("11111111111111111111111111111111");
```

</WithMentions>

<WithMentions>

- [`#[program]`](mention:one) 属性用于注解包含代表程序指令函数的模块。

```rs
// !mention one
#[program]
mod hello_anchor {
    use super::*;
    // !mention two
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !mention three
        ctx.accounts.new_account.data = data;
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}
```

在本例中，[`initialize`](mention:two) 指令接收两个参数：

- `ctx: Context<Initialize>`：将本指令所需的账户传递给函数，如 `Initialize`
  struct 中所指定。
- `data: u64`：调用指令时你必须提供的自定义参数。

函数体[将 `data` 字段设置为 `new_account`](mention:three) 所提供的 `data`
参数，然后向程序日志打印一条消息。

</WithMentions>

<WithMentions>

- [`#[derive(Accounts)]`](mention:one)
  属性定义了一个 struct，用于指定某个指令所需的账户。每个字段代表一个独立账户。

  Anchor 会通过字段类型（如 `Signer<'info>`）和约束（如
  `#[account(mut)]`）自动处理与账户校验相关的常见安全检查。

  ```rs
  // !mention one
  #[derive(Accounts)]
  pub struct Initialize<'info> {
      #[account(init, payer = signer, space = 8 + 8)]
      pub new_account: Account<'info, NewAccount>,
      #[account(mut)]
      pub signer: Signer<'info>,
      pub system_program: Program<'info, System>,
  }
  ```

</WithMentions>

<WithMentions>

- [`#[account]`](mention:one)
  属性定义了一个 struct，表示该程序创建和拥有的账户的数据类型。

```rs
// !mention one
#[account]
pub struct NewAccount {
  data: u64
}
```

</WithMentions>

<Step>

### 构建并部署程序

1. 编译程序代码

```terminal
$ build
Building...
Build successful. Completed in 1.46s.
```

<Callout>
  你的 [program ID](/docs/intro/quick-start/deploying-programs) 已在
  _rs`declare_id!()`_ 中定义，现在将被更新。
</Callout>

2. 向你的钱包添加 SOL（可选）

部署程序需要在链上账户中有 SOL。如有需要，请在继续第 3 步前，[向你的钱包添加 SOL](/docs/intro/quick-start#add-sol-to-your-wallet)。

3. 将程序部署到网络

```terminal
$ deploy
Deploying... This could take a while depending on the program size and network conditions.
Warning: 1 transaction not confirmed, retrying...
Deployment successful. Completed in 19s.
```

<Callout type="info">部署后，程序的字节码会存储在可执行程序账户中。</Callout>

**注意：** 如果你愿意，也可以使用左侧面板中的 **Build** 和 **Deploy**
按钮，如下图所示。

![构建并部署](/assets/docs/intro/quickstart/pg-build-deploy.png)

</Step>

### 测试程序

<WithMentions>

1. 打开名为 `tests/anchor.test.ts`
   的测试文件。该文件演示了如何从客户端调用程序的 [`initialize`](mention:one)
   struct。

```ts title="anchor.test.ts"
// No imports needed: web3, anchor, pg and more are globally available

describe("Test", () => {
  it("initialize", async () => {
    // Generate keypair for the new account
    const newAccountKp = new web3.Keypair();

    // Send transaction
    const data = new BN(42);
    const txHash = await pg.program.methods
      // !mention one
      .initialize(data)
      .accounts({
        newAccount: newAccountKp.publicKey,
        signer: pg.wallet.publicKey,
        systemProgram: web3.SystemProgram.programId
      })
      .signers([newAccountKp])
      .rpc();
    console.log(`Use 'solana confirm -v ${txHash}' to see the logs`);

    // Confirm transaction
    await pg.connection.confirmTransaction(txHash);

    // Fetch the created account
    const newAccount = await pg.program.account.newAccount.fetch(
      newAccountKp.publicKey
    );

    console.log("On-chain data is:", newAccount.data.toString());

    // Check whether the data on-chain is equal to local 'data'
    assert(data.eq(newAccount.data));
  });
});
```

</WithMentions>

<Step>

### 关闭程序

关闭程序以回收分配给链上程序的 SOL。操作方法：运行
`solana program close [ProgramID]`，并使用存储在 _rs`declare_id!()`_
宏中的程序地址。

```terminal
$ solana program close 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r
Closed Program Id 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r, 2.79511512 SOL reclaimed
```

<Callout type="warn">
  只有拥有升级权限的账户才能**关闭或更新**程序。每次部署时，Playground
  钱包会自动成为每个程序的升级权限账户。如果升级权限被撤销，程序将变为不可变。
</Callout>

恭喜！你已使用 Anchor 构建并部署了你的第一个 Solana 程序。
