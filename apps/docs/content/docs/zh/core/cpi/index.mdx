Cross Program Invocation（CPI）是指一个 [program](/docs/core/programs)
在执行过程中调用另一个程序的
[instruction](/docs/core/instructions)。CPI 实现了可组合性：网络上的任何程序都可以调用其他程序的指令。

## 关键要点

- **两个函数**：_rs`invoke`_（无 PDA 签名）和
  _rs`invoke_signed`_（带 PDA 签名）。
- **权限扩展**：[Account](/docs/core/accounts)
  权限（签名者、可写）会从调用方传递到被调用方。被调用方无法提升权限，只能使用调用方传递的权限。
- **共享计算预算**：被调用方消耗的 CU 会减少调用方剩余的预算。
- **重入性**：允许直接自递归（A->A->A），但不允许间接重入（A->B->A 返回
  _rs`ReentrancyNotAllowed`_）。

## `invoke` 与 `invoke_signed`

Solana 提供了两个用于进行 CPI 的函数：

| 函数            | 使用场景                                    | PDA 签名              |
| --------------- | ------------------------------------------- | --------------------- |
| `invoke`        | 所有所需签名者已在原始交易中签名的 CPI      | 否                    |
| `invoke_signed` | 调用程序需要代表其拥有的 PDA 进行签名的 CPI | 是，通过 signer seeds |

在底层，_rs`invoke`_ 实际上只是调用了 _rs`invoke_signed`_，并传入一个空的 signer
seeds 数组。当你不需要 PDA 签名时，使用
_rs`invoke`_；当程序需要代表 PDA 授权操作时，使用 _rs`invoke_signed`_。

这两个函数最终都会触发相同的 syscall（[`sol_invoke_signed_rust`](https://github.com/anza-xyz/agave/blob/v3.1.8/syscalls/src/cpi.rs#L11-L33)），并经过相同的运行时路径（[`cpi_common`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/cpi.rs#L802-L925)）。唯一的区别在于是否提供了 signer
seeds。当提供 seeds 时，运行时会派生 PDA 公钥，并在权限检查前将其加入有效签名者集合。
