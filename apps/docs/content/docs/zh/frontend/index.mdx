---
title: 前端
description: 了解如何为 Solana 程序构建界面。
---

在 Solana 上进行前端开发，涉及与 Programs、钱包以及流行的 JavaScript 框架（如 React）协作。与这些组件交互时，需要处理连接、交易创建以及读取 Solana 账户数据。

为帮助完成这些工作，Solana 提供了多种适用于不同框架的客户端库。

## 主要库

<Cards>
  <Card
    title="@solana/client"
    href="https://github.com/solana-foundation/framework-kit/tree/main/packages/client"
  >
    - 简单的 Solana 客户端，集成了 RPC、钱包、交易功能 -
    内置状态存储、操作、监听器、连接器
  </Card>
  <Card
    title="@solana/react-hooks"
    href="https://github.com/solana-foundation/framework-kit/tree/main/packages/react-hooks"
  >
    - 完整的 hooks，支持钱包、余额、转账、签名、查询等功能 - React provider
    hooks，封装 `@solana/client` 运行时状态
  </Card>
  <Card
    title="@solana/web3-compat"
    href="https://github.com/solana-foundation/framework-kit/tree/main/packages/web3-compat"
  >
    - 兼容 Web3.js 的工具包，简化升级流程 - 新的内部实现结合了 web3.js 和 kit
  </Card>
  <Card
    title="@solana/kit"
    href="https://github.com/solana-foundation/framework-kit/tree/main/packages/kit"
  >
    - 底层 Solana SDK，为其他 Solana 库（如 `@solana/react-hooks`）提供支持 -
    完全支持 tree-shaking，采用现代 Web 标准，驱动运行时
  </Card>
</Cards>

<Callout type="warn" title="@solana/web3.js 已弃用">
  许多 Solana 生态项目仍依赖已弃用的 `@solana/web3.js`。 建议优先使用
  `@solana/web3-compat`，以简化迁移路径。
</Callout>

- [@solana/client 指南](/docs/frontend/client)：精简、无界面的 RPC、钱包和交易运行时。
- [@solana/react-hooks 指南](/docs/frontend/react-hooks)：基于同一客户端运行时的 React
  hooks。
- [@solana/web3-compat 指南](/docs/frontend/web3-compat)：兼容层，帮助从
  `@solana/web3.js` 迁移到 Kit 驱动的技术栈。
