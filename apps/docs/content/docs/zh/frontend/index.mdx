在 Solana 上进行前端开发，涉及与 Programs、钱包以及流行的 JavaScript 框架（如 React）协作。与这些组件交互时，需要处理连接、交易创建以及读取 Solana 账户数据。

[Solana Kit](https://github.com/anza-xyz/kit)（`@solana/kit`）是完成此类工作的标准 JavaScript
SDK。你可以通过小型、可组合的插件来构建客户端，在需要时添加 React 绑定，并引入所需链上程序的自动生成客户端。

## 主要库

<Cards>
  <Card title="@solana/kit" href="https://github.com/anza-xyz/kit">
    - 核心 SDK：RPC、编解码器、签名者与交易 - 提供
    `createClient`，作为所有其他库扩展的插件宿主
  </Card>
  <Card title="Kit plugins" href="https://github.com/anza-xyz/kit-plugins">
    - 使用 `.use(...)` 组合客户端：RPC 连接、钱包、签名者与交易规划 - 以
    `@solana/kit-plugin-*` 包形式发布
  </Card>
  <Card
    title="@solana/react"
    href="https://github.com/anza-xyz/kit/tree/main/packages/react"
  >
    - Kit 客户端的 React 绑定 - `ClientProvider` 加上用于读取、订阅和钱包状态的
    Hooks
  </Card>
  <Card title="@solana-program/*" href="https://github.com/solana-program">
    - 链上程序的自动生成客户端（System、Token 等） -
    提供指令构建器和客户端插件，例如 `tokenProgram()`
  </Card>
</Cards>

<Callout type="warn" title="web3.js v1 and wallet-adapter are legacy">
  许多 Solana 项目仍依赖较旧的 `@solana/web3.js`（v1）和
  `@solana/wallet-adapter-*`。对于新项目，建议使用 `@solana/kit` 配合下方的 Kit
  插件；Wallet Standard 发现机制（通过 `@solana/kit-plugin-wallet`）可替代
  wallet-adapter 以支持现代钱包。
</Callout>
