`@solana/kit` 提供了
`createClient`，一个可通过插件扩展的轻量宿主。每个插件添加一项能力——RPC 连接、签名器、交易规划——组合后的客户端可同时支持脚本、服务端路由和 UI。

<Callout type="info">
  正在构建 React 应用？[React 指南](/docs/frontend/react-hooks) 使用
  `@solana/react` 封装了 同一客户端，让组件通过 hooks 读取钱包状态和 RPC 数据。
  建议从那里开始，需要更精细控制时再回到本文。
</Callout>

## 安装

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

可使用任意包管理器。Kit 版本要求 v7+，`@solana/kit-plugin-*` 包版本要求 0.13+。

## 创建客户端

使用 `.use(...)` 组合插件。先设置签名器，再配置 RPC 连接。 `solanaDevnetRpc()`
会添加 `client.rpc`、订阅、交易规划器/执行器以及 `client.sendTransaction`
辅助函数——因此应用通常无需自行连接这些模块。

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

`signer`
插件同时设置手续费支付方和应用身份。RPC 套件要求支付方已就位，这正是顺序重要的原因。

### 签名器变体

`@solana/kit-plugin-signer` 提供三种角色，确保客户端仅获得所需权限：

- `signer(x)` — 同时设置 `client.payer` 和 `client.identity`。
- `payer(x)` — 仅作为手续费支付方。
- `identity(x)` — 仅作为应用权限方。

每种角色均提供
`*FromFile(path)`（Node）、`generated*()`、`generated*WithSol(amount)` 和
`airdrop*()` 变体，适用于测试和脚本场景。

## 发送交易

使用生成的程序客户端构建指令，然后将其传递给
`client.sendTransaction`。规划器会解析区块哈希、估算计算量，使用客户端的支付方签名并提交交易。

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

`sendTransaction` 返回一个结果对象；交易签名位于
`result.context.signature`。该方法断言计划适合单笔交易，否则抛出异常——若批次可能跨越多笔交易，请使用
`sendTransactions`。

## 选择 RPC 套件

`@solana/kit-plugin-rpc` 提供集群预设：

- `solanaDevnetRpc()` — 开发网默认配置，支持空投（`client.airdrop`）。
- `solanaMainnetRpc()` — 主网；空投在编译时会报错。
- `solanaLocalRpc()` — 用于本地 validator 的 `http://127.0.0.1:8899`。
- `solanaRpc({ rpcUrl })` — 任意端点。

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### 细粒度组合

预设套件已捆绑 RPC 连接、规划器、执行器及发送助手，因此大多数应用无需单独使用各个组件。仅在需要自定义某一部分时才使用细粒度插件——例如自定义规划器或不同的订阅通道。此路径还需要
`@solana/kit-plugin-instruction-plan`，该组件在使用预设时会自动包含：

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## 添加程序客户端

生成的 `@solana-program/*`
包提供客户端插件，使程序调用挂载在客户端上。安装所需程序并通过 `.use()`
注册其插件：

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="对等依赖版本范围">
  部分 `@solana-program/*` 版本仍声明 `@solana/kit ^6.x` 对等依赖， 而 Kit
  当前为 7.x。它们可与 Kit 7
  正常配合使用——如果包管理器报错，请覆盖对等版本范围。
</Callout>

## 不使用浏览器钱包进行签名

在 API 路由、Worker 或脚本中以无头模式运行？请使用后端密钥管理服务进行签名，而非直接使用原始 keypair。请参阅
[Keychain](/docs/tools/keychain) 和
[生产环境签名](/docs/core/transactions/signing-in-production)。

## Solana 开发者常用模式

- **共享初始化**：在核心模块中构建一次客户端，并将其导入脚本、服务端路由和 UI 中。
- **服务端读取**：仅配置 `signer` +
  `solanaRpc`（无钱包插件）的客户端，可从服务端读取账户并模拟交易。
- **本地测试**：将 RPC 套件替换为 `@solana/kit-plugin-litesvm` 中的
  `.use(litesvm())`，以针对内存中的 validator 运行（仅限 Node）。
- **可测试性**：组合客户端易于模拟——无需浏览器钱包即可桩化 RPC 响应或签名者。
