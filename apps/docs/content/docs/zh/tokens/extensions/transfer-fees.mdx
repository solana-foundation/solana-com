## 什么是转账手续费？

Token Extensions Program 的 _rs`TransferFeeConfig`_
铸造扩展会对该铸造账户的每笔转账收取手续费。

在转账时：

- 转账金额会扣除配置的手续费
- 扣留的手续费会在目标代币账户上进行跟踪
- 扣留的手续费可以直接从代币账户中提取，或收集到铸造账户
- `withdraw_withheld_authority` 可以将收集的手续费转移到手续费接收者的代币账户

<Callout type="info" title="代币账户扩展">
  当为带有 _rs`TransferFeeConfig`_
  的铸造账户初始化代币账户时，该代币账户会初始化为
  _rs`TransferFeeAmount`_。当通过 [Associated Token
  Program](/docs/tokens/basics/create-token-account#how-to-create-an-associated-token-account)
  创建代币账户时，系统会计算所需的账户大小，并创建具有所需大小和免租金 lamport
  的代币账户。
</Callout>

## 如何创建带转账手续费的铸造账户

创建带转账手续费的铸造账户步骤：

1. 计算铸造账户大小以及 _rs`TransferFeeConfig`_ 扩展所需的租金。
2. 使用 _rs`CreateAccount`_ 创建铸造账户，初始化 _rs`TransferFeeConfig`_，并使用
   _rs`InitializeMint`_ 初始化铸造账户。
3. 为铸造账户创建代币账户。_rs`TransferFeeAmount`_ 会自动为代币账户启用。
4. 使用 _rs`TransferCheckedWithFee`_
   转账代币并验证预期的手续费。_rs`TransferChecked`_
   也可以使用，它会自动在目标代币账户上扣留配置的手续费。
5. 使用 _rs`WithdrawWithheldTokensFromAccounts`_
   将扣留的手续费直接从代币账户转移到手续费接收者的代币账户，由铸造账户的
   `withdraw_withheld_authority` 签名。
6. 使用无需权限的 _rs`HarvestWithheldTokensToMint`_
   将扣留的手续费从代币账户转移到铸造账户，然后使用
   _rs`WithdrawWithheldTokensFromMint`_
   将铸造账户中扣留的手续费提取到手续费接收者的代币账户，由铸造账户的
   `withdraw_withheld_authority` 签名。
7. 使用 _rs`SetTransferFee`_
   更新下一期的转账手续费配置，该配置将在两个 epoch 后生效。

## !!steps 计算账户大小

计算基础铸币账户加上 _rs`TransferFeeConfig`_ 扩展所需的铸币账户大小。这是在
_rs`CreateAccount`_ 中使用的大小。

<CodePlaceholder title="示例" />

```ts !! title="Example"
const client = await createLocalClient();
const mint = await generateKeyPairSigner();

// !focus(1:14)
const transferFeeConfigExtension = extension("TransferFeeConfig", {
  transferFeeConfigAuthority: client.payer.address,
  withdrawWithheldAuthority: client.payer.address,
  withheldAmount: 0n,
  olderTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  },
  newerTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  }
});

// !focus(1:1)
const mintSpace = BigInt(getMintSize([transferFeeConfigExtension]));
```
