---
title: Revoca Deleghe e Sottoscrizioni Abbandonate
description:
  Guida per recuperare il rent da una delega o sottoscrizione il cui
  Subscription Authority è scomparso.
---

Le deleghe e le sottoscrizioni trattengono il rent che normalmente viene
restituito al pagante registrato alla revoca. Se uno sponsor ha finanziato
l'account e l'utente in seguito chiude o sostituisce il proprio Subscription
Authority, l'account rimane bloccato e il suo rent è inaccessibile.
`RevokeAbandonedDelegation` e `RevokeAbandonedSubscription` consentono al
pagante registrato di recuperarlo.

## Quando Si Applica

Consentito solo quando il Subscription Authority dell'account è provabilmente
terminale — chiuso, oppure chiuso e ricreato (così `init_id` non corrisponde
più). Una delega o sottoscrizione attiva non può mai essere chiusa in questo
modo.

|                                       | `RevokeAbandonedDelegation`                                                                                          | `RevokeAbandonedSubscription`                                                                                                                                   |
| ------------------------------------- | -------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Chiude                                | PDA di delega fissa o ricorrente                                                                                     | PDA di sottoscrizione                                                                                                                                           |
| Firmatario                            | Pagante registrato                                                                                                   | Pagante registrato                                                                                                                                              |
| Account                               | Delega + authority registrata                                                                                        | Sottoscrizione + authority registrata + piano                                                                                                                   |
| Idoneo quando                         | Authority chiusa, o ricreata (mancata corrispondenza di `init_id`)                                                   | Lo stesso, per il mint del piano — il mint viene letto dal piano associato, quindi un'authority non correlata non può essere sostituita per forzare la chiusura |
| Usa invece la revoca ordinaria quando | La delega è [scaduta o completamente spesa](/docs/payments/subscriptions/recurring-delegation#revoke-the-delegation) | La sottoscrizione è [annullata e scaduta](/docs/payments/subscriptions/subscription-plan#cancel-and-revoke)                                                     |

`RevokeAbandonedDelegation` è anche l'unica via di recupero per il pagante
registrato nel caso di una delega senza scadenza (`expiry_ts == 0`) non
completamente spesa: la revoca ordinaria si attiva per il pagante solo alla
scadenza o, per una delega fissa, al completamento della spesa.

## Revoca di una Delega Abbandonata

Il pagante registrato firma. Passa il PDA della delega inattiva e il suo
Subscription Authority registrato (che potrebbe essere già chiuso). Il rent
viene restituito al pagante.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedDelegationInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedDelegationInstruction({
      payer: payerSigner,
      delegationAccount: delegationPda,
      subscriptionAuthority: subscriptionAuthorityPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let delegation_pda: Address = address!("DELEGATION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");

    let revoke_abandoned_ix = RevokeAbandonedDelegationBuilder::new()
        .payer(payer)
        .delegation_account(delegation_pda)
        .subscription_authority(subscription_authority)
        .instruction();
    ```

  </Tab>
</Tabs>

## Revocare una Sottoscrizione Abbandonata

Il pagante registrato firma. Passa il PDA della sottoscrizione abbandonata, la
sua Subscription Authority registrata (che potrebbe essere già chiusa) e il
piano a cui appartiene la sottoscrizione. Il rent viene restituito al pagante.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedSubscriptionInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedSubscriptionInstruction({
      payer: payerSigner,
      subscriptionAccount: subscriptionPda,
      subscriptionAuthority: subscriptionAuthorityPda,
      planPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let subscription_pda: Address = address!("SUBSCRIPTION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");
    let plan_pda: Address = address!("PLAN_PDA_ADDRESS_HERE");

    let revoke_abandoned_ix = RevokeAbandonedSubscriptionBuilder::new()
        .payer(payer)
        .subscription_account(subscription_pda)
        .subscription_authority(subscription_authority)
        .plan_pda(plan_pda)
        .instruction();
    ```

  </Tab>
</Tabs>

## Note

- Il pagante registrato firma, non il delegante, il delegatario o il
  sottoscrittore.
- Entrambe le istruzioni falliscono se la Subscription Authority è ancora attiva
  per l'account — usa invece la revoca ordinaria.
- `RevokeAbandonedDelegation` funziona sia per le deleghe fisse che per quelle
  ricorrenti.
