Accetta pagamenti su Solana utilizzando componenti drop-in, il protocollo Solana
Pay o integrazione RPC diretta.

## Convenzione per l'indirizzo di ricezione

Su Solana, **condividi il tuo indirizzo wallet, non il tuo token account**. È
tipico che il mittente derivi l'appropriato
[associated token account (ATA)](/docs/payments/how-payments-work#token-accounts)
per qualsiasi token stia inviando (vedi
[Inviare pagamenti](/docs/payments/send-payments) per maggiori dettagli).

**Esempio:** Condividi `7EcDhS...` (il tuo wallet), non `9xKpQ...` (il tuo token
account USDC).

<Callout>
  Per punti vendita ed e-commerce, vedi [Solana
  Pay](/docs/payments/accept-payments/solana-pay) per codici QR e URL di
  pagamento, o [Payment Button](/docs/payments/accept-payments/payment-button)
  per componenti React drop-in.
</Callout>

## Opzioni di integrazione

| Approccio                                                       | Ideale per                                | Complessità |
| --------------------------------------------------------------- | ----------------------------------------- | ----------- |
| [Payment Button](/docs/payments/accept-payments/payment-button) | App React, integrazione rapida            | Bassa       |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | Codici QR, link di pagamento, POS         | Media       |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Flussi personalizzati, controllo completo | Più alta    |

## Verifica

Dopo aver ricevuto un pagamento, verifica che sia arrivato utilizzando gli
[strumenti di verifica](/docs/payments/accept-payments/verification-tools). Per
applicazioni ad alto volume, vedi
[Indexing](/docs/payments/accept-payments/indexing) per il monitoraggio delle
transazioni a livello di produzione.
