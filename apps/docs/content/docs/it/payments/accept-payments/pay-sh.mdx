---

Puoi accettare pagamenti sulle richieste API tramite Pay.sh configurando un
server Pay. Installa Pay con:

```bash
brew install pay
```

oppure

```bash
npm install -g @solana/pay
```

Verifica l'installazione con:

```bash
pay --version
```

Il toolchain configurerà automaticamente il tuo indirizzo. Esegui il seguente
comando per configurare e ricaricare il tuo wallet:

```bash
pay setup
```

## Configurazione del server demo

Puoi iniziare ad accettare pagamenti avviando il tuo server. Puoi vedere un
server demo in azione con:

```bash
pay server demo
```

Questo esegue due operazioni. Avvia un server Pay attivo che puoi interrogare
tramite richieste HTTP e genera un file `pay-demo.yml` nella tua directory
corrente.

Tutte le route API sono configurate tramite il file di configurazione YAML
`endpoints`. Per accettare pagamenti, devi aggiungere un attributo `metering:`.
La route `/api/v1/reports/usage` è stata configurata come esempio per te:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Puoi verificare che i pagamenti funzionino eseguendo il seguente comando come
client su un altro terminale:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Puoi saperne di più su come definire i prezzi nella
[documentazione Pay.sh Building with Pay > Pricing](https://pay.sh/docs/building-with-pay/pricing)

Scopri di più su come configurare Pay.sh in produzione nella
[documentazione di configurazione Pay.sh](https://pay.sh/docs/toolchain/configuration).
