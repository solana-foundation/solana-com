---
title: Distribuzione su Railway
description: Distribuisci su Railway con un clic
---

[Railway](https://railway.com/) offre un'esperienza di distribuzione semplice
per i nodi Kora con SSL automatico, gestione dei domini e monitoraggio
integrato. Questa guida ti accompagnerà nella configurazione di un nodo Kora su
Railway con una semplice immagine Docker e la CLI di Railway.

## Prerequisiti

- [Account Railway](https://railway.app)
- [CLI di Railway](https://docs.railway.com/guides/cli) installata
- [CLI di Solana](https://solana.com/docs/intro/installation) per la creazione
  della chiave privata
- [Endpoint RPC Solana](https://solana.com/rpc)

## Passaggio 1: Preparare i File di Distribuzione

Crea una nuova directory per il tuo nodo Kora:

```bash
mkdir my-kora-node
cd my-kora-node
```

Aggiungi i file di esempio alla tua directory:

- [kora.toml](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/kora.toml)
- [signers.toml](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/signers.toml)
- [Dockerfile](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/Dockerfile)

La tua directory dovrebbe ora apparire così:

```bash
my-kora-node/
├── kora.toml
├── signers.toml
└── Dockerfile
```

## Passaggio 2: Distribuire su Railway

Dalla directory del tuo progetto, accedi a Railway

```bash
# Login to Railway
railway login
```

Segui le istruzioni a schermo per accedere con il tuo account.

Inizializza il progetto Railway nella tua directory.

```bash
railway init
```

Railway ti chiederà di:

1. Creare un nuovo progetto o selezionarne uno esistente
2. Scegliere un nome per il progetto (ad es., "my-kora-node")

### Distribuisci la Tua Applicazione

```bash
# Deploy the application
railway up
```

Questo effettuerà:

- Caricamento dei tuoi file su Railway
- Build dell'immagine Docker
- Distribuzione della tua applicazione

**Output previsto:**

```
Build time: 121.81 seconds
Deploy complete
Starting Container
```

Perfetto! Siamo pronti! Probabilmente noterai un errore nei tuoi log ("error: a
value is required for `--rpc-url <RPC_URL>` but none was supplied"). Questo
accade perché non abbiamo ancora configurato le variabili d'ambiente--quindi
facciamolo subito.

## Passaggio 3: Configurare le Variabili d'Ambiente

Apri la tua dashboard Railway su [railway.app](https://railway.app) e vai al tuo
progetto. Nel tuo progetto, clicca su "Settings" > "Variables"

![Variabili d'Ambiente Railway per Kora](/assets/docs/tools/kora/deploy/railway-variables.jpg)

Aggiungi queste variabili d'ambiente:

| Variabile          | Valore                                | Descrizione                                   |
| ------------------ | ------------------------------------- | --------------------------------------------- |
| `RPC_URL`          | `https://api.mainnet-beta.solana.com` | Endpoint RPC di Solana                        |
| `KORA_PRIVATE_KEY` | `your-base58-private-key`             | La tua chiave privata Solana (formato base58) |
| `RUST_LOG`         | `info`                                | Livello di log per il debugging               |

> **Avviso di Sicurezza**: La tua chiave privata ha accesso a SOL per pagare le
> commissioni. Usa un keypair dedicato per il tuo nodo Kora e finanzialo solo
> con il SOL che sei disposto a spendere per le commissioni di transazione. Per
> maggiori informazioni sui firmatari, consulta la
> [Guida ai Firmatari](/docs/tools/kora/operators/signers).

### Ottenere la Tua Chiave Privata

Se devi creare un nuovo keypair:

```bash
# Generate new keypair
solana-keygen new --outfile ~/.config/solana/kora-keypair.json
```

Assicurati di depositare SOL adeguati per testare/operare il tuo nodo.

## Passaggio 4: Ridistribuire

Dopo aver salvato le tue variabili, ti verrà richiesto di ridistribuire il tuo
server-- fallo.

## Passaggio 5: Testare la Tua Distribuzione

### Generare l'Endpoint Pubblico

Per inviare richieste al tuo endpoint, dovrai generare un dominio pubblico.
Dalla tua distribuzione, vai su "Settings" > "Generate domain".

![Configurazione Dominio Railway](/assets/docs/tools/kora/deploy/railway-domain.jpg)

Usa la porta predefinita, "8080" e clicca su "Generate Domain".

Dovresti ottenere un dominio come `https://my-kora-node.railway.app/`

### Testare l'Endpoint di Configurazione

```bash
curl -X POST https://my-kora-node.railway.app \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"getConfig","params":[]}' | jq
```

**Risposta prevista:**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "fee_payer": "YourPublicKeyHere...",
    "validation_config": {
      "max_allowed_lamports": 1000000,
      "allowed_programs": [...],
      "allowed_spl_paid_tokens": [...]
      // ...
    }
  }
}
```

Ottimo lavoro! Il tuo nodo Kora è ora attivo.
