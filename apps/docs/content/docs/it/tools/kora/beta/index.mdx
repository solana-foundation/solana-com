_Disponibile su GitHub:
[`v2.2.0-beta.7`](https://github.com/solana-foundation/kora/releases/tag/v2.2.0-beta.7)_

Kora v2.2.0-beta introduce il supporto per i bundle Jito, nuovi metodi RPC,
immagini Docker, funzionalità di sicurezza migliorate e miglioramenti dell'SDK.

> **Questa è una pre-release.** Le funzionalità qui documentate sono disponibili
> in `v2.2.0-beta.7` e potrebbero cambiare prima del rilascio stabile. Per la
> documentazione stabile, consulta la
> [documentazione principale di Kora](/docs/tools/kora).

## Punti salienti

### Supporto per i Bundle Jito

Invia fino a 5 transazioni atomicamente tramite il block engine di Jito. Nuovi
metodi RPC:

- [`signBundle`](/docs/tools/kora/beta/bundle-methods#signbundle) — Firma un
  bundle senza trasmetterlo
- [`signAndSendBundle`](/docs/tools/kora/beta/bundle-methods#signandsendbundle)
  — Firma e invia a Jito
- [`estimateBundleFee`](/docs/tools/kora/beta/bundle-methods#estimatebundlefee)
  — Stima le commissioni totali del bundle

Consulta la [Guida ai Bundle Jito](/docs/tools/kora/beta/jito-guide) per una
procedura completa.

### Immagini Docker

Immagini ufficiali multi-piattaforma (amd64/arm64) pubblicate sul GitHub
Container Registry. Consulta
[Installazione](/docs/tools/kora/beta/installation).

### Plugin per Transazioni

Un nuovo sistema di plugin consente a Kora di validare la struttura delle
transazioni durante i flussi di firma. Il primo plugin integrato, `gas_swap`,
applica regole rigorose per le transazioni di scambio token-per-SOL senza gas. I
plugin sono configurati in `kora.toml` e vengono eseguiti automaticamente
durante `signTransaction`, `signAndSendTransaction`, `signBundle`, e
`signAndSendBundle`.

Consulta
[Configurazione — Plugin per Transazioni](/docs/tools/kora/beta/configuration#transaction-plugins).

### Nuove Opzioni di Configurazione

- **Plugin per Transazioni** per la validazione della struttura delle
  transazioni (`gas_swap`)
- **reCAPTCHA v3** protezione anti-bot per gli endpoint di firma
- **Lighthouse** protezione contro il prosciugamento del fee payer
- **Limiti di Utilizzo** limitazione delle transazioni per wallet
- **Configurazione bundle** per l'integrazione con Jito

Consulta [Configurazione](/docs/tools/kora/beta/configuration).

### Modifiche all'SDK

- Supporto per i metodi bundle (`signBundle`, `signAndSendBundle`,
  `estimateBundleFee`)

Consulta [Modifiche all'SDK](/docs/tools/kora/beta/sdk-changes).

### Metodi RPC Aggiuntivi

- [`getVersion`](/docs/tools/kora/beta/bundle-methods#getversion) — Recupera le
  informazioni sulla versione del server

## Installazione

```bash
cargo install kora-cli@2.2.0-beta.7
```

Oppure tramite Docker:

```bash
docker pull ghcr.io/solana-foundation/kora:v2.2.0-beta.7
```
