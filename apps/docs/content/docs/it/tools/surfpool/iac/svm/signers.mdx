title: Firmatari SVM

Questi firmatari sono disponibili quando si utilizza il componente aggiuntivo
SVM per la firma delle transazioni.

## secret_key

Il firmatario `svm::secret_key` può essere utilizzato per firmare in modo
sincrono una transazione utilizzando una chiave segreta, una mnemonica o un file
keypair.

### Input

| Name              | Required | Type   | Description                                                          |
| ----------------- | -------- | ------ | -------------------------------------------------------------------- |
| `secret_key`      | optional | string | La chiave segreta utilizzata per firmare messaggi e transazioni      |
| `mnemonic`        | optional | string | La frase mnemonica utilizzata per generare la chiave segreta         |
| `derivation_path` | optional | string | Il percorso di derivazione utilizzato per generare la chiave segreta |
| `keypair_json`    | optional | string | Un percorso a un file keypair.json contenente la chiave segreta      |

### Output

| Name         | Type   | Description                            |
| ------------ | ------ | -------------------------------------- |
| `public_key` | string | La chiave pubblica dell'account        |
| `address`    | string | L'indirizzo SVM (alias per public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Puoi anche utilizzare una mnemonica:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Oppure un file keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

Il firmatario `svm::web_wallet` consentirà a un operatore di Runbook di firmare
la transazione con il firmatario del browser di sua scelta.

### Input

| Name               | Required | Type   | Description                                                                       |
| ------------------ | -------- | ------ | --------------------------------------------------------------------------------- |
| `expected_address` | optional | string | L'indirizzo SVM che ci si aspetta si connetta (omettere per consentire qualsiasi) |

### Output

| Name         | Type   | Description                              |
| ------------ | ------ | ---------------------------------------- |
| `address`    | string | L'indirizzo dell'account connesso        |
| `public_key` | string | La chiave pubblica dell'account connesso |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Questo signer chiederà all'utente nell'interfaccia Web di:

1. Connettere il proprio wallet
2. Fornire una pubkey tramite firma del messaggio
3. Firmare tutte le transazioni che richiedono questo signer

## squads

Il signer `svm::squads` può essere utilizzato per firmare transazioni con un
multisig vault di Squads.

### Input

| Nome      | Obbligatorio | Tipo   | Descrizione                 |
| --------- | ------------ | ------ | --------------------------- |
| `address` | facoltativo  | string | L'indirizzo del vault Squad |

### Output

| Nome         | Tipo   | Descrizione                            |
| ------------ | ------ | -------------------------------------- |
| `public_key` | string | La pubkey del vault Squad              |
| `address`    | string | L'indirizzo SVM (alias per public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

L'utilizzo del multisig di Squads consente una firma delle transazioni sicura e
multi-party senza esporre le chiavi private individuali.
