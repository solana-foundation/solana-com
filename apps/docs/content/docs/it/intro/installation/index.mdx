Un'installazione rapida per configurare il tuo ambiente di sviluppo Solana
locale con un solo comando. Installa Rust, la CLI di Solana e Anchor Framework
su Windows (WSL), Linux e Mac. (Se preferisci, puoi
[installare ogni dipendenza individualmente](/docs/intro/installation/dependencies).)

## Prerequisiti

<Accordions>
<Accordion title="Windows">

Per sviluppare programmi Solana su Windows **devi** utilizzare
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Dopo aver installato WSL, tutte le altre dipendenze verranno installate
tramite il terminale Linux.

Per installare WSL, esegui il seguente comando in Windows PowerShell:

```terminal
$ wsl --install
```

Il processo di installazione ti chiederà di creare un account utente
predefinito.

![Installazione WSL](/assets/docs/intro/installation/wsl-install.png)

Per impostazione predefinita, WSL installa Ubuntu. Puoi aprire un terminale
Linux cercando "Ubuntu" nella barra di ricerca.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Se il tuo terminale Ubuntu appare come nell'immagine qui sotto, potresti
riscontrare un problema in cui `ctrl + v` (scorciatoia da tastiera per
incollare) non funziona nel terminale.

![Terminale Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Se riscontri questo problema, apri Windows Terminal cercando "Terminal" nella
barra di ricerca.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Successivamente, chiudi Windows Terminal e riapri un terminale Linux cercando
Ubuntu di nuovo. Il terminale dovrebbe ora apparire come nell'immagine qui
sotto, dove `ctrl + v` (scorciatoia da tastiera per incollare) funziona.

![Terminale Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Se utilizzi VS Code,
l'[estensione WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) ti
consente di utilizzare WSL e VS Code insieme.

![Configurazione WSL in VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Vedrai quanto segue nella barra di stato di VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Una volta configurato WSL, sei pronto per:

- Completare l'[installazione rapida](#installation) utilizzando il terminale
  Ubuntu (Linux)
- Installare [Rust](/docs/intro/installation/dependencies#install-rust), la
  [CLI di Solana](/docs/intro/installation/dependencies#install-solana-cli) e la
  [CLI di Anchor](/docs/intro/installation/dependencies#install-anchor-cli)
  individualmente, come descritto di seguito

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Se utilizzi Debian o alternative derivate da Debian come Ubuntu o Mint, segui i passaggi seguenti:

1. Aggiorna gli elenchi dei pacchetti per assicurarti che tutti i pacchetti
   siano aggiornati:

```terminal
$ sudo apt-get update
```

2. Installa le seguenti dipendenze:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Se riscontri il seguente errore durante l'installazione di `protobuf-compiler`, assicurati
di eseguire prima _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Per i derivati RHEL come Fedora o Red Hat Linux, segui i seguenti passaggi.

Aggiorna gli elenchi dei pacchetti per ottenere le versioni più recenti dei
pacchetti eseguendo il comando.

```terminal
$ sudo dnf upgrade --refresh
```

Successivamente, installa le seguenti dipendenze:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## Installazione

1. Inserisci il seguente comando nel tuo terminale per installare tutte le
   dipendenze necessarie:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Un'installazione riuscita restituirà un output simile al seguente:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. Verifica un'installazione riuscita controllando la versione di ciascuna
   dipendenza installata.

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  Se il comando di installazione rapida fallisce, consulta la sezione [Installa
  dipendenze](/docs/intro/installation/dependencies) qui sotto per le istruzioni
  su come installare ciascuna dipendenza individualmente.
</Callout>
