Questa sezione fornisce alcuni comandi ed esempi comuni per aiutarti a iniziare
ad usare la [CLI di Anchor](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Inizializza il progetto

Crea un nuovo progetto Anchor eseguendo il comando mostrato di seguito. Creerà
una nuova directory con il nome del progetto e la userà per inizializzare un
nuovo progetto Anchor.

```terminal
$ anchor init <project-name>
```

Ad esempio, il comando seguente creerà un progetto chiamato `my-project`. La
directory `my-project` conterrà un programma Rust di base e un template di test
TypeScript.

```terminal
$ anchor init my-project
```

Quindi naviga nella directory del progetto:

```terminal
$ cd <project-name>
```

Vedi la
[struttura dei file del progetto](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
di Anchor.

</Step>
<Step>

### Compila il programma

Per compilare il tuo progetto, esegui il seguente comando:

```terminal
$ anchor build
```

Puoi trovare il programma compilato nella directory `/target/deploy`.

<Callout type="warn">

Quando esegui `anchor build`, se riscontri i seguenti errori:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Prova queste soluzioni:

1. Forza l'installazione usando il seguente comando:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Se la soluzione precedente non funziona, cancella la cache di Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Puoi risolvere questo problema modificando il campo version del file `Cargo.lock` da 4 a 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Consulta [questo issue](https://github.com/solana-foundation/anchor/issues/3392)
per ulteriori informazioni.

</Accordion>

</Accordions>

Dopo aver applicato la soluzione precedente, prova a eseguire nuovamente
`anchor build`.

</Callout>

</Step>
<Step>

### Distribuisci il programma

Per distribuire il tuo progetto, esegui il seguente comando:

```terminal
$ anchor deploy
```

Questo comando distribuisce il tuo programma sul `cluster` specificato nel file
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Testa il programma

Per testare il tuo progetto, esegui il seguente comando:

```terminal
$ anchor test
```

Questo comando compila, distribuisce ed esegue i test per il tuo progetto.

<Callout>
  Quando si utilizza il cluster `localnet`, Anchor avvia automaticamente un
  validator locale, distribuisce il programma, esegue i test e poi arresta il
  validator.
</Callout>

<Callout type="warn">

Uno dei seguenti errori potrebbe indicare che non hai Node.js o Yarn installati:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
