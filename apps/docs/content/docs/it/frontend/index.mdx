Lo sviluppo frontend su Solana implica lavorare con programmi, wallet e
framework JavaScript popolari come React. Interagire con questi componenti
richiede la gestione della connessione, la creazione di transazioni e la lettura
dagli account Solana.

[Solana Kit](https://github.com/anza-xyz/kit) (`@solana/kit`) è l'SDK JavaScript
standard per questo scopo. Si assembla un client a partire da piccoli plugin
componibili, si aggiungono i binding per React quando necessario, e si importano
i client di programma generati per i programmi on-chain che si intende
richiamare.

## Librerie principali

<Cards>
  <Card title="@solana/kit" href="https://github.com/anza-xyz/kit">
    - SDK principale: RPC, codec, signer e transazioni - Fornisce
    `createClient`, il plugin host che tutte le altre librerie estendono
  </Card>
  <Card title="Kit plugins" href="https://github.com/anza-xyz/kit-plugins">
    - Componi un client con `.use(...)`: connessione RPC, wallet, signer e
    pianificazione delle transazioni - Pubblicati come pacchetti
    `@solana/kit-plugin-*`
  </Card>
  <Card
    title="@solana/react"
    href="https://github.com/anza-xyz/kit/tree/main/packages/react"
  >
    - Binding React per un client Kit - `ClientProvider` con hook per letture,
    sottoscrizioni e stato del wallet
  </Card>
  <Card title="@solana-program/*" href="https://github.com/solana-program">
    - Client generati per i programmi on-chain (System, Token e altri) -
    Forniscono builder di istruzioni e plugin client come `tokenProgram()`
  </Card>
</Cards>

<Callout type="warn" title="web3.js v1 e wallet-adapter sono legacy">
  Molti progetti Solana si affidano ancora ai più datati `@solana/web3.js` (v1)
  e `@solana/wallet-adapter-*`. Per i nuovi sviluppi, preferire `@solana/kit`
  con i kit plugin indicati di seguito; il rilevamento tramite Wallet Standard
  (tramite `@solana/kit-plugin-wallet`) sostituisce wallet-adapter per i wallet
  moderni.
</Callout>
