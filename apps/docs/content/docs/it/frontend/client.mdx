`@solana/kit` include `createClient`, un piccolo host che si estende tramite
plugin. Ogni plugin aggiunge una funzionalità — una connessione RPC, un
firmatario, la pianificazione delle transazioni — e il client composto supporta
script, route server e UI allo stesso modo.

<Callout type="info">
  Stai costruendo un'app React? La [guida React](/docs/frontend/react-hooks)
  avvolge questo stesso client con `@solana/react` così i componenti leggono lo
  stato del wallet e i dati RPC tramite hook. Inizia da lì e torna qui quando
  hai bisogno di un controllo più granulare.
</Callout>

## Installazione

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

Usa qualsiasi package manager. Kit è v7+ e i pacchetti `@solana/kit-plugin-*`
sono 0.13+.

## Creare un client

Componi i plugin con `.use(...)`. Imposta prima un firmatario, poi la
connessione RPC. `solanaDevnetRpc()` aggiunge `client.rpc`, le sottoscrizioni,
il pianificatore/esecutore di transazioni e il helper `client.sendTransaction` —
così un'app raramente deve collegare questi elementi da sola.

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

Il plugin `signer` imposta sia il fee payer che l'identità dell'app. Il bundle
RPC richiede che un payer sia già configurato, ed è per questo che l'ordine è
importante.

### Varianti del firmatario

`@solana/kit-plugin-signer` espone tre ruoli in modo da concedere solo ciò di
cui un client ha bisogno:

- `signer(x)` — imposta sia `client.payer` che `client.identity`.
- `payer(x)` — solo fee payer.
- `identity(x)` — solo autorità dell'app.

Ognuno ha varianti `*FromFile(path)` (Node), `generated*()`,
`generated*WithSol(amount)` e `airdrop*()` per test e script.

## Inviare una transazione

Costruisci le istruzioni con un client di programma generato, poi passale a
`client.sendTransaction`. Il pianificatore risolve il blockhash, stima il
compute, firma con il payer del client e invia.

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

`sendTransaction` restituisce un oggetto risultato; la firma della transazione
si trova in `result.context.signature`. Verifica che il piano rientri in una
singola transazione e generava un errore in caso contrario — usa
`sendTransactions` quando un batch può estendersi su più transazioni.

## Scegli un bundle RPC

`@solana/kit-plugin-rpc` include preset per cluster:

- `solanaDevnetRpc()` — impostazioni predefinite per devnet, airdrop abilitato
  (`client.airdrop`).
- `solanaMainnetRpc()` — mainnet; l'airdrop genera un errore in fase di
  compilazione.
- `solanaLocalRpc()` — `http://127.0.0.1:8899` per un validator locale.
- `solanaRpc({ rpcUrl })` — qualsiasi endpoint.

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### Composizione granulare

I preset raggruppano la connessione RPC, il planner, l'executor e gli helper per
l'invio, quindi la maggior parte delle applicazioni non ha mai bisogno dei
singoli componenti. Ricorri ai plugin granulari solo per personalizzarne uno —
ad esempio un planner personalizzato o un canale di sottoscrizione diverso.
Questo percorso richiede anche `@solana/kit-plugin-instruction-plan`, che i
preset includono automaticamente:

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## Aggiungi client per i programmi

I pacchetti `@solana-program/*` generati espongono plugin client, così le
chiamate ai programmi sono disponibili direttamente sul client. Installa il
programma di cui hai bisogno e aggiungi il suo plugin con `.use()`:

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="Peer dependency ranges">
  Alcune versioni di `@solana-program/*` dichiarano ancora un peer `@solana/kit
  ^6.x` mentre Kit è alla versione 7.x. Funzionano con Kit 7 — sovrascrivi
  l'intervallo del peer nel tuo package manager se genera errori.
</Callout>

## Firma senza un wallet browser

Stai eseguendo operazioni headless in una route API, un worker o uno script?
Firma con un servizio di gestione delle chiavi backend invece di un keypair
grezzo. Consulta [Keychain](/docs/tools/keychain) e
[Firma in Produzione](/docs/core/transactions/signing-in-production).

## Pattern comuni per gli sviluppatori Solana

- **Setup condiviso**: Costruisci il client una sola volta in un modulo centrale
  e importalo in script, route server e interfaccia utente.
- **Letture lato server**: Un client con solo `signer` + `solanaRpc` (senza
  plugin wallet) legge account e simula transazioni dal server.
- **Test in locale**: Sostituisci il bundle RPC con `.use(litesvm())` da
  `@solana/kit-plugin-litesvm` per eseguire operazioni su un validator in
  memoria (solo Node).
- **Testabilità**: Il client composto è facile da simulare — stub delle risposte
  RPC o dei firmatari senza un wallet browser presente.
