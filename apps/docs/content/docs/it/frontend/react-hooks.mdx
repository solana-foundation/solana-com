  Condividi un client Kit nell'intera app React con @solana/react e gli hook del

`@solana/react` pubblica un client Kit nell'albero dei componenti tramite un
unico provider, dopodiché gli hook leggono da esso i dati RPC, le sottoscrizioni
e lo stato del wallet. Gli hook per il wallet del browser si trovano in
`@solana/kit-plugin-wallet/react` e leggono dallo stesso provider — non è
necessario un provider wallet separato.

## Installazione

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Costruire il client

Costruisci il client una sola volta, a livello di modulo, al di fuori
dell'albero React. Nel browser, `walletSigner` imposta il wallet connesso sia
come fee payer che come identity.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` aggiunge `client.wallet` (rilevamento e connessione Wallet
Standard), e `solanaRpc` aggiunge `client.rpc` più il helper
`client.sendTransaction`. Esporta il tipo del client racchiuso in `Awaited<>` in
modo che rimanga corretto se in seguito aggiungessi un plugin che si risolve in
modo asincrono, e passa quel `AppClient` a ogni chiamata `useClient`.

## Avvolgi il tuo albero una sola volta

`ClientProvider` pubblica il client. Tutti gli hook seguenti lo leggono da esso.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Hook per il wallet

Gli hook per il wallet leggono la capability `wallet` del client, quindi
funzionano solo all'interno di un `ClientProvider` il cui client ha installato
un plugin wallet.

- `useWallets()` — Wallet Standard scoperti per la chain del client.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` o `null`. `signer` è
  `null` per i wallet in sola lettura.
- `useConnect()` / `useDisconnect()` — restituiscono un `ActionResult` il cui
  `dispatch` (e `dispatchAsync`) esegue l'azione e i cui `isRunning`, `status`,
  `data`, `error` e `reset` tracciano il risultato.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Inviare una transazione

`useClient` restituisce lo stesso client pubblicato. Costruisci le istruzioni
con un client di programma generato e inviale tramite `client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Passando il tipo `AppClient` esportato a `useClient` si fornisce all'hook il
client completo esteso dal plugin, così `client.rpc` e `client.sendTransaction`
sono tipizzati.

## Lettura dei dati

`@solana/react` fornisce hook che leggono tramite il client:

- `useRequest(source)` — lettura RPC una tantum; restituisce
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — una sottoscrizione live; restituisce
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — un valore inizializzato via RPC e mantenuto
  aggiornato da una sottoscrizione.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Memoizza la sorgente (con `useMemo`, oppure `useCallback` per una funzione) in
modo che l'hook esegua nuovamente il fetch solo quando cambiano i suoi input,
anziché a ogni render.

Per la cache e la rivalidazione, i sottopercorsi `@solana/react/swr` e
`@solana/react/query` racchiudono gli stessi hook per
[SWR](https://swr.vercel.app) e [TanStack Query](https://tanstack.com/query).

## Pattern comuni per sviluppatori Solana

- **Un solo provider**: `ClientProvider` è l'unico provider di cui hai bisogno;
  gli hook per il wallet lo leggono tramite la capability `wallet` del client.
- **Compatibile con i Server Component**: Contrassegna con `"use client"` solo i
  componenti foglia che chiamano hook; le letture lato server possono usare un
  semplice client RPC di Kit senza wallet.
- **Testing**: Pubblica un client mockato tramite `ClientProvider` per simulare
  wallet e risposte RPC.

<Callout type="warn" title="Due famiglie di hook, stessi nomi">
  `useSignIn` e `useSignMessage` esistono sia in `@solana/react` (accettano un
  argomento `UiWalletAccount`) che in `@solana/kit-plugin-wallet/react` (basati
  sul client, senza argomento account, restituiscono un `ActionResult`). Usa le
  versioni di `@solana/kit-plugin-wallet/react` con il pattern client descritto
  sopra; i vecchi hook wallet di `@solana/react` sono in fase di dismissione.
</Callout>

Abbina questa guida alla [panoramica del client Kit](/docs/frontend/client) per
capire da quale client ogni hook esegue le letture.
