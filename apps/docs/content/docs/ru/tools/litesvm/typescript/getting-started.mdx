# LiteSVM для TypeScript

Тестовые наборы Solana, запускающие `solana-test-validator`, тратят секунды на
каждый тест — вы ждёте запуска процесса, установки RPC-соединений и сетевых
round-trip'ов. LiteSVM выполняет те же тесты за миллисекунды. Без внешних
процессов, без сети — всё в памяти внутри вашего тест-раннера.

<Callout type="info">
  **Протестировано с:** `@solana/kit@6.8.0`,
  `@solana/kit-plugin-litesvm@0.10.0`, `@solana/kit-plugin-signer@0.10.0`,
  `@solana-program/system@0.12.0`, `@solana-program/token@0.13.0`. Эти пакеты
  плагинов новые и активно развиваются — если вы столкнулись с несовместимостью
  версий, проверьте
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  для получения точных версий, с которыми работают документация и тестовый
  набор.
</Callout>

Вы собираете Kit-клиент из `@solana/kit`, устанавливая плагины с помощью
`.use()`: плагин подписанта
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
плагин LiteSVM
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm))
и любые необходимые плагины программ. В результате открывается доступ к
`client.sendTransaction()`, `client.rpc` и прямой доступ к состоянию аккаунта,
часам и sysvar'ам — всё не выходя за пределы вашего процесса.

### Создание клиента

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` возвращает пустую оболочку — каждый `.use(plugin)` добавляет
  к ней новые возможности. Плагин `signer()` должен идти **перед** `litesvm()`,
  поскольку транспортному плагину необходим плательщик для привязки.
  `signer(mySigner)` — это сокращение для установки одного keypair одновременно
  как `payer`, так и `identity` — используйте `payer()` или `identity()` из того
  же пакета, если хотите разделить эти роли. Такой порядок аналогичен
  `solanaRpc()` при работе с реальной сетью.
</Callout>

<Callout type="info">
  Предпочитаете быструю однострочную настройку для экспериментов? Пакет
  `litesvm()` включает airdrop, поэтому вы можете использовать цепочку
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` из
  `@solana/kit-plugin-signer` для автоматической генерации и пополнения аккаунта
  плательщика.
</Callout>

</Step>

</Steps>

## Понимание клиента

После `.use(signer(...)).use(litesvm())` клиент предоставляет:

| Свойство                    | Описание                                                                                                                                                                                                           |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `client.payer`              | `TransactionSigner`, установленный с помощью плагина `signer()` (также задаётся как `client.identity`)                                                                                                             |
| `client.identity`           | Подписант кошелька, которому принадлежат аккаунты, токены и права доступа                                                                                                                                          |
| `client.svm`                | Прямой экземпляр [`LiteSVM`](https://www.npmjs.com/package/litesvm) для управления аккаунтами, транзакциями и конфигурацией                                                                                        |
| `client.rpc`                | RPC-интерфейс, совместимый с Kit (подмножество: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Планирование, подписание и отправка инструкций в виде транзакции                                                                                                                                                   |
| `client.airdrop`            | Запрос SOL из крана LiteSVM                                                                                                                                                                                        |
| `client.getMinimumBalance`  | Вычисление минимального количества lamport для освобождения от rent                                                                                                                                                |

### Использование client.svm

Прямой доступ к LiteSVM для:

- Управление аккаунтами (`setAccount`, `getAccount`, `airdrop`)
- Выполнение транзакций (`sendTransaction`, `simulateTransaction`)
- Конфигурация (`withSigverify`, `withBlockhashCheck` и др.)
- Управление временем (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### Использование client.rpc

RPC-интерфейс, совместимый с Kit, для стандартных операций:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  RPC-слой поддерживает только **кодировку base64**. Запрос других кодировок
  приведёт к ошибке.
</Callout>

## Конфигурация

Настройте LiteSVM с помощью методов-строителей на `client.svm`:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## Отправка транзакций

Подключите программные плагины после транспорта — они добавляют строители
инструкций к клиенту, так что вам никогда не придётся работать с сырым
`TransactionMessage` напрямую:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## Поток SVM → RPC

Вы можете устанавливать состояние через SVM и читать его через RPC:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```
