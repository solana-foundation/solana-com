Эти функции доступны при использовании дополнения SVM для Solana и
SVM-совместимых блокчейнов.

## system_program_id

`svm::system_program_id` возвращает идентификатор System Program,
`11111111111111111111111111111111`.

```hcl
output "system_program_id" {
    value = svm::system_program_id()
}
// > 11111111111111111111111111111111
```

## get_instruction_data_from_idl_path

`svm::get_instruction_data_from_idl_path` создаёт закодированные instruction
data для вызова программы, обеспечивая проверку типов и сериализацию на основе
указанного файла IDL.

### Входные параметры

| Name               | Required | Type          | Description                                           |
| ------------------ | -------- | ------------- | ----------------------------------------------------- |
| `idl_path`         | required | string        | Путь к файлу IDL `.json` относительно txtx.yml        |
| `instruction_name` | required | string        | Название инструкции, для которой генерируются данные  |
| `arguments`        | optional | array[string] | Аргументы инструкции, для которых генерируются данные |

```hcl
output "data" {
    value = svm::get_instruction_data_from_idl("/path/to/idl.json", "my_instruction", ["arg1", "arg2"])
}
// > data: 0x95763bdcc47fa1b305000000776f726c64
```

## get_instruction_data_from_idl

`svm::get_instruction_data_from_idl` создаёт закодированные instruction data для
вызова программы, обеспечивая проверку типов и сериализацию на основе
предоставленных данных IDL.

### Входные параметры

| Name               | Required | Type              | Description                                                                            |
| ------------------ | -------- | ----------------- | -------------------------------------------------------------------------------------- |
| `idl`              | required | string \| SVM_IDL | IDL программы                                                                          |
| `instruction_name` | required | string            | Название инструкции, для которой генерируются данные, в соответствии с индексацией IDL |
| `arguments`        | optional | array[string]     | Аргументы инструкции, для которых генерируются данные                                  |

```hcl
output "data" {
    value = svm::get_instruction_data_from_idl(variable.contract.idl, "my_instruction", ["arg1", "arg2"])
}
// > data: 0x95763bdcc47fa1b305000000776f726c64
```

## get_program_from_anchor_project

`svm::get_program_from_anchor_project` извлекает артефакты развёртывания
программы для программы в проекте Anchor.

### Входные параметры

| Название       | Обязательный   | Тип    | Описание                               |
| -------------- | -------------- | ------ | -------------------------------------- |
| `program_name` | обязательный   | string | Название развёртываемой программы      |
| `keypair_path` | необязательный | string | Расположение файла keypair программы   |
| `idl_path`     | необязательный | string | Расположение файла IDL программы       |
| `bin_path`     | необязательный | string | Расположение бинарного файла программы |

```hcl
variable "contract" {
    value = svm::get_program_from_anchor_project("my_program")
}
output "idl" {
    value = variable.contract.idl
}
```

## sol_to_lamports

`svm::sol_to_lamports` конвертирует указанное количество SOL в lamport.

```hcl
output "lamports" {
    value = svm::sol_to_lamports(1.1)
}
// lamports: 1100000000
```

`svm::lamports_to_sol` конвертирует указанное количество lamport в SOL.

```hcl
    value = svm::lamports_to_sol(1100000000)

```

## find_pda

`svm::find_pda` находит корректный PDA, используя предоставленный идентификатор
программы и seeds.

### Входные параметры

| Название     | Обязательный   | Тип           | Описание                                                   |
| ------------ | -------------- | ------------- | ---------------------------------------------------------- |
| `program_id` | обязательный   | string        | Адрес программы, из которой выведен PDA                    |
| `seeds`      | необязательный | array[string] | Необязательный массив seeds (макс. 16, по 32 байта каждый) |

```hcl
variable "pda" {
    value = svm::find_pda("3bv3j4GvMPjvvBX9QdoX27pVoWhDSXpwKZipFF1QiVr6", ["data"])
}
output "pda" {
    value = std::encode_base58(variable.pda.pda)
}
output "bump" {
    value = variable.pda.bump_seed
}
// > pda: 4amHoWMBgLkPfM8Nq9ZP33Liq9FCuqrLoU1feejkdsUJ
// > bump: 252
```

`svm::get_associated_token_account` вычисляет адрес associated token account для

| `token_mint_address` | optional | string | Адрес токен-минта |

```hcl
variable "token_account" {
    value = svm::get_associated_token_account(signer.caller.address, "So11111111111111111111111111111111111111112")
}
```

## create_token_account_instruction

`svm::create_token_account_instruction` создаёт необработанные байты инструкции
для создания associated token account.

### Входные данные

| Name                 | Required | Type   | Description                   |
| -------------------- | -------- | ------ | ----------------------------- |
| `funding_address`    | required | string | Адрес финансирующего аккаунта |
| `wallet_address`     | required | string | Адрес кошелька                |
| `token_mint_address` | optional | string | Адрес токен-минта             |
| `token_program_id`   | optional | string | Адрес token program           |

```hcl
action "call" "svm::process_instructions" {
    signers = [signer.caller]

    instruction {
        raw_bytes = svm::create_token_account_instruction(
            signer.caller.address, // funding address
            signer.caller.address, // wallet address
            variable.token_mint, // token mint address
            variable.token_program // token program id
        )
    }
}
```

## u64

`svm::u64` создаёт представление целого числа u64 в виде массива байтов,
подходящего для использования в качестве seed при выводе PDA.

### Входные данные

| Name    | Required | Type    | Description                                        |
| ------- | -------- | ------- | -------------------------------------------------- |
| `value` | required | integer | Целое число u64 для преобразования в массив байтов |

```hcl
variable "pda" {
    value = svm::find_pda("3bv3j4GvMPjvvBX9QdoX27pVoWhDSXpwKZipFF1QiVr6", [svm::u64(1000000000)])
}
```
