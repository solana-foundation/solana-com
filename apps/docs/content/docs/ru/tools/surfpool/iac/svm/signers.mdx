Эти подписанты доступны при использовании SVM-аддона для подписания транзакций.

## secret_key

Подписант `svm::secret_key` может использоваться для синхронного подписания
транзакции с помощью секретного ключа, мнемонической фразы или файла keypair.

### Входные данные

| Название          | Обязательно | Тип    | Описание                                                           |
| ----------------- | ----------- | ------ | ------------------------------------------------------------------ |
| `secret_key`      | опционально | string | Секретный ключ, используемый для подписания сообщений и транзакций |
| `mnemonic`        | опционально | string | Мнемоническая фраза для генерации секретного ключа                 |
| `derivation_path` | опционально | string | Путь деривации для генерации секретного ключа                      |
| `keypair_json`    | опционально | string | Путь к файлу keypair.json, содержащему секретный ключ              |

### Выходные данные

| Название     | Тип    | Описание                             |
| ------------ | ------ | ------------------------------------ |
| `public_key` | string | Публичный ключ аккаунта              |
| `address`    | string | SVM-адрес (псевдоним для public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Также можно использовать мнемоническую фразу:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Или файл keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

Подписант `svm::web_wallet` позволяет оператору Runbook подписать транзакцию с
помощью браузерного подписанта на его выбор.

### Входные данные

| Название           | Обязательно | Тип    | Описание                                                                   |
| ------------------ | ----------- | ------ | -------------------------------------------------------------------------- |
| `expected_address` | опционально | string | Ожидаемый SVM-адрес для подключения (не указывайте, чтобы разрешить любой) |

### Выходные данные

| Название     | Тип    | Описание                              |
| ------------ | ------ | ------------------------------------- |
| `address`    | string | Адрес подключённого аккаунта          |
| `public_key` | string | Публичный ключ подключённого аккаунта |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

При использовании этого подписанта пользователю будет предложено в
веб-интерфейсе:

1. Подключить кошелёк
2. Предоставить pubkey через подпись сообщения
3. Подписать все транзакции, требующие этого подписанта

## squads

Подписант `svm::squads` можно использовать для подписания транзакций с помощью
мультисиг-хранилища Squads.

### Входные данные

| Название  | Обязательно | Тип    | Описание              |
| --------- | ----------- | ------ | --------------------- |
| `address` | опционально | string | Адрес хранилища squad |

### Выходные данные

| Название     | Тип    | Описание                             |
| ------------ | ------ | ------------------------------------ |
| `public_key` | string | Pubkey хранилища squad               |
| `address`    | string | SVM-адрес (псевдоним для public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Использование мультисига Squads обеспечивает безопасное подписание транзакций
несколькими сторонами без раскрытия индивидуальных приватных ключей.
