**Solana Keychain предоставляет единый интерфейс для подписи транзакций Solana
через несколько бэкендов управления ключами.** Ведите локальную разработку с
ключом в памяти, затем переключайтесь на корпоративное управление ключами,
облачные HSM или управляемые кошельки в продакшене — без переписывания кода
подписи.

### Почему Keychain?

- **Единый интерфейс**: Один трейт `SolanaSigner` работает со всеми бэкендами
- **Замена бэкенда**: Смена провайдеров управления ключами через конфигурацию, а
  не через код
- **Абстракция без накладных расходов**: Флаги функций включают только то, что
  нужно
- **Аудит**: Проверка безопасности от [Accretion](https://accretion.xyz)

### Архитектура

- **Языки**: Rust + TypeScript, с полным паритетом
- **Трейт**: Унифицированный интерфейс `SolanaSigner`
- **Бэкенды**: Memory, Vault, AWS KMS, GCP KMS, Privy, Turnkey, Fireblocks, CDP,
  Crossmint, Dfns, Openfort, Para, Utila
- **Совместимость**: `@solana/kit` и `@solana/signers` совместимо (TypeScript) |
  `solana-sdk` и `solana-sdk-v3` совместимо (Rust)

### Поддерживаемые системы

Бэкенды охватывают локальную разработку, автономное и облачное управление
ключами, а также управляемые кошельки и сервисы MPC. См.
[Выбор бэкенда](/docs/tools/keychain/choosing-a-backend) для сравнения моделей
хранения и выбора подходящего варианта.

| Бэкенд          | Сценарий использования             | Rust | TypeScript |
| --------------- | ---------------------------------- | ---- | ---------- |
| Memory          | Разработка, тестирование           | ✓    | ✓          |
| HashiCorp Vault | Автономное управление ключами      | ✓    | ✓          |
| AWS KMS         | Облачный (AWS)                     | ✓    | ✓          |
| GCP KMS         | Облачный (GCP)                     | ✓    | ✓          |
| Privy           | Встроенные кошельки                | ✓    | ✓          |
| Turnkey         | Некастодиальное управление ключами | ✓    | ✓          |
| Fireblocks      | Институциональное хранение         | ✓    | ✓          |
| CDP             | Coinbase Developer Platform        | ✓    | ✓          |
| Crossmint       | Управляемые кошельки Crossmint     | ✓    | ✓          |
| Dfns            | Инфраструктура кошельков Dfns      | ✓    | ✓          |

## Быстрый старт

### Rust

Установите Rust crate:

```bash
cargo add solana-keychain
```

Базовое использование:

```rust
use solana_keychain::{Signer, SolanaSigner};

// Create a signer from any backend
let signer = Signer::from_memory("base58_private_key")?;

// All signers share the same interface
let pubkey = signer.pubkey();
let signature = signer.sign_transaction(&mut tx).await?;
```

- [Руководство по Rust](/docs/tools/keychain/getting-started/rust) - Полная
  установка и настройка бэкенда
- [Crates.io](https://crates.io/crates/solana-keychain) - Rust crate

### TypeScript

Установите пакет TypeScript:

```bash
pnpm add @solana/keychain
```

Базовое использование:

```typescript
import { createKeychainSigner } from "@solana/keychain";
import { signTransactionWithSigners } from "@solana/signers";

// Create any signer via the unified factory
const signer = await createKeychainSigner({
  backend: "vault",
  vaultAddr: "https://vault.example.com:8200",
  vaultToken: "hvs.xxxxx",
  keyName: "my-solana-key",
  publicKey: "base58_public_key"
});

// Sign an already-compiled transaction
const signedTx = await signTransactionWithSigners(
  [signer],
  compiledTransaction
);
```

- [Руководство по TypeScript](/docs/tools/keychain/getting-started/typescript) -
  пакеты `@solana/keychain`
- [npm Package](https://www.npmjs.com/package/@solana/keychain) - npm-пакет
