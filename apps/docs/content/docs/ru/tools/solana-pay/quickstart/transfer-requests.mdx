---
title: Запросы на перевод
description: Создавайте простые URL-адреса для платежей в SOL и SPL-токенах
---

Запросы на перевод — это простейшая форма интеграции Solana Pay. Они создают
неинтерактивные URL-адреса для платежей, которые кошельки могут использовать для
прямого формирования и отправки транзакций без необходимости в сервере.

## Обзор

URL-адрес запроса на перевод имеет следующий формат:

```
solana:<recipient>
      ?amount=<amount>
      &spl-token=<spl-token>
      &reference=<reference>
      &label=<label>
      &message=<message>
      &memo=<memo>
```

## Базовый перевод SOL

Создайте простой запрос на оплату в SOL:

```typescript
import { address } from "@solana/kit";
import { encodeURL } from "@solana/pay";

// Recipient's Solana wallet address
const recipient = address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB");

// Create the payment URL
const url = encodeURL({
  recipient,
  amount: 0.01, // 0.01 SOL
  label: "My Store",
  message: "Thanks for your purchase!"
});

console.log(url.toString());
// Output: solana:FvJ8k8Hh...?amount=0.01&label=My%20Store&message=Thanks%20for%20your%20purchase!
```

## Перевод SPL-токенов

Создайте запрос на оплату в USDC:

```typescript
import { address } from "@solana/kit";
import { encodeURL } from "@solana/pay";

// USDC mint address on mainnet
const usdcMint = address("EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v");

const url = encodeURL({
  recipient: address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB"),
  amount: 10.5, // $10.50 USDC
  splToken: usdcMint,
  label: "Coffee Shop",
  message: "Grande Latte + Tip"
});

console.log(url.toString());
```

## Параметры URL

### Обязательные параметры

#### Получатель

Адрес кошелька Solana получателя (в кодировке base58):

```typescript
const recipient = address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB");
```

### Необязательные параметры

#### Сумма

Сумма платежа в удобных для пользователя единицах (SOL или единицы токена):

```typescript
// SOL amounts
const solAmount = 1.5; // 1.5 SOL
const smallAmount = 0.001; // 0.001 SOL

// Token amounts (USDC example)
const usdcAmount = 25.99; // $25.99 USDC
```

Если не указана, кошелек предложит пользователю ввести сумму.

#### SPL-токен

Для переводов токенов укажите адрес минта:

```typescript
// Common SPL tokens
const USDC = address("EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v");
const USDT = address("Es9vMFrzaCERmJfrF4H2FYD4KCoNkY11McCe8BenwNYB");
// Omit splToken for native SOL transfers
```

#### Ссылка

Уникальные идентификаторы для отслеживания платежей. Сгенерируйте адрес для
использования в качестве ключа ссылки:

```typescript
import { generateKeyPair, getAddressFromPublicKey } from "@solana/kit";

// Generate a unique reference
const keypair = await generateKeyPair();
const reference = await getAddressFromPublicKey(keypair.publicKey);
```

#### Метка

Название продавца или приложения, отображаемое пользователям:

```typescript
const label = "Acme Coffee Shop";
```

#### Сообщение

Описание платежа:

```typescript
const message = "Order #12345 - 2x Americano";
```

#### Примечание

Примечание в блокчейне (публично видимое):

```typescript
const memo = "OrderId:12345";
```

## Интеграция QR-кода

Генерируйте QR-коды с помощью встроенной функции `createQR`:

```typescript
import { address } from "@solana/kit";
import { encodeURL, createQR } from "@solana/pay";

const url = encodeURL({
  recipient: address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB"),
  amount: 0.05,
  label: "Donation",
  message: "Support our project"
});

// Generate a QR code element
const qr = createQR(url);

// Append to DOM
const element = document.getElementById("qr-code");
qr.append(element);
```

## Проверка платежа

### Использование клиента продавца

Самый простой способ найти и проверить платежи:

```typescript
import { address } from "@solana/kit";
import { createMerchantClient } from "@solana/pay";

const merchant = createMerchantClient({
  rpcUrl: "https://api.mainnet-beta.solana.com"
});

const recipient = address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB");
const reference = address("YOUR_REFERENCE_ADDRESS");

// Find the transaction by reference
const found = await merchant.pay.findReference(reference);

// Validate the transfer matches expected fields
await merchant.pay.validateTransfer(found.signature, {
  recipient,
  amount: 0.1
});

console.log("Payment confirmed!", found.signature);
```

### Использование автономных функций

```typescript
import { createSolanaRpc, address } from "@solana/kit";
import { findReference, validateTransfer } from "@solana/pay";

const rpc = createSolanaRpc("https://api.mainnet-beta.solana.com");
const reference = address("YOUR_REFERENCE_ADDRESS");

// Find transaction by reference
const found = await findReference(rpc, reference);

// Validate transaction details
await validateTransfer(rpc, found.signature, {
  recipient: address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB"),
  amount: 0.1
});

console.log("Payment confirmed!", found.signature);
```

### Отслеживание платежей (WebSocket)

Рекомендуемый подход — `watchReference` подписывается через WebSocket и
разрешается, как только поступает соответствующая транзакция. Опрос не
требуется.

```typescript
async function waitForPayment(reference, recipient, amount) {
  const merchant = createMerchantClient({
    rpcUrl: "https://api.mainnet-beta.solana.com"
  });

  // Resolves on the first transaction mentioning this reference
  const result = await merchant.pay.watchReference(reference);

  if (result.err) {
    throw new Error(`Transaction failed: ${JSON.stringify(result.err)}`);
  }

  // Validate that the transaction matches the expected payment
  await merchant.pay.validateTransfer(result.signature, { recipient, amount });

  return { signature: result.signature };
}
```

### Опрос платежей (HTTP резервный вариант)

Если подписки WebSocket недоступны (например, у некоторых RPC-провайдеров),
используйте опрос с помощью `findReference`:

```typescript
async function pollForPayment(reference, recipient, amount) {
  const merchant = createMerchantClient({
    rpcUrl: "https://api.mainnet-beta.solana.com"
  });

  const checkPayment = async () => {
    try {
      const found = await merchant.pay.findReference(reference);
      await merchant.pay.validateTransfer(found.signature, {
        recipient,
        amount
      });
      return { success: true, signature: found.signature };
    } catch {
      return { success: false };
    }
  };

  // Check every 2 seconds
  return new Promise((resolve) => {
    const interval = setInterval(async () => {
      const result = await checkPayment();
      if (result.success) {
        clearInterval(interval);
        resolve(result);
      }
    }, 2000);
  });
}
```

## Полный пример

Вот полный пример с генерацией QR-кода и мониторингом платежей:

```typescript
import { address, generateKeyPair, getAddressFromPublicKey } from "@solana/kit";
import { encodeURL, createQR, createMerchantClient } from "@solana/pay";

const recipient = address("FvJ8k8HhXp4a3zQyFMZd4FvEqcYdYE7gSZWxrEBRfBsB");
const amount = 0.1;

// Generate unique reference for this payment
const keypair = await generateKeyPair();
const reference = await getAddressFromPublicKey(keypair.publicKey);

// Create payment URL and QR code
const url = encodeURL({
  recipient,
  amount,
  reference,
  label: "Demo Store",
  message: "Test Purchase"
});
const qr = createQR(url);
qr.append(document.getElementById("qr-code"));

// Monitor for payment
const merchant = createMerchantClient({
  rpcUrl: "https://api.mainnet-beta.solana.com"
});

const interval = setInterval(async () => {
  try {
    const found = await merchant.pay.findReference(reference);
    await merchant.pay.validateTransfer(found.signature, {
      recipient,
      amount,
      reference
    });
    clearInterval(interval);
    console.log("Payment confirmed!", found.signature);
  } catch {
    // Payment not found yet, keep polling
  }
}, 2000);
```

## Лучшие практики

### Безопасность

- Всегда проверяйте платежи на стороне сервера
- Используйте уникальные ссылки для каждого платежа
- Проверяйте суммы транзакций и получателей
- Проверяйте статус успешности транзакции

### Пользовательский опыт

- Показывайте четкие суммы платежей и описания
- Предоставляйте обновления статуса платежа
- Обрабатывайте подключение кошелька корректно
- Поддерживайте как QR-коды, так и прямые ссылки

### Обработка ошибок

- Обрабатывайте сбои подключения кошелька
- Предоставляйте резервные варианты для неподдерживаемых кошельков
- Показывайте понятные сообщения об ошибках
- Реализуйте тайм-ауты платежей
