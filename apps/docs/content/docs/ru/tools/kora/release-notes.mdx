## Kora v2.2.0-beta.6

_23.03.2026_ |
[Релиз на GitHub](https://github.com/solana-foundation/kora/releases/tag/v2.2.0-beta.6)
| `cargo install kora-cli@2.2.0-beta.6`

Предварительный выпуск с поддержкой Jito-пакетов, новыми RPC-методами и
расширенными функциями безопасности.

- Добавлена поддержка Jito-пакетов с методами `signBundle`, `signAndSendBundle`
  и `estimateBundleFee`
- Добавлен RPC-метод `getVersion`
- Добавлена защита плательщика комиссий Lighthouse
- Добавлена защита от ботов reCAPTCHA v3 для эндпоинтов подписи
- Добавлены лимиты использования для каждого кошелька с поддержкой Redis
- Добавлены официальные Docker-образы в GHCR
- Добавлены детализированные правила лимитов использования
- SDK: `KoraClient` теперь принимает объект options; новый `koraPlugin` для
  `@solana/kit`

См. [Документацию по бета-версии](/docs/tools/kora/beta) для подробностей.

## Kora TypeScript SDK v0.2.0

_09.03.2026_ |
[Релиз на GitHub](https://github.com/solana-foundation/kora/releases/tag/ts-sdk-v0.2.0)
| `pnpm add @solana/kora@0.2.0`

Архитектура плагинов Kit-клиента для компонуемых транзакций без комиссий.

- Добавлен `createKitKoraClient()` с композицией плагинов Kit (плагины
  планировщика, исполнителя, плательщика)
- Обеспечивает компонуемость с плагинами программ Kit, такими как
  `tokenProgram()`
- Добавлена поддержка Token-2022 через параметр конфигурации `tokenProgramId`
- Добавлена поддержка инструкций бюджета вычислений с оценкой CU на основе
  симуляции
- Добавлено недостающее поле `signature` в ответ `signAndSendTransaction`
- Возвращается `NoopSigner` из `getPaymentInstruction` для дедупликации
  подписантов Kit v6
- Обновлена зависимость `@solana/kit` до `^6.1.0`

См. [Руководство по Kit Client](/docs/tools/kora/guides/kit-client).

## Kora v2.0.0

_19.11.2025_ |
[Релиз на GitHub](https://github.com/solana-foundation/kora/releases/tag/v2.0.0)
| `cargo install kora-cli@2.0.0`

Мажорный релиз с новой архитектурой.

- Представлена новая архитектура ноды Kora
- Опубликован TypeScript SDK `@solana/kora` v0.1.0
- Выполнена миграция на Solana SDK v3.0.x и обновлены все зависимости
- Пользовательский Kora Signer заменён на крейт solana-signers
- Удалены методы `sign_transaction_if_paid` и `get_all_signers`
- Изменены типы ответов: подписи удалены из `signAndSendTransaction` и
  `signTransaction`
- Улучшены опыт разработчиков, безопасность и производительность
