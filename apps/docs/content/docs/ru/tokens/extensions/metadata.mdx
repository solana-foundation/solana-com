## Что такое расширения Metadata Pointer и Token Metadata?

Расширение минта _rs`MetadataPointer`_ программы Token Extensions Program хранит
два поля в минте:

- Полномочие, которое может обновлять указатель
- Адрес аккаунта, хранящего метаданные токена

Этот указатель может ссылаться на любой аккаунт, принадлежащий программе,
реализующей
[интерфейс token-metadata](https://github.com/solana-program/token-metadata/tree/main/interface).

Программа Token Extensions Program также напрямую реализует этот же интерфейс
через расширение минта _rs`TokenMetadata`_. С _rs`TokenMetadata`_ минт хранит
_rs`name`_, _rs`symbol`_, _rs`uri`_, _rs`update_authority`_ и пользовательские
метаданные непосредственно в аккаунте минта.

<Callout type="info" title="URI метаданных вне блокчейна">
  Поле _rs`uri`_ указывает на JSON-метаданные вне блокчейна. См. [Формат
  метаданных вне блокчейна](/docs/tokens/metaplex#off-chain-metadata-format).
</Callout>

Эти два расширения решают разные задачи:

- _rs`MetadataPointer`_ указывает аккаунт, который хранит метаданные.
- _rs`TokenMetadata`_ хранит метаданные непосредственно в аккаунте минта.

<Callout type="info" title="Расширение переменной длины">
  _rs`TokenMetadata`_ является расширением TLV переменной длины.
  _rs`InitializeTokenMetadata`_, _rs`UpdateField`_ и _rs`RemoveKey`_ могут
  изменять размер данных аккаунта минта, но они не переводят дополнительные
  лампорты. Аккаунт минта должен иметь достаточно лампортов, чтобы оставаться
  освобождённым от арендной платы для хранимых метаданных. Если метаданные позже
  увеличатся, дополнительные лампорты необходимо перевести в аккаунт минта перед
  инструкцией, которая изменяет его размер.
</Callout>

## Как хранить метаданные токена в аккаунте минта

Для хранения метаданных в аккаунте минта:

1. Рассчитайте размер mint account и rent, необходимый для минта, расширений и
   метаданных.
2. Создайте mint account с помощью _rs`CreateAccount`_, инициализируйте
   _rs`MetadataPointer`_ и инициализируйте минт с помощью _rs`InitializeMint`_.
3. Инициализируйте _rs`TokenMetadata`_ на mint account, затем используйте
   _rs`UpdateField`_ для добавления или обновления метаданных.
4. Используйте _rs`RemoveKey`_, _rs`UpdateAuthority`_ и _rs`Emit`_ для удаления
   пользовательских метаданных, изменения или очистки полномочий на обновление
   или возврата текущих метаданных в данных возврата транзакции.

## !!steps Расчет размера аккаунта

Рассчитайте размер mint account для базового минта плюс расширение
_rs`MetadataPointer`_. Это размер, используемый в _rs`CreateAccount`_.

<CodePlaceholder title="Пример" />

```ts !! title="Example"
const client = await createLocalClient();
const mint = await generateKeyPairSigner();

// !focus(1:4)
const metadataPointerExtension = extension("MetadataPointer", {
  authority: client.payer.address,
  metadataAddress: mint.address
});

// !focus(1:1)
const mintSpace = BigInt(getMintSize([metadataPointerExtension]));
```

## !!steps Расчет rent

Рассчитайте rent, используя максимальный размер, необходимый после сохранения
_rs`TokenMetadata`_ на минте.

  authority: client.payer.address,

// !focus(1:16)

  updateAuthority: client.payer.address,
  mint: mint.address,
  name: "Example Token v2",
  symbol: "EXMPL",
  uri: "https://example.com/token.json",
  additionalMetadata: new Map([
    ["description", "Metadata stored on mint account"]
  ])

  getMintSize([metadataPointerExtension, maxTokenMetadataExtension])

  .getMinimumBalanceForRentExemption(maxMintSpace)

## !!steps Обновление, удаление или получение метаданных

После инициализации mint используйте _rs`UpdateField`_ для обновления
метаданных, _rs`UpdateMetadataPointer`_ для обновления указателя метаданных,
_rs`RemoveKey`_ для удаления пользовательских метаданных, _rs`UpdateAuthority`_
для очистки полномочий обновления и _rs`Emit`_ для возврата текущих метаданных.

<CodePlaceholder title="Пример" />

```ts !! title="Example"
const client = await createLocalClient();
const mint = await generateKeyPairSigner();

const metadataPointerExtension = extension("MetadataPointer", {
  authority: client.payer.address,
  metadataAddress: mint.address
});

const mintSpace = BigInt(getMintSize([metadataPointerExtension]));

const maxTokenMetadataExtension = extension("TokenMetadata", {
  updateAuthority: client.payer.address,
  mint: mint.address,
  name: "Example Token v2",
  symbol: "EXMPL",
  uri: "https://example.com/token.json",
  additionalMetadata: new Map([
    ["description", "Metadata stored on mint account"]
  ])
});
const maxMintSpace = BigInt(
  getMintSize([metadataPointerExtension, maxTokenMetadataExtension])
);
const mintRent = await client.rpc
  .getMinimumBalanceForRentExemption(maxMintSpace)
  .send();

await client.sendTransaction([
  getCreateAccountInstruction({
    payer: client.payer, // Account funding account creation.
    newAccount: mint, // New mint account to create.
    lamports: mintRent, // Lamports funding the mint account rent.
    space: mintSpace, // Account size in bytes for the mint plus MetadataPointer.
    programAddress: TOKEN_2022_PROGRAM_ADDRESS // Program that owns the mint account.
  }),
  getInitializeMetadataPointerInstruction({
    mint: mint.address, // Mint account that stores the MetadataPointer extension.
    authority: client.payer.address, // Authority allowed to update the metadata pointer later.
    metadataAddress: mint.address // Account address that stores the metadata.
  }),
  getInitializeMintInstruction({
    mint: mint.address, // Mint account to initialize.
    decimals: 0, // Number of decimals for the token.
    mintAuthority: client.payer.address, // Authority allowed to mint new tokens.
    freezeAuthority: client.payer.address // Authority allowed to freeze token accounts.
  }),
  getInitializeTokenMetadataInstruction({
    metadata: mint.address, // Mint account that stores the metadata.
    updateAuthority: client.payer.address, // Authority allowed to update metadata later.
    mint: mint.address, // Mint that the metadata describes.
    mintAuthority: client.payer, // Signer authorizing metadata initialization for the mint.
    name: "Example Token", // Token name stored in metadata.
    symbol: "EXMPL", // Token symbol stored in metadata.
    uri: "https://example.com/token.json" // URI pointing to off-chain JSON metadata.
  }),
  getUpdateTokenMetadataFieldInstruction({
    metadata: mint.address, // Mint account that stores the metadata.
    updateAuthority: client.payer, // Signer authorized to update metadata fields.
    field: tokenMetadataField("Key", ["description"]), // Custom metadata field to add.
    value: "Metadata stored on mint account" // Value stored for the custom metadata field.
  })
]);

// !focus(1:31)
await client.sendTransaction([
  getUpdateTokenMetadataFieldInstruction({
    metadata: mint.address,
    updateAuthority: client.payer,
    field: tokenMetadataField("Name"),
    value: "Example Token v2"
  }),
  getUpdateMetadataPointerInstruction({
    mint: mint.address,
    metadataPointerAuthority: client.payer,
    metadataAddress: mint.address
  }),
  getRemoveTokenMetadataKeyInstruction({
    metadata: mint.address,
    updateAuthority: client.payer,
    key: "description"
  }),
  getUpdateTokenMetadataUpdateAuthorityInstruction({
    metadata: mint.address,
    updateAuthority: client.payer,
    newUpdateAuthority: null
  }),
  getEmitTokenMetadataInstruction({
    metadata: mint.address
  })
]);
```

</ScrollyCoding>

<Callout type="info" title="Порядок инструкций">
  _rs`MetadataPointerInstruction::Initialize`_ должна быть выполнена перед
  _rs`InitializeMint`_. _rs`CreateAccount`_,
  _rs`MetadataPointerInstruction::Initialize`_ и _rs`InitializeMint`_ должны
  быть включены в одну и ту же транзакцию.
</Callout>
