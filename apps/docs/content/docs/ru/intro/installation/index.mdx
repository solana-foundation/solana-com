Быстрая установка для настройки локальной среды разработки Solana одной
командой. Установите Rust, Solana CLI и Anchor Framework на Windows (WSL), Linux
и Mac. (Если предпочитаете, вы можете
[установить каждую зависимость отдельно](/docs/intro/installation/dependencies).)

## Необходимые условия

<Accordions>
<Accordion title="Windows">

Для разработки программ Solana на Windows **необходимо** использовать
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). После установки WSL все остальные зависимости будут устанавливаться через
терминал Linux.

Чтобы установить WSL, выполните следующую команду в Windows PowerShell:

```terminal
$ wsl --install
```

В процессе установки вам будет предложено создать основной аккаунт пользователя.

![Установка WSL](/assets/docs/intro/installation/wsl-install.png)

По умолчанию WSL устанавливает Ubuntu. Открыть терминал Linux можно, найдя
"Ubuntu" в строке поиска.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Если ваш терминал Ubuntu выглядит как на изображении ниже, вы можете столкнуться
с проблемой, когда `ctrl + v` (горячая клавиша вставки) не работает в терминале.

![Терминал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Если вы столкнулись с этой проблемой, откройте Windows Terminal, найдя
"Terminal" в строке поиска.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Затем закройте Windows Terminal и снова откройте терминал Linux, найдя Ubuntu.
Теперь терминал должен выглядеть как на изображении ниже, и `ctrl + v` (горячая
клавиша вставки) будет работать.

![Терминал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Если вы используете VS Code,
[расширение WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
позволяет использовать WSL и VS Code вместе.

![Настройка WSL в VS Code](/assets/docs/intro/installation/wsl-vscode.png)

В строке состояния VS Code вы увидите следующее:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

После настройки WSL вы готовы либо:

- Завершить [быструю установку](#installation) с помощью терминала Ubuntu
  (Linux)
- Установить [Rust](/docs/intro/installation/dependencies#install-rust),
  [Solana CLI](/docs/intro/installation/dependencies#install-solana-cli) и
  [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli) по
  отдельности, как описано ниже

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Если вы используете Debian или производные, такие как Ubuntu или Mint, выполните следующие шаги:

1. Обновите списки пакетов, чтобы убедиться, что все пакеты актуальны:

```terminal
$ sudo apt-get update
```

2. Установите следующие зависимости:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Если при установке `protobuf-compiler` возникает ошибка,
убедитесь, что вы сначала выполнили _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

    Для дистрибутивов на базе RHEL, таких как Fedora или Red Hat Linux, выполните следующие шаги.

Обновите списки пакетов, чтобы получить более новые версии, выполнив команду.


## Установка

1. Введите следующую команду в терминал, чтобы установить все необходимые
   зависимости:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Успешная установка вернёт вывод, похожий на следующий:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. Проверьте успешность установки, проверив версию каждой установленной
   зависимости.

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  Если команда быстрой установки не сработала, пожалуйста, обратитесь к разделу
  [Установка зависимостей](/docs/intro/installation/dependencies) ниже, чтобы
  узнать, как установить каждую зависимость вручную.
</Callout>
