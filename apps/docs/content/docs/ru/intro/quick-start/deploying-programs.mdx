h1: Деплой вашей первой программы Solana

Создайте, задеплойте и протестируйте свою первую
[программу](/docs/core/programs) в блокчейне Solana с использованием фреймворка
Anchor. Это руководство поможет вам лучше познакомиться с Solana Playground и
подготовит к использованию адресов, производных от программ (PDA), и
межпрограммных вызовов (CPI).

<Steps>

<Step>

### Создание проекта Anchor

1. Откройте [Solana Playground](https://beta.solpg.io) в вашем браузере
2. В левой панели нажмите **+ Создать новый проект**
3. Введите **Имя проекта**
4. В качестве фреймворка выберите **Anchor (Rust)**
5. Нажмите **Создать**

![Новый проект](/assets/docs/intro/quickstart/pg-new-project.gif)

Откроется исходный файл `src/lib.rs`. В нём содержится одна структура с именем
`Initialize`, которая выполняет следующее:

- Принимает обязательный параметр _rs`data: u64`_
- Создаёт новый аккаунт
- Сохраняет значение _rs`data: u64`_ в новый аккаунт
- Добавляет сообщение в логи программы транзакции

<WithNotes>

```rust title="lib.rs"
use anchor_lang::prelude::*;

// This is your program's public key and it will update
// automatically when you build the project.
// !tooltip[/declare_id/] declare_id
declare_id!("11111111111111111111111111111111");

// !tooltip[/program/] program
#[program]
mod hello_anchor {
    use super::*;
    // !tooltip[/data: u64/] parameter
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !tooltip[/ctx.accounts.new_account.data = data/] initialize
        ctx.accounts.new_account.data = data;
        // !tooltip[/msg!/] msg
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}

// !tooltip[/Accounts/] accounts
#[derive(Accounts)]
pub struct Initialize<'info> {
    // We must specify the space in order to initialize an account.
    // First 8 bytes are default account discriminator,
    // next 8 bytes come from NewAccount.data being type u64.
    // (u64 = 64 bits unsigned integer = 8 bytes)
    // !tooltip[/account/] constraints
    #[account(
      // !tooltip[/init/] init
      init,
      // !tooltip[/payer/] payer
      payer = signer,
      // !tooltip[/space/] space
      space = 8 + 8
    )]
    pub new_account: Account<'info, NewAccount>,
    // !tooltip[/account/] constraints
    #[account(mut)]
    pub signer: Signer<'info>,
    pub system_program: Program<'info, System>,
}

// !tooltip[/account/] account
#[account]
pub struct NewAccount {
    data: u64
}
```

### !declare_id

Адрес вашей программы в блокчейне. Он будет обновлён при сборке проекта

### !program

Модуль, содержащий инструкции программы

### !accounts

Аннотирует структуру, содержащую аккаунты программы

### !account

Структура, представляющая типы данных аккаунтов, которые эта программа может
создавать и владеть ими

### !constraints

Ограничения аккаунта. Например, стандартные проверки или функции, повышающие
безопасность программы

### !parameter

Значение, передаваемое инструкции при её вызове

### !initialize

Обращается к полю `data` в аккаунте `new_account` и устанавливает его в значение
из `data: u64`

### !msg

Макрос _rs`msg!`_ записывает сообщение в логи программы транзакции

### !init

Вызывает System Program для создания нового аккаунта и назначает этот программой
владельцем. Как владелец, эта программа затем может записывать данные в аккаунт

### !payer

Аккаунт, который оплачивает создание нового аккаунта

### !space

Количество байт, выделяемых для аккаунта

</WithNotes>

<Accordion title="Пояснение">

<WithMentions>

- Макрос [`declare_id!`](mention:one) указывает on-chain адрес вашей программы.
  Solana Playground автоматически обновит этот адрес при сборке программы.

```rs
// !mention one
declare_id!("11111111111111111111111111111111");
```

</WithMentions>

<WithMentions>

- Атрибут [`#[program]`](mention:one) аннотирует модуль, содержащий функции,
  представляющие инструкции программы.

```rs
// !mention one
#[program]
mod hello_anchor {
    use super::*;
    // !mention two
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !mention three
        ctx.accounts.new_account.data = data;
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}
```

В этом примере инструкция [`initialize`](mention:two) принимает два параметра:

- `ctx: Context<Initialize>`: Передаёт аккаунты, необходимые для этой
  инструкции, в функцию, как указано в структуре `Initialize`.
- `data: u64`: Пользовательский параметр, который вы должны указать при вызове
  инструкции.

Тело функции [устанавливает поле `data` аккаунта `new_account`](mention:three) в
переданный аргумент `data`, затем выводит сообщение в логи программы.

</WithMentions>

<WithMentions>

- Атрибут [`#[derive(Accounts)]`](mention:one) определяет структуру, которая
  указывает аккаунты, необходимые для конкретной инструкции. Каждое поле
  представляет отдельный аккаунт.

  Anchor автоматически выполняет стандартные проверки безопасности, связанные с
  валидацией аккаунтов, через типы полей (например, `Signer<'info>`) и
  ограничения (например, `#[account(mut)]`).

  ```rs
  // !mention one
  #[derive(Accounts)]
  pub struct Initialize<'info> {
      #[account(init, payer = signer, space = 8 + 8)]
      pub new_account: Account<'info, NewAccount>,
      #[account(mut)]
      pub signer: Signer<'info>,
      pub system_program: Program<'info, System>,
  }
  ```

</WithMentions>

<WithMentions>

- Атрибут [`#[account]`](mention:one) определяет структуру, которая представляет
  тип данных для аккаунтов, которые эта программа создаёт и которыми владеет.

```rs
// !mention one
#[account]
pub struct NewAccount {
  data: u64
}
```

</WithMentions>

</Accordion>
</Accordions>

<Step>

### Сборка и деплой программы

1. Скомпилируйте код программы

```terminal
$ build
Building...
Build successful. Completed in 1.46s.
```

<Callout>
  Ваш [program ID](/docs/intro/quick-start/deploying-programs), определённый в
  _rs`declare_id!()`_, теперь будет обновлён.
</Callout>

2. Добавьте SOL в свой кошелёк _(необязательно)_

Для деплоя программы требуется SOL на ончейн-аккаунте программы. При
необходимости
[добавьте SOL в свой кошелёк](/docs/intro/quick-start#add-sol-to-your-wallet)
перед переходом к шагу 3.

3. Задеплойте программу в сеть

```terminal
$ deploy
Deploying... This could take a while depending on the program size and network conditions.
Warning: 1 transaction not confirmed, retrying...
Deployment successful. Completed in 19s.
```

<Callout type="info">
  После деплоя байткод программы сохраняется в исполняемом аккаунте программы.
</Callout>

**Примечание:** При желании вы можете использовать кнопки **Build** и **Deploy**
на левой панели, как показано на скриншоте ниже.

![Сборка и деплой](/assets/docs/intro/quickstart/pg-build-deploy.png)

</Step>

### Протестируйте программу

<WithMentions>

1. Откройте тестовый файл с именем `tests/anchor.test.ts`. В этом файле
   показано, как вызывать структуру [`initialize`](mention:one) программы с
   клиента.

```ts title="anchor.test.ts"
// No imports needed: web3, anchor, pg and more are globally available

describe("Test", () => {
  it("initialize", async () => {
    // Generate keypair for the new account
    const newAccountKp = new web3.Keypair();

    // Send transaction
    const data = new BN(42);
    const txHash = await pg.program.methods
      // !mention one
      .initialize(data)
      .accounts({
        newAccount: newAccountKp.publicKey,
        signer: pg.wallet.publicKey,
        systemProgram: web3.SystemProgram.programId
      })
      .signers([newAccountKp])
      .rpc();
    console.log(`Use 'solana confirm -v ${txHash}' to see the logs`);

    // Confirm transaction
    await pg.connection.confirmTransaction(txHash);

    // Fetch the created account
    const newAccount = await pg.program.account.newAccount.fetch(
      newAccountKp.publicKey
    );

    console.log("On-chain data is:", newAccount.data.toString());

    // Check whether the data on-chain is equal to local 'data'
    assert(data.eq(newAccount.data));
  });
});
```

</WithMentions>

<Step>

### Закройте программу

Закройте программу, чтобы вернуть SOL, выделенные для ончейн-программы. Для
этого запустите `solana program close [ProgramID]`, используя адрес программы,
сохранённый в макросе _rs`declare_id!()`_.

```terminal
$ solana program close 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r
Closed Program Id 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r, 2.79511512 SOL reclaimed
```

<Callout type="warn">
  Программы могут быть **закрыты** или обновлены только аккаунтом с правами
  апгрейда. Ваш кошелёк Playground автоматически становится апгрейд-аккаунтом
  для каждой программы при её деплое. Если права апгрейда отозваны, программа
  становится неизменяемой.
</Callout>

Поздравляем! Вы создали и задеплоили свою первую программу Solana с помощью
Anchor.
