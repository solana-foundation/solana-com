Вы можете принимать платежи по API-запросам через Pay.sh, настроив Pay-сервер.
Установите Pay с помощью:

```bash
brew install pay
```

или

```bash
npm install -g @solana/pay
```

Проверьте установку с помощью:

```bash
pay --version
```

Инструментарий автоматически настроит ваш адрес. Выполните следующую команду для
настройки и пополнения кошелька:

```bash
pay setup
```

## Настройка демо-сервера

Вы можете начать принимать платежи, запустив сервер. Посмотрите демо-сервер в
действии с помощью:

```bash
pay server demo
```

Это выполняет два действия: запускает рабочий Pay-сервер, к которому можно
обращаться через HTTP-запросы, и создаёт файл `pay-demo.yml` в вашей текущей
директории.

Все маршруты API настраиваются через YAML-конфигурацию `endpoints`. Чтобы
принимать платежи, необходимо добавить атрибут `metering:`. Маршрут
`/api/v1/reports/usage` уже настроен в качестве примера:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Вы можете проверить работу платежей, выполнив следующую команду в качестве
клиента в другом терминале:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Узнайте больше о настройке тарификации в
[документации Pay.sh: Разработка с Pay > Тарификация](https://pay.sh/docs/building-with-pay/pricing)

Узнайте больше о настройке Pay.sh в продакшене в
[документации Pay.sh по конфигурации](https://pay.sh/docs/toolchain/configuration).
