Принимайте платежи в Solana с помощью готовых компонентов, протокола Solana Pay
или прямой интеграции через RPC.

## Конвенция получения адреса

В Solana **делитесь адресом своего кошелька, а не token account**. Обычно
отправитель сам определяет подходящий
[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)
для того токена, который он отправляет (см.
[Отправка платежей](/docs/payments/send-payments) для подробностей).

**Пример:** Делитесь `7EcDhS...` (ваш кошелёк), а не `9xKpQ...` (ваш USDC token
account).

<Callout>
  Для точек продаж и электронной коммерции смотрите [Solana
  Pay](/docs/payments/accept-payments/solana-pay) для QR-кодов и платёжных
  ссылок, или [Платёжная кнопка](/docs/payments/accept-payments/payment-button)
  для готовых React-компонентов.
</Callout>

## Варианты интеграции

| Подход                                                            | Лучше всего подходит для             | Сложность |
| ----------------------------------------------------------------- | ------------------------------------ | --------- |
| [Платёжная кнопка](/docs/payments/accept-payments/payment-button) | React-приложения, быстрая интеграция | Низкая    |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)           | QR-коды, платёжные ссылки, POS       | Средняя   |
| [Прямой RPC](/docs/payments/interacting-with-solana)              | Кастомные сценарии, полный контроль  | Высокая   |

## Верификация

После получения платежа проверьте его поступление с помощью
[инструментов верификации](/docs/payments/accept-payments/verification-tools).
Для высоконагруженных приложений смотрите раздел
[Индексация](/docs/payments/accept-payments/indexing) для мониторинга транзакций
на уровне продакшена.
