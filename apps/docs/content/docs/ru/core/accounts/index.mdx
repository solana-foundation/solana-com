Аккаунт — это базовая единица данных Solana для хранения состояния. Сеть хранит
всё состояние в key-value хранилище, где каждый ключ — это 32-байтовый адрес, а
каждое значение — аккаунт.

## Ключевые факты

- **Структура**: Каждый аккаунт содержит
  [пять полей](/docs/core/accounts/account-structure): lamports, данные,
  владелец, исполняемый, rent_epoch.
- **Адрес**: Каждый аккаунт идентифицируется уникальным 32-байтовым адресом
  (публичный ключ Ed25519 или [PDA](/docs/core/pda)).
- **Владение**: Только программа-владелец аккаунта может изменять его данные или
  списывать lamports. Любая программа может зачислять lamports на любой
  доступный для записи аккаунт.
- **Rent**: Каждый аккаунт должен иметь минимальный баланс lamport,
  пропорциональный размеру данных, чтобы оставаться в сети.
