Блокчейн Solana состоит из нескольких групп валидаторов, называемых кластерами.
Каждый кластер выполняет определённые задачи в рамках общей экосистемы и
содержит выделенные API-узлы для обработки [JSON-RPC](/docs/rpc/) запросов для
своего кластера.

Отдельные узлы внутри кластера принадлежат и управляются третьими сторонами, и
для каждого из них доступен публичный эндпоинт.

## Публичные RPC-эндпоинты Solana

Организация Solana Labs управляет публичным RPC-эндпоинтом для каждого кластера.
Каждый из этих публичных эндпоинтов имеет ограничения по скорости, но доступен
для пользователей и разработчиков для взаимодействия с блокчейном Solana.

> Ограничения по скорости публичных эндпоинтов могут изменяться. Конкретные
> ограничения, указанные в этом документе, не гарантированно являются самыми
> актуальными.

### Использование обозревателей с различными кластерами

Многие популярные обозреватели блокчейна Solana поддерживают выбор любого из
кластеров, часто позволяя продвинутым пользователям добавлять
пользовательские/частные RPC-эндпоинты.

Примеры некоторых из этих обозревателей блокчейна Solana включают:

## На высоком уровне

- Mainnet: Рабочая среда для развернутых приложений.
- Devnet: Тестирование с публичным доступом для разработчиков,
  экспериментирующих со своими приложениями.
- Testnet: Стресс-тестирование для обновлений сети и производительности
  валидаторов.

**Примеры использования**: Вы можете захотеть отладить новую программу на Devnet
или проверить показатели производительности на Testnet перед развертыванием на
Mainnet.

| **Кластер** | **Эндпоинт**                     | **Назначение**                                      | **Примечания**                          |
| ----------- | -------------------------------- | --------------------------------------------------- | --------------------------------------- |
| Mainnet     | `https://api.mainnet.solana.com` | Рабочая продакшен-среда                             | Для транзакций требуется SOL            |
| Devnet      | `https://api.devnet.solana.com`  | Публичное тестирование и разработка                 | Бесплатный airdrop SOL для тестирования |
| Testnet     | `https://api.testnet.solana.com` | Тестирование валидаторов и нагрузочное тестирование | Возможны временные простои              |

## Devnet

Devnet служит площадкой для всех, кто хочет протестировать Solana — будь то
пользователь, держатель токенов, разработчик приложений или валидатор.

- Разработчикам приложений рекомендуется использовать Devnet.
- Потенциальным валидаторам также следует сначала использовать Devnet.
- Ключевые отличия между Devnet и Mainnet:
  - Токены Devnet **не являются реальными**
  - В Devnet есть faucet для airdrop токенов для тестирования приложений
  - В Devnet возможны сбросы леджера
  - Devnet обычно работает на той же ветке релиза ПО, что и Mainnet Beta, но
    может использовать более новую минорную версию, чем Mainnet.
- Gossip entrypoint для Devnet: `entrypoint.devnet.solana.com:8001`

### Эндпоинт Devnet

- `https://api.devnet.solana.com` - одиночный API-узел Solana Labs; с
  ограничением скорости

#### Пример конфигурации командной строки `solana`

Для подключения к кластеру `devnet` с помощью Solana CLI:

```shell
solana config set --url https://api.devnet.solana.com
```

### Ограничения скорости Devnet

- Максимальное количество запросов за 10 секунд с одного IP: 100
- Максимальное количество запросов за 10 секунд с одного IP для одного RPC: 40
- Максимальное количество одновременных подключений с одного IP: 40
- Максимальная скорость подключения за 10 секунд с одного IP: 40
- Максимальный объем данных за 30 секунд: 100 МБ

## Testnet

Testnet — это сеть, где основные участники Solana проводят стресс-тестирование
новых функций релизов на живом кластере, уделяя особое внимание
производительности сети, стабильности и поведению валидаторов.

- Токены Testnet **не являются реальными**
- В Testnet возможны сбросы леджера.
- В Testnet есть faucet для airdrop токенов для тестирования приложений
- Testnet обычно работает на более новой ветке релиза ПО, чем Devnet и Mainnet
- Gossip entrypoint для Testnet: `entrypoint.testnet.solana.com:8001`

### Точка подключения Testnet

- `https://api.testnet.solana.com` - одиночный API-узел Solana Labs; с
  ограничением скорости

#### Пример конфигурации командной строки `solana`

Для подключения к кластеру `testnet` с помощью Solana CLI:

```shell
solana config set --url https://api.testnet.solana.com
```

### Ограничения скорости Testnet

- Максимальное количество запросов за 10 секунд с одного IP: 100
- Максимальное количество запросов за 10 секунд с одного IP для одного RPC: 40
- Максимальное количество одновременных подключений с одного IP: 40
- Максимальная скорость подключения за 10 секунд с одного IP: 40
- Максимальный объем данных за 30 секунд: 100 МБ

## Mainnet

Разрешённый, постоянный кластер для пользователей, разработчиков, валидаторов и
держателей токенов Solana.

- Токены, выпущенные в Mainnet, являются **реальными** SOL
- Gossip entrypoint для Mainnet: `entrypoint.mainnet.solana.com:8001`

### Endpoint Mainnet

- `https://api.mainnet.solana.com` - кластер API-узлов Solana Labs,
  поддерживаемый балансировщиком нагрузки; с ограничением скорости

#### Пример конфигурации командной строки `solana`

Для подключения к кластеру `mainnet` с помощью Solana CLI:

```shell
solana config set --url https://api.mainnet.solana.com
```

### Ограничения скорости Mainnet

- Максимальное количество запросов за 10 секунд с одного IP: 100
- Максимальное количество запросов за 10 секунд с одного IP для одного RPC: 40
- Максимальное количество одновременных подключений с одного IP: 40
- Максимальная скорость подключения за 10 секунд с одного IP: 40
- Максимальный объем данных за 30 секунд: 100 МБ

> Публичные RPC-эндпоинты не предназначены для использования в производственных
> приложениях. Пожалуйста, используйте выделенные/частные RPC-серверы при
> запуске вашего приложения, выпуске NFT и т. д. Публичные сервисы подвержены
> злоупотреблениям, и лимиты запросов могут изменяться без предварительного
> уведомления. Также, сайты с высоким трафиком могут быть заблокированы без
> предварительного уведомления.

## Общие коды ошибок HTTP

- 403 -- Ваш IP-адрес или сайт был заблокирован. Пора запускать собственный
  RPC-сервер(ы) или найти частный сервис.
- 429 -- Ваш IP-адрес превышает лимиты запросов. Сбавьте скорость! Используйте
  HTTP-заголовок
  [Retry-After](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Retry-After),
  чтобы определить, сколько времени нужно подождать перед следующей попыткой.
