`@solana/react` публикует Kit-клиент в дерево компонентов через один провайдер,
а хуки считывают из него данные RPC, подписки и состояние кошелька. Хуки
браузерного кошелька находятся в `@solana/kit-plugin-wallet/react` и обращаются
к тому же провайдеру — отдельный провайдер кошелька не нужен.

## Установка

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Создание клиента

Создайте клиент один раз на уровне модуля, вне дерева React. В браузере
`walletSigner` делает подключённый кошелёк одновременно плательщиком комиссии и
идентификатором.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` добавляет `client.wallet` (обнаружение и подключение по Wallet
Standard), а `solanaRpc` добавляет `client.rpc` и вспомогательный метод
`client.sendTransaction`. Экспортируйте тип клиента, обёрнутый в `Awaited<>`,
чтобы он оставался корректным при добавлении плагина с асинхронным разрешением,
и передавайте этот `AppClient` в каждый вызов `useClient`.

## Оберните дерево один раз

`ClientProvider` публикует клиент. Все приведённые ниже хуки читают данные из
него.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Хуки кошелька

Хуки кошелька обращаются к возможности `wallet` клиента, поэтому они работают
только внутри `ClientProvider`, чей клиент установил плагин кошелька.

- `useWallets()` — кошельки Wallet Standard, обнаруженные для цепочки клиента.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` или `null`. `signer`
  равно `null` для кошельков только для чтения.
- `useConnect()` / `useDisconnect()` — возвращают `ActionResult`, чей `dispatch`
  (и `dispatchAsync`) выполняет действие, а `isRunning`, `status`, `data`,
  `error` и `reset` отслеживают его результат.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Отправка транзакции

`useClient` возвращает тот же клиент, который вы опубликовали. Создавайте
инструкции с помощью сгенерированного клиента программы и отправляйте их через
`client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Передача экспортированного типа `AppClient` в `useClient` предоставляет хуку
полный расширенный плагинами клиент, благодаря чему `client.rpc` и
`client.sendTransaction` имеют типизацию.

## Чтение данных

`@solana/react` предоставляет хуки для чтения данных через клиент:

- `useRequest(source)` — однократное RPC-чтение; возвращает
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — живая подписка; возвращает
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — значение, заполненное через RPC и поддерживаемое
  актуальным с помощью подписки.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Мемоизируйте источник (с помощью `useMemo` или `useCallback` для функции), чтобы
хук выполнял повторный запрос только при изменении входных данных, а не при
каждом рендере.

Для кэширования и ревалидации подпути `@solana/react/swr` и
`@solana/react/query` оборачивают те же хуки для [SWR](https://swr.vercel.app) и
[TanStack Query](https://tanstack.com/query).

## Типичные паттерны для Solana-разработчиков

- **Один провайдер**: `ClientProvider` — единственный необходимый провайдер;
  хуки кошелька читают его через возможность `wallet` клиента.
- **Поддержка серверных компонентов**: помечайте директивой `"use client"`
  только листовые компоненты, вызывающие хуки; серверные операции чтения могут
  использовать обычный RPC-клиент Kit без кошелька.
- **Тестирование**: передайте мокированный клиент через `ClientProvider` для
  имитации кошельков и RPC-ответов.

<Callout type="warn" title="Два семейства хуков с одинаковыми именами">
  `useSignIn` и `useSignMessage` существуют как в `@solana/react` (принимают
  аргумент `UiWalletAccount`), так и в `@solana/kit-plugin-wallet/react` (на
  основе клиента, без аргумента аккаунта, возвращают `ActionResult`).
  Используйте версии из `@solana/kit-plugin-wallet/react` с описанным выше
  паттерном клиента; старые хуки кошелька из `@solana/react` постепенно
  выводятся из использования.
</Callout>

Дополните это руководство [обзором клиента Kit](/docs/frontend/client), чтобы
понять, какой клиент использует каждый хук.
