`@solana/kit` поставляется с `createClient` — небольшим хостом, который вы
расширяете с помощью плагинов. Каждый плагин добавляет возможность:
RPC-соединение, подписант, планирование транзакций — и составной клиент
одинаково подходит для скриптов, серверных маршрутов и пользовательских
интерфейсов.

<Callout type="info">
  Создаёте React-приложение? [Руководство по React](/docs/frontend/react-hooks)
  оборачивает этот же клиент с помощью `@solana/react`, чтобы компоненты читали
  состояние кошелька и данные RPC через хуки. Начните с него и возвращайтесь
  сюда, когда вам понадобится более тонкий контроль.
</Callout>

## Установка

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

Используйте любой менеджер пакетов. Kit версии 7+ и пакеты
`@solana/kit-plugin-*` версии 0.13+.

## Создание клиента

Компонуйте плагины с помощью `.use(...)`. Сначала задайте подписант, затем
RPC-соединение. `solanaDevnetRpc()` добавляет `client.rpc`, подписки,
планировщик/исполнитель транзакций и вспомогательную функцию
`client.sendTransaction` — так что приложению редко приходится подключать эти
компоненты вручную.

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

Плагин `signer` задаёт одновременно и плательщика комиссии, и идентификатор
приложения. RPC-пакет требует, чтобы плательщик был определён заранее, поэтому
порядок имеет значение.

### Варианты подписантов

`@solana/kit-plugin-signer` предоставляет три роли, чтобы вы выдавали клиенту
только необходимые права:

- `signer(x)` — задаёт одновременно `client.payer` и `client.identity`.
- `payer(x)` — только плательщик комиссии.
- `identity(x)` — только авторизация приложения.

Каждый имеет варианты `*FromFile(path)` (Node), `generated*()`,
`generated*WithSol(amount)` и `airdrop*()` для тестов и скриптов.

## Отправка транзакции

Создайте инструкции с помощью сгенерированного клиента программы, затем
передайте их в `client.sendTransaction`. Планировщик разрешает блокхэш,
оценивает вычислительные ресурсы, подписывает с помощью плательщика клиента и
отправляет транзакцию.

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

`sendTransaction` возвращает объект результата; сигнатура транзакции находится в
`result.context.signature`. Функция проверяет, что план укладывается в одну
транзакцию, и выбрасывает исключение в противном случае — используйте
`sendTransactions`, когда пакет может охватывать несколько транзакций.

## Выберите RPC-пакет

`@solana/kit-plugin-rpc` поставляется с пресетами кластеров:

- `solanaDevnetRpc()` — настройки devnet, включён airdrop (`client.airdrop`).
- `solanaMainnetRpc()` — mainnet; airdrop вызывает ошибку компиляции.
- `solanaLocalRpc()` — `http://127.0.0.1:8899` для локального validator.
- `solanaRpc({ rpcUrl })` — произвольный endpoint.

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### Гибкая компоновка

Пресеты объединяют RPC-соединение, планировщик, исполнитель и вспомогательные
функции отправки, поэтому большинству приложений отдельные компоненты не нужны.
Используйте гранулярные плагины только тогда, когда нужно настроить один из них
— например, задать собственный планировщик или другой канал подписки. В этом
случае также потребуется `@solana/kit-plugin-instruction-plan`, который пресеты
включают автоматически:

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## Добавление клиентов программ

Сгенерированные пакеты `@solana-program/*` предоставляют клиентские плагины,
позволяющие вызывать программы через клиент. Установите нужную программу и
подключите её плагин через `.use()`:

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="Диапазоны peer-зависимостей">
  В некоторых релизах `@solana-program/*` по-прежнему объявляется
  peer-зависимость `@solana/kit ^6.x`, тогда как Kit уже версии 7.x. Они
  совместимы с Kit 7 — переопределите диапазон peer-зависимости в вашем пакетном
  менеджере, если он выдаёт предупреждение.
</Callout>

## Подписание без браузерного кошелька

Запускаете headless-режим в API-маршруте, воркере или скрипте? Используйте для
подписания серверный сервис управления ключами вместо сырого keypair. См.
[Keychain](/docs/tools/keychain) и
[Подписание в продакшне](/docs/core/transactions/signing-in-production).

## Типовые паттерны для Solana-разработчиков

- **Общая настройка**: Создайте клиент один раз в основном модуле и импортируйте
  его в скрипты, серверные маршруты и UI.
- **Серверные чтения**: Клиент только с `signer` + `solanaRpc` (без
  wallet-плагина) читает аккаунты и симулирует транзакции на сервере.
- **Локальное тестирование**: Замените RPC-пакет на `.use(litesvm())` из
  `@solana/kit-plugin-litesvm` для работы с in-memory validator (только Node).
- **Тестируемость**: Скомпонованный клиент легко мокировать — подменяйте
  RPC-ответы или подписантов без браузерного кошелька.
