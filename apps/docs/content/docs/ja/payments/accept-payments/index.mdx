ドロップインコンポーネント、Solana
Payプロトコル、または直接RPC統合を使用してSolanaで支払いを受け取ります。

## 受信アドレスの規則

Solanaでは、**token
accountではなくウォレットアドレスを共有してください**。送信者が送信するトークンに応じて適切な[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)を導出するのが一般的です(詳細は[支払いの送信](/docs/payments/send-payments)を参照してください)。

**例:** `7EcDhS...`(あなたのウォレット)を共有し、`9xKpQ...`(あなたのUSDC token
account)は共有しないでください。

<Callout>
  販売時点管理システムやeコマースについては、QRコードと支払いURLの[Solana
  Pay](/docs/payments/accept-payments/solana-pay)、またはドロップインReactコンポーネントの[Payment
  Button](/docs/payments/accept-payments/payment-button)を参照してください。
</Callout>

## 統合オプション

| アプローチ                                                      | 最適な用途                  | 複雑さ |
| --------------------------------------------------------------- | --------------------------- | ------ |
| [Payment Button](/docs/payments/accept-payments/payment-button) | Reactアプリ、迅速な統合     | 低     |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | QRコード、支払いリンク、POS | 中     |
| [Direct RPC](/docs/payments/interacting-with-solana)            | カスタムフロー、完全な制御  | 高     |

## 検証

支払いを受け取った後、[検証ツール](/docs/payments/accept-payments/verification-tools)を使用して着金を確認してください。大量のトランザクションを処理するアプリケーションについては、本番環境レベルのトランザクション監視のための[インデックス作成](/docs/payments/accept-payments/indexing)を参照してください。
