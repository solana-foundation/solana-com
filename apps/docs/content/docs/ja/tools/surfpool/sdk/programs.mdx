---
title: プログラムのデプロイ
description: ローカルのビルド成果物からSolanaプログラムをSurfnetにデプロイします。Anchorワークスペースを自動検出するか、明示的な.soパス、生バイト、IDLを指定できます。
---

SDKは、ローカルのビルド成果物から1回の呼び出しでプログラムをデプロイできます。2つの方式があります：標準的なパス構造を持つAnchor/Agaveワークスペース向けの**自動検出**と、バイトデータの取得元を自分で制御したい場合の**明示的デプロイ**です。

どちらの方式も、プログラムをIDLで宣言されたアドレス（またはkeypairファイルの公開鍵）に登録し、IDLが利用可能な場合は`surfnet_registerIdl`を通じてランタイムに登録することで、RPCクライアントがアカウントデータをデコードできるようにします。

## Anchorワークスペースの自動検出

`deployProgram` / `deploy_program`
はプログラム名を受け取り、現在の作業ディレクトリを基準として以下のファイルを検索します：

| ファイル                            | 用途                                                       |
| ----------------------------------- | ---------------------------------------------------------- |
| `target/deploy/{name}.so`           | コンパイル済みプログラムバイトコード（必須）               |
| `target/deploy/{name}-keypair.json` | プログラムkeypair — 公開鍵がプログラムIDになります（必須） |
| `target/idl/{name}.json`            | Anchor IDL JSON（任意；存在する場合は登録されます）        |

<CodeTabs storage="surfpool-lang">

```rust !! title="Rust"
use surfpool_sdk::Surfnet;

let surfnet = Surfnet::start().await?;
let cheats = surfnet.cheatcodes();

let program_id = cheats.deploy_program("my_program")?;
println!("deployed at {program_id}");
```

```ts !! title="TypeScript"
import { Surfnet } from "@solana/surfpool";

const surfnet = Surfnet.start();
try {
  const programId = surfnet.deployProgram("my_program");
  console.log("deployed at", programId);
} finally {
  surfnet.stop();
}
```

</CodeTabs>

<Callout type="info" title="デプロイ前にビルドを実行してください">
  `deployProgram` を呼び出す前に`anchor build`（または`cargo
  build-sbf`）を実行し、 `.so`
  およびkeypairの成果物がディスク上に存在することを確認してください。SDKはビルドを自動的に実行しません。
</Callout>

## 明示的デプロイ

成果物が非標準の場所にある場合（CIのビルド成果物、事前準備済みのテストフィクスチャ、埋め込みバイトスライスなど）、明示的なデプロイ設定を指定してください。

<CodeTabs storage="surfpool-lang">

```rust !! title="Rust"
use surfpool_sdk::cheatcodes::builders::DeployProgram;
use surfpool_sdk::Surfnet;

let cheats = Surfnet::start().await?.cheatcodes();

// Derive the program ID from a keypair file.
let program_id = cheats.deploy(
    DeployProgram::from_keypair_path("fixtures/my_program-keypair.json")?
        .so_path("fixtures/my_program.so")
        .idl_path("fixtures/my_program.idl.json"),
)?;
```

```ts !! title="TypeScript"
const surfnet = Surfnet.start();
const programId = "DemoProgram111111111111111111111111111111111";

surfnet.deploy({
  programId,
  soPath: "fixtures/my_program.so",
  idlPath: "fixtures/my_program.idl.json"
});

surfnet.stop();
```

</CodeTabs>

## 生バイトからのデプロイ

バイトコードがすでにメモリ上にある場合（ビルドステップで生成、成果物ストアからダウンロード、または`include_bytes!`で埋め込まれたもの）、ファイルシステムを介さずに直接デプロイできます。

<CodeTabs storage="surfpool-lang">

```rust !! title="Rust"
use surfpool_sdk::cheatcodes::builders::DeployProgram;
use surfpool_sdk::{Pubkey, Surfnet};

const PROGRAM_BYTES: &[u8] = include_bytes!("../fixtures/my_program.so");

let cheats = Surfnet::start().await?.cheatcodes();
let program_id: Pubkey = "DemoProgram111111111111111111111111111111111".parse()?;

cheats.deploy(
    DeployProgram::new(program_id).so_bytes(PROGRAM_BYTES.to_vec()),
)?;
```

```ts !! title="TypeScript"
import { readFileSync } from "node:fs";

const surfnet = Surfnet.start();
const programBytes = readFileSync("fixtures/my_program.so");

surfnet.deploy({
  programId: "DemoProgramID11111111111111111111111111111",
  soBytes: programBytes
});

surfnet.stop();
```

</CodeTabs>

## DeployProgram ビルダーリファレンス（Rust）

| コンストラクター / セッター        | シグネチャ                                        | 目的                                                                         |
| ---------------------------------- | ------------------------------------------------- | ---------------------------------------------------------------------------- |
| `DeployProgram::new`               | `(program_id: Pubkey) -> Self`                    | 既知のプログラム ID に対してデプロイをビルドする。                           |
| `DeployProgram::from_keypair_path` | `(path: impl AsRef<Path>) -> SurfnetResult<Self>` | Solana の keypair ファイルを読み込み、公開鍵をプログラム ID として使用する。 |
| `.so_path`                         | `(path: impl Into<PathBuf>)`                      | ディスク上にあるコンパイル済み `.so` アーティファクトへのパス。              |
| `.so_bytes`                        | `(bytes: Vec<u8>)`                                | プログラムのバイト列（`so_path` と相互排他）。                               |
| `.idl_path`                        | `(path: impl Into<PathBuf>)`                      | デプロイ後に登録するオプションの Anchor IDL JSON。                           |

## DeployOptions 型リファレンス（JS）

```ts
type ByteArrayLike = Uint8Array | number[];

interface DeployOptions {
  programId: string; // base58 address to deploy at
  soPath?: string; // path to .so on disk
  soBytes?: ByteArrayLike; // or raw bytes in memory
  idlPath?: string; // optional Anchor IDL JSON
}
```

`soPath` と `soBytes` は相互排他です — どちらか一方のみを指定してください。

## 登録される内容

デプロイが成功すると、ランタイムは次の3つの処理を行います：

1. **program account を書き込む** `programId` に、`.so` バイト列を
   `surfnet_writeProgram` 経由で書き込みます。
2. **アカウントを実行可能としてマーク**
   し、通常の RPC 呼び出しで起動できるようにします。
3. **IDL を登録する**（指定された場合）`surfnet_registerIdl`
   経由で登録し、RPC クライアントがプログラムの IDL スキーマを使ってアカウントデータをデコードできるようにします。

<Callout type="info" title="レアケース：実行不可能な program account">
  プログラムのバイト列をアカウントに書き込みつつ実行不可能な状態に保ちたい場合は、
  `deploy` の代わりに `SetAccount` を直接使用してください。`deploy`
  のパスでは常に アカウントが実行可能としてマークされます。
</Callout>

## エラー

| 原因                                                                        | 症状                                                                             |
| --------------------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| `.so` ファイルが存在しないか読み取れない                                    | `SurfnetError::Cheatcode("...failed to read...")` またはJSエラーがスローされる   |
| プログラム ID が keypair ファイルと一致しない                               | `SurfnetError::Cheatcode("...mismatched program id...")`                         |
| IDL JSON の形式が不正                                                       | `SurfnetError::Cheatcode("...invalid IDL...")`                                   |
| Surfnet がオフラインでかつ program account がアップストリームに既に存在する | デプロイは成功するが、ローカルのコピーがアップストリームのバージョンを上書きする |
