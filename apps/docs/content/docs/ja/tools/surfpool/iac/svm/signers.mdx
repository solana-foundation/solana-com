---
title: SVMシグナー
description: SolanaおよびSVM互換ブロックチェーン向けシグナー
---

これらのシグナーは、トランザクション署名にSVMアドオンを使用する際に利用可能です。

## secret_key

`svm::secret_key`
シグナーは、秘密鍵、ニーモニック、またはkeypairファイルを使用してトランザクションに同期的に署名するために使用できます。

### 入力

| 名前              | 必須 | 型     | 説明                                                   |
| ----------------- | ---- | ------ | ------------------------------------------------------ |
| `secret_key`      | 任意 | string | メッセージおよびトランザクションの署名に使用する秘密鍵 |
| `mnemonic`        | 任意 | string | 秘密鍵の生成に使用するニーモニックフレーズ             |
| `derivation_path` | 任意 | string | 秘密鍵の生成に使用する導出パス                         |
| `keypair_json`    | 任意 | string | 秘密鍵を含むkeypair.jsonファイルへのパス               |

### 出力

| 名前         | 型     | 説明                            |
| ------------ | ------ | ------------------------------- |
| `public_key` | string | アカウントのpubkey              |
| `address`    | string | SVMアドレス（public_keyの別名） |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

ニーモニックを使用することもできます：

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

またはkeypairファイルを使用する場合：

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

`svm::web_wallet`
シグナーを使用すると、Runbookオペレーターが任意のブラウザシグナーでトランザクションに署名できます。

### 入力

| 名前               | 必須 | 型     | 説明                                                          |
| ------------------ | ---- | ------ | ------------------------------------------------------------- |
| `expected_address` | 任意 | string | 接続が想定されるSVMアドレス（任意の接続を許可する場合は省略） |

### 出力

| 名前         | 型     | 説明                           |
| ------------ | ------ | ------------------------------ |
| `address`    | string | 接続されたアカウントのアドレス |
| `public_key` | string | 接続されたアカウントのpubkey   |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

このサイナーは、Web UIでユーザーに以下の操作を求めます：

1. ウォレットを接続する
2. メッセージ署名を通じてpubkeyを提供する
3. このサイナーを必要とするすべてのトランザクションに署名する

## squads

`svm::squads`
サイナーは、Squads マルチシグ vault でトランザクションに署名するために使用できます。

### 入力

| 名前      | 必須 | 型     | 説明                   |
| --------- | ---- | ------ | ---------------------- |
| `address` | 任意 | string | squad vault のアドレス |

### 出力

| 名前         | 型     | 説明                              |
| ------------ | ------ | --------------------------------- |
| `public_key` | string | squad vault のpubkey              |
| `address`    | string | SVM アドレス（public_key の別名） |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Squads マルチシグを使用することで、個々の秘密鍵を公開することなく、安全なマルチパーティトランザクション署名が可能になります。
