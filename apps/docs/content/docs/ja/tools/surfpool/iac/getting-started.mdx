Surfpoolは、SolanaエコシステムにInfrastructure as
Code（IaC）を導入し、開発者が最小限のオーバーヘッドで再現性が高く、安全で自動化されたデプロイメントを実現できるようにします。シミュレーションはSurfpoolのコア機能ですが、基盤となるIaCシステムも、現代のSolana開発ワークフローを支える上で同様に重要な役割を果たしています。

## 概要

Crypto Infrastructure as Code（Crypto
IaC）のコンセプトは、従来のDevOpsでおなじみの利点—自動化、監査可能性、モジュール性—をWeb3の世界にもたらします。このモデルは、複雑なクリプト環境全体にわたって、一貫性とスケーラビリティを備えたデプロイメントプラクティスを確保するために設計されています。

SurfpoolのCrypto IaCへのアプローチは、Pyth、Wormhole Core、Circle CCTP、Bitcoin
Ordinalsなど高度なプロトコルとの長年にわたる実践的な経験に基づいています。

堅牢なCrypto
IaCシステムは、通常3つのカテゴリのコンポーネントで構成されます：オンチェーンインフラストラクチャ、署名インフラストラクチャ、そしてオフチェーンインフラストラクチャです。

## オンチェーンインフラストラクチャ

このカテゴリには以下が含まれます：

- Solanaプログラムのデプロイメント
- アップグレード
- ステートマイグレーション

適切に構造化されたIaC実装により、これらのアクションは再現可能で、バージョン管理され、監査可能な状態が保たれます。開発者はすべてのデプロイメントを追跡し、実行前にその影響を把握することができます。

## 署名インフラストラクチャ

安全なトランザクション署名は、本番環境において不可欠です。ローカルのkeypair（重大なセキュリティリスクをもたらす）に依存する代わりに、現代のIaCシステムは以下をサポートします：

- ハードウェアウォレット
- 閾値暗号技術
- マルチシグウォレット

Surfpool
IaCの署名モジュールはモジュール式で設定可能です。ハードコードされた秘密鍵から高度な署名スキーム（例：Squads マルチシグ）への移行は、設定を更新するだけで完了します—コードの書き直しは不要です。

## オフチェーンインフラストラクチャ

これには以下が含まれます：

- インデクサー
- ステートウォッチャー
- ウォレットセンチネル
- 自動化スクリプト

これらのコンポーネントはオンチェーンイベントに反応することが多く、デプロイメントパイプラインの一部として扱う必要があります。他のコードアーティファクトと同様に、テスト可能・ポータブル・バージョン管理されていることが求められます。

> **注意:**
> SurfpoolのIaCは、RPCおよびノードのプロビジョニングを意図的に対象外としています。これらの低レベルな関心事は、Terraform、Ansible、クラウドネイティブサービスなどのツールによって既に十分にサポートされており、アプリケーションレベルのインフラ管理のスコープ外となります。

## 言語設計の原則

重要な設計目標は静的解析です。開発者はコードを実行することなく、関与するプログラム、期待される署名者、アクセスされるアカウント、および推定コストを含む実行計画を作成できる必要があります。

主要な原則：

- 宣言的な構文（JS DSLなし、命令型シェルスクリプトなし）
- コンポーザブルな構造
- 最小限の学習コスト
- インフラとロジックの関心の分離

これにより、署名プロバイダーやネットワークエンドポイントなどのコンポーネントを切り替えるだけで、ローカル開発からメインネットのデプロイまでをシームレスに進めることができます。

## Web3 ランブック

SurfpoolのIaCエンジンは、ブロックチェーンデプロイメント専用に構築されたドメイン固有のフレームワークであるWeb3
Runbooksによって動作しています。このシステムは以下を確保するために数ヶ月にわたって開発されました：

- 使いやすさ
- ルーティンのコンポーザビリティ
- セキュアなランタイム実行

SolanaおよびEthereum互換チェーンの両方をサポートしています。

Ethereum環境では、RunbooksはすでにEigenlayerおよび複数のLayer
2ネットワークが関わる複雑なプロトコルデプロイメントにおいて、ConsensysのInfuraチームなどのハイパフォーマンスチームによって活用されています。

## Surfpool インテグレーション

Surfpoolでは、Runbooksをさらに一歩進め、開発者エクスペリエンスを洗練させ、Solanaのユースケースに最適化しました。目標は、Solana開発を初日からモジュール化・テスト可能・本番対応にすることです。

SurfnetsとCrypto IaCを組み合わせることで、Surfpoolは以下を提供します：

- ローカルファーストの開発者スタック
- シンプルな設定変更によるメインネットへのスムーズな移行
- 完全に定義された再現可能なネットワーク環境
