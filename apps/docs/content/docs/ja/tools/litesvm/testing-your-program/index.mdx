## セットアップ

プログラムをテストする前に、以下を確認してください：

1. `.so`ファイルを生成するためにプログラムをビルドする
2. `litesvm`を`Cargo.toml`ファイルのdev dependencyとして追加する
3. `solana-sdk`や`litesvm-token`など、必要な追加のdev dependencyを追加する
4. `tests`ディレクトリにテストファイルを作成する

## 基本的なプログラムテストの構成

以下は、Solanaプログラムでinstructionsを呼び出して実行するために必要な基本的な手順です。

```rust title="tests/program_test.rs"
use litesvm::LiteSVM;
use solana_sdk::{
    instruction::{AccountMeta, Instruction},
    pubkey::Pubkey,
    signature::{Keypair, Signer},
    transaction::Transaction,
};

use my_program;

#[test]
fn test_my_program() {
    // Initialize the test environment
    let mut svm = LiteSVM::new();

    // Deploy your program to the test environment
    let program_id = Pubkey::from(my_program::ID);
    let program_bytes = include_bytes!("../../../target/deploy/my_program.so");
    svm.add_program(program_id, program_bytes);

    // Create and fund test accounts
    let payer = Keypair::new();
    svm.airdrop(&payer.pubkey(), 10_000_000_000).unwrap();

    // Create your instruction
    let instruction = Instruction {
        program_id,
        accounts: vec![
            AccountMeta::new(payer.pubkey(), true),
        ],
        data: vec![],
    };

    // Build transaction
    let tx = Transaction::new_signed_with_payer(
        &[instruction],
        Some(&payer.pubkey()),
        &[&payer],
        svm.latest_blockhash(),
    );

    // Send transaction
    let result = svm.send_transaction(tx).unwrap();

    // Check transaction succeeded
    println!("Transaction logs: {:?}", result.logs);

    println!("Program executed successfully!");
}
```

`LiteSVM::new()`で生成されるデフォルトのテスト環境には、すべてのランタイム機能、デフォルトのsysvar、プリコンパイル、splプログラム、署名検証、およびSystem
Programを含むすべての組み込みプログラムが有効になっています。

<Callout type="info">
  デフォルトのテスト環境に含まれていないプログラムを使用する場合は、そのプログラムをテスト環境にデプロイする必要があります。
</Callout>
