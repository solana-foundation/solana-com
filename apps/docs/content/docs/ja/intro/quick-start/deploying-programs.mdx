Anchorフレームワークを使用して、最初の[プログラム](/docs/core/programs)をソラナブロックチェーンに構築、デプロイ、テストしましょう。このガイドは、ソラナプレイグラウンドに慣れるのに役立ち、Program
Derived Address（PDA）やCross Program
Invocation（CPI）を使用する準備をするのに役立ちます。

<Steps>

<Step>

### Anchorプロジェクトを作成する

1. ウェブブラウザで[ソラナプレイグラウンド](https://beta.solpg.io)を開きます
2. 左側のパネルで**+ Create a new project**をクリックします
3. **Project name**を入力します
4. フレームワークとして**Anchor (Rust)**を選択します
5. **Create**をクリックします

![新規プロジェクト](/assets/docs/intro/quickstart/pg-new-project.gif)

`src/lib.rs`ソースファイルが開きます。このファイルには`Initialize`という構造体が1つ含まれており、以下の処理を行います。

- *rs`data: u64`*という必須パラメータを受け取ります
- 新しいアカウントを作成します
- *rs`data: u64`*の値を新しいアカウントに保存します
- トランザクションのプログラムログにメッセージを追加します

<WithNotes>

```rust title="lib.rs"
use anchor_lang::prelude::*;

// This is your program's public key and it will update
// automatically when you build the project.
// !tooltip[/declare_id/] declare_id
declare_id!("11111111111111111111111111111111");

// !tooltip[/program/] program
#[program]
mod hello_anchor {
    use super::*;
    // !tooltip[/data: u64/] parameter
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !tooltip[/ctx.accounts.new_account.data = data/] initialize
        ctx.accounts.new_account.data = data;
        // !tooltip[/msg!/] msg
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}

// !tooltip[/Accounts/] accounts
#[derive(Accounts)]
pub struct Initialize<'info> {
    // We must specify the space in order to initialize an account.
    // First 8 bytes are default account discriminator,
    // next 8 bytes come from NewAccount.data being type u64.
    // (u64 = 64 bits unsigned integer = 8 bytes)
    // !tooltip[/account/] constraints
    #[account(
      // !tooltip[/init/] init
      init,
      // !tooltip[/payer/] payer
      payer = signer,
      // !tooltip[/space/] space
      space = 8 + 8
    )]
    pub new_account: Account<'info, NewAccount>,
    // !tooltip[/account/] constraints
    #[account(mut)]
    pub signer: Signer<'info>,
    pub system_program: Program<'info, System>,
}

// !tooltip[/account/] account
#[account]
pub struct NewAccount {
    data: u64
}
```

### !declare_id

プログラムのオンチェーンアドレスです。プロジェクトをビルドすると更新されます

### !program

プログラムのinstructionsを含むモジュール

### !accounts

プログラムのアカウントを含む構造体に注釈を付けます

### !account

このプログラムが作成して所有できるアカウントデータ型を表す構造体

### !constraints

アカウントの制約。例えば、一般的なチェックやプログラムのセキュリティを強化する機能

### !parameter

instructionが呼び出されるときに渡される値

### !initialize

`new_account`アカウントの`data`フィールドにアクセスし、`data: u64`の値を設定します

### !msg

*rs`msg!`*マクロは、トランザクションのプログラムログにメッセージを記録します

### !init

System
Programを呼び出して新しいアカウントを作成し、このプログラムを所有者にします。プログラム所有者として、このプログラムはアカウントデータに書き込むことができます

### !payer

新しいアカウントの支払いを行うアカウント

### !space

アカウントに割り当てるバイト数

</WithNotes>

<Accordion title="説明">

<WithMentions>

- [`declare_id!`](mention:one)マクロは、プログラムのオンチェーンアドレスを指定します。Solana
  Playgroundは、プログラムをビルドする際にこのアドレスを自動的に更新します。

```rs
// !mention one
declare_id!("11111111111111111111111111111111");
```

</WithMentions>

<WithMentions>

- [`#[program]`](mention:one)属性は、プログラムのinstructionsを表す関数を含むモジュールに注釈を付けます。

```rs
// !mention one
#[program]
mod hello_anchor {
    use super::*;
    // !mention two
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !mention three
        ctx.accounts.new_account.data = data;
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}
```

この例では、[`initialize`](mention:two)instructionは2つのパラメータを受け取ります。

- `ctx: Context<Initialize>`:
  `Initialize`構造体で指定されているように、このinstructionに必要なアカウントを関数に渡します。
- `data: u64`:
  instructionを呼び出す際に指定する必要があるカスタムパラメータです。

関数本体は、[`new_account`の`data`フィールドを設定](mention:three)し、指定された`data`引数の値を設定した後、プログラムログにメッセージを出力します。

</WithMentions>

<WithMentions>

- [`#[derive(Accounts)]`](mention:one)属性は、特定のinstructionに必要なアカウントを指定する構造体を定義します。各フィールドは個別のアカウントを表します。

  Anchorは、フィールドタイプ(例: `Signer<'info>`)や制約(例:
  `#[account(mut)]`)を通じて、アカウント検証に関連する一般的なセキュリティチェックを自動的に処理します。

  ```rs
  // !mention one
  #[derive(Accounts)]
  pub struct Initialize<'info> {
      #[account(init, payer = signer, space = 8 + 8)]
      pub new_account: Account<'info, NewAccount>,
      #[account(mut)]
      pub signer: Signer<'info>,
      pub system_program: Program<'info, System>,
  }
  ```

</WithMentions>

<WithMentions>

- [`#[account]`](mention:one)属性は、このプログラムが作成および所有するアカウントのデータ型を表す構造体を定義します。

```rs
// !mention one
#[account]
pub struct NewAccount {
  data: u64
}
```

</WithMentions>

</Accordion>
</Accordions>

<Step>

### プログラムのビルドとデプロイ

1. プログラムコードをコンパイルする

```terminal
$ build
Building...
Build successful. Completed in 1.46s.
```

<Callout>
  _rs`declare_id!()`_で定義されている[プログラムID](/docs/intro/quick-start/deploying-programs)が更新されます。
</Callout>

2. ウォレットにSOLを追加する（任意）

プログラムをデプロイするには、プログラムのオンチェーンアカウントにSOLが必要です。必要に応じて、ステップ3に進む前に[ウォレットにSOLを追加](/docs/intro/quick-start#add-sol-to-your-wallet)してください。

3. プログラムをネットワークにデプロイする

```terminal
$ deploy
Deploying... This could take a while depending on the program size and network conditions.
Warning: 1 transaction not confirmed, retrying...
Deployment successful. Completed in 19s.
```

<Callout type="info">
  デプロイされると、プログラムのバイトコードは実行可能なprogram
  accountに保存されます。
</Callout>

**注意：**
必要に応じて、以下のスクリーンショットに示すように、左側のパネルにある**ビルド**と**デプロイ**ボタンを使用することもできます。

![ビルドとデプロイ](/assets/docs/intro/quickstart/pg-build-deploy.png)

</Step>

### プログラムをテストする

<WithMentions>

1. `tests/anchor.test.ts`というテストファイルを開きます。このファイルは、クライアントからプログラムの[`initialize`](mention:one)構造体を呼び出す方法を示しています。

```ts title="anchor.test.ts"
// No imports needed: web3, anchor, pg and more are globally available

describe("Test", () => {
  it("initialize", async () => {
    // Generate keypair for the new account
    const newAccountKp = new web3.Keypair();

    // Send transaction
    const data = new BN(42);
    const txHash = await pg.program.methods
      // !mention one
      .initialize(data)
      .accounts({
        newAccount: newAccountKp.publicKey,
        signer: pg.wallet.publicKey,
        systemProgram: web3.SystemProgram.programId
      })
      .signers([newAccountKp])
      .rpc();
    console.log(`Use 'solana confirm -v ${txHash}' to see the logs`);

    // Confirm transaction
    await pg.connection.confirmTransaction(txHash);

    // Fetch the created account
    const newAccount = await pg.program.account.newAccount.fetch(
      newAccountKp.publicKey
    );

    console.log("On-chain data is:", newAccount.data.toString());

    // Check whether the data on-chain is equal to local 'data'
    assert(data.eq(newAccount.data));
  });
});
```

</WithMentions>

<Step>

### プログラムを閉じる

オンチェーンプログラムに割り当てられたSOLを回収するには、プログラムを閉じます。これを行うには、*rs`declare_id!()`*マクロに保存されているプログラムアドレスを使用して`solana program close [ProgramID]`を実行します。

```terminal
$ solana program close 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r
Closed Program Id 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r, 2.79511512 SOL reclaimed
```

<Callout type="warn">
  プログラムは、アップグレード権限を持つアカウントによって**のみ**閉じたり更新したりすることができます。Playgroundウォレットは、プログラムがデプロイされるとき、自動的に各プログラムのアップグレード権限を持ちます。アップグレード権限が取り消されると、プログラムは不変になります。
</Callout>
