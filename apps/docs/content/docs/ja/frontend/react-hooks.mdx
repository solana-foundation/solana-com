`@solana/react`
は、1つのプロバイダーを通じてKitクライアントをコンポーネントツリーに公開し、フックはそこからRPCデータ、サブスクリプション、およびウォレットの状態を読み取ります。ブラウザのウォレットフックは
`@solana/kit-plugin-wallet/react`
に含まれており、同じプロバイダーを参照します — 別途ウォレットプロバイダーは必要ありません。

## インストール

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## クライアントのビルド

クライアントは一度だけ、Reactツリーの外側のモジュールスコープでビルドします。ブラウザでは、`walletSigner`
が接続されたウォレットを手数料の支払者かつアイデンティティとして設定します。

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` は `client.wallet`（Wallet
Standard の検出と接続）を追加し、`solanaRpc` は `client.rpc` と
`client.sendTransaction`
ヘルパーを追加します。後でプラグインを非同期に解決する場合でも正確な型を維持できるよう、クライアントの型を
`Awaited<>` でラップしてエクスポートし、その `AppClient` をすべての `useClient`
呼び出しに渡してください。

## ツリーを一度ラップする

`ClientProvider`
がクライアントを公開します。以下のすべてのフックはここから読み取ります。

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## ウォレットフック

ウォレットフックはクライアントの `wallet`
ケイパビリティを参照するため、ウォレットプラグインをインストールしたクライアントを持つ
`ClientProvider` の内部でのみ機能します。

- `useWallets()` — クライアントのチェーンで検出されたWallet Standardウォレット。
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`。
- `useConnectedWallet()` — `{ account, signer, wallet }` または `null`。`signer`
  は読み取り専用ウォレットの場合 `null` になります。
- `useConnect()` / `useDisconnect()` — `dispatch`（および
  `dispatchAsync`）でアクションを実行し、`isRunning`、`status`、`data`、`error`、`reset`
  でその結果を追跡する `ActionResult` を返します。

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## トランザクションの送信

`useClient`
は公開したクライアントを返します。生成されたプログラムクライアントを使用して instructions を構築し、`client.sendTransaction`
を通じて送信します。

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

エクスポートされた `AppClient` 型を `useClient`
に渡すことで、フックはプラグインで拡張されたフルクライアントを取得し、`client.rpc`
と `client.sendTransaction` が型付けされます。

## データの読み取り

`@solana/react`
には、クライアントを通じてデータを読み取るフックが用意されています：

- `useRequest(source)` — 単発のRPC読み取り。`{ data, error, status, refresh }`
  を返します。
- `useSubscription(source)`
  — ライブサブスクリプション。`{ data, error, reconnect, status }` を返します。
- `useTrackedData(spec)`
  — サブスクリプションによって最新状態に保たれる、RPCで初期化された値。

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

フックが毎回のレンダリングではなく入力が変わったときのみ再フェッチするよう、ソースを（関数の場合は
`useMemo`、または `useCallback` で）メモ化してください。

キャッシュと再検証のために、`@solana/react/swr` と `@solana/react/query`
のサブパスが [SWR](https://swr.vercel.app) と
[TanStack Query](https://tanstack.com/query)
向けに同じフックをラップしています。

## Solana開発者向けの一般的なパターン

- **プロバイダーは1つ**: `ClientProvider`
  が唯一必要なプロバイダーです。ウォレットフックはクライアントの `wallet`
  ケイパビリティを通じてこれを読み取ります。
- **サーバーコンポーネント対応**: フックを呼び出すリーフコンポーネントにのみ
  `"use client"`
  を付けてください。サーバーの読み取りはウォレットなしのシンプルなKit
  RPCクライアントを使用できます。
- **テスト**: `ClientProvider`
  を通じてモック済みクライアントを公開し、ウォレットとRPCレスポンスをシミュレートします。

<Callout type="warn" title="同名の2つのフックファミリー">
  `useSignIn` と `useSignMessage` は、`@solana/react`（`UiWalletAccount`
  引数を取る）と
  `@solana/kit-plugin-wallet/react`（クライアントベースで、アカウント引数なし、`ActionResult`
  を返す）の両方に存在します。上記のクライアントパターンでは
  `@solana/kit-plugin-wallet/react` バージョンを使用してください。古い
  `@solana/react` のウォレットフックは廃止されつつあります。
</Callout>
