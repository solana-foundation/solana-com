`@solana/kit`は`createClient`を提供します。これはプラグインで拡張できる小さなホストです。各プラグインはRPC接続、署名者、トランザクション計画といった機能を追加し、組み合わされたクライアントはスクリプト、サーバールート、UIのいずれにも対応します。

<Callout type="info">
  Reactアプリを構築していますか？[Reactガイド](/docs/frontend/react-hooks)では、このクライアントを`@solana/react`でラップし、コンポーネントがフックを通じてウォレットの状態とRPCデータを読み取れるようにしています。まずそちらから始め、より細かい制御が必要になったらここに戻ってください。
</Callout>

## インストール

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

パッケージマネージャーはどれでも使用できます。KitはバージョンはKit
v7以上、`@solana/kit-plugin-*`パッケージは0.13以上が必要です。

## クライアントを作成する

`.use(...)`でプラグインを組み合わせます。最初に署名者を設定し、次にRPC接続を設定します。`solanaDevnetRpc()`は`client.rpc`、サブスクリプション、トランザクションプランナー/エグゼキューター、および`client.sendTransaction`ヘルパーを追加するため、アプリがこれらの部品を自分で接続する必要はほとんどありません。

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

`signer`プラグインは手数料支払者とアプリのIDの両方を設定します。RPCバンドルは先に支払者が設定されている必要があるため、順序が重要です。

### 署名者のバリアント

`@solana/kit-plugin-signer`は3つのロールを提供しており、クライアントに必要な権限のみを付与できます：

- `signer(x)` — `client.payer`と`client.identity`の両方を設定します。
- `payer(x)` — 手数料支払者のみ。
- `identity(x)` — アプリの権限のみ。

それぞれに`*FromFile(path)`（Node）、`generated*()`、`generated*WithSol(amount)`、および`airdrop*()`のバリアントがあり、テストやスクリプトに利用できます。

## トランザクションを送信する

生成されたプログラムクライアントでinstructionsを構築し、`client.sendTransaction`に渡します。プランナーはブロックハッシュを解決し、コンピュートを見積もり、クライアントの支払者で署名して送信します。

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

`sendTransaction`は結果オブジェクトに解決され、トランザクション署名は`result.context.signature`にあります。プランが単一のトランザクションに収まることを表明し、そうでない場合はエラーをスローします。バッチが複数のトランザクションにまたがる可能性がある場合は`sendTransactions`を使用してください。

## RPCバンドルを選択する

`@solana/kit-plugin-rpc` はクラスタープリセットを提供しています：

- `solanaDevnetRpc()` —
  devnetのデフォルト、エアドロップ有効（`client.airdrop`）。
- `solanaMainnetRpc()` — mainnet；エアドロップはコンパイル時エラー。
- `solanaLocalRpc()` — ローカルvalidator用の`http://127.0.0.1:8899`。
- `solanaRpc({ rpcUrl })` — 任意のエンドポイント。

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### 詳細に組み合わせる

プリセットはRPC接続、プランナー、エグゼキューター、および送信ヘルパーをまとめているため、ほとんどのアプリでは各パーツを個別に使用する必要はありません。カスタムプランナーや別のサブスクリプションチャネルなど、1つをカスタマイズする場合にのみ、詳細なプラグインに切り替えてください。このパスには
`@solana/kit-plugin-instruction-plan`
も必要です。これはプリセットに含まれています：

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## プログラムクライアントを追加する

生成された `@solana-program/*`
パッケージはクライアントプラグインを公開しており、プログラム呼び出しをクライアントにぶら下げることができます。必要なプログラムをインストールし、そのプラグインを
`.use()` してください：

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="ピア依存関係の範囲">
  一部の `@solana-program/*` リリースでは、Kitがバージョン`@solana/kit ^6.x`
  のピアを宣言している一方、Kitは7.xです。これらはKit 7で動作します —
  パッケージマネージャーが警告を出す場合は、ピアの範囲をオーバーライドしてください。
</Callout>

## ブラウザウォレットなしで署名する

APIルート、ワーカー、またはスクリプトでヘッドレス実行していますか？生のkeypairの代わりに、バックエンドの鍵管理サービスで署名してください。[Keychain](/docs/tools/keychain)および[本番環境での署名](/docs/core/transactions/signing-in-production)を参照してください。

## Solana開発者向けの一般的なパターン

- **共有セットアップ**：コアモジュールで一度クライアントを構築し、スクリプト、サーバールート、UIにインポートします。
- **サーバー側の読み取り**：`signer` + `solanaRpc`
  のみのクライアント（ウォレットプラグインなし）は、サーバーからアカウントを読み取り、トランザクションをシミュレートします。
- **ローカルテスト**：`@solana/kit-plugin-litesvm` の `.use(litesvm())`
  にRPCバンドルを切り替えて、インメモリvalidatorに対して実行します（Nodeのみ）。
- **テスト容易性**：構成されたクライアントはモックが簡単です — ブラウザウォレットがなくても、RPCレスポンスや署名者をスタブ化できます。
