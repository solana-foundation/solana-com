Solanaのフロントエンド開発では、プログラム、ウォレット、Reactなどの人気のあるJavaScriptフレームワークを扱います。これらのコンポーネントとやり取りするには、接続、トランザクション作成、Solanaアカウントからの読み取りを処理する必要があります。

[Solana Kit](https://github.com/anza-xyz/kit)
(`@solana/kit`) は、この用途向けの標準的なJavaScript
SDKです。小さく組み合わせ可能なプラグインからクライアントを構築し、必要に応じてReactバインディングを追加し、呼び出すオンチェーンプログラム用に生成されたプログラムクライアントを取り込みます。

## 主要なライブラリ

<Cards>
  <Card title="@solana/kit" href="https://github.com/anza-xyz/kit">
    - コアSDK：RPC、コーデック、署名者、トランザクション -
    他のすべてのライブラリが拡張するプラグインホストである `createClient` を提供
  </Card>
  <Card title="Kit plugins" href="https://github.com/anza-xyz/kit-plugins">
    - `.use(...)`
    でクライアントを構成：RPC接続、ウォレット、署名者、トランザクション計画 -
    `@solana/kit-plugin-*` パッケージとして公開
  </Card>
  <Card
    title="@solana/react"
    href="https://github.com/anza-xyz/kit/tree/main/packages/react"
  >
    - KitクライアントのReactバインディング - `ClientProvider`
    と読み取り、サブスクリプション、ウォレット状態のためのフック
  </Card>
  <Card title="@solana-program/*" href="https://github.com/solana-program">
    - オンチェーンプログラム（System、Tokenなど）用の生成済みクライアント -
    instructions ビルダーと `tokenProgram()`
    のようなクライアントプラグインを提供
  </Card>
</Cards>

<Callout type="warn" title="web3.js v1 と wallet-adapter はレガシーです">
  多くのSolanaプロジェクトは、旧来の `@solana/web3.js` (v1) と
  `@solana/wallet-adapter-*`
  に依存しています。新しい開発では、以下のkitプラグインと共に `@solana/kit`
  を優先してください。Wallet Standard discovery（`@solana/kit-plugin-wallet`
  経由）が、モダンなウォレット向けにwallet-adapterを置き換えます。
</Callout>
