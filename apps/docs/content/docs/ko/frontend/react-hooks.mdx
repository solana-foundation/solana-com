`@solana/react`는 하나의 프로바이더를 통해 컴포넌트 트리에 Kit 클라이언트를
제공하며, 훅은 해당 프로바이더에서 RPC 데이터, 구독 정보, 지갑 상태를
읽어옵니다. 브라우저 지갑 훅은 `@solana/kit-plugin-wallet/react`에 포함되어
있으며 동일한 프로바이더를 사용합니다 — 별도의 지갑 프로바이더는 필요하지
않습니다.

## 설치

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## 클라이언트 빌드

React 트리 외부의 모듈 스코프에서 클라이언트를 한 번 빌드하세요. 브라우저에서
`walletSigner`는 연결된 지갑을 수수료 지불자이자 신원으로 설정합니다.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner`는 `client.wallet`(Wallet Standard 검색 및 연결)를 추가하고,
`solanaRpc`는 `client.rpc`와 `client.sendTransaction` 헬퍼를 추가합니다. 나중에
비동기적으로 처리되는 플러그인을 추가하더라도 타입이 올바르게 유지되도록
`Awaited<>`로 감싼 클라이언트 타입을 내보내고, 해당 `AppClient`를 모든
`useClient` 호출에 전달하세요.

## 트리를 한 번만 감싸기

`ClientProvider`는 클라이언트를 제공합니다. 아래의 모든 훅은 이 프로바이더에서
데이터를 읽습니다.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## 지갑 훅

지갑 훅은 클라이언트의 `wallet` 기능을 사용하므로, 지갑 플러그인이 설치된
클라이언트를 가진 `ClientProvider` 내부에서만 동작합니다.

- `useWallets()` — 클라이언트의 체인에서 검색된 Wallet Standard 지갑 목록.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` 또는 `null`. 읽기 전용
  지갑의 경우 `signer`는 `null`입니다.
- `useConnect()` / `useDisconnect()` — `dispatch`(및 `dispatchAsync`)가 액션을
  실행하고 `isRunning`, `status`, `data`, `error`, `reset`가 결과를 추적하는
  `ActionResult`를 반환합니다.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## 트랜잭션 전송

`useClient`는 제공된 클라이언트를 반환합니다. 생성된 프로그램 클라이언트로
명령어를 빌드하고 `client.sendTransaction`를 통해 전송하세요.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

내보낸 `AppClient` 타입을 `useClient`에 전달하면 훅이 플러그인이 확장된 전체
클라이언트를 받아 `client.rpc`와 `client.sendTransaction`의 타입이 지정됩니다.

## 데이터 읽기

`@solana/react`는 클라이언트를 통해 읽는 훅을 제공합니다:

- `useRequest(source)` — 단발성 RPC 읽기; `{ data, error, status, refresh }`를
  반환합니다.
- `useSubscription(source)` — 실시간 구독;
  `{ data, error, reconnect, status }`를 반환합니다.
- `useTrackedData(spec)` — 구독으로 최신 상태를 유지하는 RPC 초기값.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

훅이 매 렌더링마다 재요청하지 않고 입력값이 변경될 때만 재요청하도록 소스를
메모이제이션하세요 (`useMemo` 사용, 함수의 경우 `useCallback` 사용).

캐싱 및 재검증을 위해 `@solana/react/swr` 및 `@solana/react/query` 서브패스는
[SWR](https://swr.vercel.app) 및
[TanStack Query](https://tanstack.com/query)용으로 동일한 훅을 래핑합니다.

## Solana 개발자를 위한 일반적인 패턴

- **단일 프로바이더**: `ClientProvider`는 필요한 유일한 프로바이더이며, 지갑
  훅은 클라이언트의 `wallet` 기능을 통해 이를 읽습니다.
- **서버 컴포넌트 지원**: 훅을 호출하는 리프 컴포넌트에만 `"use client"`를
  표시하세요; 서버 읽기는 지갑 없이 일반 Kit RPC 클라이언트를 사용할 수
  있습니다.
- **테스트**: `ClientProvider`를 통해 모의 클라이언트를 제공하여 지갑과 RPC
  응답을 시뮬레이션합니다.

<Callout type="warn" title="두 훅 패밀리, 동일한 이름">
  `useSignIn`와 `useSignMessage`는 `@solana/react`(`UiWalletAccount` 인수를
  받음)와 `@solana/kit-plugin-wallet/react`(클라이언트 기반, 계정 인수 없음,
  `ActionResult` 반환) 모두에 존재합니다. 위의 클라이언트 패턴에서는
  `@solana/kit-plugin-wallet/react` 버전을 사용하세요; 기존 `@solana/react` 지갑
  훅은 대체되고 있습니다.
</Callout>

각 훅이 읽는 클라이언트를 이해하려면 이 가이드를
[Kit 클라이언트 개요](/docs/frontend/client)와 함께 참고하세요.
