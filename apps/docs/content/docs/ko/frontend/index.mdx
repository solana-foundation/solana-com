Solana의 프론트엔드 개발은 프로그램, 지갑 및 React와 같은 인기 있는 JavaScript
프레임워크를 다루는 작업을 포함합니다. 이러한 구성 요소와 상호 작용하려면 연결
처리, 트랜잭션 생성 및 Solana 계정에서 읽기가 필요합니다.

[Solana Kit](https://github.com/anza-xyz/kit) (`@solana/kit`)은 이 작업을 위한
표준 JavaScript SDK입니다. 작고 조합 가능한 플러그인으로 클라이언트를 구성하고,
필요할 때 React 바인딩을 추가하며, 호출하는 온체인 프로그램에 맞는 생성된
프로그램 클라이언트를 가져올 수 있습니다.

## 주요 라이브러리

<Cards>
  <Card title="@solana/kit" href="https://github.com/anza-xyz/kit">
    - 핵심 SDK: RPC, 코덱, 서명자 및 트랜잭션 - 다른 모든 라이브러리가 확장하는
    플러그인 호스트 `createClient` 제공
  </Card>
  <Card title="Kit plugins" href="https://github.com/anza-xyz/kit-plugins">
    - `.use(...)`로 클라이언트 구성: RPC 연결, 지갑, 서명자 및 트랜잭션 계획 -
    `@solana/kit-plugin-*` 패키지로 게시됨
  </Card>
  <Card
    title="@solana/react"
    href="https://github.com/anza-xyz/kit/tree/main/packages/react"
  >
    - Kit 클라이언트용 React 바인딩 - 읽기, 구독 및 지갑 상태를 위한
    `ClientProvider` 및 훅
  </Card>
  <Card title="@solana-program/*" href="https://github.com/solana-program">
    - 온체인 프로그램용 생성된 클라이언트 (System, Token 등) -
    `tokenProgram()`와 같은 명령어 빌더 및 클라이언트 플러그인 제공
  </Card>
</Cards>

<Callout type="warn" title="web3.js v1 및 wallet-adapter는 레거시입니다">
  많은 Solana 프로젝트가 여전히 구버전 `@solana/web3.js` (v1) 및
  `@solana/wallet-adapter-*`에 의존하고 있습니다. 새로운 작업에는 아래 kit
  플러그인과 함께 `@solana/kit`를 사용하는 것을 권장합니다. Wallet Standard 검색
  (`@solana/kit-plugin-wallet`를 통해) 은 최신 지갑을 위해 wallet-adapter를
  대체합니다.
</Callout>

- [Kit 클라이언트 가이드](/docs/frontend/client): RPC, 서명 및 트랜잭션 전송을
  위한 플러그인으로 클라이언트 구축.
- [React 가이드](/docs/frontend/react-hooks): 동일한 클라이언트 위에 계층화된
  `@solana/react` 바인딩 및 지갑 훅.
- [Next.js 튜토리얼](/docs/frontend/nextjs-solana): App Router에서 지갑 연결 및
  SOL 전송 앱 전체 구현.
- [Kit으로 마이그레이션](/docs/frontend/web3-compat): web3.js 및 wallet-adapter
  개념이 Kit에 어떻게 매핑되는지 안내.
