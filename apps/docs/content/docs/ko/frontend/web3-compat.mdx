`@solana/web3.js` (v1), `@solana/wallet-adapter-*`, 또는 이전 프레임워크-kit
패키지(`@solana/client`, `@solana/react-hooks`, `@solana/web3-compat`)를
사용하고 계셨다면, 해당 방식들은 더 이상 권장되지 않습니다. 새로운 앱은
`@solana/kit`와 kit 플러그인을 기반으로 직접 구축합니다. 이 페이지는 기존에 알고
계신 개념들을 Kit의 동등한 개념으로 매핑합니다.

## 개념 매핑

| web3.js v1 / wallet-adapter            | Kit                                                                       |
| -------------------------------------- | ------------------------------------------------------------------------- |
| `new Connection(url)`                  | `createSolanaRpc(url)`, 또는 클라이언트에서 `.use(solanaRpc({ rpcUrl }))` |
| `PublicKey`                            | `Address` — `address("...")`로 문자열 변환                                |
| `Keypair`                              | `generateKeyPairSigner()` 또는 `@solana/kit-plugin-signer` 플러그인       |
| `SystemProgram.transfer(...)`          | `@solana-program/system`의 `getTransferSolInstruction(...)`               |
| `LAMPORTS_PER_SOL` 수학 연산           | `@solana/kit`의 `lamports(...)`                                           |
| `Transaction` / `VersionedTransaction` | 트랜잭션 플래너가 빌드 및 패킹하는 트랜잭션 메시지                        |
| `sendAndConfirmTransaction(...)`       | `client.sendTransaction([...])` (플래너가 서명, 전송 및 확인)             |
| `@solana/wallet-adapter-*`             | `@solana/kit-plugin-wallet` + Wallet Standard 검색                        |

## web3.js에서 오셨나요?

대규모 `@solana/web3.js` v1 코드베이스를 유지 관리 중이며 Kit으로의 전체
재작성이 아직 현실적이지 않다면, web3.js v3가 그 연결 다리 역할을 합니다. 현재
`@solana/web3.js@rc` (`3.0.0-rc` 라인)로 출시되어 있으며, `@solana/kit` 내부를
기반으로 기존 클래스 기반 API — `Connection`, `Keypair`, `Transaction` — 를
재구성합니다. Kit 위에 구축되어 있어 상호 운용성이 긴밀합니다: `PublicKey`는
`Address`의 deprecated 별칭이며, v3 `Keypair`는 구조적으로 Kit의
`KeyPairSigner`를 만족하므로, Kit API, 플러그인 및 Codama 생성 클라이언트에 직접
전달할 수 있습니다.

v3는 아직 **릴리스 후보** 단계입니다 — 정확한 버전을 고정하고 RC 간 일부 API
변경을 예상하세요. 이는 새로운 개발의 기본값이 아닌 레거시 상호 운용 경로이므로,
새 앱은 Kit을 기반으로 직접 구축해야 합니다. 마이그레이션 자체에 대해서는 공식
[web3.js v1 → v3 마이그레이션 가이드](https://github.com/solana-foundation/solana-web3.js/blob/v3.x/docs/web3js-v1-to-v3-migration.md)를
참고하세요.

## 지갑

Kit은 wallet-adapter를 사용하지 않습니다. `@solana/kit-plugin-wallet`는 Wallet
Standard 지갑을 검색하고 `client.wallet` 및
[React 훅](/docs/frontend/react-hooks)을 통해 노출합니다. 최신 지갑(Phantom,
Solflare, Backpack 등)은 Wallet Standard를 통해 자신을 알리므로 지갑별 어댑터
패키지가 필요하지 않습니다.

[@solana/connector](https://www.npmjs.com/package/@solana/connector)와 같은 사전
빌드된 도구를 사용할 수도 있습니다. 이 도구는 Kit 기본 요소를 직접 사용하여
지갑에 연결합니다.

<Callout type="info" title="아직 wallet-adapter를 사용 중이신가요?">
  `@solana/wallet-adapter-*` 기반으로 구축된 앱을 유지 관리 중이며 아직
  마이그레이션할 준비가 되지 않은 경우, [wallet-adapter
  저장소](https://github.com/anza-xyz/wallet-adapter)가 해당 스택의 참조 자료로
  유지됩니다. 새로운 프로젝트에는 Kit 지갑 플러그인을 사용하는 것을 권장합니다.
</Callout>
