계정은 상태를 저장하기 위한 Solana의 기본 데이터 단위입니다. 네트워크는 모든
상태를 키-값 저장소에 저장하며, 각 키는 32바이트 주소이고 각 값은 계정입니다.

## 주요 사항

- **구조**: 모든 계정은 동일한
  [5개의 필드](/docs/core/accounts/account-structure)를 가집니다: lamport,
  데이터, 소유자, 실행 가능, rent_epoch.
- **주소**: 각 계정은 고유한 32바이트
  주소([Ed25519 공개 키](https://ko.wikipedia.org/wiki/EdDSA) 또는
  [PDA](/docs/core/pda))로 식별됩니다.
- **소유권**: 계정의 소유자 프로그램만 해당 데이터를 수정하거나 lamport를 차감할
  수 있습니다. 모든 프로그램은 쓰기 가능한 계정에 lamport를 입금할 수 있습니다.
- **Rent**: 모든 계정은 온체인에 유지되기 위해 데이터 크기에 비례하는 최소
  lamport 잔액을 보유해야 합니다.
