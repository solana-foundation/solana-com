일곱 가지 개념이 솔라나 개발의 기초를 형성합니다. 계정, 프로그램, 명령어,
트랜잭션, 수수료를 순서대로 읽으세요. 각 개념은 이전 개념을 기반으로 합니다.
PDA를 읽기 전에 계정과 프로그램을 먼저 읽으세요. CPI를 읽기 전에 프로그램,
명령어, PDA를 먼저 읽으세요.

<Cards>
  <Card title="계정" href="/docs/core/accounts">
    솔라나는 모든 상태를 계정에 저장합니다. 계정은 32바이트 주소로 키가 지정된
    키-값 저장소입니다. 각 계정에는 lamports, data, owner, executable,
    rent_epoch의 다섯 가지 필드가 있습니다.
  </Card>
  <Card title="프로그램" href="/docs/core/programs">
    sBPF로 컴파일된 솔라나의 스마트 컨트랙트입니다. 프로그램은 상태를 저장하지
    않으며, 모든 변경 가능한 상태는 명령어를 통해 전달되는 별도의 데이터 계정에
    저장됩니다.
  </Card>
  <Card title="명령어" href="/docs/core/instructions">
    프로그램 ID, 계정 목록, 불투명 데이터 바이트를 포함하는 하나의 프로그램 실행
    요청입니다. 여러 명령어가 하나의 트랜잭션으로 구성됩니다.
  </Card>
  <Card title="트랜잭션" href="/docs/core/transactions">
    솔라나의 원자적 실행 단위입니다. 서명과 메시지(계정 주소, 최근 블록해시,
    컴파일된 명령어)를 포함합니다. 최대 1232바이트입니다.
  </Card>
  <Card title="수수료" href="/docs/core/fees">
    서명당 5,000 lamports의 기본 수수료와 요청된 컴퓨팅 유닛에 기반한 선택적
    우선순위 수수료가 있습니다. 트랜잭션당 최대 140만 컴퓨팅 유닛입니다.
  </Card>
  <Card title="프로그램 파생 주소 (PDA)" href="/docs/core/pda">
    프로그램 ID와 시드에서 파생된 결정론적 주소입니다. 소유 프로그램만
    invoke_signed를 통해 PDA를 대신하여 서명할 수 있습니다.
  </Card>
  <Card title="크로스 프로그램 호출 (CPI)" href="/docs/core/cpi">
    invoke 또는 invoke_signed를 통해 한 프로그램이 다른 프로그램을 호출하는
    것입니다. 서명자 및 쓰기 가능 권한이 호출된 프로그램으로 확장됩니다. 최대
    명령어 스택 깊이는 5입니다(최상위 레벨 + CPI).
  </Card>
</Cards>
