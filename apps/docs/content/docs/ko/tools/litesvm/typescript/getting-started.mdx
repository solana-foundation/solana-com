# TypeScript용 LiteSVM

`solana-test-validator`를 실행하는 Solana 테스트 스위트는 테스트당 수 초가
걸립니다 — 프로세스 시작, RPC 연결, 네트워크 왕복 대기 시간 때문입니다.
LiteSVM은 동일한 테스트를 밀리초 단위로 실행합니다. 외부 프로세스도 없고,
네트워크도 없으며, 모든 것이 테스트 러너 내부의 인메모리에서 처리됩니다.

<Callout type="info">
  **테스트 대상:** `@solana/kit@6.8.0`, `@solana/kit-plugin-litesvm@0.10.0`,
  `@solana/kit-plugin-signer@0.10.0`, `@solana-program/system@0.12.0`,
  `@solana-program/token@0.13.0`. 이 플러그인 패키지들은 새롭고 빠르게 발전하고
  있습니다 — 버전 불일치가 발생하면,
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)에서
  문서 및 테스트 스위트가 실행되는 정확한 버전을 확인하세요.
</Callout>

`.use()`로 플러그인을 설치하여 `@solana/kit`에서 Kit 클라이언트를 구성합니다:
서명자 플러그인
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
LiteSVM 플러그인
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm)),
그리고 필요한 프로그램 플러그인들. 결과적으로 `client.sendTransaction()`,
`client.rpc`, 그리고 계정 상태, 클락, 시스바에 대한 직접 접근이 가능하며 — 모두
프로세스를 벗어나지 않고 사용할 수 있습니다.

### 클라이언트 생성

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()`는 빈 셸을 반환하며 — 각 `.use(plugin)`가 그 위에 기능을
  추가합니다. `signer()` 플러그인은 `litesvm()`보다 **앞에** 위치해야 합니다.
  트랜스포트 플러그인이 연결할 페이어가 필요하기 때문입니다.
  `signer(mySigner)`는 동일한 keypair를 `payer`와 `identity` 둘 다로 설치하는
  축약형입니다 — 역할을 분리하고 싶다면 같은 패키지의 `payer()` 또는
  `identity()`를 사용하세요. 실제 네트워크에서 `solanaRpc()`를 사용할 때도 이
  순서는 동일합니다.
</Callout>

<Callout type="info">
  빠른 실험을 위한 한 줄 설정을 선호하시나요? `litesvm()` 번들에는 에어드롭이
  포함되어 있으므로, `@solana/kit-plugin-signer`의
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))`를 체이닝하여 페이어를
  자동 생성하고 자금을 충전할 수 있습니다.
</Callout>

</Step>

</Steps>

## 클라이언트 이해하기

`.use(signer(...)).use(litesvm())` 이후, 클라이언트는 다음을 노출합니다:

| 속성                        | 설명                                                                                                                                                                                                     |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | `signer()` 플러그인과 함께 설치한 `TransactionSigner` (`client.identity`로도 설정됨)                                                                                                                     |
| `client.identity`           | 계정, 토큰, 권한을 소유하는 지갑 서명자                                                                                                                                                                  |
| `client.svm`                | 계정 관리, 트랜잭션 및 구성을 위한 직접 [`LiteSVM`](https://www.npmjs.com/package/litesvm) 인스턴스                                                                                                      |
| `client.rpc`                | Kit 호환 RPC 인터페이스 (지원 메서드: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | 명령어를 트랜잭션으로 계획, 서명 및 전송                                                                                                                                                                 |
| `client.airdrop`            | LiteSVM 파우셋에서 SOL 요청                                                                                                                                                                              |
| `client.getMinimumBalance`  | rent 면제를 위한 최소 lamport 계산                                                                                                                                                                       |

### client.svm 사용하기

LiteSVM에 직접 접근하여 다음을 수행합니다:

- 계정 관리 (`setAccount`, `getAccount`, `airdrop`)
- 트랜잭션 실행 (`sendTransaction`, `simulateTransaction`)
- 구성 (`withSigverify`, `withBlockhashCheck` 등)
- 시간 조작 (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### client.rpc 사용하기

표준 작업을 위한 Kit 호환 RPC 인터페이스:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  RPC 레이어는 **base64 인코딩**만 지원합니다. 다른 인코딩을 요청하면 오류가
  발생합니다.
</Callout>

## 구성

`client.svm`의 빌더 메서드를 사용하여 LiteSVM을 구성합니다:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## 트랜잭션 전송

전송 레이어 이후에 프로그램 플러그인을 추가하세요 — 플러그인은 클라이언트에
명령어 빌더를 연결하므로 원시 `TransactionMessage`를 직접 다룰 필요가 없습니다:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## SVM → RPC 흐름

SVM으로 상태를 설정하고 RPC로 읽을 수 있습니다:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```

## 다음 단계

API 레퍼런스는 [API 문서](/docs/tools/litesvm/typescript/api-reference)를
참고하세요.
