이 서명자들은 트랜잭션 서명을 위해 SVM 애드온을 사용할 때 사용할 수 있습니다.

## secret_key

`svm::secret_key` 서명자는 시크릿 키, 니모닉, 또는 keypair 파일을 사용하여
트랜잭션에 동기적으로 서명하는 데 사용할 수 있습니다.

### 입력

| 이름              | 필수 여부 | 타입   | 설명                                         |
| ----------------- | --------- | ------ | -------------------------------------------- |
| `secret_key`      | 선택      | string | 메시지 및 트랜잭션 서명에 사용되는 시크릿 키 |
| `mnemonic`        | 선택      | string | 시크릿 키 생성에 사용되는 니모닉 구문        |
| `derivation_path` | 선택      | string | 시크릿 키 생성에 사용되는 파생 경로          |
| `keypair_json`    | 선택      | string | 시크릿 키가 포함된 keypair.json 파일 경로    |

### 출력

| 이름         | 타입   | 설명                         |
| ------------ | ------ | ---------------------------- |
| `public_key` | string | 계정의 공개 키               |
| `address`    | string | SVM 주소 (public_key의 별칭) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

니모닉을 사용할 수도 있습니다:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

또는 keypair 파일을 사용할 수도 있습니다:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

`svm::web_wallet` 서명자를 통해 Runbook 운영자는 원하는 브라우저 서명자를
사용하여 트랜잭션에 서명할 수 있습니다.

### 입력

| 이름               | 필수 여부 | 타입   | 설명                                         |
| ------------------ | --------- | ------ | -------------------------------------------- |
| `expected_address` | 선택      | string | 연결이 예상되는 SVM 주소 (임의 허용 시 생략) |

### 출력

| 이름         | 타입   | 설명                  |
| ------------ | ------ | --------------------- |
| `address`    | string | 연결된 계정의 주소    |
| `public_key` | string | 연결된 계정의 공개 키 |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

이 서명자는 Web UI에서 사용자에게 다음을 요청합니다:

1. 지갑 연결
2. 메시지 서명을 통한 pubkey 제공
3. 이 서명자가 필요한 모든 트랜잭션 서명

## squads

`svm::squads` 서명자는 Squads 멀티시그 볼트로 트랜잭션에 서명하는 데 사용할 수
있습니다.

### 입력

| 이름      | 필수 여부 | 유형   | 설명             |
| --------- | --------- | ------ | ---------------- |
| `address` | 선택 사항 | string | 스쿼드 볼트 주소 |

### 출력

| 이름         | 유형   | 설명                         |
| ------------ | ------ | ---------------------------- |
| `public_key` | string | 스쿼드 볼트의 pubkey         |
| `address`    | string | SVM 주소 (public_key의 별칭) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Squads 멀티시그를 사용하면 개인 프라이빗 키를 노출하지 않고 안전한 다자간
트랜잭션 서명이 가능합니다.
