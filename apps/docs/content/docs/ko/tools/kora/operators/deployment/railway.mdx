[Railway](https://railway.com/)는 자동 SSL, 도메인 관리 및 내장 모니터링을 통해
Kora 노드를 쉽게 배포할 수 있는 환경을 제공합니다. 이 가이드는 간단한 Docker
이미지와 Railway CLI를 사용하여 Railway에서 Kora 노드를 설정하는 방법을
안내합니다.

## 사전 요구 사항

- [Railway 계정](https://railway.app)
- [Railway CLI](https://docs.railway.com/guides/cli) 설치됨
- 개인 키 생성을 위한 [Solana CLI](https://solana.com/docs/intro/installation)
- [Solana RPC 엔드포인트](https://solana.com/rpc)

## 1단계: 배포 파일 준비

Kora 노드를 위한 새 디렉토리를 생성하세요:

```bash
mkdir my-kora-node
cd my-kora-node
```

디렉토리에 샘플 파일을 추가하세요:

- [kora.toml](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/kora.toml)
- [signers.toml](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/signers.toml)
- [Dockerfile](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/Dockerfile)

이제 디렉토리는 다음과 같이 보여야 합니다:

```bash
my-kora-node/
├── kora.toml
├── signers.toml
└── Dockerfile
```

## 2단계: Railway에 배포

프로젝트 디렉토리에서 Railway에 로그인하세요

```bash
# Login to Railway
railway login
```

화면의 안내에 따라 계정으로 로그인하세요.

디렉토리에서 Railway 프로젝트를 초기화하세요.

```bash
railway init
```

Railway는 다음을 요청합니다:

1. 새 프로젝트 생성 또는 기존 프로젝트 선택
2. 프로젝트 이름 선택 (예: "my-kora-node")

### 애플리케이션 배포

```bash
# Deploy the application
railway up
```

이 명령은 다음을 수행합니다:

- Railway에 파일 업로드
- Docker 이미지 빌드
- 애플리케이션 배포

**예상 출력:**

```
Build time: 121.81 seconds
Deploy complete
Starting Container
```

좋습니다! 모든 설정이 완료되었습니다! 로그에 오류가 표시될 수 있습니다 ("error:
a value is required for `--rpc-url <RPC_URL>` but none was supplied "). 이는
아직 환경 변수를 구성하지 않았기 때문입니다. 이제 환경 변수를 설정해 보겠습니다.

## 3단계: 환경 변수 구성

[railway.app](https://railway.app)에서 Railway 대시보드를 열고 프로젝트로
이동하세요. 프로젝트에서 "Settings" > "Variables"를 클릭하세요

![Kora를 위한 Railway 환경 변수](/assets/docs/tools/kora/deploy/railway-variables.jpg)

다음 환경 변수를 추가하세요:

| 변수               | 값                                    | 설명                         |
| ------------------ | ------------------------------------- | ---------------------------- |
| `RPC_URL`          | `https://api.mainnet-beta.solana.com` | Solana RPC 엔드포인트        |
| `KORA_PRIVATE_KEY` | `your-base58-private-key`             | Solana 개인 키 (base58 형식) |
| `RUST_LOG`         | `info`                                | 디버깅을 위한 로그 레벨      |

> **보안 경고**: 개인 키는 수수료 지불을 위한 SOL에 접근할 수 있습니다. Kora
> 노드 전용 keypair를 사용하고 거래 수수료로 사용할 의향이 있는 SOL만
> 입금하세요. 서명자에 대한 자세한 내용은
> [서명자 가이드](/docs/tools/kora/operators/signers)를 참조하세요.

### 개인 키 가져오기

새 keypair를 생성해야 하는 경우:

```bash
# Generate new keypair
solana-keygen new --outfile ~/.config/solana/kora-keypair.json
```

테스트/노드 운영을 위해 충분한 SOL을 입금해야 합니다.

## 4단계: 재배포

변수를 저장한 후 서버를 재배포하라는 메시지가 표시됩니다. 재배포를 진행하세요.

## 5단계: 배포 테스트

### 공개 엔드포인트 생성

엔드포인트로 요청을 보내려면 공개 도메인을 생성해야 합니다. 배포에서
"Settings" > "Generate domain"으로 이동하세요.

![Railway 도메인 설정](/assets/docs/tools/kora/deploy/railway-domain.jpg)

기본 포트인 "8080"을 사용하고 "Generate Domain"을 클릭하세요.

`https://my-kora-node.railway.app/`와 같은 도메인을 받아야 합니다

### 테스트 구성 엔드포인트

```bash
curl -X POST https://my-kora-node.railway.app \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"getConfig","params":[]}' | jq
```

**예상 응답:**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "fee_payer": "YourPublicKeyHere...",
    "validation_config": {
      "max_allowed_lamports": 1000000,
      "allowed_programs": [...],
      "allowed_spl_paid_tokens": [...]
      // ...
    }
  }
}
```

잘하셨습니다! 이제 Kora 노드가 실행 중입니다.
