드롭인 컴포넌트, Solana Pay 프로토콜 또는 직접 RPC 통합을 사용하여 Solana에서
결제를 수락하세요.

## 수신 주소 규칙

Solana에서는 **토큰 계정이 아닌 지갑 주소를 공유하세요**. 일반적으로 발신자가
전송하는 토큰에 대해 적절한
[연결 토큰 계정(ATA)](/docs/payments/how-payments-work#token-accounts)을
파생합니다(자세한 내용은 [결제 전송](/docs/payments/send-payments) 참조).

**예시:** `7EcDhS...`(지갑)을 공유하고, `9xKpQ...`(USDC 토큰 계정)은 공유하지
마세요.

<Callout>
  POS 및 전자상거래의 경우 QR 코드 및 결제 URL은 [Solana
  Pay](/docs/payments/accept-payments/solana-pay)를, 드롭인 React 컴포넌트는
  [결제 버튼](/docs/payments/accept-payments/payment-button)을 참조하세요.
</Callout>

## 통합 옵션

| 접근 방식                                                  | 적합한 용도                | 복잡도 |
| ---------------------------------------------------------- | -------------------------- | ------ |
| [결제 버튼](/docs/payments/accept-payments/payment-button) | React 앱, 빠른 통합        | 낮음   |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)    | QR 코드, 결제 링크, POS    | 보통   |
| [직접 RPC](/docs/payments/interacting-with-solana)         | 맞춤형 플로우, 완전한 제어 | 높음   |

## 검증

결제를 받은 후 [검증 도구](/docs/payments/accept-payments/verification-tools)를
사용하여 결제가 완료되었는지 확인하세요. 대량 트랜잭션 애플리케이션의 경우
프로덕션급 트랜잭션 모니터링을 위해
[인덱싱](/docs/payments/accept-payments/indexing)을 참조하세요.
