description: Pay.sh로 결제를 수락하세요.
---

Pay 서버를 설정하여 Pay.sh를 통해 API 요청에 대한 결제를 수락할 수 있습니다.
Pay를 설치하려면 다음을 실행하세요:

```bash
brew install pay
```

또는

```bash
npm install -g @solana/pay
```

다음 명령어로 설치를 확인하세요:

```bash
pay --version
```

툴체인이 자동으로 주소를 설정해 드립니다. 다음을 실행하여 지갑을 설정하고
충전하세요:

```bash
pay setup
```

## 데모 서버 설정

서버를 시작하여 결제 수락을 시작할 수 있습니다. 다음 명령어로 데모 서버가
동작하는 것을 확인하세요:

```bash
pay server demo
```

이 명령어는 두 가지 작업을 수행합니다. HTTP 요청을 통해 쿼리할 수 있는 라이브
Pay 서버를 시작하고, 현재 디렉터리에 `pay-demo.yml` 파일을 생성합니다.

모든 API 경로는 `endpoints` YAML 설정 파일을 통해 구성됩니다. 결제를 수락하려면
`metering:` 속성을 추가해야 합니다. `/api/v1/reports/usage` 경로는 예시로 이미
구성되어 있습니다:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

다른 터미널에서 클라이언트로 다음 명령어를 실행하여 결제가 정상적으로 작동하는지
테스트할 수 있습니다:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

가격 책정 방법에 대해 더 알아보려면
[Pay.sh 빌딩 가이드 > 가격 책정 문서](https://pay.sh/docs/building-with-pay/pricing)를
참고하세요.

프로덕션 환경에서 Pay.sh를 설정하는 방법에 대해 더 알아보려면
[Pay.sh 구성 문서](https://pay.sh/docs/toolchain/configuration)를 참고하세요.
