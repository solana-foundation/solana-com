이전 섹션에서는 계정을 가져와 Solana 네트워크에서
[데이터를 읽는 방법](/docs/intro/quick-start/reading-from-network)을 배웠습니다.
Solana 네트워크에 데이터를 쓰려면 [트랜잭션](/docs/core/transactions)이
필요합니다. 트랜잭션에는 하나 이상의 [명령어](/docs/core/instructions)가
포함되며, 각 명령어는 프로그램을 호출합니다.

프로그램은 각 명령어의 비즈니스 로직을 정의합니다. 트랜잭션을 전송하면 Solana
런타임이 트랜잭션의 명령어를 순서대로 실행합니다. 트랜잭션은 원자적입니다.
트랜잭션 내의 모든 명령어가 성공하거나, 전체 트랜잭션이 실패합니다.

이 섹션의 예제에서는 다음을 수행하는 방법을 보여줍니다:

1. 계정 간 SOL 전송
2. 새 토큰 민트 생성

## SOL 전송

아래 예제는 한 계정에서 다른 계정으로 SOL을 전송합니다. 계정의 소유자로 지정된
프로그램만 계정의 데이터를 수정하거나 잔액에서 lamport를 차감할 수 있습니다.
지갑 계정은 [System Program](/docs/core/programs#the-system-program)이
소유하므로, 지갑 계정 간 SOL 전송에는 System Program의
[transfer](https://github.com/anza-xyz/agave/blob/v2.1.11/programs/system/src/system_processor.rs#L183-L213)
명령어를 호출하는 명령어가 필요합니다. 소스 계정도 트랜잭션에 서명해야 합니다.

<WithNotes>

<CodeTabs flags="r">

```ts !! title="Transfer SOL"
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { solanaRpc, rpcAirdrop } from "@solana/kit-plugin-rpc";
import { generatedPayer, airdropPayer } from "@solana/kit-plugin-signer";
import { getTransferSolInstruction } from "@solana-program/system";

// !tooltip[/createClient/] client
const client = await createClient()
  .use(generatedPayer())
  .use(
    solanaRpc({
      rpcUrl: "http://localhost:8899",
      rpcSubscriptionsUrl: "ws://localhost:8900"
    })
  )
  .use(rpcAirdrop())
  .use(airdropPayer(lamports(1_000_000_000n)));

// !tooltip[/receiver/] receiver
const receiver = await generateKeyPairSigner();

// !tooltip[/transferInstruction/] transferInstruction
const transferInstruction = getTransferSolInstruction({
  source: client.payer,
  destination: receiver.address,
  amount: lamports(10_000_000n)
});

// !tooltip[/sendTransaction/] sendTransaction
const result = await client.sendTransaction([transferInstruction]);

console.log("Transaction Signature:", result.context.signature);

// !tooltip[/balances/] balances
const { value: senderBalance } = await client.rpc
  .getBalance(client.payer.address)
  .send();
const { value: receiverBalance } = await client.rpc
  .getBalance(receiver.address)
  .send();

console.log("Sender Balance:", senderBalance);
console.log("Receiver Balance:", receiverBalance);
```

</CodeTabs>

### !client

로컬 validator용 Kit 클라이언트를 생성합니다. `generatedPayer()`는 payer
서명자를 추가하고, `solanaRpc()`는 RPC 및 트랜잭션 전송 기능을 추가하며,
`rpcAirdrop()`는 `client.airdrop`를 추가하고, `airdropPayer()`는 payer에게
테스트 SOL을 충전합니다.

### !receiver

SOL을 수신할 계정의 서명자를 생성합니다. 수신자는 이 전송에 서명할 필요가
없습니다. 수신자 주소에 아직 계정이 없는 경우, SOL을 수신하면 해당 주소에 시스템
계정이 생성됩니다.

### !transferInstruction

클라이언트 지불자에서 수신자로 lamport를 전송하는 System Program 명령어를
빌드합니다.

### !sendTransaction

명령어를 트랜잭션으로 전송합니다. Kit 클라이언트는 최근 블록해시를 가져오고,
수수료 지불자를 설정하고, 필요한 서명자로 서명하고, 트랜잭션을 전송한 후 확인을
기다립니다.

### !balances

트랜잭션이 확인된 후 송신자와 수신자의 잔액을 가져옵니다.

</WithNotes>

## 토큰 생성

아래 예제는 [Token Extensions Program](/docs/tokens/extensions)을 사용하여
새로운 토큰 mint를 생성합니다. mint account는 소수점 자릿수, 공급량, mint 권한
및 동결 권한과 같은 토큰의 전역 설정을 정의하는 계정입니다.

mint account를 생성하려면 두 가지 명령어가 필요합니다:

1. System Program을 호출하여 Token Extensions Program이 소유하는 새 계정을
   생성합니다.
2. Token Extensions Program을 호출하여 해당 계정을 mint로 초기화합니다.

<WithNotes>

<CodeTabs flags="r">

```ts !! title="Create mint account"
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { solanaRpc, rpcAirdrop } from "@solana/kit-plugin-rpc";
import { generatedPayer, airdropPayer } from "@solana/kit-plugin-signer";
import { getCreateAccountInstruction } from "@solana-program/system";
import {
  fetchMint,
  getInitializeMintInstruction,
  getMintSize,
  TOKEN_2022_PROGRAM_ADDRESS
} from "@solana-program/token-2022";

// !tooltip[/createClient/] client
const client = await createClient()
  .use(generatedPayer())
  .use(
    solanaRpc({
      rpcUrl: "http://localhost:8899",
      rpcSubscriptionsUrl: "ws://localhost:8900"
    })
  )
  .use(rpcAirdrop())
  .use(airdropPayer(lamports(1_000_000_000n)));

// !tooltip[/mint/] mint
const mint = await generateKeyPairSigner();

// !tooltip[/space/] space
const space = BigInt(getMintSize());

// !tooltip[/rent/] rent
const rent = await client.rpc.getMinimumBalanceForRentExemption(space).send();

// !tooltip[/createAccountInstruction/] createAccountInstruction
const createAccountInstruction = getCreateAccountInstruction({
  payer: client.payer,
  newAccount: mint,
  space,
  lamports: rent,
  programAddress: TOKEN_2022_PROGRAM_ADDRESS
});

// !tooltip[/initializeMintInstruction/] initializeMintInstruction
const initializeMintInstruction = getInitializeMintInstruction({
  mint: mint.address,
  decimals: 2,
  mintAuthority: client.payer.address,
  freezeAuthority: client.payer.address,
  tokenProgram: TOKEN_2022_PROGRAM_ADDRESS
});

// !tooltip[/sendTransaction/] sendTransaction
const result = await client.sendTransaction([
  createAccountInstruction,
  initializeMintInstruction
]);

console.log("Mint Address:", mint.address);
console.log("Transaction Signature:", result.context.signature);

// !tooltip[/mintAccount/] mintAccount
const mintAccount = await fetchMint(client.rpc, mint.address);
console.log("Mint Account:", mintAccount);
```

</CodeTabs>

### !client

로컬 validator용 Kit 클라이언트를 생성하고 자금을 지원합니다. 자금이 있는
payer는 계정 rent와 트랜잭션 수수료를 지불합니다.

### !mint

새 mint account의 주소로 사용할 서명자를 생성합니다. 이 서명자는 해당 계정
생성을 승인합니다.

### !space

확장 기능이 없는 mint account에 필요한 바이트 수를 계산합니다.

### !rent

mint account를 rent 면제 상태로 만들기 위해 필요한 lamport를 계산합니다.

### !createAccountInstruction

mint account를 생성하고 해당 계정을 Token Extensions Program에 할당하는 System
Program 명령어를 작성합니다.

### !initializeMintInstruction

새 계정에 초기 mint 데이터를 기록하는 Token Extensions Program 명령어를
작성합니다.

### !sendTransaction

두 명령어를 하나의 트랜잭션으로 전송합니다. 명령어 순서가 중요합니다: 계정을
초기화하기 전에 먼저 생성되어야 합니다.

### !mintAccount

확인 후 mint account를 가져옵니다. 헬퍼는 계정을 읽고 mint account의 데이터
필드를 Mint 타입으로 역직렬화합니다.

</WithNotes>

<WithMentions>

두 번째 명령어는 Token Extensions Program을 호출합니다. 이 명령어는
[`mint`](mention:mint-account) 주소를 [`decimals`](mention:decimals) 값,
[`mintAuthority`](mention:mint-authority),
[`freezeAuthority`](mention:freeze-authority)로 초기화하고, mint account를
소유하는 [`tokenProgram`](mention:token-program)을 지정합니다.
