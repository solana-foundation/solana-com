description:
  이 섹션에서는 시작하는 데 도움이 되는 일반적인 Anchor CLI 명령어를 안내합니다.
---

이 섹션에서는 [Anchor CLI](https://www.anchor-lang.com/docs) 사용을 시작하는 데
도움이 되는 일반적인 명령어와 예제를 제공합니다.

<Steps>
<Step>

### 프로젝트 초기화

아래 명령어를 실행하여 새로운 Anchor 프로젝트를 생성하세요. 프로젝트 이름으로 새
디렉토리를 생성하고 이를 사용하여 새로운 Anchor 프로젝트를 초기화합니다.

```terminal
$ anchor init <project-name>
```

예를 들어, 아래 명령어는 `my-project`라는 프로젝트를 생성합니다. `my-project`
디렉토리에는 기본 Rust 프로그램과 TypeScript 테스트 템플릿이 포함됩니다.

```terminal
$ anchor init my-project
```

그런 다음 프로젝트 디렉토리로 이동하세요:

```terminal
$ cd <project-name>
```

Anchor의
[프로젝트 파일 구조](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)를
참조하세요.

</Step>
<Step>

### 프로그램 빌드

프로젝트를 빌드하려면 다음 명령어를 실행하세요:

```terminal
$ anchor build
```

컴파일된 프로그램은 `/target/deploy` 디렉토리에서 찾을 수 있습니다.

<Callout type="warn">

`anchor build`를 실행할 때 다음 오류가 발생하면:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

다음 해결 방법을 시도해 보세요:

1. 다음 명령어를 사용하여 강제 설치하세요:

```terminal
$ cargo build-sbf --force-tools-install
```

2. 앞의 해결 방법이 작동하지 않으면 Solana 캐시를 삭제하세요:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
`Cargo.lock` 파일의 version 필드를 4에서 3으로 변경하여 이 문제를 해결할 수 있습니다:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

자세한 내용은
[이 이슈](https://github.com/solana-foundation/anchor/issues/3392)를 참조하세요.

</Accordion>

</Accordions>

앞의 해결 방법을 적용한 후 *shell`anchor build`*를 다시 실행해 보세요.

</Callout>

</Step>
<Step>

### 프로그램 배포

프로젝트를 배포하려면 다음 명령어를 실행하세요:

```terminal
$ anchor deploy
```

이 명령어는
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml) 파일에
지정된 `cluster`에 프로그램을 배포합니다.

</Step>
<Step>

### 프로그램 테스트

프로젝트를 테스트하려면 다음 명령어를 실행하세요:

```terminal
$ anchor test
```

이 명령어는 프로젝트를 빌드하고 배포한 후 테스트를 실행합니다.

<Callout>
  `localnet` 클러스터를 사용할 때 Anchor는 자동으로 로컬 validator를 시작하고
  프로그램을 배포한 후 테스트를 실행하고 validator를 중지합니다.
</Callout>

<Callout type="warn">

다음 오류 중 하나가 발생하면 Node.js 또는 Yarn이 설치되지 않았음을 나타낼 수
있습니다:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
