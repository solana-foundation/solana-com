한 번의 명령으로 로컬 Solana 개발 환경을 설정하는 빠른 설치 방법입니다.
Windows(WSL), Linux, Mac에서 Rust, Solana CLI, Anchor Framework를 설치하세요.
(원하시는 경우
[각 종속성을 개별적으로 설치](/docs/intro/installation/dependencies)할 수도
있습니다.)

## 사전 요구 사항

<Accordions>
<Accordion title="Windows">

Windows에서 Solana 프로그램을 개발하려면 **반드시**
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)(WSL)를
사용해야 합니다. WSL을 설치한 후 다른 모든 종속성은 Linux 터미널을 통해
설치됩니다.

WSL을 설치하려면 Windows PowerShell에서 다음 명령을 실행하세요:

```terminal
$ wsl --install
```

설치 과정에서 기본 사용자 계정을 생성하라는 메시지가 표시됩니다.

![WSL 설치](/assets/docs/intro/installation/wsl-install.png)

기본적으로 WSL은 Ubuntu를 설치합니다. 검색 창에서 "Ubuntu"를 검색하여 Linux
터미널을 열 수 있습니다.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Ubuntu 터미널이 아래 이미지와 같이 보인다면 `ctrl + v`(붙여넣기 단축키)가
터미널에서 작동하지 않는 문제가 발생할 수 있습니다.

![Ubuntu 터미널](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

이 문제가 발생하면 검색 창에서 "Terminal"을 검색하여 Windows Terminal을 여세요.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

다음으로 Windows Terminal을 닫고 Ubuntu를 다시 검색하여 Linux 터미널을 다시
여세요. 이제 터미널이 아래 이미지처럼 보여야 하며 `ctrl + v`(붙여넣기 단축키)가
작동합니다.

![Ubuntu 터미널](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

VS Code를 사용하는 경우
[WSL 확장 프로그램](https://code.visualstudio.com/docs/remote/wsl-tutorial)을
사용하면 WSL과 VS Code를 함께 사용할 수 있습니다.

![VS Code에서 WSL 설정](/assets/docs/intro/installation/wsl-vscode.png)

VS Code 상태 표시줄에 다음과 같이 표시됩니다:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

WSL 설정이 완료되면 다음 중 하나를 진행할 수 있습니다:

- Ubuntu(Linux) 터미널을 사용하여 [빠른 설치](#installation) 완료하기
- 아래 설명된 대로 [Rust](/docs/intro/installation/dependencies#install-rust),
  [Solana CLI](/docs/intro/installation/dependencies#install-solana-cli) 및
  [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli)를
  개별적으로 설치하기

</Accordion>

<Tab value=".deb" >
Ubuntu나 Mint와 같은 Debian 또는 Debian 기반 배포판을 사용하는 경우 아래 단계를 따르세요:

1. 모든 패키지가 최신 상태인지 확인하기 위해 패키지 목록을 업데이트합니다:

```terminal
$ sudo apt-get update
```

2. 다음 의존성을 설치합니다:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
`protobuf-compiler` 설치 시 다음 오류가 발생하면,
먼저 `sudo apt-get update`를 실행해야 합니다

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Fedora 또는 Red Hat Linux와 같은 RHEL 기반 배포판의 경우 다음 단계를 따르세요.

다음 명령어를 실행하여 패키지의 최신 버전을 가져오기 위해 패키지 목록을
업데이트합니다.

```terminal
$ sudo dnf upgrade --refresh
```

다음으로, 다음 의존성을 설치합니다:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## 설치

1. 터미널에 다음 명령어를 입력하여 필요한 모든 의존성을 설치합니다:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

설치가 성공하면 다음과 같은 출력이 반환됩니다:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. 설치된 각 종속성의 버전을 확인하여 설치가 성공했는지 확인합니다.

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  빠른 설치 명령이 실패하는 경우, 각 종속성을 개별적으로 설치하는 방법은 아래의
  [종속성 설치](/docs/intro/installation/dependencies) 섹션을 참조하세요.
</Callout>
