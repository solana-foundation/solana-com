<Callout type="info" title="Resumen">
  Verificaciones del runtime después de cada instrucción: solo el propietario
  puede debitar lamports o modificar datos, los datos pueden crecer máximo 10
  KiB por instrucción, los cambios de propietario requieren datos inicializados
  en cero, y la bandera ejecutable es irreversible.
</Callout>

El runtime de Solana aplica estas reglas después de que cada instrucción se
ejecuta mediante los métodos
[`BorrowedInstructionAccount`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L722-L727).
Cada regla se verifica en el punto de modificación, y la transacción se revierte
si alguna verificación falla.

## Reglas de lamports

| Regla                                                               | Aplicación                                                                                                                                                                                                                                       | Error                         |
| ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------- |
| Solo el propietario puede debitar lamports                          | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L785): verifica `is_owned_by_current_program()` cuando `lamports < current`                                                                 | `ExternalAccountLamportSpend` |
| Las cuentas de solo lectura no pueden tener lamports modificados    | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L788-L789): verifica `is_writable()`                                                                                                             | `ReadonlyLamportChange`       |
| Cualquier programa puede acreditar lamports a una cuenta escribible | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L789): la verificación de propiedad solo aplica cuando el nuevo saldo es menor que el saldo actual; la verificación de escritura aún aplica | `ReadonlyLamportChange`       |
| Los lamports deben balancearse a través de una instrucción          | [`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L366-L398): verifica `get_lamports_delta() == 0`                                                                                      | `UnbalancedInstruction`       |

## Transiciones de estado de rent

Las cuentas existen en uno de tres valores
[`RentState`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L19-L30):
_rs`Uninitialized`_ (0 lamports), _rs`RentPaying`_ (por encima de 0 pero por
debajo del mínimo exento de rent), y _rs`RentExempt`_ (igual o por encima del
mínimo). Las transiciones no permitidas producen
_rs`TransactionError::InsufficientFundsForRent`_.

El runtime aplica estas reglas mediante
[`transition_allowed()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L107-L126):

- Cualquier cuenta puede transicionar a _rs`Uninitialized`_ (cerrar) o
  _rs`RentExempt`_.
- Ninguna cuenta puede entrar en _rs`RentPaying`_ desde _rs`Uninitialized`_ o
  _rs`RentExempt`_. Todas las cuentas nuevas deben estar exentas de rent.

## Reglas de préstamo de cuentas

Durante la ejecución de instrucciones, el runtime aplica semántica de préstamo
de escritor único en las cuentas. Un programa puede obtener una referencia
mutable o múltiples referencias inmutables a una cuenta, pero no ambas
simultáneamente. Si un programa intenta tomar prestada una cuenta que ya está
prestada de forma mutable (o prestar de forma mutable una cuenta que ya está
prestada de forma inmutable), la instrucción falla con _rs`AccountBorrowFailed`_
mediante
[`try_borrow()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L498-L508)
y
[`try_borrow_mut()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L512-L520).
Si una instrucción se completa mientras un préstamo aún está pendiente, el
runtime devuelve _rs`AccountBorrowOutstanding`_ en
[`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L374-L385).
