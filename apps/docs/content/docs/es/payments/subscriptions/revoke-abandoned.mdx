---
title: Revocar Delegaciones y Suscripciones Abandonadas
description:
  Guía para recuperar rent de una delegación o suscripción cuya Autoridad de
  Suscripción ha desaparecido.
---

Las delegaciones y suscripciones retienen rent que normalmente se devuelve a su
pagador registrado al revocarlas. Si un patrocinador financió la cuenta y el
usuario posteriormente cierra o reemplaza su Autoridad de Suscripción, la cuenta
queda abandonada y su rent bloqueada. `RevokeAbandonedDelegation` e
`RevokeAbandonedSubscription` permiten que el pagador registrado la recupere.

## Cuándo Aplica

Solo se permite cuando la Autoridad de Suscripción de la cuenta es
demostrablemente terminal — cerrada, o cerrada y recreada (de modo que `init_id`
ya no coincide). Una delegación o suscripción activa nunca puede cerrarse de
esta manera.

|                                  | `RevokeAbandonedDelegation`                                                                                                | `RevokeAbandonedSubscription`                                                                                                                            |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Cierra                           | PDA de delegación fija o recurrente                                                                                        | PDA de suscripción                                                                                                                                       |
| Firmante                         | Pagador registrado                                                                                                         | Pagador registrado                                                                                                                                       |
| Cuentas                          | Delegación + autoridad registrada                                                                                          | Suscripción + autoridad registrada + plan                                                                                                                |
| Elegible cuando                  | Autoridad cerrada o recreada (discrepancia en `init_id`)                                                                   | Lo mismo, para el mint del plan — el mint se lee del plan vinculado, por lo que no se puede sustituir una autoridad no relacionada para forzar el cierre |
| Usar revocación ordinaria cuando | La delegación está [vencida o totalmente gastada](/docs/payments/subscriptions/recurring-delegation#revoke-the-delegation) | La suscripción está [cancelada y vencida](/docs/payments/subscriptions/subscription-plan#cancel-and-revoke)                                              |

`RevokeAbandonedDelegation` es también el único camino de recuperación del
pagador registrado para una delegación que nunca vence (`expiry_ts == 0`) y que
no está completamente gastada: la revocación ordinaria se activa para el pagador
solo al vencer o, en el caso de una delegación fija, al gastarse por completo.

## Revocar una Delegación Abandonada

El pagador registrado firma. Se pasa el PDA de la delegación inactiva y su
Autoridad de Suscripción registrada (que puede ya estar cerrada). El rent se
devuelve al pagador.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedDelegationInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedDelegationInstruction({
      payer: payerSigner,
      delegationAccount: delegationPda,
      subscriptionAuthority: subscriptionAuthorityPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let delegation_pda: Address = address!("DELEGATION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");

    let revoke_abandoned_ix = RevokeAbandonedDelegationBuilder::new()
        .payer(payer)
        .delegation_account(delegation_pda)
        .subscription_authority(subscription_authority)
        .instruction();
    ```

  </Tab>
</Tabs>

## Revocar una Suscripción Abandonada

El pagador registrado firma. Pasa el PDA de la suscripción abandonada, su
Autoridad de Suscripción registrada (que puede que ya esté cerrada) y el plan al
que pertenece la suscripción. El rent se devuelve al pagador.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedSubscriptionInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedSubscriptionInstruction({
      payer: payerSigner,
      subscriptionAccount: subscriptionPda,
      subscriptionAuthority: subscriptionAuthorityPda,
      planPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let subscription_pda: Address = address!("SUBSCRIPTION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");
    let plan_pda: Address = address!("PLAN_PDA_ADDRESS_HERE");

    let revoke_abandoned_ix = RevokeAbandonedSubscriptionBuilder::new()
        .payer(payer)
        .subscription_account(subscription_pda)
        .subscription_authority(subscription_authority)
        .plan_pda(plan_pda)
        .instruction();
    ```

  </Tab>
</Tabs>

## Notas

- El pagador registrado firma, no el delegante, el delegado ni el suscriptor.
- Ambas instrucciones fallan si la Autoridad de Suscripción sigue activa para la
  cuenta — usa la revocación ordinaria en su lugar.
- `RevokeAbandonedDelegation` funciona tanto para delegaciones fijas como
  recurrentes.
