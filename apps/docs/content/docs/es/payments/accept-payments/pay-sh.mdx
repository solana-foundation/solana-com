---
title: Con Pay.sh
description: Acepta pagos con Pay.sh.
---

Puedes aceptar pagos en solicitudes de API a través de Pay.sh configurando un
servidor Pay. Instala Pay con:

```bash
brew install pay
```

o

```bash
npm install -g @solana/pay
```

Verifica la instalación con:

```bash
pay --version
```

La cadena de herramientas configurará tu dirección automáticamente. Ejecuta lo
siguiente para configurar y recargar tu billetera:

```bash
pay setup
```

## Configuración del servidor de demostración

Puedes comenzar a aceptar pagos iniciando tu servidor. Puedes ver un servidor de
demo en acción con:

```bash
pay server demo
```

Esto hace dos cosas. Inicia un servidor Pay en vivo que puedes consultar
mediante solicitudes HTTP y genera un archivo `pay-demo.yml` en tu directorio
actual.

Todas las rutas de la API se configuran mediante el archivo de configuración
YAML `endpoints`. Para aceptar pagos, debes agregar un atributo `metering:`. La
ruta `/api/v1/reports/usage` ya está hecha como ejemplo:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Puedes comprobar que los pagos funcionan ejecutando el siguiente comando como
cliente en otra terminal:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Puedes obtener más información sobre cómo definir precios en la
[documentación de Pay.sh: Construir con Pay > Precios](https://pay.sh/docs/building-with-pay/pricing)

Obtén más información sobre cómo configurar Pay.sh en producción en la
[documentación de configuración de Pay.sh](https://pay.sh/docs/toolchain/configuration).
