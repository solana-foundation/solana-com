Acepta pagos en Solana usando componentes listos para usar, el protocolo Solana
Pay o integración directa con RPC.

## Convención de dirección de recepción

En Solana, **comparte tu dirección de wallet, no tu token account**. Es típico
que el remitente derive la
[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)
apropiada para el token que está enviando (consulta
[Enviar pagos](/docs/payments/send-payments) para más detalles).

**Ejemplo:** Comparte `7EcDhS...` (tu wallet), no `9xKpQ...` (tu token account
de USDC).

<Callout>
  Para punto de venta y comercio electrónico, consulta [Solana
  Pay](/docs/payments/accept-payments/solana-pay) para códigos QR y URLs de
  pago, o [Payment Button](/docs/payments/accept-payments/payment-button) para
  componentes React listos para usar.
</Callout>

## Opciones de integración

| Enfoque                                                         | Ideal para                           | Complejidad |
| --------------------------------------------------------------- | ------------------------------------ | ----------- |
| [Payment Button](/docs/payments/accept-payments/payment-button) | Apps React, integración rápida       | Baja        |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | Códigos QR, enlaces de pago, POS     | Media       |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Flujos personalizados, control total | Mayor       |

## Verificación

Después de recibir un pago, verifica que se haya recibido usando
[Herramientas de verificación](/docs/payments/accept-payments/verification-tools).
Para aplicaciones de alto volumen, consulta
[Indexing](/docs/payments/accept-payments/indexing) para monitoreo de
transacciones de nivel producción.
