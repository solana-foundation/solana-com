## ¿Qué son las comisiones de transferencia?

La extensión de mint _rs`TransferFeeConfig`_ del Token Extensions Program aplica
una comisión a cada transferencia para ese mint.

Al momento de la transferencia:

- La cantidad transferida se reduce según la comisión configurada
- La comisión retenida se registra en la cuenta de tokens de destino
- Las comisiones retenidas pueden retirarse directamente desde las cuentas de
  tokens o recolectarse en el mint
- El `withdraw_withheld_authority` puede mover las comisiones recolectadas a una
  cuenta de tokens receptora de comisiones

<Callout type="info" title="Extensiones de cuenta de tokens">
  Cuando se inicializa una cuenta de tokens para un mint con
  _rs`TransferFeeConfig`_, la cuenta de tokens se inicializa con
  _rs`TransferFeeAmount`_. Cuando la cuenta de tokens se crea a través del
  [Associated Token
  Program](/docs/tokens/basics/create-token-account#how-to-create-an-associated-token-account),
  se calcula el tamaño de cuenta requerido y la cuenta de tokens se crea con el
  tamaño necesario y los lamports exentos de renta.
</Callout>

## Cómo crear un mint con comisiones de transferencia

Para crear un mint con comisiones de transferencia:

1. Calcula el tamaño de la cuenta del mint y la renta necesaria para el mint y
   la extensión _rs`TransferFeeConfig`_.
2. Crea la cuenta del mint con _rs`CreateAccount`_, inicializa
   _rs`TransferFeeConfig`_ e inicializa el mint con _rs`InitializeMint`_.
3. Crea cuentas de tokens para el mint. _rs`TransferFeeAmount`_ se habilita
   automáticamente para las cuentas de tokens.
4. Usa _rs`TransferCheckedWithFee`_ para transferir tokens y verifica la
   comisión esperada. _rs`TransferChecked`_ también funciona y automáticamente
   retiene la comisión configurada en la cuenta de tokens de destino.
5. Usa _rs`WithdrawWithheldTokensFromAccounts`_ para mover las comisiones
   retenidas directamente desde las cuentas de tokens a una cuenta de tokens
   receptora de comisiones, firmado por el `withdraw_withheld_authority` del
   mint.
6. Usa _rs`HarvestWithheldTokensToMint`_ sin permisos para mover las comisiones
   retenidas desde las cuentas de tokens a la cuenta del mint, luego usa
   _rs`WithdrawWithheldTokensFromMint`_ para retirar las comisiones retenidas de
   la cuenta del mint a una cuenta de tokens receptora de comisiones, firmado
   por el `withdraw_withheld_authority` del mint.
7. Usa _rs`SetTransferFee`_ para actualizar la siguiente configuración de
   comisión de transferencia, que entrará en vigor dos epochs después.

## !!steps Calcular el tamaño de la cuenta

Calcula el tamaño de la cuenta mint para el mint base más la extensión
_rs`TransferFeeConfig`_. Este es el tamaño utilizado en _rs`CreateAccount`_.

<CodePlaceholder title="Ejemplo" />

```ts !! title="Example"
const client = await createClient()
  .use(generatedPayer())
  .use(
    solanaRpc({
      rpcUrl: "http://localhost:8899",
      rpcSubscriptionsUrl: "ws://localhost:8900"
    })
  )
  .use(rpcAirdrop())
  .use(airdropPayer(lamports(1_000_000_000n)));

const mint = await generateKeyPairSigner();

// !focus(1:14)
const transferFeeConfigExtension = extension("TransferFeeConfig", {
  transferFeeConfigAuthority: client.payer.address,
  withdrawWithheldAuthority: client.payer.address,
  withheldAmount: 0n,
  olderTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  },
  newerTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  }
});

// !focus(1:1)
const mintSpace = BigInt(getMintSize([transferFeeConfigExtension]));
```
