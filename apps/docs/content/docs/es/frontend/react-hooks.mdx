`@solana/react` publica un cliente Kit en tu árbol de componentes a través de un
proveedor, y luego los hooks leen datos RPC, suscripciones y el estado de la
wallet desde él. Los hooks de wallet del navegador se encuentran en
`@solana/kit-plugin-wallet/react` y leen del mismo proveedor — no existe un
proveedor de wallet independiente.

## Instalación

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Construir el cliente

Construye el cliente una sola vez, en el ámbito del módulo, fuera del árbol de
React. En el navegador, `walletSigner` establece la wallet conectada como el
pagador de comisiones y la identidad.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` añade `client.wallet` (descubrimiento y conexión con Wallet
Standard), y `solanaRpc` añade `client.rpc` junto con el helper
`client.sendTransaction`. Exporta el tipo de cliente envuelto en `Awaited<>`
para que permanezca correcto si más adelante añades un plugin que se resuelva de
forma asíncrona, y pasa ese `AppClient` a cada llamada a `useClient`.

## Envuelve tu árbol una sola vez

`ClientProvider` publica el cliente. Todos los hooks que se muestran a
continuación leen desde él.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Hooks de wallet

Los hooks de wallet leen la capacidad `wallet` del cliente, por lo que solo
funcionan dentro de un `ClientProvider` cuyo cliente tenga instalado un plugin
de wallet.

- `useWallets()` — Wallets de Wallet Standard descubiertas para la cadena del
  cliente.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` o `null`. `signer` es
  `null` para wallets de solo lectura.
- `useConnect()` / `useDisconnect()` — devuelven un `ActionResult` cuyo
  `dispatch` (y `dispatchAsync`) ejecuta la acción y cuyos `isRunning`,
  `status`, `data`, `error` e `reset` rastrean su resultado.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Enviar una transacción

`useClient` devuelve el mismo cliente que publicaste. Construye instrucciones
con un cliente de programa generado y envíalas a través de
`client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Pasar el tipo `AppClient` exportado a `useClient` le otorga al hook el cliente
completo extendido por plugins, de modo que `client.rpc` e
`client.sendTransaction` están tipados.

## Leer datos

`@solana/react` incluye hooks que leen a través del cliente:

- `useRequest(source)` — lectura RPC puntual; devuelve
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — una suscripción en vivo; devuelve
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — un valor inicializado por RPC que se mantiene
  actualizado mediante una suscripción.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Memoiza la fuente (con `useMemo`, o `useCallback` para una función) para que el
hook vuelva a obtener datos solo cuando cambien sus entradas y no en cada
renderizado.

Para el almacenamiento en caché y la revalidación, las subrutas
`@solana/react/swr` e `@solana/react/query` envuelven los mismos hooks para
[SWR](https://swr.vercel.app) y [TanStack Query](https://tanstack.com/query).

## Patrones comunes para desarrolladores de Solana

- **Un solo proveedor**: `ClientProvider` es el único proveedor que necesitas;
  los hooks de wallet lo leen a través de la capacidad `wallet` del cliente.
- **Compatible con componentes de servidor**: Marca solo los componentes hoja
  que invocan hooks con `"use client"`; las lecturas del servidor pueden usar un
  cliente RPC de Kit simple sin wallet.
- **Pruebas**: Publica un cliente simulado a través de `ClientProvider` para
  simular wallets y respuestas RPC.

<Callout type="warn" title="Dos familias de hooks, mismos nombres">
  `useSignIn` e `useSignMessage` existen tanto en `@solana/react` (reciben un
  argumento `UiWalletAccount`) como en `@solana/kit-plugin-wallet/react`
  (basados en cliente, sin argumento de cuenta, devuelven un `ActionResult`).
  Usa las versiones de `@solana/kit-plugin-wallet/react` con el patrón de
  cliente descrito arriba; los hooks de wallet de `@solana/react` más antiguos
  están siendo reemplazados.
</Callout>

Complementa esta guía con la
[descripción general del cliente Kit](/docs/frontend/client) para comprender el
cliente del que lee cada hook.
