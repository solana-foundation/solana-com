title: Migración a Kit
description:
  Cómo los conceptos de @solana/web3.js y wallet-adapter se corresponden con
  @solana/kit.
---

Si vienes de `@solana/web3.js` (v1), `@solana/wallet-adapter-*`, o los paquetes
anteriores del framework-kit (`@solana/client`, `@solana/react-hooks`,
`@solana/web3-compat`), esos enfoques han sido reemplazados. Las nuevas
aplicaciones se construyen directamente sobre `@solana/kit` y los plugins de
Kit. Esta página relaciona los conceptos que ya conoces con sus equivalentes en
Kit.

## Correspondencia de conceptos

| web3.js v1 / wallet-adapter            | Kit                                                                                     |
| -------------------------------------- | --------------------------------------------------------------------------------------- |
| `new Connection(url)`                  | `createSolanaRpc(url)`, o `.use(solanaRpc({ rpcUrl }))` en un cliente                   |
| `PublicKey`                            | `Address` — convierte un string con `address("...")`                                    |
| `Keypair`                              | `generateKeyPairSigner()` o los plugins de `@solana/kit-plugin-signer`                  |
| `SystemProgram.transfer(...)`          | `getTransferSolInstruction(...)` de `@solana-program/system`                            |
| `LAMPORTS_PER_SOL` math                | `lamports(...)` de `@solana/kit`                                                        |
| `Transaction` / `VersionedTransaction` | Mensajes de transacción construidos y empaquetados por el planificador de transacciones |
| `sendAndConfirmTransaction(...)`       | `client.sendTransaction([...])` (el planificador firma, envía y confirma)               |
| `@solana/wallet-adapter-*`             | `@solana/kit-plugin-wallet` + descubrimiento mediante Wallet Standard                   |

## ¿Vienes de web3.js?

Si mantienes una base de código grande con `@solana/web3.js` v1 y una
reescritura completa a Kit aún no es viable, web3.js v3 es el puente.
Actualmente se distribuye como `@solana/web3.js@rc` (la línea `3.0.0-rc`) y
reconstruye la API clásica basada en clases — `Connection`, `Keypair`,
`Transaction` — sobre los componentes internos de `@solana/kit`. Como se basa en
Kit, la interoperabilidad es estrecha: `PublicKey` es un alias obsoleto de
`Address`, y un `Keypair` de v3 satisface estructuralmente el `KeyPairSigner` de
Kit, por lo que puedes pasarlo directamente a las APIs, plugins y clientes
generados por Codama de Kit.

v3 sigue siendo un **candidato a versión final** — fija versiones exactas y
espera algunos cambios en la API entre RCs. Es una ruta de interoperabilidad con
código heredado, no el punto de partida para proyectos nuevos; las nuevas
aplicaciones deben construirse directamente sobre Kit. Para la migración en sí,
consulta la
[guía oficial de migración de web3.js v1 → v3](https://github.com/solana-foundation/solana-web3.js/blob/v3.x/docs/web3js-v1-to-v3-migration.md).

## Wallets

Kit no utiliza wallet-adapter. `@solana/kit-plugin-wallet` descubre las wallets
de Wallet Standard y las expone a través de `client.wallet` y los
[hooks de React](/docs/frontend/react-hooks). Las wallets modernas (Phantom,
Solflare, Backpack, entre otras) se anuncian a través de Wallet Standard, por lo
que no se requiere ningún paquete adaptador por wallet.

También puedes utilizar herramientas prediseñadas como
[@solana/connector](https://www.npmjs.com/package/@solana/connector), que usa
primitivas de Kit directamente para conectarse a las wallets.

<Callout type="info" title="¿Todavía usas wallet-adapter?">
  Si mantienes una aplicación desarrollada con `@solana/wallet-adapter-*` y aún
  no estás listo para migrar, el [repositorio de wallet-adapter
  ](https://github.com/anza-xyz/wallet-adapter) sigue siendo la referencia para
  ese stack. Para cualquier proyecto nuevo, se recomienda el plugin de wallet de
  Kit.
</Callout>
