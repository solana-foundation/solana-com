Ponte en marcha con Solana Attestations en solo unos pocos pasos. Esta guía te
guiará a través de los conceptos fundamentales y te ayudará a crear tu primera
attestation.

## Lo que aprenderás

- Instalar y configurar la biblioteca de Attestations
- Crear tu primera credencial (autoridad de attestation)
- Definir un esquema para tus attestations
- Emitir tu primera attestation
- Obtener y verificar attestations

## Requisitos previos

- Conocimientos básicos de desarrollo en Solana
- Node.js 16+ o entorno de desarrollo Rust
- Una billetera de Solana con algo de SOL para tarifas de transacción

## Pasos de inicio rápido

### 1. Instalación

Instala los paquetes necesarios y configura tu entorno de desarrollo.

**[→ Guía de instalación](/docs/tools/attestations/quickstart/installation)**

### 2. Crear una credencial

Configura una autoridad de attestation que pueda emitir attestations.

**[→ Guía para crear credenciales](/docs/tools/attestations/instructions/create-credential)**

### 3. Definir un esquema

Crea un esquema que defina la estructura de tus attestations.

**[→ Guía para crear esquemas](/docs/tools/attestations/instructions/create-schema)**

### 4. Emitir una attestation

Crea tu primera declaración verificable usando la credencial y el esquema.

**[→ Guía para crear attestations](/docs/tools/attestations/instructions/create-attestation)**

## Conceptos fundamentales

### Credenciales

Una credencial representa una autoridad de attestation. Define quién puede
emitir attestations y contiene una lista de firmantes autorizados.

### Esquemas

Los esquemas definen la estructura y las reglas de validación para las
attestations. Especifican qué campos contiene una attestation y sus tipos de
datos.

### Certificaciones

Las certificaciones son declaraciones o afirmaciones verificables creadas por
firmantes autorizados bajo una credencial, siguiendo una estructura de esquema
específica.

## Casos de Uso

Las certificaciones se pueden utilizar para:

- **Verificación de Identidad**: Verificar identidades de usuarios y
  cumplimiento normativo
- **Sistemas de Reputación**: Construir puntuaciones de confianza y métricas de
  reputación
- **Certificaciones**: Emitir certificados profesionales y calificaciones
- **Cumplimiento**: Garantizar el cumplimiento normativo para servicios
  financieros
- **Cadena de Suministro**: Rastrear autenticidad y procedencia de productos
