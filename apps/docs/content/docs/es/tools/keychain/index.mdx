**Solana Keychain proporciona una interfaz unificada para firmar transacciones
de Solana a través de múltiples backends de gestión de claves.** Desarrolla
localmente con una clave en memoria, luego cambia a gestión de claves
empresarial, HSMs en la nube o servicios de billetera gestionados en producción
— sin reescribir tu código de firma.

### ¿Por qué Keychain?

- **Interfaz Única**: Un trait `SolanaSigner` que funciona con todos los
  backends
- **Intercambia Backends**: Cambia de proveedor de gestión de claves mediante
  configuración, no código
- **Abstracción de Costo Cero**: Los feature flags incluyen solo lo que
  necesitas
- **Auditado**: Revisado por seguridad por [Accretion](https://accretion.xyz)

### Arquitectura

- **Lenguajes**: Rust + TypeScript, con paridad total
- **Trait**: Interfaz unificada `SolanaSigner`
- **Backends**: Memory, Vault, AWS KMS, GCP KMS, Privy, Turnkey, Fireblocks,
  CDP, Crossmint, Dfns, Openfort, Para, Utila
- **Compatibilidad**: `@solana/kit` e `@solana/signers` compatibles (TypeScript)
  | `solana-sdk` y `solana-sdk-v3` compatibles (Rust)

### Backends Soportados

Los backends abarcan el desarrollo local, la gestión de claves autoalojada y en
la nube, y servicios de billetera gestionados o MPC. Consulta
[Cómo elegir un backend](/docs/tools/keychain/choosing-a-backend) para comparar
modelos de custodia y elegir el más adecuado.

| Backend         | Caso de Uso                         | Rust | TypeScript |
| --------------- | ----------------------------------- | ---- | ---------- |
| Memory          | Desarrollo, pruebas                 | ✓    | ✓          |
| HashiCorp Vault | Gestión de claves autoalojada       | ✓    | ✓          |
| AWS KMS         | Nativo en la nube (AWS)             | ✓    | ✓          |
| GCP KMS         | Nativo en la nube (GCP)             | ✓    | ✓          |
| Privy           | Billeteras integradas               | ✓    | ✓          |
| Turnkey         | Gestión de claves no custodial      | ✓    | ✓          |
| Fireblocks      | Custodia institucional              | ✓    | ✓          |
| CDP             | Coinbase Developer Platform         | ✓    | ✓          |
| Crossmint       | Billeteras gestionadas de Crossmint | ✓    | ✓          |
| Dfns            | Infraestructura de billeteras Dfns  | ✓    | ✓          |
| Openfort        | Billeteras backend de Openfort      | ✓    | ✓          |
| Para            | Billeteras MPC de Para              | ✓    | ✓          |
| Utila           | Custodia MPC de Utila               | ✓    | ✓          |

## Inicio rápido

### Rust

Instala el crate de Rust:

```bash
cargo add solana-keychain
```

Uso básico:

```rust
use solana_keychain::{Signer, SolanaSigner};

// Create a signer from any backend
let signer = Signer::from_memory("base58_private_key")?;

// All signers share the same interface
let pubkey = signer.pubkey();
let signature = signer.sign_transaction(&mut tx).await?;
```

- [Guía de Rust](/docs/tools/keychain/getting-started/rust) - Instalación
  completa y configuración del backend
- [Crates.io](https://crates.io/crates/solana-keychain) - Crate de Rust

### TypeScript

Instala el paquete de TypeScript:

```bash
pnpm add @solana/keychain
```

Uso básico:

```typescript
import { createKeychainSigner } from "@solana/keychain";
import { signTransactionWithSigners } from "@solana/signers";

// Create any signer via the unified factory
const signer = await createKeychainSigner({
  backend: "vault",
  vaultAddr: "https://vault.example.com:8200",
  vaultToken: "hvs.xxxxx",
  keyName: "my-solana-key",
  publicKey: "base58_public_key"
});

// Sign an already-compiled transaction
const signedTx = await signTransactionWithSigners(
  [signer],
  compiledTransaction
);
```

- [Guía de TypeScript](/docs/tools/keychain/getting-started/typescript) -
  Paquetes `@solana/keychain`
- [Paquete npm](https://www.npmjs.com/package/@solana/keychain) - Paquete npm
