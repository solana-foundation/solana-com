title: Firmantes SVM
description: Firmantes para Solana y Blockchains Compatibles con SVM

Estos firmantes están disponibles al usar el complemento SVM para la firma de
transacciones.

## secret_key

El firmante `svm::secret_key` puede usarse para firmar una transacción de forma
síncrona utilizando una clave secreta, mnemónico o archivo keypair.

### Entradas

| Name              | Required | Type   | Description                                                      |
| ----------------- | -------- | ------ | ---------------------------------------------------------------- |
| `secret_key`      | optional | string | La clave secreta utilizada para firmar mensajes y transacciones  |
| `mnemonic`        | optional | string | La frase mnemónica utilizada para generar la clave secreta       |
| `derivation_path` | optional | string | La ruta de derivación utilizada para generar la clave secreta    |
| `keypair_json`    | optional | string | Una ruta a un archivo keypair.json que contiene la clave secreta |

### Salidas

| Name         | Type   | Description                            |
| ------------ | ------ | -------------------------------------- |
| `public_key` | string | La clave pública de la cuenta          |
| `address`    | string | La dirección SVM (alias de public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

También puedes usar un mnemónico:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

O un archivo keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

El firmante `svm::web_wallet` permitirá a un operador de Runbook firmar la
transacción con el firmante de navegador de su elección.

### Entradas

| Name               | Required | Type   | Description                                                               |
| ------------------ | -------- | ------ | ------------------------------------------------------------------------- |
| `expected_address` | optional | string | La dirección SVM que se espera conectar (omitir para permitir cualquiera) |

### Salidas

| Name         | Type   | Description                             |
| ------------ | ------ | --------------------------------------- |
| `address`    | string | La dirección de la cuenta conectada     |
| `public_key` | string | La clave pública de la cuenta conectada |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Este firmante solicitará al usuario en la interfaz web que:

1. Conecte su billetera
2. Proporcione una pubkey mediante firma de mensaje
3. Firme todas las transacciones que requieran este firmante

## squads

El firmante `svm::squads` puede utilizarse para firmar transacciones con una
bóveda multisig de Squads.

### Entradas

| Nombre    | Requerido | Tipo   | Descripción                         |
| --------- | --------- | ------ | ----------------------------------- |
| `address` | opcional  | string | La dirección de la bóveda del squad |

### Salidas

| Nombre       | Tipo   | Descripción                            |
| ------------ | ------ | -------------------------------------- |
| `public_key` | string | La pubkey de la bóveda del squad       |
| `address`    | string | La dirección SVM (alias de public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

El uso del multisig de Squads permite la firma de transacciones segura y entre
múltiples partes sin exponer las claves privadas individuales.
