<Callout type="info" title="Samenvatting">
  Runtime controles na elke instructie: alleen de eigenaar kan lamports
  debiteren of data wijzigen, data kan maximaal 10 KiB per instructie groeien,
  eigenaar wijzigingen vereisen zero-initialized data, en de executable flag is
  onomkeerbaar.
</Callout>

De Solana runtime dwingt deze regels af na elke instructie via
[`BorrowedInstructionAccount`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L722-L727)
methoden. Elke regel wordt gecontroleerd op het moment van wijziging, en de
transactie wordt teruggedraaid als een controle faalt.

## Lamports regels

| Regel                                                              | Afdwinging                                                                                                                                                                                                                                          | Fout                          |
| ------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Alleen de eigenaar kan lamports debiteren                          | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L785): controleert `is_owned_by_current_program()` wanneer `lamports < current`                                                                | `ExternalAccountLamportSpend` |
| Read-only accounts kunnen geen lamports wijzigingen hebben         | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L788-L789): controleert `is_writable()`                                                                                                             | `ReadonlyLamportChange`       |
| Elk programma kan lamports crediteren naar een schrijfbaar account | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L789): de eigendomscontrole geldt alleen wanneer het nieuwe saldo lager is dan het huidige saldo; de schrijfbaarheidscontrole geldt nog steeds | `ReadonlyLamportChange`       |
| Lamports moeten in balans zijn over een instructie                 | [`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L366-L398): verifieert `get_lamports_delta() == 0`                                                                                       | `UnbalancedInstruction`       |

## Rent state-overgangen

Accounts bestaan in een van drie
[`RentState`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L19-L30)
waarden: _rs`Uninitialized`_ (0 lamports), _rs`RentPaying`_ (boven 0 maar onder
het rent-exempt minimum), en _rs`RentExempt`_ (op of boven het minimum).
Niet-toegestane overgangen produceren
_rs`TransactionError::InsufficientFundsForRent`_.

De runtime handhaaft deze regels via
[`transition_allowed()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L107-L126):

- Elk account kan overgaan naar _rs`Uninitialized`_ (sluiten) of
  _rs`RentExempt`_.
- Geen enkel account kan _rs`RentPaying`_ binnengaan vanuit _rs`Uninitialized`_
  of _rs`RentExempt`_. Alle nieuwe accounts moeten rent-exempt zijn.

## Account borrow-regels

Tijdens instructie-uitvoering handhaaft de runtime single-writer borrow
semantics op accounts. Een programma kan ofwel één muteerbare referentie of
meerdere onveranderlijke referenties naar een account verkrijgen, maar niet
beide tegelijkertijd. Als een programma probeert een account te lenen dat al
muteerbaar geleend is (of muteerbaar een account probeert te lenen dat al
onveranderlijk geleend is), mislukt de instructie met _rs`AccountBorrowFailed`_
via
[`try_borrow()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L498-L508)
en
[`try_borrow_mut()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L512-L520).
Als een instructie voltooid wordt terwijl een borrow nog uitstaat, retourneert
de runtime _rs`AccountBorrowOutstanding`_ in
[`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L374-L385).
