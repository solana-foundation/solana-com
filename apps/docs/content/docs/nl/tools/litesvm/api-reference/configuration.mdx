---
title: Omgevingsconfiguratie
description:
  Builder-patroonmethoden voor het configureren van LiteSVM-testomgevingen
---

Alle configuratiemethoden gebruiken het builder-patroon met
`LiteSVM::default()`.

## with_compute_budget

```rust
pub fn with_compute_budget(self, compute_budget: ComputeBudget) -> Self
```

Stel limieten voor rekeneenheden en heapgrootte in.

```rust
use solana_compute_budget::compute_budget::ComputeBudget;

let mut svm = LiteSVM::default()
    .with_compute_budget(ComputeBudget {
        compute_unit_limit: 1_400_000,
        heap_size: 256 * 1024,
        ..ComputeBudget::default()
    });
```

## with_sigverify

```rust
pub fn with_sigverify(self, sigverify: bool) -> Self
```

Schakel handtekeningverificatie in of uit. Standaard: `true`.

```rust
// Disable for faster tests
let mut svm = LiteSVM::default()
    .with_sigverify(false);
```

## with_blockhash_check

```rust
pub fn with_blockhash_check(self, check: bool) -> Self
```

Schakel blokhashvalidatie in of uit. Standaard: `true`.

```rust
// Disable to use any blockhash
let mut svm = LiteSVM::default()
    .with_blockhash_check(false);
```

## with_transaction_history

```rust
pub fn with_transaction_history(self, capacity: usize) -> Self
```

Stel het aantal transacties in dat bewaard blijft in de geschiedenis. Standaard:
`0`.

```rust
// Keep last 100 transactions
let mut svm = LiteSVM::default()
    .with_transaction_history(100);
```

## with_log_bytes_limit

```rust
pub fn with_log_bytes_limit(self, limit: Option<usize>) -> Self
```

Stel het maximale aantal bytes in voor transactielogboeken. Standaard:
`Some(10_000)`.

```rust
// Unlimited logs
let mut svm = LiteSVM::default()
    .with_log_bytes_limit(None);

// Custom limit (50KB)
let mut svm = LiteSVM::default()
    .with_log_bytes_limit(Some(50_000));
```

## with_lamports

```rust
pub fn with_lamports(self, lamports: u64) -> Self
```

Stel initiële lamports in voor het innen van vergoedingen. Standaard:
`1_000_000_000`.

```rust
let mut svm = LiteSVM::default()
    .with_lamports(10_000_000_000);
```

## with_default_programs

```rust
pub fn with_default_programs(self) -> Self
```

Inclusief standaard SPL-programma's (Token, Associated Token, etc.).

```rust
let mut svm = LiteSVM::default()
    .with_default_programs();
```

## with_sysvars

```rust
pub fn with_sysvars(self) -> Self
```

Inclusief standaard sysvars (Clock, Rent, EpochSchedule, SlotHashes,
StakeHistory, etc.).

```rust
let mut svm = LiteSVM::default()
    .with_sysvars();
```

## with_feature_set

```rust
pub fn with_feature_set(self, feature_set: FeatureSet) -> Self
```

Configureer de Solana-functieset voor het testen van specifieke
runtime-functies.

```rust
use agave_feature_set::FeatureSet;

let mut svm = LiteSVM::default()
    .with_feature_set(FeatureSet::all_enabled());
```

## with_precompiles

```rust
#[cfg(feature = "precompiles")]
pub fn with_precompiles(self) -> Self
```

Inclusief standaard geprecompileerde programma's (ed25519, secp256k1,
secp256r1).

```rust
let mut svm = LiteSVM::default()
    .with_precompiles();
```

<Callout type="info">
  Vereist de `precompiles` feature flag:

```toml
[dev-dependencies]
litesvm = { version = "0.11", features = ["precompiles"] }
```

</Callout>

## with_feature_accounts

```rust
pub fn with_feature_accounts(self) -> Self
```

Voeg feature gate-accounts toe aan de testomgeving. Dit zorgt ervoor dat het
door feature gates bepaalde gedrag van de Solana-runtime nauwkeurig wordt
weerspiegeld in uw tests.

```rust
let mut svm = LiteSVM::default()
    .with_feature_accounts();
```

## register-tracing functie

De `register-tracing` functie maakt gedetailleerde SBF-registertracering
mogelijk tijdens de uitvoering van transacties. Het impliceert de
`invocation-inspect-callback` functie.

```toml
[dev-dependencies]
litesvm = { version = "0.11", features = ["register-tracing"] }
```

Gebruik `LiteSVM::new_debuggable(true)` om een instantie te maken met
registertracering ingeschakeld:

```rust
#[cfg(feature = "register-tracing")]
let mut svm = LiteSVM::new_debuggable(true);
```

Wanneer ingeschakeld, stel de omgevingsvariabele `SBF_TRACE_DISASSEMBLE` in om
ook disassembly-uitvoer naast registertraces te dumpen:

```bash
SBF_TRACE_DISASSEMBLE=1 cargo test
```

## set_invocation_inspect_callback

```rust
pub fn set_invocation_inspect_callback<C: InvocationInspectCallback + 'static>(&mut self, callback: C)
```

Registreer een callback die wordt uitgevoerd vóór en na elke programma-aanroep.
Vereist de `invocation-inspect-callback` functie.

```toml
[dev-dependencies]
litesvm = { version = "0.11", features = ["invocation-inspect-callback"] }
```

```rust
use litesvm::{LiteSVM, types::InvocationInspectCallback};
use solana_svm::invoke_context::InvokeContext;
use solana_svm_transaction::svm_message::SVMMessage;

struct MyInspector;

impl InvocationInspectCallback for MyInspector {
    fn before_invocation(
        &self,
        _svm: &LiteSVM,
        _tx: &impl SVMMessage,
        _program_indices: &[u16],
        _invoke_context: &InvokeContext,
    ) {
        println!("Before program invocation");
    }

    fn after_invocation(
        &self,
        _svm: &LiteSVM,
        _invoke_context: &InvokeContext,
        _enable_register_tracing: bool,
    ) {
        println!("After program invocation");
    }
}

let mut svm = LiteSVM::new();
svm.set_invocation_inspect_callback(MyInspector);
```
