# Transacties

Methoden voor het verzenden, simuleren en beheren van transacties.

## sendTransaction

```typescript
sendTransaction(tx: Transaction): TransactionMetadata | FailedTransactionMetadata
```

Een transactie uitvoeren. Geeft metadata terug bij succes of foutdetails bij een
fout.

```typescript
const result = client.svm.sendTransaction(signedTx);

// Check for errors
if ("err" in result && result.err()) {
  console.error("Transaction failed:", result.err());
  return;
}

// Access result properties (note: these are getter functions)
console.log("Signature:", result.signature());
console.log("Compute units:", result.computeUnitsConsumed());
console.log("Logs:", result.logs());
```

<Callout type="warn">
  Eigenschappen van transactieresultaten zoals `err()`,
  `computeUnitsConsumed()`, `logs()` en `signature()` zijn **getter-functies**,
  geen eigenschappen. Roep ze altijd aan met haakjes.
</Callout>

### TransactionMetadata-methoden

| Methode                  | Geeft terug             | Beschrijving                     |
| ------------------------ | ----------------------- | -------------------------------- |
| `signature()`            | `Uint8Array`            | Handtekeningbytes van transactie |
| `computeUnitsConsumed()` | `bigint`                | Gebruikte compute-eenheden       |
| `logs()`                 | `string[]`              | Programmalogboeken               |
| `innerInstructions()`    | `InnerInstruction[]`    | Interne (CPI) instructies        |
| `returnData()`           | `TransactionReturnData` | Retourdata van programma's       |
| `prettyLogs()`           | `string`                | Opgemaakte logboeken met kleuren |

## simulateTransaction

```typescript
simulateTransaction(tx: Transaction): SimulatedTransactionInfo | FailedTransactionMetadata
```

Een transactie simuleren zonder de toestand te wijzigen.

```typescript
const simResult = client.svm.simulateTransaction(signedTx);

if ("err" in simResult && simResult.err()) {
  console.error("Simulation failed:", simResult.err());
  return;
}

// Use .meta() to get TransactionMetadata
const meta = simResult.meta();
console.log("Would use", meta.computeUnitsConsumed(), "compute units");
console.log("Logs:", meta.logs());

// Get post-execution account states
const postAccounts = simResult.postAccounts();
console.log("Post-state accounts:", postAccounts);
```

<Callout type="info">
  Simulatie wijzigt de toestand niet. Gebruik het om transacties te controleren
  voordat u ze verstuurt.
</Callout>

## Blockhash-beheer

### latestBlockhash

```typescript
latestBlockhash(): Blockhash
```

De huidige blockhash ophalen.

```typescript
const blockhash = client.svm.latestBlockhash();
console.log("Blockhash:", blockhash);
```

### getLatestBlockhash (via client.rpc)

De blockhash ophalen met levensduurinformatie voor het handmatig opbouwen van
transacties. Gebruik de RPC-helper omdat die zowel `blockhash` als
`lastValidBlockHeight` teruggeeft.

```typescript
import {
  createTransactionMessage,
  setTransactionMessageLifetimeUsingBlockhash
} from "@solana/kit";

const { value: blockhashLifetime } = await client.rpc
  .getLatestBlockhash()
  .send();

const tx = setTransactionMessageLifetimeUsingBlockhash(
  blockhashLifetime,
  createTransactionMessage({ version: 0 })
);
```

<Callout type="info">
  Wanneer u programma-plugins zoals `systemProgram()` of `tokenProgram()`
  gebruikt, hoeft u de blockhash nooit zelf op te halen —
  `client.sendTransaction()` (en de `.sendTransaction()`-helper op
  instructieketens) doet dit voor u.
</Callout>

### expireBlockhash

```typescript
expireBlockhash(): LiteSVM
```

Ga naar een nieuwe blockhash. Handig voor het testen van scenario's waarbij
blockhashes verlopen.

```typescript
const oldBlockhash = client.svm.latestBlockhash();
client.svm.expireBlockhash();
const newBlockhash = client.svm.latestBlockhash();

console.log("Old:", oldBlockhash);
console.log("New:", newBlockhash);
// Blockhashes will be different
```

## Transactiegeschiedenis

Schakel transactiegeschiedenis in om transacties op te halen na het verzenden:

```typescript
// Enable history storage (stores last N transactions)
client.svm.withTransactionHistory(100n);

// Send a transaction
const result = client.svm.sendTransaction(signedTx);

// Later, retrieve it by signature (base58 string)
// Note: getTransaction expects a base58-encoded signature string
```
