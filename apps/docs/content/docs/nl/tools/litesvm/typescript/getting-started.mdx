# LiteSVM voor TypeScript

Solana-testsuites die `solana-test-validator` opstarten kosten seconden per test
— je wacht op het opstarten van het proces, RPC-verbindingen en
netwerk-roundtrips. LiteSVM voert dezelfde tests uit in milliseconden. Geen
extern proces, geen netwerk, alles in-memory binnen je testrunner.

<Callout type="info">
  **Getest met:** `@solana/kit@6.8.0`, `@solana/kit-plugin-litesvm@0.10.0`,
  `@solana/kit-plugin-signer@0.10.0`, `@solana-program/system@0.12.0`,
  `@solana-program/token@0.13.0`. Deze pluginpakketten zijn nieuw en ontwikkelen
  zich snel — als je een versieconflict tegenkomt, raadpleeg dan
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  voor de exacte versies waartegen de documentatie en testsuite draaien.
</Callout>

Je stelt een Kit-client samen vanuit `@solana/kit` door plugins te installeren
met `.use()`: een signer-plugin
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
de LiteSVM-plugin
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm)),
en eventuele programma-plugins die je nodig hebt. Het resultaat geeft toegang
tot `client.sendTransaction()`, `client.rpc`, en directe toegang tot
accountstatus, klok en sysvars — allemaal zonder je proces te verlaten.

### Je client aanmaken

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` geeft een lege basisstructuur terug — elke `.use(plugin)`
  voegt mogelijkheden toe. De `signer()`-plugin moet **voor** `litesvm()` komen
  omdat de transportplugin een betaler nodig heeft om aan te koppelen.
  `signer(mySigner)` is een afkorting voor het installeren van hetzelfde keypair
  als zowel `payer` als `identity` — gebruik `payer()` of `identity()` uit
  hetzelfde pakket als je die rollen wilt splitsen. Deze volgorde is hetzelfde
  voor `solanaRpc()` wanneer je op een echt netwerk werkt.
</Callout>

<Callout type="info">
  Wil je liever een eenregelige setup voor snelle experimenten? De
  `litesvm()`-bundel bevat een airdrop, zodat je
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` kunt koppelen vanuit
  `@solana/kit-plugin-signer` om automatisch een betaler aan te maken en te
  financieren.
</Callout>

</Step>

</Steps>

## De client begrijpen

Na `.use(signer(...)).use(litesvm())` stelt de client het volgende beschikbaar:

| Eigenschap                  | Beschrijving                                                                                                                                                                                              |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | De `TransactionSigner` die je hebt geïnstalleerd met de `signer()` plugin (ook ingesteld als `client.identity`)                                                                                           |
| `client.identity`           | De wallet-ondertekenaar die accounts, tokens en bevoegdheden beheert                                                                                                                                      |
| `client.svm`                | Directe [`LiteSVM`](https://www.npmjs.com/package/litesvm) instantie voor accountbeheer, transacties en configuratie                                                                                      |
| `client.rpc`                | Kit-compatibele RPC-interface (subset: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Instructies plannen, ondertekenen en verzenden als een transactie                                                                                                                                         |
| `client.airdrop`            | SOL opvragen bij de LiteSVM-faucet                                                                                                                                                                        |
| `client.getMinimumBalance`  | Minimale lamports berekenen voor rent-vrijstelling                                                                                                                                                        |

### client.svm gebruiken

Directe toegang tot LiteSVM voor:

- Accountbeheer (`setAccount`, `getAccount`, `airdrop`)
- Transactie-uitvoering (`sendTransaction`, `simulateTransaction`)
- Configuratie (`withSigverify`, `withBlockhashCheck`, enz.)
- Tijdsmanipulatie (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### client.rpc gebruiken

Kit-compatibele RPC-interface voor standaardbewerkingen:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  De RPC-laag ondersteunt alleen **base64-codering**. Het opvragen van andere
  coderingen zal een fout veroorzaken.
</Callout>

## Configuratie

Configureer LiteSVM met behulp van bouwmethoden op `client.svm`:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## Transacties verzenden

Voeg programma-plugins toe na de transport — ze koppelen instructiebouwers aan
de client, zodat je nooit direct een `TransactionMessage` hoeft aan te raken:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## SVM → RPC-stroom

Je kunt de toestand instellen via SVM en uitlezen via RPC:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```

## Wat is het volgende

Dit behandelt de kern van de client — accounts aanmaken, instructies verzenden
en toestand uitlezen. Als je tests SPL-tokens omvatten (mints, ATA's,
overdrachten), laat de
[handleiding voor token-testen](/docs/tools/litesvm/typescript/tokens) zien hoe
de `tokenProgram()` plugin ~40 regels setup terugbrengt tot 3 aanroepen. Voor
een volledige API-referentie, zie de
[API-documentatie](/docs/tools/litesvm/typescript/api-reference).
