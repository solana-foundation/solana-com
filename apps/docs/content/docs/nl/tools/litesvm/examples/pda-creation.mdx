## Volledige Test

```rust title="tests/pda_test.rs"
use litesvm::LiteSVM;
use solana_account::Account;
use solana_sdk::pubkey::Pubkey;

#[test]
fn test_pda_creation() {
    let mut svm = LiteSVM::new();

    // Program that owns the PDA
    let program_id = Pubkey::new_unique();

    // Derive PDA address
    let seed = b"my_pda";
    let (pda, bump) = Pubkey::find_program_address(
        &[seed],
        &program_id
    );

    println!("PDA: {}", pda);
    println!("Bump: {}", bump);

    // Create PDA account manually (for testing)
    svm.set_account(pda, Account {
        lamports: 1_000_000,
        data: vec![bump], // Store bump seed in data
        owner: program_id,
        executable: false,
        rent_epoch: 0,
    }).unwrap();

    // Verify PDA was created
    let account = svm.get_account(&pda).unwrap();
    assert_eq!(account.owner, program_id);
    assert_eq!(account.data[0], bump);

    println!("PDA created successfully!");
}
```

## Belangrijke Punten

1. **PDA-afleiding**: Gebruik `Pubkey::find_program_address()` om PDA-adressen
   af te leiden
2. **Seeds**: PDA's worden afgeleid van seeds en de programma-ID
3. **Bump Seed**: De bump seed zorgt ervoor dat het adres buiten de
   ed25519-curve valt
4. **Accountaanmaak**: Gebruik `set_account()` om accounts handmatig aan te
   maken voor tests
5. **Eigenaarschap**: PDA's moeten eigendom zijn van het programma dat ze heeft
   afgeleid
