---
title: SVM Ondertekenaars
description: Ondertekenaars voor Solana en SVM-compatibele blockchains
---

Deze ondertekenaars zijn beschikbaar bij gebruik van de SVM-addon voor
transactieondertekening.

## secret_key

De `svm::secret_key` ondertekenaar kan worden gebruikt om een transactie
synchroon te ondertekenen met behulp van een geheime sleutel, mnemonic of
keypair-bestand.

### Invoer

| Name              | Required | Type   | Description                                                                       |
| ----------------- | -------- | ------ | --------------------------------------------------------------------------------- |
| `secret_key`      | optional | string | De geheime sleutel die wordt gebruikt om berichten en transacties te ondertekenen |
| `mnemonic`        | optional | string | De mnemonische zin die wordt gebruikt om de geheime sleutel te genereren          |
| `derivation_path` | optional | string | Het afleidingspad dat wordt gebruikt om de geheime sleutel te genereren           |
| `keypair_json`    | optional | string | Een pad naar een keypair.json-bestand dat de geheime sleutel bevat                |

### Uitvoer

| Name         | Type   | Description                           |
| ------------ | ------ | ------------------------------------- |
| `public_key` | string | De publieke sleutel van het account   |
| `address`    | string | Het SVM-adres (alias voor public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

U kunt ook een mnemonic gebruiken:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Of een keypair-bestand:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

De `svm::web_wallet` ondertekenaar stelt een Runbook-operator in staat de
transactie te ondertekenen met de browserondertekenaar van hun keuze.

### Invoer

| Name               | Required | Type   | Description                                                                             |
| ------------------ | -------- | ------ | --------------------------------------------------------------------------------------- |
| `expected_address` | optional | string | Het verwachte SVM-adres om verbinding mee te maken (weglaten om elk adres toe te staan) |

### Uitvoer

| Name         | Type   | Description                                   |
| ------------ | ------ | --------------------------------------------- |
| `address`    | string | Het adres van het verbonden account           |
| `public_key` | string | De publieke sleutel van het verbonden account |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Deze ondertekenaar vraagt de gebruiker in de Web UI om:

1. Verbind hun wallet
2. Geef een pubkey op via berichthandtekening
3. Onderteken alle transacties die deze ondertekenaar vereisen

## squads

De `svm::squads` ondertekenaar kan worden gebruikt om transacties te
ondertekenen met een Squads multisig-vault.

### Invoer

| Naam      | Vereist   | Type   | Beschrijving                 |
| --------- | --------- | ------ | ---------------------------- |
| `address` | optioneel | string | Het adres van de squad-vault |

### Uitvoer

| Naam         | Type   | Beschrijving                          |
| ------------ | ------ | ------------------------------------- |
| `public_key` | string | De pubkey van de squad-vault          |
| `address`    | string | Het SVM-adres (alias voor public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Het gebruik van Squads multisig maakt veilige transactieondertekening door
meerdere partijen mogelijk zonder dat individuele privésleutels worden
blootgesteld.
