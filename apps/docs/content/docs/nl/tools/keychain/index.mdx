**Solana Keychain biedt een uniforme interface voor het ondertekenen van
Solana-transacties via meerdere sleutelbeheer-backends.** Ontwikkel lokaal met
een in-memory sleutel en schakel vervolgens over naar enterprise sleutelbeheer,
cloud-HSM's of beheerde walletservices in productie — zonder uw
ondertekeningscode te herschrijven.

### Waarom Keychain?

- **Enkelvoudige interface**: Één `SolanaSigner` trait werkt met alle backends
- **Backends wisselen**: Wijzig sleutelbeheer-providers via configuratie, niet
  via code
- **Kostenloze abstractie**: Feature flags nemen alleen op wat u nodig heeft
- **Geauditeerd**: Beveiligingscontrole door [Accretion](https://accretion.xyz)

### Architectuur

- **Talen**: Rust + TypeScript, volledig equivalent
- **Trait**: Uniforme `SolanaSigner` interface
- **Backends**: Memory, Vault, AWS KMS, GCP KMS, Privy, Turnkey, Fireblocks,
  CDP, Crossmint, Dfns, Openfort, Para, Utila
- **Compatibiliteit**: `@solana/kit` en `@solana/signers` compatibel
  (TypeScript) | `solana-sdk` en `solana-sdk-v3` compatibel (Rust)

### Ondersteunde backends

Backends omvatten lokale ontwikkeling, zelfgehoste en cloud-sleutelbeheer, en
beheerde of MPC-walletservices. Zie
[Een backend kiezen](/docs/tools/keychain/choosing-a-backend) om bewaarmodellen
te vergelijken en de juiste te selecteren.

| Backend         | Gebruiksscenario              | Rust | TypeScript |
| --------------- | ----------------------------- | ---- | ---------- |
| Memory          | Ontwikkeling, testen          | ✓    | ✓          |
| HashiCorp Vault | Zelfgehoste sleutelbeheer     | ✓    | ✓          |
| AWS KMS         | Cloud-native (AWS)            | ✓    | ✓          |
| GCP KMS         | Cloud-native (GCP)            | ✓    | ✓          |
| Privy           | Embedded wallets              | ✓    | ✓          |
| Turnkey         | Niet-custodiaal sleutelbeheer | ✓    | ✓          |
| Fireblocks      | Institutionele bewaring       | ✓    | ✓          |
| CDP             | Coinbase Developer Platform   | ✓    | ✓          |
| Crossmint       | Crossmint beheerde wallets    | ✓    | ✓          |
| Dfns            | Dfns wallet-infrastructuur    | ✓    | ✓          |
| Openfort        | Openfort backend wallets      | ✓    | ✓          |
| Para            | Para MPC-wallets              | ✓    | ✓          |
| Utila           | Utila MPC-bewaring            | ✓    | ✓          |

## Snelle start

### Rust

Installeer de Rust crate:

```bash
cargo add solana-keychain
```

Basisgebruik:

```rust
use solana_keychain::{Signer, SolanaSigner};

// Create a signer from any backend
let signer = Signer::from_memory("base58_private_key")?;

// All signers share the same interface
let pubkey = signer.pubkey();
let signature = signer.sign_transaction(&mut tx).await?;
```

- [Rust-gids](/docs/tools/keychain/getting-started/rust) - Volledige installatie
  en backendconfiguratie
- [Crates.io](https://crates.io/crates/solana-keychain) - Rust crate

### TypeScript

Installeer het TypeScript-pakket:

```bash
pnpm add @solana/keychain
```

Basisgebruik:

```typescript
import { createKeychainSigner } from "@solana/keychain";
import { signTransactionWithSigners } from "@solana/signers";

// Create any signer via the unified factory
const signer = await createKeychainSigner({
  backend: "vault",
  vaultAddr: "https://vault.example.com:8200",
  vaultToken: "hvs.xxxxx",
  keyName: "my-solana-key",
  publicKey: "base58_public_key"
});

// Sign an already-compiled transaction
const signedTx = await signTransactionWithSigners(
  [signer],
  compiledTransaction
);
```

- [TypeScript-gids](/docs/tools/keychain/getting-started/typescript) -
  `@solana/keychain` pakketten
- [npm-pakket](https://www.npmjs.com/package/@solana/keychain) - npm-pakket
