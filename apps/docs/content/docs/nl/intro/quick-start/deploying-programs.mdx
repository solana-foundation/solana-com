h1: Je eerste Solana-programma implementeren
---

Bouw, implementeer en test je eerste [programma](/docs/core/programs) op de
Solana Blockchain met behulp van het Anchor-framework. Deze gids helpt je
vertrouwd te raken met de Solana Playground en bereidt je voor op het gebruik
van Program Derived Addresses (PDA) en Cross Program Invocation (CPI).

<Steps>

<Step>

### Anchor-project aanmaken

1. Open de [Solana Playground](https://beta.solpg.io) in je webbrowser
2. Klik in het linkerpaneel op **+ Create a new project**
3. Voer een **Project name** in
4. Selecteer **Anchor (Rust)** als framework
5. Klik op **Create**

![Nieuw project](/assets/docs/intro/quickstart/pg-new-project.gif)

Het `src/lib.rs` bronbestand wordt geopend. Het bevat één struct genaamd
`Initialize`, die het volgende doet:

- Ontvangt een verplichte parameter genaamd _rs`data: u64`_
- Maakt een nieuw account aan
- Slaat de waarde van _rs`data: u64`_ op in het nieuwe account
- Voegt een bericht toe aan de programmalogs van de transactie

<WithNotes>

```rust title="lib.rs"
use anchor_lang::prelude::*;

// This is your program's public key and it will update
// automatically when you build the project.
// !tooltip[/declare_id/] declare_id
declare_id!("11111111111111111111111111111111");

// !tooltip[/program/] program
#[program]
mod hello_anchor {
    use super::*;
    // !tooltip[/data: u64/] parameter
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !tooltip[/ctx.accounts.new_account.data = data/] initialize
        ctx.accounts.new_account.data = data;
        // !tooltip[/msg!/] msg
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}

// !tooltip[/Accounts/] accounts
#[derive(Accounts)]
pub struct Initialize<'info> {
    // We must specify the space in order to initialize an account.
    // First 8 bytes are default account discriminator,
    // next 8 bytes come from NewAccount.data being type u64.
    // (u64 = 64 bits unsigned integer = 8 bytes)
    // !tooltip[/account/] constraints
    #[account(
      // !tooltip[/init/] init
      init,
      // !tooltip[/payer/] payer
      payer = signer,
      // !tooltip[/space/] space
      space = 8 + 8
    )]
    pub new_account: Account<'info, NewAccount>,
    // !tooltip[/account/] constraints
    #[account(mut)]
    pub signer: Signer<'info>,
    pub system_program: Program<'info, System>,
}

// !tooltip[/account/] account
#[account]
pub struct NewAccount {
    data: u64
}
```

### !declare_id

Het on-chain adres van je programma. Dit wordt bijgewerkt wanneer je het project
bouwt

### !program

De module die de instructies van het programma bevat

### !accounts

Annoteert een struct die de accounts van het programma bevat

### !account

Een struct die de accountgegevenstypen vertegenwoordigt die dit programma kan
aanmaken en bezitten

### !constraints

Accountbeperkingen. Bijvoorbeeld, algemene controles of functies die de
programmabeveiliging verbeteren

### !parameter

De waarde die aan de instructie wordt doorgegeven wanneer deze wordt aangeroepen

### !initialize

Benadert het `data` veld van het `new_account` account en stelt het in op de
waarde van `data: u64`

### !msg

De _rs`msg!`_ macro logt een bericht naar de programmalogs van de transactie

### !init

Roept het System Program aan om een nieuw account te maken en maakt dit
programma de eigenaar. Als programma-eigenaar kan dit programma dan naar de
accountgegevens schrijven

### !payer

Het account dat betaalt voor het nieuwe account

### !space

Het aantal bytes dat voor het account moet worden gereserveerd

</WithNotes>

<Accordion title="Uitleg">

<WithMentions>

- De [`declare_id!`](mention:one) macro specificeert het on-chain adres van je
  programma. Solana Playground zal dit adres automatisch bijwerken wanneer je
  het programma bouwt.

```rs
// !mention one
declare_id!("11111111111111111111111111111111");
```

</WithMentions>

<WithMentions>

- Het [`#[program]`](mention:one) attribuut annoteert een module die functies
  bevat welke de instructies van het programma vertegenwoordigen.

```rs
// !mention one
#[program]
mod hello_anchor {
    use super::*;
    // !mention two
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !mention three
        ctx.accounts.new_account.data = data;
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}
```

In dit voorbeeld neemt de [`initialize`](mention:two) instructie twee
parameters:

- `ctx: Context<Initialize>`: Geeft de accounts door die vereist zijn voor deze
  instructie aan de functie, zoals gespecificeerd in de `Initialize` struct.
- `data: u64`: Een aangepaste parameter die je moet opgeven wanneer je de
  instructie aanroept.

De functiebody [stelt het `data` veld van `new_account`](mention:three) in op
het opgegeven `data` argument, en drukt vervolgens een bericht af naar de
programmalogs.

</WithMentions>

<WithMentions>

- Het [`#[derive(Accounts)]`](mention:one) attribuut definieert een struct die
  de accounts specificeert die vereist zijn voor een bepaalde instructie. Elk
  veld vertegenwoordigt een afzonderlijk account.

  Anchor handelt automatisch veelvoorkomende beveiligingscontroles met
  betrekking tot accountvalidatie af via veldtypes (bijv. `Signer<'info>`) en
  beperkingen (bijv. `#[account(mut)]`).

  ```rs
  // !mention one
  #[derive(Accounts)]
  pub struct Initialize<'info> {
      #[account(init, payer = signer, space = 8 + 8)]
      pub new_account: Account<'info, NewAccount>,
      #[account(mut)]
      pub signer: Signer<'info>,
      pub system_program: Program<'info, System>,
  }
  ```

</WithMentions>

<WithMentions>

- Het [`#[account]`](mention:one) attribuut definieert een struct die het
  datatype vertegenwoordigt voor accounts die dit programma aanmaakt en bezit.

```rs
// !mention one
#[account]
pub struct NewAccount {
  data: u64
}
```

</WithMentions>

</Accordion>

<Step>

### Het programma bouwen en implementeren

1. Compileer de programmacode

```terminal
$ build
Building...
Build successful. Completed in 1.46s.
```

<Callout>
  Je [programma-ID](/docs/intro/quick-start/deploying-programs), gedefinieerd in
  _rs`declare_id!()`_, wordt nu bijgewerkt.
</Callout>

2. Voeg SOL toe aan je wallet _(Optioneel)_

Het implementeren van een programma vereist SOL in het on-chain account van het
programma. Indien nodig,
[voeg SOL toe aan je wallet](/docs/intro/quick-start#add-sol-to-your-wallet)
voordat je verdergaat naar stap 3.

3. Implementeer het programma op het netwerk

```terminal
$ deploy
Deploying... This could take a while depending on the program size and network conditions.
Warning: 1 transaction not confirmed, retrying...
Deployment successful. Completed in 19s.
```

<Callout type="info">
  Wanneer het programma wordt geïmplementeerd, wordt de bytecode van het
  programma opgeslagen in een uitvoerbaar program account.
</Callout>

**Opmerking:** Indien gewenst, kun je ook de knoppen **Build** en **Deploy**
gebruiken in het paneel aan de linkerkant, zoals weergegeven in de onderstaande
schermafbeelding.

![Build en Deploy](/assets/docs/intro/quickstart/pg-build-deploy.png)

</Step>

### Het programma testen

<WithMentions>

1. Open het testbestand genaamd `tests/anchor.test.ts`. Dit bestand demonstreert
   hoe je de [`initialize`](mention:one) struct van het programma aanroept
   vanuit de client.

```ts title="anchor.test.ts"
// No imports needed: web3, anchor, pg and more are globally available

describe("Test", () => {
  it("initialize", async () => {
    // Generate keypair for the new account
    const newAccountKp = new web3.Keypair();

    // Send transaction
    const data = new BN(42);
    const txHash = await pg.program.methods
      // !mention one
      .initialize(data)
      .accounts({
        newAccount: newAccountKp.publicKey,
        signer: pg.wallet.publicKey,
        systemProgram: web3.SystemProgram.programId
      })
      .signers([newAccountKp])
      .rpc();
    console.log(`Use 'solana confirm -v ${txHash}' to see the logs`);

    // Confirm transaction
    await pg.connection.confirmTransaction(txHash);

    // Fetch the created account
    const newAccount = await pg.program.account.newAccount.fetch(
      newAccountKp.publicKey
    );

    console.log("On-chain data is:", newAccount.data.toString());

    // Check whether the data on-chain is equal to local 'data'
    assert(data.eq(newAccount.data));
  });
});
```

</WithMentions>

<Step>

### Sluit het programma

Sluit het programma om de SOL die aan het on-chain programma is toegewezen terug
te vorderen. Voer hiervoor `solana program close [ProgramID]` uit, met het
programma-adres dat is opgeslagen in de _rs`declare_id!()`_ macro.

```terminal
$ solana program close 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r
Closed Program Id 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r, 2.79511512 SOL reclaimed
```

<Callout type="warn">
  Programma's kunnen **alleen** worden gesloten of bijgewerkt door een account
  met upgrade authority. Je Playground-wallet wordt automatisch de upgrade
  authority voor elk programma wanneer het wordt geïmplementeerd. Als de upgrade
  authority wordt ingetrokken, wordt het programma onveranderlijk.
</Callout>

Gefeliciteerd! Je hebt je eerste Solana-programma gebouwd en geïmplementeerd met
Anchor.
