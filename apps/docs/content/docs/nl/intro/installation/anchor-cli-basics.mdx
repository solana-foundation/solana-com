Dit gedeelte biedt enkele veelgebruikte commando's en voorbeelden om je op weg
te helpen met het gebruik van de [Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Het project initialiseren

Maak een nieuw Anchor-project door het onderstaande commando uit te voeren. Het
zal een nieuwe map maken met de projectnaam en deze gebruiken om een nieuw
Anchor- project te initialiseren.

```terminal
$ anchor init <project-name>
```

Het onderstaande commando creëert bijvoorbeeld een project genaamd `my-project`.
De `my-project` directory bevat een basis Rust-programma en
TypeScript-testsjabloon.

```terminal
$ anchor init my-project
```

Navigeer vervolgens naar de projectmap:

```terminal
$ cd <project-name>
```

Bekijk Anchor's
[projectbestandsstructuur](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure).

</Step>
<Step>

### Het programma bouwen

Om je project te bouwen, voer het volgende commando uit:

```terminal
$ anchor build
```

Je vindt het gecompileerde programma in de `/target/deploy` directory.

<Callout type="warn">

Als je bij het uitvoeren van `anchor build` de volgende foutmeldingen tegenkomt:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Probeer deze oplossingen:

1. Forceer de installatie met het volgende commando:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Als de voorgaande oplossing niet werkt, wis de Solana-cache:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Je kunt dit oplossen door het versieveld van het `Cargo.lock` bestand te wijzigen van 4 naar 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Zie [deze issue](https://github.com/solana-foundation/anchor/issues/3392) voor
meer informatie.

</Accordion>

</Accordions>

Probeer na het toepassen van de voorgaande oplossing opnieuw
_shell`anchor build`_ uit te voeren.

</Callout>

</Step>
<Step>

### Het programma implementeren

Om je project te implementeren, voer je het volgende commando uit:

```terminal
$ anchor deploy
```

Dit commando implementeert je programma naar de `cluster` die is opgegeven in
het [`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml)
bestand.

</Step>
<Step>

### Het programma testen

Om je project te testen, voer je het volgende commando uit:

```terminal
$ anchor test
```

Dit commando bouwt, implementeert en voert de tests uit voor je project.

<Callout>
  Bij gebruik van de `localnet` cluster start Anchor automatisch een lokale
  validator, implementeert het programma, voert tests uit en stopt vervolgens de
  validator.
</Callout>

<Callout type="warn">

Een van de volgende fouten kan aangeven dat je Node.js of Yarn niet hebt
geïnstalleerd:

```
Permission denied (os error 13)
```

</Step>
</Steps>
