Deze sectie biedt enkele veelvoorkomende commando's en voorbeelden om je te
helpen bij het gebruik van de [Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Initialiseer het project

Maak een nieuw Anchor-project aan door het onderstaande commando uit te voeren.
Het zal een nieuwe map aanmaken met de projectnaam en deze gebruiken om een
nieuw Anchor-project te initialiseren.

```terminal
$ anchor init <project-name>
```

Bijvoorbeeld, het onderstaande commando zal een project genaamd `my-project`
aanmaken. De `my-project` map zal een basis Rust-programma en
TypeScript-testsjabloon bevatten.

```terminal
$ anchor init my-project
```

Navigeer vervolgens naar de projectmap:

```terminal
$ cd <project-name>
```

Zie Anchor's
[projectbestandsstructuur](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure).

</Step>
<Step>

### Bouw het programma

Om je project te bouwen, voer je het volgende commando uit:

```terminal
$ anchor build
```

Je vindt het gecompileerde programma in de `/target/deploy` map.

<Callout type="warn">

Bij het uitvoeren van `anchor build`, als je de volgende fouten tegenkomt:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Probeer deze oplossingen:

1. Forceer installatie met het volgende commando:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Als de voorgaande oplossing niet werkt, wis dan de Solana-cache:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Je kunt dit oplossen door het versieveld van het `Cargo.lock`-bestand te wijzigen van 4 naar 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Zie [deze issue](https://github.com/solana-foundation/anchor/issues/3392) voor
meer informatie.

</Accordion>

</Accordions>

Probeer na het toepassen van de voorgaande oplossing opnieuw
_shell`anchor build`_ uit te voeren.

</Callout>

</Step>
<Step>

### Deploy het programma

Om je project te deployen, voer het volgende commando uit:

```terminal
$ anchor deploy
```

Dit commando deployt je programma naar het `cluster` dat is opgegeven in het
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml)-bestand.

</Step>
<Step>

### Test het programma

Om je project te testen, voer het volgende commando uit:

```terminal
$ anchor test
```

Dit commando bouwt, deployt en voert de tests voor je project uit.

<Callout>
  Bij gebruik van het `localnet`-cluster start Anchor automatisch een lokale
  validator, deployt het programma, voert tests uit en stopt vervolgens de
  validator.
</Callout>
