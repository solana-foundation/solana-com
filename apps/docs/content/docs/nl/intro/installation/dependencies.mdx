Een handleiding voor het opzetten van je lokale Solana-ontwikkelomgeving. Leer
hoe je Rust, de Solana CLI en Anchor Framework installeert op Windows (WSL),
Linux en Mac. Gebruik deze handleiding als je liever elke afhankelijkheid
afzonderlijk installeert, of als de
[snelle installatie](/docs/intro/installation/) om welke reden dan ook mislukt.

### Vereisten

<Accordions>
<Accordion title="Windows">

Om Solana-programma's op Windows te ontwikkelen **moet** je
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL) gebruiken. Na het installeren van WSL worden alle andere afhankelijkheden
geïnstalleerd via de Linux-terminal.

Om WSL te installeren, voer je het volgende commando uit in Windows PowerShell:

```terminal
$ wsl --install
```

Het installatieproces zal je vragen om een standaard gebruikersaccount aan te
maken.

![WSL Install](/assets/docs/intro/installation/wsl-install.png)

Standaard installeert WSL Ubuntu. Je kunt een Linux-terminal openen door
"Ubuntu" te zoeken in de zoekbalk.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Als je Ubuntu-terminal eruitziet zoals de afbeelding hieronder, kun je een
probleem tegenkomen waar `ctrl + v` (plakken sneltoets) niet werkt in de
terminal.

![Ubuntu Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Als je dit probleem tegenkomt, open dan Windows Terminal door te zoeken naar
"Terminal" in de zoekbalk.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Sluit vervolgens de Windows Terminal en open opnieuw een Linux-terminal door
opnieuw naar Ubuntu te zoeken. De terminal zou er nu uit moeten zien zoals de
afbeelding hieronder, waar `ctrl + v` (plakken sneltoets) werkt.

![Ubuntu Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Als je VS Code gebruikt, stelt de
[WSL-extensie](https://code.visualstudio.com/docs/remote/wsl-tutorial) je in
staat om WSL en VS Code samen te gebruiken.

![WSL Setup in VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Je ziet het volgende in de statusbalk van VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Zodra je WSL hebt ingesteld, ben je klaar om:

- De [snelle installatie](/docs/intro/installation) te voltooien met behulp van
  de Ubuntu (Linux) terminal
- [Rust](#install-rust), de [Solana CLI](#install-solana-cli) en de
  [Anchor CLI](#install-anchor-cli) afzonderlijk te installeren, zoals hieronder
  beschreven

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Als je Debian of Debian-afgeleide alternatieven zoals Ubuntu of Mint gebruikt, volg dan de onderstaande stappen:

1. Update de pakketlijsten om ervoor te zorgen dat alle pakketten up-to-date
   zijn:

```terminal
$ sudo apt-get update
```

2. Installeer de volgende afhankelijkheden:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Als je de volgende foutmelding tegenkomt bij het installeren van `protobuf-compiler`, zorg er dan
voor dat je eerst _shell`sudo apt-get update` uitvoert

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Voor RHEL-derivaten zoals Fedora of Red Hat Linux, volg je de volgende stappen.

Update de pakketlijsten om nieuwere versies van pakketten te krijgen door het
commando uit te voeren.

```terminal
$ sudo dnf upgrade --refresh
```

Installeer vervolgens de volgende afhankelijkheden:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Installeer Rust

Ontwikkelaars bouwen Solana-programma's met behulp van de programmeertaal
[Rust](https://www.rust-lang.org/).

1. Installeer Rust met behulp van
   [rustup](https://www.rust-lang.org/tools/install) door het volgende commando
   in je terminal in te voeren:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Na een succesvolle installatie zie je het volgende bericht:

<Accordions>
<Accordion title="Succesvol Rust installatiebericht">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Herlaad je PATH-omgevingsvariabele om de bin-directory van
   [Cargo](https://github.com/rust-lang/cargo) op te nemen:

```terminal
$ . "$HOME/.cargo/env"
```

3. Controleer of de installatie succesvol was.

```terminal
$ rustc --version
```

Je zult output zien zoals het volgende:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Installeer Solana CLI

De Solana CLI biedt alle tools die nodig zijn om Solana-programma's te bouwen en
te deployen.

1. Installeer de Solana CLI-toolsuite met behulp van het officiële
   installatiecommando:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Je kunt `stable` vervangen door de release-tag die overeenkomt met de
  softwareversie van je gewenste release (bijv. `v2.0.3`), of gebruik een van de
  drie symbolische kanaalnamen: `stable`, `beta`, of `edge`.
</Callout>

2. Voeg een PATH-omgevingsvariabele toe

Een eerste installatie van de Solana CLI kan je vragen om een `PATH`
omgevingsvariabele toe te voegen. Om dit te doen, sluit en heropen je terminal
of voer het volgende uit in je shell:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Update je PATH-omgevingsvariabele

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Als je **Linux** of **WSL** gebruikt, moet je het Solana CLI-binaire bestand aan
je PATH toevoegen zodat het commando beschikbaar is in je terminal. Volg
hiervoor de onderstaande stappen:

**a.** Controleer welke shell je gebruikt:

```terminal
$ echo $SHELL
```

- Als de output `/bash` bevat, gebruik dan `.bashrc`.
- Als de output `/zsh` bevat, gebruik dan `.zshrc`.

**b.** Voer het juiste commando uit, afhankelijk van je shell.

Voor Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Voor Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Als je een Mac gebruikt met `zsh`, blijft het standaard `export PATH` commando
dat vanuit de installatie wordt gelogd niet behouden zodra je je terminal sluit.
Voeg in plaats daarvan het PATH toe aan je shell-configuratiebestand door het
volgende commando uit te voeren:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  De bovenstaande stappen zijn niet van toepassing op `bash` gebruikers, omdat
  de wijzigingen aan de PATH variabele standaard behouden blijven.
</Callout>

</Tab>
</Tabs>

4. Herstart je terminal of voer het volgende commando uit om de terminal sessie
   te vernieuwen:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Controleer of de installatie is geslaagd door de Solana CLI versie te
   controleren:

```terminal
$ solana --version
```

Je ziet output zoals het volgende:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Je kunt alle beschikbare versies bekijken op de
[Agave Github repo](https://github.com/anza-xyz/agave/releases). Agave is de
validator client van [Anza](https://www.anza.xyz/), voorheen bekend als Solana
Labs validator client.

</Callout>

5. Update de Solana CLI naar de nieuwste versie, indien nodig _(Optioneel)_

```terminal
$ agave-install update
```

</Step>

### Installeer Anchor CLI

[Anchor](https://www.anchor-lang.com/) is een framework voor het ontwikkelen van
Solana-programma's. Het Anchor-framework maakt gebruik van Rust-macro's om het
proces van het schrijven van Solana-programma's te vereenvoudigen.

#### Vereisten

Het standaard Anchor-projecttestbestand (TypeScript) dat is aangemaakt met het
commando _shell`anchor init`_ vereist Node.js en Yarn. (De Rust-testtemplate is
beschikbaar via _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Node-installatie">

1. Installeer Node met behulp van
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) door het volgende
   commando uit te voeren:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Herstart je terminal en bevestig dat het commando `nvm` succesvol werkt:

```terminal
$ command -v nvm
```

3. Gebruik `nvm` om Node te installeren:

```terminal
$ nvm install node
```

4. Controleer of de installatie is geslaagd door de Node-versie te controleren:

```terminal
$ node --version
```

Je ziet output zoals het volgende:

```
v23.7.0
```

</Accordion>
<Accordion title="Yarn-installatie">

1. Installeer Yarn:

```terminal
$ npm install --global yarn
```

2. Controleer of de installatie is geslaagd door de Yarn-versie te controleren:

```terminal
$ yarn --version
```

Je ziet de volgende output:

```
1.22.1
```

</Accordion>
</Accordions>

#### Installatie

Je kunt de Anchor CLI en tooling op twee manieren installeren:

- Anchor Version Manager (AVM) — aanbevolen installatiemethode
- Zonder AVM — installeer rechtstreeks vanaf GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

De Anchor Version Manager (AVM) stelt je in staat om verschillende
Anchor-versies op je systeem te installeren en te beheren, en om in de toekomst
eenvoudig Anchor-versies bij te werken. Volg de onderstaande stappen om Anchor
te installeren met AVM:

1. Installeer AVM met het volgende commando:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Bevestig dat AVM succesvol is geïnstalleerd:

```terminal
$ avm --version
```

3. Installeer Anchor CLI met behulp van AVM:

Om de nieuwste versie te installeren:

```terminal
$ avm install latest
$ avm use latest
```

Om een specifieke versie te installeren, geef het versienummer op:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Bij het installeren van de Anchor CLI op Linux of WSL, kun je deze foutmelding
tegenkomen:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Als je deze foutmelding ziet, volg dan deze stappen:

1. Installeer de afhankelijkheden die vermeld staan in de sectie
   [Linux](/docs/intro/installation#install-dependencies) bovenaan deze pagina.
2. Probeer de Anchor CLI opnieuw te installeren.

</Callout>

4. Verifieer dat de installatie is geslaagd door de Anchor CLI-versie te
   controleren:

```terminal
$ anchor --version
```

Je ziet output zoals het volgende:

```
anchor-cli 0.31.1
```

<Callout type="info">

Vergeet niet om het commando _shell`avm use`_ uit te voeren om de Anchor
CLI-versie te declareren die op je systeem moet draaien.

- Als je de versie `latest` hebt geïnstalleerd, voer dan _shell`avm use latest`_
  uit.
- Als je de versie `0.30.1` hebt geïnstalleerd, voer dan _shell`avm use 0.30.1`_
  uit.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Installeer een specifieke versie van de Anchor CLI met het volgende commando:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Verifieer dat de installatie is gelukt door de Anchor CLI-versie te
   controleren:

```terminal
$ anchor --version
```

Je ziet output zoals het volgende:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>

### Installeer Surfpool CLI

[Surfpool](https://surfpool.run/) is een tool voor lokale ontwikkeling en een
verbeterde vervanging voor `solana-test-validator`. Lees meer over de functies
van Surfpool in de [Surfpool-documentatie](https://docs.surfpool.run/).

1. Installeer Surfpool met het volgende commando:

```terminal
$ curl -sL https://run.surfpool.run/ | bash
```

2. Verifieer dat de installatie is gelukt door de Surfpool CLI-versie te
   controleren:

```terminal
$ surfpool --version
```

Je ziet output zoals het volgende:

```
surfpool 0.12.0
```

</Step>
