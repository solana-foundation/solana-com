[Codama](https://github.com/codama-idl/codama) genereert TypeScript- en Rust-
clients vanuit het IDL-bestand van een programma. De gegenereerde clients
bevatten functies voor het aanroepen van instructies en het ophalen van
accounts, waardoor het niet meer nodig is om handmatig programma-accounts of
instruction data te serialiseren en deserialiseren.

<Callout type="warn">
  Codama is in actieve ontwikkeling en onderhevig aan veranderingen. Raadpleeg
  de gelinkte repositories in de snelle referentie voor de meest recente
  informatie.
</Callout>

## Snelle referentie

### CLI-opdrachten

| Commando      | Beschrijving                                             |
| ------------- | -------------------------------------------------------- |
| `codama init` | Genereer een codama-configuratiebestand                  |
| `codama run`  | Genereer programmaclients vanuit het configuratiebestand |

### Client-generators

| Package                                                                                                  | Beschrijving                                                                             |
| -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| [`@codama/renderers-js`](https://github.com/codama-idl/renderers-js)                                     | Genereer [`@solana/kit`](https://www.solanakit.com) compatibele TypeScript-clients       |
| [`@codama/renderers-rust`](https://github.com/codama-idl/renderers-rust)                                 | Genereer [`solana_sdk`](https://github.com/anza-xyz/solana-sdk) compatibele Rust-clients |
| [`@codama/nodes-from-anchor`](https://github.com/codama-idl/codama/tree/main/packages/nodes-from-anchor) | Converteer Anchor IDL's naar Codama IDL's voor het genereren van clients                 |

## !!steps Maak een Codama configuratiebestand

Codama gebruikt een configuratiebestand om aan te geven waar het IDL-bestand
zich bevindt, welke cliënttalen gegenereerd moeten worden en waar de
gegenereerde code moet worden opgeslagen. Maak dit configuratiebestand door het
volgende uit te voeren:

```terminal
$ npx codama init
```

Dit zal je vragen stellen om een `codama.json`-bestand aan te maken in de
hoofdmap van je project en de benodigde Codama-afhankelijkheden te installeren.

## !!steps Genereer clients

Zodra het configuratiebestand is aangemaakt, voer je het generatie-commando uit
om de clients te genereren:

```terminal
$ npx codama run --all
```

<WithMentions>

Dit zal de programmaclients genereren in de [bestandspaden](mention:path) die
zijn gespecificeerd in het codama-configuratiebestand.

```json title="codama.json"
{
  "idl": "idl.json",
  "before": [],
  "scripts": {
    "js": {
      "from": "@codama/renderers-js",
      // !mark
      // !mention path
      "args": ["clients/js/src/generated"]
    },
    "rust": {
      "from": "@codama/renderers-rust",
      "args": [
        // !mark
        // !mention path
        "clients/rust/src/generated",
        {
          "crateFolder": "clients/rust",
          "formatCode": true
        }
      ]
    }
  }
}
```

</WithMentions>

## De gegenereerde clients gebruiken

De gegenereerde clients bevatten broncode voor interactie met je programma, maar
bevatten geen `package.json` (voor TypeScript) of `Cargo.toml` (voor Rust)
bestanden.

Je moet ofwel een nieuw `package.json`- en `Cargo.toml`-bestand aanmaken, of de
vereiste afhankelijkheden van de gegenereerde clients toevoegen aan de bestaande
bestanden van je project.

### Geïntegreerd in je applicatie

Genereer de client rechtstreeks in de bronmap van je applicatie en voeg
vervolgens de vereiste afhankelijkheden toe aan je bestaande `package.json` of
`Cargo.toml`.

Gebruik deze aanpak bij het bouwen van een applicatie waarbij de clientcode niet
gedeeld of gepubliceerd hoeft te worden.

### Zelfstandige packages of crates

Maak aparte TypeScript-packages of Rust-crates voor elke client. Voeg
bijvoorbeeld `package.json` toe in `clients/js/` en `Cargo.toml` in
`clients/rust/`.

Gebruik deze aanpak om de client te publiceren als een herbruikbare bibliotheek
(npm of crates.io) of om de client te delen tussen meerdere applicaties in een
monorepo.
