AI-agenten worden steeds autonomer in hun workflows: code schrijven, data
ophalen, resources inrichten en meer. Om volledig onafhankelijk te kunnen
werken, hebben ze een manier nodig om programmatisch "microbetalingen" te doen
voor diensten, in real-time en tegen minimale kosten. Traditionele
betalingssystemen werken niet voor microbetalingen. De transactiekosten van
minder dan een cent en de afwikkeling in minder dan een seconde van Solana maken
dit technisch en economisch haalbaar.

## Waarom is dit belangrijk?

Stel je bijvoorbeeld voor dat een ontwikkelaar een LLM gebruikt om een
applicatie te bouwen en data nodig heeft van een betaalde API. Om belangrijke
data op te halen, zou een ontwikkelaar hun workflow moeten onderbreken, API's
moeten onderzoeken, een provider selecteren, een account aanmaken, een
betaalmethode toevoegen, een API-sleutel genereren, de sleutel in hun omgeving
kopiëren en vervolgens hun workflow hervatten. Deze wrijving doorbreekt de flow.
Voor AI-ondersteunde workflows is het een blokkade.

In plaats daarvan kan de agent met agentische betalingen het verzoek doen,
ervoor betalen en doorgaan met de workflow. Geen accounts. Geen API-sleutels.
Geen handmatige tussenkomst.

### Betaal voor API's met pay.sh

[pay.sh](https://pay.sh) is een betalingslaag voor HTTP-agents en command-line
tools. Raadpleeg de [pay.sh-documentatie](https://pay.sh/docs) om `pay` te
installeren. De `pay` CLI omhult tools zoals `curl`, `codex` en `claude` zodat
ze betaalde API's kunnen aanroepen zonder provideraccounts of API-sleutels.
Wanneer een API een MPP- of x402 `402 Payment Required`-challenge retourneert,
vraagt `pay` je lokale wallet om het ondertekenen goed te keuren.

Je kunt Pay installeren met:

```bash
brew install pay
```

of

```bash
npm install -g @solana/pay
```

Verifieer de installatie met:

```bash
pay --version
```

De toolchain stelt je adres automatisch in. Voer het volgende uit om je wallet
in te stellen en te laden:

```bash
pay setup
```

## Het x402-protocol

Agentische betalingen hebben een manier nodig waarop clients en servers
betalingsvoorwaarden kunnen onderhandelen via standaard webinfrastructuur zonder
traditionele webinterfaces te verstoren. De micropayment-ruimte is nog in
ontwikkeling en nieuwe tools en standaarden zullen waarschijnlijk ontstaan
naarmate de ruimte evolueert. Het [x402-protocol](https://www.x402.org/) heeft
zich echter als een vroege standaard met sterke ecosysteemondersteuning
gevestigd. x402 gebruikt
[HTTP's 402 "Payment Required"](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status/402)
statuscode—een standaard sinds HTTP/1.1 die pas praktisch werd met
blockchain-afwikkeling.

### Hoe het werkt

De client doet een verzoek, ontvangt een 402 met betalingsvoorwaarden en
probeert het opnieuw met een ondertekende betaling. De server delegeert
verificatie en afwikkeling aan een facilitator, een optionele tussenpersoon die
de onchain transactie-indiening afhandelt. Zodra de facilitator de afwikkeling
bevestigt, retourneert de server de gevraagde inhoud. Deze scheiding stelt
API-aanbieders in staat betalingen te accepteren zonder onchain-infrastructuur
rechtstreeks te beheren.

![x402 stroomdiagram](/assets/guides/x402/x402-flow-solana.png)

Het protocol is stateloos (geen sessies of API-sleutels), werkt met standaard
HTTP-infrastructuur en ondersteunt elk SPL-token dat de server accepteert. Hier
is een fragment van hoe je een Express-server instelt met x402-middleware:

```ts
app.use(
  paymentMiddleware(
    {
      "GET /costly-data": {
        accepts: [
          {
            scheme: "exact",
            price: "$0.001",
            network: "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
            payTo: svmAddress
          }
        ],
        description: "Costly data",
        mimeType: "application/json"
      }
    },
    new x402ResourceServer(facilitatorClient).register(
      "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
      new ExactSvmScheme()
    )
  )
);

app.get("/costly-data", (req, res) => {
  res.send({
    report: {
      data: "costly data"
    }
  });
});
```
