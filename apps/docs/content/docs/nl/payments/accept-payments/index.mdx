Accepteer betalingen op Solana met behulp van kant-en-klare componenten, het
Solana Pay-protocol of directe RPC-integratie.

## Conventie voor ontvangstadressen

Op Solana **deel je je wallet-adres, niet je token account**. Het is
gebruikelijk dat de verzender het juiste
[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)
aï¬‚eidt voor welke token ze ook versturen (zie
[Betalingen verzenden](/docs/payments/send-payments) voor meer details).

**Voorbeeld:** Deel `7EcDhS...` (je wallet), niet `9xKpQ...` (je USDC token
account).

<Callout>
  Voor point-of-sale en e-commerce, zie [Solana
  Pay](/docs/payments/accept-payments/solana-pay) voor QR-codes en
  betalings-URL's, of [Payment
  Button](/docs/payments/accept-payments/payment-button) voor kant-en-klare
  React-componenten.
</Callout>

## Integratieopties

| Aanpak                                                          | Het beste voor                       | Complexiteit |
| --------------------------------------------------------------- | ------------------------------------ | ------------ |
| [Payment Button](/docs/payments/accept-payments/payment-button) | React-apps, snelle integratie        | Laag         |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | QR-codes, betalingslinks, POS        | Gemiddeld    |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Aangepaste flows, volledige controle | Hoger        |

## Verificatie

Controleer na ontvangst van een betaling of deze is binnengekomen met behulp van
[Verificatietools](/docs/payments/accept-payments/verification-tools). Voor
applicaties met een hoog volume, zie
[Indexing](/docs/payments/accept-payments/indexing) voor productie-waardige
transactiemonitoring.
