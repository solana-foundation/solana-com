Je kunt betalingen accepteren op API-verzoeken via Pay.sh door een Pay-server in
te stellen. Installeer Pay met:

```bash
brew install pay
```

of

```bash
npm install -g @solana/pay
```

Verifieer de installatie met:

```bash
pay --version
```

De toolchain stelt je adres voor je in. Voer het volgende uit om je wallet in te
stellen en te voeden:

```bash
pay setup
```

## De demoserver instellen

Je kunt betalingen gaan accepteren door je server te starten. Je kunt een
demoserver in actie zien met:

```bash
pay server demo
```

Dit doet twee dingen. Het start een live Pay-server die je via een HTTP-verzoek
kunt bevragen, en het genereert een `pay-demo.yml` bestand in je huidige map.

Alle API-routes worden geconfigureerd via de `endpoints` YAML-configuratie. Om
betalingen te accepteren, moet je een `metering:` attribuut toevoegen. De
`/api/v1/reports/usage` route is als voorbeeld voor je uitgewerkt:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Je kunt testen of de betalingen werken door het volgende commando als client uit
te voeren in een andere terminal:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```
