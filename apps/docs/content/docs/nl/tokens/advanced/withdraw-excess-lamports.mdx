## Wat Doet Overtollige Lamports Opnemen?

_rs`WithdrawExcessLamports`_ verplaatst SOL die door een Token Program-account
wordt aangehouden boven het huurvrije minimum naar een bestemmingsaccount.
Tokenbalansen en mint-voorraad blijven onaangetast — alleen de overtollige
lamports worden verplaatst.

Dit herstelt SOL die eerder vast zat: lamports die naar een token account, mint
of multisig (allemaal eigendom van het Token Program) werden gestuurd, konden
anders niet worden verplaatst.

## Accounts

| Account       | Beschrijving                                                                |
| ------------- | --------------------------------------------------------------------------- |
| `source`      | Schrijfbaar. Het token account, de mint of het multisig om van op te nemen. |
| `destination` | Schrijfbaar. Ontvangt de overtollige lamports.                              |
| `authority`   | Ondertekent op basis van het type bronaccount (zie hieronder).              |

## Wie Kan Ondertekenen

De vereiste ondertekenaar hangt af van het type `source`-account:

| Bronaccount                     | Ondertekenaar                                                                   |
| ------------------------------- | ------------------------------------------------------------------------------- |
| Token account                   | De eigenaar van het token account (of de bijbehorende multisig-ondertekenaars). |
| Mint met een mint-autoriteit    | De mint-autoriteit.                                                             |
| Mint zonder een mint-autoriteit | Het mint account zelf moet ondertekenen (zie de opmerking hieronder).           |
| Multisig                        | De geconfigureerde M-van-N-ondertekenaars.                                      |

<Callout type="info">

Het geval "mint zonder mint authority" is ongebruikelijk. Wanneer dit van
toepassing is, is het mint-account de ondertekenaar: een programma-eigendom
(off-curve) mint ondertekent via een CPI van het eigenaarsprogramma, en een
keypair (on-curve) mint ondertekent met zijn sleutel.

</Callout>

<Callout type="warn">

Native (wrapped SOL) token accounts worden niet ondersteund en retourneren
_rs`NativeNotSupported`_. De opname kan het bronaccount niet onder het minimale
rent-exempt bedrag brengen.

</Callout>

## Hoe overtollige lamports op te nemen

Het voorbeeld gebruikt een geconfigureerde
[`@solana/kit`](/docs/clients/official/javascript#solana-kit) client — zie
[Tokens overmaken](/docs/tokens/basics/transfer-tokens) voor de
clientinstelling.

```ts title="Withdraw excess lamports (Kit)"
import { getWithdrawExcessLamportsInstruction } from "@solana-program/token";

// source/destination: addresses of the account to drain and the lamport recipient.
// authority: a TransactionSigner that signs per the source account type.
const instruction = getWithdrawExcessLamportsInstruction({
  source,
  destination,
  authority
});

await client.sendTransaction([instruction]);
```

Voor een multisig authority geeft u de mede-ondertekenaars mee met
_ts`multiSigners`_.
