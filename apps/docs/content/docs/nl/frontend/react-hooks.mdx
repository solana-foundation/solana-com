  Deel een Kit-client in je React-app met @solana/react en de
  wallet-plugin-hooks.
---

`@solana/react` publiceert een Kit-client naar je componentenstructuur via één
provider, waarna hooks RPC-gegevens, abonnementen en wallet-status daaruit
uitlezen. De browser-wallet-hooks bevinden zich in
`@solana/kit-plugin-wallet/react` en lezen van dezelfde provider — er is geen
aparte walletprovider.

## Installeren

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## De client bouwen

Bouw de client eenmalig op moduleniveau, buiten de React-boom. In de browser
maakt `walletSigner` de verbonden wallet zowel de betaler van de
transactiekosten als de identiteit.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` voegt `client.wallet` toe (Wallet Standard-detectie en
-verbinding), en `solanaRpc` voegt `client.rpc` plus de hulpfunctie
`client.sendTransaction` toe. Exporteer het clienttype gewrapped in `Awaited<>`
zodat het correct blijft als je later een plugin toevoegt die asynchroon
resolvet, en geef dat `AppClient` door aan elke `useClient`-aanroep.

## Je boom eenmalig inpakken

`ClientProvider` publiceert de client. Alle onderstaande hooks lezen daaruit.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Wallet-hooks

De wallet-hooks lezen de `wallet`-capability van de client, dus ze werken alleen
binnen een `ClientProvider` waarvan de client een wallet-plugin heeft
geïnstalleerd.

- `useWallets()` — Wallet Standard-wallets gedetecteerd voor de chain van de
  client.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` of `null`. `signer` is
  `null` voor alleen-lezen wallets.
- `useConnect()` / `useDisconnect()` — retourneren een `ActionResult` waarvan
  `dispatch` (en `dispatchAsync`) de actie uitvoert en waarvan `isRunning`,
  `status`, `data`, `error` en `reset` het resultaat bijhouden.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Een transactie verzenden

`useClient` retourneert dezelfde client die je hebt gepubliceerd. Bouw
instructies met een gegenereerde programma-client en verzend ze via
`client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Het doorgeven van het geëxporteerde `AppClient` type aan `useClient` geeft de
hook de volledige plugin-uitgebreide client, zodat `client.rpc` en
`client.sendTransaction` getypeerd zijn.

## Gegevens lezen

`@solana/react` biedt hooks die via de client gegevens uitlezen:

- `useRequest(source)` — eenmalige RPC-leesactie; geeft
  `{ data, error, status, refresh }` terug.
- `useSubscription(source)` — een live abonnement; geeft
  `{ data, error, reconnect, status }` terug.
- `useTrackedData(spec)` — een via RPC gevulde waarde die actueel wordt gehouden
  via een abonnement.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Memoïseer de bron (met `useMemo`, of `useCallback` voor een functie) zodat de
hook alleen opnieuw ophaalt wanneer de invoer verandert en niet bij elke render.

Voor caching en hervalidatie wikkelen de subpaden `@solana/react/swr` en
`@solana/react/query` dezelfde hooks in voor [SWR](https://swr.vercel.app) en
[TanStack Query](https://tanstack.com/query).

## Veelvoorkomende patronen voor Solana-ontwikkelaars

- **Één provider**: `ClientProvider` is de enige provider die je nodig hebt;
  wallet-hooks lezen deze via de `wallet` capability van de client.
- **Bewust van servercomponenten**: Markeer alleen leaf-componenten die hooks
  aanroepen met `"use client"`; server-reads kunnen een gewone Kit RPC-client
  zonder wallet gebruiken.
- **Testen**: Publiceer een gemockte client via `ClientProvider` om wallets en
  RPC-responses te simuleren.

<Callout type="warn" title="Twee hook-families, zelfde namen">
  `useSignIn` en `useSignMessage` bestaan in zowel `@solana/react` (ze nemen een
  `UiWalletAccount` argument) als `@solana/kit-plugin-wallet/react`
  (client-gebaseerd, geen account-argument, geeft een `ActionResult` terug).
  Gebruik de `@solana/kit-plugin-wallet/react` versies met het bovenstaande
  clientpatroon; de oudere `@solana/react` wallet-hooks worden vervangen.
</Callout>

Combineer deze handleiding met het [Kit client-overzicht](/docs/frontend/client)
om te begrijpen welke client elke hook uitleest.
