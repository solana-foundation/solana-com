Als je afkomstig bent van `@solana/web3.js` (v1), `@solana/wallet-adapter-*`, of
de eerdere framework-kit-pakketten (`@solana/client`, `@solana/react-hooks`,
`@solana/web3-compat`), zijn die benaderingen achterhaald. Nieuwe apps worden
rechtstreeks gebouwd op `@solana/kit` en de kit-plugins. Deze pagina koppelt de
concepten die je al kent aan hun Kit-equivalenten.

## Conceptmapping

| web3.js v1 / wallet-adapter            | Kit                                                                           |
| -------------------------------------- | ----------------------------------------------------------------------------- |
| `new Connection(url)`                  | `createSolanaRpc(url)`, of `.use(solanaRpc({ rpcUrl }))` op een client        |
| `PublicKey`                            | `Address` — converteer een string met `address("...")`                        |
| `Keypair`                              | `generateKeyPairSigner()` of de `@solana/kit-plugin-signer`-plugins           |
| `SystemProgram.transfer(...)`          | `getTransferSolInstruction(...)` van `@solana-program/system`                 |
| `LAMPORTS_PER_SOL` rekenkunde          | `lamports(...)` van `@solana/kit`                                             |
| `Transaction` / `VersionedTransaction` | Transactieberichten gebouwd en samengesteld door de transactieplanner         |
| `sendAndConfirmTransaction(...)`       | `client.sendTransaction([...])` (planner ondertekent, verstuurt en bevestigt) |
| `@solana/wallet-adapter-*`             | `@solana/kit-plugin-wallet` + Wallet Standard-detectie                        |

## Afkomstig van web3.js?

Als je een grote `@solana/web3.js` v1-codebase onderhoudt en een volledige
herschrijving naar Kit nog niet haalbaar is, is web3.js v3 de brug. Het wordt
momenteel uitgebracht als `@solana/web3.js@rc` (de `3.0.0-rc`-lijn) en herbouwt
de klassieke op klassen gebaseerde API — `Connection`, `Keypair`, `Transaction`
— bovenop `@solana/kit`- internals. Omdat het op Kit is gebouwd, is de
interoperabiliteit nauw: `PublicKey` is een verouderde alias van `Address`, en
een v3 `Keypair` voldoet structureel aan Kit's `KeyPairSigner`, zodat je het
direct kunt doorgeven aan Kit-API's, plugins en Codama-gegenereerde clients.

v3 is nog steeds een **release candidate** — pin exacte versies en verwacht
enige API- wijzigingen tussen RC's. Het is een pad voor verouderde
interoperabiliteit, geen standaard voor nieuw werk; nieuwe apps moeten
rechtstreeks op Kit worden gebouwd. Voor de migratie zelf, zie de officiële
[web3.js v1 → v3-migratiegids](https://github.com/solana-foundation/solana-web3.js/blob/v3.x/docs/web3js-v1-to-v3-migration.md).

## Wallets

Kit gebruikt geen wallet-adapter. `@solana/kit-plugin-wallet` ontdekt Wallet
Standard-wallets en stelt ze beschikbaar via `client.wallet` en de
[React hooks](/docs/frontend/react-hooks). Moderne wallets (Phantom, Solflare,
Backpack en anderen) melden zichzelf aan via Wallet Standard, dus er is geen
per-wallet adapterpakket vereist.

Je kunt ook vooraf gebouwde tools gebruiken zoals
[@solana/connector](https://www.npmjs.com/package/@solana/connector), die
Kit-primitieven rechtstreeks gebruikt om verbinding te maken met wallets.

<Callout type="info" title="Nog op wallet-adapter?">
  Als je een app onderhoudt die gebouwd is op `@solana/wallet-adapter-*` en nog
  niet klaar bent om te migreren, blijft de [wallet-adapter
  repository](https://github.com/anza-xyz/wallet-adapter) de referentie voor dat
  stack. Voor alles wat nieuw is, geef de voorkeur aan de Kit wallet-plugin.
</Callout>
