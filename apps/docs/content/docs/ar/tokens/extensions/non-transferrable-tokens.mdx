## ما هي الرموز غير القابلة للتحويل؟

إن امتداد mint الخاص بـ _rs`NonTransferable`_ في Token Extensions Program يجعل
كل token account لذلك الـ mint غير قابل للتحويل. بعد سك الرموز، لا يمكن لحاملي
الرموز نقلها إلى token account آخر باستخدام _rs`Transfer`_ أو
_rs`TransferChecked`_.

هذا النمط مفيد للأصول التي يجب أن تبقى مرتبطة بمحفظة واحدة.

لا تزال الرموز غير القابلة للتحويل قابلة لـ:

- السك بواسطة صلاحية الـ mint
- الحرق بواسطة مالك الـ token account أو مفوض معتمد
- يمكن إغلاق حسابات الرموز بعد وصول الرصيد إلى الصفر

<Callout type="info" title="امتدادات حساب الرمز">
  عندما يتم تهيئة token account لـ mint مع _rs`NonTransferable`_، يتم تهيئة الـ
  token account مع _rs`NonTransferableAccount`_ و _rs`ImmutableOwner`_. عندما
  يتم إنشاء الـ token account من خلال [Associated Token
  Program](/docs/tokens/basics/create-token-account#how-to-create-an-associated-token-account)،
  يتم حساب حجم الحساب المطلوب وإنشاء الـ token account بالحجم المطلوب والـ
  lamports المعفاة من الـ rent.
</Callout>

## كيفية إنشاء Mint غير قابل للتحويل

لإنشاء mint غير قابل للتحويل:

1. احسب حجم حساب الـ mint والـ rent المطلوب للـ mint وامتداد
   _rs`NonTransferable`_.
2. أنشئ حساب الـ mint باستخدام _rs`CreateAccount`_، وهيئ _rs`NonTransferable`_،
   وهيئ الـ mint باستخدام _rs`InitializeMint`_.
3. أنشئ حسابات رموز للـ mint. يتم تمكين _rs`NonTransferableAccount`_ و
   _rs`ImmutableOwner`_ تلقائيًا لحسابات الرموز.
4. تفشل _rs`Transfer`_ و _rs`TransferChecked`_ مع
   _rs`TokenError::NonTransferable`_.

<ScrollyCoding>

## !!steps احسب حجم الحساب

احسب حجم حساب الـ mint للـ mint الأساسي بالإضافة إلى امتداد
_rs`NonTransferable`_. هذا هو الحجم المستخدم في _rs`CreateAccount`_.
