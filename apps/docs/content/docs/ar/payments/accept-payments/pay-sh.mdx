---

يمكنك قبول المدفوعات على طلبات API عبر Pay.sh من خلال إعداد خادم Pay. ثبّت Pay
باستخدام:

```bash
brew install pay
```

أو

```bash
npm install -g @solana/pay
```

تحقق من التثبيت باستخدام:

```bash
pay --version
```

ستقوم سلسلة الأدوات بإعداد عنوانك تلقائيًا. شغّل الأمر التالي لإعداد محفظتك
وتعبئتها:

```bash
pay setup
```

## إعداد الخادم التجريبي

يمكنك البدء في قبول المدفوعات عن طريق تشغيل الخادم الخاص بك. يمكنك مشاهدة الخادم
التجريبي في العمل باستخدام:

```bash
pay server demo
```

يؤدي هذا وظيفتين: يشغّل خادم Pay مباشرةً يمكنك الاستعلام عنه عبر طلب HTTP،
ويُنشئ ملف `pay-demo.yml` في دليلك الحالي.

يتم تكوين جميع مسارات API عبر إعداد YAML الخاص بـ `endpoints`. لقبول المدفوعات،
يجب عليك إضافة سمة `metering:`. وقد تم تنفيذ مسار `/api/v1/reports/usage` كمثال
توضيحي لك:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

يمكنك اختبار عمل المدفوعات عن طريق تشغيل الأمر التالي كعميل في محطة طرفية أخرى:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

يمكنك الاطلاع على مزيد من المعلومات حول كيفية تحديد الأسعار في
[وثائق Pay.sh - البناء مع Pay > التسعير](https://pay.sh/docs/building-with-pay/pricing)

تعرف على المزيد حول إعداد Pay.sh في بيئة الإنتاج في
[وثائق تكوين Pay.sh](https://pay.sh/docs/toolchain/configuration).
