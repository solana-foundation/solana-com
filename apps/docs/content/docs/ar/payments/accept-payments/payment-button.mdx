يوفر مكون `PaymentButton` من
[Commerce Kit](https://www.npmjs.com/package/@solana-commerce/kit) واجهة دفع
كاملة—تتعامل مع اتصال المحفظة، واختيار الرمز المميز، ومعالجة المعاملات، وحالة
واجهة المستخدم بشكل جاهز.

## التثبيت

```bash
pnpm add @solana-commerce/kit
```

## الاستخدام الأساسي

```tsx
import { PaymentButton } from "@solana-commerce/kit";

function Checkout() {
  return (
    <PaymentButton
      config={{
        merchant: { name: "My Store", wallet: "your-wallet-address" },
        mode: "tip"
      }}
      onPaymentSuccess={(signature) => {
        console.log("Payment confirmed:", signature);
      }}
    />
  );
}
```

تحقق من
[Commerce Kit Playground](https://launch.solana.com/products/commercekit/playground)
لتكوين واختبار المكون.

## أوضاع الدفع

يحدد خيار `mode` سلوك الزر:

| الوضع | الوصف                                            |
| ----- | ------------------------------------------------ |
| `tip` | يختار المستخدم المبلغ الخاص به (تبرعات/إكراميات) |

## التكوين

### التكوين المطلوب

| الخاصية           | النوع    | الوصف                                  |
| ----------------- | -------- | -------------------------------------- |
| `merchant.name`   | `string` | اسم النشاط التجاري المعروض أثناء الدفع |
| `merchant.wallet` | `string` | عنوان محفظة سولانا لاستقبال المدفوعات  |
| `mode`            | 'tip'`   | نوع تدفق الدفع                         |

### التكوين الاختياري

| الخاصية        | النوع                   | الوصف                                          |
| -------------- | ----------------------- | ---------------------------------------------- |
| `network`      | `'mainnet' \| 'devnet'` | شبكة سولانا (افتراضي: `'mainnet'`)             |
| `rpcUrl`       | `string`                | نقطة نهاية RPC مخصصة                           |
| `allowedMints` | `string[]`              | تقييد الرموز المميزة المقبولة حسب عنوان السك   |
| `showQR`       | `boolean`               | تفعيل خيار الدفع برمز QR                       |
| `theme`        | `ThemeConfig`           | التخصيص المرئي (الألوان، نصف قطر الحدود، إلخ.) |

{/\*

### إعدادات الدفع

بالنسبة لوضعي `buyNow` و `cart`، قدم المنتجات عبر `paymentConfig`:

```tsx
paymentConfig={{
  products: [
    { id: '1', name: 'T-Shirt', price: 25, quantity: 2 },
    { id: '2', name: 'Hoodie', price: 45, quantity: 1 }
  ]
}}
```

\*/}

## استدعاءات الأحداث

| الاستدعاء          | المعاملات             | الوصف                         |
| ------------------ | --------------------- | ----------------------------- |
| `onPaymentSuccess` | `(signature: string)` | تم تأكيد المعاملة على السلسلة |
| `onPaymentError`   | `(error: Error)`      | فشل الدفع في أي مرحلة         |
| `onPaymentStart`   | `()`                  | تم بدء تدفق الدفع             |
| `onCancel`         | `()`                  | ألغى المستخدم عملية الدفع     |

## محفز مخصص

استبدل الزر الافتراضي بعنصرك الخاص:

```tsx
<PaymentButton
  config={{
    merchant: { name: "Shop", wallet: "address" },
    mode: "tip"
  }}
>
  <button className="my-custom-button">Pay with Solana</button>
</PaymentButton>
```
