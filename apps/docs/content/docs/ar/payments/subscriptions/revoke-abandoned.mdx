---
title: إلغاء التفويضات والاشتراكات المهجورة واستعادة الـ rent
description:
  دليل لاستعادة الـ rent من تفويض أو اشتراك اختفت سلطة الاشتراك الخاصة به.
---

تحتفظ التفويضات والاشتراكات بالـ rent الذي يعود عادةً إلى الدافع المسجّل عند
الإلغاء. إذا موّل راعٍ الحساب وقام المستخدم لاحقاً بإغلاق سلطة الاشتراك الخاصة
به أو استبدالها، يصبح الحساب معلقاً ويتعذّر استرداد الـ rent الخاص به. يتيح كلٌّ
من `RevokeAbandonedDelegation` و`RevokeAbandonedSubscription` للدافع المسجّل
استعادته.

## متى يُطبَّق

لا يُسمح بذلك إلا عندما تكون سلطة الاشتراك للحساب منتهية بشكل قاطع — مغلقة، أو
مغلقة وأُعيد إنشاؤها (بحيث لم يعد `init_id` مطابقاً). لا يمكن إغلاق تفويض أو
اشتراك نشط بهذه الطريقة أبداً.

|                                   | `RevokeAbandonedDelegation`                                                                                           | `RevokeAbandonedSubscription`                                                                                        |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| يُغلق                             | PDA تفويض ثابت أو متكرر                                                                                               | PDA اشتراك                                                                                                           |
| الموقّع                           | الدافع المسجّل                                                                                                        | الدافع المسجّل                                                                                                       |
| الحسابات                          | التفويض + السلطة المسجّلة                                                                                             | الاشتراك + السلطة المسجّلة + الخطة                                                                                   |
| يكون مؤهلاً عندما                 | السلطة مغلقة، أو أُعيد إنشاؤها (عدم تطابق `init_id`)                                                                  | نفس الشرط، لعملة سك الخطة — يُقرأ عنوان السك من الخطة المرتبطة، لذلك لا يمكن استبدال سلطة غير ذات صلة لإجبار الإغلاق |
| استخدم الإلغاء العادي عوضاً عندما | التفويض [منتهي الصلاحية أو مُستنفَد بالكامل](/docs/payments/subscriptions/recurring-delegation#revoke-the-delegation) | الاشتراك [ملغى ومنتهي الصلاحية](/docs/payments/subscriptions/subscription-plan#cancel-and-revoke)                    |

يُعدّ `RevokeAbandonedDelegation` أيضاً المسار الوحيد لاسترداد الدافع المسجّل
للـ rent من تفويض لا تنتهي صلاحيته أبداً (`expiry_ts == 0`) ولم يُستنفَد
بالكامل: إذ لا يُفعَّل الإلغاء العادي للدافع إلا عند انتهاء الصلاحية، أو عند
الإنفاق الكامل في حالة التفويض الثابت.

## إلغاء تفويض مهجور

يقوم الدافع المسجّل بالتوقيع. مرّر PDA التفويض المنتهي وسلطة الاشتراك المسجّلة
الخاصة به (والتي قد تكون مغلقة بالفعل). يعود الـ rent إلى الدافع.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedDelegationInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedDelegationInstruction({
      payer: payerSigner,
      delegationAccount: delegationPda,
      subscriptionAuthority: subscriptionAuthorityPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let delegation_pda: Address = address!("DELEGATION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");

    let revoke_abandoned_ix = RevokeAbandonedDelegationBuilder::new()
        .payer(payer)
        .delegation_account(delegation_pda)
        .subscription_authority(subscription_authority)
        .instruction();
    ```

  </Tab>
</Tabs>

## إلغاء اشتراك متروك

يوقّع الدافع المسجَّل. مرِّر PDA الخاص بالاشتراك المتروك، وسلطة الاشتراك
المسجَّلة (التي قد تكون مغلقة بالفعل)، والخطة التي ينتمي إليها الاشتراك. يعود
rent إلى الدافع.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedSubscriptionInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedSubscriptionInstruction({
      payer: payerSigner,
      subscriptionAccount: subscriptionPda,
      subscriptionAuthority: subscriptionAuthorityPda,
      planPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let subscription_pda: Address = address!("SUBSCRIPTION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");
    let plan_pda: Address = address!("PLAN_PDA_ADDRESS_HERE");

    let revoke_abandoned_ix = RevokeAbandonedSubscriptionBuilder::new()
        .payer(payer)
        .subscription_account(subscription_pda)
        .subscription_authority(subscription_authority)
        .plan_pda(plan_pda)
        .instruction();
    ```

  </Tab>
</Tabs>

## ملاحظات

- يوقّع الدافع المسجَّل، وليس المفوِّض أو المفوَّض إليه أو المشترك.
- يفشل كلا التعليمتين إذا كانت سلطة الاشتراك لا تزال نشطة للحساب — استخدم
  الإلغاء العادي بدلاً من ذلك.
- `RevokeAbandonedDelegation` تعمل مع كلٍّ من التفويضات الثابتة والمتكررة.
