تثبيت سريع لإعداد بيئة تطوير سولانا المحلية الخاصة بك بأمر واحد. تثبيت Rust
وواجهة سطر أوامر سولانا وإطار عمل Anchor على Windows (WSL) و Linux و Mac. (إذا
كنت تفضل ذلك، يمكنك
[تثبيت كل تبعية على حدة](/docs/intro/installation/dependencies).)

## المتطلبات الأساسية

<Accordions>
<Accordion title="Windows">

لتطوير برامج سولانا على Windows **يجب** عليك استخدام
[نظام Windows الفرعي لـ Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). بعد تثبيت WSL، سيتم تثبيت جميع التبعيات الأخرى من خلال طرفية Linux.

لتثبيت WSL، قم بتشغيل الأمر التالي في Windows PowerShell:

```terminal
$ wsl --install
```

ستطالبك عملية التثبيت بإنشاء حساب مستخدم افتراضي.

![تثبيت WSL](/assets/docs/intro/installation/wsl-install.png)

بشكل افتراضي، يقوم WSL بتثبيت Ubuntu. يمكنك فتح طرفية Linux عن طريق البحث عن
"Ubuntu" في شريط البحث.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

إذا كانت طرفية Ubuntu الخاصة بك تبدو مثل الصورة أدناه، فقد تواجه مشكلة حيث
`ctrl + v` (اختصار لوحة المفاتيح للصق) لا يعمل في الطرفية.

![طرفية Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

إذا واجهت هذه المشكلة، افتح Windows Terminal عن طريق البحث عن "Terminal" في شريط
البحث.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

بعد ذلك، أغلق Windows Terminal وأعد فتح طرفية Linux عن طريق البحث عن Ubuntu مرة
أخرى. يجب أن تبدو الطرفية الآن مثل الصورة أدناه، حيث `ctrl + v` (اختصار لوحة
المفاتيح للصق) يعمل.

![طرفية Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

إذا كنت تستخدم VS Code، فإن
[إضافة WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) تمكنك من
استخدام WSL و VS Code معًا.

![إعداد WSL في VS Code](/assets/docs/intro/installation/wsl-vscode.png)

سترى ما يلي في شريط حالة VS Code:

![WSL: أوبونتو](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

بمجرد إعداد WSL، ستكون جاهزًا إما لـ:

- إكمال [التثبيت السريع](#installation) باستخدام طرفية أوبونتو (لينكس)
- تثبيت [Rust](/docs/intro/installation/dependencies#install-rust) و
  [واجهة سطر أوامر سولانا](/docs/intro/installation/dependencies#install-solana-cli)
  و
  [واجهة سطر أوامر Anchor](/docs/intro/installation/dependencies#install-anchor-cli)
  بشكل منفصل، كما هو موضح أدناه

<Tab value=".deb" >
إذا كنت تستخدم Debian أو البدائل المشتقة من Debian مثل Ubuntu أو Mint، اتبع الخطوات أدناه:

1. قم بتحديث قوائم الحزم للتأكد من أن جميع الحزم محدثة:

```terminal
$ sudo apt-get update
```

2. قم بتثبيت التبعيات التالية:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
إذا واجهت الخطأ التالي عند تثبيت `protobuf-compiler`، تأكد من
تشغيل _shell`sudo apt-get update` أولاً

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    بالنسبة لمشتقات RHEL مثل Fedora أو Red Hat Linux، اتبع الخطوات التالية.

قم بتحديث قوائم الحزم للحصول على إصدارات أحدث من الحزم عن طريق تشغيل الأمر.

```terminal
$ sudo dnf upgrade --refresh
```

بعد ذلك، قم بتثبيت التبعيات التالية:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## التثبيت

1. أدخل الأمر التالي في الطرفية الخاصة بك لتثبيت جميع التبعيات الضرورية:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

سيُرجع التثبيت الناجح مخرجات مثل التالي:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. تحقق من نجاح التثبيت عن طريق فحص إصدار كل تبعية مثبتة.

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  إذا فشل أمر التثبيت السريع، يُرجى الرجوع إلى قسم [تثبيت
  التبعيات](/docs/intro/installation/dependencies) أدناه للحصول على تعليمات
  تثبيت كل تبعية على حدة.
</Callout>
