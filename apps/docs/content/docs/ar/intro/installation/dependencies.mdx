دليل لإعداد بيئة تطوير سولانا المحلية الخاصة بك. تعلم كيفية تثبيت Rust وواجهة
سطر أوامر سولانا وإطار عمل Anchor على Windows (WSL) وLinux و Mac. استخدم هذا
الدليل إذا كنت تفضل تثبيت كل تبعية على حدة، أو إذا فشل
[التثبيت السريع](/docs/intro/installation/) لأي سبب.

### المتطلبات الأساسية

<Accordions>
<Accordion title="Windows">

لتطوير برامج سولانا على Windows **يجب** عليك استخدام
[نظام Windows الفرعي لـ Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). بعد تثبيت WSL، سيتم تثبيت جميع التبعيات الأخرى من خلال طرفية Linux.

لتثبيت WSL، قم بتشغيل الأمر التالي في Windows PowerShell:

```terminal
$ wsl --install
```

ستطالبك عملية التثبيت بإنشاء حساب مستخدم افتراضي.

![تثبيت WSL](/assets/docs/intro/installation/wsl-install.png)

بشكل افتراضي، يقوم WSL بتثبيت Ubuntu. يمكنك فتح طرفية Linux عن طريق البحث عن
"Ubuntu" في شريط البحث.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

إذا كانت طرفية Ubuntu الخاصة بك تبدو مثل الصورة أدناه، فقد تواجه مشكلة حيث لا
يعمل `ctrl + v` (اختصار لوحة المفاتيح للصق) في الطرفية.

![طرفية Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

إذا واجهت هذه المشكلة، افتح Windows Terminal عن طريق البحث عن "Terminal" في شريط
البحث.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

بعد ذلك، أغلق Windows Terminal وأعد فتح طرفية Linux عن طريق البحث عن Ubuntu مرة
أخرى. يجب أن تبدو الطرفية الآن مثل الصورة أدناه، حيث يعمل `ctrl + v` (اختصار
لوحة المفاتيح للصق).

![طرفية Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

إذا كنت تستخدم VS Code، فإن
[إضافة WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) تمكنك من
استخدام WSL وVS Code معاً.

![إعداد WSL في VS Code](/assets/docs/intro/installation/wsl-vscode.png)

سترى ما يلي في شريط الحالة في VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

بمجرد إعداد WSL، ستكون جاهزًا إما:

- إكمال [التثبيت السريع](/docs/intro/installation) باستخدام طرفية Ubuntu (Linux)
- تثبيت [Rust](#install-rust) و[Solana CLI](#install-solana-cli) و
  [Anchor CLI](#install-anchor-cli) بشكل فردي، كما هو موضح أدناه

</Accordion>

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
إذا كنت تستخدم Debian أو البدائل المشتقة من Debian مثل Ubuntu أو Mint، اتبع الخطوات أدناه:

1. قم بتحديث قوائم الحزم للتأكد من أن جميع الحزم محدثة:

```terminal
$ sudo apt-get update
```

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

تشغيل _shell`sudo apt-get update` أولاً

```

This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

```terminal
$ sudo dnf upgrade --refresh
```

بعد ذلك، قم بتثبيت التبعيات التالية:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### تثبيت Rust

يقوم المطورون ببناء برامج سولانا باستخدام لغة البرمجة
[Rust](https://www.rust-lang.org/).

1. قم بتثبيت Rust باستخدام [rustup](https://www.rust-lang.org/tools/install) عن
   طريق إدخال الأمر التالي في الطرفية الخاصة بك:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

بعد التثبيت الناجح سترى الرسالة التالية:

<Accordions>
<Accordion title="رسالة تثبيت Rust الناجح">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. أعد تحميل متغير بيئة PATH الخاص بك لتضمين دليل bin الخاص بـ
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. تحقق من نجاح التثبيت.

```terminal
$ rustc --version
```

سترى مخرجات مثل ما يلي:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### تثبيت Solana CLI

توفر Solana CLI جميع الأدوات المطلوبة لبناء ونشر برامج سولانا.

1. قم بتثبيت مجموعة أدوات Solana CLI باستخدام أمر التثبيت الرسمي:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  يمكنك استبدال `stable` بعلامة الإصدار المطابقة لإصدار البرنامج الذي تريده (مثل
  `v2.0.3`)، أو استخدام أحد أسماء القنوات الرمزية الثلاثة: `stable`، `beta`، أو
  `edge`.
</Callout>

2. أضف متغير بيئة PATH

قد يطالبك التثبيت الأول لـ Solana CLI بإضافة متغير بيئة `PATH`. للقيام بذلك،
أغلق وأعد فتح الطرفية الخاصة بك أو قم بتشغيل الأمر التالي في shell الخاص بك:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. حدّث متغير بيئة PATH الخاص بك

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

إذا كنت تستخدم **Linux** أو **WSL**، يجب عليك إضافة ملف Solana CLI الثنائي إلى
PATH الخاص بك حتى يكون الأمر متاحًا في الطرفية الخاصة بك. للقيام بذلك، اتبع
الخطوات أدناه:

**أ.** تحقق من shell الذي تستخدمه:

```terminal
$ echo $SHELL
```

- إذا كانت المخرجات تحتوي على `/bash`، استخدم `.bashrc`.
- إذا كانت المخرجات تحتوي على `/zsh`، استخدم `.zshrc`.

**ب.** قم بتشغيل الأمر المناسب، بناءً على الصدفة (shell) التي تستخدمها.

لـ Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

لـ Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

إذا كنت تستخدم جهاز Mac مع `zsh`، فإن تشغيل أمر `export PATH` الافتراضي المسجل
من التثبيت لا يستمر بمجرد إغلاق الطرفية. بدلاً من ذلك، أضف المسار (PATH) إلى ملف
تكوين الصدفة الخاص بك عن طريق تشغيل الأمر التالي:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  الخطوات أعلاه لا تنطبق على مستخدمي `bash`، حيث ستستمر التغييرات على متغير PATH
  بشكل افتراضي.
</Callout>

</Tab>
</Tabs>

4. أعد تشغيل الطرفية أو قم بتشغيل الأمر التالي لتحديث جلسة الطرفية:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. تحقق من نجاح التثبيت عن طريق فحص إصدار واجهة سطر أوامر سولانا:

```terminal
$ solana --version
```

سترى مخرجات مشابهة لما يلي:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

يمكنك عرض جميع الإصدارات المتاحة على
[مستودع Agave على Github](https://github.com/anza-xyz/agave/releases). Agave هو
عميل المدقق (validator) من [Anza](https://www.anza.xyz/)، المعروف سابقاً باسم
عميل المدقق من Solana Labs.

</Callout>

5. قم بتحديث واجهة سطر أوامر سولانا إلى أحدث إصدار، حسب الحاجة _(اختياري)_

```terminal
$ agave-install update
```

</Step>

### تثبيت واجهة سطر أوامر Anchor

[Anchor](https://www.anchor-lang.com/) هو إطار عمل لتطوير برامج سولانا. يستفيد
إطار عمل Anchor من وحدات ماكرو Rust لتبسيط عملية كتابة برامج سولانا.

#### المتطلبات الأساسية

ملف اختبار مشروع Anchor الافتراضي (TypeScript) الذي تم إنشاؤه باستخدام الأمر
_shell`anchor init`_ يتطلب Node.js وYarn. (قالب اختبار Rust متاح باستخدام
_shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="تثبيت Node">

1. قم بتثبيت Node باستخدام
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) عن طريق تشغيل
   الأمر التالي:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. أعد تشغيل الطرفية الخاصة بك وتأكد من أن الأمر `nvm` يعمل بنجاح:

```terminal
$ command -v nvm
```

3. استخدم `nvm` لتثبيت Node:

```terminal
$ nvm install node
```

4. تحقق من نجاح التثبيت، عن طريق التحقق من إصدار Node:

```terminal
$ node --version
```

سترى مخرجات مشابهة لما يلي:

```
v23.7.0
```

</Accordion>
<Accordion title="تثبيت Yarn">

1. قم بتثبيت Yarn:

```terminal
$ npm install --global yarn
```

2. تحقق من نجاح التثبيت، عن طريق التحقق من إصدار Yarn:

```terminal
$ yarn --version
```

سترى المخرجات التالية:

```
1.22.1
```

</Accordion>
</Accordions>

#### التثبيت

يمكنك تثبيت Anchor CLI والأدوات بطريقتين:

- Anchor Version Manager (AVM) — طريقة التثبيت الموصى بها
- بدون AVM — التثبيت مباشرة من GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

يتيح لك مدير إصدارات Anchor (AVM) تثبيت وإدارة إصدارات Anchor المختلفة على نظامك
وتحديث إصدارات Anchor بسهولة في المستقبل. لتثبيت Anchor باستخدام AVM، اتبع
الخطوات أدناه:

1. قم بتثبيت AVM باستخدام الأمر التالي:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. تأكد من نجاح تثبيت AVM:

```terminal
$ avm --version
```

3. قم بتثبيت واجهة سطر أوامر Anchor باستخدام AVM:

لتثبيت أحدث إصدار:

```terminal
$ avm install latest
$ avm use latest
```

لتثبيت إصدار محدد، حدد رقم الإصدار:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

عند تثبيت واجهة سطر أوامر Anchor على Linux أو WSL، قد تواجه هذا الخطأ:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

إذا رأيت رسالة الخطأ هذه، اتبع الخطوات التالية:

1. قم بتثبيت التبعيات المدرجة في قسم
   [Linux](/docs/intro/installation#install-dependencies) في أعلى هذه الصفحة.
2. أعد محاولة تثبيت واجهة سطر أوامر Anchor.

</Callout>

4. تحقق من نجاح التثبيت، من خلال التحقق من إصدار واجهة سطر أوامر Anchor:

```terminal
$ anchor --version
```

سترى مخرجات مشابهة لما يلي:

```
anchor-cli 0.31.1
```

<Callout type="info">

لا تنس تشغيل أمر _shell`avm use`_ لتحديد إصدار واجهة سطر أوامر Anchor الذي سيتم
تشغيله على نظامك.

- إذا قمت بتثبيت الإصدار `latest`، قم بتشغيل _shell`avm use latest`_.
- إذا قمت بتثبيت الإصدار `0.30.1`، قم بتشغيل _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. قم بتثبيت إصدار محدد من واجهة سطر أوامر Anchor باستخدام الأمر التالي:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. تحقق من نجاح التثبيت، من خلال التحقق من إصدار واجهة سطر أوامر Anchor:

```terminal
$ anchor --version
```

سترى مخرجات مشابهة لما يلي:

```
anchor-cli 0.31.1
```

</Tab>

### تثبيت واجهة سطر أوامر Surfpool

[Surfpool](https://surfpool.run/) هي أداة للتطوير المحلي وبديل محسّن لـ
`solana-test-validator`. تعرف على المزيد حول ميزات Surfpool في
[وثائق Surfpool](https://docs.surfpool.run/).

1. قم بتثبيت Surfpool باستخدام الأمر التالي:

```terminal
$ curl -sL https://run.surfpool.run/ | bash
```

2. تحقق من نجاح التثبيت من خلال التحقق من إصدار واجهة سطر أوامر Surfpool:

```terminal
$ surfpool --version
```

سترى مخرجات مشابهة لما يلي:

```
surfpool 0.12.0
```
