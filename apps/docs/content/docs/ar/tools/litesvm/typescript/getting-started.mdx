# LiteSVM لـ TypeScript

مجموعات اختبار سولانا التي تُشغّل `solana-test-validator` تستغرق ثوانٍ لكل
اختبار — أنت تنتظر بدء تشغيل العملية، واتصالات RPC، والرحلات ذهاباً وإياباً عبر
الشبكة. يُشغّل LiteSVM نفس الاختبارات في ميلي ثانية. لا عملية خارجية، لا شبكة،
كل شيء في الذاكرة داخل مشغّل الاختبار الخاص بك.

<Callout type="info">
  **تم الاختبار ضد:** `@solana/kit@6.8.0`، `@solana/kit-plugin-litesvm@0.10.0`،
  `@solana/kit-plugin-signer@0.10.0`، `@solana-program/system@0.12.0`،
  `@solana-program/token@0.13.0`. هذه الحزم الإضافية جديدة وتتطور بسرعة — إذا
  واجهت تعارضاً، تحقق من
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  للإصدارات الدقيقة التي تعمل عليها المستندات ومجموعة الاختبار.
</Callout>

تقوم بتجميع عميل Kit من `@solana/kit` عن طريق تثبيت الإضافات باستخدام `.use()`:
إضافة للتوقيع
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer))،
وإضافة LiteSVM
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm))،
وأي إضافات برامج تحتاجها. تُتيح النتيجة `client.sendTransaction()`،
و`client.rpc`، والوصول المباشر إلى حالة الحساب والساعة وsysvars — كل ذلك dون
مغادرة عمليتك.

### إنشاء عميلك

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` تُعيد غلافاً فارغاً — كل `.use(plugin)` يُضيف إمكانيات إليه.
  يجب أن تأتي إضافة `signer()` **قبل** `litesvm()` لأن إضافة النقل تحتاج إلى
  دافع للارتباط به. `signer(mySigner)` هو اختصار لتثبيت نفس keypair بوصفه
  `payer` و`identity` معاً — استخدم `payer()` أو `identity()` من نفس الحزمة إذا
  أردت فصل تلك الأدوار. هذا الترتيب هو نفسه لـ `solanaRpc()` عند استخدام شبكة
  حقيقية.
</Callout>

<Callout type="info">
  تفضّل إعداداً بسطر واحد للتجارب السريعة؟ تتضمن حزمة `litesvm()` إسقاط رصيد
  تلقائي، لذا يمكنك تسلسل
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` من
  `@solana/kit-plugin-signer` لتوليد دافع وتمويله تلقائياً.
</Callout>

</Step>

</Steps>

## فهم العميل

بعد `.use(signer(...)).use(litesvm())`، يكشف العميل عن:

| الخاصية                     | الوصف                                                                                                                                                                                                      |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | `TransactionSigner` الذي قمت بتثبيته مع إضافة `signer()` (مُعيَّن أيضاً بوصفه `client.identity`)                                                                                                           |
| `client.identity`           | موقِّع المحفظة الذي يمتلك الحسابات والرموز المميزة والصلاحيات                                                                                                                                              |
| `client.svm`                | نسخة مباشرة من [`LiteSVM`](https://www.npmjs.com/package/litesvm) لإدارة الحسابات والمعاملات والإعدادات                                                                                                    |
| `client.rpc`                | واجهة RPC متوافقة مع Kit (مجموعة فرعية: `getAccountInfo`، `getBalance`، `getEpochSchedule`، `getLatestBlockhash`، `getMinimumBalanceForRentExemption`، `getMultipleAccounts`، `getSlot`، `requestAirdrop`) |
| `client.sendTransaction(s)` | تخطيط التعليمات وتوقيعها وإرسالها كمعاملة                                                                                                                                                                  |
| `client.airdrop`            | طلب SOL من صنبور LiteSVM                                                                                                                                                                                   |
| `client.getMinimumBalance`  | احتساب الحد الأدنى من lamport لإعفاء rent                                                                                                                                                                  |

### استخدام client.svm

وصول مباشر إلى LiteSVM من أجل:

- إدارة الحسابات (`setAccount`، `getAccount`، `airdrop`)
- تنفيذ المعاملات (`sendTransaction`، `simulateTransaction`)
- الإعدادات (`withSigverify`، `withBlockhashCheck`، وغيرها)
- معالجة الوقت (`warpToSlot`، `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### استخدام client.rpc

واجهة RPC متوافقة مع Kit للعمليات الاعتيادية:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  تدعم طبقة RPC **ترميز base64 فقط**. سيؤدي طلب ترميزات أخرى إلى إطلاق خطأ.
</Callout>

## الإعدادات

قم بضبط إعدادات LiteSVM باستخدام أساليب البناء على `client.svm`:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## إرسال المعاملات

أضف إضافات البرنامج بعد وحدة النقل — فهي تُعلِّق منشئي التعليمات على العميل حتى
لا تضطر إلى التعامل مع `TransactionMessage` الخام:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## التدفق من SVM إلى RPC

يمكنك تعيين الحالة عبر SVM وقراءتها عبر RPC:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```

## ما التالي

يغطي هذا القسم العميل الأساسي — إنشاء الحسابات وإرسال التعليمات وقراءة الحالة.
إذا كانت اختباراتك تتضمن رموز SPL المميزة (المضارب، وATAs، والتحويلات)، فإن
[دليل اختبار الرموز المميزة](/docs/tools/litesvm/typescript/tokens) يوضح كيف
تُلخِّص إضافة `tokenProgram()` ما يزيد على 40 سطراً من الإعداد في 3 استدعاءات
فقط. للاطلاع على مرجع API الكامل، راجع
[توثيق API](/docs/tools/litesvm/typescript/api-reference).
