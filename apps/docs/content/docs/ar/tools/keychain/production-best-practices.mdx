يُجرّد Keychain خلفية التوقيع، غير أن تشغيل الموقّع في بيئة الإنتاج يبقى
مسؤوليتك. تنطبق هذه الممارسات على جميع الخلفيات؛ راجع
[اختيار خلفية](/docs/tools/keychain/choosing-a-backend) لتحديد الأنسب لك.

## فصل الموقّعين الساخن والبارد

استخدم موقّعاً تشغيلياً برصيد منخفض ("ساخن") للمعاملات المتكررة والآلية،
وموقّعاً منفصلاً بقيمة عالية ("بارد") لتحركات الخزينة. ادعم الموقّع البارد
بحضانة MPC أو HSM سحابي، واشترط سياسات موافقة قبل توقيعه. ونظراً لأن الواجهة
متطابقة، يمكن لكل دور استخدام خلفية مختلفة دون تغيير نقاط الاستدعاء.

## تدوير المفاتيح بشكل مدروس

في سولانا، يُمثّل مفتاح التوقيع _عنوان الحساب_ ذاته، لذا فإن تدوير المفتاح يُنشئ
عنواناً جديداً. خطّط للتدوير كعملية ترحيل: أنشئ مفتاحاً جديداً، وأعد تعيين
الصلاحيات المعنية (أو انقل الأموال) إلى العنوان الجديد، ثم أوقف العمل بالمفتاح
القديم. استخدم مفاتيح مستقلة لكل بيئة حتى لا تُعاد استخدام مفاتيح devnet في
mainnet.

## منح أقل الصلاحيات اللازمة

قيّد بيانات اعتماد كل خلفية بالحد الأدنى اللازم للتوقيع. بالنسبة لـ AWS KMS،
يتمثّل ذلك في `kms:Sign` و `kms:DescribeKey` على المفتاح المحدد؛ وفضّل أدوار IAM
على بيانات الاعتماد الثابتة. طبّق المبدأ ذاته على سياسات Vault ومفاتيح API
للمحافظ المُدارة.

## التخطيط للتوافر

تعتمد الخلفيات البعيدة على خدمة شبكية. استدعِ `isAvailable()` قبل الاعتماد على
الموقّع، وطبّق إعادة المحاولة مع التراجع التدريجي عند الإخفاقات العابرة، وحدّد
مساراً احتياطياً للتعامل مع حالات التوقف. يفرض Keychain استخدام HTTPS للخلفيات
البعيدة ويطبّق مهلة افتراضية للطلبات.

## حماية الأسرار والنقل

- تتواصل جميع الواجهات الخلفية البعيدة عبر HTTPS؛ ويتم رفض أي نقاط نهاية لا
  تستخدم HTTPS.
- قم بتوفير مفاتيح API والرموز المميزة وبيانات الاعتماد من مدير الأسرار أو
  متغيرات البيئة — ولا تقم أبدًا بتضمينها في الكود المصدري أو إيداعها في نظام
  التحكم بالإصدارات.
- يقوم Keychain بتعقيم استجابات الأخطاء لتجنب تسرب الأسرار إلى السجلات؛ احرص على
  إبقاء سجلاتك الخاصة خالية من بيانات الاعتماد الخام ومواد المفاتيح.

## مراقبة عمليات التوقيع وتدقيقها

تتبّع معدل نجاح التوقيع والكمون وعدد حالات الفشل، وقم بإعداد تنبيهات عند
ارتفاعها — راجع المقاييس في
[جاهزية الإنتاج](/docs/payments/production-readiness#monitoring). قم بتمكين
سجلات التدقيق على مستوى المزود (KMS أو Vault أو الحارس) وقم بمطابقتها مع
المعاملات التي يُرسلها تطبيقك.
