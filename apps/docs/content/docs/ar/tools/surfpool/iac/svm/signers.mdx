تتوفر هذه الموقّعات عند استخدام إضافة SVM لتوقيع المعاملات.

## secret_key

يمكن استخدام الموقّع `svm::secret_key` لتوقيع معاملة بشكل متزامن باستخدام مفتاح
سري أو عبارة استذكارية أو ملف keypair.

### المدخلات

| الاسم             | مطلوب   | النوع | الوصف                                              |
| ----------------- | ------- | ----- | -------------------------------------------------- |
| `secret_key`      | اختياري | نص    | المفتاح السري المستخدم لتوقيع الرسائل والمعاملات   |
| `mnemonic`        | اختياري | نص    | العبارة الاستذكارية المستخدمة لتوليد المفتاح السري |
| `derivation_path` | اختياري | نص    | مسار الاشتقاق المستخدم لتوليد المفتاح السري        |
| `keypair_json`    | اختياري | نص    | مسار إلى ملف keypair.json يحتوي على المفتاح السري  |

### المخرجات

| الاسم        | النوع | الوصف                                |
| ------------ | ----- | ------------------------------------ |
| `public_key` | نص    | المفتاح العام للحساب                 |
| `address`    | نص    | عنوان SVM (اسم مستعار لـ public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

يمكنك أيضاً استخدام عبارة استذكارية:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

أو ملف keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

سيتيح الموقّع `svm::web_wallet` لمشغّل Runbook توقيع المعاملة باستخدام موقّع
المتصفح الذي يختاره.

### المدخلات

| الاسم              | مطلوب   | النوع | الوصف                                              |
| ------------------ | ------- | ----- | -------------------------------------------------- |
| `expected_address` | اختياري | نص    | عنوان SVM المتوقع للاتصال (احذفه للسماح بأي عنوان) |

### المخرجات

| الاسم        | النوع | الوصف                       |
| ------------ | ----- | --------------------------- |
| `address`    | نص    | عنوان الحساب المتصل         |
| `public_key` | نص    | المفتاح العام للحساب المتصل |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

سيطلب هذا الموقّع من المستخدم في واجهة المستخدم على الويب ما يلي:

1. ربط محفظته
2. تقديم pubkey عبر توقيع الرسالة
3. توقيع جميع المعاملات التي تتطلب هذا الموقّع

## squads

يمكن استخدام الموقّع `svm::squads` للتوقيع على المعاملات باستخدام خزينة Squads
متعدد التوقيعات.

### المدخلات

| الاسم     | مطلوب   | النوع      | الوصف              |
| --------- | ------- | ---------- | ------------------ |
| `address` | اختياري | سلسلة نصية | عنوان خزينة الفريق |

### المخرجات

| الاسم        | النوع      | الوصف                                |
| ------------ | ---------- | ------------------------------------ |
| `public_key` | سلسلة نصية | pubkey لخزينة الفريق                 |
| `address`    | سلسلة نصية | عنوان SVM (اسم مستعار لـ public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

يتيح استخدام Squads متعدد التوقيعات إجراء توقيع آمن على المعاملات من قِبَل أطراف
متعددة دون الكشف عن المفاتيح الخاصة الفردية.
