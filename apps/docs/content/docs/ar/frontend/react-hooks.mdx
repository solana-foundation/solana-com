`@solana/react` ينشر عميل Kit إلى شجرة المكوّنات من خلال موفّر واحد، ثم تقرأ
الخطافات بيانات RPC والاشتراكات وحالة المحفظة منه. تقع خطافات محفظة المتصفح في
`@solana/kit-plugin-wallet/react` وتقرأ من الموفّر نفسه — لا يوجد موفّر محفظة
منفصل.

## التثبيت

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## بناء العميل

ابنِ العميل مرةً واحدة، على مستوى الوحدة، خارج شجرة React. في المتصفح، يجعل
`walletSigner` المحفظة المتصلة هي دافع الرسوم والهوية في آنٍ واحد.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

يضيف `walletSigner` الإضافة `client.wallet` (اكتشاف Wallet Standard والاتصال
به)، ويضيف `solanaRpc` الإضافة `client.rpc` إلى جانب المساعد
`client.sendTransaction`. صدِّر نوع العميل مُغلَّفاً في `Awaited<>` حتى يظل
صحيحاً إذا أضفت لاحقاً إضافةً تُحلَّل بشكل غير متزامن، ومرّر ذلك `AppClient` إلى
كل استدعاء `useClient`.

## لفّ الشجرة مرةً واحدة

`ClientProvider` ينشر العميل. كل خطاف أدناه يقرأ منه.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## خطافات المحفظة

تقرأ خطافات المحفظة قدرة `wallet` الخاصة بالعميل، لذا تعمل فقط داخل
`ClientProvider` الذي ثبّت فيه العميلُ إضافةَ محفظة.

- `useWallets()` — محافظ Wallet Standard المكتشفة لسلسلة العميل.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` أو `null`. القيمة
  `signer` هي `null` للمحافظ للقراءة فقط.
- `useConnect()` / `useDisconnect()` — تُعيد `ActionResult` يُنفّذ `dispatch`
  منها (و `dispatchAsync`) الإجراءَ، فيما تتتبع `isRunning` و `status` و `data`
  و `error` و `reset` نتيجته.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## إرسال معاملة

يُعيد `useClient` العميلَ ذاته الذي نشرته. ابنِ التعليمات باستخدام عميل برنامج
مُولَّد وأرسلها عبر `client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

تمرير نوع `AppClient` المُصدَّر إلى `useClient` يمنح الخطاف العميلَ الكامل
الموسَّع بالإضافات، بحيث يكون `client.rpc` و `client.sendTransaction` مكتوبَي
النوع.

## قراءة البيانات

يوفر `@solana/react` خطاطيف تقرأ البيانات عبر العميل:

- `useRequest(source)` — قراءة RPC أحادية؛ يُعيد
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — اشتراك مباشر؛ يُعيد
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — قيمة مُغذَّاة من RPC تُحافظ على حداثتها عبر اشتراك.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

قم بحفظ المصدر في ذاكرة التخزين المؤقت (باستخدام `useMemo`، أو `useCallback`
للدوال) حتى لا يُعيد الخطاف جلب البيانات إلا عند تغيُّر مدخلاته، لا عند كل عملية
تصيير.

للتخزين المؤقت وإعادة التحقق، يغلِّف المسارا `@solana/react/swr` و
`@solana/react/query` الخطاطيفَ ذاتها لـ [SWR](https://swr.vercel.app) و
[TanStack Query](https://tanstack.com/query).

## الأنماط الشائعة لمطوري سولانا

- **مزوِّد واحد**: `ClientProvider` هو المزوِّد الوحيد الذي تحتاجه؛ تقرأ خطاطيف
  المحفظة منه عبر قدرة `wallet` الخاصة بالعميل.
- **دعم مكونات الخادم**: اكتفِ بتمييز مكونات الأوراق التي تستدعي الخطاطيف بـ
  `"use client"`؛ يمكن لعمليات القراءة على الخادم استخدام عميل Kit RPC عادي بدون
  محفظة.
- **الاختبار**: انشر عميلاً وهمياً عبر `ClientProvider` لمحاكاة المحافظ
  واستجابات RPC.

<Callout type="warn" title="عائلتا خطاطيف بنفس الأسماء">
  يتواجد `useSignIn` و `useSignMessage` في كلٍّ من `@solana/react` (حيث يأخذان
  وسيطة `UiWalletAccount`) و `@solana/kit-plugin-wallet/react` (يعتمد على
  العميل، بدون وسيطة حساب، ويُعيد `ActionResult`). استخدم إصدارات
  `@solana/kit-plugin-wallet/react` مع نمط العميل الموضَّح أعلاه؛ فخطاطيف
  المحفظة القديمة في `@solana/react` في طور الاستبدال.
</Callout>

اقرن هذا الدليل مع [نظرة عامة على عميل Kit](/docs/frontend/client) لفهم العميل
الذي يقرأ منه كل خطاف.
