Opas paikallisen Solana-kehitysympäristön määrittämiseen. Opi asentamaan Rust,
Solana CLI ja Anchor Framework Windowsiin (WSL), Linuxiin ja Maciin. Käytä tätä
opasta, jos haluat asentaa jokaisen riippuvuuden erikseen tai jos
[pika-asennus](/docs/intro/installation/) epäonnistuu jostain syystä.

### Edellytykset

<Accordions>
<Accordion title="Windows">

Solana-ohjelmien kehittämiseen Windowsissa **on käytettävä**
[Windows Subsystem for Linuxia](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). WSL:n asennuksen jälkeen kaikki muut riippuvuudet asennetaan
Linux-terminaalin kautta.

Asenna WSL suorittamalla seuraava komento Windows PowerShellissä:

```terminal
$ wsl --install
```

Asennusprosessi kehottaa luomaan oletuskäyttäjätilin.

![WSL-asennus](/assets/docs/intro/installation/wsl-install.png)

Oletuksena WSL asentaa Ubuntun. Voit avata Linux-terminaalin etsimällä "Ubuntu"
hakupalkista.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Jos Ubuntu-terminaalisi näyttää alla olevan kuvan kaltaiselta, saatat kohdata
ongelman, jossa `ctrl + v` (liitä-pikanäppäin) ei toimi terminaalissa.

![Ubuntu-terminaali](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Jos kohtaat tämän ongelman, avaa Windows Terminal etsimällä "Terminal"
hakupalkista.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Sulje seuraavaksi Windows Terminal ja avaa Linux-terminaali uudelleen etsimällä
Ubuntu uudelleen. Terminaalin pitäisi nyt näyttää alla olevan kuvan kaltaiselta,
jossa `ctrl + v` (liitä-pikanäppäin) toimii.

![Ubuntu-terminaali](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Jos käytät VS Codea,
[WSL-laajennus](https://code.visualstudio.com/docs/remote/wsl-tutorial)
mahdollistaa WSL:n ja VS Coden käytön yhdessä.

![WSL-asennus VS Codessa](/assets/docs/intro/installation/wsl-vscode.png)

Näet seuraavan VS Coden tilapalkissa:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Kun WSL on asennettu, olet valmis joko:

- Suorittamaan [pika-asennuksen](/docs/intro/installation) Ubuntu (Linux)
  -päätteellä
- Asentamaan [Rustin](#install-rust), [Solana CLI:n](#install-solana-cli) ja
  [Anchor CLI:n](#install-anchor-cli) erikseen, kuten alla on kuvattu

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Jos käytät Debiania tai Debianiin pohjautuvia vaihtoehtoja kuten Ubuntua tai Mintiä, seuraa alla olevia ohjeita:

1. Päivitä pakettiluettelot varmistaaksesi, että kaikki paketit ovat ajan
   tasalla:

```terminal
$ sudo apt-get update
```

2. Asenna seuraavat riippuvuudet:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Jos kohtaat seuraavan virheen asennettaessa `protobuf-compiler`, varmista että suoritat ensin _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    RHEL-johdannaisille kuten Fedora tai Red Hat Linux, seuraa seuraavia ohjeita.

Päivitä pakettiluettelot saadaksesi uudemmat versiot paketeista suorittamalla
komento.

```terminal
$ sudo dnf upgrade --refresh
```

Seuraavaksi asenna seuraavat riippuvuudet:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Asenna Rust

Kehittäjät rakentavat Solana-ohjelmia käyttäen
[Rust](https://www.rust-lang.org/) ohjelmointikieltä.

1. Asenna Rust käyttämällä [rustup](https://www.rust-lang.org/tools/install)
   -työkalua syöttämällä seuraava komento päätteessäsi:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Onnistuneen asennuksen jälkeen näet seuraavan viestin:

<Accordions>
<Accordion title="Onnistuneen Rust-asennuksen viesti">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Lataa PATH-ympäristömuuttujasi uudelleen sisältämään
   [Cargon](https://github.com/rust-lang/cargo) bin-hakemisto:

```terminal
$ . "$HOME/.cargo/env"
```

3. Varmista, että asennus onnistui.

```terminal
$ rustc --version
```

Näet seuraavanlaisen tulosteen:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Asenna Solana CLI

Solana CLI tarjoaa kaikki työkalut, joita tarvitaan Solana-ohjelmien
rakentamiseen ja käyttöönottoon.

1. Asenna Solana CLI -työkalukokoelma käyttämällä virallista asennuskomentoa:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Voit korvata `stable` julkaisutunnuksella, joka vastaa haluamasi julkaisun
  ohjelmistoversiota (esim. `v2.0.3`), tai käyttää yhtä kolmesta symbolisesta
  kanavan nimestä: `stable`, `beta` tai `edge`.
</Callout>

2. Lisää PATH-ympäristömuuttuja

Solana CLI:n ensimmäinen asennus saattaa kehottaa sinua lisäämään `PATH`
-ympäristömuuttujan. Tee tämä sulkemalla ja avaamalla terminaalisi uudelleen tai
suorittamalla seuraava komento shellissäsi:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Päivitä PATH-ympäristömuuttujasi

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Jos käytät **Linuxia** tai **WSL**:ää, sinun täytyy lisätä Solana CLI -binääri
PATH-muuttujaasi, jotta komento on käytettävissä päätteessäsi. Tee tämä
noudattamalla alla olevia ohjeita:

**a.** Tarkista mitä komentotulkkia käytät:

```terminal
$ echo $SHELL
```

- Jos tuloste sisältää `/bash`, käytä `.bashrc`.
- Jos tuloste sisältää `/zsh`, käytä `.zshrc`.

**b.** Suorita asianmukainen komento shellisi perusteella.

Bashille (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Zshille (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Jos käytät Macia `zsh` -prosessorilla, asennuksesta kirjatun oletusarvoisen
`export PATH` -komennon suorittaminen ei säily terminaalin sulkemisen jälkeen.
Lisää sen sijaan PATH shell-konfiguraatiotiedostoosi suorittamalla seuraava
komento:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Yllä olevat vaiheet eivät koske `bash` -käyttäjiä, sillä muutokset
  PATH-muuttujaan säilyvät oletuksena.
</Callout>

</Tab>
</Tabs>

4. Käynnistä terminaali uudelleen tai suorita seuraava komento päivittääksesi
   terminaali-istunnon:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Varmista asennuksen onnistuminen tarkistamalla Solana CLI -versio:

```terminal
$ solana --version
```

Näet seuraavanlaisen tulosteen:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Voit tarkastella kaikkia saatavilla olevia versioita
[Agave Github -repossa](https://github.com/anza-xyz/agave/releases). Agave on
validator-asiakasohjelma [Anzalta](https://www.anza.xyz/), joka tunnettiin
aiemmin nimellä Solana Labs validator -asiakasohjelma.

</Callout>

5. Päivitä Solana CLI uusimpaan versioon tarvittaessa _(Valinnainen)_

```terminal
$ agave-install update
```

</Step>

### Asenna Anchor CLI

[Anchor](https://www.anchor-lang.com/) on kehys Solana-ohjelmien kehittämiseen.
Anchor-kehys hyödyntää Rust-makroja yksinkertaistaakseen Solana-ohjelmien
kirjoitusprosessia.

#### Edellytykset

Oletusarvoinen Anchor-projektin testitiedosto (TypeScript), joka luodaan
_shell`anchor init`_ -komennolla, vaatii Node.js:n ja Yarnin.
(Rust-testitemplatti on saatavilla käyttämällä
_shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Node-asennus">

1. Asenna Node käyttämällä
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) suorittamalla
   seuraava komento:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Käynnistä terminaali uudelleen ja varmista, että `nvm` -komento toimii
   onnistuneesti:

```terminal
$ command -v nvm
```

3. Käytä `nvm` -komentoa Noden asentamiseen:

```terminal
$ nvm install node
```

4. Varmista, että asennus onnistui tarkistamalla Node-versio:

```terminal
$ node --version
```

Näet seuraavanlaisen tulosteen:

```
v23.7.0
```

</Accordion>
<Accordion title="Yarn-asennus">

1. Asenna Yarn:

```terminal
$ npm install --global yarn
```

2. Varmista, että asennus onnistui tarkistamalla Yarn-versio:

```terminal
$ yarn --version
```

Näet seuraavan tulosteen:

```
1.22.1
```

</Accordion>
</Accordions>

#### Asennus

Voit asentaa Anchor CLI:n ja työkalut kahdella tavalla:

- Anchor Version Manager (AVM) — Suositeltu asennustapa
- Ilman AVM:ää — Asenna suoraan GitHubista

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Anchor Version Manager (AVM) mahdollistaa eri Anchor-versioiden asentamisen ja
hallinnan järjestelmässäsi sekä helpottaa Anchor-versioiden päivittämistä
tulevaisuudessa. Asentaaksesi Anchorin AVM:n avulla, seuraa alla olevia ohjeita:

1. Asenna AVM seuraavalla komennolla:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Varmista, että AVM asentui onnistuneesti:

```terminal
$ avm --version
```

3. Asenna Anchor CLI käyttäen AVM:ää:

Asentaaksesi uusimman version:

```terminal
$ avm install latest
$ avm use latest
```

Asentaaksesi tietyn version, määritä versionumero:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Kun asennat Anchor CLI:n Linuxissa tai WSL:ssä, saatat kohdata tämän virheen:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Jos näet tämän virheilmoituksen, seuraa näitä ohjeita:

1. Asenna riippuvuudet, jotka on listattu
   [Linux](/docs/intro/installation#install-dependencies) -osiossa tämän sivun
   alussa.
2. Yritä asentaa Anchor CLI uudelleen.

</Callout>

4. Varmista, että asennus onnistui tarkistamalla Anchor CLI:n versio:

```terminal
$ anchor --version
```

Näet seuraavanlaisen tulosteen:

```
anchor-cli 0.31.1
```

<Callout type="info">

Muista suorittaa _shell`avm use`_ -komento määrittääksesi Anchor CLI:n version,
jota järjestelmässäsi käytetään.

- Jos asensit `latest` -version, suorita _shell`avm use latest`_.
- Jos asensit version `0.30.1`, suorita _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Asenna tietty versio Anchor CLI:stä seuraavalla komennolla:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Varmista, että asennus onnistui tarkistamalla Anchor CLI:n versio:

```terminal
$ anchor --version
```

Näet seuraavanlaisen tulosteen:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>

### Asenna Surfpool CLI

[Surfpool](https://surfpool.run/) on työkalu paikalliseen kehitykseen ja
parannettu korvike `solana-test-validator`:lle. Lue lisää Surfpoolin
ominaisuuksista [Surfpoolin dokumentaatiosta](https://docs.surfpool.run/).

1. Asenna Surfpool seuraavalla komennolla:

```terminal
$ curl -sL https://run.surfpool.run/ | bash
```

2. Varmista, että asennus onnistui tarkistamalla Surfpool CLI:n versio:

```terminal
$ surfpool --version
```

Näet seuraavanlaisen tulosteen:

```
surfpool 0.12.0
```
