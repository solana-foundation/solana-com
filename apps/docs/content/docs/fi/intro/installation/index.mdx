Pika-asennus paikallisen Solana-kehitysympäristön määrittämiseen yhdellä
komennolla. Asenna Rust, Solana CLI ja Anchor Framework Windowsiin (WSL),
Linuxiin ja Maciin. (Halutessasi voit myös
[asentaa jokaisen riippuvuuden erikseen](/docs/intro/installation/dependencies).)

## Edellytykset

<Accordions>
<Accordion title="Windows">

Solana-ohjelmien kehittämiseen Windowsissa **on käytettävä**
[Windows Subsystem for Linuxia](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). WSL:n asennuksen jälkeen kaikki muut riippuvuudet asennetaan
Linux-päätteen kautta.

Asenna WSL suorittamalla seuraava komento Windows PowerShellissä:

```terminal
$ wsl --install
```

Asennusprosessi kehottaa luomaan oletuskäyttäjätilin.

![WSL-asennus](/assets/docs/intro/installation/wsl-install.png)

Oletuksena WSL asentaa Ubuntun. Voit avata Linux-päätteen etsimällä "Ubuntu"
hakupalkista.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Jos Ubuntu-päätteesi näyttää alla olevan kuvan kaltaiselta, saatat kohdata
ongelman, jossa `ctrl + v` (liitä-pikanäppäin) ei toimi päätteessä.

![Ubuntu-pääte](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Jos kohtaat tämän ongelman, avaa Windows Terminal etsimällä "Terminal"
hakupalkista.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Sulje seuraavaksi Windows Terminal ja avaa Linux-pääte uudelleen etsimällä
Ubuntu uudelleen. Päätteen pitäisi nyt näyttää alla olevan kuvan kaltaiselta,
jossa `ctrl + v` (liitä-pikanäppäin) toimii.

![Ubuntu-pääte](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Jos käytät VS Codea,
[WSL-laajennus](https://code.visualstudio.com/docs/remote/wsl-tutorial)
mahdollistaa WSL:n ja VS Coden käytön yhdessä.

![WSL-asetukset VS Codessa](/assets/docs/intro/installation/wsl-vscode.png)

Näet seuraavan VS Coden tilapalkissa:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Kun WSL on määritetty, voit joko:

- Suorittaa [pika-asennuksen](#installation) käyttämällä Ubuntu (Linux)
  -terminaalia
- Asentaa [Rustin](/docs/intro/installation/dependencies#install-rust),
  [Solana CLI:n](/docs/intro/installation/dependencies#install-solana-cli) ja
  [Anchor CLI:n](/docs/intro/installation/dependencies#install-anchor-cli)
  erikseen, kuten alla kuvataan

</Accordion>

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Jos käytät Debiania tai Debian-pohjaisia vaihtoehtoja kuten Ubuntua tai Minttiä, seuraa alla olevia ohjeita:

1. Päivitä pakettiluettelot varmistaaksesi, että kaikki paketit ovat ajan
   tasalla:

```terminal
$ sudo apt-get update
```

2. Asenna seuraavat riippuvuudet:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Jos kohtaat seuraavan virheen asennettaessa `protobuf-compiler`, varmista
ensin, että suoritat _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    RHEL-johdannaisille kuten Fedoralle tai Red Hat Linuxille, seuraa seuraavia ohjeita.

Päivitä pakettiluettelot saadaksesi uudemmat versiot paketeista suorittamalla
komento.

```terminal
$ sudo dnf upgrade --refresh
```

Seuraavaksi asenna seuraavat riippuvuudet:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## Asennus

1. Syötä seuraava komento terminaaliisi asentaaksesi kaikki tarvittavat
   riippuvuudet:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Onnistunut asennus palauttaa seuraavanlaisen tulosteen:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. Varmista onnistunut asennus tarkistamalla kunkin asennetun riippuvuuden
   versio.

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  Jos pika-asennuskomento epäonnistuu, katso alla olevasta [Asenna
  riippuvuudet](/docs/intro/installation/dependencies) -osiosta ohjeet kunkin
  riippuvuuden asentamiseen erikseen.
</Callout>
