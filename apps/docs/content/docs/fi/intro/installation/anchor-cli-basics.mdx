Tämä osio tarjoaa joitakin yleisiä komentoja ja esimerkkejä, jotka auttavat
sinua alkuun [Anchor CLI:n](https://www.anchor-lang.com/docs) käytössä.

<Steps>
<Step>

### Alusta projekti

Luo uusi Anchor-projekti suorittamalla alla näkyvä komento. Se luo uuden
hakemiston projektin nimellä ja käyttää sitä uuden Anchor-projektin
alustamiseen.

```terminal
$ anchor init <project-name>
```

Esimerkiksi alla oleva komento luo projektin nimeltä `my-project`.
`my-project`-hakemisto sisältää perustason Rust-ohjelman ja
TypeScript-testimallin.

```terminal
$ anchor init my-project
```

Siirry sitten projektihakemistoon:

```terminal
$ cd <project-name>
```

Katso Anchorin
[projektitiedostorakenne](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure).

</Step>
<Step>

### Rakenna ohjelma

Rakentaaksesi projektisi, suorita seuraava komento:

```terminal
$ anchor build
```

Löydät käännetyn ohjelman `/target/deploy`-hakemistosta.

<Callout type="warn">

Kun suoritat `anchor build` -komentoa, jos kohtaat seuraavat virheet:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Kokeile näitä ratkaisuja:

1. Pakota asennus käyttämällä seuraavaa komentoa:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Jos edellinen ratkaisu ei toimi, tyhjennä Solanan välimuisti:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Voit korjata tämän muuttamalla `Cargo.lock`-tiedoston version kentän arvosta 4 arvoon 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Katso [tämä issue](https://github.com/solana-foundation/anchor/issues/3392)
lisätietoja varten.

</Accordion>

</Accordions>

Edellisen ratkaisun soveltamisen jälkeen yritä suorittaa _shell`anchor build`_
uudelleen.

</Callout>

</Step>
<Step>

### Julkaise ohjelma

Julkaistaksesi projektisi, suorita seuraava komento:

```terminal
$ anchor deploy
```

Tämä komento julkaisee ohjelmasi
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml)-tiedostossa
määriteltyyn `cluster`-klusteriin.

</Step>
<Step>

### Testaa ohjelma

Testataksesi projektisi, suorita seuraava komento:

```terminal
$ anchor test
```

Tämä komento kääntää, julkaisee ja suorittaa projektisi testit.

<Callout>
  Kun käytät `localnet`-klusteria, Anchor käynnistää automaattisesti paikallisen
  validatorin, julkaisee ohjelman, suorittaa testit ja pysäyttää sitten
  validatorin.
</Callout>

<Callout type="warn">

Jompikumpi seuraavista virheistä voi viitata siihen, että sinulla ei ole
Node.js:ää tai Yarnia asennettuna:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
