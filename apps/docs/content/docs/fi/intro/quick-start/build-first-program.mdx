Tämä pikakäynnistysopas käyttää
[Anchor-sovelluskehyksen](https://www.anchor-lang.com/docs) aloitusprojektia,
jonka `anchor init` luo. Luot projektin paikallisesti, suoritat sen testit,
rakennat ohjelman ja käyt läpi ohjelmakoodin, jonka Anchor generoi.

## Edellytykset

Ennen kuin aloitat, [asenna Solana-kehitystyökalut](/docs/intro/installation).
Asennus sisältää Rustin, Solana CLI:n ja Anchor CLI:n.

Käytä tähän malliin Anchor CLI -versiota 1.1.2 tai uudempaa. Tarkista asennettu
versio:

```terminal
$ anchor --version
```

<Steps>

<Step>

### Luo projekti

Suorita seuraavat komennot terminaalissasi:

```terminal
$ anchor init my-program
$ cd my-program
```

Aloitusprojekti sisältää yhden Solana-ohjelman hakemistossa
`programs/my-program`. Ohjelma sisältää kaksi käskyä: yhden laskuritilin
alustamiseen ja yhden laskurin kasvattamiseen.

Osa mallin osioista havainnollistaa yleisiä Solana-ohjelmointimalleja:
[PDA](/docs/core/pda)-tilin osoitteiden johtaminen, Cross Program Invocation
([CPI](/docs/core/cpi)) SOL:n siirtämiseksi sekä mukautettujen
virheentarkistusten käyttö käskyn pysäyttämiseksi, kun ehto ei täyty.

<Step>

### Rakenna ohjelma

Suorita `anchor build` kääntääksesi aloitusohjelman:

```terminal
$ anchor build
```

Käännetty ohjelma kirjoitetaan kohteeseen `target/deploy/my_program.so`. Kun
ohjelma otetaan käyttöön, tämän `.so`-tiedoston sisältö tallennetaan ketjussa
olevalle tilille.

</Step>

<Step>

### Suorita testi

Suorita oletustesti:

```terminal
$ anchor test
```

Tämän mallin `Anchor.toml` käyttää Rust-testikomentoa:

```toml title="Anchor.toml"
skip_local_validator = true

[scripts]
test = "cargo test"
```

Testi lataa käännetyn ohjelman [LiteSVM](https://www.litesvm.com/)-ympäristöön,
luo maksajan, lähettää _rs`initialize`_- ja _rs`increment`_-käskyt, jonka
jälkeen tarkistaa laskuritilin tilan.

`anchor test`-komennon suorittaminen kääntää myös ohjelman, joten sinun ei
tarvitse ajaa `anchor build`-komentoa ensin paikallista testausta varten.

</Step>

<Step>

### Ota ohjelma käyttöön

Paikalliset testit tarjoavat nopeimman palautesilmukan. Kun olet valmis ottamaan
käyttöön verkossa, esimerkiksi devnetissä, käännä ensin ja ota sitten käyttöön
klusterissa.

Solana-ohjelman käyttöönottovaatimuksena on SOL, koska ohjelma tallennetaan
program account -tilille, jonka täytyy maksaa käyttämästään tilasta. Devnetissä
voit pyytää ilmaista devnet-SOL:ia
[Solana Faucetista](https://faucet.solana.com/) tai Solana CLI:llä:

```terminal
$ solana airdrop 2 --url devnet
```

```terminal
$ anchor build
$ anchor deploy --provider.cluster devnet
```

</Step>

</Steps>

## Lähdetiedostot

`src`-hakemisto sisältää Solana-ohjelman. Anchor'in
[Ohjelmarakenne](https://www.anchor-lang.com/docs/basics/program-structure)
-dokumentaatio selittää tässä käytetyt keskeiset makrot, mukaan lukien
_rs`declare_id!`_, _rs`#[program]`_, _rs`#[derive(Accounts)]`_ ja
_rs`#[account]`_. Tässä osiossa käydään läpi mallin tiedostot.

### lib.rs

`lib.rs` on ohjelman sisääntulopiste. Se yhdistää lähdetiedostot, määrittelee
ohjelman osoitteen ja määrittelee ohjelmakomennot, joita käyttäjät voivat
kutsua.

<WithNotes>

```rust title="programs/my-program/src/lib.rs" -w
// !tooltip[/pub mod constants/] constantsMod
pub mod constants;
// !tooltip[/pub mod error/] errorMod
pub mod error;
// !tooltip[/pub mod instructions/] instructionsMod
pub mod instructions;
// !tooltip[/pub mod state/] stateMod
pub mod state;

// !tooltip[/anchor_lang::prelude/] prelude
use anchor_lang::prelude::*;

pub use constants::*;
pub use instructions::*;
pub use state::*;

// !tooltip[/declare_id!/] declareId
declare_id!("82sFkffP9wxwpyfZyeaKHH2chQoJPUGsJZSPi9mrUuXd");

// !tooltip[/#\[program\]/] programMacro
#[program]
pub mod my_program {
    use super::*;

    // !tooltip[/pub fn initialize/] publicInstructions
    pub fn initialize(ctx: Context<Initialize>) -> Result<()> {
        // !tooltip[/handle_initialize/] handlers
        crate::instructions::initialize::handle_initialize(ctx)
    }

    // !tooltip[/pub fn increment/] publicInstructions
    pub fn increment(ctx: Context<Increment>) -> Result<()> {
        // !tooltip[/handle_increment/] handlers
        crate::instructions::increment::handle_increment(ctx)
    }
}
```

### !constantsMod

_rs`pub mod constants`_ lisää `constants.rs` osaksi ohjelmacratea. Tänne malli
tallentaa jaetut ohjelma-arvot, kuten PDA seed, siirtosumma ja laskurin
enimmäisarvo.

### !errorMod

_rs`pub mod error`_ lisää `error.rs` osaksi ohjelmacratea. Tämä tiedosto
määrittelee mukautetut virheet, jotka ohjelma voi palauttaa, kun käsky on
virheellinen.

### !instructionsMod

_rs`pub mod instructions`_ lisää `instructions.rs` osaksi ohjelmacratea. Tämä
tiedosto toimii hakemistona käskykohtaisille tiedostoille
`src/instructions/`-kansiossa.

### !stateMod

_rs`pub mod state`_ lisää `state.rs` osaksi ohjelmacratea. Tämä tiedosto
määrittelee tietopohjan tämän ohjelman omistamille tileille, mukaan lukien
_rs`Counter`_ tilin.

### !prelude

_rs`anchor_lang::prelude::*`_ tuo Anchor-tyypit ja makrot, joita käytetään
Solana-ohjelman määrittelemiseen, mukaan lukien _rs`Context`_, _rs`Result`_,
_rs`Pubkey`_, _rs`Account`_, _rs`Signer`_ sekä Anchor-tilimääritteet.

### !reexports

_rs`pub use`_-rivit tekevät alimoduulien julkiset kohteet saataville craten
juuresta. Tämä pitää käskyjen sisääntulopisteet lyhyinä, kuten
_rs`Context<Initialize>`_ pitkän moduulipolun sijaan.

### !declareId

_rs`declare_id!`_ määrittelee ohjelman osoitteen. Sen tulee vastata osoitetta
`Anchor.toml`-tiedostossa, käyttöönotetulla program account -tilillä sekä
kaikilla käyttäjillä tai testeillä, jotka kutsuvat ohjelmaa.

### !programMacro

_rs`#[program]`_ käskee Anchoria generoimaan Solana-ohjelman sisääntulopisteen
ja käskyjen reitityksen. Tämän moduulin julkisista funktioista tulee käskyjä,
joita käyttäjät voivat kutsua.

### !publicInstructions

_rs`initialize`_ ja _rs`increment`_ ovat kaksi käskyä, joita käyttäjät voivat
kutsua. Kumpikin vastaanottaa _rs`Context<T>`_-arvon. _rs`Context`_-sisällä
oleva tyyppi määrittelee käskyn vaatimat tilit, ja Anchor validoi nämä tilit
ennen käskyn suoritusta.

### !handlers

Käsittelijäfunktio sisältää instruktion logiikan.

</WithNotes>

### constants.rs

`constants.rs` pitää jaetut arvot yhdessä paikassa. Tässä mallissa
_rs`COUNTER_SEED`_ johtaa laskurin PDA:n, _rs`HELLO_WORLD_LAMPORTS`_ siirretään
alustuksen aikana ja _rs`MAX_COUNT`_ tarkistetaan ennen kasvattamista.

<WithNotes>

```rust title="programs/my-program/src/constants.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[constant\]/] anchorConstant
// !tooltip[/COUNTER_SEED/] counterSeed
#[constant]
pub const COUNTER_SEED: &[u8] = b"counter";

// !tooltip[/HELLO_WORLD_LAMPORTS/] helloLamports
#[constant]
pub const HELLO_WORLD_LAMPORTS: u64 = 1;

// !tooltip[/MAX_COUNT/] maxCount
#[constant]
pub const MAX_COUNT: u64 = 10;
```

### !anchorConstant

_rs`#[constant]`_ käskee Anchoria sisällyttämään arvon ohjelman IDL:ään. Tämä
tekee tärkeistä ohjelmakonstanteista näkyviä generoitujen asiakkaiden ja
työkalujen käyttöön.

### !counterSeed

_rs`COUNTER_SEED`_ on tavumerkkijono, jota käytetään laskurin PDA:n johtamiseen.
Sekä ohjelma että testi käyttävät tätä seed-arvoa, joten ne ovat yksimielisiä
tilin osoitteesta ilman erillisen keypair-arvon tallentamista. Tämä malli
käyttää yhtä kiinteää seed-arvoa, joten ohjelmalla on yksi laskuriosoite. Oikeat
ohjelmat sisällyttävät usein käyttäjän osoitteen tai muun tunnisteen
seed-arvoihin luodakseen monia toisiinsa liittyviä tilejä.

### !helloLamports

_rs`HELLO_WORLD_LAMPORTS`_ on lamport-määrä, joka siirretään maksajalta
laskuritilille _rs`initialize`_-funktion sisällä. Se on tarkoituksellisesti
pieni, koska malli havainnollistaa CPI:tä System Programille.

### !maxCount

_rs`MAX_COUNT`_ on yläraja, jonka _rs`increment`_ tarkistaa. Sen pitäminen
täällä antaa ohjelmalle, testeille ja generoidulle IDL:lle yhteisen nimen
säännölle.

</WithNotes>

```rust title="initialize.rs" -w

    #[account(
        init,

    // ...

    // !mark

    Ok(())

```

### !right

`initialize.rs` käyttää kahta vakiota:

- [_rs`COUNTER_SEED`_](mention:counter-seed) johtaa laskurin PDA-osoitteen.
- [_rs`HELLO_WORLD_LAMPORTS`_](mention:hello-world-lamports) määrittää summan,
  joka siirretään maksajalta laskuritilille.

</SideBySide>

<SideBySide>

### !left

```rust title="increment.rs" -w
pub fn handle_increment(ctx: Context<Increment>) -> Result<()> {
    // ...

    require!(
        // !mark
        // !mention max-count

        ErrorCode::CounterOverflow,

    ctx.accounts.counter.count += 1;
```

### !right

`increment.rs` käyttää [_rs`MAX_COUNT`_](mention:max-count) laskurin ylänä
rajana. Jos nykyinen arvo on jo maksimissa, _rs`require!`_ palauttaa
_rs`CounterOverflow`_ eikä tilin tietoja muuteta.

</SideBySide>

### state.rs

`state.rs` määrittää mukautetut tietotyypit tileille, jotka ohjelma luo ja
omistaa. Ohjelma määrittää ohjeet tietojen luomiseen, alustamiseen ja
päivittämiseen, mutta laskuridata ei ole tallennettu itse ohjelmaan. Se on
tallennettu erilliselle tilille, jolla on oma osoitteensa.

<WithNotes>

```rust title="programs/my-program/src/state.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[account\]/] accountMacro
#[account]
// !tooltip[/InitSpace/] initSpace
#[derive(InitSpace)]
pub struct Counter {
    // !tooltip[/count/] count
    pub count: u64,
    // !tooltip[/authority/] authority
    pub authority: Pubkey,
}
```

### !accountMacro

_rs`#[account]`_ kertoo Anchorille, että tämä struct on dataa, joka voidaan
tallentaa tämän ohjelman omistamalle ketjussa olevalle tilille. Anchor käyttää
sitä sarjallistamiseen, purkamiseen ja _rs`Counter`_-tilien validointiin.

### !initSpace

_rs`InitSpace`_ antaa Anchorin laskea _rs`Counter::INIT_SPACE`_, eli tarvittavan
tavumäärän tämän structin kentille. 8-tavuinen Anchor-tilin erottelija lisätään
erikseen tilin luomisen yhteydessä.

### !count

_rs`count`_ on tilin dataan tallennettu laskuriarvo.

### !authority

_rs`authority`_ tallentaa osoitteen, jolla on oikeus kasvattaa laskuria.
_rs`initialize`_ kirjoittaa tämän osoitteen, ja _rs`increment`_ vaatii saman
osoitteen allekirjoituksen.

</WithNotes>

<SideBySide>

### !left

```rust title="initialize.rs" -w
#[account(
    init,
    payer = payer,
    // !mark
    // !mention counter-space
    space = 8 + Counter::INIT_SPACE,
    seeds = [COUNTER_SEED],
    bump
)]
pub counter: Account<'info, Counter>,
// ...

pub fn handle_initialize(ctx: Context<Initialize>) -> Result<()> {
    // !mark(1:2)
    // !mention(1:2) initialize-fields
    ctx.accounts.counter.count = 0;
    ctx.accounts.counter.authority = ctx.accounts.payer.key();

    Ok(())
}
```

```rust title="increment.rs" -w
pub fn handle_increment(ctx: Context<Increment>) -> Result<()> {
    // ...

    // !mark
    // !mention increment-count
    ctx.accounts.counter.count += 1;
    Ok(())
}
```

### !right

`state.rs` määrittelee _rs`Counter`_-tilin tiedot. Ohjetiedostot käyttävät tätä
tyyppiä luodessaan ja päivittäessään tilin:

- [_rs`Counter::INIT_SPACE`_](mention:counter-space) määrittää tilin koon
  `state.rs`:ssä määriteltyjen kenttien mukaan.
- [_rs`count`_ ja _rs`authority`_](mention:initialize-fields) ovat kenttien
  arvoja, jotka kirjoitetaan tilin alustuksen yhteydessä.
- [_rs`count += 1`_](mention:increment-count) päivittää tallennetun laskuriarvon
  validoinnin läpäisemisen jälkeen.

</SideBySide>

### error.rs

`error.rs` määrittelee ohjelman mukautetut virheet. Tässä mallissa virheet
havainnollistavat, kuinka ohjetoiminnot pysähtyvät, kun kutsujalla ei ole
oikeutta päivittää laskuria tai laskuri on jo saavuttanut _rs`MAX_COUNT`_-arvon.

<WithNotes>

```rust title="programs/my-program/src/error.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[error_code\]/] errorCode
#[error_code]
pub enum ErrorCode {
    // !tooltip[/Unauthorized/] unauthorized
    #[msg("Only the counter authority can update this counter")]
    Unauthorized,
    // !tooltip[/CounterOverflow/] counterOverflow
    #[msg("Counter has reached the maximum value")]
    CounterOverflow,
}
```

### !errorCode

_rs`#[error_code]`_ käskee Anchoria muuttamaan nämä variantit mukautetuiksi
ohjelmavirheiksi. Asiakkaat ja testit voivat käyttää näitä virheitä sen sijaan,
että arvailtaisiin raakojen virhekoodien perusteella.

### !unauthorized

_rs`Unauthorized`_ palautetaan, kun tapahtuman allekirjoittaja ei ole
laskuritiliin tallennettu auktoriteetti.

### !counterOverflow

_rs`CounterOverflow`_ palautetaan, kun laskurin kasvattaminen rikkoisi ohjelman
_rs`MAX_COUNT`_-säännön.

</WithNotes>

<SideBySide>

### !left

```rust title="increment.rs" -w
pub fn handle_increment(ctx: Context<Increment>) -> Result<()> {
    require_keys_eq!(
        ctx.accounts.counter.authority,
        ctx.accounts.authority.key(),
        // !mark
        // !mention unauthorized-error
        ErrorCode::Unauthorized,
    );
    require!(
        ctx.accounts.counter.count < MAX_COUNT,
        // !mark
        // !mention overflow-error
        ErrorCode::CounterOverflow,
    );

    ctx.accounts.counter.count += 1;
    msg!("Hello, world! Counter is now {}", ctx.accounts.counter.count);
    Ok(())
}
```

### !right

`error.rs` nimeää virheet, joita ohje voi palauttaa:

- [_rs`ErrorCode::Unauthorized`_](mention:unauthorized-error) palautetaan, kun
  allekirjoittaja ei ole laskuritiliin tallennettu auktoriteetti.
- [_rs`ErrorCode::CounterOverflow`_](mention:overflow-error) palautetaan, kun
  laskuri on jo saavuttanut _rs`MAX_COUNT`_-arvon.

</SideBySide>

### instructions.rs

`instructions.rs` yhdistää ohjetiedostot ohjelmapaketille, jotta `lib.rs` voi
käyttää _rs`initialize`_- ja _rs`increment`_-ohjeistuksen koodia. Jokainen
ohjetiedosto määrittelee kyseisen ohjeen vaatimat tilit sekä
käsittelijälogiikan, joka suoritetaan sen jälkeen, kun Anchor on vahvistanut
nämä tilit.

<WithNotes>

```rust title="programs/my-program/src/instructions.rs" -w
// !tooltip[/pub mod initialize/] moduleDeclarations
pub mod initialize;
pub mod increment;

// !tooltip[/pub use initialize/] reexports
pub use initialize::*;
pub use increment::*;
```

### !moduleDeclarations

_rs`pub mod`_-rivit tekevät `src/instructions/initialize.rs`:sta ja
`src/instructions/increment.rs`:sta osan ohjelmapakettia.

### !reexports

_rs`pub use`_-rivit vievät julkiset kohteet uudelleen näistä tiedostoista. Tämä
tekee ohjeen tilirakenteet ja käsittelijäfunktiot saataville
_rs`instructions`_-moduulin kautta.

</WithNotes>

### initialize.rs

`initialize.rs` määrittelee laskuritilin luomiseen tarvittavat tilit ja
kirjoittaa sitten tilin ensimmäiset arvot. _rs`#[derive(Accounts)]`_-rakenne
käyttää Anchorin
[tiliehtojen](https://www.anchor-lang.com/docs/references/account-constraints)
määrityksiä ilmaisemaan, mitkä tilit vaaditaan ja miten uusi laskuritili
luodaan.

<WithNotes>

```rust title="programs/my-program/src/instructions/initialize.rs" -w
use anchor_lang::prelude::*;

use crate::{constants::*, state::Counter};

// !tooltip[/derive\(Accounts\)/] accountsStruct
#[derive(Accounts)]
pub struct Initialize<'info> {
    // !tooltip[/#\[account\(mut\)\]/] payerMutable
    #[account(mut)]
    // !tooltip[/Signer<'info>/] payerSigner
    pub payer: Signer<'info>,
    // !tooltip[/#\[account\(/] counterConstraints
    #[account(
        // !tooltip[/init/] initConstraint
        init,
        // !tooltip[/payer = payer/] payerConstraint
        payer = payer,
        // !tooltip[/space = 8 \+ Counter::INIT_SPACE/] space
        space = 8 + Counter::INIT_SPACE,
        // !tooltip[/seeds = \[COUNTER_SEED\]/] seeds
        seeds = [COUNTER_SEED],
        // !tooltip[/bump/] bump
        bump
    )]
    // !tooltip[/Account<'info, Counter>/] counterAccountType
    pub counter: Account<'info, Counter>,
    // !tooltip[/Program<'info, System>/] systemProgram
    pub system_program: Program<'info, System>,
}

// !tooltip[/handle_initialize/] handler
pub fn handle_initialize(ctx: Context<Initialize>) -> Result<()> {
    // !tooltip[/counter.count = 0/] initialCount
    ctx.accounts.counter.count = 0;
    // !tooltip[/counter.authority = ctx.accounts.payer.key\(\)/] initialAuthority
    ctx.accounts.counter.authority = ctx.accounts.payer.key();

    // !tooltip[/system_program::Transfer/] cpiAccounts
    let cpi_accounts = anchor_lang::system_program::Transfer {
        from: ctx.accounts.payer.to_account_info(),
        to: ctx.accounts.counter.to_account_info(),
    };
    // !tooltip[/CpiContext::new/] cpiContext
    let cpi_ctx = CpiContext::new(anchor_lang::system_program::ID, cpi_accounts);
    // !tooltip[/system_program::transfer/] transfer
    anchor_lang::system_program::transfer(cpi_ctx, HELLO_WORLD_LAMPORTS)?;

    // !tooltip[/msg!/] programLog
    msg!("Hello, world! Counter initialized");
    Ok(())
}
```

## !!steps Account Context

_rs`Initialize`_-rakenne määrittelee tilit, jotka on sisällytettävä, kun
käyttäjä kutsuu _rs`initialize`_-ohjetta. Anchor tarkistaa nämä tilit ennen
käsittelijän suorittamista.

<CodePlaceholder title="initialize.rs" />

```rust !! title="initialize.rs"
// !focus(1:2)

pub struct Initialize<'info> {
    #[account(mut)]
    pub payer: Signer<'info>,
    #[account(
        init,
        payer = payer,
        space = 8 + Counter::INIT_SPACE,
        seeds = [COUNTER_SEED],
        bump
    )]
    pub counter: Account<'info, Counter>,
    pub system_program: Program<'info, System>,
}
```

## !!steps Payer Account

_rs`payer`_-tili maksaa laskuritilin luomisesta. _rs`Signer<'info>`_-tyyppi
tarkoittaa, että maksajan on allekirjoitettava transaktio, ja
_rs`#[account(mut)]`_ tarkoittaa, että maksajatiliä voidaan muuttaa, koska
lamport-yksiköitä vähennetään siitä.

<CodePlaceholder title="initialize.rs" />

```rust !! title="initialize.rs"
#[derive(Accounts)]
pub struct Initialize<'info> {
    // !focus(1:2)
    #[account(mut)]
    pub payer: Signer<'info>,
    #[account(
        init,
        payer = payer,
        space = 8 + Counter::INIT_SPACE,
        seeds = [COUNTER_SEED],
        bump
    )]
    pub counter: Account<'info, Counter>,
```

_rs`msg!`_ kirjoittaa päivitetyn lukeman ohjelmalokiin. _rs`Ok(())`_ ilmaisee,
että ohje palautui onnistuneesti.

<CodePlaceholder title="increment.rs" />

```rust !! title="increment.rs"
#[derive(Accounts)]
pub struct Increment<'info> {
    #[account(
        mut,
        seeds = [COUNTER_SEED],
        bump
    )]
    pub counter: Account<'info, Counter>,
    pub authority: Signer<'info>,
}

pub fn handle_increment(ctx: Context<Increment>) -> Result<()> {
    require_keys_eq!(
        ctx.accounts.counter.authority,
        ctx.accounts.authority.key(),
        ErrorCode::Unauthorized,
    );
    require!(
        ctx.accounts.counter.count < MAX_COUNT,
        ErrorCode::CounterOverflow,
    );

    ctx.accounts.counter.count += 1;
    // !focus(1:2)
    msg!("Hello, world! Counter is now {}", ctx.accounts.counter.count);
    Ok(())
}
```

</ScrollyCoding>

### !incrementAccounts

_rs`Increment`_ on _rs`increment`_-ohjeen tilikonteksti. Anchor käyttää sitä
muuttuvan laskuritilin ja vaaditun allekirjoittajan validointiin.

### !counterMutable

_rs`mut`_ kertoo Anchorille, että laskuritili on oltava kirjoitettavissa, koska
käsittelijä päivittää tallennettua _rs`count`_-arvoa.

### !counterSeeds

_rs`seeds = [COUNTER_SEED]`_ varmistaa, että annettu laskuritili on tästä
seed:stä ja ohjelmatunnuksesta johdettu PDA.

### !counterBump

_rs`bump`_ käskee Anchoria etsimään ja vahvistamaan laskuritilin PDA-bumpin.

### !incrementCounterAccountType

_rs`Account<'info, Counter>`_ käskee Anchoria deserialisoimaan tilin datan
_rs`Counter`_-tyypiksi `state.rs`:stä. Anchor tarkistaa myös, että tili on tämän
ohjelman omistama.

### !authoritySigner

_rs`Signer`_ vaatii auktoriteetin allekirjoituksen. Käsittelijä vertaa tämän
allekirjoittajan osoitetta laskuritilille tallennettuun auktoriteetin
osoitteeseen.

### !incrementHandler

Käsittelijä suoritetaan sen jälkeen, kun Anchor on vahvistanut tililuettelon,
tarkistanut omistajuuden ja deserialisoinut laskuritilin.

### !authorityCheck

_rs`require_keys_eq!`_ vertaa kahta julkista avainta ja keskeyttää ohjeen
suorituksen, ellei allekirjoittajan osoite vastaa laskuritilille tallennettua
auktoriteettia. Tämä on mallin sovellustason valtuutussääntö.

### !maxCheck

_rs`require!`_ tarkistaa boolean-säännön ja keskeyttää ohjeen suorituksen
virheellä _rs`CounterOverflow`_, jos laskuri on jo saavuttanut arvon
_rs`MAX_COUNT`_. Tämä mallisääntö havainnollistaa mukautettuja virheitä ja
epäonnistumista ennen muutosta.

### !mutateCount

Käsittelijä päivittää laskuritilin datan vasta, kun molemmat tarkistukset ovat
läpäisty. Jos kumpi tahansa tarkistus epäonnistuu, transaktio epäonnistuu eikä
tiliä muuteta.

### !programLog

_rs`msg!`_ kirjoittaa viestin ohjelman lokeihin, kun ohjelma käynnistetään.

```rust title="programs/my-program/tests/test_initialize.rs" -w
use {
    anchor_lang::{
```

### !litesvm

_rs`LiteSVM`_ on prosessinsisäinen Solana-virtuaalikone testausta varten. Se
mahdollistaa käännetyn ohjelman rekisteröinnin, tilien luomisen, transaktioiden
lähettämisen ja tilitilan tarkastamisen ilman validator-prosessin
käynnistämistä.

### !setup

Testi hakee ohjelman osoitteen _rs`declare_id!`_-arvosta
_rs`my_program::id()`_-funktion kautta ja luo uuden payer keypair -avaimen
testitransaktiota varten.

### !counterPda

Laskurin osoite johdetaan samasta _rs`COUNTER_SEED`_ ohjelman käyttämästä. Tämä
tekee testistä kohdennetun siihen PDA:han, jota `initialize.rs` odottaa.

### !programBytes

_rs`include_bytes!`_ lukee buildin tuottaman käännetyn `my_program.so`
tiedoston. Nämä tavut ovat suoritettava ohjelmakoodi, jota LiteSVM ajaa
testissä.

### !addProgram

_rs`svm.add_program`_ rekisteröi käännetyt ohjelmatavut ohjelmaosoitteeseen
LiteSVM:ssä. Airdrop antaa maksajalle riittävästi paikallista SOL:ia transaktion
ja tilin luomisen kattamiseen.

### !testInstruction

_rs`Instruction::new_with_bytes`_ luo käskyn ohjelman kutsumiseksi. Se sisältää
ohjelmatunnuksen, instruction data:n ja käskyn tilit.

### !blockhash

Jokainen Solana-transaktio tarvitsee tuoreen lohkohashin. Tässä testissä LiteSVM
tarjoaa käytettävän lohkohashin transaktiota varten.

### !message

Viesti sisältää käskyn, maksajan ja lohkohashin. Tämä on transaktiotieto, joka
allekirjoitetaan.

### !transaction

_rs`VersionedTransaction::try_new`_ allekirjoittaa viestin maksajan keypair:lla.
Maksaja allekirjoittaa, koska se maksaa transaktiomaksun ja rahoittaa uuden
laskuritilin.

### !sendTx

_rs`svm.send_transaction`_ suorittaa transaktion LiteSVM:ssä.

### !getCounter

Transaktion suorittamisen jälkeen testi lukee laskuritilin suoraan LiteSVM:stä,
jotta se voi tarkastella tilin tietoja.

### !deserialize

Laskuritili tallentaa sarjallistetun datan. _rs`Counter::try_deserialize`_
tarkistaa Anchor:n tilien erottelijan ja muuntaa tilin datan takaisin
_rs`Counter`_ tyypiksi `state.rs`:stä.

### !initAssert

Kun _rs`initialize`_ onnistuu, testi sarjatulkitsee laskuritilin ja tarkistaa,
että alkutila vastaa sitä, mitä \_rs`handle_initialize` kirjoitti.

### !incrementAssert

Kun _rs`increment`_ onnistuu, samalla laskuritilillä pitäisi olla _rs`count`_
asetettuna arvoon _rs`1`_ samalla säilyttäen sama valtuutus.
