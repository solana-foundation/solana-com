`@solana/react` julkaisee Kit-asiakkaan komponenttipuuhusi yhden tarjoajan
kautta, minkä jälkeen koukut lukevat RPC-dataa, tilauksia ja lompakkotilaa
siitä. Selaimen lompakkokoukut sijaitsevat
`@solana/kit-plugin-wallet/react`-paketissa ja lukevat samaa tarjoajaa —
erillistä lompakkotarjoajaa ei tarvita.

## Asennus

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Rakenna asiakas

Rakenna asiakas kerran, moduulitasolla, React-puun ulkopuolella. Selaimessa
`walletSigner` asettaa yhdistetyn lompakon sekä maksunmaksajaksi että
identiteetiksi.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` lisää `client.wallet` -ominaisuuden (Wallet Standard -löytäminen
ja yhdistäminen), ja `solanaRpc` lisää `client.rpc` sekä
`client.sendTransaction`-apufunktion. Vie asiakastyyppi `Awaited<>`-kääreeseen,
jotta se pysyy oikeana, jos lisäät myöhemmin asynkronisesti ratkeavan
liitännäisen, ja välitä kyseinen `AppClient` jokaiselle `useClient`-kutsulle.

## Kääri puusi kerran

`ClientProvider` julkaisee asiakkaan. Kaikki alla olevat koukut lukevat siitä.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Lompakkokoukut

Lompakkokoukut lukevat asiakkaan `wallet`-ominaisuutta, joten ne toimivat vain
`ClientProvider`-komponentin sisällä, jonka asiakas on asentanut
lompakkoliitännäisen.

- `useWallets()` — Wallet Standard -lompakot, jotka on löydetty asiakkaan
  ketjulle.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` tai `null`. `signer` on
  `null` vain luettaville lompakoille.
- `useConnect()` / `useDisconnect()` — palauttavat `ActionResult`-olion, jonka
  `dispatch` (ja `dispatchAsync`) suorittaa toiminnon ja jonka `isRunning`,
  `status`, `data`, `error` ja `reset` seuraavat sen tulosta.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Lähetä transaktio

`useClient` palauttaa saman asiakkaan, jonka julkaisit. Rakenna käskyt
generoidun ohjelma-asiakkaan avulla ja lähetä ne
`client.sendTransaction`-funktion kautta.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Viennän `AppClient`-tyypin välittäminen `useClient`:lle antaa hookille täyden
laajennuksilla varustetun asiakkaan, joten `client.rpc` ja
`client.sendTransaction` ovat tyypitettyjä.

## Lue tietoja

`@solana/react` sisältää hookit, jotka lukevat asiakkaan kautta:

- `useRequest(source)` — kertaluonteinen RPC-luku; palauttaa
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — reaaliaikainen tilaus; palauttaa
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — RPC:llä alustettu arvo, joka pidetään ajantasaisena
  tilauksen avulla.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Muistita lähde (käyttämällä `useMemo`:a tai `useCallback` funktiota varten),
jotta hook hakee uudelleen vain syötteiden muuttuessa eikä jokaisella
renderöintikerralla.

Välimuistitusta ja uudelleenvalidointia varten `@solana/react/swr`- ja
`@solana/react/query`- alipolut käärivät samat hookit
[SWR](https://swr.vercel.app)- ja
[TanStack Query](https://tanstack.com/query)-kirjastoja varten.

## Yleiset käytännöt Solana-kehittäjille

- **Yksi provider**: `ClientProvider` on ainoa tarvitsemasi provider;
  lompakkohookit lukevat sen asiakkaan `wallet`-kyvyn kautta.
- **Palvelinkomponentit huomioitu**: Merkitse vain lehtikomponentit, jotka
  kutsuvat hookkeja `"use client"`:llä; palvelinluvut voivat käyttää tavallista
  Kit RPC -asiakasta ilman lompakkoa.
- **Testaus**: Julkaise mockattu asiakas `ClientProvider`:n kautta lompakkojen
  ja RPC-vastausten simuloimiseksi.

<Callout type="warn" title="Kaksi hookki-perhettä, samat nimet">
  `useSignIn` ja `useSignMessage` esiintyvät sekä `@solana/react`:ssä (ne
  ottavat `UiWalletAccount`-argumentin) että
  `@solana/kit-plugin-wallet/react`:ssä (asiakaspohjainen, ei tiliargumenttia,
  palauttaa `ActionResult`:n). Käytä `@solana/kit-plugin-wallet/react`-versioita
  yllä olevan asiakasmallin kanssa; vanhemmat `@solana/react`-lompakkohookit
  ovat jäämässä pois käytöstä.
</Callout>

Yhdistä tämä opas [Kit-asiakkaan yleiskatsaukseen](/docs/frontend/client)
ymmärtääksesi asiakkaan, josta kukin hookki lukee.
