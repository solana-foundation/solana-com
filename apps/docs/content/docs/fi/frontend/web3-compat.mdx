Jos olet käyttänyt `@solana/web3.js` (v1), `@solana/wallet-adapter-*` tai
aikaisempia framework-kit-paketteja (`@solana/client`, `@solana/react-hooks`,
`@solana/web3-compat`), nämä lähestymistavat ovat vanhentuneita. Uudet
sovellukset rakennetaan suoraan `@solana/kit`:n ja kit-laajennusten päälle.
Tällä sivulla tutut käsitteet yhdistetään niitä vastaaviin Kit-käsitteisiin.

## Käsitteiden vastaavuudet

| web3.js v1 / wallet-adapter            | Kit                                                                                   |
| -------------------------------------- | ------------------------------------------------------------------------------------- |
| `new Connection(url)`                  | `createSolanaRpc(url)`, tai `.use(solanaRpc({ rpcUrl }))` asiakkaalla                 |
| `PublicKey`                            | `Address` — muunna merkkijono `address("...")`:lla                                    |
| `Keypair`                              | `generateKeyPairSigner()` tai `@solana/kit-plugin-signer`-laajennukset                |
| `SystemProgram.transfer(...)`          | `getTransferSolInstruction(...)` paketista `@solana-program/system`                   |
| `LAMPORTS_PER_SOL`-laskenta            | `lamports(...)` paketista `@solana/kit`                                               |
| `Transaction` / `VersionedTransaction` | Tapahtumaviestit, jotka transaktiosuunnittelija rakentaa ja pakkaa                    |
| `sendAndConfirmTransaction(...)`       | `client.sendTransaction([...])` (suunnittelija allekirjoittaa, lähettää ja vahvistaa) |
| `@solana/wallet-adapter-*`             | `@solana/kit-plugin-wallet` + Wallet Standard -löytäminen                             |

## Tulitko web3.js:stä?

Jos ylläpidät laajaa `@solana/web3.js` v1 -koodikantaa eikä täydellinen
siirtyminen Kitiin ole vielä käytännöllistä, web3.js v3 toimii siltana. Se
julkaistaan tällä hetkellä nimellä `@solana/web3.js@rc` (`3.0.0-rc`-linja) ja
rakentaa perinteisen luokkapohjaisen API:n — `Connection`, `Keypair`,
`Transaction` — `@solana/kit`:n sisäisten osien päälle. Koska se rakentuu Kitin
päälle, yhteensopivuus on tiivis: `PublicKey` on vanhentunut alias
`Address`:lle, ja v3:n `Keypair` täyttää rakenteellisesti Kitin
`KeyPairSigner`:n vaatimukset, joten voit välittää sen suoraan Kit API:lle,
laajennuksille ja Codama-generoituihin asiakkaisiin.

v3 on edelleen **julkaisuehdokas** — kiinnitä tarkat versiot ja varaudu
joihinkin API-muutoksiin RC-versioiden välillä. Se on vanhan koodin
yhteensopivuuspolku, ei oletusvalinta uusille projekteille; uudet sovellukset
tulisi rakentaa suoraan Kitin päälle. Itse siirtymistä varten katso virallinen
[web3.js v1 → v3 -siirtymisopas](https://github.com/solana-foundation/solana-web3.js/blob/v3.x/docs/web3js-v1-to-v3-migration.md).

## Lompakot

Kit ei käytä wallet-adapteria. `@solana/kit-plugin-wallet` tunnistaa Wallet
Standard -lompakot ja tarjoaa ne käytettäväksi `client.wallet`:n ja
[React-koukkujen](/docs/frontend/react-hooks) kautta. Modernit lompakot
(Phantom, Solflare, Backpack ja muut) ilmoittavat itsensä Wallet Standardin
kautta, joten erillistä lompakohtaista adapteripaketia ei tarvita.

Voit myös käyttää valmiita työkaluja, kuten
[@solana/connector](https://www.npmjs.com/package/@solana/connector), joka
hyödyntää Kitin primitiivejä suoraan lompakoihin yhdistämiseen.

<Callout type="info" title="Käytätkö vielä wallet-adapteria?">
  Jos ylläpidät `@solana/wallet-adapter-*`:lle rakennettua sovellusta etkä ole
  valmis siirtymään uuteen,
  [wallet-adapter-repositorio](https://github.com/anza-xyz/wallet-adapter) on
  edelleen kyseisen pinon viitteellinen toteutus. Kaikkea uutta varten
  suosittelemme Kitin lompakon liitännäistä.
</Callout>
