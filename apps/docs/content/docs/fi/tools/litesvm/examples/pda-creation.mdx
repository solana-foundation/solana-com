## Täydellinen testi

```rust title="tests/pda_test.rs"
use litesvm::LiteSVM;
use solana_account::Account;
use solana_sdk::pubkey::Pubkey;

#[test]
fn test_pda_creation() {
    let mut svm = LiteSVM::new();

    // Program that owns the PDA
    let program_id = Pubkey::new_unique();

    // Derive PDA address
    let seed = b"my_pda";
    let (pda, bump) = Pubkey::find_program_address(
        &[seed],
        &program_id
    );

    println!("PDA: {}", pda);
    println!("Bump: {}", bump);

    // Create PDA account manually (for testing)
    svm.set_account(pda, Account {
        lamports: 1_000_000,
        data: vec![bump], // Store bump seed in data
        owner: program_id,
        executable: false,
        rent_epoch: 0,
    }).unwrap();

    // Verify PDA was created
    let account = svm.get_account(&pda).unwrap();
    assert_eq!(account.owner, program_id);
    assert_eq!(account.data[0], bump);

    println!("PDA created successfully!");
}
```

## Keskeiset kohdat

1. **PDA:n johtaminen**: Käytä `Pubkey::find_program_address()` PDA-osoitteiden
   johtamiseen
2. **Seeds**: PDA:t johdetaan seed-arvoista ja ohjelman tunnuksesta
3. **Bump seed**: bump seed varmistaa, että osoite on ed25519-käyrän
   ulkopuolella
4. **Tilin luominen**: Käytä `set_account()` tilien manuaaliseen luomiseen
   testausta varten
5. **Omistajuus**: PDA:t on omistettava ohjelman toimesta, joka johti ne
