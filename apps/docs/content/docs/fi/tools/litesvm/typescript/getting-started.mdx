# LiteSVM TypeScriptille

Solana-testipaketit, jotka käynnistävät `solana-test-validator`, vievät
sekunteja testiä kohden — odottelet prosessin käynnistystä, RPC-yhteyksiä ja
verkon edestakaisin-matkoja. LiteSVM suorittaa samat testit millisekunteissa. Ei
ulkoista prosessia, ei verkkoa, kaikki muistissa testisuorittimesi sisällä.

<Callout type="info">
  **Testattu versioita vastaan:** `@solana/kit@6.8.0`,
  `@solana/kit-plugin-litesvm@0.10.0`, `@solana/kit-plugin-signer@0.10.0`,
  `@solana-program/system@0.12.0`, `@solana-program/token@0.13.0`. Nämä
  laajennuspaketit ovat uusia ja kehittyvät nopeasti — jos kohtaat
  yhteensopimattomuuden, tarkista
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  tarkat versiot, joita dokumentaatio ja testipaketit käyttävät.
</Callout>

Koostat Kit-asiakkaan `@solana/kit`:stä asentamalla laajennuksia `.use()`
avulla: allekirjoittajalaajennuksen
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
LiteSVM-laajennuksen
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm))
ja tarvitsemasi ohjelmaLaajennukset. Tulos tarjoaa `client.sendTransaction()`:n,
`client.rpc`:n sekä suoran pääsyn tilitilaan, kelloon ja sysvar-muuttujiin —
kaikki poistumatta prosessistasi.

### Luo asiakkaasi

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` palauttaa tyhjän pohjan — jokainen `.use(plugin)` lisää
  siihen ominaisuuksia. `signer()`-laajennuksen on tultava **ennen**
  `litesvm()`:ää, koska kuljetuslaajennukselle täytyy liittää maksaja.
  `signer(mySigner)` on lyhenne saman keypair:n asentamiselle sekä `payer`:ksi
  että `identity`:ksi — käytä `payer()`:ää tai `identity()`:ää samasta
  paketista, jos haluat erottaa nämä roolit. Tämä järjestys on sama
  `solanaRpc()`:lle, kun käytät oikeaa verkkoa.
</Callout>

<Callout type="info">
  Haluatko yksiriviseen asennuksen nopeisiin kokeiluihin? `litesvm()`-paketti
  sisältää airdropin, joten voit ketjuttaa
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))`
  `@solana/kit-plugin-signer`:stä luodaksesi ja rahoittaaksesi maksajan
  automaattisesti.
</Callout>

</Step>

</Steps>

## Asiakkaan ymmärtäminen

`.use(signer(...)).use(litesvm())`:n jälkeen asiakas tarjoaa:

| Ominaisuus                  | Kuvaus                                                                                                                                                                                                        |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | `TransactionSigner`, jonka asensit `signer()`-liitännäisellä (asetettu myös nimellä `client.identity`)                                                                                                        |
| `client.identity`           | Lompakkoallekirjoittaja, joka omistaa tilit, tokenit ja valtuudet                                                                                                                                             |
| `client.svm`                | Suora [`LiteSVM`](https://www.npmjs.com/package/litesvm)-instanssi tilien hallintaan, transaktioihin ja konfigurointiin                                                                                       |
| `client.rpc`                | Kit-yhteensopiva RPC-rajapinta (osajoukko: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Suunnittele, allekirjoita ja lähetä ohjeet transaktiona                                                                                                                                                       |
| `client.airdrop`            | Pyydä SOL LiteSVM-hanasta                                                                                                                                                                                     |
| `client.getMinimumBalance`  | Laske rent-vapautukseen vaadittava lamport-vähimmäismäärä                                                                                                                                                     |

### client.svm:n käyttö

Suora pääsy LiteSVM:ään seuraaviin tarkoituksiin:

- Tilien hallinta (`setAccount`, `getAccount`, `airdrop`)
- Transaktioiden suoritus (`sendTransaction`, `simulateTransaction`)
- Konfigurointi (`withSigverify`, `withBlockhashCheck` jne.)
- Aikamanipulaatio (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### client.rpc:n käyttö

Kit-yhteensopiva RPC-rajapinta tavallisiin operaatioihin:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  RPC-kerros tukee vain **base64-enkoodausta**. Muiden enkoodausten pyytäminen
  aiheuttaa virheen.
</Callout>

## Konfigurointi

Konfiguroi LiteSVM käyttämällä rakentajametodeja `client.svm`:ssä:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## Transaktioiden lähettäminen

Lisää ohjelma-liitännäiset kuljetuskerroksen jälkeen — ne kiinnittävät ohjeiden
rakentajat asiakkaaseen, joten sinun ei tarvitse koskaan käsitellä raakaa
`TransactionMessage`:ä:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## SVM → RPC-kulku

Voit asettaa tilan SVM:n kautta ja lukea sen RPC:n kautta:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```
