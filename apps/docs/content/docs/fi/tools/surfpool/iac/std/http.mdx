## send_http_request

`std::send_http_request` tekee HTTP-pyynnön annettuun URL-osoitteeseen ja vie
vastauksen ulos.

### Syötteet

| Nimi             | Pakollinen  | Tyyppi  | Kuvaus                                                              |
| ---------------- | ----------- | ------- | ------------------------------------------------------------------- |
| `url`            | pakollinen  | string  | Pyynnön URL-osoite. Tuetut skeemat ovat http ja https               |
| `body`           | valinnainen | string  | Pyynnön runko merkkijonona tai JSON-objektina                       |
| `method`         | valinnainen | string  | Pyynnön HTTP-metodi. Sallitut metodit: GET, HEAD, POST              |
| `timeout_ms`     | valinnainen | integer | Pyynnön aikakatkaisu millisekunteina                                |
| `headers`        | valinnainen | object  | Kartta pyynnön otsikkokenttien nimistä ja arvoista                  |
| `pre_condition`  | valinnainen | map     | Esiehtojen kartta, joka arvioidaan ennen komennon suorittamista     |
| `post_condition` | valinnainen | map     | Jälkiehtojen kartta, joka arvioidaan komennon suorittamisen jälkeen |

### Esiehdot

Esiehdot ovat väitteitä, jotka arvioidaan ennen komennon suorittamista. Niillä
voidaan määrittää, tulisiko komento suorittaa tai tulisiko jokin tietty
toimintatapa suorittaa väitteen tuloksen perusteella.

| Ominaisuus  | Kuvaus                                                                                                              |
| ----------- | ------------------------------------------------------------------------------------------------------------------- |
| `behavior`  | Toimintatapa, jos esiehtoväite ei täyty. Vaihtoehdot: `halt` (oletus), `log`, `skip`                                |
| `assertion` | Tarkistettava väite. Sen tulisi palauttaa totuusarvo tai käyttää `std::assert_eq`-funktiota ja vastaavia funktioita |

### Jälkiehdot

Jälkiehdot ovat väittämiä, jotka arvioidaan komennon suorittamisen jälkeen.
Niiden avulla voidaan määrittää, tulisiko komento suorittaa uudelleen.

| Ominaisuus   | Kuvaus                                                                                            |
| ------------ | ------------------------------------------------------------------------------------------------- |
| `retries`    | Kuinka monta kertaa komento suoritetaan uudelleen, jos väittämä epäonnistuu. Oletus: 0            |
| `backoff_ms` | Millisekuntia odotettava ennen uudelleensuoritusta. Oletus: 1000                                  |
| `behavior`   | Toiminta, jos jälkiehtoväittämä ei täyty. Vaihtoehdot: `halt` (oletus), `log`, `skip`, `continue` |
| `assertion`  | Tarkistettava väittämä                                                                            |

### Tulosteet

Kun `send_http_request`-toiminto suoritetaan onnistuneesti, seuraavat tulosteet
liitetään toimintoon:

| Nimi            | Tyyppi  | Kuvaus                                  |
| --------------- | ------- | --------------------------------------- |
| `response_body` | string  | Vastausteksti palautettuna merkkijonona |
| `status_code`   | integer | HTTP-vastauksen statuskoodi             |

### Esimerkki

```hcl
action "example" "std::send_http_request" {
  url = "https://example.com"
}

output "status" {
  value = action.example.status_code
}
// > status: 200
```

### POST-pyyntöesimerkki

```hcl
action "create_resource" "std::send_http_request" {
    description = "Create a new resource"
    url = "https://api.example.com/resources"
    method = "POST"
    headers = {
        "Content-Type" = "application/json"
    }
    body = {
        "name" = variable.name,
        "value" = variable.value
    }
}
```

### Pyyntö todennuksen kanssa

```hcl
action "api_call" "std::send_http_request" {
    description = "Fetch data from API"
    url = "https://api.example.com/data"
    method = "GET"
    headers = {
        "Authorization" = "Bearer ${variable.token}"
    }
}

output "response" {
    value = action.api_call.response_body
}

output "status" {
    value = action.api_call.status_code
}
```
