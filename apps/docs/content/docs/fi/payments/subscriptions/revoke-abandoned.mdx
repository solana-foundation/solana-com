---
title: Peruuta Hylätyt Delegoinnit Ja Tilaukset
description:
  Opas rent-maksujen takaisinperintään delegoinnista tai tilauksesta, jonka
  Subscription Authority on poistunut.
---

Delegoinnit ja tilaukset pitävät hallussaan rent-maksuja, jotka normaalisti
palautuvat niille kirjatulle maksajalle peruutuksen yhteydessä. Jos sponsori
rahoitti tilin ja käyttäjä myöhemmin sulkee tai korvaa Subscription
Authoritynsa, tili jää jumiin ja sen rent-maksut jumiutuvat.
`RevokeAbandonedDelegation` ja `RevokeAbandonedSubscription` antavat kirjatulle
maksajalle mahdollisuuden periä ne takaisin.

## Milloin Se Soveltuu

Sallittu ainoastaan silloin, kun tilin Subscription Authority on todistettavasti
päättynyt — suljettu tai suljettu ja uudelleenluotu (jolloin `init_id` ei enää
täsmää). Aktiivista delegointia tai tilausta ei voi koskaan sulkea tällä tavoin.

|                                            | `RevokeAbandonedDelegation`                                                                                              | `RevokeAbandonedSubscription`                                                                                                                |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| Sulkee                                     | Kiinteän tai toistuvan delegoinnin PDA:n                                                                                 | Tilauksen PDA:n                                                                                                                              |
| Allekirjoittaja                            | Kirjattu maksaja                                                                                                         | Kirjattu maksaja                                                                                                                             |
| Tilit                                      | Delegointi + kirjattu authority                                                                                          | Tilaus + kirjattu authority + suunnitelma                                                                                                    |
| Kelpaa kun                                 | Authority suljettu tai uudelleenluotu (`init_id` ei täsmää)                                                              | Sama, suunnitelman mintille — mint luetaan sidotusta suunnitelmasta, joten liittymätöntä authoritya ei voi korvata sulkemisen pakottamiseksi |
| Käytä tavallista peruutusta sen sijaan kun | Delegointi on [vanhentunut tai täysin käytetty](/docs/payments/subscriptions/recurring-delegation#revoke-the-delegation) | Tilaus on [peruutettu ja vanhentunut](/docs/payments/subscriptions/subscription-plan#cancel-and-revoke)                                      |

`RevokeAbandonedDelegation` on myös kirjatun maksajan ainoa palautuskeino
ikuisesti voimassa olevalle delegoinnille (`expiry_ts == 0`), jota ei ole täysin
käytetty: tavallinen peruutus aktivoituu maksajalle vain vanhentumisen
yhteydessä tai, kiinteän delegoinnin tapauksessa, täyden käytön myötä.

## Peruuta Hylätty Delegointi

Kirjattu maksaja allekirjoittaa. Anna päättyneen delegoinnin PDA ja sen kirjattu
Subscription Authority (joka saattaa jo olla suljettu). Rent palautuu
maksajalle.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedDelegationInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedDelegationInstruction({
      payer: payerSigner,
      delegationAccount: delegationPda,
      subscriptionAuthority: subscriptionAuthorityPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let delegation_pda: Address = address!("DELEGATION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");

    let revoke_abandoned_ix = RevokeAbandonedDelegationBuilder::new()
        .payer(payer)
        .delegation_account(delegation_pda)
        .subscription_authority(subscription_authority)
        .instruction();
    ```

  </Tab>
</Tabs>

## Peruuta hylätty tilaus

Rekisteröity maksaja allekirjoittaa. Anna hylätyn tilauksen PDA, sen
rekisteröity Subscription Authority (joka saattaa olla jo suljettu) ja
suunnitelma, johon tilaus kuuluu. rent palautuu maksajalle.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedSubscriptionInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedSubscriptionInstruction({
      payer: payerSigner,
      subscriptionAccount: subscriptionPda,
      subscriptionAuthority: subscriptionAuthorityPda,
      planPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let subscription_pda: Address = address!("SUBSCRIPTION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");
    let plan_pda: Address = address!("PLAN_PDA_ADDRESS_HERE");

    let revoke_abandoned_ix = RevokeAbandonedSubscriptionBuilder::new()
        .payer(payer)
        .subscription_account(subscription_pda)
        .subscription_authority(subscription_authority)
        .plan_pda(plan_pda)
        .instruction();
    ```

  </Tab>
</Tabs>

## Huomioita

- Rekisteröity maksaja allekirjoittaa, ei valtuuttaja, valtuutettu eikä tilaaja.
- Molemmat ohjeet epäonnistuvat, jos Subscription Authority on edelleen
  aktiivinen tilille — käytä sen sijaan tavallista peruutusta.
- `RevokeAbandonedDelegation` toimii sekä kiinteille että
  toistuvaisvaltuutuksille.
