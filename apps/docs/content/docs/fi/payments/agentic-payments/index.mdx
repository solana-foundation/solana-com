Tekoälyagentit ovat yhä itsenäisempiä työnkuluissaan: ne kirjoittavat koodia,
hakevat dataa, varaavat resursseja ja paljon muuta. Toimiakseen täysin
itsenäisesti niiden täytyy pystyä suorittamaan "mikromaksuja" palveluista
ohjelmallisesti, reaaliajassa, mahdollisimman alhaisin kustannuksin. Perinteiset
maksujärjestelmät eivät toimi mikromaksuille. Solanan alle sentin maksut ja alle
sekunnin lopullisuus tekevät tästä teknisesti ja taloudellisesti mahdollista.

## Miksi tämä on tärkeää?

Kuvittele esimerkiksi kehittäjä, joka käyttää LLM:ää sovelluksen rakentamiseen
ja tarvitsee dataa maksullisesta API:sta. Tärkeän datan hakemiseksi kehittäjän
täytyisi keskeyttää työnkulkunsa, tutkia API:ja, valita palveluntarjoaja, luoda
tili, lisätä maksutapa, generoida API-avain, kopioida avain ympäristöönsä ja
sitten jatkaa työnkulkuaan. Tämä kitka rikkoo flow-tilan. Tekoälyavusteisissa
työnkuluissa se on este.

Sen sijaan agenttipohjaisilla maksuilla agentti voi tehdä pyynnön, maksaa siitä
ja jatkaa työnkulkuaan. Ei tilejä. Ei API-avaimia. Ei manuaalista toimintaa.

### Maksa API:sta pay.sh:lla

[pay.sh](https://pay.sh) on maksukerros HTTP-agenteille ja
komentorivityökaluille. Tutustu [pay.sh-dokumentaatioon](https://pay.sh/docs)
asentaaksesi `pay`:n. `pay` CLI käärii työkaluja kuten `curl`, `codex` ja
`claude`, jotta ne voivat kutsua maksullisia API:ja ilman palveluntarjoajan
tilejä tai API-avaimia. Kun API palauttaa MPP- tai x402 `402 Payment Required`
-haasteen, `pay` pyytää paikallista lompakkoasi hyväksymään allekirjoituksen.

Voit asentaa Pay:n seuraavasti:

```bash
brew install pay
```

tai

```bash
npm install -g @solana/pay
```

Tarkista asennus seuraavasti:

```bash
pay --version
```

Työkaluketju määrittää osoitteesi automaattisesti. Suorita seuraava komento
lompakkosi käyttöönoton ja täydennyksen yhteydessä:

```bash
pay setup
```

## x402-protokolla

Agenttimaksut tarvitsevat tavan, jolla asiakkaat ja palvelimet voivat neuvotella
maksuehdoista tavallisen web-infrastruktuurin kautta häiritsemättä perinteisiä
web-rajapintoja. Mikromaksualue on vielä kehittymisvaiheessa, ja uusia työkaluja
sekä standardeja syntyy todennäköisesti alan kehittyessä.
[x402-protokolla](https://www.x402.org/) on kuitenkin noussut varhaiseksi
standardiksi vahvalla ekosysteemituella. x402 hyödyntää
[HTTP:n 402 "Payment Required"](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status/402)
-tilakoodi – standardi HTTP/1.1:stä lähtien, josta tuli käytännöllinen vasta
lohkoketjuselvityksen myötä.

### Kuinka se toimii

Asiakas tekee pyynnön, vastaanottaa 402-vastauksen maksuehtoineen ja lähettää
pyynnön uudelleen allekirjoitetun maksun kanssa. Palvelin delegoi vahvistuksen
ja selvityksen fasilitaattorille – valinnaiselle välittäjälle, joka hoitaa
ketjunsisäisen tapahtuman lähettämisen. Kun fasilitaattori vahvistaa
selvityksen, palvelin palauttaa pyydetyn sisällön. Tämä erottelu mahdollistaa
sen, että API-tarjoajat voivat vastaanottaa maksuja ilman suoraa ketjunsisäisen
infrastruktuurin hallintaa.

![x402-vuokaavio](/assets/guides/x402/x402-flow-solana.png)

Protokolla on tilaton (ei istuntoja eikä API-avaimia), toimii tavallisen
HTTP-infrastruktuurin kanssa ja tukee mitä tahansa palvelimen hyväksymää
SPL-tokenia. Tässä on esimerkki Express-palvelimen käyttöönotosta
x402-väliohjelmiston avulla:

```ts
app.use(
  paymentMiddleware(
    {
      "GET /costly-data": {
        accepts: [
          {
            scheme: "exact",
            price: "$0.001",
            network: "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
            payTo: svmAddress
          }
        ],
        description: "Costly data",
        mimeType: "application/json"
      }
    },
    new x402ResourceServer(facilitatorClient).register(
      "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
      new ExactSvmScheme()
    )
  )
);

app.get("/costly-data", (req, res) => {
  res.send({
    report: {
      data: "costly data"
    }
  });
});
```
