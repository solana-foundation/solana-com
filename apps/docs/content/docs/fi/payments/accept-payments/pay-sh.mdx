Voit hyväksyä maksuja API-pyynöissä Pay.sh:n kautta määrittämällä
Pay-palvelimen. Asenna Pay komennolla:

```bash
brew install pay
```

tai

```bash
npm install -g @solana/pay
```

Tarkista asennus komennolla:

```bash
pay --version
```

Työkaluketju määrittää osoitteesi automaattisesti. Suorita seuraava komento
lompakkosi määrittämiseksi ja täydentämiseksi:

```bash
pay setup
```

## Demopalvelimen määrittäminen

Voit aloittaa maksujen hyväksymisen käynnistämällä palvelimesi. Näet
demopalvelimen toiminnassa komennolla:

```bash
pay server demo
```

Tämä tekee kaksi asiaa. Se käynnistää live-Pay-palvelimen, jota voit kutsua
HTTP-pyynnöllä, ja luo `pay-demo.yml`-tiedoston nykyiseen hakemistoosi.

Kaikki API-reitit määritetään `endpoints` YAML-konfiguraation kautta. Maksujen
hyväksymiseksi sinun täytyy lisätä `metering:`-attribuutti.
`/api/v1/reports/usage`-reitti on tehty esimerkiksi sinulle:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Voit testata maksujen toimivuuden suorittamalla seuraavan komennon asiakkaana
toisessa terminaalissa:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```
