Vastaanota maksuja Solanassa käyttämällä valmiita komponentteja, Solana Pay
-protokollaa tai suoraa RPC-integraatiota.

## Vastaanotto-osoitteen käytäntö

Solanassa **jaa lompakko-osoitteesi, ei token account -tiliäsi**. Tyypillisesti
lähettäjä johtaa sopivan
[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)
-tilin riippumatta siitä, mitä tokenia he lähettävät (katso
[Maksujen lähettäminen](/docs/payments/send-payments) lisätietoja varten).

**Esimerkki:** Jaa `7EcDhS...` (lompakkosi), ei `9xKpQ...` (USDC token account
-tilisi).

<Callout>
  Myyntipiste- ja verkkokauppasovelluksiin katso [Solana
  Pay](/docs/payments/accept-payments/solana-pay) QR-koodeja ja maksulinkkejä
  varten, tai [Payment Button](/docs/payments/accept-payments/payment-button)
  valmiita React-komponentteja varten.
</Callout>

## Integrointivaihtoehdot

| Lähestymistapa                                                  | Paras käyttötarkoitus                 | Monimutkaisuus |
| --------------------------------------------------------------- | ------------------------------------- | -------------- |
| [Payment Button](/docs/payments/accept-payments/payment-button) | React-sovellukset, nopea integraatio  | Matala         |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | QR-koodit, maksulinkit, myyntipisteet | Keskitaso      |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Mukautetut prosessit, täysi hallinta  | Korkeampi      |

## Vahvistus

Maksun vastaanotettuasi vahvista sen saapuminen käyttämällä
[Verification Tools](/docs/payments/accept-payments/verification-tools)
-työkaluja. Suurten volyymien sovelluksiin katso
[Indexing](/docs/payments/accept-payments/indexing) tuotantotason tapahtumien
seurantaa varten.
