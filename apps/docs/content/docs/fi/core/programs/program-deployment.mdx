## Ohjelmien päivittäminen

Ohjelman päivittämiseksi tilin on pidettävä hallussaan päivitysoikeutta
(tyypillisesti tili, joka alun perin
[otti ohjelman käyttöön](/docs/programs/deploying)). Käyttöönoton tai
päivityksen aikana uusi tavukoodi ladataan väliaikaiselle puskuritilille ja
kirjoitetaan sitten program data -tilille. Päivitysoikeuden asettaminen arvoon
_rs`None`_ tekee ohjelmasta muuttumattoman ja estää pysyvästi kaikki tulevat
päivitykset.

### Päivitysmekanismi

Kun
[`UpgradeableLoaderInstruction::Upgrade`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L662)
käsitellään, ajonaikaympäristö:

1. Varmistaa, että program account on kirjoitettavissa ja loader-v3:n omistama.
2. Varmistaa, että buffer account sisältää _rs`Buffer`_-tilan oikealla
   oikeudella.
3. Varmistaa, että programdata account -tilin `upgrade_authority_address` täsmää
   eikä ole _rs`None`_.
4. Varmistaa, että ohjelmaa ei ole jo otettu käyttöön nykyisessä slotissa
   (`clock.slot != slot`).
5. Lataa ja varmistaa uudet ELF-tavut puskurista.
6. Kopioi uuden tavukoodin puskurista programdata account -tilille ja nollaa
   jäljelle jäävät tavut.
7. Rahoittaa programdata account -tilin vuokravapauteen.
8. Tyhjentää puskuritilin (asettaa lamportit arvoon 0) ja katkaisee sen datan.
9. Uusi versio tulee voimaan seuraavassa slotissa (`deployment_slot + 1`).

Itse `Program`-tili (sen tila ja `programdata_address`-osoitin) ei muutu
päivityksen aikana. Vain _rs`ProgramData`_-tilin tavukoodi ja slot-metatiedot
päivitetään.

## Loader-v3-käskyviite

Loader-v3 (BPF Loader Upgradeable) on nykyinen oletuslataaja ohjelmien
käyttöönotolle Solanassa.

| Ohje                                                                                                         | Kuvaus                                                                                                                                                                                                                                                      |
| ------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`InitializeBuffer`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L454)      | Asettaa puskuritilin tilan `Buffer`:ksi määritetyllä oikeudella. Epäonnistuu, jos tili on jo alustettu.                                                                                                                                                     |
| [`Write`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L469)                 | Kirjoittaa tavut määritettyyn sijaintiin puskuritilissä. Oikeuden on allekirjoitettava.                                                                                                                                                                     |
| [`DeployWithMaxDataLen`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L498)  | Luo uuden ohjelman: johtaa ProgramData-osoitteen, luo ProgramData-tilin CPI:n avulla System Program -ohjelmaan, varmentaa ja ottaa käyttöön ELF:n puskurista, asettaa Program-tilin tilan ja merkitsee sen suoritettavaksi.                                 |
| [`Upgrade`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L662)               | Korvaa olemassa olevan ohjelman ProgramData-tilin tavukoodin puskurista.                                                                                                                                                                                    |
| [`SetAuthority`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L835)          | Muuttaa Buffer- tai ProgramData-tilin oikeutta. Asettaminen `None`:ksi ProgramData-tilissä tekee ohjelmasta muuttumattoman. Buffer-oikeutta ei voi asettaa `None`:ksi.                                                                                      |
| [`SetAuthorityChecked`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L892)   | Kuten `SetAuthority`, mutta vaatii myös uuden oikeuden allekirjoittamaan transaktion.                                                                                                                                                                       |
| [`Close`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L960)                 | Sulkee puskuri-, alustamattoman tai ProgramData-tilin siirtämällä sen lamportit vastaanottajalle. ProgramData-tilin sulkeminen kirjoittaa myös `Closed`-hautakiven ohjelmavälimuistiin. Ei voi sulkea ohjelmaa, joka otettiin käyttöön nykyisessä slotissa. |
| [`ExtendProgramChecked`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L1071) | Laajentaa ProgramData-tilin varauksen lisätavuilla rahoittaen ylimääräisen vuokran maksajalta.                                                                                                                                                              |

## Ohjelmien varmentaminen

Solana tukee [varmentavia käännöksiä](/docs/programs/verified-builds), joiden
avulla käyttäjät voivat vahvistaa, että ohjelman ketjussa oleva tavukoodi vastaa
sen julkista lähdekoodia. Anchor-kehys tarjoaa
[sisäänrakennetun tuen](https://www.anchor-lang.com/docs/verifiable-builds)
varmentaville käännöksille.

Tarkistaaksesi käyttöönotetun ohjelman vahvistustilan, etsi sen ohjelmatunnus
[Solana Explorerista](https://explorer.solana.com/address/PhoeNiXZ8ByJGLkxNfZRnkUfjvmuYqLR89jjFHGqdXY)
tai käytä Ellipsis Labsin
[Solana Verifiable Build CLI](https://github.com/Ellipsis-Labs/solana-verifiable-build)
-työkalua ketjussa olevien ohjelmien itsenäiseen vahvistamiseen.
