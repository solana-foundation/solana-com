## Mitä ovat siirtomaksut?

Token Extensions Program -ohjelman _rs`TransferFeeConfig`_ mint-laajennus
soveltaa maksua jokaiseen siirtoon kyseiselle mint-tilille.

Siirron yhteydessä:

- Siirrettävää määrää vähennetään määritetyllä maksulla
- Pidätetty maksu seurataan kohteen token-tilillä
- Pidätetyt maksut voidaan nostaa suoraan token-tileiltä tai kerätä mint-tilille
- `withdraw_withheld_authority` voi siirtää kerätyt maksut maksujen
  vastaanottajan token-tilille

<Callout type="info" title="Token-tilin laajennukset">
  Kun token-tili alustetaan mint-tilille, jossa on _rs`TransferFeeConfig`_,
  token-tili alustetaan _rs`TransferFeeAmount`_ -laajennuksella. Kun token-tili
  luodaan [Associated Token
  Program](/docs/tokens/basics/create-token-account#how-to-create-an-associated-token-account)
  -ohjelman kautta, vaadittu tilin koko lasketaan ja token-tili luodaan
  vaaditulla koolla ja vuokravapaina lamporteina.
</Callout>

## Miten luodaan siirtomaksu-mint

Siirtomaksu-mintin luominen:

1. Laske mint-tilin koko ja tarvittava vuokra mint-tilille ja
   _rs`TransferFeeConfig`_ -laajennukselle.
2. Luo mint-tili käyttäen _rs`CreateAccount`_ -komentoa, alusta
   _rs`TransferFeeConfig`_ ja alusta mint käyttäen _rs`InitializeMint`_
   -komentoa.
3. Luo token-tilit mint-tilille. _rs`TransferFeeAmount`_ otetaan automaattisesti
   käyttöön token-tileille.
4. Käytä _rs`TransferCheckedWithFee`_ -komentoa tokenien siirtämiseen ja
   odotetun maksun tarkistamiseen. _rs`TransferChecked`_ toimii myös ja pidättää
   automaattisesti määritetyn maksun kohteen token-tilillä.
5. Käytä _rs`WithdrawWithheldTokensFromAccounts`_ -komentoa siirtääksesi
   pidätetyt maksut suoraan token-tileiltä maksujen vastaanottajan
   token-tilille, allekirjoitettuna mintin `withdraw_withheld_authority`-avaimen
   toimesta.
6. Käytä lupavapaata _rs`HarvestWithheldTokensToMint`_ -komentoa siirtääksesi
   pidätetyt maksut token-tileiltä mint-tilille, ja käytä sitten
   _rs`WithdrawWithheldTokensFromMint`_ -komentoa nostaaksesi mint-tilin
   pidätetyt maksut maksujen vastaanottajan token-tilille, allekirjoitettuna
   mintin `withdraw_withheld_authority`-avaimen toimesta.
7. Käytä _rs`SetTransferFee`_ -komentoa päivittääksesi seuraavan siirtomaksun
   asetukset, jotka astuvat voimaan kahden epocin kuluttua.

## !!steps Laske tilin koko

Laske mint accountin koko perus-mintille sekä _rs`TransferFeeConfig`_
-laajennukselle. Tämä on koko, jota käytetään _rs`CreateAccount`_ -toiminnossa.

<CodePlaceholder title="Esimerkki" />

```ts !! title="Example"
const client = await createLocalClient();
const mint = await generateKeyPairSigner();

// !focus(1:14)
const transferFeeConfigExtension = extension("TransferFeeConfig", {
  transferFeeConfigAuthority: client.payer.address,
  withdrawWithheldAuthority: client.payer.address,
  withheldAmount: 0n,
  olderTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  },
  newerTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  }
});

// !focus(1:1)

```
