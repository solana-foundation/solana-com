## Mitä Metadata Pointer- ja Token Metadata -laajennukset ovat?

Token Extensions Program -ohjelman _rs`MetadataPointer`_ mint-laajennus
tallentaa kaksi kenttää mintille:

- Valtuutus, joka voi päivittää osoittimen
- Tilin osoite, joka tallentaa tokenin metadatan

Osoitin voi viitata mihin tahansa tiliin, jonka omistaa ohjelma, joka toteuttaa
[token-metadata-rajapinnan](https://github.com/solana-program/token-metadata/tree/main/interface).

Token Extensions Program toteuttaa myös saman rajapinnan suoraan
_rs`TokenMetadata`_ mint-laajennuksen kautta. _rs`TokenMetadata`_-laajennuksella
mint tallentaa tokenin _rs`name`_-, _rs`symbol`_-, _rs`uri`_-,
_rs`update_authority`_-kentät ja mukautetun metadatan suoraan mint-tilille.

<Callout type="info" title="Off-chain-metadatan URI">
  _rs`uri`_-kenttä osoittaa off-chain JSON-metadataan. Katso
  [Off-chain-metadatan muoto](/docs/tokens/metaplex#off-chain-metadata-format).
</Callout>

Nämä kaksi laajennusta ratkaisevat eri ongelmia:

- _rs`MetadataPointer`_ määrittää tilin, joka tallentaa metadatan.
- _rs`TokenMetadata`_ tallentaa metadatan suoraan mint-tilille.

<Callout type="info" title="Muuttuvan pituuden laajennus">
  _rs`TokenMetadata`_ on muuttuvan pituuden TLV-laajennus.
  _rsINLINE_CODE_PLACEHOLDER_662394a59107c294_-,
  _rsINLINE_CODE_PLACEHOLDER_57ad0e221016adef_- ja
  _rsINLINE_CODE_PLACEHOLDER_491037e2b9d5e1ed_-toiminnot voivat muuttaa
  mint-tilin datan kokoa, mutta ne eivät siirrä lisää lamportteja. Mint-tilillä
  täytyy olla riittävästi lamportteja pysyäkseen vuokravapaa tallennettavalle
  metadatalle. Jos metadata kasvaa myöhemmin, lisää lamportteja täytyy siirtää
  mint-tilille ennen kokomuutosta suorittavaa käskyä.
</Callout>

## Token-metadatan tallentaminen mint-tilille

Tallentaaksesi metadatan mint-tilille:

1. Laske mint-tilin koko ja tarvittava vuokra mintille, laajennuksille ja
   metadatalle.
2. Luo mint-tili käyttäen _rs`CreateAccount`_, alusta _rs`MetadataPointer`_ ja
   alusta mint käyttäen _rs`InitializeMint`_.
3. Alusta _rs`TokenMetadata`_ mint-tilille ja käytä sitten _rs`UpdateField`_
   -toimintoa metadatan lisäämiseen tai päivittämiseen.
4. Käytä _rs`RemoveKey`_, _rs`UpdateAuthority`_ ja _rs`Emit`_ -toimintoja
   mukautetun metadatan poistamiseen, päivitysvaltuuden muuttamiseen tai
   tyhjentämiseen taikka nykyisen metadatan palauttamiseen transaktion
   paluudatassa.

## !!steps Laske tilin koko

Laske mint-tilin koko perus-mintille lisättynä _rs`MetadataPointer`_
-laajennuksella. Tätä kokoa käytetään _rs`CreateAccount`_ -toiminnossa.

<CodePlaceholder title="Esimerkki" />

```ts !! title="Example"
const client = await createLocalClient();
const mint = await generateKeyPairSigner();

// !focus(1:4)
const metadataPointerExtension = extension("MetadataPointer", {
  authority: client.payer.address,
  metadataAddress: mint.address
});

// !focus(1:1)
const mintSpace = BigInt(getMintSize([metadataPointerExtension]));
```

  updateAuthority: client.payer.address,

  additionalMetadata: new Map([
    ["description", "Metadata stored on mint account"]

## !!steps Päivitä, poista tai lähetä metatietoja

Kun mint on alustettu, käytä _rs`UpdateField`_ -toimintoa metatietojen
päivittämiseen, _rs`UpdateMetadataPointer`_ -toimintoa metatietojen osoittimen
päivittämiseen, _rs`RemoveKey`_ -toimintoa mukautettujen metatietojen
poistamiseen, _rs`UpdateAuthority`_ -toimintoa päivitysvaltuuden tyhjentämiseen
ja _rs`Emit`_ -toimintoa nykyisten metatietojen palauttamiseen.

<CodePlaceholder title="Esimerkki" />

```ts !! title="Example"
const client = await createLocalClient();
const mint = await generateKeyPairSigner();

const metadataPointerExtension = extension("MetadataPointer", {
  authority: client.payer.address,
  metadataAddress: mint.address
});

const mintSpace = BigInt(getMintSize([metadataPointerExtension]));

const maxTokenMetadataExtension = extension("TokenMetadata", {
  updateAuthority: client.payer.address,
  mint: mint.address,
  name: "Example Token v2",
  symbol: "EXMPL",
  uri: "https://example.com/token.json",
  additionalMetadata: new Map([
    ["description", "Metadata stored on mint account"]
  ])
});
const maxMintSpace = BigInt(
  getMintSize([metadataPointerExtension, maxTokenMetadataExtension])
);
const mintRent = await client.rpc
  .getMinimumBalanceForRentExemption(maxMintSpace)
  .send();

await client.sendTransaction([
  getCreateAccountInstruction({
    payer: client.payer, // Account funding account creation.
    newAccount: mint, // New mint account to create.
    lamports: mintRent, // Lamports funding the mint account rent.
    space: mintSpace, // Account size in bytes for the mint plus MetadataPointer.
    programAddress: TOKEN_2022_PROGRAM_ADDRESS // Program that owns the mint account.
  }),
  getInitializeMetadataPointerInstruction({
    mint: mint.address, // Mint account that stores the MetadataPointer extension.
    authority: client.payer.address, // Authority allowed to update the metadata pointer later.
    metadataAddress: mint.address // Account address that stores the metadata.
  }),
  getInitializeMintInstruction({
    mint: mint.address, // Mint account to initialize.
    decimals: 0, // Number of decimals for the token.
    mintAuthority: client.payer.address, // Authority allowed to mint new tokens.
    freezeAuthority: client.payer.address // Authority allowed to freeze token accounts.
  }),
  getInitializeTokenMetadataInstruction({
    metadata: mint.address, // Mint account that stores the metadata.
    updateAuthority: client.payer.address, // Authority allowed to update metadata later.
    mint: mint.address, // Mint that the metadata describes.
    mintAuthority: client.payer, // Signer authorizing metadata initialization for the mint.
    name: "Example Token", // Token name stored in metadata.
    symbol: "EXMPL", // Token symbol stored in metadata.
    uri: "https://example.com/token.json" // URI pointing to off-chain JSON metadata.
  }),
  getUpdateTokenMetadataFieldInstruction({
    metadata: mint.address, // Mint account that stores the metadata.
    updateAuthority: client.payer, // Signer authorized to update metadata fields.
    field: tokenMetadataField("Key", ["description"]), // Custom metadata field to add.
    value: "Metadata stored on mint account" // Value stored for the custom metadata field.
  })
]);

// !focus(1:31)
await client.sendTransaction([
  getUpdateTokenMetadataFieldInstruction({
    metadata: mint.address,
    updateAuthority: client.payer,
    field: tokenMetadataField("Name"),
    value: "Example Token v2"
  }),
  getUpdateMetadataPointerInstruction({
    mint: mint.address,
    metadataPointerAuthority: client.payer,
    metadataAddress: mint.address
  }),
  getRemoveTokenMetadataKeyInstruction({
    metadata: mint.address,
    updateAuthority: client.payer,
    key: "description"
  }),
  getUpdateTokenMetadataUpdateAuthorityInstruction({
    metadata: mint.address,
    updateAuthority: client.payer,
    newUpdateAuthority: null
  }),
  getEmitTokenMetadataInstruction({
    metadata: mint.address
  })
]);
```
