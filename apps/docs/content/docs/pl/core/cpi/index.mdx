Cross Program Invocation (CPI) to sytuacja, gdy jeden
[program](/docs/core/programs) wywołuje [instrukcję](/docs/core/instructions) w
innym programie podczas wykonywania. CPI umożliwiają kompozycyjność: instrukcje
dowolnego programu mogą być wywoływane przez każdy inny program w sieci.

## Najważniejsze informacje

- **Dwie funkcje**: _rs`invoke`_ (bez podpisu PDA) oraz _rs`invoke_signed`_ (z
  podpisem PDA).
- **Rozszerzanie uprawnień**: Uprawnienia [konta](/docs/core/accounts)
  (sygnatariusz, zapisywalność) są przekazywane od wywołującego do wywoływanego.
  Wywoływany nie może zwiększyć uprawnień ponad to, co przekazał wywołujący.
- **Wspólny budżet obliczeniowy**: Zużycie CU przez wywoływanego zmniejsza
  pozostały budżet wywołującego.
- **Reentrancy**: Dozwolona jest bezpośrednia rekurencja (A->A->A). Pośrednia
  rekurencja nie jest dozwolona (A->B->A zwraca _rs`ReentrancyNotAllowed`_).

## `invoke` kontra `invoke_signed`

Solana udostępnia dwie funkcje do wykonywania CPI:

| Funkcja         | Zastosowanie                                                                           | Podpisywanie przez PDA  |
| --------------- | -------------------------------------------------------------------------------------- | ----------------------- |
| `invoke`        | CPI, w których wszyscy wymagani sygnatariusze już podpisali oryginalną transakcję      | Nie                     |
| `invoke_signed` | CPI, w których wywołujący program musi podpisać w imieniu posiadanego przez siebie PDA | Tak, przez signer seeds |

Pod maską _rs`invoke`_ po prostu wywołuje _rs`invoke_signed`_ z pustą tablicą
signer seeds. Użyj _rs`invoke`_, gdy nie potrzebujesz podpisywania przez PDA, a
_rs`invoke_signed`_, gdy program musi autoryzować akcję w imieniu PDA.

Obie funkcje ostatecznie wywołują ten sam syscall
([`sol_invoke_signed_rust`](https://github.com/anza-xyz/agave/blob/v3.1.8/syscalls/src/cpi.rs#L11-L33))
i przechodzą przez tę samą ścieżkę wykonania w runtime
([`cpi_common`](https://github.com/anza-xyz/agave/blob/v3.1.8/program-runtime/src/cpi.rs#L802-L925)).
Jedyną różnicą jest to, czy signer seeds są przekazane. Gdy są, runtime
wyprowadza klucze publiczne PDA i dodaje je do zestawu ważnych sygnatariuszy
przed sprawdzeniem uprawnień.
