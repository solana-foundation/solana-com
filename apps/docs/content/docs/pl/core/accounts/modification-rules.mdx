<Callout type="info" title="Podsumowanie">
  Kontrole w czasie wykonywania po każdej instrukcji: tylko właściciel może
  obciążać konto lamportami lub modyfikować dane, dane mogą rosnąć maksymalnie o
  10 KiB na instrukcję, zmiana właściciela wymaga danych wyzerowanych, a flaga
  wykonywalności jest nieodwracalna.
</Callout>

Środowisko uruchomieniowe Solana egzekwuje te zasady po wykonaniu każdej
instrukcji za pomocą metod
[`BorrowedInstructionAccount`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L722-L727).
Każda zasada jest sprawdzana w momencie modyfikacji, a transakcja jest
wycofywana, jeśli którakolwiek kontrola się nie powiedzie.

## Zasady dotyczące lamportów

| Zasada                                                     | Egzekwowanie                                                                                                                                                                                                                                | Błąd                          |
| ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Tylko właściciel może obciążać konto lamportami            | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L785): sprawdza `is_owned_by_current_program()`, gdy `lamports < current`                                                              | `ExternalAccountLamportSpend` |
| Konta tylko do odczytu nie mogą mieć zmienianych lamportów | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L788-L789): sprawdza `is_writable()`                                                                                                        | `ReadonlyLamportChange`       |
| Każdy program może dodać lamporty do zapisywalnego konta   | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L789): kontrola właściciela dotyczy tylko sytuacji, gdy nowy bilans jest mniejszy niż obecny; kontrola zapisywalności nadal obowiązuje | `ReadonlyLamportChange`       |
| Bilans lamportów musi się zgadzać w ramach instrukcji      | [`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L366-L398): weryfikuje `get_lamports_delta() == 0`                                                                               | `UnbalancedInstruction`       |

## Przejścia stanów rent

Konta istnieją w jednym z trzech
[`RentState`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L19-L30)
stanów: _rs`Uninitialized`_ (0 lamportów), _rs`RentPaying`_ (powyżej 0, ale
poniżej minimum zwolnienia z rent), oraz _rs`RentExempt`_ (na poziomie minimum
lub powyżej). Niedozwolone przejścia powodują
_rs`TransactionError::InsufficientFundsForRent`_.

Środowisko wykonawcze egzekwuje te zasady przez
[`transition_allowed()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L107-L126):

- Każde konto może przejść do _rs`Uninitialized`_ (zamknięcie) lub
  _rs`RentExempt`_.
- Żadne konto nie może wejść w _rs`RentPaying`_ z _rs`Uninitialized`_ ani
  _rs`RentExempt`_. Wszystkie nowe konta muszą być zwolnione z rent.

## Zasady pożyczania konta

Podczas wykonywania instrukcji środowisko wykonawcze egzekwuje semantykę
pojedynczego zapisu na kontach. Program może uzyskać albo jedno mutowalne
odwołanie, albo wiele niemutowalnych odwołań do konta, ale nie oba jednocześnie.
Jeśli program spróbuje pożyczyć konto, które jest już mutowalnie pożyczone (lub
mutowalnie pożyczyć konto, które jest już niemutowalnie pożyczone), instrukcja
zakończy się niepowodzeniem z _rs`AccountBorrowFailed`_ przez
[`try_borrow()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L498-L508)
oraz
[`try_borrow_mut()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L512-L520).
Jeśli instrukcja zakończy się, gdy pożyczka nadal trwa, środowisko wykonawcze
zwraca _rs`AccountBorrowOutstanding`_ w
[`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L374-L385).
