Instrukcja to żądanie wykonania określonej funkcji w programie Solana.
Instrukcje są podstawowym elementem operacji on-chain. Każda instrukcja wskazuje
dokładnie jeden program do wywołania, wymagane konta oraz tablicę bajtów z
danymi, które program interpretuje (zwykle jest to dyskryminator do
identyfikacji obsługiwanej instrukcji, a następnie zserializowane argumenty).
Logika wykonania każdej instrukcji jest przechowywana w
[programie](/docs/core/programs), gdzie każdy program definiuje własny zestaw
instrukcji. Aby wejść w interakcję z siecią Solana, jedna lub więcej instrukcji
jest dodawana do [transakcji](/docs/core/transactions) i wysyłana do sieci do
przetworzenia.

## Najważniejsze informacje

- **Jeden program**: Każda instrukcja jest skierowana dokładnie do jednego
  programu przez `program_id`.
- **Metadane konta**: Tablica `accounts` zawiera
  [`AccountMeta`](/docs/core/instructions/instruction-structure#account-metadata)
  dla każdego konta, które instrukcja odczytuje lub zapisuje.
- **Nieprzezroczyste dane**: Pole `data` to tablica bajtów, której format jest
  definiowany przez docelowy program.
