[Codama](https://github.com/codama-idl/codama) generuje klientów TypeScript i
Rust na podstawie pliku IDL programu. Wygenerowani klienci zawierają funkcje do
wywoływania instrukcji i pobierania kont, eliminując potrzebę ręcznego
serializowania i deserializowania kont programu lub danych instrukcji.

<Callout type="warn">
  Codama jest w fazie aktywnego rozwoju i może ulec zmianie. Odwołaj się do
  podlinkowanych repozytoriów w sekcji szybkiego odniesienia, aby uzyskać
  najnowsze informacje.
</Callout>

## Szybkie odniesienie

### Polecenia CLI

| Polecenie     | Opis                                                           |
| ------------- | -------------------------------------------------------------- |
| `codama init` | Generuje plik konfiguracyjny codama                            |
| `codama run`  | Generuje klienty programów na podstawie pliku konfiguracyjnego |

### Generatory klientów

| Pakiet                                                                                                   | Opis                                                                                        |
| -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| [`@codama/renderers-js`](https://github.com/codama-idl/renderers-js)                                     | Generuje klienty TypeScript kompatybilne z [`@solana/kit`](https://www.solanakit.com)       |
| [`@codama/renderers-rust`](https://github.com/codama-idl/renderers-rust)                                 | Generuje klienty Rust kompatybilne z [`solana_sdk`](https://github.com/anza-xyz/solana-sdk) |
| [`@codama/nodes-from-anchor`](https://github.com/codama-idl/codama/tree/main/packages/nodes-from-anchor) | Konwertuje Anchor IDL na Codama IDL do generowania klienta                                  |

## !!steps Utwórz plik konfiguracyjny Codama

Codama używa pliku konfiguracyjnego do określenia, gdzie znajduje się plik IDL,
jakie generować języki klienckie oraz gdzie zapisać wygenerowany kod. Utwórz tę
konfigurację, uruchamiając:

```terminal
$ npx codama init
```

To polecenie wyświetli pytania, aby utworzyć plik `codama.json` w katalogu
głównym twojego projektu i zainstalować wymagane zależności Codama.

## !!steps Generowanie klientów

Po utworzeniu pliku konfiguracyjnego uruchom polecenie generowania, aby
wygenerować klientów:

```terminal
$ npx codama run --all
```

<WithMentions>

To wygeneruje klientów programu w [ścieżkach plików](mention:path) określonych w
pliku konfiguracyjnym Codama.

```json title="codama.json"
{
  "idl": "idl.json",
  "before": [],
  "scripts": {
    "js": {
      "from": "@codama/renderers-js",
      // !mark
      // !mention path
      "args": ["clients/js/src/generated"]
    },
    "rust": {
      "from": "@codama/renderers-rust",
      "args": [
        // !mark
        // !mention path
        "clients/rust/src/generated",
        {
          "crateFolder": "clients/rust",
          "formatCode": true
        }
      ]
    }
  }
}
```

</WithMentions>

<Callout type="info">
  Nie wszystkie wygenerowane pliki są pokazane w odpowiadających fragmentach
  kodu.
</Callout>


## Korzystanie z wygenerowanych klientów

Wygenerowani klienci zawierają kod źródłowy do komunikacji z twoim programem,
ale nie zawierają plików `package.json` (dla TypeScript) ani `Cargo.toml` (dla
Rust).

Musisz utworzyć nowe pliki `package.json` i `Cargo.toml` lub dodać wymagane
zależności z wygenerowanych klientów do istniejących plików w twoim projekcie.

### Zintegrowane z Twoją aplikacją

Wygeneruj klienta bezpośrednio do katalogu źródłowego twojej aplikacji, a
następnie dodaj wymagane zależności do istniejącego pliku `package.json` lub
`Cargo.toml`.

Użyj tego podejścia podczas tworzenia aplikacji, w której kod klienta nie musi
być udostępniany ani publikowany.

### Samodzielne pakiety lub crate'y

Utwórz osobne pakiety TypeScript lub crate’y Rust dla każdego klienta. Na
przykład, dodaj `package.json` w `clients/js/` oraz `Cargo.toml` w
`clients/rust/`.

Użyj tego podejścia, aby opublikować klienta jako bibliotekę wielokrotnego
użytku (npm lub crates.io) lub udostępnić klienta w wielu aplikacjach w
monorepozytorium.
