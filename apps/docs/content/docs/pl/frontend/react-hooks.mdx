`@solana/react` publikuje klienta Kit do drzewa komponentów przez jeden
dostawca, a następnie hooki odczytują dane RPC, subskrypcje i stan portfela.
Hooki przeglądarki dla portfela znajdują się w `@solana/kit-plugin-wallet/react`
i odczytują tego samego dostawcę — nie ma oddzielnego dostawcy portfela.

## Instalacja

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Zbuduj klienta

Zbuduj klienta raz, w zakresie modułu, poza drzewem React. W przeglądarce
`walletSigner` ustawia podłączony portfel zarówno jako płatnika opłat, jak i
tożsamość.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` dodaje `client.wallet` (wykrywanie i połączenie Wallet Standard),
a `solanaRpc` dodaje `client.rpc` oraz helper `client.sendTransaction`.
Eksportuj typ klienta opakowany w `Awaited<>`, aby pozostał poprawny, jeśli
później dodasz plugin rozwiązywany asynchronicznie, i przekaż ten `AppClient` do
każdego wywołania `useClient`.

## Opakuj drzewo raz

`ClientProvider` publikuje klienta. Każdy hook poniżej odczytuje dane od niego.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Hooki portfela

Hooki portfela odczytują możliwość `wallet` klienta, więc działają tylko
wewnątrz `ClientProvider`, którego klient zainstalował plugin portfela.

- `useWallets()` — portfele Wallet Standard wykryte dla łańcucha klienta.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` lub `null`. `signer`
  wynosi `null` dla portfeli tylko do odczytu.
- `useConnect()` / `useDisconnect()` — zwracają `ActionResult`, którego
  `dispatch` (oraz `dispatchAsync`) wykonuje akcję, a `isRunning`, `status`,
  `data`, `error` i `reset` śledzą jej wynik.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Wyślij transakcję

`useClient` zwraca tego samego klienta, którego opublikowałeś. Buduj instrukcje
za pomocą wygenerowanego klienta programu i wysyłaj je przez
`client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Przekazanie wyeksportowanego typu `AppClient` do `useClient` daje hookowi pełny
klient rozszerzony przez wtyczki, dzięki czemu `client.rpc` i
`client.sendTransaction` są typowane.

## Odczyt danych

`@solana/react` dostarcza hooki odczytujące dane przez klienta:

- `useRequest(source)` — jednorazowy odczyt RPC; zwraca
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — subskrypcja na żywo; zwraca
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — wartość zasilana przez RPC, odświeżana przez
  subskrypcję.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Memoizuj źródło (za pomocą `useMemo` lub `useCallback` dla funkcji), aby hook
ponownie pobierał dane tylko przy zmianie wejść, a nie przy każdym renderowaniu.

W celu buforowania i rewalidacji ścieżki `@solana/react/swr` i
`@solana/react/query` opakowują te same hooki dla [SWR](https://swr.vercel.app)
i [TanStack Query](https://tanstack.com/query).

## Typowe wzorce dla deweloperów Solana

- **Jeden provider**: `ClientProvider` to jedyny potrzebny provider; hooki
  portfela odczytują go przez możliwość `wallet` klienta.
- **Świadomość komponentów serwerowych**: Oznaczaj `"use client"` tylko liściowe
  komponenty wywołujące hooki; odczyty serwerowe mogą korzystać ze zwykłego
  klienta RPC Kit bez portfela.
- **Testowanie**: Przekaż zamockowanego klienta przez `ClientProvider`, aby
  symulować portfele i odpowiedzi RPC.

<Callout type="warn" title="Dwie rodziny hooków, te same nazwy">
  `useSignIn` i `useSignMessage` istnieją zarówno w `@solana/react` (przyjmują
  argument `UiWalletAccount`), jak i w `@solana/kit-plugin-wallet/react` (oparte
  na kliencie, bez argumentu konta, zwracają `ActionResult`). Używaj wersji
  `@solana/kit-plugin-wallet/react` ze wzorcem klienta opisanym powyżej; starsze
  hooki portfela `@solana/react` są zastępowane.
</Callout>

Połącz ten przewodnik z [przeglądem klienta Kit](/docs/frontend/client), aby
zrozumieć, z jakiego klienta korzysta każdy hook.
