## Przegląd

Po wdrożeniu programu do LiteSVM musisz wykonać instrukcje, aby wejść w
interakcję z wdrożonym programem.

LiteSVM udostępnia proste API do tworzenia, wysyłania i symulowania transakcji,
które jest w pełni zgodne z modelem transakcji Solany.

## Podstawowy przepływ transakcji

Typowy przepływ wykonywania instrukcji wygląda następująco:

1. **Utwórz instrukcję** – Określ, który program wywołać i z jakimi danymi
2. **Zbuduj wiadomość** – Połącz jedną lub więcej instrukcji
3. **Utwórz transakcję** – Podpisz wiadomość wymaganymi sygnatariuszami
4. **Wyślij lub zasymuluj** – Wykonaj transakcję i obsłuż wyniki

## Tworzenie instrukcji

### Podstawowa struktura instrukcji

```rust
use solana_instruction::{Instruction, AccountMeta};
use solana_pubkey::Pubkey;

let instruction = Instruction {
    program_id: Pubkey::new_unique(),  // The program to call
    accounts: vec![                     // Accounts the program needs
        AccountMeta::new(account_pubkey, false),        // Writable, not signer
        AccountMeta::new_readonly(readonly_pubkey, false), // Read-only, not signer
        AccountMeta::new(signer_pubkey, true),          // Writable, signer
    ],
    data: vec![0, 1, 2, 3],             // Instruction data (program-specific)
};
```

## Budowanie i wysyłanie transakcji

### Metoda 1: Podstawowa transakcja

```rust
use litesvm::LiteSVM;
use solana_keypair::Keypair;
use solana_message::Message;
use solana_transaction::Transaction;
use solana_signer::Signer;

let mut svm = LiteSVM::new();
let payer = Keypair::new();

// Airdrop SOL for fees
svm.airdrop(&payer.pubkey(), 1_000_000_000).unwrap();

// Create instruction
let instruction = /* your instruction */;

// Build message with payer
let message = Message::new(&[instruction], Some(&payer.pubkey()));

// Create and sign transaction
let tx = Transaction::new_signed_with_payer(
    &[instruction],                 // Your instructions
    Some(&payer.pubkey()),          // Who pays transaction fees
    &[&payer],                      // All required signers
    svm.latest_blockhash(),         // Recent blockhash
);

// Send transaction
let result = svm.send_transaction(tx);
```

### Metoda 2: Transakcja z niestandardową wiadomością

```rust
let payer = Keypair::new();

// Manually construct message
let message = Message::new(&[instruction], Some(&payer.pubkey()));

// Create transaction with signers, message, and blockhash
let tx = Transaction::new(
    &[&payer],
    message,
    svm.latest_blockhash(),
);

let result = svm.send_transaction(tx);
```

## Wersjonowane transakcje

LiteSVM obsługuje zarówno starsze, jak i wersjonowane transakcje:

```rust
use solana_transaction::versioned::VersionedTransaction;
use solana_message::VersionedMessage;

// Legacy transaction (most common)
let legacy_msg = Message::new(&[instruction], Some(&payer.pubkey()));
let versioned_tx = VersionedTransaction::try_new(
    VersionedMessage::Legacy(legacy_msg),
    &[&payer]
).unwrap();

// Send versioned transaction
let result = svm.send_transaction(versioned_tx);
```

## Wyniki transakcji

### Pomyślna transakcja

```rust
match svm.send_transaction(tx) {
    Ok(meta) => {
        println!("Signature: {}", meta.signature);
        println!("Compute units: {}", meta.compute_units_consumed);
        println!("Logs:");
        for log in &meta.logs {
            println!("  {}", log);
        }
    }
    Err(err) => {
        println!("Transaction failed: {:?}", err.err);
        // Logs are still available on failure
        println!("Failure logs: {:?}", err.meta.logs);
    }
}
```

### Pola metadanych transakcji

```rust
pub struct TransactionMetadata {
    pub signature: Signature,
    pub logs: Vec<String>,
    pub inner_instructions: InnerInstructionsList,
    pub compute_units_consumed: u64,
    pub return_data: TransactionReturnData,
    pub fee: u64,
}
```

## Symulowanie transakcji

Symulacja transakcji pozwala testować transakcje bez zmiany stanu:

```rust
// Simulate instead of sending
match svm.simulate_transaction(tx) {
    Ok(sim_result) => {
        println!("Simulation successful!");
        println!("Logs: {:?}", sim_result.meta.logs);
        println!("Compute units: {}", sim_result.meta.compute_units_consumed);
    }
    Err(err) => {
        println!("Simulation failed: {:?}", err.err);
    }
}
```

## Obsługa błędów

### Typowe błędy transakcji

```rust
use solana_transaction_error::TransactionError;
use solana_instruction::error::InstructionError;

match svm.send_transaction(tx) {
    Err(failed_tx) => {
        match failed_tx.err {
            TransactionError::InsufficientFundsForFee => {
                println!("Not enough SOL for fees");
            }
            TransactionError::InvalidProgramForExecution => {
                println!("Program doesn't exist or isn't executable");
            }
            TransactionError::InstructionError(index, err) => {
                println!("Instruction {} failed: {:?}", index, err);
                match err {
                    InstructionError::Custom(code) => {
                        println!("Custom error code: {}", code);
                    }
                    InstructionError::AccountNotFound => {
                        println!("An account doesn't exist");
                    }
                    _ => {}
                }
            }
            TransactionError::BlockhashNotFound => {
                println!("Blockhash expired or invalid");
            }
            _ => println!("Other error: {:?}", failed_tx.err),
        }
    }
    Ok(_) => {}
}
```

## Praca z logami programu

### Dostęp do logów

```rust
let result = svm.send_transaction(tx).unwrap();
```

## Konfiguracja budżetu obliczeniowego

### Ustawianie globalnego budżetu obliczeniowego

```rust
use solana_compute_budget::compute_budget::ComputeBudget;

let mut svm = LiteSVM::new()
    .with_compute_budget(ComputeBudget {
        compute_unit_limit: 200_000,
        ..Default::default()
    });
```

### Budżet obliczeniowy per transakcja

```rust
use solana_compute_budget_interface::ComputeBudgetInstruction;

let instructions = vec![
    // Set compute budget for this transaction
    ComputeBudgetInstruction::set_compute_unit_limit(400_000),
    ComputeBudgetInstruction::set_compute_unit_price(1),

    // Your actual instruction
    your_instruction,
];
```

## Instrukcje System Program

LiteSVM obsługuje System Program:

```rust
use solana_keypair::Keypair;
use solana_signer::Signer;
use solana_system_interface::instruction as system_instruction;
use solana_transaction::Transaction;

// Transfer SOL
let transfer_ix = system_instruction::transfer(
    &alice.pubkey(),
    &bob.pubkey(),
    1_000_000_000, // 1 SOL
);

// Create new account
let create_ix = system_instruction::create_account(
    &payer_pubkey,
    &new_account_pubkey,
    lamports,
    space as u64,
    &owner_program_id,
);
```

## Podsumowanie

Wykonywanie instrukcji w LiteSVM odbywa się zgodnie ze standardowym modelem
transakcji Solana:

1. Twórz obiekty `Instruction` z identyfikatorem programu, kontami i danymi
2. Zbuduj `Transaction` z instrukcją i wymaganymi sygnatariuszami
3. Użyj `send_transaction()` do wykonania lub `simulate_transaction()` do
   testowania
4. Obsługuj wyniki, sprawdzając `TransactionMetadata` lub szczegóły błędu

LiteSVM zapewnia natychmiastowe wykonanie ze szczegółowymi logami i informacjami
diagnostycznymi, co czyni go idealnym narzędziem do wydajnego testowania
programów Solana.
