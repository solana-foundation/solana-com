# LiteSVM dla TypeScript

Zestawy testów Solana uruchamiające `solana-test-validator` zajmują sekundy na
test — czekasz na uruchomienie procesu, połączenia RPC i podróże w obie strony
przez sieć. LiteSVM uruchamia te same testy w milisekundach. Żadnego
zewnętrznego procesu, żadnej sieci, wszystko w pamięci wewnątrz twojego
narzędzia do testowania.

<Callout type="info">
  **Testowano z:** `@solana/kit@6.8.0`, `@solana/kit-plugin-litesvm@0.10.0`,
  `@solana/kit-plugin-signer@0.10.0`, `@solana-program/system@0.12.0`,
  `@solana-program/token@0.13.0`. Te pakiety wtyczek są nowe i szybko się
  rozwijają — jeśli natkniesz się na niezgodność, sprawdź
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  w celu uzyskania dokładnych wersji, na których działają dokumentacja i zestaw
  testów.
</Callout>

Składasz klienta Kit z `@solana/kit`, instalując wtyczki za pomocą `.use()`:
wtyczkę podpisującą
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
wtyczkę LiteSVM
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm))
oraz dowolne potrzebne wtyczki programów. Wynik udostępnia
`client.sendTransaction()`, `client.rpc` i bezpośredni dostęp do stanu konta,
zegara i sysvarów — wszystko bez opuszczania twojego procesu.

### Utwórz swojego klienta

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` zwraca pustą powłokę — każdy `.use(plugin)` nakłada na nią
  kolejne możliwości. Wtyczka `signer()` musi być użyta **przed** `litesvm()`,
  ponieważ wtyczka transportowa potrzebuje płatnika do dołączenia.
  `signer(mySigner)` to skrót do zainstalowania tego samego keypair zarówno jako
  `payer`, jak i `identity` — użyj `payer()` lub `identity()` z tego samego
  pakietu, jeśli chcesz rozdzielić te role. Ta kolejność jest taka sama dla
  `solanaRpc()`, gdy korzystasz z prawdziwej sieci.
</Callout>

<Callout type="info">
  Wolisz konfigurację w jednej linii do szybkich eksperymentów? Pakiet
  `litesvm()` zawiera airdrop, więc możesz połączyć
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` z
  `@solana/kit-plugin-signer`, aby automatycznie wygenerować i zasilić płatnika.
</Callout>

</Step>

</Steps>

## Omówienie klienta

Po `.use(signer(...)).use(litesvm())` klient udostępnia:

| Właściwość                  | Opis                                                                                                                                                                                                     |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | `TransactionSigner` zainstalowany przez plugin `signer()` (ustawiony również jako `client.identity`)                                                                                                     |
| `client.identity`           | Sygnatariusz portfela będący właścicielem kont, tokenów i uprawnień                                                                                                                                      |
| `client.svm`                | Bezpośrednia instancja [`LiteSVM`](https://www.npmjs.com/package/litesvm) do zarządzania kontami, transakcjami i konfiguracją                                                                            |
| `client.rpc`                | Interfejs RPC zgodny z Kit (podzbiór: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Planowanie, podpisywanie i wysyłanie instrukcji jako transakcji                                                                                                                                          |
| `client.airdrop`            | Żądanie SOL z kranu LiteSVM                                                                                                                                                                              |
| `client.getMinimumBalance`  | Obliczanie minimalnej liczby lamport do zwolnienia z rent                                                                                                                                                |

### Korzystanie z client.svm

Bezpośredni dostęp do LiteSVM dla:

- Zarządzanie kontami (`setAccount`, `getAccount`, `airdrop`)
- Wykonywanie transakcji (`sendTransaction`, `simulateTransaction`)
- Konfiguracja (`withSigverify`, `withBlockhashCheck`, itd.)
- Manipulacja czasem (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### Korzystanie z client.rpc

Interfejs RPC zgodny z Kit do standardowych operacji:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  Warstwa RPC obsługuje wyłącznie **kodowanie base64**. Żądanie innych kodowań
  spowoduje błąd.
</Callout>

## Konfiguracja

Skonfiguruj LiteSVM za pomocą metod budowania na `client.svm`:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## Wysyłanie transakcji

Dodaj warstwy pluginów programu po transportcie — wieszają one konstruktory
instrukcji na kliencie, dzięki czemu nigdy nie dotykasz surowego
`TransactionMessage`:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## Przepływ SVM → RPC

Możesz ustawiać stan przez SVM i odczytywać go przez RPC:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```

odczytywanie stanu. Jeśli Twoje testy dotyczą tokenów SPL (minty, ATA,

pokazuje, jak plugin `tokenProgram()` redukuje ~40 linii konfiguracji do 3
wywołań. Pełne odniesienie do API znajdziesz w
[dokumentacji API](/docs/tools/litesvm/typescript/api-reference).
