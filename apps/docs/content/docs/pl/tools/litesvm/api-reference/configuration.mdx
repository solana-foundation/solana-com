title: Konfiguracja środowiska
description:
  Metody wzorca konstruktora do konfigurowania środowisk testowych LiteSVM
---

Wszystkie metody konfiguracyjne używają wzorca konstruktora z
`LiteSVM::default()`.

## with_compute_budget

```rust
pub fn with_compute_budget(self, compute_budget: ComputeBudget) -> Self
```

Ustaw limity jednostek obliczeniowych i rozmiar sterty.

```rust
use solana_compute_budget::compute_budget::ComputeBudget;

let mut svm = LiteSVM::default()
    .with_compute_budget(ComputeBudget {
        compute_unit_limit: 1_400_000,
        heap_size: 256 * 1024,
        ..ComputeBudget::default()
    });
```

## with_sigverify

```rust
pub fn with_sigverify(self, sigverify: bool) -> Self
```

Włącz lub wyłącz weryfikację podpisów. Domyślnie: `true`.

```rust
// Disable for faster tests
let mut svm = LiteSVM::default()
    .with_sigverify(false);
```

## with_blockhash_check

```rust
pub fn with_blockhash_check(self, check: bool) -> Self
```

Włącz lub wyłącz walidację skrótu bloku. Domyślnie: `true`.

```rust
// Disable to use any blockhash
let mut svm = LiteSVM::default()
    .with_blockhash_check(false);
```

## with_transaction_history

```rust
pub fn with_transaction_history(self, capacity: usize) -> Self
```

Ustaw liczbę transakcji do przechowywania w historii. Domyślnie: `0`.

```rust
// Keep last 100 transactions
let mut svm = LiteSVM::default()
    .with_transaction_history(100);
```

## with_log_bytes_limit

```rust
pub fn with_log_bytes_limit(self, limit: Option<usize>) -> Self
```

Ustaw maksymalną liczbę bajtów dla logów transakcji. Domyślnie: `Some(10_000)`.

```rust
// Unlimited logs
let mut svm = LiteSVM::default()
    .with_log_bytes_limit(None);

// Custom limit (50KB)
let mut svm = LiteSVM::default()
    .with_log_bytes_limit(Some(50_000));
```

## with_lamports

```rust
pub fn with_lamports(self, lamports: u64) -> Self
```

Ustaw początkowe lamport dla pobierania opłat. Domyślnie: `1_000_000_000`.

```rust
let mut svm = LiteSVM::default()
    .with_lamports(10_000_000_000);
```

## with_default_programs

```rust
pub fn with_default_programs(self) -> Self
```

Dołącz domyślne programy SPL (Token, Associated Token itp.).

```rust
let mut svm = LiteSVM::default()
    .with_default_programs();
```

## with_sysvars

```rust
pub fn with_sysvars(self) -> Self
```

Zawiera domyślne sysvary (Clock, Rent, EpochSchedule, SlotHashes, StakeHistory
itp.).

```rust
let mut svm = LiteSVM::default()
    .with_sysvars();
```

## with_feature_set

```rust
pub fn with_feature_set(self, feature_set: FeatureSet) -> Self
```

Konfiguruj zestaw funkcji Solana do testowania określonych funkcji środowiska
uruchomieniowego.

```rust
use agave_feature_set::FeatureSet;

let mut svm = LiteSVM::default()
    .with_feature_set(FeatureSet::all_enabled());
```

## with_precompiles

```rust
#[cfg(feature = "precompiles")]
pub fn with_precompiles(self) -> Self
```

Zawiera standardowe programy prekompilowane (ed25519, secp256k1, secp256r1).

```rust
let mut svm = LiteSVM::default()
    .with_precompiles();
```

```toml
litesvm = { version = "0.11", features = ["precompiles"] }
```

</Callout>

## with_feature_accounts

```rust
pub fn with_feature_accounts(self) -> Self
```

Dołącz konta bramkowane funkcjami do środowiska testowego. Dzięki temu
zachowanie środowiska uruchomieniowego Solana sterowane funkcjami jest dokładnie
odzwierciedlone w Twoich testach.

```rust
let mut svm = LiteSVM::default()
    .with_feature_accounts();
```

## Funkcja register-tracing

Funkcja `register-tracing` umożliwia szczegółowe śledzenie rejestrów SBF podczas
wykonywania transakcji. Implikuje funkcję `invocation-inspect-callback`.

```toml
[dev-dependencies]
litesvm = { version = "0.11", features = ["register-tracing"] }
```

Użyj `LiteSVM::new_debuggable(true)`, aby utworzyć instancję ze śledzeniem
rejestrów włączonym:

```rust
#[cfg(feature = "register-tracing")]
let mut svm = LiteSVM::new_debuggable(true);
```

Po włączeniu ustaw zmienną środowiskową `SBF_TRACE_DISASSEMBLE`, aby dodatkowo
zrzucić wyjście deasemblacji obok śladów rejestrów:

```bash
SBF_TRACE_DISASSEMBLE=1 cargo test
```

## set_invocation_inspect_callback

```rust
pub fn set_invocation_inspect_callback<C: InvocationInspectCallback + 'static>(&mut self, callback: C)
```

Rejestruje wywołanie zwrotne, które jest wywoływane przed i po każdym wywołaniu
programu. Wymaga funkcji `invocation-inspect-callback`.

```toml
[dev-dependencies]
litesvm = { version = "0.11", features = ["invocation-inspect-callback"] }
```

```rust
use litesvm::{LiteSVM, types::InvocationInspectCallback};
use solana_svm::invoke_context::InvokeContext;
use solana_svm_transaction::svm_message::SVMMessage;

struct MyInspector;

impl InvocationInspectCallback for MyInspector {
    fn before_invocation(
        &self,
        _svm: &LiteSVM,
        _tx: &impl SVMMessage,
        _program_indices: &[u16],
        _invoke_context: &InvokeContext,
    ) {
        println!("Before program invocation");
    }

    fn after_invocation(
        &self,
        _svm: &LiteSVM,
        _invoke_context: &InvokeContext,
        _enable_register_tracing: bool,
    ) {
        println!("After program invocation");
    }
}

let mut svm = LiteSVM::new();
svm.set_invocation_inspect_callback(MyInspector);
```
