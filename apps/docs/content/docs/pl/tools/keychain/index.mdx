**Solana Keychain zapewnia ujednolicony interfejs do podpisywania transakcji
Solana w wielu backendach zarządzania kluczami.** Twórz lokalnie z kluczem w
pamięci, a następnie przełącz się na enterprise key management, chmurowe HSM lub
zarządzane usługi portfeli w środowisku produkcyjnym — bez przepisywania kodu
podpisywania.

### Dlaczego Keychain?

- **Jednolity interfejs**: Jeden trait `SolanaSigner` działa we wszystkich
  backendach
- **Wymiana backendów**: Zmień dostawców zarządzania kluczami przez
  konfigurację, nie kod
- **Abstrakcja bez kosztów**: Flagi funkcji dołączają tylko to, czego
  potrzebujesz
- **Audytowany**: Przegląd bezpieczeństwa przeprowadzony przez
  [Accretion](https://accretion.xyz)

### Architektura

- **Języki**: Rust + TypeScript, w pełnej parytecie
- **Trait**: Ujednolicony interfejs `SolanaSigner`
- **Backendy**: Memory, Vault, AWS KMS, GCP KMS, Privy, Turnkey, Fireblocks,
  CDP, Crossmint, Dfns, Openfort, Para, Utila
- **Kompatybilność**: Zgodny z `@solana/kit` i `@solana/signers` (TypeScript) |
  Zgodny z `solana-sdk` i `solana-sdk-v3` (Rust)

### Obsługiwane backendy

Backendy obejmują lokalne środowiska deweloperskie, samodzielnie hostowane i
chmurowe zarządzanie kluczami oraz zarządzane usługi portfeli lub MPC. Zobacz
[Wybór backendu](/docs/tools/keychain/choosing-a-backend), aby porównać modele
custody i wybrać odpowiedni.

| Backend         | Przypadek użycia                 | Rust | TypeScript |
| --------------- | -------------------------------- | ---- | ---------- |
| Memory          | Programowanie, testowanie        | ✓    | ✓          |
| HashiCorp Vault | Samodzielne zarządzanie kluczami | ✓    | ✓          |
| AWS KMS         | Natywna chmura (AWS)             | ✓    | ✓          |
| GCP KMS         | Natywna chmura (GCP)             | ✓    | ✓          |
| Privy           | Wbudowane portfele               | ✓    | ✓          |
| Turnkey         | Zarządzanie kluczami bez custody | ✓    | ✓          |
| Fireblocks      | Custody instytucjonalny          | ✓    | ✓          |
| CDP             | Coinbase Developer Platform      | ✓    | ✓          |
| Crossmint       | Zarządzane portfele Crossmint    | ✓    | ✓          |
| Dfns            | Infrastruktura portfeli Dfns     | ✓    | ✓          |
| Openfort        | Portfele backendowe Openfort     | ✓    | ✓          |
| Para            | Portfele MPC Para                | ✓    | ✓          |
| Utila           | Custody MPC Utila                | ✓    | ✓          |

## Szybki start

### Rust

Zainstaluj pakiet Rust:

```bash
cargo add solana-keychain
```

Podstawowe użycie:

```rust
use solana_keychain::{Signer, SolanaSigner};

// Create a signer from any backend
let signer = Signer::from_memory("base58_private_key")?;

// All signers share the same interface
let pubkey = signer.pubkey();
let signature = signer.sign_transaction(&mut tx).await?;
```

- [Przewodnik Rust](/docs/tools/keychain/getting-started/rust) - Pełna
  instalacja i konfiguracja backendu
- [Crates.io](https://crates.io/crates/solana-keychain) - pakiet Rust

### TypeScript

Zainstaluj pakiet TypeScript:

```bash
pnpm add @solana/keychain
```

Podstawowe użycie:

```typescript
import { createKeychainSigner } from "@solana/keychain";
import { signTransactionWithSigners } from "@solana/signers";

// Create any signer via the unified factory
const signer = await createKeychainSigner({
  backend: "vault",
  vaultAddr: "https://vault.example.com:8200",
  vaultToken: "hvs.xxxxx",
  keyName: "my-solana-key",
  publicKey: "base58_public_key"
});

// Sign an already-compiled transaction
const signedTx = await signTransactionWithSigners(
  [signer],
  compiledTransaction
);
```

- [Przewodnik TypeScript](/docs/tools/keychain/getting-started/typescript) -
  `@solana/keychain` pakiety
- [Pakiet npm](https://www.npmjs.com/package/@solana/keychain) - pakiet npm
