Wybierz pakiety Commerce Kit, których potrzebujesz, w zależności od tego, co
budujesz. Każdy pakiet można zainstalować niezależnie i używać samodzielnie lub
w połączeniu z innymi.

## Wymagania systemowe

- **Node.js**: Wersja 18 lub wyższa (zalecana wersja LTS)
- **Menedżer pakietów**: pnpm, npm lub yarn
- **TypeScript**: Wersja 4.5+ (zalecany, ale niewymagany)

## Popularne zależności równorzędne

Solana Commerce Kit jest zbudowany w oparciu o bibliotekę Solana Kit, dzięki
czemu jest kompatybilny z Solana Kit, Gill, Solana Programs
[Clients](https://github.com/solana-program) oraz
[klientami JS wygenerowanymi przez Codama](https://github.com/codama-idl/renderers-js).

```bash
pnpm add gill # or @solana/kit
pnpm add @solana-program/token # or @solana-program/target-program
pnpm add @my-program/codama-js-client
```

## Konfiguracja TypeScript

Jeśli używasz TypeScript, upewnij się, że Twój plik `tsconfig.json` zawiera:

```json
{
  "compilerOptions": {
    "module": "ESNext",
    "target": "ES2020",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "jsx": "react-jsx",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  }
}
```

## Konfiguracja Next.js

Jeśli używasz Next.js, zaktualizuj plik `next.config.js`, aby transpilować
pakiety Commerce Kit:

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  transpilePackages: [
    "@solana-commerce/react",
    "@solana-commerce/connector",
    "@solana-commerce/headless",
    "@solana-commerce/solana-pay",
    "@solana-commerce/sdk"
  ]
};

export default nextConfig;
```
