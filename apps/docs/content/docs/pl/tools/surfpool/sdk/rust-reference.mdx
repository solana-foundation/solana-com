Skrzynka `surfpool-sdk` udostępnia niewielką powierzchnię skoncentrowaną wokół
struktury `Surfnet`. Ta strona zawiera listę wszystkich publicznych elementów;
opis użytkowania narracyjnego znajdziesz w przewodnikach zaczynając od
[Przeglądu](/docs/tools/surfpool/sdk/overview).

## Eksporty korzenia skrzynki

| Element               | Rodzaj     | Uwagi                                                                |
| --------------------- | ---------- | -------------------------------------------------------------------- |
| `Surfnet`             | struct     | Działająca instancja Surfnet — endpointy RPC/WS, płatnik, cheatcodes |
| `SurfnetBuilder`      | struct     | Płynny builder używany przez `Surfnet::builder()`                    |
| `Cheatcodes<'_>`      | struct     | Pożyczony pomocnik cheatcode zwracany przez `Surfnet::cheatcodes()`  |
| `SurfnetError`        | enum       | Wszystkie warianty błędów zwracane przez SDK                         |
| `SurfnetResult<T>`    | alias typu | `Result<T, SurfnetError>`                                            |
| `Keypair`             | re-eksport | `solana_keypair::Keypair`                                            |
| `Pubkey`              | re-eksport | `solana_pubkey::Pubkey`                                              |
| `Signer`              | re-eksport | `solana_signer::Signer`                                              |
| `RpcClient`           | re-eksport | `solana_rpc_client::rpc_client::RpcClient`                           |
| `BlockProductionMode` | re-eksport | `surfpool_types::BlockProductionMode`                                |
| `SimnetCommand`       | re-eksport | `surfpool_types::SimnetCommand`                                      |
| `SimnetEvent`         | re-eksport | `surfpool_types::SimnetEvent`                                        |
| `SvmFeatureConfig`    | re-eksport | `surfpool_types::SvmFeatureConfig`                                   |

## Surfnet

```rust
pub struct Surfnet { /* private */ }
```

### Konstruktory

```rust
impl Surfnet {
    pub async fn start() -> SurfnetResult<Self>;
    pub fn builder() -> SurfnetBuilder;
}
```

### Akcesory

```rust
impl Surfnet {
    pub fn rpc_url(&self) -> &str;
    pub fn ws_url(&self) -> &str;
    pub fn payer(&self) -> &Keypair;
    pub fn instance_id(&self) -> &str;
    pub fn rpc_client(&self) -> RpcClient;
    pub fn cheatcodes(&self) -> Cheatcodes<'_>;
    pub fn events(&self) -> &crossbeam_channel::Receiver<SimnetEvent>;
    pub fn send_command(&self, cmd: SimnetCommand) -> SurfnetResult<()>;
}
```

### Cykl życia

```rust
impl Surfnet {
    pub fn stop(&mut self) -> SurfnetResult<()>;
}

impl Drop for Surfnet { /* calls stop() if not already stopped */ }
```

`stop` czeka do pięciu sekund na zwolnienie portów przez środowisko
uruchomieniowe. Błąd podczas `Drop` jest cichy; błąd podczas jawnego wywołania
`stop` zwraca `SurfnetError::Runtime`.

## SurfnetBuilder


Każdy setter zwraca `Self`, dzięki czemu wywołania można łączyć w łańcuch.
Wywołaniem końcowym jest `start().await`.

```rust
    pub fn disable_feature(self, feature_id: Pubkey) -> Self;
```

## SurfnetError

```rust
#[derive(Debug)]
pub enum SurfnetError {
    PortAllocation(String),
    Startup(String),
    Runtime(String),
    Cheatcode(String),
    Aborted(String),
}
```

| Wariant          | Przyczyna                                                                        |
| ---------------- | -------------------------------------------------------------------------------- |
| `PortAllocation` | Nie można było przypisać wolnego portu dla RPC lub WebSocket.                    |
| `Startup`        | Inicjalizacja środowiska uruchomieniowego nie powiodła się.                      |
| `Runtime`        | Wątek środowiska uruchomieniowego uległ panice lub zakończył się nieoczekiwanie. |
| `Cheatcode`      | Wywołanie JSON-RPC cheatcode'u zwróciło błąd.                                    |
| `Aborted`        | Środowisko uruchomieniowe zostało wyłączone lub anulowane podczas uruchamiania.  |

Crate implementuje `std::error::Error` i `Display` na `SurfnetError`, dzięki
czemu działa z `?` i `anyhow` bez żadnej dodatkowej konfiguracji.
