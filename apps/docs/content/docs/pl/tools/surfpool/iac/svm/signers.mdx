Te moduły podpisujące są dostępne podczas korzystania z dodatku SVM do
podpisywania transakcji.

## secret_key

Moduł podpisujący `svm::secret_key` może być używany do synchronicznego
podpisywania transakcji przy użyciu klucza tajnego, mnemonika lub pliku keypair.

### Dane wejściowe

| Name              | Required | Type   | Description                                                 |
| ----------------- | -------- | ------ | ----------------------------------------------------------- |
| `secret_key`      | optional | string | Klucz tajny używany do podpisywania wiadomości i transakcji |
| `mnemonic`        | optional | string | Fraza mnemoniczna używana do wygenerowania klucza tajnego   |
| `derivation_path` | optional | string | Ścieżka derywacji używana do wygenerowania klucza tajnego   |
| `keypair_json`    | optional | string | Ścieżka do pliku keypair.json zawierającego klucz tajny     |

### Dane wyjściowe

| Name         | Type   | Description                      |
| ------------ | ------ | -------------------------------- |
| `public_key` | string | Klucz publiczny konta            |
| `address`    | string | Adres SVM (alias dla public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Możesz również użyć mnemonika:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Lub pliku keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

Moduł podpisujący `svm::web_wallet` umożliwi operatorowi Runbooka podpisanie
transakcji przy użyciu wybranego przez niego sygnatariusza przeglądarki.

### Dane wejściowe

| Name               | Required | Type   | Description                                                         |
| ------------------ | -------- | ------ | ------------------------------------------------------------------- |
| `expected_address` | optional | string | Oczekiwany adres SVM do połączenia (pomiń, aby zezwolić na dowolny) |

### Dane wyjściowe

| Name         | Type   | Description                       |
| ------------ | ------ | --------------------------------- |
| `address`    | string | Adres połączonego konta           |
| `public_key` | string | Klucz publiczny połączonego konta |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Ten podpisujący wyświetli użytkownikowi w interfejsie Web UI monit o:

1. Połączenie portfela
2. Podanie pubkey poprzez podpis wiadomości
3. Podpisanie wszystkich transakcji wymagających tego podpisującego

## squads

Podpisujący `svm::squads` może być używany do podpisywania transakcji za pomocą
multisig vault Squads.

### Dane wejściowe

| Nazwa     | Wymagane   | Typ    | Opis              |
| --------- | ---------- | ------ | ----------------- |
| `address` | opcjonalne | string | Adres vault squad |

### Dane wyjściowe

| Nazwa        | Typ    | Opis                             |
| ------------ | ------ | -------------------------------- |
| `public_key` | string | Klucz publiczny vault squad      |
| `address`    | string | Adres SVM (alias dla public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Użycie multisig Squads umożliwia bezpieczne podpisywanie transakcji przez wiele
stron bez ujawniania indywidualnych kluczy prywatnych.
