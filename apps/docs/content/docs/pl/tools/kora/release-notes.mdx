## Kora v2.2.0-beta.6

_23.03.2026_ |
[Wydanie na GitHubie](https://github.com/solana-foundation/kora/releases/tag/v2.2.0-beta.6)
| `cargo install kora-cli@2.2.0-beta.6`

Wersja przedpremierowa z obsługą pakietów Jito, nowymi metodami RPC oraz
rozszerzonymi funkcjami bezpieczeństwa.

- Dodano obsługę pakietów Jito z metodami `signBundle`, `signAndSendBundle` i
  `estimateBundleFee`
- Dodano metodę RPC `getVersion`
- Dodano ochronę opłacającego Lighthouse
- Dodano ochronę przed botami reCAPTCHA v3 dla punktów końcowych podpisywania
- Dodano limity użycia per portfel z obsługą Redis
- Dodano oficjalne obrazy Dockera na GHCR
- Dodano szczegółowe zasady limitów użycia
- SDK: `KoraClient` przyjmuje teraz obiekt opcji; nowy `koraPlugin` dla
  `@solana/kit`

Zobacz [Dokumentacja wersji beta](/docs/tools/kora/beta), aby poznać szczegóły.

## Kora TypeScript SDK v0.2.0

_09.03.2026_ |
[Wydanie na GitHubie](https://github.com/solana-foundation/kora/releases/tag/ts-sdk-v0.2.0)
| `pnpm add @solana/kora@0.2.0`

Architektura wtyczek klienta Kit do tworzenia komponowalnych transakcji bez
opłat za gaz.

- Dodano `createKitKoraClient()` z kompozycją wtyczek Kit (wtyczki planera,
  executora, płatnika)
- Umożliwia kompatybilność z wtyczkami programów Kit, takimi jak
  `tokenProgram()`
- Dodano obsługę Token-2022 przez opcję konfiguracyjną `tokenProgramId`
- Dodano obsługę instrukcji budżetu obliczeniowego z szacowaniem CU opartym na
  symulacji
- Dodano brakujące pole `signature` do odpowiedzi `signAndSendTransaction`
- Zwraca `NoopSigner` z `getPaymentInstruction` dla deduplikacji podpisujących
  Kit v6
- Zaktualizowano zależność peer `@solana/kit` do `^6.1.0`

Zobacz [Przewodnik po kliencie Kit](/docs/tools/kora/guides/kit-client).

## Kora v2.0.0

_2025-11-19_ |
[Wydanie na GitHubie](https://github.com/solana-foundation/kora/releases/tag/v2.0.0)
| `cargo install kora-cli@2.0.0`

Główne wydanie z nową architekturą.

- Wprowadzono nową architekturę dla węzła Kora
- Opublikowano TypeScript SDK `@solana/kora` v0.1.0
- Migracja do Solana SDK v3.0.x i aktualizacja wszystkich zależności
- Zastąpiono niestandardowy Kora Signer pakietem solana-signers
- Usunięto metody `sign_transaction_if_paid` i `get_all_signers`
- Zmieniono typy odpowiedzi: usunięto podpisy z `signAndSendTransaction` i
  `signTransaction`
- Ulepszone doświadczenie deweloperskie, bezpieczeństwo i wydajność
