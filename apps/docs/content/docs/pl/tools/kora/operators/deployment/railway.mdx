---
title: Wdrożenie na Railway
description: Wdrożenie na Railway jednym kliknięciem
---

[Railway](https://railway.com/) oferuje łatwe wdrożenie węzłów Kora z
automatycznym SSL, zarządzaniem domenami i wbudowanym monitoringiem. Ten
przewodnik przeprowadzi Cię przez konfigurację węzła Kora na Railway przy użyciu
prostego obrazu Dockera i interfejsu CLI Railway.

## Wymagania wstępne

- [Konto Railway](https://railway.app)
- Zainstalowane [Railway CLI](https://docs.railway.com/guides/cli)
- [Solana CLI](https://solana.com/docs/intro/installation) do tworzenia kluczy
  prywatnych
- [Punkt końcowy Solana RPC](https://solana.com/rpc)

## Krok 1: Przygotowanie plików wdrożeniowych

Utwórz nowy katalog dla swojego węzła Kora:

```bash
mkdir my-kora-node
cd my-kora-node
```

Dodaj przykładowe pliki do swojego katalogu:

- [kora.toml](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/kora.toml)
- [signers.toml](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/signers.toml)
- [Dockerfile](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/Dockerfile)

Twój katalog powinien teraz wyglądać następująco:

```bash
my-kora-node/
├── kora.toml
├── signers.toml
└── Dockerfile
```

## Krok 2: Wdrożenie na Railway

Z katalogu projektu zaloguj się do Railway

```bash
# Login to Railway
railway login
```

Postępuj zgodnie z instrukcjami wyświetlanymi na ekranie, aby zalogować się na
swoje konto.

Zainicjuj projekt Railway w swoim katalogu.

```bash
railway init
```

Railway wyświetli monit o:

1. Utworzenie nowego projektu lub wybór istniejącego
2. Wybranie nazwy projektu (np. "moj-wezel-kora")

### Wdróż swoją aplikację

```bash
# Deploy the application
railway up
```

To spowoduje:

- Przesłanie plików na Railway
- Zbudowanie obrazu Dockera
- Wdrożenie aplikacji

**Oczekiwany wynik:**

```
Build time: 121.81 seconds
Deploy complete
Starting Container
```

Świetnie! Wszystko jest gotowe! Prawdopodobnie zauważysz błąd w logach ("error:
a value is required for `--rpc-url <RPC_URL>` but none was supplied"). Dzieje
się tak, ponieważ nie skonfigurowaliśmy jeszcze zmiennych środowiskowych —
zajmijmy się tym teraz.

## Krok 3: Konfiguracja zmiennych środowiskowych

Otwórz swój panel Railway pod adresem [railway.app](https://railway.app) i
przejdź do swojego projektu. W projekcie kliknij "Settings" > "Variables"

![Zmienne środowiskowe Railway dla Kora](/assets/docs/tools/kora/deploy/railway-variables.jpg)

Dodaj następujące zmienne środowiskowe:

| Zmienna            | Wartość                               | Opis                                       |
| ------------------ | ------------------------------------- | ------------------------------------------ |
| `RPC_URL`          | `https://api.mainnet-beta.solana.com` | Punkt końcowy RPC Solana                   |
| `KORA_PRIVATE_KEY` | `your-base58-private-key`             | Twój klucz prywatny Solana (format base58) |
| `RUST_LOG`         | `info`                                | Poziom logowania do debugowania            |

> **Ostrzeżenie bezpieczeństwa**: Twój klucz prywatny ma dostęp do SOL na
> opłacanie prowizji. Użyj dedykowanej pary kluczy dla swojego węzła Kora i
> doładuj ją tylko taką ilością SOL, którą jesteś gotów wydać na opłaty
> transakcyjne. Więcej informacji o podpisujących znajdziesz w
> [Przewodniku po podpisujących](/docs/tools/kora/operators/signers).

### Pobieranie klucza prywatnego

Jeśli musisz utworzyć nową parę kluczy:

```bash
# Generate new keypair
solana-keygen new --outfile ~/.config/solana/kora-keypair.json
```

Upewnij się, że wpłacisz wystarczającą ilość SOL do testowania/obsługi swojego
węzła.

## Krok 4: Ponowne wdrożenie

Po zapisaniu zmiennych zostaniesz poproszony o ponowne wdrożenie serwera — zrób
to.

## Krok 5: Przetestuj wdrożenie

### Wygeneruj publiczny punkt końcowy

Aby wysyłać żądania do swojego punktu końcowego, musisz wygenerować publiczną
domenę. W swoim wdrożeniu przejdź do "Settings" > "Generate domain".

![Konfiguracja domeny Railway](/assets/docs/tools/kora/deploy/railway-domain.jpg)

Użyj domyślnego portu "8080" i kliknij "Generate Domain".

Powinieneś uzyskać domenę taką jak `https://my-kora-node.railway.app/`

### Testowanie endpointu konfiguracji

```bash
curl -X POST https://my-kora-node.railway.app \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"getConfig","params":[]}' | jq
```

**Oczekiwana odpowiedź:**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "fee_payer": "YourPublicKeyHere...",
    "validation_config": {
      "max_allowed_lamports": 1000000,
      "allowed_programs": [...],
      "allowed_spl_paid_tokens": [...]
      // ...
    }
  }
}
```

Świetna robota! Twój węzeł Kora jest teraz aktywny.
