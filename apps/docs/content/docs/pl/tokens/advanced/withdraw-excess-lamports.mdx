## Co robi wypłata nadwyżki lamportów?

_rs`WithdrawExcessLamports`_ przenosi SOL przechowywany przez konto Token
Program powyżej jego minimum zwolnienia z czynszu na konto docelowe. Salda
tokenów i podaż minta pozostają nienaruszone — przemieszczają się wyłącznie
nadwyżkowe lamporty.

Pozwala to odzyskać SOL, który wcześniej był zablokowany: lamporty wysłane do
token account, minta lub multisig (wszystkich należących do Token Program) nie
mogły być w inny sposób przenoszone.

## Konta

| Konto         | Opis                                                                |
| ------------- | ------------------------------------------------------------------- |
| `source`      | Zapisywalne. Token account, mint lub multisig, z którego wypłacamy. |
| `destination` | Zapisywalne. Otrzymuje nadwyżkowe lamporty.                         |
| `authority`   | Podpisuje na podstawie typu konta źródłowego (patrz poniżej).       |

## Kto może podpisać

Wymagany podpisujący zależy od typu konta `source`:

| Konto źródłowe                | Podpisujący                                                 |
| ----------------------------- | ----------------------------------------------------------- |
| Token account                 | Właściciel token account (lub jego sygnatariusze multisig). |
| Mint z uprawnieniem do minta  | Uprawnienie do minta.                                       |
| Mint bez uprawnienia do minta | Sam mint account musi podpisać (patrz uwaga poniżej).       |
| Multisig                      | Skonfigurowane sygnatariusze M-z-N.                         |

<Callout type="info">

Przypadek „mennica bez władzy mennicy“ jest rzadki. Gdy ma zastosowanie, konto
mennicy jest sygnatariuszem: mennica będąca własnością programu (poza krzywą)
podpisuje za pośrednictwem CPI ze swojego programu właściciela, a mennica
keypair (na krzywej) podpisuje swoim kluczem.

</Callout>

<Callout type="warn">

Natywne token account (opakowane SOL) nie są obsługiwane i zwracają
_rs`NativeNotSupported`_. Wypłata nie może obniżyć salda konta źródłowego
poniżej minimalnego progu zwolnienia z czynszu.

</Callout>

## Jak wypłacić nadwyżkę lamport

Przykład używa skonfigurowanego klienta
[`@solana/kit`](/docs/clients/official/javascript#solana-kit) — zobacz
[Transferowanie tokenów](/docs/tokens/basics/transfer-tokens), aby skonfigurować
klienta.

```ts title="Withdraw excess lamports (Kit)"
import { getWithdrawExcessLamportsInstruction } from "@solana-program/token";

// source/destination: addresses of the account to drain and the lamport recipient.
// authority: a TransactionSigner that signs per the source account type.
const instruction = getWithdrawExcessLamportsInstruction({
  source,
  destination,
  authority
});

await client.sendTransaction([instruction]);
```

W przypadku władzy multisig przekaż współsygnatariuszy za pomocą
_ts`multiSigners`_.
