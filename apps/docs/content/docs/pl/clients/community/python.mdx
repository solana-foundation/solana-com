## Solana Python SDK

Istnieją dwa pakiety, które są przydatne do programowania w Pythonie:

[`@kevinheavy/solana.py`](https://github.com/michaelhly/solana-py)

- Solana.py to podstawowa biblioteka Pythona do interakcji z Solana.
- Możesz jej używać do budowania transakcji i interakcji z
  [Solana JSON RPC API](/docs/rpc), podobnie jak z
  [solana-web3.js](https://github.com/solana-labs/solana-web3.js/).

- Obejmuje również [SPL Token Program](https://spl.solana.com/token).

- [Najnowsza dokumentacja](https://michaelhly.github.io/solana-py/).

[`@kevinheavey/solders`](https://github.com/kevinheavey/solders)

- `solders` to wydajny toolkit Pythona dla Solana, napisany w Rust. Zapewnia
  solidne rozwiązania w następujących obszarach:

- Podstawowa funkcjonalność SDK: keypairs, pubkeys, podpisywanie i serializacja
  transakcji.
- RPC: budowanie zapytań i parsowanie odpowiedzi (do obsługi sieci i
  asynchronicznych API zalecany jest
  [solana-py](https://michaelhly.github.io/solana-py/rpc/async_api/)).
- Testy integracyjne: moduł `solders.litesvm` to alternatywa dla
  `solana-test-validator`, która jest znacznie wygodniejsza i **dużo** szybsza.
  Bazuje na [solana-program-test](https://crates.io/crates/solana-program-test).

## Solana Client Python SDK

Jeśli chcesz wchodzić w interakcje z kontraktami Solana za pomocą Pythona,
możesz użyć klientów SDK, aby przekształcić kontrakty Solana napisane w Rust na
klientów Pythona, pod warunkiem, że posiadasz plik IDL kontraktu.

[`Codama-py`](https://www.npmjs.com/package/codama-py)

- Standaryzuj dowolny program kontraktu on-chain Solana do
  [`Codama IDL`](https://github.com/codama-idl/codama) i generuj interaktywnego
  klienta Pythona.
- Obsługuje zarówno Anchor, jak i programy niestandardowe.

[`Anchor-py`](https://github.com/kevinheavey/anchorpy)

- Anchor-py to klient Pythona dla programów Anchor. Może generować klientów z
  plików IDL Anchor i wchodzić w interakcje z programami Anchor.
