Szybka instalacja, która pozwala skonfigurować lokalne środowisko deweloperskie
Solana za pomocą jednego polecenia. Zainstaluj Rust, Solana CLI i Anchor
Framework na Windows (WSL), Linuksie i Macu. (Jeśli wolisz, możesz
[zainstalować każdą zależność osobno](/docs/intro/installation/dependencies).)

## Wymagania wstępne

<Accordions>
<Accordion title="Windows">

Aby tworzyć programy Solana na Windows, **musisz** użyć
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Po zainstalowaniu WSL wszystkie pozostałe zależności będą instalowane
przez terminal Linuksa.

Aby zainstalować WSL, uruchom poniższe polecenie w Windows PowerShell:

```terminal
$ wsl --install
```

W trakcie instalacji zostaniesz poproszony o utworzenie domyślnego konta
użytkownika.

![Instalacja WSL](/assets/docs/intro/installation/wsl-install.png)

Domyślnie WSL instaluje Ubuntu. Możesz otworzyć terminal Linuksa, wyszukując
"Ubuntu" na pasku wyszukiwania.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Jeśli Twój terminal Ubuntu wygląda jak na poniższym obrazku, możesz napotkać
problem, w którym `ctrl + v` (skrót klawiaturowy wklejania) nie działa w
terminalu.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Jeśli napotkasz ten problem, otwórz Windows Terminal, wyszukując "Terminal" na
pasku wyszukiwania.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Następnie zamknij Windows Terminal i ponownie otwórz terminal Linuksa,
wyszukując ponownie Ubuntu. Terminal powinien teraz wyglądać jak na poniższym
obrazku, gdzie `ctrl + v` (skrót klawiaturowy wklejania) działa.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Jeśli korzystasz z VS Code,
[rozszerzenie WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
umożliwia używanie WSL i VS Code razem.

![Konfiguracja WSL w VS Code](/assets/docs/intro/installation/wsl-vscode.png)

W pasku stanu VS Code zobaczysz następujący widok:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Gdy WSL jest już skonfigurowany, możesz przejść do:

- Wykonania [szybkiej instalacji](#installation) w terminalu Ubuntu (Linux)
- Zainstalowania [Rust](/docs/intro/installation/dependencies#install-rust),
  [Solana CLI](/docs/intro/installation/dependencies#install-solana-cli) oraz
  [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli) osobno,
  zgodnie z poniższym opisem

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Jeśli korzystasz z Debiana lub pochodnych, takich jak Ubuntu czy Mint, wykonaj poniższe kroki:

1. Zaktualizuj listy pakietów, aby upewnić się, że wszystkie pakiety są
   aktualne:

```terminal
$ sudo apt-get update
```

2. Zainstaluj następujące zależności:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Jeśli podczas instalacji `protobuf-compiler` pojawi się poniższy błąd,
upewnij się, że najpierw uruchomiłeś _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Jeśli korzystasz z dystrybucji RHEL, takich jak Fedora lub Red Hat Linux, wykonaj następujące kroki.

Zaktualizuj listy pakietów, aby pobrać nowsze wersje pakietów, uruchamiając
poniższe polecenie.

```terminal
$ sudo dnf upgrade --refresh
```

Następnie zainstaluj następujące zależności:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## Instalacja

1. Wpisz poniższe polecenie w terminalu, aby zainstalować wszystkie niezbędne
   zależności:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Pomyślna instalacja zwróci wynik podobny do poniższego:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. Zweryfikuj poprawność instalacji, sprawdzając wersję każdej zainstalowanej
   zależności.

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  Jeśli szybka instalacja się nie powiedzie, zapoznaj się z sekcją [Instalacja
  zależności](/docs/intro/installation/dependencies) poniżej, aby uzyskać
  instrukcje dotyczące instalacji każdej zależności osobno.
</Callout>
