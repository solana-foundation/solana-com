Ta sekcja zawiera kilka popularnych poleceń i przykładów, które pomogą Ci
rozpocząć pracę z [Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Inicjalizacja projektu

Utwórz nowy projekt Anchor, uruchamiając poniższe polecenie. Utworzy ono nowy
katalog o nazwie projektu i wykorzysta go do zainicjowania nowego projektu
Anchor.

```terminal
$ anchor init <project-name>
```

Na przykład poniższe polecenie utworzy projekt o nazwie `my-project`. Katalog
`my-project` będzie zawierał podstawowy program w Rust oraz szablon testów w
TypeScript.

```terminal
$ anchor init my-project
```

Następnie przejdź do katalogu projektu:

```terminal
$ cd <project-name>
```

Zobacz
[strukturę plików projektu](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
Anchor.

</Step>
<Step>

### Budowanie programu

Aby zbudować swój projekt, uruchom następujące polecenie:

```terminal
$ anchor build
```

Skompilowany program znajdziesz w katalogu `/target/deploy`.

<Callout type="warn">

Podczas uruchamiania `anchor build`, jeśli napotkasz następujące błędy:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Wypróbuj te rozwiązania:

1. Wymuś instalację, używając poniższego polecenia:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Jeśli powyższe rozwiązanie nie działa, wyczyść pamięć podręczną Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 wymaga `-Znext-lockfile-bump">
Możesz to naprawić, zmieniając pole version w pliku `Cargo.lock` z 4 na 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Zobacz [ten problem](https://github.com/solana-foundation/anchor/issues/3392),
aby uzyskać więcej informacji.

</Accordion>

</Accordions>

Po zastosowaniu powyższego rozwiązania spróbuj ponownie uruchomić
_shell`anchor build`_.

</Callout>

</Step>
<Step>

### Wdróż program

Aby wdrożyć swój projekt, uruchom następujące polecenie:

```terminal
$ anchor deploy
```

To polecenie wdraża Twój program do `cluster` określonego w pliku
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Przetestuj program

Aby przetestować swój projekt, uruchom następujące polecenie:

```terminal
$ anchor test
```

To polecenie buduje, wdraża i uruchamia testy Twojego projektu.

<Callout>
  Podczas korzystania z klastra `localnet` Anchor automatycznie uruchamia
  lokalnego validatora, wdraża program, uruchamia testy, a następnie zatrzymuje
  validatora.
</Callout>

<Callout type="warn">

Którykolwiek z poniższych błędów może oznaczać, że nie masz zainstalowanego
Node.js lub Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
