Możesz akceptować płatności w żądaniach API za pośrednictwem Pay.sh,
konfigurując serwer Pay. Zainstaluj Pay za pomocą:

```bash
brew install pay
```

lub

```bash
npm install -g @solana/pay
```

Zweryfikuj instalację za pomocą:

```bash
pay --version
```

Łańcuch narzędzi skonfiguruje Twój adres automatycznie. Uruchom poniższe
polecenie, aby skonfigurować i doładować swój portfel:

```bash
pay setup
```

## Konfiguracja serwera demonstracyjnego

Możesz rozpocząć akceptowanie płatności, uruchamiając swój serwer. Możesz
zobaczyć serwer demonstracyjny w działaniu za pomocą:

```bash
pay server demo
```

Wykonuje to dwie rzeczy. Uruchamia działający serwer Pay, który możesz odpytywać
za pomocą żądań HTTP, oraz generuje plik `pay-demo.yml` w bieżącym katalogu.

Wszystkie trasy API są konfigurowane za pośrednictwem konfiguracji YAML
`endpoints`. Aby akceptować płatności, musisz dodać atrybut `metering:`. Trasa
`/api/v1/reports/usage` została przygotowana jako przykład:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Możesz przetestować działanie płatności, uruchamiając następujące polecenie jako
klient w innym terminalu:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Dowiedz się więcej o definiowaniu cen w
[dokumentacji Pay.sh Building with Pay > Pricing](https://pay.sh/docs/building-with-pay/pricing)

Dowiedz się więcej o konfiguracji Pay.sh w środowisku produkcyjnym w
[dokumentacji konfiguracji Pay.sh](https://pay.sh/docs/toolchain/configuration).
