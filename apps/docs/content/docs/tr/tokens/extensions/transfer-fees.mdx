## Transfer Ücretleri Nedir?

Token Extensions Program'ın _rs`TransferFeeConfig`_ mint uzantısı, bu mint için
her transfere bir ücret uygular.

Transfer sırasında:

- Transfer edilen miktar, yapılandırılmış ücret kadar azaltılır
- Alıkonulan ücret, hedef token hesabında izlenir
- Alıkonulan ücretler doğrudan token hesaplarından çekilebilir veya mint'e
  toplanabilir
- `withdraw_withheld_authority`, toplanan ücretleri bir ücret alıcısı token
  hesabına taşıyabilir

<Callout type="info" title="Token Hesap Uzantıları">
  Bir token hesabı, _rs`TransferFeeConfig`_ içeren bir mint için
  başlatıldığında, token hesabı _rs`TransferFeeAmount`_ ile başlatılır. Token
  hesabı [Associated Token
  Program](/docs/tokens/basics/create-token-account#how-to-create-an-associated-token-account)
  üzerinden oluşturulduğunda, gerekli hesap boyutu hesaplanır ve token hesabı
  gerekli boyut ve rent muafiyeti lamportları ile oluşturulur.
</Callout>

## Transfer Ücreti Mint'i Nasıl Oluşturulur

Bir transfer ücreti mint'i oluşturmak için:

1. Mint hesabı boyutunu ve mint ile _rs`TransferFeeConfig`_ uzantısı için
   gereken rent'i hesaplayın.
2. _rs`CreateAccount`_ ile mint hesabını oluşturun, _rs`TransferFeeConfig`_
   uzantısını başlatın ve mint'i _rs`InitializeMint`_ ile başlatın.
3. Mint için token hesapları oluşturun. _rs`TransferFeeAmount`_, token hesapları
   için otomatik olarak etkinleştirilir.
4. Token transferi yapmak ve beklenen ücreti doğrulamak için
   _rs`TransferCheckedWithFee`_ kullanın. _rs`TransferChecked`_ de çalışır ve
   hedef token hesabında yapılandırılmış ücreti otomatik olarak alıkoyar.
5. Alıkonulan ücretleri token hesaplarından doğrudan bir ücret alıcısı token
   hesabına taşımak için _rs`WithdrawWithheldTokensFromAccounts`_ kullanın;
   mint'in `withdraw_withheld_authority` tarafından imzalanmalıdır.
6. Alıkonulan ücretleri token hesaplarından mint hesabına taşımak için izinsiz
   _rs`HarvestWithheldTokensToMint`_ kullanın, ardından mint hesabının
   alıkonulan ücretlerini bir ücret alıcısı token hesabına çekmek için
   _rs`WithdrawWithheldTokensFromMint`_ kullanın; mint'in
   `withdraw_withheld_authority` tarafından imzalanmalıdır.
7. İki epoch sonra yürürlüğe girecek bir sonraki transfer ücreti
   yapılandırmasını güncellemek için _rs`SetTransferFee`_ kullanın.

## !!steps Hesap boyutunu hesaplayın

Temel mint için mint hesap boyutunu _rs`TransferFeeConfig`_ uzantısıyla birlikte
hesaplayın. Bu, _rs`CreateAccount`_ içinde kullanılan boyuttur.

<CodePlaceholder title="Örnek" />

```ts !! title="Example"
const client = await createLocalClient();
const mint = await generateKeyPairSigner();

// !focus(1:14)
const transferFeeConfigExtension = extension("TransferFeeConfig", {
  transferFeeConfigAuthority: client.payer.address,
  withdrawWithheldAuthority: client.payer.address,
  withheldAmount: 0n,
  olderTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  },
  newerTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  }
});

// !focus(1:1)
const mintSpace = BigInt(getMintSize([transferFeeConfigExtension]));
```
