Token'lar, çeşitli varlık kategorileri üzerindeki mülkiyeti temsil eden dijital
varlıklardır. Tokenizasyon, mülkiyet haklarının dijitalleştirilmesini sağlar.
Solana'daki token'lara SPL
([Solana Program Library](https://github.com/solana-program)) token'ları denir.

Bu bölüm, token'ların Solana'da nasıl temsil edildiğine dair temel kavramları
kapsar. Kod örnekleri için [SPL token temelleri](/docs/tokens/basics) bölümüne
bakın.

## Önemli noktalar

- [Token Program'lar](#token-program), ağdaki token'larla etkileşim için tüm
  talimat mantığını içerir (hem değiştirilebilir hem de değiştirilemez).

- [Mint account](#mint-account), belirli bir token'ı temsil eder ve token
  hakkında toplam arz ve mint yetkisi (bir token'ın yeni birimlerini oluşturmaya
  yetkili adres) gibi global meta verileri saklar.

- [Token account](#token-account), belirli bir sahip için belirli bir mint
  account'a ait token'ların bireysel sahipliğini takip eder.

- [Associated token account](#associated-token-account), sahip ve mint account
  adreslerinden türetilen bir adresle oluşturulan bir token account'tır.

## Token Program'lar

Solana ekosisteminin iki ana Token Program'ı vardır. Her iki programın kaynak
kodu aşağıdadır.

<Cards>
  <Card title="Token Program (orijinal)" href="https://github.com/solana-program/token">
   - Temel token yetenekleri (mint, transfer vb.)
   - Değiştirilemez ve yaygın olarak kullanılır
  </Card>

  <Card title="Token Extension Program (Token 2022)" href="https://github.com/solana-program/token-2022">
   - Orijinal Token Program'ın tüm özelliklerini içerir
   - "Uzantılar" aracılığıyla özellikler ekler
  </Card>
</Cards>

Token Program'lar, ağdaki token'larla etkileşim için tüm talimat mantığını
içerir (hem değiştirilebilir hem de değiştirilemez). Solana'daki tüm token'lar
aslında bir Token Program'a ait
[veri hesaplarıdır](/docs/core/accounts#data-account).

![Token Program](/assets/docs/core/tokens/token-program.svg)

### Mint account

Solana'daki token'lar, Token Program'a ait bir
[mint account](https://github.com/solana-program/token/blob/6d18ff73b1dd30703a30b1ca941cb0f1d18c2b2a/program/src/state.rs#L16-L30)
adresine göre benzersiz şekilde tanımlanır. Bu hesap, belirli bir token için
global bir sayaç görevi görür ve şu gibi verileri saklar:

- **Supply**: Token'ın toplam arzı
- **Decimals**: Token'ın ondalık hassasiyeti
- **Mint authority**: Token'ın yeni birimlerini oluşturmaya yetkili hesap, arzı
  artırır
- **Freeze authority**: Token Account'taki tokenları dondurmaya yetkili hesap,
  transfer edilmelerini veya yakılmalarını engeller

![Mint Account](/assets/docs/core/tokens/mint-account.svg)

Her Mint Account'ta saklanan tam detaylar şunları içerir:

```rust title="Mint Account State"
pub struct Mint {
    /// Optional authority used to mint new tokens. The mint authority may only
    /// be provided during mint creation. If no mint authority is present
    /// then the mint has a fixed supply and no further tokens may be
    /// minted.
    pub mint_authority: COption<Pubkey>,
    /// Total supply of tokens.
    pub supply: u64,
    /// Number of base 10 digits to the right of the decimal place.
    pub decimals: u8,
    /// Is `true` if this structure has been initialized
    pub is_initialized: bool,
    /// Optional authority to freeze token accounts.
    pub freeze_authority: COption<Pubkey>,
}
```

Referans olarak, işte
[USDC Mint Account](https://explorer.solana.com/address/EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v)'a
ait Solana Explorer bağlantısı.

### Token Account

Token Program, her token biriminin bireysel sahipliğini takip etmek için
[Token Account'lar](https://github.com/solana-program/token/blob/6d18ff73b1dd30703a30b1ca941cb0f1d18c2b2a/program/src/state.rs#L87-L108)
oluşturur. Bir Token Account şu gibi verileri saklar:

- **Mint**: Token Account'ın birimlerini tuttuğu token
- **Owner**: Token Account'tan token transfer etmeye yetkili hesap
- **Amount**: Token Account'ın şu anda tuttuğu token sayısı

![Token Account](/assets/docs/core/tokens/token-account.svg)

Her Token Account'ta saklanan tam detaylar şunları içerir:

```rust title="Token Account State"
pub struct Account {
    /// The mint associated with this account
    pub mint: Pubkey,
    /// The owner of this account.
    pub owner: Pubkey,
    /// The amount of tokens this account holds.
    pub amount: u64,
    /// If `delegate` is `Some` then `delegated_amount` represents
    /// the amount authorized by the delegate
    pub delegate: COption<Pubkey>,
    /// The account's state
    pub state: AccountState,
    /// If is_native.is_some, this is a native token, and the value logs the
    /// rent-exempt reserve. An Account is required to be rent-exempt, so
    /// the value is used by the Processor to ensure that wrapped SOL
    /// accounts do not drop below this threshold.
    pub is_native: COption<u64>,
    /// The amount delegated
    pub delegated_amount: u64,
    /// Optional authority to close the account.
    pub close_authority: COption<Pubkey>,
}
```

Bir cüzdanın tutmak istediği her token (mint) için bir token hesabına ihtiyacı
vardır ve cüzdan adresi token hesabının sahibi olarak ayarlanır. Her cüzdan aynı
token (mint) için birden fazla token hesabına sahip olabilir, ancak bir token
hesabının yalnızca bir sahibi olabilir ve bir token'ın (mint) birimlerini
tutabilir.

![Hesap İlişkisi](/assets/docs/core/tokens/token-account-relationship.svg)

<Callout type="info">

Her Token Account'ın verisinin, Token Account üzerinde kimin yetkiye sahip
olduğunu belirten bir `owner` alanı içerdiğini unutmayın. Bu, tüm Token
Account'lar için Token Program olan temel
[Account](/docs/core/accounts#account-type) tipinde belirtilen program
sahibinden farklıdır.

</Callout>

### Associated Token Account

Associated Token Account'lar, belirli bir mint ve sahip için bir token account
adresini bulma sürecini basitleştirir. Associated Token Account'u, belirli bir
mint ve sahip için "varsayılan" token account olarak düşünün.

Bir Associated Token Account, sahibin adresinden ve mint account'un adresinden
türetilen bir adresle oluşturulur. Bir Associated Token Account'un, belirli bir
adrese sahip sadece bir token account olduğunu anlamak önemlidir.

Bu, Solana geliştirmede önemli bir kavramı ortaya koyar:
[Program Derived Address (PDA)](/docs/core/pda). Bir PDA, önceden tanımlanmış
girdileri kullanarak deterministik olarak bir adres türetir ve bu da bir hesabın
adresini bulmayı kolaylaştırır.

![Associated Token Account](/assets/docs/core/tokens/associated-token-account.svg)

Her cüzdanın aynı mint'ten token tutmak için kendi token account'una ihtiyacı
olduğunu unutmayın.

![Genişletilmiş Hesap İlişkileri](/assets/docs/core/tokens/token-account-relationship-ata.svg)

## Token CLI örnekleri

_sh`spl-token`_ CLI, SPL token'larıyla deney yapmanıza yardımcı olur. Aşağıdaki
örnekler, yerel kurulum gerektirmeden CLI komutlarını doğrudan tarayıcıda
çalıştırmak için [Solana Playground](https://beta.solpg.io/) terminalini
kullanır.

Token ve hesap oluşturmak, hesap rent depozitoları ve işlem ücretleri için SOL
gerektirir. İlk kez Solana Playground kullanan kullanıcılar için bir Playground
cüzdanı oluşturun ve Playground terminalinde _sh`solana airdrop`_ komutunu
çalıştırın. Ayrıca genel [web faucet](https://faucet.solana.com/)'ten devnet SOL
alabilirsiniz.

```terminal
$ solana airdrop 2
```

Tüm mevcut komutları görmek için _sh`spl-token --help`_ komutunu çalıştırın.

```terminal
$ spl-token --help
```

_sh`spl-token`_ CLI'yi yerel olarak kurmak için şunu çalıştırın:

```terminal
$ cargo install spl-token-cli
```

_sh`spl-token`_ [dokümantasyonunda](https://www.solana-program.com/docs/token)
daha fazla örnek bulabilirsiniz.

<Callout type="info">

Terminal çıktınızda gösterilen hesap adresleri aşağıdaki örneklerden farklıdır.
Takip ederken Playground terminalinizdeki adresleri kullanın. Örneğin,
_sh`create-token`_ komutu, Playground cüzdanınızı mint authority olarak içeren
bir mint account adresi çıktısı verir.

</Callout>

### Yeni bir token oluşturma

Yeni bir token (mint account) oluşturmak için şu komutu çalıştırın:

```terminal
$ spl-token create-token
```

Çıktı şöyle görünür:

```sh title="Terminal Output"
Creating token 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg

Address:  99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
Decimals:  9

Signature: 44fvKfT1ezBUwdzrCys3fvCdFxbLMnNvBstds76QZyE6cXag5NupBprSXwxPTzzjrC3cA6nvUZaLFTvmcKyzxrm1
```

Yeni bir mint account sıfır arz ile başlar. Mevcut arzı kontrol etmek için:

```terminal
$ spl-token supply <TOKEN_ADDRESS>
```

Yeni bir token için _sh`supply`_ komutu `0` döndürür:

```sh title="Example"
spl-token supply 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
```

Yeni bir Mint Account oluşturmak, iki talimat içeren bir işlem gerektirir. İşte
[Solana Playground](https://beta.solpg.io/660ce32ecffcf4b13384d00f) üzerinde bir
Javascript örneği.

1. System Program, Mint Account verisi için alan içeren yeni bir hesap oluşturur
   ve sahipliği Token Program'a aktarır.

2. Token Program, yeni hesabın verisini bir Mint Account olarak başlatır

### Token Account oluşturma

Belirli bir mint'in tokenlarını tutmak için bir token account oluşturun:

```terminal
$ spl-token create-account <TOKEN_ADDRESS>
```

Örnek komut:

```sh title="Example"
spl-token create-account 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
```

Çıktı:

```sh title="Terminal Output"
Creating account AfB7uwBEsGtrrBqPTVqEgzWed5XdYfM1psPNLmf7EeX9

Signature: 2BtrynuCLX9CNofFiaw6Yzbx6hit66pup9Sk7aFjwU2NEbFz7NCHD9w9sWhrCfEd73XveAGK1DxFpJoQZPXU9tS1
```

_sh`create-account`_ komutu, cüzdan adresinizi sahip olarak kullanarak bir
associated token account oluşturur.

Farklı bir sahiple token account oluşturmak için:

```terminal
$ spl-token create-account --owner <OWNER_ADDRESS> <TOKEN_ADDRESS>
```

Not: `<TOKEN_ADDRESS>`, token account için mint account'u belirtir.

Örnek komut:

```sh title="Example"
spl-token create-account --owner 2i3KvjDCZWxBsqcxBHpdEaZYQwQSYE6LXUMx5VjY5XrR 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
```

Çıktı:

```shell title="Terminal Output" /Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt/
Creating account Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt

Signature: 44vqKdfzspT592REDPY4goaRJH3uJ3Ce13G4BCuUHg35dVUbHuGTHvqn4ZjYF9BGe9QrjMfe9GmuLkQhSZCBQuEt
```

Bir associated token account oluşturmak,
[Associated Token Program](https://github.com/solana-program/associated-token-account)'ı
çağıran tek bir talimat gerektirir. İşte
[Solana Playground](https://beta.solpg.io/660ce868cffcf4b13384d011)'daki bir
Javascript örneği.

Associated Token Program, şunlar için
[Cross Program Invocations](/docs/core/cpi) kullanır:

- Sağlanan PDA'yı adres olarak kullanarak yeni bir hesap oluşturmak için
  [System Program'ı çağırır](https://github.com/solana-program/associated-token-account/blob/9d94201e8158f06015ff80ad47fefac62a2ec450/program/src/tools/account.rs#L19)
- Token Account verisini başlatmak için
  [Token Program'ı çağırır](https://github.com/solana-program/associated-token-account/blob/9d94201e8158f06015ff80ad47fefac62a2ec450/program/src/processor.rs#L138-L161)

Bir associated token account adresi yerine yeni bir keypair ile yeni bir token
account oluşturmak için, iki talimatlı bir işlem gönderin. İşte
[Solana Playground](https://beta.solpg.io/660ce716cffcf4b13384d010)'daki bir
Javascript örneği.

1. System Program, token account verisi için alan içeren yeni bir hesap
   oluşturur ve sahipliği Token Program'a aktarır.

2. Token Program, veriyi bir token account olarak başlatır

### Token basma

Bir tokenin yeni birimlerini oluşturmak için, bir token account'a token basın:

```terminal
$ spl-token mint [OPTIONS] <TOKEN_ADDRESS> <TOKEN_AMOUNT> [--] [RECIPIENT_TOKEN_ACCOUNT_ADDRESS]
```

Örnek komut:

```sh title="Example"
spl-token mint 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg 100
```

Çıktı:

```shell title="Terminal Output"
Minting 100 tokens
  Token: 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
  Recipient: AfB7uwBEsGtrrBqPTVqEgzWed5XdYfM1psPNLmf7EeX9

Signature: 2NJ1m7qCraPSBAVxbr2ssmWZmBU9Jc8pDtJAnyZsZJRcaYCYMqq1oRY1gqA4ddQno3g3xcnny5fzr1dvsnFKMEqG
```

Farklı bir token account'a token basmak için:

```sh title="Example"
spl-token mint 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg 100 -- Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt
```

Çıktı:

```shell title="Terminal Output"
Minting 100 tokens
  Token: 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
  Recipient: Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt

Signature: 3SQvNM3o9DsTiLwcEkSPT1Edr14RgE2wC54TEjonEP2swyVCp2jPWYWdD6RwXUGpvDNUkKWzVBZVFShn5yntxVd7
```

Token Program üzerindeki _rs`MintTo`_ talimatı yeni tokenler oluşturur. Mint
yetkisi işlemi imzalamalıdır. Talimat, bir token account'a token basar ve Mint
Account üzerindeki toplam arzı artırır. İşte
[Solana Playground](https://beta.solpg.io/660cea45cffcf4b13384d012)'daki bir
Javascript örneği.

### Token transferi

Token account'lar arasında token transfer etmek için:

```sh title="Terminal"
spl-token transfer [OPTIONS] <TOKEN_ADDRESS> <TOKEN_AMOUNT> <RECIPIENT_ADDRESS or RECIPIENT_TOKEN_ACCOUNT_ADDRESS>
```

Örnek komut:

```sh title="Example"
spl-token transfer 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg 100 Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt
```

Çıktı:

```shell title="Terminal Output"
Transfer 100 tokens
  Sender: AfB7uwBEsGtrrBqPTVqEgzWed5XdYfM1psPNLmf7EeX9
  Recipient: Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt

Signature: 5y6HVwV8V2hHGLTVmTmdySRiEUCZnWmkasAvJ7J6m7JR46obbGKCBqUFgLpZu5zQGwM4Xy6GZ4M5LKd1h6Padx3o
```

Token Program'daki _rs`Transfer`_ talimatı token transferlerini yönetir.
Gönderenin token hesabının sahibi işlemi imzalamalıdır. Talimat, tokenları token
hesapları arasında taşır. İşte
[Solana Playground](https://beta.solpg.io/660ced84cffcf4b13384d013) üzerinde bir
Javascript örneği.

Hem gönderenin hem de alıcının belirli token (mint) için token hesaplarına
ihtiyacı vardır. Gönderen, alıcının token hesabını oluşturmak için talimatları
aynı işleme dahil edebilir.

### Token metadata'sı oluşturma

Token Extensions Program, metadata'yı (ad, sembol, görsel bağlantısı) doğrudan
mint hesabında saklamanıza olanak tanır.

Metadata uzantısı ile token oluşturmak için:

```sh title="Example"
spl-token create-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb --enable-metadata
```

Çıktı:

```shell title="Terminal Output"
Creating token BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP under program TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb
To initialize metadata inside the mint, please run `spl-token initialize-metadata BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP <YOUR_TOKEN_NAME> <YOUR_TOKEN_SYMBOL> <YOUR_TOKEN_URI>`, and sign with the mint authority.

Address:  BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP
Decimals:  9

Signature: 5iQofFeXdYhMi9uTzZghcq8stAaa6CY6saUwcdnELST13eNSifiuLbvR5DnRt311frkCTUh5oecj8YEvZSB3wfai
```

Metadata'yı başlatmak için:

```sh title="Terminal"
spl-token initialize-metadata <TOKEN_MINT_ADDRESS> <YOUR_TOKEN_NAME> <YOUR_TOKEN_SYMBOL> <YOUR_TOKEN_URI>
```

Token URI'si, zincir dışı metadata'ya bağlantı verir. Örnek bir JSON formatını
[buradan](https://raw.githubusercontent.com/solana-developers/opos-asset/main/assets/DeveloperPortal/metadata.json)
görebilirsiniz.

Örnek komut:

```sh title="Example"
spl-token initialize-metadata BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP "TokenName" "TokenSymbol" "https://raw.githubusercontent.com/solana-developers/opos-asset/main/assets/DeveloperPortal/metadata.json"
```

Metadata'yı
[Solana Explorer](https://explorer.solana.com/address/BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP?cluster=devnet)
üzerinde görüntüleyin.

Daha fazla bilgi için
[Metadata Extension rehberini](/developers/guides/token-extensions/metadata-pointer)
inceleyin. Token Extensions hakkında ayrıntılı bilgi için Token Extensions
[başlangıç rehberine](/developers/guides/token-extensions/getting-started) ve
[SPL belgelerine](https://www.solana-program.com/docs/token-2022/extensions)
bakın.
