`@solana/react`, bir Kit istemcisini bileşen ağacınıza tek bir sağlayıcı
aracılığıyla yayınlar; ardından kancalar RPC verilerini, abonelikleri ve cüzdan
durumunu buradan okur. Tarayıcı cüzdan kancaları
`@solana/kit-plugin-wallet/react` içinde yer alır ve aynı sağlayıcıyı okur —
ayrı bir cüzdan sağlayıcısı yoktur.

## Kurulum

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## İstemciyi oluşturun

İstemciyi, React ağacının dışında, modül kapsamında bir kez oluşturun.
Tarayıcıda, `walletSigner` bağlı cüzdanı hem ücret ödeyici hem de kimlik olarak
ayarlar.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner`, `client.wallet` (Wallet Standard keşfi ve bağlantısı) ekler;
`solanaRpc` ise `client.rpc` ve `client.sendTransaction` yardımcısını ekler.
İstemci türünü `Awaited<>` ile sararak dışa aktarın; böylece daha sonra asenkron
çözümlenen bir eklenti ekleseniz bile tür doğru kalır. Bu `AppClient` değerini
her `useClient` çağrısına iletin.

## Ağacınızı bir kez sarın

`ClientProvider` istemciyi yayınlar. Aşağıdaki her kanca buradan okur.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Cüzdan kancaları

Cüzdan kancaları, istemcinin `wallet` yeteneğini okur; bu nedenle yalnızca
cüzdan eklentisi yüklenmiş bir istemciye sahip `ClientProvider` içinde
çalışırlar.

- `useWallets()` — İstemcinin zinciri için keşfedilen Wallet Standard
  cüzdanları.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` veya `null`. Salt
  okunur cüzdanlar için `signer`, `null` değerindedir.
- `useConnect()` / `useDisconnect()` — `dispatch` (ve `dispatchAsync`) işlemi
  çalıştıran, `isRunning`, `status`, `data`, `error` ve `reset` alanlarıyla
  sonucu izleyen bir `ActionResult` döndürür.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## İşlem gönderin

`useClient`, yayınladığınız istemciyi döndürür. Oluşturulan bir program
istemcisiyle talimatları derleyin ve `client.sendTransaction` aracılığıyla
gönderin.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Dışa aktarılan `AppClient` türünü `useClient`'e geçirmek, hook'a tam eklenti
genişletmeli istemciyi sağlar; böylece `client.rpc` ve `client.sendTransaction`
türlendirilmiş olur.

## Veri okuma

`@solana/react`, istemci üzerinden okuma yapan hook'lar içerir:

- `useRequest(source)` — tek seferlik RPC okuması;
  `{ data, error, status, refresh }` döndürür.
- `useSubscription(source)` — canlı abonelik;
  `{ data, error, reconnect, status }` döndürür.
- `useTrackedData(spec)` — bir abonelik tarafından güncel tutulan, RPC ile
  başlatılmış değer.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Kaynağı memoize edin (`useMemo` ile veya bir fonksiyon için `useCallback` ile);
böylece hook her render'da değil, yalnızca girdileri değiştiğinde yeniden veri
çeker.

Önbellekleme ve yeniden doğrulama için `@solana/react/swr` ve
`@solana/react/query` alt yolları, [SWR](https://swr.vercel.app) ve
[TanStack Query](https://tanstack.com/query) için aynı hook'ları sarar.

## Solana geliştiricileri için yaygın desenler

- **Tek provider**: `ClientProvider`, ihtiyaç duyduğunuz tek provider'dır;
  cüzdan hook'ları buna istemcinin `wallet` özelliği aracılığıyla erişir.
- **Sunucu bileşeni desteği**: Yalnızca hook çağıran yaprak bileşenleri
  `"use client"` ile işaretleyin; sunucu okumaları cüzdan olmaksızın düz bir Kit
  RPC istemcisi kullanabilir.
- **Test**: Cüzdanları ve RPC yanıtlarını simüle etmek için `ClientProvider`
  aracılığıyla mock'lanmış bir istemci yayımlayın.

<Callout type="warn" title="İki hook ailesi, aynı isimler">
  `useSignIn` ve `useSignMessage` hem `@solana/react`'de (bir `UiWalletAccount`
  argümanı alır) hem de `@solana/kit-plugin-wallet/react`'de (istemci tabanlı,
  hesap argümanı yok, `ActionResult` döndürür) bulunur. Yukarıdaki istemci
  deseniyle `@solana/kit-plugin-wallet/react` sürümlerini kullanın; eski
  `@solana/react` cüzdan hook'larının yerini bunlar almaktadır.
</Callout>

Her hook'un hangi istemciden okuduğunu anlamak için bu kılavuzu
[Kit istemciye genel bakış](/docs/frontend/client) ile birlikte inceleyin.
