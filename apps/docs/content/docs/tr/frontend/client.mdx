  RPC, imzalama ve işlem gönderme için Kit eklentilerinden bir Solana istemcisi

`@solana/kit`, eklentilerle genişlettiğiniz küçük bir konak olan
`createClient`'i içerir. Her eklenti bir yetenek ekler — RPC bağlantısı,
imzalayıcı, işlem planlama — ve oluşturulan istemci; betikleri, sunucu
rotalarını ve kullanıcı arayüzlerini eşit biçimde destekler.

<Callout type="info">
  Bir React uygulaması mı geliştiriyorsunuz? [React
  rehberi](/docs/frontend/react-hooks), bu istemciyi `@solana/react` ile sararak
  bileşenlerin cüzdan durumunu ve RPC verilerini hook'lar aracılığıyla okumasını
  sağlar. Oradan başlayın ve daha ince kontrole ihtiyaç duyduğunuzda buraya
  dönün.
</Callout>

## Kurulum

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

Herhangi bir paket yöneticisi kullanabilirsiniz. Kit v7+ sürümündedir ve
`@solana/kit-plugin-*` paketleri 0.13+ sürümündedir.

## İstemci oluşturma

Eklentileri `.use(...)` ile birleştirin. Önce imzalayıcıyı, ardından RPC
bağlantısını ayarlayın. `solanaDevnetRpc()`; `client.rpc`, abonelikler, işlem
planlayıcısı/yürütücüsü ve `client.sendTransaction` yardımcısını ekler — bu
sayede bir uygulama bu parçaları nadiren kendisi birleştirmek zorunda kalır.

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

`signer` eklentisi hem ücret ödeyiciyi hem de uygulama kimliğini ayarlar. RPC
paketi, önce bir ödeyicinin tanımlanmasını gerektirdiğinden sıra önem taşır.

### İmzalayıcı varyantları

`@solana/kit-plugin-signer`, bir istemcinin yalnızca ihtiyaç duyduğu yetkiyi
vermek için üç rol sunar:

- `signer(x)` — hem `client.payer` hem de `client.identity`'i ayarlar.
- `payer(x)` — yalnızca ücret ödeyici.
- `identity(x)` — yalnızca uygulama yetkisi.

Her birinin testler ve betikler için `*FromFile(path)` (Node), `generated*()`,
`generated*WithSol(amount)` ve `airdrop*()` varyantları mevcuttur.

## İşlem gönderme

Oluşturulmuş bir program istemcisiyle talimatlar oluşturun, ardından bunları
`client.sendTransaction`'e iletin. Planlayıcı blok hash'ini çözümler,
hesaplamayı tahmin eder, istemcinin ödeyicisiyle imzalar ve gönderir.

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

`sendTransaction` bir sonuç nesnesine dönüşür; işlem imzası
`result.context.signature` konumundadır. Planın tek bir işleme sığdığını
doğrular ve aksi hâlde hata fırlatır — bir toplu iş birden fazla işleme
yayılabilecekse `sendTransactions` kullanın.

## Bir RPC paketi seçin

`@solana/kit-plugin-rpc` küme ön ayarlarıyla birlikte gelir:

- `solanaDevnetRpc()` — devnet varsayılanları, airdrop etkin (`client.airdrop`).
- `solanaMainnetRpc()` — mainnet; airdrop derleme zamanı hatasıdır.
- `solanaLocalRpc()` — yerel validator için `http://127.0.0.1:8899`.
- `solanaRpc({ rpcUrl })` — herhangi bir uç nokta.

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### Ayrıntılı düzeyde oluşturun

Ön ayarlar; RPC bağlantısını, planlayıcıyı, yürütücüyü ve gönderme
yardımcılarını bir araya getirir; bu nedenle çoğu uygulama parçalara ayrı ayrı
ihtiyaç duymaz. Yalnızca birini özelleştirmek istediğinizde — örneğin özel bir
planlayıcı veya farklı bir abonelik kanalı için — ayrıntılı eklentilere geçin.
Bu yol aynı zamanda `@solana/kit-plugin-instruction-plan` gerektirir; ön ayarlar
bunu sizin için otomatik olarak içerir:

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## Program istemcileri ekleyin

Oluşturulan `@solana-program/*` paketleri, program çağrılarının istemci
üzerinden gerçekleştirilebilmesi için istemci eklentileri sunar. İhtiyacınız
olan programı yükleyin ve eklentisini `.use()` ile ekleyin:

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="Eş bağımlılık aralıkları">
  Bazı `@solana-program/*` sürümleri, Kit 7.x sürümündeyken hâlâ `@solana/kit
  ^6.x` eş bağımlılığı bildirir. Kit 7 ile birlikte çalışırlar — paket
  yöneticiniz şikâyet ederse eş bağımlılık aralığını geçersiz kılın.
</Callout>

## Tarayıcı cüzdanı olmadan imzalama

Bir API rotasında, worker'da veya betikte başsız (headless) olarak mı
çalışıyorsunuz? Ham keypair yerine bir arka uç anahtar yönetim hizmetiyle
imzalayın. Bkz. [Keychain](/docs/tools/keychain) ve
[Üretimde İmzalama](/docs/core/transactions/signing-in-production).

## Solana geliştiricileri için yaygın desenler

- **Paylaşılan kurulum**: İstemciyi bir çekirdek modülde tek seferinde oluşturun
  ve betiklere, sunucu rotalarına ve kullanıcı arayüzünüze aktarın.
- **Sunucu taraflı okumalar**: Yalnızca `signer` + `solanaRpc` içeren bir
  istemci (cüzdan eklentisi olmadan) sunucudan hesapları okur ve işlemleri
  simüle eder.
- **Yerel test**: RPC paketini `@solana/kit-plugin-litesvm` içindeki
  `.use(litesvm())` ile değiştirerek bellek içi validator'a karşı çalıştırın
  (yalnızca Node).
- **Test edilebilirlik**: Oluşturulan istemci kolayca taklit edilebilir —
  tarayıcı cüzdanı olmadan RPC yanıtlarını veya imzalayıcıları taklit edin.
