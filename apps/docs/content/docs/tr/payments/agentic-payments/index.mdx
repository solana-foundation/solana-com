AI aracıları iş akışlarında giderek daha özerk hale geliyor: kod yazıyor, veri
çekiyor, kaynak sağlıyor ve daha fazlasını yapıyor. Tamamen bağımsız çalışmak
için, hizmetler için programatik olarak, gerçek zamanlı ve minimum maliyetle
"mikro ödemeler" yapabilecek bir yönteme ihtiyaçları var. Geleneksel ödeme
altyapıları mikro ödemeler için işe yaramıyor. Solana'nın sent altı ücretleri ve
saniye altı kesinliği bunu teknik ve ekonomik olarak uygulanabilir kılıyor.

## Bu neden önemli?

Örneğin, bir uygulama geliştirmek için LLM kullanan bir geliştiricinin ücretli
bir API'den veriye ihtiyacı olduğunu düşünün. Önemli bir veriyi çekmek için
geliştiricinin iş akışını durdurması, API'leri araştırması, bir sağlayıcı
seçmesi, hesap oluşturması, ödeme yöntemi eklemesi, API anahtarı oluşturması,
anahtarı ortamına kopyalaması ve ardından iş akışına devam etmesi gerekir. Bu
sürtünme akış durumunu bozar. AI destekli iş akışları için bir engeldir.

Bunun yerine, aracı ödemelerle, aracı isteği yapabilir, ödemesini
gerçekleştirebilir ve iş akışına devam edebilir. Hesap yok. API anahtarı yok.
Manuel müdahale yok.

### pay.sh ile API'ler için ödeme yapın

[pay.sh](https://pay.sh), HTTP ajanları ve komut satırı araçları için bir ödeme
katmanıdır. `pay`'yi kurmak için [pay.sh dokümantasyonuna](https://pay.sh/docs)
bakın. `pay` CLI, `curl`, `codex` ve `claude` gibi araçları sarmalayarak
sağlayıcı hesapları veya API anahtarları olmadan ücretli API'leri çağırmalarını
sağlar. Bir API, MPP veya x402 `402 Payment Required` sorgusu döndürdüğünde,
`pay` yerel cüzdanınızdan imzalamayı onaylamanızı ister.

Pay'i şu şekilde kurabilirsiniz:

```bash
brew install pay
```

veya

```bash
npm install -g @solana/pay
```

Kurulumu şu komutla doğrulayın:

```bash
pay --version
```

Araç zinciri adresinizi otomatik olarak ayarlayacaktır. Cüzdanınızı kurmak ve
bakiye yüklemek için aşağıdaki komutu çalıştırın:

```bash
pay setup
```

## x402 Protokolü

Agentic ödemeler, istemciler ve sunucuların geleneksel web arayüzlerini bozmadan
standart web altyapısı üzerinden ödeme koşullarını müzakere etmesine olanak
tanıyan bir yönteme ihtiyaç duyar. Mikro ödeme alanı henüz gelişmekte olup, alan
evrimleştikçe yeni araçlar ve standartlar ortaya çıkması beklenmektedir.
[x402 protokolü](https://www.x402.org/) ise güçlü ekosistem desteğiyle erken bir
standart olarak öne çıkmıştır. x402,
[HTTP'nin 402 "Ödeme Gerekli"](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status/402)
durum kodunu kullanır; bu kod HTTP/1.1'den bu yana bir standart olmakla birlikte
blokzincir tabanlı mutabakat ile pratik hale gelmiştir.

### Nasıl çalışır

İstemci bir istek gönderir, ödeme koşullarıyla birlikte 402 yanıtı alır,
ardından imzalı bir ödemeyle isteği yeniler. Sunucu, doğrulama ve mutabakatı;
zincir üzeri işlem gönderimini yöneten isteğe bağlı bir aracı olan
kolaylaştırıcıya devreder. Kolaylaştırıcı mutabakatı onayladığında, sunucu
istenen içeriği döndürür. Bu ayrım, API sağlayıcılarının doğrudan herhangi bir
zincir üzeri altyapı yönetmeksizin ödeme kabul etmesine olanak tanır.

![x402 Akış Diyagramı](/assets/guides/x402/x402-flow-solana.png)

Protokol durumsuz bir yapıya sahiptir (oturum veya API anahtarı gerektirmez),
standart HTTP altyapısıyla uyumludur ve sunucunun kabul ettiği her SPL token'ı
destekler. x402 ara yazılımını kullanan bir Express sunucusunun nasıl
kurulacağına dair örnek bir kod parçası:

```ts
app.use(
  paymentMiddleware(
    {
      "GET /costly-data": {
        accepts: [
          {
            scheme: "exact",
            price: "$0.001",
            network: "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
            payTo: svmAddress
          }
        ],
        description: "Costly data",
        mimeType: "application/json"
      }
    },
    new x402ResourceServer(facilitatorClient).register(
      "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
      new ExactSvmScheme()
    )
  )
);

app.get("/costly-data", (req, res) => {
  res.send({
    report: {
      data: "costly data"
    }
  });
});
```
