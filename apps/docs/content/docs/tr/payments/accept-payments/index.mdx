Solana'da hazır bileşenler, Solana Pay protokolü veya doğrudan RPC entegrasyonu
kullanarak ödeme kabul edin.

## Alıcı adres kuralı

Solana'da **token hesabınızı değil, cüzdan adresinizi paylaşın**. Gönderenin,
gönderdikleri token için uygun
[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)
adresini türetmesi tipiktir (daha fazla ayrıntı için
[Ödeme gönderme](/docs/payments/send-payments) bölümüne bakın).

**Örnek:** `7EcDhS...` (cüzdanınız) adresini paylaşın, `9xKpQ...` (USDC token
hesabınız) adresini değil.

<Callout>
  Satış noktası ve e-ticaret için QR kodları ve ödeme URL'leri için [Solana
  Pay](/docs/payments/accept-payments/solana-pay) bölümüne veya hazır React
  bileşenleri için [Payment
  Button](/docs/payments/accept-payments/payment-button) bölümüne bakın.
</Callout>

## Entegrasyon seçenekleri

| Yaklaşım                                                        | En uygun olduğu durumlar              | Karmaşıklık |
| --------------------------------------------------------------- | ------------------------------------- | ----------- |
| [Payment Button](/docs/payments/accept-payments/payment-button) | React uygulamaları, hızlı entegrasyon | Düşük       |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | QR kodları, ödeme bağlantıları, POS   | Orta        |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Özel akışlar, tam kontrol             | Yüksek      |

## Doğrulama

Bir ödeme aldıktan sonra,
[Verification Tools](/docs/payments/accept-payments/verification-tools)
kullanarak ödemenin ulaştığını doğrulayın. Yüksek hacimli uygulamalar için,
üretim düzeyinde işlem izleme için
[Indexing](/docs/payments/accept-payments/indexing) bölümüne bakın.
