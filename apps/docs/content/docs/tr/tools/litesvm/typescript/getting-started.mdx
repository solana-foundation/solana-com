# TypeScript için LiteSVM

`solana-test-validator` başlatan Solana test paketleri, test başına saniyeler
alır — süreç başlatma, RPC bağlantıları ve ağ gidiş-dönüşlerini bekliyorsunuz.
LiteSVM aynı testleri milisaniyeler içinde çalıştırır. Harici süreç yok, ağ yok,
her şey test çalıştırıcınızın içinde bellekte.

<Callout type="info">
  **Test edilen sürümler:** `@solana/kit@6.8.0`,
  `@solana/kit-plugin-litesvm@0.10.0`, `@solana/kit-plugin-signer@0.10.0`,
  `@solana-program/system@0.12.0`, `@solana-program/token@0.13.0`. Bu eklenti
  paketleri yeni ve hızla gelişiyor — bir uyumsuzlukla karşılaşırsanız,
  dokümanların ve test paketinin çalıştığı tam sürümler için
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  adresini inceleyin.
</Callout>

`@solana/kit` ile bir Kit istemcisi oluşturmak için `.use()` ile eklentiler
yüklersiniz: bir imzalayan eklentisi
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
LiteSVM eklentisi
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm))
ve ihtiyaç duyduğunuz program eklentileri. Sonuç; `client.sendTransaction()`,
`client.rpc` ve sürecinizden hiç çıkmadan hesap durumuna, saate ve sysvarlara
doğrudan erişimi ortaya koyar.

### İstemcinizi Oluşturun

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` boş bir iskelet döndürür — her `.use(plugin)` üzerine
  yetenekler ekler. `signer()` eklentisi, `litesvm()`'den **önce** gelmelidir
  çünkü taşıma eklentisinin bağlanacak bir ödeyiciye ihtiyacı vardır.
  `signer(mySigner)`, aynı keypair'i hem `payer` hem de `identity` olarak
  yüklemek için bir kısayoldur — bu rolleri ayırmak istiyorsanız aynı paketten
  `payer()` veya `identity()` kullanın. Bu sıralama, gerçek bir ağda
  `solanaRpc()` için de aynıdır.
</Callout>

<Callout type="info">
  Hızlı denemeler için tek satırlık bir kurulum mu tercih ediyorsunuz?
  `litesvm()` paketi bir airdrop içerir; bu sayede bir ödeyiciyi otomatik
  oluşturup fonlamak için `@solana/kit-plugin-signer` paketinden
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` zincirini
  kullanabilirsiniz.
</Callout>

</Step>

</Steps>

## İstemciyi Anlamak

`.use(signer(...)).use(litesvm())` sonrasında istemci şunları sunar:

| Özellik                     | Açıklama                                                                                                                                                                                             |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | `signer()` eklentisiyle kurduğunuz `TransactionSigner` (aynı zamanda `client.identity` olarak da ayarlanır)                                                                                          |
| `client.identity`           | Hesaplara, tokenlara ve yetkililere sahip olan cüzdan imzalayıcısı                                                                                                                                   |
| `client.svm`                | Hesap yönetimi, işlemler ve yapılandırma için doğrudan [`LiteSVM`](https://www.npmjs.com/package/litesvm) örneği                                                                                     |
| `client.rpc`                | Kit uyumlu RPC arayüzü (alt küme: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Talimatları bir işlem olarak planla, imzala ve gönder                                                                                                                                                |
| `client.airdrop`            | LiteSVM faucet'inden SOL talep et                                                                                                                                                                    |
| `client.getMinimumBalance`  | rent muafiyeti için minimum lamport hesapla                                                                                                                                                          |

### client.svm Kullanımı

LiteSVM'e doğrudan erişim şunlar için:

- Hesap yönetimi (`setAccount`, `getAccount`, `airdrop`)
- İşlem yürütme (`sendTransaction`, `simulateTransaction`)
- Yapılandırma (`withSigverify`, `withBlockhashCheck`, vb.)
- Zaman manipülasyonu (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### client.rpc Kullanımı

Standart işlemler için Kit uyumlu RPC arayüzü:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  RPC katmanı yalnızca **base64 kodlamasını** destekler. Başka kodlamalar talep
  edilirse hata fırlatılır.
</Callout>

## Yapılandırma

`client.svm` üzerindeki builder metodlarını kullanarak LiteSVM'i yapılandırın:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## İşlem Gönderme

Taşıma katmanından sonra program eklentilerini ekleyin — bu eklentiler talimat
bildirgelerini istemciye bağlar, böylece ham `TransactionMessage` ile hiç
uğraşmazsınız:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## SVM → RPC Akışı

Durumu SVM aracılığıyla ayarlayabilir ve RPC aracılığıyla okuyabilirsiniz:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```

## Sırada Ne Var

Bu bölüm, hesap oluşturma, talimat gönderme ve durum okuma gibi temel istemci
işlevlerini kapsar. Testleriniz SPL tokenlarını (mint'ler, ATA'lar, transferler)
içeriyorsa, [token test kılavuzu](/docs/tools/litesvm/typescript/tokens)
`tokenProgram()` eklentisinin yaklaşık 40 satırlık kurulumu nasıl 3 çağrıya
indirdiğini gösterir. Tam API referansı için
[API belgelerine](/docs/tools/litesvm/typescript/api-reference) bakın.
