## send_http_request

`std::send_http_request`, belirtilen URL'ye bir HTTP isteği gönderir ve yanıtı
dışa aktarır.

### Girdiler

| Ad               | Zorunlu      | Tür     | Açıklama                                                         |
| ---------------- | ------------ | ------- | ---------------------------------------------------------------- |
| `url`            | zorunlu      | string  | İstek için URL. Desteklenen şemalar: http ve https               |
| `body`           | isteğe bağlı | string  | Dize veya JSON nesnesi olarak istek gövdesi                      |
| `method`         | isteğe bağlı | string  | İstek için HTTP Yöntemi. İzin verilen yöntemler: GET, HEAD, POST |
| `timeout_ms`     | isteğe bağlı | integer | Milisaniye cinsinden istek zaman aşımı                           |
| `headers`        | isteğe bağlı | object  | İstek başlığı alan adları ve değerlerinin haritası               |
| `pre_condition`  | isteğe bağlı | map     | Bir komut yürütülmeden önce değerlendirilen ön koşullar          |
| `post_condition` | isteğe bağlı | map     | Bir komut yürütüldükten sonra değerlendirilen son koşullar       |

### Ön Koşullar

Ön koşullar, bir komut yürütülmeden önce değerlendirilen onaylardır. Komutun
yürütülüp yürütülmeyeceğini veya onayın sonucuna göre belirli bir davranışın
uygulanıp uygulanmayacağını belirlemek için kullanılabilirler.

| Özellik     | Açıklama                                                                                                             |
| ----------- | -------------------------------------------------------------------------------------------------------------------- |
| `behavior`  | Ön koşul onayı başarısız olduğunda gerçekleşecek davranış. Seçenekler: `halt` (varsayılan), `log`, `skip`            |
| `assertion` | Kontrol edilecek onay. Boolean olarak değerlendirilmeli veya `std::assert_eq` ve benzeri fonksiyonları kullanmalıdır |

### Ön Koşullar Sonrası

Son koşullar, bir komut yürütüldükten sonra değerlendirilen doğrulamalardır.
Komutun yeniden yürütülüp yürütülmemesi gerektiğini belirlemek için
kullanılabilirler.

| Özellik      | Açıklama                                                                                                        |
| ------------ | --------------------------------------------------------------------------------------------------------------- |
| `retries`    | Doğrulama başarısız olursa komutu yeniden yürütme sayısı. Varsayılan: 0                                         |
| `backoff_ms` | Yeniden yürütmeden önce beklenecek milisaniye. Varsayılan: 1000                                                 |
| `behavior`   | Son koşul doğrulaması geçmezse uygulanacak davranış. Seçenekler: `halt` (varsayılan), `log`, `skip`, `continue` |
| `assertion`  | Kontrol edilecek doğrulama                                                                                      |

### Çıktılar

`send_http_request` eylemi başarıyla yürütüldüğünde, aşağıdaki çıktılar eyleme
eklenir:

| Ad              | Tür     | Açıklama                             |
| --------------- | ------- | ------------------------------------ |
| `response_body` | string  | Dize olarak döndürülen yanıt gövdesi |
| `status_code`   | integer | HTTP yanıt durum kodu                |

### Örnek

```hcl
action "example" "std::send_http_request" {
  url = "https://example.com"
}

output "status" {
  value = action.example.status_code
}
// > status: 200
```

### POST İstek Örneği

```hcl
action "create_resource" "std::send_http_request" {
    description = "Create a new resource"
    url = "https://api.example.com/resources"
    method = "POST"
    headers = {
        "Content-Type" = "application/json"
    }
    body = {
        "name" = variable.name,
        "value" = variable.value
    }
}
```

### Kimlik Doğrulamalı İstek

```hcl
action "api_call" "std::send_http_request" {
    description = "Fetch data from API"
    url = "https://api.example.com/data"
    method = "GET"
    headers = {
        "Authorization" = "Bearer ${variable.token}"
    }
}

output "response" {
    value = action.api_call.response_body
}

output "status" {
    value = action.api_call.status_code
}
```
