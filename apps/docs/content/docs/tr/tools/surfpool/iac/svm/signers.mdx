Bu imzalayıcılar, işlem imzalama için SVM eklentisi kullanılırken mevcuttur.

## secret_key

`svm::secret_key` imzalayıcısı, bir gizli anahtar, anımsatıcı ifade veya keypair
dosyası kullanılarak bir işlemi eş zamanlı olarak imzalamak için kullanılabilir.

### Girdiler

| Ad                | Zorunlu      | Tür    | Açıklama                                                       |
| ----------------- | ------------ | ------ | -------------------------------------------------------------- |
| `secret_key`      | isteğe bağlı | string | İletileri ve işlemleri imzalamak için kullanılan gizli anahtar |
| `mnemonic`        | isteğe bağlı | string | Gizli anahtarı oluşturmak için kullanılan anımsatıcı ifade     |
| `derivation_path` | isteğe bağlı | string | Gizli anahtarı oluşturmak için kullanılan türetme yolu         |
| `keypair_json`    | isteğe bağlı | string | Gizli anahtarı içeren bir keypair.json dosyasının yolu         |

### Çıktılar

| Ad           | Tür    | Açıklama                              |
| ------------ | ------ | ------------------------------------- |
| `public_key` | string | Hesabın pubkey değeri                 |
| `address`    | string | SVM adresi (public_key için takma ad) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Ayrıca bir anımsatıcı ifade de kullanabilirsiniz:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Ya da bir keypair dosyası:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

`svm::web_wallet` imzalayıcısı, bir Runbook operatörünün işlemi kendi seçtiği
tarayıcı imzalayıcısıyla imzalamasına olanak tanır.

### Girdiler

| Ad                 | Zorunlu      | Tür    | Açıklama                                                                      |
| ------------------ | ------------ | ------ | ----------------------------------------------------------------------------- |
| `expected_address` | isteğe bağlı | string | Bağlanması beklenen SVM adresi (herhangi birine izin vermek için belirtmeyin) |

### Çıktılar

| Ad           | Tür    | Açıklama                    |
| ------------ | ------ | --------------------------- |
| `address`    | string | Bağlı hesabın adresi        |
| `public_key` | string | Bağlı hesabın pubkey değeri |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Bu imzalayıcı, Web Arayüzünde kullanıcıdan şunları yapmasını ister:

1. Cüzdanını bağla
2. Mesaj imzası aracılığıyla bir pubkey sağla
3. Bu imzalayıcıyı gerektiren tüm işlemleri imzala

## squads

`svm::squads` imzalayıcısı, işlemleri bir Squads çoklu imza kasasıyla imzalamak
için kullanılabilir.

### Girdiler

| Ad        | Zorunlu      | Tür    | Açıklama          |
| --------- | ------------ | ------ | ----------------- |
| `address` | isteğe bağlı | string | Squad kasa adresi |

### Çıktılar

| Ad           | Tür    | Açıklama                              |
| ------------ | ------ | ------------------------------------- |
| `public_key` | string | Squad kasasının pubkey'i              |
| `address`    | string | SVM adresi (public_key için takma ad) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Squads çoklu imzasını kullanmak, bireysel özel anahtarları ifşa etmeden güvenli,
çok taraflı işlem imzalamaya olanak tanır.
