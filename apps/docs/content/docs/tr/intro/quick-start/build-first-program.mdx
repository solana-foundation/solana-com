Bu hızlı başlangıç kılavuzu, `anchor init` tarafından oluşturulan
[Anchor framework](https://www.anchor-lang.com/docs) başlangıç projesini
kullanır. Projeyi yerel olarak oluşturacak, testlerini çalıştıracak, programı
derleyecek ve Anchor'ın oluşturduğu program kodunu inceleyeceksiniz.

## Ön Koşullar

Başlamadan önce,
[Solana geliştirme araçlarını yükleyin](/docs/intro/installation). Kurulum;
Rust, Solana CLI ve Anchor CLI'yi içerir.

Bu şablon için Anchor CLI sürüm 1.1.2 veya daha yüksek bir sürüm kullanın. Yüklü
sürümünüzü kontrol edin:

```terminal
$ anchor --version
```

<Steps>

<Step>

### Projeyi Oluşturun

Terminalinizde aşağıdaki komutları çalıştırın:

```terminal
$ anchor init my-program
$ cd my-program
```

Başlangıç projesi, `programs/my-program` altında bir Solana programı içerir.
Program, bir sayaç hesabını başlatmak ve sayacı artırmak için iki talimat
içerir.

Şablonun bazı bölümleri yaygın Solana program kalıplarını gösterir:
[PDA](/docs/core/pda) hesap adreslerini türetme, SOL aktarmak için Cross Program
Invocation ([CPI](/docs/core/cpi)) yapma ve bir koşul başarısız olduğunda
talimatı durdurmak için özel hata denetimleri kullanma.

</Step>

<Step>

### Programı Derleyin

Başlangıç programını derlemek için `anchor build` komutunu çalıştırın:

```terminal
$ anchor build
```

Derlenen program, `target/deploy/my_program.so` konumuna yazılır. Program
dağıtıldığında, bu `.so` dosyasının içeriği zincir üzerinde bir hesapta
depolanır.

</Step>

<Step>

### Testi Çalıştır

Varsayılan testi çalıştırın:

```terminal
$ anchor test
```

Bu şablonun `Anchor.toml`'i Rust test komutunu kullanır:

```toml title="Anchor.toml"
skip_local_validator = true

[scripts]
test = "cargo test"
```

Test, derlenmiş programı [LiteSVM](https://www.litesvm.com/) içine yükler, bir
ödeyici oluşturur, _rs`initialize`_ ve _rs`increment`_ talimatlarını gönderir,
ardından sayaç hesabının durumunu kontrol eder.

`anchor test` komutunun çalıştırılması programı da derler; bu nedenle yerel
testlerde `anchor build` komutunu önceden çalıştırmanız gerekmez.

</Step>

<Step>

### Programı Dağıt

Yerel testler en hızlı geri bildirim döngüsüdür. Bir ağa, örneğin devnet'e
dağıtmaya hazır olduğunuzda önce derleyin, ardından bir kümeye dağıtın.

Bir Solana programının dağıtılması SOL gerektirir; çünkü program bir hesapta
depolanır ve hesap kullandığı alan için ücret ödemek zorundadır. Devnet'te,
[Solana Faucet](https://faucet.solana.com/) adresinden veya Solana CLI ile
ücretsiz devnet SOL talep edebilirsiniz:

```terminal
$ solana airdrop 2 --url devnet
```

```terminal
$ anchor build
$ anchor deploy --provider.cluster devnet
```

</Step>

</Steps>

## Kaynak dosyalar

`src` dizini Solana programını içerir. Anchor'ın
[Program Yapısı](https://www.anchor-lang.com/docs/basics/program-structure)
dokümantasyonu, _rs`declare_id!`_, _rs`#[program]`_, _rs`#[derive(Accounts)]`_
ve _rs`#[account]`_ dahil olmak üzere burada kullanılan temel makroları açıklar.
Bu bölüm şablon dosyalarını adım adım ele alır.

### lib.rs

`lib.rs`, programın giriş noktasıdır. Kaynak dosyaları birbirine bağlar, program
adresini tanımlar ve kullanıcıların çağırabileceği program talimatlarını
tanımlar.

<WithNotes>

```rust title="programs/my-program/src/lib.rs" -w
// !tooltip[/pub mod constants/] constantsMod
pub mod constants;
// !tooltip[/pub mod error/] errorMod
pub mod error;
// !tooltip[/pub mod instructions/] instructionsMod
pub mod instructions;
// !tooltip[/pub mod state/] stateMod
pub mod state;

// !tooltip[/anchor_lang::prelude/] prelude
use anchor_lang::prelude::*;

pub use constants::*;
pub use instructions::*;
pub use state::*;

// !tooltip[/declare_id!/] declareId
declare_id!("82sFkffP9wxwpyfZyeaKHH2chQoJPUGsJZSPi9mrUuXd");

// !tooltip[/#\[program\]/] programMacro
#[program]
pub mod my_program {
    use super::*;

    // !tooltip[/pub fn initialize/] publicInstructions
    pub fn initialize(ctx: Context<Initialize>) -> Result<()> {
        // !tooltip[/handle_initialize/] handlers
        crate::instructions::initialize::handle_initialize(ctx)
    }

    // !tooltip[/pub fn increment/] publicInstructions
    pub fn increment(ctx: Context<Increment>) -> Result<()> {
        // !tooltip[/handle_increment/] handlers
        crate::instructions::increment::handle_increment(ctx)
    }
}
```

### !constantsMod

_rs`pub mod constants`_ ifadesi, `constants.rs` öğesini program crate'inin bir
parçası yapar. Şablon, PDA seed, transfer miktarı ve maksimum sayaç değeri gibi
paylaşılan program değerlerini burada saklar.

### !errorMod

_rs`pub mod error`_ ifadesi, `error.rs` öğesini program crate'inin bir parçası
yapar. Bu dosya, bir talimat geçersiz olduğunda programın döndürebileceği özel
hataları tanımlar.

### !instructionsMod

_rs`pub mod instructions`_ ifadesi, `instructions.rs` öğesini program crate'inin
bir parçası yapar. Bu dosya, `src/instructions/` içindeki talimata özgü dosyalar
için bir dizin görevi görür.

### !stateMod

_rs`pub mod state`_ ifadesi, `state.rs` öğesini program crate'inin bir parçası
yapar. Bu dosya, _rs`Counter`_ hesabı da dahil olmak üzere bu programa ait
hesapların veri düzenini tanımlar.

### !prelude

_rs`anchor_lang::prelude::*`_, bir Solana programı tanımlamak için kullanılan
Anchor türlerini ve makrolarını içe aktarır; bunlar arasında _rs`Context`_,
_rs`Result`_, _rs`Pubkey`_, _rs`Account`_, _rs`Signer`_ ve Anchor hesap
nitelikleri yer alır.

### !reexports

_rs`pub use`_ satırları, alt modüllerdeki genel öğeleri crate kökünden
erişilebilir kılar. Bu sayede talimat giriş noktaları, uzun bir modül yolu
yerine _rs`Context<Initialize>`_ gibi kısa tutulur.

### !declareId

_rs`declare_id!`_, programın adresini tanımlar. Bu adres; `Anchor.toml`,
dağıtılmış program account ve programı çağıran tüm kullanıcılar ya da testlerle
eşleşmelidir.

### !programMacro

_rs`#[program]`_, Anchor'a Solana program giriş noktasını ve talimat dağıtım
kodunu oluşturmasını söyler. Bu modüldeki genel fonksiyonlar, kullanıcıların
çağırabileceği talimatlar haline gelir.

### !publicInstructions

_rs`initialize`_ ve _rs`increment`_, kullanıcıların çağırabileceği iki talimat
adıdır. Her biri bir _rs`Context<T>`_ alır. _rs`Context`_ içindeki tür, o
talimat için gereken hesapları tanımlar ve Anchor, talimat çalışmadan önce bu
hesapları doğrular.

### !handlers

İşleyici fonksiyon, talimat için gereken mantığı içerir.

</WithNotes>

### constants.rs

`constants.rs`, paylaşılan değerleri tek bir yerde tutar. Bu şablonda
_rs`COUNTER_SEED`_ sayaç PDA'sını türetir, _rs`HELLO_WORLD_LAMPORTS`_ başlatma
sırasında aktarılır ve _rs`MAX_COUNT`_ artırmadan önce kontrol edilir.

<WithNotes>

```rust title="programs/my-program/src/constants.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[constant\]/] anchorConstant
// !tooltip[/COUNTER_SEED/] counterSeed
#[constant]
pub const COUNTER_SEED: &[u8] = b"counter";

// !tooltip[/HELLO_WORLD_LAMPORTS/] helloLamports
#[constant]
pub const HELLO_WORLD_LAMPORTS: u64 = 1;

// !tooltip[/MAX_COUNT/] maxCount
#[constant]
pub const MAX_COUNT: u64 = 10;
```

### !anchorConstant

_rs`#[constant]`_, Anchor'a değeri program IDL'ine dahil etmesini söyler. Bu
sayede önemli program sabitleri, oluşturulan istemciler ve araçlar tarafından
görünür hâle gelir.

### !counterSeed

_rs`COUNTER_SEED`_, sayaç PDA'sını türetmek için kullanılan bayt dizisidir.
Program ve test, ayrı bir keypair saklamaksızın hesap adresi üzerinde mutabık
kalmak için bu seed'i birlikte kullanır. Bu şablon tek bir sabit seed
kullandığından programın tek bir sayaç adresi vardır. Gerçek programlar,
birbiriyle ilişkili pek çok hesap oluşturmak için seeds içine genellikle bir
kullanıcı adresi ya da başka bir tanımlayıcı ekler.

### !helloLamports

_rs`HELLO_WORLD_LAMPORTS`_, _rs`initialize`_ içinde ödeyiciden sayaç hesabına
aktarılan lamport miktarıdır. Şablon, System Program'a yapılan bir CPI'ı
gösterdiğinden bu değer kasıtlı olarak çok küçük tutulmuştur.

### !maxCount

_rs`MAX_COUNT`_, _rs`increment`_ tarafından kontrol edilen üst sınırdır. Bu
değerin burada tutulması; program, testler ve oluşturulan IDL'nin kural için tek
bir ortak ad kullanmasını sağlar.

</WithNotes>

- [_rs`COUNTER_SEED`_](mention:counter-seed) sayaç PDA adresini türetir.
- [_rs`HELLO_WORLD_LAMPORTS`_](mention:hello-world-lamports) ödeyiciden sayaç
  hesabına aktarılacak miktarı belirler.

</SideBySide>

### !right

`increment.rs`, sayacın üst sınırı olarak [_rs`MAX_COUNT`_](mention:max-count)

</SideBySide>

### state.rs

`state.rs`, programın oluşturduğu ve sahip olduğu hesaplar için özel veri
türlerini tanımlar. Program bu veriyi oluşturmak, başlatmak ve güncellemek için
talimatlar tanımlar; ancak sayaç verisi programın kendisinde saklanmaz. Kendi
adresiyle ayrı bir hesapta saklanır.

<WithNotes>

```rust title="programs/my-program/src/state.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[account\]/] accountMacro
#[account]
// !tooltip[/InitSpace/] initSpace
#[derive(InitSpace)]
pub struct Counter {
    // !tooltip[/count/] count
    pub count: u64,
    // !tooltip[/authority/] authority
    pub authority: Pubkey,
}
```

### !accountMacro

_rs`#[account]`_, Anchor'a bu struct'ın bu program tarafından sahiplenilen
zincir üstü bir hesapta saklanabilen veri olduğunu bildirir. Anchor bunu
_rs`Counter`_ hesaplarını serileştirmek, seri dışı bırakmak ve doğrulamak için
kullanır.

### !initSpace

_rs`InitSpace`_, Anchor'ın bu struct'ın alanları için gereken bayt sayısı olan
_rs`Counter::INIT_SPACE`_ değerini hesaplamasını sağlar. 8 baytlık Anchor hesap
ayrıştırıcısı, hesap oluşturulurken ayrıca eklenir.

### !count

_rs`count`_, hesap verisinde saklanan sayaç değeridir.

### !authority

_rs`authority`_, sayacı artırma izni olan adresi saklar. _rs`initialize`_ bu
adresi yazar, _rs`increment`_ ise aynı adresin imzalamasını zorunlu kılar.

</WithNotes>

<SideBySide>

### !left

```rust title="initialize.rs" -w

    payer = payer,
    // !mark
    // !mention counter-space

    seeds = [COUNTER_SEED],

pub counter: Account<'info, Counter>,
// ...

pub fn handle_initialize(ctx: Context<Initialize>) -> Result<()> {
    // !mark(1:2)
    // !mention(1:2) initialize-fields
    ctx.accounts.counter.count = 0;
    ctx.accounts.counter.authority = ctx.accounts.payer.key();

```

```rust title="increment.rs" -w

    // ...

    // !mark
    // !mention increment-count
    ctx.accounts.counter.count += 1;
    Ok(())
}
```

### !right

`state.rs`, _rs`Counter`_ hesap verilerini tanımlar. Talimat dosyaları, hesabı
oluştururken ve güncellerken bu türü kullanır:

- [_rs`Counter::INIT_SPACE`_](mention:counter-space), `state.rs` içinde
  tanımlanan alanlar için hesabı boyutlandırır.
- [_rs`count`_ ve _rs`authority`_](mention:initialize-fields), hesap
  başlatıldığında yazılan alan değerleridir.
- [_rs`count += 1`_](mention:increment-count), doğrulama geçtikten sonra
  depolanan sayaç değerini günceller.

</SideBySide>

### error.rs

`error.rs`, programın özel hatalarını tanımlar. Bu şablonda hatalar, bir çağıran
sayacı güncelleme yetkisine sahip olmadığında veya sayaç zaten _rs`MAX_COUNT`_
değerine ulaştığında talimat işleyicilerinin nasıl durduğunu göstermektedir.

<WithNotes>

```rust title="programs/my-program/src/error.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[error_code\]/] errorCode
#[error_code]
pub enum ErrorCode {
    // !tooltip[/Unauthorized/] unauthorized
    #[msg("Only the counter authority can update this counter")]
    Unauthorized,
    // !tooltip[/CounterOverflow/] counterOverflow
    #[msg("Counter has reached the maximum value")]
    CounterOverflow,
}
```

### !errorCode

_rs`#[error_code]`_, Anchor'a bu varyantları özel program hatalarına
dönüştürmesini söyler. İstemciler ve testler, ham hata kodlarını tahmin etmek
yerine bu hataları kullanabilir.

### !unauthorized

_rs`Unauthorized`_, işlem imzalayıcısı sayaç hesabında depolanan yetkili
olmadığında döndürülür.

### !counterOverflow

_rs`CounterOverflow`_, artırma işlemi programın _rs`MAX_COUNT`_ kuralını ihlal
edeceğinde döndürülür.

</WithNotes>

<SideBySide>

### !left

```rust title="increment.rs" -w
pub fn handle_increment(ctx: Context<Increment>) -> Result<()> {
    require_keys_eq!(
        ctx.accounts.counter.authority,
        ctx.accounts.authority.key(),
        // !mark
        // !mention unauthorized-error
        ErrorCode::Unauthorized,
    );
    require!(
        ctx.accounts.counter.count < MAX_COUNT,
        // !mark
        // !mention overflow-error
        ErrorCode::CounterOverflow,
    );

    ctx.accounts.counter.count += 1;
    msg!("Hello, world! Counter is now {}", ctx.accounts.counter.count);
    Ok(())
}
```

### !right

`error.rs`, talimatın döndürebileceği hataları listeler:

- [_rs`ErrorCode::Unauthorized`_](mention:unauthorized-error), imzalayıcı sayaç
  hesabında depolanan yetkili olmadığında döndürülür.
- [_rs`ErrorCode::CounterOverflow`_](mention:overflow-error), sayaç zaten
  _rs`MAX_COUNT`_ değerine ulaştığında döndürülür.

</SideBySide>

### instructions.rs

`instructions.rs`, talimat dosyalarını program crate'ine bağlar; böylece
`lib.rs`, _rs`initialize`_ ve _rs`increment`_ talimat kodlarına erişebilir. Her
talimat dosyası, o talimat için gereken hesapları ve Anchor bu hesapları
doğruladıktan sonra çalışan işleyici mantığını tanımlar.

<WithNotes>

```rust title="programs/my-program/src/instructions.rs" -w
// !tooltip[/pub mod initialize/] moduleDeclarations
pub mod initialize;
pub mod increment;

// !tooltip[/pub use initialize/] reexports
pub use initialize::*;
pub use increment::*;
```

### !moduleDeclarations

_rs`pub mod`_ satırları, `src/instructions/initialize.rs` ve
`src/instructions/increment.rs` öğelerini program crate'inin bir parçası hâline
getirir.

### !reexports

_rs`pub use`_ satırları, bu dosyalardan genel öğeleri yeniden dışa aktarır. Bu
sayede talimat hesap yapıları ve işleyici fonksiyonlar _rs`instructions`_ modülü
aracılığıyla erişilebilir hâle gelir.

</WithNotes>

### initialize.rs

`initialize.rs`, sayaç hesabını oluşturmak için gereken hesapları tanımlar ve
hesabın ilk değerlerini yazar. _rs`#[derive(Accounts)]`_ yapısı, hangi
hesapların gerekli olduğunu ve yeni sayaç hesabının nasıl oluşturulduğunu
belirtmek için Anchor
[hesap kısıtlamalarını](https://www.anchor-lang.com/docs/references/account-constraints)
kullanır.

<WithNotes>

```rust title="programs/my-program/src/instructions/initialize.rs" -w
use anchor_lang::prelude::*;

use crate::{constants::*, state::Counter};

// !tooltip[/derive\(Accounts\)/] accountsStruct
#[derive(Accounts)]
pub struct Initialize<'info> {
    // !tooltip[/#\[account\(mut\)\]/] payerMutable
    #[account(mut)]
    // !tooltip[/Signer<'info>/] payerSigner
    pub payer: Signer<'info>,
    // !tooltip[/#\[account\(/] counterConstraints
    #[account(
        // !tooltip[/init/] initConstraint
        init,
        // !tooltip[/payer = payer/] payerConstraint
        payer = payer,
        // !tooltip[/space = 8 \+ Counter::INIT_SPACE/] space
        space = 8 + Counter::INIT_SPACE,
        // !tooltip[/seeds = \[COUNTER_SEED\]/] seeds
        seeds = [COUNTER_SEED],
        // !tooltip[/bump/] bump
        bump
    )]
    // !tooltip[/Account<'info, Counter>/] counterAccountType
    pub counter: Account<'info, Counter>,
    // !tooltip[/Program<'info, System>/] systemProgram
    pub system_program: Program<'info, System>,
}

// !tooltip[/handle_initialize/] handler
pub fn handle_initialize(ctx: Context<Initialize>) -> Result<()> {
    // !tooltip[/counter.count = 0/] initialCount
    ctx.accounts.counter.count = 0;
    // !tooltip[/counter.authority = ctx.accounts.payer.key\(\)/] initialAuthority
    ctx.accounts.counter.authority = ctx.accounts.payer.key();

    // !tooltip[/system_program::Transfer/] cpiAccounts
    let cpi_accounts = anchor_lang::system_program::Transfer {
        from: ctx.accounts.payer.to_account_info(),
        to: ctx.accounts.counter.to_account_info(),
    };
    // !tooltip[/CpiContext::new/] cpiContext
    let cpi_ctx = CpiContext::new(anchor_lang::system_program::ID, cpi_accounts);
    // !tooltip[/system_program::transfer/] transfer
    anchor_lang::system_program::transfer(cpi_ctx, HELLO_WORLD_LAMPORTS)?;

    // !tooltip[/msg!/] programLog
    msg!("Hello, world! Counter initialized");
    Ok(())
}
```

dahil edilmesi gereken hesapları tanımlar. Anchor, işleyici çalışmadan önce bu
hesapları doğrular.

        payer = payer,
        space = 8 + Counter::INIT_SPACE,
        seeds = [COUNTER_SEED],
        bump

    pub counter: Account<'info, Counter>,
    pub system_program: Program<'info, System>,

pub fn handle_increment(ctx: Context<Increment>) -> Result<()> {
    require_keys_eq!(
        ctx.accounts.counter.authority,
        ctx.accounts.authority.key(),
        ErrorCode::Unauthorized,
    );
    // !focus(1:4)
    require!(
        ctx.accounts.counter.count < MAX_COUNT,
        ErrorCode::CounterOverflow,
    );

    ctx.accounts.counter.count += 1;
    msg!("Hello, world! Counter is now {}", ctx.accounts.counter.count);
    Ok(())
}

use {
    anchor_lang::{
        prelude::Pubkey,
