Yerel Solana geliştirme ortamınızı kurma rehberi. Windows (WSL), Linux ve Mac'te
Rust, Solana CLI ve Anchor Framework'ü nasıl yükleyeceğinizi öğrenin. Her
bağımlılığı ayrı ayrı yüklemeyi tercih ediyorsanız veya
[hızlı kurulum](/docs/intro/installation/) herhangi bir nedenle başarısız olursa
bu rehberi kullanın.

### Ön koşullar

<Accordions>
<Accordion title="Windows">

Windows'ta Solana programları geliştirmek için
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL) kullanmanız **zorunludur**. WSL'yi yükledikten sonra, diğer tüm
bağımlılıklar Linux terminali üzerinden yüklenecektir.

WSL'yi yüklemek için Windows PowerShell'de aşağıdaki komutu çalıştırın:

```terminal
$ wsl --install
```

Kurulum işlemi sırasında varsayılan bir kullanıcı hesabı oluşturmanız
istenecektir.

![WSL Kurulumu](/assets/docs/intro/installation/wsl-install.png)

Varsayılan olarak WSL, Ubuntu'yu yükler. Arama çubuğunda "Ubuntu" aratarak Linux
terminalini açabilirsiniz.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Ubuntu terminaliniz aşağıdaki görüntü gibi görünüyorsa, `ctrl + v` (yapıştırma
klavye kısayolu) terminalinde çalışmayan bir sorunla karşılaşabilirsiniz.

![Ubuntu Terminali](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Bu sorunla karşılaşırsanız, Arama çubuğunda "Terminal" aratarak Windows
Terminal'i açın.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Ardından, Windows Terminal'i kapatın ve Ubuntu'yu tekrar aratarak Linux
terminalini yeniden açın. Terminal artık aşağıdaki görüntü gibi görünmeli ve
`ctrl + v` (yapıştırma klavye kısayolu) çalışmalıdır.

![Ubuntu Terminali](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

VS Code kullanıyorsanız,
[WSL eklentisi](https://code.visualstudio.com/docs/remote/wsl-tutorial) WSL ve
VS Code'u birlikte kullanmanızı sağlar.

![VS Code'da WSL Kurulumu](/assets/docs/intro/installation/wsl-vscode.png)

VS Code durum çubuğunda aşağıdakileri göreceksiniz:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

WSL kurulumunu tamamladıktan sonra şunlardan birini yapmaya hazırsınız:

- Ubuntu (Linux) terminalini kullanarak
  [hızlı kurulumu](/docs/intro/installation) tamamlayın
- Aşağıda açıklandığı gibi [Rust](#install-rust),
  [Solana CLI](#install-solana-cli) ve [Anchor CLI](#install-anchor-cli)'yi ayrı
  ayrı kurun

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Debian veya Ubuntu ya da Mint gibi Debian türevi alternatiflerdeyseniz, aşağıdaki adımları izleyin:

1. Tüm paketlerin güncel olduğundan emin olmak için paket listelerini
   güncelleyin:

```terminal
$ sudo apt-get update
```

2. Aşağıdaki bağımlılıkları kurun:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
`protobuf-compiler` kurulumu sırasında aşağıdaki hatayla karşılaşırsanız, önce `sudo apt-get update` komutunu çalıştırdığınızdan emin olun

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Fedora veya Red Hat Linux gibi RHEL türevleri için aşağıdaki adımları izleyin.

Paketlerin daha yeni sürümlerini almak için komutu çalıştırarak paket
listelerini güncelleyin.

```terminal
$ sudo dnf upgrade --refresh
```

Ardından, aşağıdaki bağımlılıkları kurun:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

  </Tab>
</Tabs>

### Rust kurulumu

Geliştiriciler Solana programlarını [Rust](https://www.rust-lang.org/)
programlama dilini kullanarak oluştururlar.

1. Terminalinize aşağıdaki komutu girerek
   [rustup](https://www.rust-lang.org/tools/install) kullanarak Rust'ı yükleyin:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Başarılı bir kurulumdan sonra aşağıdaki mesajı göreceksiniz:

<Accordions>
<Accordion title="Başarılı Rust kurulum mesajı">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. [Cargo'nun](https://github.com/rust-lang/cargo) bin dizinini içerecek şekilde
   PATH ortam değişkeninizi yeniden yükleyin:

```terminal
$ . "$HOME/.cargo/env"
```

3. Kurulumun başarılı olduğunu doğrulayın.

```terminal
$ rustc --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Solana CLI'yi yükleyin

Solana CLI, Solana programları oluşturmak ve dağıtmak için gereken tüm araçları
sağlar.

1. Resmi kurulum komutunu kullanarak Solana CLI araç paketini yükleyin:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  `stable` yerine istediğiniz sürümün yazılım sürümüyle eşleşen sürüm etiketini
  (örn. `v2.0.3`) kullanabilir veya üç sembolik kanal adından birini
  kullanabilirsiniz: `stable`, `beta` veya `edge`.
</Callout>

2. PATH ortam değişkeni ekleyin

Solana CLI'nin ilk kez kurulumu sırasında bir `PATH` ortam değişkeni eklemeniz
istenebilir. Bunu yapmak için terminalinizi kapatıp yeniden açın veya
kabuğunuzda aşağıdaki komutu çalıştırın:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. PATH ortam değişkeninizi güncelleyin

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

**Linux** veya **WSL** kullanıyorsanız, komutun terminalinizde kullanılabilir
olması için Solana CLI binary dosyasını PATH'inize eklemeniz gerekir. Bunu
yapmak için aşağıdaki adımları izleyin:

**a.** Hangi kabuğu kullandığınızı kontrol edin:

```terminal
$ echo $SHELL
```

- Çıktı `/bash` içeriyorsa, `.bashrc` kullanın.
- Çıktı `/zsh` içeriyorsa, `.zshrc` kullanın.

**b.** Shell'inize göre uygun komutu çalıştırın.

Bash için (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Zsh için (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

`zsh` kullanan bir Mac kullanıyorsanız, kurulumdan kaydedilen varsayılan
`export PATH` komutunu çalıştırmak terminalinizi kapattığınızda kalıcı olmaz.
Bunun yerine, aşağıdaki komutu çalıştırarak PATH'i shell yapılandırma dosyanıza
ekleyin:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Yukarıdaki adımlar `bash` kullanıcıları için geçerli değildir, çünkü PATH
  değişkenindeki değişiklikler varsayılan olarak kalıcı olacaktır.
</Callout>

</Tab>
</Tabs>

4. Terminalinizi yeniden başlatın veya terminal oturumunu yenilemek için
   aşağıdaki komutu çalıştırın:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Solana CLI sürümünü kontrol ederek kurulumun başarılı olduğunu doğrulayın:

```terminal
$ solana --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Tüm mevcut sürümleri
[Agave Github deposunda](https://github.com/anza-xyz/agave/releases)
görüntüleyebilirsiniz. Agave, [Anza](https://www.anza.xyz/)'dan gelen validator
istemcisidir, eskiden Solana Labs validator istemcisi olarak biliniyordu.

</Callout>

5. Gerektiğinde Solana CLI'yi en son sürüme güncelleyin _(İsteğe bağlı)_

```terminal
$ agave-install update
```

</Step>

### Anchor CLI'yi yükleyin

[Anchor](https://www.anchor-lang.com/), Solana programları geliştirmek için bir
framework'tür. Anchor framework'ü, Solana programları yazma sürecini
basitleştirmek için Rust makrolarından yararlanır.

#### Ön koşullar

_shell`anchor init`_ komutuyla oluşturulan varsayılan Anchor proje test dosyası
(TypeScript), Node.js ve Yarn gerektirir. (Rust test şablonu
_shell`anchor init --test-template rust`_ kullanılarak mevcuttur)

<Accordions>
<Accordion title="Node kurulumu">

1. Aşağıdaki komutu çalıştırarak
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) kullanarak Node'u
   yükleyin:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Terminalinizi yeniden başlatın ve `nvm` komutunun başarıyla çalıştığını
   doğrulayın:

```terminal
$ command -v nvm
```

3. Node'u yüklemek için `nvm` kullanın:

```terminal
$ nvm install node
```

4. Node sürümünü kontrol ederek kurulumun başarılı olduğunu doğrulayın:

```terminal
$ node --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
v23.7.0
```

</Accordion>
<Accordion title="Yarn kurulumu">

1. Yarn'ı yükleyin:

```terminal
$ npm install --global yarn
```

2. Yarn sürümünü kontrol ederek kurulumun başarılı olduğunu doğrulayın:

```terminal
$ yarn --version
```

Aşağıdaki çıktıyı göreceksiniz:

```
1.22.1
```

</Accordion>
</Accordions>

#### Kurulum

Anchor CLI ve araçlarını iki şekilde yükleyebilirsiniz:

- Anchor Version Manager (AVM) — Önerilen kurulum yöntemi
- AVM olmadan — Doğrudan GitHub'dan yükleme

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Anchor Version Manager (AVM), sisteminizde farklı Anchor sürümlerini kurmanıza
ve yönetmenize olanak tanır ve gelecekte Anchor sürümlerini kolayca
güncellemenizi sağlar. Anchor'ı AVM kullanarak kurmak için aşağıdaki adımları
izleyin:

1. AVM'yi aşağıdaki komutla kurun:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. AVM'nin başarıyla kurulduğunu doğrulayın:

```terminal
$ avm --version
```

3. AVM kullanarak Anchor CLI'yi kurun:

En son sürümü kurmak için:

```terminal
$ avm install latest
$ avm use latest
```

Belirli bir sürümü kurmak için sürüm numarasını belirtin:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Anchor CLI'yi Linux veya WSL üzerinde kurarken şu hatayla karşılaşabilirsiniz:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Bu hata mesajını görürseniz şu adımları izleyin:

1. Bu sayfanın üst kısmındaki
   [Linux](/docs/intro/installation#install-dependencies) bölümünde listelenen
   bağımlılıkları kurun.
2. Anchor CLI kurulumunu yeniden deneyin.

</Callout>

4. Anchor CLI sürümünü kontrol ederek kurulumun başarılı olduğunu doğrulayın:

```terminal
$ anchor --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
anchor-cli 0.31.1
```

<Callout type="info">

Sisteminizde çalıştırılacak Anchor CLI sürümünü bildirmek için _shell`avm use`_
komutunu çalıştırmayı unutmayın.

- `latest` sürümünü kurduysanız _shell`avm use latest`_ komutunu çalıştırın.
- `0.30.1` sürümünü kurduysanız _shell`avm use 0.30.1`_ komutunu çalıştırın.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Aşağıdaki komutla Anchor CLI'nin belirli bir sürümünü kurun:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Kurulumun başarılı olduğunu, Anchor CLI sürümünü kontrol ederek doğrulayın:

```terminal
$ anchor --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>

### Surfpool CLI'yi kurun

[Surfpool](https://surfpool.run/), yerel geliştirme için bir araçtır ve
`solana-test-validator` için geliştirilmiş bir alternatiftir. Surfpool
özellikleri hakkında daha fazla bilgi için
[Surfpool belgelerine](https://docs.surfpool.run/) göz atın.

1. Aşağıdaki komutla Surfpool'u kurun:

```terminal
$ curl -sL https://run.surfpool.run/ | bash
```

2. Kurulumun başarılı olduğunu, Surfpool CLI sürümünü kontrol ederek doğrulayın:

```terminal
$ surfpool --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
surfpool 0.12.0
```
