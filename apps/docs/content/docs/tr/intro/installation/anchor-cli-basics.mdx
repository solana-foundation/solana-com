  Bu bölüm, başlamanız için bazı yaygın Anchor CLI komutlarını açıklar.
---

Bu bölüm, [Anchor CLI](https://www.anchor-lang.com/docs) kullanmaya başlamanıza
yardımcı olacak bazı yaygın komutlar ve örnekler sunmaktadır.

<Steps>
<Step>

### Projeyi başlat

Aşağıda gösterilen komutu çalıştırarak yeni bir Anchor projesi oluşturun. Bu
komut, proje adıyla yeni bir dizin oluşturacak ve bunu yeni bir Anchor projesi
başlatmak için kullanacaktır.

```terminal
$ anchor init <project-name>
```

Örneğin, aşağıdaki komut `my-project` adında bir proje oluşturacaktır.
`my-project` dizini, temel bir Rust programı ve TypeScript test şablonu
içerecektir.

```terminal
$ anchor init my-project
```

Ardından proje dizinine gidin:

```terminal
$ cd <project-name>
```

Anchor'un
[proje dosya yapısına](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
bakın.

</Step>
<Step>

### Programı derle

Projenizi derlemek için aşağıdaki komutu çalıştırın:

```terminal
$ anchor build
```

Derlenmiş programı `/target/deploy` dizininde bulabilirsiniz.

<Callout type="warn">

`anchor build` komutunu çalıştırırken aşağıdaki hatalarla karşılaşırsanız:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Bu çözümleri deneyin:

1. Aşağıdaki komutu kullanarak zorla yükleme yapın:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Önceki çözüm işe yaramazsa, Solana önbelleğini temizleyin:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Bunu, `Cargo.lock` dosyasının version alanını 4'ten 3'e değiştirerek düzeltebilirsiniz:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Daha fazla bilgi için
[bu soruna](https://github.com/solana-foundation/anchor/issues/3392) bakın.

</Accordion>

</Accordions>

Yukarıdaki çözümü uyguladıktan sonra, _shell`anchor build`_ komutunu tekrar
çalıştırmayı deneyin.

</Callout>

</Step>
<Step>

### Programı dağıt

Projenizi dağıtmak için aşağıdaki komutu çalıştırın:

```terminal
$ anchor deploy
```

Bu komut, programınızı
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml)
dosyasında belirtilen `cluster` üzerine dağıtır.

</Step>
<Step>

### Programı test et

Projenizi test etmek için aşağıdaki komutu çalıştırın:

```terminal
$ anchor test
```

Bu komut, projenizi derler, dağıtır ve testleri çalıştırır.

<Callout>
  `localnet` kümesini kullanırken, Anchor otomatik olarak yerel bir validator
  başlatır, programı dağıtır, testleri çalıştırır ve ardından validator'ı
  durdurur.
</Callout>

<Callout type="warn">

Aşağıdaki hatalardan herhangi biri, Node.js veya Yarn'ın kurulu olmadığını
gösterebilir:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
