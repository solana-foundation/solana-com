Αυτή η ενότητα παρέχει μερικές κοινές εντολές και παραδείγματα για να σας
βοηθήσει να ξεκινήσετε τη χρήση του
[Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Αρχικοποίηση του έργου

Δημιουργήστε ένα νέο έργο Anchor εκτελώντας την εντολή που φαίνεται παρακάτω. Θα
δημιουργήσει έναν νέο κατάλογο με το όνομα του έργου και θα τον χρησιμοποιήσει
για να αρχικοποιήσει ένα νέο έργο Anchor.

```terminal
$ anchor init <project-name>
```

Για παράδειγμα, η παρακάτω εντολή θα δημιουργήσει ένα έργο με το όνομα
`my-project`. Ο κατάλογος `my-project` θα περιέχει ένα βασικό πρόγραμμα Rust και
ένα πρότυπο δοκιμής TypeScript.

```terminal
$ anchor init my-project
```

Στη συνέχεια, μεταβείτε στον κατάλογο του έργου:

```terminal
$ cd <project-name>
```

Δείτε τη
[δομή αρχείων έργου](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
του Anchor.

</Step>
<Step>

### Κατασκευή του προγράμματος

Για να κατασκευάσετε το έργο σας, εκτελέστε την ακόλουθη εντολή:

```terminal
$ anchor build
```

Μπορείτε να βρείτε το μεταγλωττισμένο πρόγραμμα στον κατάλογο `/target/deploy`.

<Callout type="warn">

Όταν εκτελείτε την εντολή `anchor build`, εάν αντιμετωπίσετε τα ακόλουθα
σφάλματα:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Δοκιμάστε αυτές τις λύσεις:

1. Εγκαταστήστε με επιβολή χρησιμοποιώντας την ακόλουθη εντολή:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Εάν η προηγούμενη λύση δεν λειτουργήσει, διαγράψτε την προσωρινή μνήμη του
   Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Μπορείτε να το διορθώσετε αλλάζοντας το πεδίο version του αρχείου `Cargo.lock` από 4 σε 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Δείτε [αυτό το ζήτημα](https://github.com/solana-foundation/anchor/issues/3392)
για περισσότερες πληροφορίες.

</Accordion>

</Accordions>

Αφού εφαρμόσετε την προηγούμενη λύση, προσπαθήστε να εκτελέσετε ξανά
_shell`anchor build`_.

</Callout>

</Step>
<Step>

### Ανάπτυξη του προγράμματος

Για να αναπτύξετε το έργο σας, εκτελέστε την ακόλουθη εντολή:

```terminal
$ anchor deploy
```

Αυτή η εντολή αναπτύσσει το πρόγραμμά σας στο `cluster` που καθορίζεται στο
αρχείο [`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Δοκιμή του προγράμματος

Για να δοκιμάσετε το έργο σας, εκτελέστε την ακόλουθη εντολή:

```terminal
$ anchor test
```

Αυτή η εντολή δημιουργεί, αναπτύσσει και εκτελεί τις δοκιμές για το έργο σας.

<Callout>
  Όταν χρησιμοποιείτε το cluster `localnet`, το Anchor ξεκινά αυτόματα έναν
  τοπικό validator, αναπτύσσει το πρόγραμμα, εκτελεί τις δοκιμές και στη
  συνέχεια σταματά τον validator.
</Callout>

<Callout type="warn">

Οποιοδήποτε από τα ακόλουθα σφάλματα μπορεί να υποδεικνύει ότι δεν έχετε
εγκαταστήσει το Node.js ή το Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
