Αυτή η γρήγορη εκκίνηση χρησιμοποιεί το αρχικό έργο του
[πλαισίου Anchor](https://www.anchor-lang.com/docs) που δημιουργείται από το
`anchor init`. Θα δημιουργήσετε το έργο τοπικά, θα εκτελέσετε τις δοκιμές του,
θα κατασκευάσετε το πρόγραμμα και θα εξερευνήσετε τον κώδικα που δημιουργεί το
Anchor.

## Προαπαιτούμενα

Πριν ξεκινήσετε,
[εγκαταστήστε τα εργαλεία ανάπτυξης Solana](/docs/intro/installation). Η
εγκατάσταση περιλαμβάνει Rust, το Solana CLI και το Anchor CLI.

Χρησιμοποιήστε την έκδοση 1.1.2 ή νεότερη του Anchor CLI για αυτό το πρότυπο.
Ελέγξτε την εγκατεστημένη έκδοσή σας:

```terminal
$ anchor --version
```

<Steps>

<Step>

### Δημιουργία του έργου

Εκτελέστε τις παρακάτω εντολές στο τερματικό σας:

```terminal
$ anchor init my-program
$ cd my-program
```

Το αρχικό έργο περιλαμβάνει ένα πρόγραμμα Solana στο `programs/my-program`. Το
πρόγραμμα περιλαμβάνει δύο εντολές: μία για την αρχικοποίηση ενός λογαριασμού
μετρητή και μία για την αύξηση του μετρητή.

Ορισμένα μέρη του προτύπου επιδεικνύουν συνήθη μοτίβα προγραμμάτων Solana:
παραγωγή διευθύνσεων λογαριασμών [PDA](/docs/core/pda), εκτέλεση Cross Program
Invocation ([CPI](/docs/core/cpi)) για μεταφορά SOL, και χρήση προσαρμοσμένων
ελέγχων σφαλμάτων για τη διακοπή μιας εντολής όταν μια συνθήκη αποτυγχάνει.

</Step>

<Step>

### Κατασκευή Προγράμματος

Εκτελέστε `anchor build` για να μεταγλωττίσετε το αρχικό πρόγραμμα:

```terminal
$ anchor build
```

Το μεταγλωττισμένο πρόγραμμα αποθηκεύεται στο `target/deploy/my_program.so`.
Κατά την ανάπτυξη του προγράμματος, τα περιεχόμενα αυτού του αρχείου `.so`
αποθηκεύονται σε έναν λογαριασμό onchain.

</Step>

<Step>

### Εκτέλεση Δοκιμής

Εκτελέστε την προεπιλεγμένη δοκιμή:

```terminal
$ anchor test
```

Το `Anchor.toml` αυτού του προτύπου χρησιμοποιεί την εντολή δοκιμής Rust:

```toml title="Anchor.toml"
skip_local_validator = true

[scripts]
test = "cargo test"
```

Η δοκιμή φορτώνει το μεταγλωττισμένο πρόγραμμα στο
[LiteSVM](https://www.litesvm.com/), δημιουργεί έναν πληρωτή, αποστέλλει τις
εντολές _rs`initialize`_ και _rs`increment`_, και στη συνέχεια ελέγχει την
κατάσταση του program account.

Η εκτέλεση του `anchor test` μεταγλωττίζει επίσης το πρόγραμμα, επομένως δεν
χρειάζεται να εκτελέσετε το `anchor build` πρώτα κατά την τοπική δοκιμή.

</Step>

<Step>

### Ανάπτυξη Προγράμματος

Οι τοπικές δοκιμές είναι ο ταχύτερος κύκλος ανατροφοδότησης. Όταν είστε έτοιμοι
να αναπτύξετε σε ένα δίκτυο, για παράδειγμα στο devnet, κάντε πρώτα build και
στη συνέχεια αναπτύξτε σε ένα cluster.

Η ανάπτυξη ενός προγράμματος Solana απαιτεί SOL επειδή το πρόγραμμα αποθηκεύεται
σε ένα λογαριασμό, και ο λογαριασμός πρέπει να πληρώσει για τον χώρο που
χρησιμοποιεί. Στο devnet, ζητήστε δωρεάν devnet SOL από το
[Solana Faucet](https://faucet.solana.com/) ή με το Solana CLI:

```terminal
$ solana airdrop 2 --url devnet
```

```terminal
$ anchor build
$ anchor deploy --provider.cluster devnet
```

</Step>

</Steps>

## Αρχεία πηγής

Ο κατάλογος `src` περιέχει το πρόγραμμα Solana. Τα έγγραφα
[Program Structure](https://www.anchor-lang.com/docs/basics/program-structure)
του Anchor εξηγούν τις βασικές μακροεντολές που χρησιμοποιούνται εδώ,
συμπεριλαμβανομένων των _rs`declare_id!`_, _rs`#[program]`_,
_rs`#[derive(Accounts)]`_ και _rs`#[account]`_. Αυτή η ενότητα αναλύει τα αρχεία
προτύπου.

### lib.rs

Το `lib.rs` είναι το σημείο εισόδου του προγράμματος. Συνδέει τα αρχεία πηγής,
ορίζει τη διεύθυνση του προγράμματος και ορίζει τις εντολές του προγράμματος που
μπορούν να καλέσουν οι χρήστες.

<WithNotes>

```rust title="programs/my-program/src/lib.rs" -w
// !tooltip[/pub mod constants/] constantsMod
pub mod constants;
// !tooltip[/pub mod error/] errorMod
pub mod error;
// !tooltip[/pub mod instructions/] instructionsMod
pub mod instructions;
// !tooltip[/pub mod state/] stateMod
pub mod state;

// !tooltip[/anchor_lang::prelude/] prelude
use anchor_lang::prelude::*;

pub use constants::*;
pub use instructions::*;
pub use state::*;

// !tooltip[/declare_id!/] declareId
declare_id!("82sFkffP9wxwpyfZyeaKHH2chQoJPUGsJZSPi9mrUuXd");

// !tooltip[/#\[program\]/] programMacro
#[program]
pub mod my_program {
    use super::*;

    // !tooltip[/pub fn initialize/] publicInstructions
    pub fn initialize(ctx: Context<Initialize>) -> Result<()> {
        // !tooltip[/handle_initialize/] handlers
        crate::instructions::initialize::handle_initialize(ctx)
    }

    // !tooltip[/pub fn increment/] publicInstructions
    pub fn increment(ctx: Context<Increment>) -> Result<()> {
        // !tooltip[/handle_increment/] handlers
        crate::instructions::increment::handle_increment(ctx)
    }
}
```

### !constantsMod

_rs`pub mod constants`_ εντάσσει το `constants.rs` στο crate του προγράμματος.
Εδώ το πρότυπο διατηρεί κοινές τιμές του προγράμματος, όπως το seed του PDA, το
ποσό μεταφοράς και η μέγιστη τιμή μετρητή.

### !errorMod

_rs`pub mod error`_ εντάσσει το `error.rs` στο crate του προγράμματος. Αυτό το
αρχείο ορίζει τα προσαρμοσμένα σφάλματα που μπορεί να επιστρέψει το πρόγραμμα
όταν μια εντολή είναι άκυρη.

### !instructionsMod

_rs`pub mod instructions`_ εντάσσει το `instructions.rs` στο crate του
προγράμματος. Αυτό το αρχείο λειτουργεί ως ευρετήριο για τα αρχεία που αφορούν
συγκεκριμένες εντολές μέσα στο `src/instructions/`.

### !stateMod

_rs`pub mod state`_ εντάσσει το `state.rs` στο crate του προγράμματος. Αυτό το
αρχείο ορίζει τη διάταξη δεδομένων για τους λογαριασμούς που ανήκουν σε αυτό το
πρόγραμμα, συμπεριλαμβανομένου του λογαριασμού _rs`Counter`_.

### !prelude

Το _rs`anchor_lang::prelude::*`_ εισάγει τους τύπους και τα macros του Anchor
που χρησιμοποιούνται για τον ορισμό ενός προγράμματος Solana,
συμπεριλαμβανομένων των _rs`Context`_, _rs`Result`_, _rs`Pubkey`_,
_rs`Account`_, _rs`Signer`_ και των χαρακτηριστικών λογαριασμού του Anchor.

### !reexports

Οι γραμμές _rs`pub use`_ καθιστούν δημόσια προσβάσιμα τα στοιχεία των θυγατρικών
ενοτήτων από τη ρίζα του crate. Αυτό διατηρεί τα σημεία εισόδου εντολών σύντομα,
όπως το _rs`Context<Initialize>`_ αντί για μια μεγαλύτερη διαδρομή ενότητας.

### !declareId

Το _rs`declare_id!`_ ορίζει τη διεύθυνση του προγράμματος. Θα πρέπει να
ταιριάζει με τη διεύθυνση στο `Anchor.toml`, το program account που έχει
αναπτυχθεί, και οποιουσδήποτε χρήστες ή δοκιμές που καλούν το πρόγραμμα.

### !programMacro

Το _rs`#[program]`_ δίνει εντολή στο Anchor να δημιουργήσει το σημείο εισόδου
του προγράμματος Solana και τον κώδικα κατανομής εντολών. Οι δημόσιες
συναρτήσεις σε αυτή την ενότητα γίνονται οι εντολές που μπορούν να καλέσουν οι
χρήστες.

### !publicInstructions

Τα _rs`initialize`_ και _rs`increment`_ είναι τα δύο ονόματα εντολών που μπορούν
να καλέσουν οι χρήστες. Κάθε μία λαμβάνει ένα _rs`Context<T>`_. Ο τύπος μέσα στο
_rs`Context`_ ορίζει τους λογαριασμούς που απαιτούνται από αυτή την εντολή, και
το Anchor επικυρώνει αυτούς τους λογαριασμούς πριν εκτελεστεί η εντολή.

### !handlers

Η συνάρτηση χειριστή περιέχει τη λογική για την εντολή.

</WithNotes>

Η ίδια διεύθυνση προγράμματος εμφανίζεται στη διαμόρφωση και στον κώδικα. Το
`Anchor.toml` λέει στο Anchor ποια διεύθυνση να αναπτύξει ή να καλέσει για ένα
cluster. Το _rs`declare_id!`_ ορίζει τη διεύθυνση του προγράμματος μέσα στο
πρόγραμμα για ελέγχους ασφαλείας.

### constants.rs

Το `constants.rs` διατηρεί τις κοινές τιμές σε ένα μέρος. Σε αυτό το πρότυπο, το
_rs`COUNTER_SEED`_ εξάγει το PDA του μετρητή, το _rs`HELLO_WORLD_LAMPORTS`_
μεταφέρεται κατά την αρχικοποίηση, και το _rs`MAX_COUNT`_ ελέγχεται πριν την
αύξηση.

<WithNotes>

```rust title="programs/my-program/src/constants.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[constant\]/] anchorConstant
// !tooltip[/COUNTER_SEED/] counterSeed
#[constant]
pub const COUNTER_SEED: &[u8] = b"counter";

// !tooltip[/HELLO_WORLD_LAMPORTS/] helloLamports
#[constant]
pub const HELLO_WORLD_LAMPORTS: u64 = 1;

// !tooltip[/MAX_COUNT/] maxCount
#[constant]
pub const MAX_COUNT: u64 = 10;
```

### !anchorConstant

Το _rs`#[constant]`_ λέει στο Anchor να συμπεριλάβει την τιμή στο IDL του
προγράμματος. Αυτό καθιστά σημαντικές σταθερές του προγράμματος ορατές στους
παραγόμενους clients και στα εργαλεία.

### !counterSeed

Το _rs`COUNTER_SEED`_ είναι η συμβολοσειρά bytes που χρησιμοποιείται για την
εξαγωγή του PDA του μετρητή. Το πρόγραμμα και οι δοκιμές χρησιμοποιούν αυτό το
seed ώστε να συμφωνούν στη διεύθυνση του λογαριασμού χωρίς να αποθηκεύουν
ξεχωριστό keypair. Αυτό το πρότυπο χρησιμοποιεί ένα σταθερό seed, οπότε το
πρόγραμμα έχει μία διεύθυνση μετρητή. Τα πραγματικά προγράμματα συχνά
συμπεριλαμβάνουν τη διεύθυνση ενός χρήστη ή άλλο αναγνωριστικό στα seeds για τη
δημιουργία πολλών σχετικών λογαριασμών.

### !helloLamports

Το _rs`HELLO_WORLD_LAMPORTS`_ είναι το ποσό των lamport που μεταφέρονται από τον
πληρωτή στον λογαριασμό μετρητή μέσα στο _rs`initialize`_. Είναι σκόπιμα μικρό
επειδή το πρότυπο επιδεικνύει ένα CPI στο System Program.

### !maxCount

Το _rs`MAX_COUNT`_ είναι το ανώτατο όριο που ελέγχεται από το _rs`increment`_. Η
διατήρησή του εδώ δίνει στο πρόγραμμα, στις δοκιμές και στο παραγόμενο IDL ένα
κοινό όνομα για τον κανόνα.

</WithNotes>

```rust title="initialize.rs" -w

    // ...
    #[account(
        init,
        payer = payer,

    // ...

pub fn handle_initialize(ctx: Context<Initialize>) -> Result<()> {
    // ...

    // !mark
```

### !right

Το `initialize.rs` χρησιμοποιεί δύο σταθερές:

- [_rs`COUNTER_SEED`_](mention:counter-seed) παράγει τη διεύθυνση PDA του
  μετρητή.
- [_rs`HELLO_WORLD_LAMPORTS`_](mention:hello-world-lamports) ορίζει το ποσό που
  μεταφέρεται από τον πληρωτή στον λογαριασμό μετρητή.

</SideBySide>

### !left

```rust title="increment.rs" -w
pub fn handle_increment(ctx: Context<Increment>) -> Result<()> {
    // ...

    require!(
        // !mark
        // !mention max-count
        ctx.accounts.counter.count < MAX_COUNT,
        ErrorCode::CounterOverflow,
    );

    ctx.accounts.counter.count += 1;
    Ok(())

```

### !right

Το `increment.rs` χρησιμοποιεί το [_rs`MAX_COUNT`_](mention:max-count) ως το
ανώτατο όριο του μετρητή. Εάν η τρέχουσα τιμή είναι ήδη στο μέγιστο, το
_rs`require!`_ επιστρέφει _rs`CounterOverflow`_ και τα δεδομένα του λογαριασμού
δεν αλλάζουν.

</SideBySide>

### state.rs

Το `state.rs` ορίζει προσαρμοσμένους τύπους δεδομένων για τους λογαριασμούς που
δημιουργεί και διαχειρίζεται το πρόγραμμα. Το πρόγραμμα ορίζει οδηγίες για τη
δημιουργία, αρχικοποίηση και ενημέρωση αυτών των δεδομένων, αλλά τα δεδομένα του
μετρητή δεν αποθηκεύονται μέσα στο ίδιο το πρόγραμμα. Αποθηκεύονται σε έναν
ξεχωριστό λογαριασμό με τη δική του διεύθυνση.

<WithNotes>

```rust title="programs/my-program/src/state.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[account\]/] accountMacro
#[account]
// !tooltip[/InitSpace/] initSpace
#[derive(InitSpace)]
pub struct Counter {
    // !tooltip[/count/] count
    pub count: u64,
    // !tooltip[/authority/] authority
    pub authority: Pubkey,
}
```

### !accountMacro

Το _rs`#[account]`_ ενημερώνει το Anchor ότι αυτό το struct είναι δεδομένα που
μπορούν να αποθηκευτούν σε έναν onchain λογαριασμό που ανήκει σε αυτό το
πρόγραμμα. Το Anchor το χρησιμοποιεί για να σειριοποιεί, αποσειριοποιεί και να
επαληθεύει τους λογαριασμούς _rs`Counter`_.

### !initSpace

Το _rs`InitSpace`_ επιτρέπει στο Anchor να υπολογίζει το
_rs`Counter::INIT_SPACE`_, δηλαδή τον αριθμό των bytes που απαιτούνται για τα
πεδία αυτού του struct. Ο 8-byte διακριτής λογαριασμού του Anchor προστίθεται
ξεχωριστά κατά τη δημιουργία του λογαριασμού.

### !count

Το _rs`count`_ είναι η τιμή του μετρητή που αποθηκεύεται μέσα στα δεδομένα του
λογαριασμού.

### !authority

Το _rs`authority`_ αποθηκεύει τη διεύθυνση που επιτρέπεται να αυξάνει τον
μετρητή. Το _rs`initialize`_ γράφει αυτή τη διεύθυνση, και το _rs`increment`_
απαιτεί την ίδια διεύθυνση για υπογραφή.

</WithNotes>

<SideBySide>

### !left

```rust title="initialize.rs" -w
#[account(
    init,
    payer = payer,
    // !mark
    // !mention counter-space
    space = 8 + Counter::INIT_SPACE,
    seeds = [COUNTER_SEED],
    bump

pub counter: Account<'info, Counter>,
// ...

pub fn handle_initialize(ctx: Context<Initialize>) -> Result<()> {
    // !mark(1:2)
    // !mention(1:2) initialize-fields
    ctx.accounts.counter.count = 0;
    ctx.accounts.counter.authority = ctx.accounts.payer.key();

    Ok(())
}
```

```rust title="increment.rs" -w

    // ...

    // !mark
    // !mention increment-count
    ctx.accounts.counter.count += 1;
    Ok(())

```

### !right

`state.rs` ορίζει τα δεδομένα λογαριασμού _rs`Counter`_. Τα αρχεία εντολών
χρησιμοποιούν αυτόν τον τύπο όταν δημιουργούν και ενημερώνουν τον λογαριασμό:

- [_rs`Counter::INIT_SPACE`_](mention:counter-space) καθορίζει το μέγεθος του
  λογαριασμού για τα πεδία που ορίζονται στο `state.rs`.
- [_rs`count`_ και _rs`authority`_](mention:initialize-fields) είναι οι τιμές
  των πεδίων που εγγράφονται κατά την αρχικοποίηση του λογαριασμού.
- [_rs`count += 1`_](mention:increment-count) ενημερώνει την αποθηκευμένη τιμή
  μετρητή μετά την επιτυχή επικύρωση.

</SideBySide>

### error.rs

`error.rs` ορίζει τα προσαρμοσμένα σφάλματα του προγράμματος. Σε αυτό το
πρότυπο, τα σφάλματα δείχνουν πώς οι χειριστές εντολών σταματούν όταν ένας καλών
δεν επιτρέπεται να ενημερώσει τον μετρητή ή ο μετρητής έχει ήδη φτάσει στο
_rs`MAX_COUNT`_.

<WithNotes>

```rust title="programs/my-program/src/error.rs" -w
use anchor_lang::prelude::*;

// !tooltip[/#\[error_code\]/] errorCode
#[error_code]
pub enum ErrorCode {
    // !tooltip[/Unauthorized/] unauthorized
    #[msg("Only the counter authority can update this counter")]
    Unauthorized,
    // !tooltip[/CounterOverflow/] counterOverflow
    #[msg("Counter has reached the maximum value")]
    CounterOverflow,
}
```

### !errorCode

_rs`#[error_code]`_ λέει στο Anchor να μετατρέψει αυτές τις παραλλαγές σε
προσαρμοσμένα σφάλματα προγράμματος. Οι πελάτες και οι δοκιμές μπορούν να
χρησιμοποιούν αυτά τα σφάλματα αντί να μαντεύουν από ακατέργαστους κωδικούς
σφάλματος.

### !unauthorized

_rs`Unauthorized`_ επιστρέφεται όταν ο υπογράφων της συναλλαγής δεν είναι η αρχή
που είναι αποθηκευμένη στον λογαριασμό μετρητή.

### !counterOverflow

_rs`CounterOverflow`_ επιστρέφεται όταν η αύξηση θα παραβίαζε τον κανόνα
_rs`MAX_COUNT`_ του προγράμματος.

</WithNotes>

<SideBySide>

### !left

```rust title="increment.rs" -w
pub fn handle_increment(ctx: Context<Increment>) -> Result<()> {
    require_keys_eq!(
        ctx.accounts.counter.authority,
        ctx.accounts.authority.key(),
        // !mark
        // !mention unauthorized-error
        ErrorCode::Unauthorized,
    );
    require!(
        ctx.accounts.counter.count < MAX_COUNT,
        // !mark
        // !mention overflow-error
        ErrorCode::CounterOverflow,
    );

    ctx.accounts.counter.count += 1;
    msg!("Hello, world! Counter is now {}", ctx.accounts.counter.count);
    Ok(())
}
```

### !right

`error.rs` ονομάζει τα σφάλματα που μπορεί να επιστρέψει η εντολή:

- [_rs`ErrorCode::Unauthorized`_](mention:unauthorized-error) επιστρέφεται όταν
  ο υπογράφων δεν είναι η αρχή που είναι αποθηκευμένη στον λογαριασμό μετρητή.
- [_rs`ErrorCode::CounterOverflow`_](mention:overflow-error) επιστρέφεται όταν ο
  μετρητής έχει ήδη φτάσει στο _rs`MAX_COUNT`_.

</SideBySide>

### initialize.rs

`initialize.rs` ορίζει τους λογαριασμούς που απαιτούνται για τη δημιουργία του
λογαριασμού μετρητή και στη συνέχεια γράφει τις αρχικές τιμές του λογαριασμού. Η
δομή _rs`#[derive(Accounts)]`_ χρησιμοποιεί τους
[περιορισμούς λογαριασμού](https://www.anchor-lang.com/docs/references/account-constraints)
του Anchor για να δηλώσει ποιοι λογαριασμοί απαιτούνται και πώς δημιουργείται ο
νέος λογαριασμός μετρητή.

<WithNotes>

```rust title="programs/my-program/src/instructions/initialize.rs" -w
use anchor_lang::prelude::*;

use crate::{constants::*, state::Counter};

// !tooltip[/derive\(Accounts\)/] accountsStruct
#[derive(Accounts)]
pub struct Initialize<'info> {
    // !tooltip[/#\[account\(mut\)\]/] payerMutable
    #[account(mut)]
    // !tooltip[/Signer<'info>/] payerSigner
    pub payer: Signer<'info>,
    // !tooltip[/#\[account\(/] counterConstraints
    #[account(
        // !tooltip[/init/] initConstraint
        init,
        // !tooltip[/payer = payer/] payerConstraint
        payer = payer,
        // !tooltip[/space = 8 \+ Counter::INIT_SPACE/] space
        space = 8 + Counter::INIT_SPACE,
        // !tooltip[/seeds = \[COUNTER_SEED\]/] seeds
        seeds = [COUNTER_SEED],
        // !tooltip[/bump/] bump
        bump
    )]
    // !tooltip[/Account<'info, Counter>/] counterAccountType
    pub counter: Account<'info, Counter>,
    // !tooltip[/Program<'info, System>/] systemProgram
    pub system_program: Program<'info, System>,
}

// !tooltip[/handle_initialize/] handler
pub fn handle_initialize(ctx: Context<Initialize>) -> Result<()> {
    // !tooltip[/counter.count = 0/] initialCount
    ctx.accounts.counter.count = 0;
    // !tooltip[/counter.authority = ctx.accounts.payer.key\(\)/] initialAuthority
    ctx.accounts.counter.authority = ctx.accounts.payer.key();

    // !tooltip[/system_program::Transfer/] cpiAccounts
    let cpi_accounts = anchor_lang::system_program::Transfer {
        from: ctx.accounts.payer.to_account_info(),
        to: ctx.accounts.counter.to_account_info(),
    };
    // !tooltip[/CpiContext::new/] cpiContext
    let cpi_ctx = CpiContext::new(anchor_lang::system_program::ID, cpi_accounts);
    // !tooltip[/system_program::transfer/] transfer
    anchor_lang::system_program::transfer(cpi_ctx, HELLO_WORLD_LAMPORTS)?;

    // !tooltip[/msg!/] programLog
    msg!("Hello, world! Counter initialized");
    Ok(())
}
```

### !imports

Αυτό το αρχείο εισάγει τις κοινές σταθερές και τον τύπο λογαριασμού
_rs`Counter`_. Η εντολή χρησιμοποιεί αυτούς τους ορισμούς για να εξάγει το PDA,
να επιλέξει το ποσό μεταφοράς και να εκχωρήσει τον λογαριασμό με το σωστό
μέγεθος δεδομένων.

### !accountsStruct

_rs`Initialize`_ είναι το πλαίσιο λογαριασμών για την εντολή _rs`initialize`_.
Κάθε πεδίο στη δομή αντιπροσωπεύει έναν ξεχωριστό λογαριασμό που πρέπει να
συμπεριληφθεί όταν καλείται η εντολή. Το Anchor χρησιμοποιεί τη δομή για να
επικυρώσει αυτούς τους εισερχόμενους λογαριασμούς πριν εκτελεστεί ο χειριστής.

### !payerMutable

_rs`#[account(mut)]`_ ενημερώνει το Anchor ότι ο λογαριασμός πληρωτή πρέπει να
είναι εγγράψιμος. Αυτό απαιτείται επειδή η δημιουργία λογαριασμού και το CPI
μεταφοράς αφαιρούν lamports από τον πληρωτή.

### !payerSigner

_rs`Signer<'info>`_ ενημερώνει το Anchor ότι ο πληρωτής πρέπει να υπογράψει τη
συναλλαγή.

### !counterConstraints

Το μπλοκ _rs`#[account(...)]`_ ενημερώνει το Anchor πώς να δημιουργήσει και να
επικυρώσει τον λογαριασμό μετρητή πριν εκτελεστεί ο χειριστής.

### !initConstraint

_rs`init`_ λέει στο Anchor να δημιουργήσει τον λογαριασμό _rs`counter`_ πριν
εκτελεστεί ο handler. Ο νέος λογαριασμός ανήκει σε αυτό το πρόγραμμα, οπότε το
πρόγραμμα μπορεί να αποθηκεύσει δεδομένα _rs`Counter`_ σε αυτόν.

### !payerConstraint

_rs`payer = payer`_ ορίζει τον λογαριασμό _rs`payer`_ ως τον πληρωτή για τη
δημιουργία του νέου λογαριασμού μετρητή.

### !space

_rs`space = 8 + Counter::INIT_SPACE`_ δεσμεύει bytes για τον διακριτή
λογαριασμού του Anchor καθώς και για τα πεδία _rs`Counter`_ από το `state.rs`. Ο
διακριτής βοηθά το Anchor να επιβεβαιώσει ότι ο λογαριασμός αποθηκεύει τον
αναμενόμενο τύπο πριν τον διαβάσει.

### !seeds

_rs`seeds = [COUNTER_SEED]`_ ορίζει ότι η διεύθυνση του λογαριασμού μετρητή
πρέπει να είναι το PDA που προκύπτει από αυτό το seed και το αναγνωριστικό
προγράμματος. Με ένα σταθερό seed, υπάρχει ένα PDA μετρητή για αυτό το
πρόγραμμα. Για να δημιουργήσετε ξεχωριστούς μετρητές σε ένα πραγματικό
πρόγραμμα, συνήθως θα προσθέτατε ένα επιπλέον seed, όπως η διεύθυνση ενός
χρήστη.

### !bump

_rs`bump`_ λέει στο Anchor να βρει και να επικυρώσει το bump του PDA, το οποίο
είναι το επιπλέον byte που καθιστά έγκυρη την παραγόμενη διεύθυνση ως PDA.

### !counterAccountType

_rs`Account<'info, Counter>`_ λέει στο Anchor να αντιμετωπίσει τα δεδομένα του
λογαριασμού ως τον τύπο _rs`Counter`_ από το `state.rs`. Το Anchor επαληθεύει
επίσης ότι ο λογαριασμός ανήκει σε αυτό το πρόγραμμα.

### !systemProgram

_rs`Program<'info, System>`_ απαιτεί από τον καλούντα να παρέχει το System
Program. Το Anchor το χρησιμοποιεί για τον περιορισμό δημιουργίας λογαριασμού
_rs`init`_. Το CPI μεταφοράς του handler επικαλείται επίσης το System Program.

### !handler

Ο handler εκτελείται αφού το Anchor επικυρώσει τους λογαριασμούς και
δημιουργήσει τον λογαριασμό μετρητή, οπότε μπορεί να γράψει με ασφάλεια τα
αρχικά δεδομένα του λογαριασμού.

### !initialCount

### increment.rs

`increment.rs` ορίζει τους λογαριασμούς που απαιτούνται για την ενημέρωση ενός

```rust title="programs/my-program/src/instructions/increment.rs" -w
#[derive(Accounts)]
pub struct Increment<'info> {

        // !tooltip[/mut/] counterMutable
        mut,
        // !tooltip[/seeds = \[COUNTER_SEED\]/] counterSeeds
```

```rust title="programs/my-program/tests/test_initialize.rs" -w
    // !tooltip[/Counter::try_deserialize/] deserialize
    let counter_state = my_program::state::Counter::try_deserialize(&mut data).unwrap();
    // !tooltip[/assert_eq!\(counter_state.count, 1\)/] incrementAssert
    assert_eq!(counter_state.count, 1);
    assert_eq!(counter_state.authority, payer.pubkey());
}
```

### !litesvm

Το _rs`LiteSVM`_ είναι ένα ενσωματωμένο Solana VM για δοκιμές. Επιτρέπει στο
τεστ να καταχωρεί το μεταγλωττισμένο πρόγραμμα, να δημιουργεί λογαριασμούς, να
αποστέλλει συναλλαγές και να επιθεωρεί την κατάσταση των λογαριασμών χωρίς να
εκκινεί μια διαδικασία validator.

### !setup

Το τεστ λαμβάνει τη διεύθυνση του προγράμματος από την τιμή _rs`declare_id!`_
μέσω του _rs`my_program::id()`_ και δημιουργεί ένα νέο keypair πληρωτή για τη
συναλλαγή δοκιμής.

### !counterPda

Η διεύθυνση του μετρητή προέρχεται από το ίδιο _rs`COUNTER_SEED`_ που
χρησιμοποιείται από το πρόγραμμα. Αυτό κάνει το τεστ να στοχεύει το PDA που
περιμένει το `initialize.rs`.

### !programBytes

Το _rs`include_bytes!`_ διαβάζει το μεταγλωττισμένο αρχείο `my_program.so` που
παράγεται από την κατασκευή. Αυτά τα bytes είναι ο εκτελέσιμος κώδικας του
προγράμματος που θα εκτελέσει το LiteSVM στο τεστ.

### !addProgram

Το _rs`svm.add_program`_ καταχωρεί τα μεταγλωττισμένα bytes του προγράμματος στη
διεύθυνση του προγράμματος στο LiteSVM. Το airdrop δίνει στον πληρωτή αρκετά
τοπικά SOL για να πληρώσει για τη συναλλαγή και τη δημιουργία λογαριασμού.

### !testInstruction

Το _rs`Instruction::new_with_bytes`_ δημιουργεί μια εντολή για την κλήση του
προγράμματος. Περιλαμβάνει το αναγνωριστικό προγράμματος, instruction data και
λογαριασμούς εντολής.

### !blockhash

Κάθε συναλλαγή Solana χρειάζεται ένα πρόσφατο blockhash. Σε αυτό το τεστ, το
LiteSVM παρέχει ένα blockhash για χρήση στη συναλλαγή.

### !message

Το μήνυμα περιέχει την εντολή, τον πληρωτή τελών και το blockhash. Αυτά είναι τα
δεδομένα συναλλαγής που θα υπογραφούν.

### !transaction

Το _rs`VersionedTransaction::try_new`_ υπογράφει το μήνυμα με το keypair του
πληρωτή. Ο πληρωτής υπογράφει διότι καταβάλλει το τέλος συναλλαγής και
χρηματοδοτεί τον νέο λογαριασμό μετρητή.

### !sendTx

Το _rs`svm.send_transaction`_ εκτελεί τη συναλλαγή στο LiteSVM.

### !getCounter

Μετά την εκτέλεση της συναλλαγής, το τεστ διαβάζει τον λογαριασμό μετρητή
απευθείας από το LiteSVM ώστε να επιθεωρήσει τα δεδομένα του λογαριασμού.

### !deserialize

Ο λογαριασμός μετρητή αποθηκεύει σειριοποιημένα δεδομένα. Το
_rs`Counter::try_deserialize`_ ελέγχει τον διακριτικό χαρακτήρα λογαριασμού του
Anchor και μετατρέπει τα δεδομένα λογαριασμού πίσω στον τύπο _rs`Counter`_ από
το `state.rs`.

### !initAssert

Αφού το _rs`initialize`_ ολοκληρωθεί επιτυχώς, το τεστ αποσειριοποιεί τον
λογαριασμό μετρητή και ελέγχει ότι η αρχική κατάσταση ταιριάζει με αυτό που
έγραψε το _rs`handle_initialize`_.

### !incrementAssert

Αφού το _rs`increment`_ ολοκληρωθεί επιτυχώς, ο ίδιος λογαριασμός μετρητή θα
πρέπει να έχει το _rs`count`_ ορισμένο σε _rs`1`_ διατηρώντας παράλληλα την ίδια
εξουσιοδότηση.

</WithNotes>

## Διαμόρφωση έργου

Τα αρχεία ρίζας του έργου ενημερώνουν το Anchor και το Cargo για το πώς να
κατασκευάσουν, να δοκιμάσουν και να αναπτύξουν το πρόγραμμα. Για πλήρη αναφορά,
δείτε την τεκμηρίωση του Anchor για τη
[διαμόρφωση Anchor.toml](https://www.anchor-lang.com/docs/references/anchor-toml)
και το [Anchor CLI](https://www.anchor-lang.com/docs/references/cli).

<WithNotes>

```toml title="Anchor.toml"
# !tooltip[/skip_local_validator/] skipValidator
skip_local_validator = true

[toolchain]

[features]
resolution = true
skip-lint = false

# !tooltip[/programs.localnet/] programs
[programs.localnet]
my_program = "82sFkffP9wxwpyfZyeaKHH2chQoJPUGsJZSPi9mrUuXd"

# !tooltip[/provider/] provider
[provider]
cluster = "localnet"
wallet = "~/.config/solana/id.json"

# !tooltip[/scripts/] scripts
[scripts]
test = "cargo test"

[hooks]
```

### !skipValidator

Το `skip_local_validator = true` λέει στο Anchor να μην εκκινήσει ένα validator
για το `anchor test`. Η δοκιμή Rust χρησιμοποιεί το LiteSVM.

### !programs

Το `[programs.localnet]` αντιστοιχίζει το όνομα του Rust crate στη διεύθυνση του
program account που χρησιμοποιείται για τοπικές κατασκευές και δοκιμές.

### !provider

Το `[provider]` ορίζει το προεπιλεγμένο cluster και πορτοφόλι για εντολές όπως η
`anchor deploy`.

### !scripts

Το `[scripts]` αλλάζει τι εκτελεί το `anchor test`. Σε αυτό το πρότυπο, το
`anchor test` εκτελεί το `cargo test`.

</WithNotes>
