Οι AI agents γίνονται όλο και πιο αυτόνομοι στις ροές εργασίας τους: γράφουν
κώδικα, ανακτούν δεδομένα, παρέχουν πόρους και πολλά άλλα. Για να λειτουργούν
πλήρως ανεξάρτητα, χρειάζονται έναν τρόπο να πραγματοποιούν «μικροπληρωμές» για
υπηρεσίες προγραμματιστικά, σε πραγματικό χρόνο, με ελάχιστο κόστος. Τα
παραδοσιακά συστήματα πληρωμών δεν λειτουργούν για μικροπληρωμές. Τα τέλη κάτω
του ενός σεντ και η οριστικοποίηση σε κλάσματα δευτερολέπτου του Solana
καθιστούν αυτό τεχνικά και οικονομικά εφικτό.

## Γιατί είναι σημαντικό;

Για παράδειγμα, φανταστείτε έναν developer που χρησιμοποιεί ένα LLM για να
δημιουργήσει μια εφαρμογή και χρειάζεται δεδομένα από ένα API επί πληρωμή. Για
να ανακτήσει κάποια σημαντικά δεδομένα, ένας developer θα έπρεπε να διακόψει τη
ροή εργασίας του, να ερευνήσει APIs, να επιλέξει έναν πάροχο, να δημιουργήσει
λογαριασμό, να προσθέσει μέθοδο πληρωμής, να δημιουργήσει ένα API key, να
αντιγράψει το key στο περιβάλλον του και στη συνέχεια να συνεχίσει τη ροή
εργασίας του. Αυτή η τριβή διακόπτει την κατάσταση ροής. Για ροές εργασίας
υποβοηθούμενες από AI, αποτελεί εμπόδιο.

Αντίθετα, με τις πληρωμές από agents, ο agent μπορεί να κάνει το αίτημα, να
πληρώσει για αυτό και να συνεχίσει τη ροή εργασίας του. Χωρίς λογαριασμούς.
Χωρίς API keys. Χωρίς χειροκίνητη παρέμβαση.

### Πληρώστε για APIs με το pay.sh

Το [pay.sh](https://pay.sh) είναι ένα επίπεδο πληρωμών για HTTP agents και
εργαλεία γραμμής εντολών. Ανατρέξτε στην
[τεκμηρίωση του pay.sh](https://pay.sh/docs) για να εγκαταστήσετε το `pay`. Το
CLI `pay` ενσωματώνει εργαλεία όπως τα `curl`, `codex` και `claude` ώστε να
μπορούν να καλούν APIs επί πληρωμή χωρίς λογαριασμούς παρόχων ή κλειδιά API.
Όταν ένα API επιστρέφει μια πρόκληση MPP ή x402 `402 Payment Required`, το `pay`
ζητά από το τοπικό σας πορτοφόλι να εγκρίνει την υπογραφή.

## Το Πρωτόκολλο x402

Οι πληρωμές για agents χρειάζονται έναν τρόπο για τους clients και τους servers
να διαπραγματεύονται τους όρους πληρωμής μέσω της τυπικής υποδομής του web χωρίς
να διαταράσσουν τις παραδοσιακές διεπαφές ιστού. Ο χώρος των μικροπληρωμών
βρίσκεται ακόμα σε πρώιμο στάδιο και νέα εργαλεία και πρότυπα πιθανώς θα
προκύψουν καθώς ο τομέας εξελίσσεται. Το
[πρωτόκολλο x402](https://www.x402.org/), ωστόσο, έχει αναδειχθεί ως ένα πρώιμο
πρότυπο με ισχυρή υποστήριξη από το οικοσύστημα. Το x402 χρησιμοποιεί τον
[κωδικό κατάστασης HTTP 402 "Payment Required"](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Status/402)—ένα
πρότυπο από την εποχή του HTTP/1.1 που έγινε πρακτικό μόνο με τον διακανονισμό
μέσω blockchain.

### Πώς λειτουργεί

Ο client κάνει ένα αίτημα, λαμβάνει έναν κωδικό 402 με τους όρους πληρωμής και
στη συνέχεια επαναλαμβάνει το αίτημα με μια υπογεγραμμένη πληρωμή. Ο server
αναθέτει την επαλήθευση και τον διακανονισμό σε έναν facilitator, έναν
προαιρετικό μεσάζοντα που χειρίζεται την υποβολή της onchain συναλλαγής. Μόλις ο
facilitator επιβεβαιώσει τον διακανονισμό, ο server επιστρέφει το ζητούμενο
περιεχόμενο. Αυτός ο διαχωρισμός επιτρέπει στους παρόχους API να δέχονται
πληρωμές χωρίς να διαχειρίζονται άμεσα οποιαδήποτε onchain υποδομή.

![Διάγραμμα Ροής x402](/assets/guides/x402/x402-flow-solana.png)

Το πρωτόκολλο είναι stateless (χωρίς sessions ή κλειδιά API), λειτουργεί με την
τυπική υποδομή HTTP και υποστηρίζει οποιοδήποτε SPL token αποδέχεται ο server.
Ακολουθεί ένα απόσπασμα κώδικα για το πώς να ρυθμίσετε έναν Express server
χρησιμοποιώντας x402 middleware:

```ts
app.use(
  paymentMiddleware(
    {
      "GET /costly-data": {
        accepts: [
          {
            scheme: "exact",
            price: "$0.001",
            network: "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
            payTo: svmAddress
          }
        ],
        description: "Costly data",
        mimeType: "application/json"
      }
    },
    new x402ResourceServer(facilitatorClient).register(
      "solana:EtWTRABZaYq6iMfeYKouRu166VU2xqa1",
      new ExactSvmScheme()
    )
  )
);

app.get("/costly-data", (req, res) => {
  res.send({
    report: {
      data: "costly data"
    }
  });
});
```
