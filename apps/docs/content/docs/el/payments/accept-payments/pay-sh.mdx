---
title: Με το Pay.sh
description: Αποδεχτείτε πληρωμές με το Pay.sh.
---

Μπορείτε να αποδέχεστε πληρωμές σε αιτήματα API μέσω του Pay.sh, ρυθμίζοντας
έναν διακομιστή Pay. Εγκαταστήστε το Pay με:

```bash
brew install pay
```

ή

```bash
npm install -g @solana/pay
```

Επαληθεύστε την εγκατάσταση με:

```bash
pay --version
```

Η αλυσίδα εργαλείων θα ρυθμίσει αυτόματα τη διεύθυνσή σας. Εκτελέστε τα παρακάτω
για να ρυθμίσετε και να τροφοδοτήσετε το πορτοφόλι σας:

```bash
pay setup
```

## Ρύθμιση του διακομιστή επίδειξης

Μπορείτε να αρχίσετε να αποδέχεστε πληρωμές εκκινώντας τον διακομιστή σας. Δείτε
έναν διακομιστή επίδειξης σε λειτουργία με:

```bash
pay server demo
```

Αυτό κάνει δύο πράγματα. Εκκινεί έναν ζωντανό διακομιστή Pay τον οποίο μπορείτε
να αναζητήσετε μέσω αιτήματος HTTP, και δημιουργεί ένα αρχείο `pay-demo.yml`
στον τρέχοντα κατάλογό σας.

Όλες οι διαδρομές API διαμορφώνονται μέσω της ρύθμισης YAML `endpoints`. Για να
αποδέχεστε πληρωμές, πρέπει να προσθέσετε ένα χαρακτηριστικό `metering:`. Η
διαδρομή `/api/v1/reports/usage` έχει ρυθμιστεί ως παράδειγμα για εσάς:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Μπορείτε να ελέγξετε ότι οι πληρωμές λειτουργούν εκτελώντας την παρακάτω εντολή
ως πελάτης σε άλλο τερματικό:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Μπορείτε να μάθετε περισσότερα σχετικά με τον τρόπο ορισμού τιμολόγησης στα
[Pay.sh Building with Pay > Pricing docs](https://pay.sh/docs/building-with-pay/pricing)

Μάθετε περισσότερα για τη ρύθμιση του Pay.sh σε παραγωγή στα
[Pay.sh Configuration docs](https://pay.sh/docs/toolchain/configuration).
