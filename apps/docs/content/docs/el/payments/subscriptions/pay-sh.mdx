---

description: Δημιουργήστε συνδρομές με το Pay.sh.
---

Μπορείτε να ρυθμίσετε συνδρομές μέσω του Pay.sh εγκαθιστώντας ένα Pay client στη
συσκευή σας ή ρυθμίζοντας έναν Pay server. Εγκαταστήστε το Pay με:

```bash
brew install pay
```

ή

```bash
npm install -g @solana/pay
```

Επαληθεύστε την εγκατάσταση με:

```bash
pay --version
```

Η αλυσίδα εργαλείων θα ρυθμίσει αυτόματα τη διεύθυνσή σας. Εκτελέστε τα παρακάτω
για να ρυθμίσετε και να τροφοδοτήσετε το πορτοφόλι σας:

```bash
pay setup
```

## Συνδρομές ως client

Οι χρήστες εγγράφονται σε endpoints API που απαιτούν συνδρομή καλώντας το pay
κανονικά. Απόκειται στο API να αποστείλει τις λεπτομέρειες της συνδρομής στην
αρχική απόκριση `402 Status`. Το Pay διαχειρίζεται αυτόματα τη συνδρομή,
συμπεριλαμβανομένης της επίκλησης εντολών στο Solana SPL Subscriptions Program.

Οι χρήστες μπορούν να διαχειριστούν τις συνδρομές τους χρησιμοποιώντας τις
παρακάτω εντολές:

```bash
pay subscriptions                       # default: list + show available subcommands
pay subscriptions list                  # same, scriptable
pay subscriptions list --network mainnet
pay subscriptions list --json
```

Μπορείτε να μάθετε περισσότερα σχετικά με τη διαχείριση των συνδρομών σας ως
χρήστης στα
[Pay.sh > Χρήση του Pay > Έγγραφα Διαχείρισης Συνδρομών](https://pay.sh/docs/using-pay/subscriptions).

## Διαχείριση συνδρομών ως πάροχος (demo)

Οι Pay servers επιτρέπουν τη ρύθμιση παραμέτρων συνδρομητών και των συνδρομών
τους για κάθε endpoint. Παρακάτω παρουσιάζεται ένα παράδειγμα endpoint που
χρησιμοποιεί το pay και μπορείτε να δοκιμάσετε με το έργο σας:

```yaml
# monthly.yml
name: Subscriptions Example
subdomain: subscriptions-example
title: "Subscription Example"
description: "Demo Subscriptions"
category: ai_ml
version: v1
routing:
  type: respond
operator:
  currencies:
    usd: ["USDC", "USDT", "CASH"]
  network: "localnet"
  fee_payer: true
  challenge_binding_secret: d858d9104afd728f58a1f73d806dd75d8f32ebd37c9707145816b490e8631d41
  # insert your own challenge_binding_secret here. use `openssl rand -hex 32`

endpoints:
  - method: GET
    path: "api/v1/pro/feed"
    resource: "pro-feed"
    description: "Pro feed — monthly subscription, 30-day billing period."
    subscription:
      period: "30d"
      price_usd: 9.99
      currency: USDC
      plan_id_numeric: 1
```

Το Pay διαχειρίζεται τις συνδρομές χρησιμοποιώντας το χαρακτηριστικό
`subscription:` στην προδιαγραφή YAML `endpoints`. Αυτό διαφέρει από το
χαρακτηριστικό `metering:` που χρησιμοποιείται για endpoints πληρωμής ανά χρήση.

Η εκκίνηση του server στο παραπάνω παράδειγμα δημιουργεί ένα endpoint με το
όνομα `api/v1/pro/feed` που απαιτεί από τον χρήστη να πραγματοποιεί
επαναλαμβανόμενες πληρωμές 9,99 USDC κάθε 30 ημέρες.

Μπορείτε να το δοκιμάσετε εκτελώντας την παρακάτω εντολή:

```bash
pay server start monthly.yml
```

Η εκτέλεση αυτής της εντολής θα σας ζητήσει να δημιουργήσετε ένα πλάνο. Αυτό
δημιουργεί ένα πλάνο και ενημερώνει το αρχείο yml. Εάν σκοπεύετε να το
χρησιμοποιήσετε ξανά, να θυμάστε τη διεύθυνση του πλάνου και να τη
χρησιμοποιείτε στη ρύθμισή σας καθώς κάνετε αλλαγές.

Μπορείτε να δοκιμάσετε ότι οι συνδρομές λειτουργούν εκτελώντας τις παρακάτω
εντολές ως πελάτης σε άλλο τερματικό:

```bash
# First call — activates the subscription, $9.99 USDC charge settles.
pay --sandbox curl http://127.0.0.1:1402/api/v1/pro/feed

# Same call within 30 days — no payment prompt, just the response.
pay --sandbox curl http://127.0.0.1:1402/api/v1/pro/feed
```

Θα πρέπει να δείτε μια κατάσταση OK ως απόκριση, ως εξής:

```json
{ "status": "ok" }
```

Μπορείτε να μάθετε περισσότερα σχετικά με τη διαχείριση συνδρομών σε έναν
διακομιστή Pay μέσω της
[τεκμηρίωσης Pay.sh > Δημιουργία με Pay > Συνδρομές > Προδιαγραφή YAML](https://pay.sh/docs/building-with-pay/subscriptions/yaml-specification).
