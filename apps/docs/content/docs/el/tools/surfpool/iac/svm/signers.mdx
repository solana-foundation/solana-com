Αυτά τα signers είναι διαθέσιμα κατά τη χρήση του SVM addon για υπογραφή
συναλλαγών.

## secret_key

Το signer `svm::secret_key` μπορεί να χρησιμοποιηθεί για σύγχρονη υπογραφή
συναλλαγής χρησιμοποιώντας ένα μυστικό κλειδί, mnemonic ή αρχείο keypair.

### Είσοδοι

| Name              | Required | Type   | Description                                                                      |
| ----------------- | -------- | ------ | -------------------------------------------------------------------------------- |
| `secret_key`      | optional | string | Το μυστικό κλειδί που χρησιμοποιείται για την υπογραφή μηνυμάτων και συναλλαγών  |
| `mnemonic`        | optional | string | Η φράση mnemonic που χρησιμοποιείται για τη δημιουργία του μυστικού κλειδιού     |
| `derivation_path` | optional | string | Η διαδρομή παραγωγής που χρησιμοποιείται για τη δημιουργία του μυστικού κλειδιού |
| `keypair_json`    | optional | string | Μια διαδρομή προς ένα αρχείο keypair.json που περιέχει το μυστικό κλειδί         |

### Έξοδοι

| Name         | Type   | Description                                   |
| ------------ | ------ | --------------------------------------------- |
| `public_key` | string | Το pubkey του λογαριασμού                     |
| `address`    | string | Η διεύθυνση SVM (ψευδώνυμο για το public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Μπορείτε επίσης να χρησιμοποιήσετε ένα mnemonic:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Ή ένα αρχείο keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

Το signer `svm::web_wallet` θα επιτρέψει σε έναν χειριστή Runbook να υπογράψει
τη συναλλαγή με τον browser signer της επιλογής του.

### Είσοδοι

| Name               | Required | Type   | Description                                                                           |
| ------------------ | -------- | ------ | ------------------------------------------------------------------------------------- |
| `expected_address` | optional | string | Η διεύθυνση SVM που αναμένεται να συνδεθεί (παραλείψτε για να επιτρέψετε οποιαδήποτε) |

### Έξοδοι

| Name         | Type   | Description                              |
| ------------ | ------ | ---------------------------------------- |
| `address`    | string | Η διεύθυνση του συνδεδεμένου λογαριασμού |
| `public_key` | string | Το pubkey του συνδεδεμένου λογαριασμού   |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Αυτός ο υπογράφων θα ζητήσει από τον χρήστη στο Web UI να:

1. Συνδέσει το πορτοφόλι του
2. Παράσχει ένα pubkey μέσω υπογραφής μηνύματος
3. Υπογράψει όλες τις συναλλαγές που απαιτούν αυτόν τον υπογράφοντα

## squads

Ο υπογράφων `svm::squads` μπορεί να χρησιμοποιηθεί για την υπογραφή συναλλαγών
με ένα Squads multisig vault.

### Εισόδοι

| Όνομα     | Απαιτείται  | Τύπος  | Περιγραφή                   |
| --------- | ----------- | ------ | --------------------------- |
| `address` | προαιρετικό | string | Η διεύθυνση του squad vault |

### Έξοδοι

| Όνομα        | Τύπος  | Περιγραφή                                  |
| ------------ | ------ | ------------------------------------------ |
| `public_key` | string | Το pubkey του squad vault                  |
| `address`    | string | Η διεύθυνση SVM (ψευδώνυμο για public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Η χρήση του Squads multisig επιτρέπει την ασφαλή υπογραφή συναλλαγών από πολλά
μέρη χωρίς αποκάλυψη μεμονωμένων ιδιωτικών κλειδιών.
