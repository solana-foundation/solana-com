Το Keychain αφαιρεί την πολυπλοκότητα του backend υπογραφής, αλλά η λειτουργία
ενός υπογράφοντος σε παραγωγή παραμένει δική σας ευθύνη. Αυτές οι πρακτικές
εφαρμόζονται σε όλα τα backends· δείτε το
[Επιλογή backend](/docs/tools/keychain/choosing-a-backend) για την επιλογή
κατάλληλου.

## Διαχωρισμός hot και cold υπογραφόντων

Χρησιμοποιήστε έναν λειτουργικό υπογράφοντα χαμηλού υπολοίπου ("hot") για
συχνές, αυτοματοποιημένες συναλλαγές, και έναν ξεχωριστό υπογράφοντα υψηλής
αξίας ("cold") για κινήσεις ταμείου. Υποστηρίξτε τον cold υπογράφοντα με MPC
custody ή cloud HSM και απαιτήστε πολιτικές έγκρισης πριν την υπογραφή. Επειδή η
διεπαφή είναι πανομοιότυπη, κάθε ρόλος μπορεί να χρησιμοποιεί διαφορετικό
backend χωρίς αλλαγές στα σημεία κλήσης.

## Εναλλαγή κλειδιών με πρόβλεψη

Στο Solana, το κλειδί υπογραφής _είναι_ η διεύθυνση του λογαριασμού, οπότε η
εναλλαγή κλειδιού παράγει νέα διεύθυνση. Σχεδιάστε την εναλλαγή ως
μετεγκατάσταση: δημιουργήστε νέο κλειδί, εκχωρήστε εκ νέου τις σχετικές
αρμοδιότητες (ή μεταφέρετε κεφάλαια) στη νέα διεύθυνση, και στη συνέχεια
αποσύρετε το παλιό. Χρησιμοποιήστε διαφορετικά κλειδιά ανά περιβάλλον, ώστε τα
κλειδιά devnet να μην επαναχρησιμοποιούνται ποτέ στο mainnet.

## Παραχώρηση πρόσβασης ελάχιστων προνομίων

Περιορίστε τα διαπιστευτήρια κάθε backend στο ελάχιστο απαραίτητο για την
υπογραφή. Για το AWS KMS, αυτό είναι `kms:Sign` και `kms:DescribeKey` στο
συγκεκριμένο κλειδί· προτιμήστε IAM roles έναντι στατικών διαπιστευτηρίων.
Εφαρμόστε την ίδια αρχή στις πολιτικές Vault και στα API keys διαχειριζόμενου
πορτοφολιού.

## Σχεδιασμός για διαθεσιμότητα

Τα απομακρυσμένα backends εξαρτώνται από μια υπηρεσία δικτύου. Καλέστε το
`isAvailable()` πριν βασιστείτε σε έναν υπογράφοντα, εφαρμόστε επαναλήψεις με
backoff σε παροδικές αποτυχίες, και ορίστε εναλλακτική διαδρομή για διακοπές. Το
Keychain επιβάλλει HTTPS για απομακρυσμένα backends και εφαρμόζει προεπιλεγμένο
timeout αιτήματος.

## Προστατέψτε τα μυστικά και τη μεταφορά

- Όλα τα απομακρυσμένα backends επικοινωνούν μέσω HTTPS· τα endpoints που δεν
  χρησιμοποιούν HTTPS απορρίπτονται.
- Παρέχετε κλειδιά API, tokens και διαπιστευτήρια από έναν secret manager ή
  μεταβλητές περιβάλλοντος — μην τα hard-code-άρετε ποτέ και μην τα αποθηκεύετε
  στο source control.
- Το Keychain εξυγιαίνει τις αποκρίσεις σφαλμάτων για να αποφύγει τη διαρροή
  μυστικών στα logs· διατηρήστε τα δικά σας logs ελεύθερα από ανεπεξέργαστα
  διαπιστευτήρια και υλικό κλειδιών.

## Παρακολουθήστε και ελέγξτε την υπογραφή

Παρακολουθήστε τον ρυθμό επιτυχίας υπογραφής, την καθυστέρηση και τον αριθμό
αποτυχιών, και ορίστε ειδοποιήσεις για αιχμές — δείτε τις μετρήσεις στην ενότητα
[Ετοιμότητα Παραγωγής](/docs/payments/production-readiness#monitoring).
Ενεργοποιήστε τα audit logs στην πλευρά του παρόχου (KMS, Vault ή θεματοφύλακα)
και συγκρίνετέ τα με τις συναλλαγές που υποβάλλει η εφαρμογή σας.
