## Ξεκινώντας

```rust
let mut svm = LiteSVM::new();
```

Αυτό δημιουργεί τη βασική δοκιμαστική παρουσία litesvm, η οποία περιλαμβάνει
όλες τις ενεργοποιημένες λειτουργίες χρόνου εκτέλεσης, προεπιλεγμένες sysvars,
precompiles, προγράμματα spl, sigverify, και όλα τα ενσωματωμένα προγράμματα
όπως το System Program.

Για να αλληλεπιδράσετε με οποιοδήποτε άλλο πρόγραμμα στις δοκιμές σας, πρέπει να
αναπτύξετε αυτό το πρόγραμμα στο περιβάλλον δοκιμών σας.

1. Δημιουργήστε και αναπτύξτε το πρόγραμμα που θέλετε να δοκιμάσετε

2. Βρείτε όλα τα προγράμματα στα οποία το παραπάνω πρόγραμμα πραγματοποιεί
   κλήσεις CPI και αναπτύξτε αυτά τα προγράμματα

<Callout type="info">
  **Παράδειγμα:** Εάν το πρόγραμμά σας χρησιμοποιεί ροές δεδομένων από το Pyth,
  θα χρειαστεί να έχετε το πρόγραμμα Pyth αναπτυγμένο στη δοκιμαστική παρουσία
  σας.
</Callout>

## Βασική Ανάπτυξη Προγράμματος

### Μέθοδος 1: Ανάπτυξη από Bytes

Αναπτύξτε προγράμματα απευθείας από πίνακες bytes για την ταχύτερη εκτέλεση
δοκιμών:

```rust
use litesvm::LiteSVM;
use solana_sdk::signature::{read_keypair_file, Signer};

let mut svm = LiteSVM::new();

// Read the program's keypair to get correct ID
let program_keypair = read_keypair_file("target/deploy/my_program-keypair.json")
    .expect("Program keypair file not found");
let program_id = program_keypair.pubkey();

// Include bytes at compile time
let program_bytes = include_bytes!("../target/deploy/my_program.so");

// Deploy from bytes
svm.add_program(program_id, program_bytes)
    .expect("Failed to deploy program");

// Verify deployment
assert!(svm.get_account(&program_id).is_some(), "Program account not created");
assert!(svm.get_account(&program_id).unwrap().executable, "Program not executable");
```

`include_bytes!` ενσωματώνει το δυαδικό αρχείο απευθείας στο εκτελέσιμο δοκιμής
κατά τη στιγμή της μεταγλώττισης, εξαλείφοντας τις λειτουργίες I/O κατά την
εκτέλεση των δοκιμών.

### Μέθοδος 2: Ανάπτυξη από Αρχείο (.so)

Φορτώστε προγράμματα από το σύστημα αρχείων όταν αλλάζουν συχνά:

```rust
use litesvm::LiteSVM;
use solana_sdk::signature::{read_keypair_file, Signer};

let mut svm = LiteSVM::new();

// Read keypair for correct program ID
let program_keypair = read_keypair_file("target/deploy/program-keypair.json").unwrap();
let program_id = program_keypair.pubkey();

// Deploy from file
svm.add_program_from_file(program_id, "target/deploy/program.so")
    .expect("Failed to deploy program from file");

// Always verify
assert!(svm.get_account(&program_id).unwrap().executable);
```

## Λήψη Προγραμμάτων από το Mainnet/Devnet

Όταν χρειάζεστε ένα πρόγραμμα για το οποίο δεν έχετε τον κώδικα, χρησιμοποιήστε
το Solana CLI για εξαγωγή προγραμμάτων από ενεργά clusters και χρησιμοποιήστε τα
σε δοκιμές:

```bash
# Dump a program from mainnet
solana program dump gSbePebfvPy7tRqimPoVecS2UsBvYv46ynrzWocc92s pyth.so --url mainnet-beta

# Dump from devnet
solana program dump YourProgramID program.so --url devnet
```

Στη συνέχεια φορτώστε το στις δοκιμές σας:

```rust
svm.add_program_from_file(
    pubkey!("gSbePebfvPy7tRqimPoVecS2UsBvYv46ynrzWocc92s"),
    "pyth.so"
).unwrap();
```

## Βέλτιστες Πρακτικές

### 1. Επιλέξτε τη Σωστή Μέθοδο Ανάπτυξης

- **Χρησιμοποιήστε `add_program()`** όταν:
  - Τα προγράμματα είναι ενσωματωμένα στο δυαδικό αρχείο δοκιμών κατά τη
    μεταγλώττιση
  - Θέλετε ταχύτερη εκτέλεση δοκιμών (χωρίς I/O κατά το χρόνο εκτέλεσης)
  - Αυτόνομες δοκιμές

- **Χρησιμοποιήστε το `add_program_from_file()`** όταν:
  - Τα προγράμματα δημιουργούνται ξεχωριστά (δηλ. ανάκτηση από το devnet)
  - Το πρόγραμμα αλλάζει συχνά κατά την ανάπτυξη (δυνατότητα επανεκτέλεσης
    δοκιμών χωρίς επαναμεταγλώττιση)
  - Διαθέτετε περιβάλλοντα CI/CD όπου τα αρχεία .so είναι artifacts

### Κατανόηση των keypair Προγραμμάτων

Όταν δημιουργείτε ένα πρόγραμμα Solana:

1. Το **Anchor** δημιουργεί `target/deploy/program-keypair.json`
2. Οι **Native** builds δημιουργούν ένα αρχείο keypair δίπλα στο αρχείο `.so`
3. Η **διεύθυνση on-chain** του προγράμματος είναι το public key αυτού του
   keypair, γνωστό επίσης ως program ID.
4. Για **PDAs**, το `find_program_address()` απαιτεί το program ID ως seed
5. Για **CPIs**, το program ID χρησιμοποιείται για την κλήση άλλων προγραμμάτων

## Συνηθισμένα Προβλήματα και Λύσεις

### Αθόρυβες Αποτυχίες Ανάπτυξης

**Πρόβλημα**: Το πρόγραμμα φαίνεται να αναπτύσσεται αλλά οι συναλλαγές
αποτυγχάνουν με το μήνυμα "InvalidProgramForExecution".

**Αιτία**: Το program ID δεν ταιριάζει με το keypair.

**Λύση**:

```rust
// Always use the keypair-derived ID
let keypair = read_keypair_file("target/deploy/program-keypair.json").unwrap();
let program_id = keypair.pubkey();

svm.add_program(program_id, &program_bytes).unwrap();

// Verify with detailed checks
let account = svm.get_account(&program_id)
    .expect("Program account doesn't exist");

assert!(account.executable, "Program not executable");
assert_eq!(account.owner, solana_sdk::bpf_loader::id(), "Wrong owner");
assert!(!account.data.is_empty(), "Program data is empty");

println!("✅ Program verified at {}", program_id);
```

### Σφάλμα: Το Πρόγραμμα Δεν Βρέθηκε

**Σφάλμα**: `TransactionError::InvalidProgramForExecution`

**Συνηθισμένες Αιτίες**:

1. Το πρόγραμμα δεν έχει αναπτυχθεί
2. Χρησιμοποιήθηκε λάθος program ID στην εντολή
3. Αναντιστοιχία program ID με keypair

**Βήματα Εντοπισμού Σφαλμάτων**:

```rust
// Step 1: Check if program exists
if let Some(account) = svm.get_account(&program_id) {
    println!("Program exists");
    println!("  Executable: {}", account.executable);
    println!("  Owner: {}", account.owner);
    println!("  Data length: {}", account.data.len());
} else {
    println!("Program does not exist at {}", program_id);
}

// Step 2: Verify the ID matches keypair
let expected_keypair = read_keypair_file("target/deploy/program-keypair.json").unwrap();
let expected_id = expected_keypair.pubkey();

if program_id != expected_id {
    println!("  ID mismatch!");
    println!("  Used: {}", program_id);
    println!("  Expected: {}", expected_id);
}

// Step 3: Check instruction program ID
println!("Instruction program_id: {}", instruction.program_id);
assert_eq!(instruction.program_id, program_id, "Instruction uses wrong program ID");
```

### CPI σε Μη Αναπτυγμένα Προγράμματα

**Πρόβλημα**: Το πρόγραμμά σας πραγματοποιεί μια κλήση CPI σε ένα πρόγραμμα που
δεν έχει αναπτυχθεί.

**Λύση**: Αναπτύξτε όλα τα προγράμματα εξαρτήσεων:

```rust
// Find all dependency programs and make sure they are deployed
svm.add_program_from_file(
    pubkey!("gSbePebfvPy7tRqimPoVecS2UsBvYv46ynrzWocc92s"),
    "pyth.so"
).unwrap();

```

Αν χρειάζεται να ανακτήσετε το πρόγραμμα από ένα cluster, ανατρέξτε στην
[ενότητα αυτή](#pulling-programs-from-mainnetdevnet).

### Ανεπαρκή lamport για Ανάπτυξη

**Σημείωση**: Το LiteSVM χειρίζεται αυτόματα το rent-exempt υπόλοιπο για τα
προγράμματα.

```rust
// No need to manually fund - LiteSVM handles this
svm.add_program(program_id, &program_bytes).unwrap();

// But you can check the rent if needed
let program_len = program_bytes.len();
let required_lamports = svm.minimum_balance_for_rent_exemption(program_len);
println!("Program requires {} lamports for rent exemption", required_lamports);
```
