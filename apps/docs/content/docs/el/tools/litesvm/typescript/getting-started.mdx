# LiteSVM για TypeScript

Οι σουίτες δοκιμών Solana που εκκινούν `solana-test-validator` χρειάζονται
δευτερόλεπτα ανά δοκιμή — περιμένετε την εκκίνηση διεργασιών, τις συνδέσεις RPC
και τις μεταδόσεις δικτύου. Το LiteSVM εκτελεί τις ίδιες δοκιμές σε χιλιοστά του
δευτερολέπτου. Χωρίς εξωτερική διεργασία, χωρίς δίκτυο, τα πάντα στη μνήμη μέσα
στο test runner σας.

<Callout type="info">
  **Δοκιμάστηκε με:** `@solana/kit@6.8.0`, `@solana/kit-plugin-litesvm@0.10.0`,
  `@solana/kit-plugin-signer@0.10.0`, `@solana-program/system@0.12.0`,
  `@solana-program/token@0.13.0`. Αυτά τα πακέτα plugin είναι νέα και
  εξελίσσονται γρήγορα — αν αντιμετωπίσετε αναντιστοιχία, ελέγξτε το
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  για τις ακριβείς εκδόσεις που χρησιμοποιούν τα έγγραφα και η σουίτα δοκιμών.
</Callout>

Συναρμολογείτε ένα Kit client από `@solana/kit` εγκαθιστώντας plugins με
`.use()`: ένα plugin υπογραφής
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
το plugin LiteSVM
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm)),
και όποια plugin προγραμμάτων χρειάζεστε. Το αποτέλεσμα εκθέτει
`client.sendTransaction()`, `client.rpc`, και άμεση πρόσβαση στην κατάσταση
λογαριασμού, το ρολόι και τα sysvars — όλα χωρίς να εγκαταλείπετε τη διεργασία
σας.

### Δημιουργία του Client σας

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  Το `createClient()` επιστρέφει ένα κενό κέλυφος — κάθε `.use(plugin)`
  προσθέτει δυνατότητες σε αυτό. Το plugin `signer()` πρέπει να έρχεται **πριν**
  από το `litesvm()` επειδή το plugin μεταφοράς χρειάζεται έναν payer για να
  συνδεθεί. Το `signer(mySigner)` είναι συντομογραφία για την εγκατάσταση του
  ίδιου keypair τόσο ως `payer` όσο και ως `identity` — χρησιμοποιήστε το
  `payer()` ή το `identity()` από το ίδιο πακέτο αν θέλετε να διαχωρίσετε αυτούς
  τους ρόλους. Αυτή η σειρά είναι ίδια για το `solanaRpc()` όταν βρίσκεστε σε
  πραγματικό δίκτυο.
</Callout>

<Callout type="info">
  Προτιμάτε γρήγορη ρύθμιση σε μία γραμμή για γρήγορα πειράματα; Το πακέτο
  `litesvm()` περιλαμβάνει airdrop, οπότε μπορείτε να αλυσιδώσετε
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` από
  `@solana/kit-plugin-signer` για αυτόματη δημιουργία και χρηματοδότηση ενός
  payer.
</Callout>

</Step>

</Steps>

## Κατανοώντας τον Client

Μετά από `.use(signer(...)).use(litesvm())`, ο client εκθέτει:

| Ιδιότητα                    | Περιγραφή                                                                                                                                                                                                 |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | Το `TransactionSigner` που εγκαταστήσατε με το plugin `signer()` (ορίζεται επίσης ως `client.identity`)                                                                                                   |
| `client.identity`           | Ο υπογράφων του πορτοφολιού που κατέχει λογαριασμούς, tokens και εξουσιοδοτήσεις                                                                                                                          |
| `client.svm`                | Άμεση πρόσβαση στο [`LiteSVM`](https://www.npmjs.com/package/litesvm) για διαχείριση λογαριασμών, συναλλαγές και διαμόρφωση                                                                               |
| `client.rpc`                | Διεπαφή RPC συμβατή με Kit (υποσύνολο: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Σχεδιασμός, υπογραφή και αποστολή εντολών ως συναλλαγή                                                                                                                                                    |
| `client.airdrop`            | Αίτηση SOL από το faucet του LiteSVM                                                                                                                                                                      |
| `client.getMinimumBalance`  | Υπολογισμός ελάχιστων lamports για εξαίρεση από rent                                                                                                                                                      |

### Χρήση του client.svm

Άμεση πρόσβαση στο LiteSVM για:

- Διαχείριση λογαριασμών (`setAccount`, `getAccount`, `airdrop`)
- Εκτέλεση συναλλαγών (`sendTransaction`, `simulateTransaction`)
- Διαμόρφωση (`withSigverify`, `withBlockhashCheck`, κ.λπ.)
- Χειρισμός χρόνου (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### Χρήση του client.rpc

Διεπαφή RPC συμβατή με Kit για τυπικές λειτουργίες:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  Το επίπεδο RPC υποστηρίζει μόνο **κωδικοποίηση base64**. Η αίτηση άλλων
  κωδικοποιήσεων θα προκαλέσει σφάλμα.
</Callout>

## Διαμόρφωση

Ρυθμίστε το LiteSVM χρησιμοποιώντας μεθόδους builder στο `client.svm`:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## Αποστολή Συναλλαγών

Προσθέστε plugins προγράμματος μετά το transport — αναρτούν builders εντολών
στον client ώστε να μην χρειάζεται ποτέ να αγγίξετε ακατέργαστο
`TransactionMessage`:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## Ροή SVM → RPC

Μπορείτε να ορίσετε κατάσταση μέσω SVM και να την διαβάσετε μέσω RPC:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```

## Επόμενα Βήματα

Αυτό καλύπτει τον βασικό client — δημιουργία λογαριασμών, αποστολή εντολών και
ανάγνωση κατάστασης. Εάν οι δοκιμές σας περιλαμβάνουν SPL tokens (mints, ATAs,
μεταφορές), ο [οδηγός δοκιμής tokens](/docs/tools/litesvm/typescript/tokens)
δείχνει πώς το plugin `tokenProgram()` συμπυκνώνει ~40 γραμμές ρύθμισης σε 3
κλήσεις. Για πλήρη αναφορά API, δείτε τα
[έγγραφα API](/docs/tools/litesvm/typescript/api-reference).
