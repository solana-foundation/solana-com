`@solana/kit` παρέχει `createClient`, έναν μικρό host που επεκτείνετε με
plugins. Κάθε plugin προσθέτει μια δυνατότητα — σύνδεση RPC, signer, σχεδιασμός
συναλλαγών — και ο σύνθετος client υποστηρίζει scripts, διαδρομές server και UIs
εξίσου.

<Callout type="info">
  Δημιουργείτε εφαρμογή React; Ο [οδηγός React](/docs/frontend/react-hooks)
  τυλίγει τον ίδιο client με `@solana/react` ώστε τα components να διαβάζουν την
  κατάσταση wallet και δεδομένα RPC μέσω hooks. Ξεκινήστε από εκεί και
  επιστρέψτε εδώ όταν χρειάζεστε μεγαλύτερο έλεγχο.
</Callout>

## Εγκατάσταση

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer
```

Χρησιμοποιήστε οποιονδήποτε package manager. Το Kit είναι v7+ και τα πακέτα
`@solana/kit-plugin-*` είναι 0.13+.

## Δημιουργία client

Συνδυάστε plugins με `.use(...)`. Ορίστε πρώτα έναν signer, στη συνέχεια τη
σύνδεση RPC. Το `solanaDevnetRpc()` προσθέτει `client.rpc`, subscriptions, τον
planner/executor συναλλαγών και το βοηθητικό `client.sendTransaction` — οπότε
μια εφαρμογή σπάνια χρειάζεται να συνδέσει αυτά τα κομμάτια μόνη της.

```ts
import { createClient, generateKeyPairSigner } from "@solana/kit";
import { solanaDevnetRpc } from "@solana/kit-plugin-rpc";
import { signer } from "@solana/kit-plugin-signer";

const payer = await generateKeyPairSigner();

const client = createClient().use(signer(payer)).use(solanaDevnetRpc());

const slot = await client.rpc.getSlot().send();
console.log("current slot", slot);
```

Το plugin `signer` ορίζει τόσο τον fee payer όσο και την ταυτότητα της
εφαρμογής. Το RPC bundle απαιτεί να υπάρχει ήδη ένας payer, γι' αυτό η σειρά
έχει σημασία.

### Παραλλαγές Signer

Το `@solana/kit-plugin-signer` εκθέτει τρεις ρόλους ώστε να παραχωρείτε μόνο
ό,τι χρειάζεται ένας client:

- `signer(x)` — ορίζει τόσο το `client.payer` όσο και το `client.identity`.
- `payer(x)` — μόνο fee payer.
- `identity(x)` — μόνο app authority.

Κάθε ένα διαθέτει παραλλαγές `*FromFile(path)` (Node), `generated*()`,
`generated*WithSol(amount)` και `airdrop*()` για δοκιμές και scripts.

## Αποστολή συναλλαγής

Δημιουργήστε εντολές με έναν παραγόμενο program client και στη συνέχεια
παραδώστε τις στο `client.sendTransaction`. Ο planner επιλύει το blockhash,
εκτιμά το compute, υπογράφει με τον payer του client και υποβάλλει.

```ts
import { getTransferSolInstruction } from "@solana-program/system";
import { address, lamports } from "@solana/kit";

const transfer = getTransferSolInstruction({
  source: client.payer,
  destination: address("Fke...address"),
  amount: lamports(10_000_000n) // 0.01 SOL
});

const { context } = await client.sendTransaction([transfer]);
console.log("submitted", context.signature);
```

Το `sendTransaction` επιστρέφει ένα αντικείμενο αποτελέσματος· η υπογραφή
συναλλαγής βρίσκεται στο `result.context.signature`. Επιβεβαιώνει ότι το plan
χωράει σε μία μόνο συναλλαγή και σε διαφορετική περίπτωση εκπέμπει σφάλμα —
χρησιμοποιήστε το `sendTransactions` όταν μια δέσμη ενδέχεται να εκτείνεται σε
πολλαπλές συναλλαγές.

## Επιλέξτε ένα πακέτο RPC

Το `@solana/kit-plugin-rpc` περιλαμβάνει προεπιλογές cluster:

- `solanaDevnetRpc()` — προεπιλογές devnet, με ενεργοποιημένο airdrop
  (`client.airdrop`).
- `solanaMainnetRpc()` — mainnet· το airdrop αποτελεί σφάλμα κατά τη
  μεταγλώττιση.
- `solanaLocalRpc()` — `http://127.0.0.1:8899` για τοπικό validator.
- `solanaRpc({ rpcUrl })` — οποιοδήποτε endpoint.

```ts
import { solanaRpc } from "@solana/kit-plugin-rpc";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpc({ rpcUrl: "https://your-rpc-provider.example" }));
```

### Σύνθεση σε λεπτομερές επίπεδο

Τα προκαθορισμένα πακέτα συνδυάζουν τη σύνδεση RPC, τον planner, τον executor
και τα βοηθητικά αποστολής, οπότε οι περισσότερες εφαρμογές δεν χρειάζονται ποτέ
τα επιμέρους στοιχεία. Χρησιμοποιήστε τα granular plugins μόνο όταν θέλετε να
προσαρμόσετε ένα από αυτά — για παράδειγμα έναν custom planner ή διαφορετικό
κανάλι συνδρομής. Αυτή η προσέγγιση απαιτεί επίσης
`@solana/kit-plugin-instruction-plan`, το οποίο τα προκαθορισμένα πακέτα
συμπεριλαμβάνουν αυτόματα:

```ts
import {
  solanaRpcConnection,
  rpcAirdrop,
  rpcTransactionPlanner,
  rpcTransactionPlanExecutor
} from "@solana/kit-plugin-rpc";
import { planAndSendTransactions } from "@solana/kit-plugin-instruction-plan";

const client = createClient()
  .use(signer(payer))
  .use(solanaRpcConnection({ rpcUrl: "https://api.devnet.solana.com" }))
  .use(rpcAirdrop())
  .use(rpcTransactionPlanner())
  .use(rpcTransactionPlanExecutor())
  .use(planAndSendTransactions());
```

## Προσθήκη clients προγράμματος

Τα παραγόμενα πακέτα `@solana-program/*` εκθέτουν client plugins ώστε οι κλήσεις
προγράμματος να συνδέονται με τον client. Εγκαταστήστε το πρόγραμμα που
χρειάζεστε και `.use()` το plugin του:

```ts
import { tokenProgram } from "@solana-program/token";

const client = createClient()
  .use(signer(payer))
  .use(solanaDevnetRpc())
  .use(tokenProgram()); // adds client.token
```

<Callout type="warn" title="Peer dependency ranges">
  Ορισμένες εκδόσεις `@solana-program/*` εξακολουθούν να δηλώνουν peer
  `@solana/kit ^6.x` ενώ το Kit βρίσκεται στην έκδοση 7.x. Λειτουργούν με το Kit
  7 — παρακάμψτε το εύρος peer στον package manager σας αν εμφανιστεί παράπονο.
</Callout>

## Υπογραφή χωρίς πορτοφόλι browser

Εκτελείτε headless σε ένα API route, worker ή script; Υπογράψτε με μια υπηρεσία
διαχείρισης κλειδιών backend αντί για ένα άμεσο keypair. Δείτε
[Keychain](/docs/tools/keychain) και
[Υπογραφή σε Παραγωγή](/docs/core/transactions/signing-in-production).

## Συνήθη μοτίβα για προγραμματιστές Solana

- **Κοινή διαμόρφωση**: Δημιουργήστε τον client μία φορά σε ένα βασικό module
  και εισάγετέ τον σε scripts, server routes και το UI σας.
- **Αναγνώσεις από την πλευρά του server**: Ένας client με μόνο `signer` +
  `solanaRpc` (χωρίς wallet plugin) διαβάζει λογαριασμούς και προσομοιώνει
  συναλλαγές από τον server.
- **Τοπικές δοκιμές**: Αντικαταστήστε το πακέτο RPC με `.use(litesvm())` από το
  `@solana/kit-plugin-litesvm` για εκτέλεση σε έναν in-memory validator (μόνο
  Node).
- **Δυνατότητα δοκιμών**: Ο σύνθετος client είναι εύκολο να γίνει mock — ορίστε
  stub για RPC responses ή signers χωρίς να απαιτείται wallet browser.
