  Πώς οι έννοιες του @solana/web3.js και του wallet-adapter αντιστοιχίζονται στο
  @solana/kit.
---

Αν προέρχεστε από το `@solana/web3.js` (v1), το `@solana/wallet-adapter-*`, ή τα
παλαιότερα πακέτα framework-kit (`@solana/client`, `@solana/react-hooks`,
`@solana/web3-compat`), αυτές οι προσεγγίσεις έχουν αντικατασταθεί. Οι νέες
εφαρμογές δομούνται απευθείας πάνω στο `@solana/kit` και τα plugins του kit.
Αυτή η σελίδα αντιστοιχίζει τις έννοιες που ήδη γνωρίζετε στα ισοδύναμά τους στο
Kit.

## Αντιστοίχιση εννοιών

| web3.js v1 / wallet-adapter            | Kit                                                                                |
| -------------------------------------- | ---------------------------------------------------------------------------------- |
| `new Connection(url)`                  | `createSolanaRpc(url)`, ή `.use(solanaRpc({ rpcUrl }))` σε ένα client              |
| `PublicKey`                            | `Address` — μετατροπή string με `address("...")`                                   |
| `Keypair`                              | `generateKeyPairSigner()` ή τα plugins `@solana/kit-plugin-signer`                 |
| `SystemProgram.transfer(...)`          | `getTransferSolInstruction(...)` από το `@solana-program/system`                   |
| `LAMPORTS_PER_SOL` math                | `lamports(...)` από το `@solana/kit`                                               |
| `Transaction` / `VersionedTransaction` | Μηνύματα συναλλαγών που δομούνται και συσκευάζονται από τον transaction planner    |
| `sendAndConfirmTransaction(...)`       | `client.sendTransaction([...])` (ο planner υπογράφει, αποστέλλει και επιβεβαιώνει) |
| `@solana/wallet-adapter-*`             | `@solana/kit-plugin-wallet` + ανακάλυψη μέσω Wallet Standard                       |

## Έρχεστε από το web3.js;

Αν συντηρείτε μια μεγάλη κωδικοβάση `@solana/web3.js` v1 και μια πλήρης
επανεγγραφή σε Kit δεν είναι ακόμα πρακτική, το web3.js v3 αποτελεί τη γέφυρα.
Αυτή τη στιγμή αποστέλλεται ως `@solana/web3.js@rc` (η σειρά `3.0.0-rc`) και
ανακατασκευάζει το κλασικό API βασισμένο σε κλάσεις — `Connection`, `Keypair`,
`Transaction` — πάνω στα εσωτερικά του `@solana/kit`. Επειδή βασίζεται στο Kit,
η διαλειτουργικότητα είναι στενή: το `PublicKey` είναι ένα deprecated alias του
`Address`, και ένα `Keypair` v3 ικανοποιεί δομικά το `KeyPairSigner` του Kit,
οπότε μπορείτε να το περνάτε απευθείας σε Kit APIs, plugins και clients που
δημιουργούνται από το Codama.

Το v3 είναι ακόμα **υποψήφια έκδοση (release candidate)** — καρφώστε ακριβείς
εκδόσεις και αναμένετε κάποιες αλλαγές στο API μεταξύ των RC. Πρόκειται για ένα
μονοπάτι legacy-interop, όχι για την προεπιλογή για νέες εργασίες· νέες
εφαρμογές πρέπει να δομούνται απευθείας στο Kit. Για την ίδια τη μετάβαση,
ανατρέξτε στον επίσημο
[οδηγό μετάβασης web3.js v1 → v3](https://github.com/solana-foundation/solana-web3.js/blob/v3.x/docs/web3js-v1-to-v3-migration.md).

## Πορτοφόλια

Το Kit δεν χρησιμοποιεί wallet-adapter. Το `@solana/kit-plugin-wallet`
ανακαλύπτει πορτοφόλια Wallet Standard και τα εκθέτει μέσω του `client.wallet`
και των [React hooks](/docs/frontend/react-hooks). Τα σύγχρονα πορτοφόλια
(Phantom, Solflare, Backpack και άλλα) διαφημίζουν τον εαυτό τους μέσω του
Wallet Standard, οπότε δεν απαιτείται πακέτο προσαρμογέα ανά πορτοφόλι.

Μπορείτε επίσης να χρησιμοποιήσετε προκατασκευασμένα εργαλεία όπως το
[@solana/connector](https://www.npmjs.com/package/@solana/connector), το οποίο
χρησιμοποιεί απευθείας τα primitives του Kit για σύνδεση με πορτοφόλια.

<Callout type="info" title="Χρησιμοποιείτε ακόμα wallet-adapter;">
  Αν διαχειρίζεστε μια εφαρμογή που έχει κατασκευαστεί στο
  `@solana/wallet-adapter-*` και δεν είστε έτοιμοι να μεταβείτε, το [αποθετήριο
  wallet-adapter ](https://github.com/anza-xyz/wallet-adapter) παραμένει η
  αναφορά για αυτό το stack. Για οτιδήποτε νέο, προτιμήστε το plugin πορτοφολιού
  του Kit.
</Callout>
