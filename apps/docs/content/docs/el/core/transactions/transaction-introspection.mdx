<Callout type="info" title="Σύνοψη">
  Το `@solana/transaction-introspection` μετατρέπει μια απόκριση RPC
  `getTransaction` σε μια λίστα οδηγιών με πραγματικές διευθύνσεις. Επιλύει
  δείκτες λογαριασμών (συμπεριλαμβανομένων διευθύνσεων πίνακα αναζήτησης),
  αποκωδικοποιεί εσωτερικές οδηγίες Cross Program Invocation και διατρέχει το
  πλήρες δέντρο οδηγιών με τη σειρά εξερεύνησης.
</Callout>

<Callout type="info">
  Αναζητάτε πώς ένα πρόγραμμα επιθεωρεί συγγενικές οδηγίες _on-chain_ κατά το
  χρόνο εκτέλεσης; Αυτό είναι το sysvar Οδηγιών - δείτε [Ανάλυση
  Οδηγιών](/docs/core/instructions/instruction-introspection). Αυτή η σελίδα
  αφορά την αποκωδικοποίηση μιας συναλλαγής _off-chain_ από μια απόκριση RPC.
</Callout>

Μια απόκριση [`getTransaction`](/docs/rpc/http/gettransaction) δεν σας δίνει μια
έτοιμη λίστα οδηγιών. Οι λογαριασμοί αποθηκεύονται ως αριθμητικοί δείκτες, οι
[εκδοχές συναλλαγών](/docs/core/transactions/versioned-transactions) χωρίζουν
αυτούς τους λογαριασμούς σε στατικά κλειδιά και διευθύνσεις που φορτώνονται από
[Πίνακες Αναζήτησης Διευθύνσεων](/docs/core/transactions/versioned-transactions),
και εσωτερικές οδηγίες από Cross Program Invocation (CPIs) φτάνουν ως
κωδικοποιημένα blobs στα μεταδεδομένα της συναλλαγής.

Το `@solana/transaction-introspection` κάνει αυτή την επίλυση για εσάς.
Αποκωδικοποιεί τη συναλλαγή wire, αντιστοιχίζει κάθε δείκτη λογαριασμού στο
`Address` του, κανονικοποιεί τις εσωτερικές οδηγίες και επιστρέφει οδηγίες με
την ίδια σειρά εξωτερικών-μετά-εσωτερικών που εμφανίζει ένας block explorer. Οι
επιστρεφόμενες οδηγίες συνδέονται απευθείας με τα βοηθητικά `parseXInstruction`
των πελατών που δημιουργήθηκαν από codama (π.χ., `@solana-program/*`).

```shell title="Terminal"
npm install @solana/transaction-introspection @solana/kit @solana/instructions @solana-program/token
```

## Αποκωδικοποίηση της απόκρισης

Ανακτήστε τη συναλλαγή με κωδικοποίηση wire-format και περάστε την απόκριση στο
`decodeTransactionFromRpcResponse`.

```ts title="Decode a transaction"
import { createSolanaRpc, signature } from "@solana/kit";
import { decodeTransactionFromRpcResponse } from "@solana/transaction-introspection";

const rpc = createSolanaRpc("https://api.mainnet-beta.solana.com");

const txid =
  "3jUKrQp1UGq5ih6FTDUUt2kkqUfoG2o4kY5T1DoVHK2tXXDLdxJSXzuJGY4JPoRivgbi45U2bc7LZfMa6C4R3szX";

const rpcTx = await rpc
  .getTransaction(signature(txid), {
    commitment: "confirmed",
    encoding: "base64",
    maxSupportedTransactionVersion: 0
  })
  .send();
if (!rpcTx) throw new Error(`Transaction ${txid} not found`);

const { compiledMessage, loadedAddresses } =
  decodeTransactionFromRpcResponse(rpcTx);
```

Τα υπόλοιπα αποσπάσματα επαναχρησιμοποιούν τα `rpcTx`, `compiledMessage` και
`loadedAddresses` από αυτό το βήμα.

<Callout type="warn">
  Ανακτήστε με κωδικοποίηση `base64`, `base58` ή `json`. Το
  `decodeTransactionFromRpcResponse` απορρίπτει αποκρίσεις `jsonParsed` - αυτή η
  κωδικοποίηση ζητά από τον κόμβο RPC να αναλύσει τις οδηγίες στην πλευρά του
  διακομιστή, οπότε δεν απομένει τίποτα να αποκωδικοποιηθεί στον πελάτη.
</Callout>

## Διάσχιση των οδηγιών

Η `walkInstructions` επιστρέφει κάθε οδηγία - εξωτερική και εσωτερική - ως
πίνακα με τη σειρά που εμφανίζει ένας block explorer: κάθε εξωτερική οδηγία
ακολουθούμενη αμέσως από τις εσωτερικές οδηγίες που παρήγαγαν τα CPI της. Κάθε
οδηγία φέρει ένα πεδίο `trace` που περιγράφει τη θέση της στην ιεραρχία κλήσεων.

```ts title="Walk every instruction"
import { walkInstructions } from "@solana/transaction-introspection";

for (const ix of walkInstructions({
  compiledMessage,
  loadedAddresses,
  meta: rpcTx.meta
})) {
  const location =
    ix.trace.kind === "outer"
      ? `outer[${ix.trace.index}]`
      : `inner[${ix.trace.outerIndex}/${ix.trace.innerIndex}]`;
  console.log(location, ix.programAddress, ix.accounts?.length ?? 0);
}
```

Το `trace` είναι μια διακριτή ένωση τύπου: `{ kind: "outer", index }` για μια
οδηγία ανώτατου επιπέδου, ή
`{ kind: "inner", outerIndex, innerIndex, stackHeight? }` για μια οδηγία CPI
εμφωλευμένη κάτω από μια εξωτερική.

Το `meta` είναι προαιρετικό - περάστε το `rpcTx.meta` απευθείας ακόμα και όταν
μπορεί να είναι `null`. Χωρίς meta, η `walkInstructions` επιστρέφει μόνο τις
εξωτερικές οδηγίες.

## Ανάλυση με πρόγραμμα-πελάτη

Κάθε οδηγία από την `walkInstructions` είναι `ResolvedInstruction`, οπότε
λειτουργεί απευθείας με τα κατηγορήματα από το `@solana/instructions` και τους
βοηθούς `identifyXInstruction` / `parseXInstruction` που δημιουργούνται από τα
προγράμματα-πελάτες `@solana-program/*` - χωρίς βήμα μετατροπής.

Αυτό το παράδειγμα ελέγχει κάθε οδηγία Token Program `SyncNative` σε μια
συναλλαγή, είτε εκτελέστηκε στο ανώτατο επίπεδο είτε μέσα σε ένα CPI.

```ts title="Find and parse SyncNative instructions"
import {
  isInstructionForProgram,
  isInstructionWithAccounts,
  isInstructionWithData
} from "@solana/instructions";
import { walkInstructions } from "@solana/transaction-introspection";
import {
  identifyTokenInstruction,
  parseSyncNativeInstruction,
  TOKEN_PROGRAM_ADDRESS,
  TokenInstruction
} from "@solana-program/token";

for (const ix of walkInstructions({
  compiledMessage,
  loadedAddresses,
  meta: rpcTx.meta
})) {
  if (!isInstructionForProgram(ix, TOKEN_PROGRAM_ADDRESS)) continue;
  if (!isInstructionWithData(ix) || !isInstructionWithAccounts(ix)) continue;
  if (identifyTokenInstruction(ix) !== TokenInstruction.SyncNative) continue;

  const parsed = parseSyncNativeInstruction(ix);
  console.log(ix.trace, parsed);
}
```

## Βοηθοί χαμηλότερου επιπέδου

Η `walkInstructions` καλύπτει τις περισσότερες ανάγκες, αλλά το πακέτο εκθέτει
επίσης τα επιμέρους στοιχεία στα οποία βασίζεται:

- Η
  `getInstructionsFromCompiledTransactionMessage(compiledMessage, loadedAddresses?)`
  επιστρέφει μόνο τις εξωτερικές οδηγίες, με επιλυμένους λογαριασμούς και
  αποκωδικοποιημένα δεδομένα.
- Η `getInnerInstructionsFromMeta(meta, accountMetas)` αποκωδικοποιεί τις
  εσωτερικές οδηγίες CPI από τα μεταδεδομένα της συναλλαγής.
- Η
  `getAccountMetasFromCompiledTransactionMessage(compiledMessage, loadedAddresses?)`
  επιστρέφει την πλήρη λίστα λογαριασμών με τη σειρά της συναλλαγής - την είσοδο
  που χρειάζεται ο προηγούμενος βοηθός για την επίλυση των δεικτών λογαριασμών
  εσωτερικών οδηγιών.

```ts title="Resolve outer and inner instructions separately"
import {
  getAccountMetasFromCompiledTransactionMessage,
  getInnerInstructionsFromMeta,
  getInstructionsFromCompiledTransactionMessage
} from "@solana/transaction-introspection";

const outer = getInstructionsFromCompiledTransactionMessage(
  compiledMessage,
  loadedAddresses
);

const accountMetas = getAccountMetasFromCompiledTransactionMessage(
  compiledMessage,
  loadedAddresses
);

if (!rpcTx.meta) throw new Error("Transaction metadata missing");
const inner = getInnerInstructionsFromMeta(rpcTx.meta, accountMetas);
```

<Callout type="info" title="Χρησιμοποιείτε Rust?">
  Δεν υπάρχει ισοδύναμο Rust αυτού του πακέτου. Για να λάβετε αναλυμένες οδηγίες
  από Rust, ζητήστε `UiTransactionEncoding::JsonParsed` από
  [`getTransaction`](/docs/rpc/http/gettransaction) και αφήστε τον κόμβο RPC να
  επιλύσει λογαριασμούς και εσωτερικές οδηγίες για εσάς.
</Callout>
