---
title: Відкликання покинутих делегувань та підписок
description:
  Посібник із повернення rent від делегування або підписки, чий Subscription
  Authority зник.
---

Делегування та підписки утримують rent, який зазвичай повертається зафіксованому
платнику під час відкликання. Якщо спонсор фінансував обліковий запис, а
користувач пізніше закрив або замінив свій Subscription Authority, обліковий
запис залишається покинутим, а його rent — заблокованим.
`RevokeAbandonedDelegation` та `RevokeAbandonedSubscription` дозволяють
зафіксованому платнику повернути його.

## Коли це застосовується

Дозволено лише тоді, коли Subscription Authority облікового запису є доведено
кінцевим — закритим або закритим і відтвореним (тобто `init_id` більше не
збігається). Активне делегування або підписку неможливо закрити таким чином.

|                                           | `RevokeAbandonedDelegation`                                                                                                   | `RevokeAbandonedSubscription`                                                                                                                 |
| ----------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Закриває                                  | PDA фіксованого або повторюваного делегування                                                                                 | PDA підписки                                                                                                                                  |
| Підписант                                 | Зафіксований платник                                                                                                          | Зафіксований платник                                                                                                                          |
| Облікові записи                           | Делегування + зафіксований authority                                                                                          | Підписка + зафіксований authority + план                                                                                                      |
| Умова прийнятності                        | Authority закрито або відтворено (невідповідність `init_id`)                                                                  | Те саме — для mint плану; mint зчитується зі зв'язаного плану, тому непов'язаний authority не може бути підставлений для примусового закриття |
| Використовуйте звичайне відкликання, якщо | Делегування є [простроченим або повністю витраченим](/docs/payments/subscriptions/recurring-delegation#revoke-the-delegation) | Підписку [скасовано та прострочено](/docs/payments/subscriptions/subscription-plan#cancel-and-revoke)                                         |

`RevokeAbandonedDelegation` також є єдиним способом відновлення для
зафіксованого платника у випадку делегування без терміну дії (`expiry_ts == 0`),
яке не витрачено повністю: звичайне відкликання спрацьовує для платника лише
після закінчення терміну або, у випадку фіксованого делегування, після повного
витрачання.

## Відкликання покинутого делегування

Зафіксований платник підписує. Передайте покинутий PDA делегування та його
зафіксований Subscription Authority (який може вже бути закритим). Rent
повертається платнику.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedDelegationInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedDelegationInstruction({
      payer: payerSigner,
      delegationAccount: delegationPda,
      subscriptionAuthority: subscriptionAuthorityPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let delegation_pda: Address = address!("DELEGATION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");

    let revoke_abandoned_ix = RevokeAbandonedDelegationBuilder::new()
        .payer(payer)
        .delegation_account(delegation_pda)
        .subscription_authority(subscription_authority)
        .instruction();
    ```

  </Tab>
</Tabs>

## Відкликання покинутої підписки

Зареєстрований платник підписує. Передайте PDA покинутої підписки, її
зареєстрований Authority підписки (який може бути вже закритий) і план, до якого
належить підписка. rent повертається платнику.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedSubscriptionInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedSubscriptionInstruction({
      payer: payerSigner,
      subscriptionAccount: subscriptionPda,
      subscriptionAuthority: subscriptionAuthorityPda,
      planPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let subscription_pda: Address = address!("SUBSCRIPTION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");
    let plan_pda: Address = address!("PLAN_PDA_ADDRESS_HERE");

    let revoke_abandoned_ix = RevokeAbandonedSubscriptionBuilder::new()
        .payer(payer)
        .subscription_account(subscription_pda)
        .subscription_authority(subscription_authority)
        .plan_pda(plan_pda)
        .instruction();
    ```

  </Tab>
</Tabs>

## Примітки

- Підписує зареєстрований платник, а не делегатор, делегат або підписник.
- Обидві інструкції завершуються помилкою, якщо Authority підписки все ще
  активний для облікового запису — замість цього використовуйте звичайне
  відкликання.
- `RevokeAbandonedDelegation` працює як для фіксованих, так і для повторюваних
  делегувань.
