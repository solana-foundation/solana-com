title: З Pay.sh

---

Ви можете приймати платежі за API-запитами через Pay.sh, налаштувавши
Pay-сервер. Встановіть Pay за допомогою:

```bash
brew install pay
```

або

```bash
npm install -g @solana/pay
```

Перевірте встановлення за допомогою:

```bash
pay --version
```

Інструментальний ланцюжок налаштує вашу адресу автоматично. Виконайте наступне,
щоб налаштувати та поповнити ваш гаманець:

```bash
pay setup
```

## Налаштування демонстраційного сервера

Ви можете почати приймати платежі, запустивши свій сервер. Подивіться на
демонстраційний сервер у дії за допомогою:

```bash
pay server demo
```

Це виконує дві речі. Запускається живий Pay-сервер, до якого можна звертатися
через HTTP-запити, а також у вашому поточному каталозі генерується файл
`pay-demo.yml`.

Всі маршрути API налаштовуються через YAML-конфігурацію `endpoints`. Щоб
приймати платежі, необхідно додати атрибут `metering:`. Маршрут
`/api/v1/reports/usage` наведено як приклад:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Ви можете перевірити, що платежі працюють, виконавши наступну команду як клієнт
в іншому терміналі:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Дізнайтеся більше про визначення цін у
[документації Pay.sh «Розробка з Pay > Ціноутворення»](https://pay.sh/docs/building-with-pay/pricing)

Дізнайтеся більше про налаштування Pay.sh у продакшені в
[документації Pay.sh з конфігурації](https://pay.sh/docs/toolchain/configuration).
