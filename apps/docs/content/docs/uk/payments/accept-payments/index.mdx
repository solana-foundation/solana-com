Приймайте платежі на Solana, використовуючи готові компоненти, протокол Solana
Pay або пряму інтеграцію з RPC.

## Конвенція адреси отримувача

На Solana **надавайте адресу свого гаманця, а не токен-акаунта**. Зазвичай
відправник сам визначає відповідний
[асоційований токен-акаунт (ATA)](/docs/payments/how-payments-work#token-accounts)
для токена, який він надсилає (детальніше див.
[Надсилання платежів](/docs/payments/send-payments)).

**Приклад:** Надавайте `7EcDhS...` (ваш гаманець), а не `9xKpQ...` (ваш USDC
токен-акаунт).

<Callout>
  Для торгових точок та електронної комерції див. [Solana
  Pay](/docs/payments/accept-payments/solana-pay) для QR-кодів та платіжних URL,
  або [Payment Button](/docs/payments/accept-payments/payment-button) для
  готових React-компонентів.
</Callout>

## Варіанти інтеграції

| Підхід                                                          | Найкраще для                        | Складність |
| --------------------------------------------------------------- | ----------------------------------- | ---------- |
| [Payment Button](/docs/payments/accept-payments/payment-button) | React-застосунки, швидка інтеграція | Низька     |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | QR-коди, платіжні посилання, POS    | Середня    |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Кастомні процеси, повний контроль   | Вища       |

## Верифікація

Після отримання платежу перевірте його надходження за допомогою
[інструментів верифікації](/docs/payments/accept-payments/verification-tools).
Для високонавантажених застосунків див.
[індексування](/docs/payments/accept-payments/indexing) для моніторингу
транзакцій промислового рівня.
