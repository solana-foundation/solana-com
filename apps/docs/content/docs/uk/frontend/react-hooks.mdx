`@solana/react` публікує Kit-клієнт у дерево компонентів через один провайдер, а
хуки зчитують дані RPC, підписки та стан гаманця з нього. Хуки браузерного
гаманця розміщені в `@solana/kit-plugin-wallet/react` і читають той самий
провайдер — окремий провайдер гаманця не потрібен.

## Встановлення

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Створення клієнта

Створіть клієнт один раз на рівні модуля, поза деревом React. У браузері
`walletSigner` робить підключений гаманець одночасно платником комісій та
ідентифікатором.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` додає `client.wallet` (виявлення та підключення за стандартом
Wallet Standard), а `solanaRpc` додає `client.rpc` і хелпер
`client.sendTransaction`. Експортуйте тип клієнта, обгорнутий у `Awaited<>`, щоб
він залишався коректним, якщо пізніше ви додасте плагін з асинхронним
виконанням, і передавайте цей `AppClient` у кожен виклик `useClient`.

## Оберніть дерево один раз

`ClientProvider` публікує клієнт. Усі хуки нижче читають дані з нього.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Хуки гаманця

Хуки гаманця зчитують можливість `wallet` клієнта, тому вони працюють лише
всередині `ClientProvider`, чий клієнт встановив плагін гаманця.

- `useWallets()` — гаманці за стандартом Wallet Standard, виявлені для ланцюга
  клієнта.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` або `null`. `signer`
  має значення `null` для гаманців лише для читання.
- `useConnect()` / `useDisconnect()` — повертають `ActionResult`, чий `dispatch`
  (і `dispatchAsync`) виконує дію, а `isRunning`, `status`, `data`, `error` та
  `reset` відстежують її результат.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Надіслати транзакцію

`useClient` повертає той самий клієнт, який ви опублікували. Створюйте
інструкції за допомогою згенерованого клієнта програми та надсилайте їх через
`client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Передача експортованого типу `AppClient` до `useClient` надає хуку повний
розширений плагінами клієнт, тому `client.rpc` і `client.sendTransaction` є
типізованими.

## Читання даних

`@solana/react` постачає хуки для читання через клієнт:

- `useRequest(source)` — одноразове RPC-читання; повертає
  `{ data, error, status, refresh }`.
- `useSubscription(source)` — живе підписування; повертає
  `{ data, error, reconnect, status }`.
- `useTrackedData(spec)` — значення, засіяне через RPC та підтримуване
  актуальним за допомогою підписки.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Мемоізуйте джерело (за допомогою `useMemo` або `useCallback` для функції), щоб
хук повторно отримував дані лише при зміні вхідних значень, а не при кожному
рендері.

Для кешування та ревалідації підшляхи `@solana/react/swr` і
`@solana/react/query` огортають ті самі хуки для [SWR](https://swr.vercel.app) і
[TanStack Query](https://tanstack.com/query).

## Типові патерни для розробників Solana

- **Один провайдер**: `ClientProvider` — єдиний провайдер, який вам потрібен;
  хуки гаманця читають його через можливість `wallet` клієнта.
- **Сумісність із серверними компонентами**: Позначайте лише листові компоненти,
  що викликають хуки, за допомогою `"use client"`; серверні читання можуть
  використовувати простий RPC-клієнт Kit без гаманця.
- **Тестування**: Передайте замокований клієнт через `ClientProvider` для
  симуляції гаманців і RPC-відповідей.

<Callout type="warn" title="Дві родини хуків з однаковими іменами">
  `useSignIn` і `useSignMessage` існують як у `@solana/react` (вони приймають
  аргумент `UiWalletAccount`), так і в `@solana/kit-plugin-wallet/react` (на
  основі клієнта, без аргументу account, повертають `ActionResult`).
  Використовуйте версії `@solana/kit-plugin-wallet/react` з патерном клієнта
  вище; старіші хуки гаманця `@solana/react` поступово замінюються.
</Callout>

Поєднайте цей посібник з [оглядом клієнта Kit](/docs/frontend/client), щоб
зрозуміти клієнт, з якого читає кожен хук.
