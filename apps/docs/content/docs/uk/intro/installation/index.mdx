Швидке встановлення для налаштування локального середовища розробки Solana
однією командою. Встановіть Rust, Solana CLI та Anchor Framework на Windows
(WSL), Linux та Mac. (За бажанням ви можете
[встановити кожну залежність окремо](/docs/intro/installation/dependencies).)

## Передумови

<Accordions>
<Accordion title="Windows">

Для розробки програм Solana на Windows ви **повинні** використовувати
[підсистему Windows для Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Після встановлення WSL усі інші залежності встановлюються через термінал
Linux.

Щоб встановити WSL, виконайте наступну команду в Windows PowerShell:

```terminal
$ wsl --install
```

Процес встановлення запропонує вам створити обліковий запис користувача за
замовчуванням.

![Встановлення WSL](/assets/docs/intro/installation/wsl-install.png)

За замовчуванням WSL встановлює Ubuntu. Ви можете відкрити термінал Linux,
шукаючи «Ubuntu» в рядку пошуку.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Якщо ваш термінал Ubuntu виглядає як на зображенні нижче, ви можете зіткнутися з
проблемою, коли `ctrl + v` (комбінація клавіш для вставки) не працює в
терміналі.

![Термінал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Якщо ви зіткнулися з цією проблемою, відкрийте Windows Terminal, шукаючи
«Terminal» у рядку пошуку.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Далі закрийте Windows Terminal і знову відкрийте термінал Linux, шукаючи Ubuntu.
Тепер термінал повинен виглядати як на зображенні нижче, де `ctrl + v`
(комбінація клавіш для вставки) працює.

![Термінал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Якщо ви використовуєте VS Code,
[розширення WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
дозволяє використовувати WSL та VS Code разом.

![Налаштування WSL у VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Ви побачите наступне в рядку стану VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Після налаштування WSL ви готові:

- Завершити [швидке встановлення](#installation) за допомогою терміналу Ubuntu
  (Linux)
- Встановити [Rust](/docs/intro/installation/dependencies#install-rust),
  [Solana CLI](/docs/intro/installation/dependencies#install-solana-cli) та
  [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli) окремо,
  як описано нижче

</Accordion>

Якщо ви використовуєте Debian або похідні від Debian дистрибутиви, такі як Ubuntu або Mint, виконайте наступні кроки:

1. Оновіть списки пакетів, щоб переконатися, що всі пакети актуальні:

```terminal
$ sudo apt-get update
```

2. Встановіть наступні залежності:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Якщо ви зіткнулися з наступною помилкою під час встановлення `protobuf-compiler`, переконайтеся,
що спочатку виконали _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Для похідних від RHEL дистрибутивів, таких як Fedora або Red Hat Linux, виконайте наступні кроки.

Оновіть списки пакетів, щоб отримати новіші версії пакетів, виконавши команду.

```terminal
$ sudo dnf upgrade --refresh
```

Далі встановіть наступні залежності:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## Встановлення

1. Введіть наступну команду у ваш термінал, щоб встановити всі необхідні
   залежності:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Успішна інсталяція поверне результат, подібний до наступного:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. Перевірте успішність інсталяції, переглянувши версію кожної встановленої
   залежності.

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  Якщо команда швидкої інсталяції не спрацювала, зверніться до розділу
  [Встановлення залежностей](/docs/intro/installation/dependencies) нижче для
  отримання інструкцій щодо окремого встановлення кожної залежності.
</Callout>
