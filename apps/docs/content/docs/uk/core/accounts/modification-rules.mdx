<Callout type="info" title="Підсумок">
  Перевірки середовища виконання після кожної інструкції: лише власник може
  списувати lamports або змінювати дані, дані можуть зростати максимум на 10 КіБ
  за інструкцію, зміна власника вимагає нульових даних, а прапорець executable є
  незворотним.
</Callout>

Середовище виконання Solana застосовує ці правила після виконання кожної
інструкції через методи
[`BorrowedInstructionAccount`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L722-L727).
Кожне правило перевіряється в момент модифікації, і транзакція відкочується,
якщо будь-яка перевірка не вдається.

## Правила lamports

| Правило                                                                    | Застосування                                                                                                                                                                                                                                  | Помилка                       |
| -------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Лише власник може списувати lamports                                       | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L785): перевіряє `is_owned_by_current_program()`, коли `lamports < current`                                                              | `ExternalAccountLamportSpend` |
| Облікові записи лише для читання не можуть мати змінені lamports           | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L788-L789): перевіряє `is_writable()`                                                                                                         | `ReadonlyLamportChange`       |
| Будь-яка програма може зараховувати lamports на обліковий запис для запису | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L789): перевірка власності застосовується лише коли новий баланс менший за поточний; перевірка можливості запису все одно застосовується | `ReadonlyLamportChange`       |
| Lamports мають балансуватися в межах інструкції                            | [`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L366-L398): перевіряє `get_lamports_delta() == 0`                                                                                  | `UnbalancedInstruction`       |

## Переходи стану rent

Акаунти існують в одному з трьох
[`RentState`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L19-L30)
значень: _rs`Uninitialized`_ (0 lamports), _rs`RentPaying`_ (вище 0, але нижче
мінімуму для звільнення від rent), та _rs`RentExempt`_ (на рівні або вище
мінімуму). Заборонені переходи призводять до
_rs`TransactionError::InsufficientFundsForRent`_.

Середовище виконання застосовує ці правила через
[`transition_allowed()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L107-L126):

- Будь-який акаунт може перейти до _rs`Uninitialized`_ (закрити) або
  _rs`RentExempt`_.
- Жоден акаунт не може перейти до _rs`RentPaying`_ з _rs`Uninitialized`_ або
  _rs`RentExempt`_. Усі нові акаунти мають бути звільнені від rent.

## Правила запозичення акаунтів

Під час виконання інструкції середовище виконання застосовує семантику одного
записувача для запозичення акаунтів. Програма може отримати або одне змінне
посилання, або кілька незмінних посилань на акаунт, але не обидва одночасно.
Якщо програма намагається запозичити акаунт, який уже запозичено зі зміною (або
запозичити зі зміною акаунт, який уже запозичено без зміни), інструкція
завершується помилкою _rs`AccountBorrowFailed`_ через
[`try_borrow()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L498-L508)
та
[`try_borrow_mut()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L512-L520).
Якщо інструкція завершується, поки запозичення все ще активне, середовище
виконання повертає _rs`AccountBorrowOutstanding`_ у
[`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L374-L385).
