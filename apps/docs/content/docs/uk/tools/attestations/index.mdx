**Solana Attestations — це програма суспільного блага для асоціювання
позаланцюгових даних з онлайн-акаунтами.** Система надає фреймворк для створення
верифікованих тверджень про акаунти, дозволяючи розробникам створювати системи
репутації, інструменти відповідності регуляторним вимогам та інші додатки,
засновані на довірі.

## Навіщо потрібні атестації?

- **Верифіковані твердження**: Створюйте криптографічно верифіковані заяви про
  акаунти
- **Гнучкі схеми**: Визначайте власні структури даних для різних типів атестацій
- **Управління повноваженнями**: Контролюйте, хто може видавати атестації через
  систему облікових даних
- **Інтеграція з токенами**: Прив'язуйте атестації до конкретних token account
  для цільових тверджень
- **Зберігання в ланцюзі**: Зберігайте дані атестацій у Solana для прозорості та
  постійності

## Швидкий старт

Встановіть бібліотеку Solana Attestations:

```bash
# Using pnpm (recommended)
pnpm add sas-lib

# Using npm
npm install sas-lib

# Using yarn
yarn add sas-lib
```

Для Rust-проєктів:

```bash
cargo add solana-attestation-service-client
```

## Основні концепції

Система атестацій побудована навколо трьох основних компонентів:

### Облікові дані

Облікові дані представляють орган атестації, який може видавати атестації. Кожні
облікові дані мають авторизованих підписантів, які можуть створювати атестації
від їхнього імені.

### Схеми

Схема визначає структуру та правила валідації для атестацій. Схеми слугують
шаблонами, що специфікують, які поля повинна містити атестація та як вони мають
бути відформатовані.

### Атестації

Атестація представляє верифіковане твердження або заяву. Атестації створюються
авторизованими підписантами під обліковими даними та відповідають певній
структурі схеми.

## Варіанти використання

Атестації відкривають широкий спектр застосувань:

- **Верифікація ідентичності**: Перевіряйте особи користувачів та відповідність
  вимогам KYC
- **Системи репутації**: Створюйте показники довіри та метрики репутації
- **Інструменти відповідності**: Забезпечуйте дотримання регуляторних вимог для
  DeFi-протоколів
- **Програми сертифікації**: Видавайте та перевіряйте професійні сертифікати
- **Відстеження ланцюга поставок**: Відстежуйте походження та автентичність
  товарів
- **Соціальне підтвердження**: Перевіряйте акаунти соціальних мереж та членство
  в спільноті

## Архітектура

Система має ієрархічну структуру:

1. **Органи** створюють облікові дані, які визначають, хто може видавати
   атестації
2. **Схеми** створюються в рамках облікових даних для визначення структури даних
   атестації
3. **Атестації** видаються уповноваженими підписантами відповідно до формату
   схеми
4. **Токен-акаунти** можуть бути пов'язані з атестаціями для цільових заяв
