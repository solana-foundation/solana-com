**Solana Keychain надає уніфікований інтерфейс для підписання транзакцій Solana
через різні бекенди управління ключами.** Розробляйте локально з ключем у
пам'яті, а потім переходьте до корпоративного управління ключами, хмарних HSM
або керованих гаманців у продакшні — без переписування коду підписання.

### Чому Keychain?

- **Єдиний інтерфейс**: Один трейт `SolanaSigner` працює з усіма бекендами
- **Заміна бекендів**: Змінюйте постачальників управління ключами через
  конфігурацію, а не код
- **Абстракція без витрат**: Прапорці функцій підключають лише те, що потрібно
- **Перевірено**: Аудит безпеки від [Accretion](https://accretion.xyz)

### Архітектура

- **Мови**: Rust + TypeScript, з повним паритетом
- **Трейт**: Уніфікований інтерфейс `SolanaSigner`
- **Бекенди**: Memory, Vault, AWS KMS, GCP KMS, Privy, Turnkey, Fireblocks, CDP,
  Crossmint, Dfns, Openfort, Para, Utila
- **Сумісність**: `@solana/kit` та `@solana/signers` сумісні (TypeScript) |
  `solana-sdk` та `solana-sdk-v3` сумісні (Rust)

### Підтримувані бекенди

Бекенди охоплюють локальну розробку, самостійне та хмарне управління ключами, а
також керовані та MPC-гаманці. Перегляньте
[Вибір бекенду](/docs/tools/keychain/choosing-a-backend), щоб порівняти моделі
зберігання та вибрати відповідну.

| Бекенд          | Сценарій використання             | Rust | TypeScript |
| --------------- | --------------------------------- | ---- | ---------- |
| Memory          | Розробка, тестування              | ✓    | ✓          |
| HashiCorp Vault | Самостійне управління ключами     | ✓    | ✓          |
| AWS KMS         | Хмарний (AWS)                     | ✓    | ✓          |
| GCP KMS         | Хмарний (GCP)                     | ✓    | ✓          |
| Privy           | Вбудовані гаманці                 | ✓    | ✓          |
| Turnkey         | Некастодіальне управління ключами | ✓    | ✓          |
| Fireblocks      | Інституційне зберігання           | ✓    | ✓          |
| CDP             | Coinbase Developer Platform       | ✓    | ✓          |
| Crossmint       | Керовані гаманці Crossmint        | ✓    | ✓          |
| Dfns            | Інфраструктура гаманців Dfns      | ✓    | ✓          |
| Openfort        | Бекенд-гаманці Openfort           | ✓    | ✓          |
| Para            | MPC-гаманці Para                  | ✓    | ✓          |
| Utila           | MPC-зберігання Utila              | ✓    | ✓          |

## Швидкий старт

### Rust

Встановіть крейт Rust:

```bash
cargo add solana-keychain
```

Базове використання:

```rust
use solana_keychain::{Signer, SolanaSigner};

// Create a signer from any backend
let signer = Signer::from_memory("base58_private_key")?;

// All signers share the same interface
let pubkey = signer.pubkey();
let signature = signer.sign_transaction(&mut tx).await?;
```

- [Посібник з Rust](/docs/tools/keychain/getting-started/rust) - Повна
  інсталяція та конфігурація бекенду
- [Crates.io](https://crates.io/crates/solana-keychain) - Rust crate

### TypeScript

Встановіть пакет TypeScript:

```bash
pnpm add @solana/keychain
```

Базове використання:

```typescript
import { createKeychainSigner } from "@solana/keychain";
import { signTransactionWithSigners } from "@solana/signers";

// Create any signer via the unified factory
const signer = await createKeychainSigner({
  backend: "vault",
  vaultAddr: "https://vault.example.com:8200",
  vaultToken: "hvs.xxxxx",
  keyName: "my-solana-key",
  publicKey: "base58_public_key"
});

// Sign an already-compiled transaction
const signedTx = await signTransactionWithSigners(
  [signer],
  compiledTransaction
);
```

- [Посібник з TypeScript](/docs/tools/keychain/getting-started/typescript) -
  `@solana/keychain` пакети
- [npm Package](https://www.npmjs.com/package/@solana/keychain) - npm-пакет
