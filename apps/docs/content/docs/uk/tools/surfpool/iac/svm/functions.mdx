Ці функції доступні при використанні SVM-аддону для Solana та SVM-сумісних
блокчейнів.

## system_program_id

`svm::system_program_id` повертає ідентифікатор System Program,
`11111111111111111111111111111111`.

```hcl
output "system_program_id" {
    value = svm::system_program_id()
}
// > 11111111111111111111111111111111
```

output "default_pubkey" {

## get_instruction_data_from_idl_path

`svm::get_instruction_data_from_idl_path` створює закодовані instruction data
для виклику програми, забезпечуючи перевірку типів та серіалізацію на основі
наданого файлу IDL.

### Вхідні дані

| Name               | Required | Type          | Description                                     |
| ------------------ | -------- | ------------- | ----------------------------------------------- |
| `idl_path`         | required | string        | Шлях відносно txtx.yml до файлу IDL `.json`     |
| `instruction_name` | required | string        | Назва інструкції, для якої генеруються дані     |
| `arguments`        | optional | array[string] | Аргументи інструкції, для яких генеруються дані |

```hcl
output "data" {
    value = svm::get_instruction_data_from_idl("/path/to/idl.json", "my_instruction", ["arg1", "arg2"])
}
// > data: 0x95763bdcc47fa1b305000000776f726c64
```

## get_instruction_data_from_idl

`svm::get_instruction_data_from_idl` створює закодовані instruction data для
виклику програми, забезпечуючи перевірку типів та серіалізацію на основі наданих
даних IDL.

### Вхідні дані

| Name               | Required | Type              | Description                                                              |
| ------------------ | -------- | ----------------- | ------------------------------------------------------------------------ |
| `idl`              | required | string \| SVM_IDL | IDL програми                                                             |
| `instruction_name` | required | string            | Назва інструкції, для якої генеруються дані відповідно до індексації IDL |
| `arguments`        | optional | array[string]     | Аргументи інструкції, для яких генеруються дані                          |

```hcl
output "data" {
    value = svm::get_instruction_data_from_idl(variable.contract.idl, "my_instruction", ["arg1", "arg2"])
}
// > data: 0x95763bdcc47fa1b305000000776f726c64
```

## get_program_from_anchor_project

`svm::get_program_from_anchor_project` отримує артефакти розгортання програми
для програми в проєкті Anchor.

### Вхідні дані

| Назва          | Обов'язково   | Тип    | Опис                                  |
| -------------- | ------------- | ------ | ------------------------------------- |
| `program_name` | обов'язково   | string | Назва програми, що розгортається      |
| `keypair_path` | необов'язково | string | Розташування файлу keypair програми   |
| `idl_path`     | необов'язково | string | Розташування файлу IDL програми       |
| `bin_path`     | необов'язково | string | Розташування бінарного файлу програми |

```hcl
variable "contract" {
    value = svm::get_program_from_anchor_project("my_program")
}
output "idl" {
    value = variable.contract.idl
}
```

## sol_to_lamports

`svm::sol_to_lamports` перетворює вказану суму SOL на lamport.

```hcl
output "lamports" {
    value = svm::sol_to_lamports(1.1)
}
// lamports: 1100000000
```

## lamports_to_sol

`svm::lamports_to_sol` перетворює вказану кількість lamport на SOL.

```hcl
output "sol" {
    value = svm::lamports_to_sol(1100000000)
}
// sol: 1.1
```

## find_pda

`svm::find_pda` знаходить дійсний PDA, використовуючи вказаний ідентифікатор
програми та seeds.

### Вхідні дані

| Назва        | Обов'язково   | Тип           | Опис                                                        |
| ------------ | ------------- | ------------- | ----------------------------------------------------------- |
| `program_id` | обов'язково   | string        | Адреса програми, з якої отримано PDA                        |
| `seeds`      | необов'язково | array[string] | Необов'язковий масив seeds (максимум 16, по 32 байти кожен) |

```hcl
variable "pda" {
    value = svm::find_pda("3bv3j4GvMPjvvBX9QdoX27pVoWhDSXpwKZipFF1QiVr6", ["data"])
}
output "pda" {
    value = std::encode_base58(variable.pda.pda)
}
output "bump" {
    value = variable.pda.bump_seed
}
// > pda: 4amHoWMBgLkPfM8Nq9ZP33Liq9FCuqrLoU1feejkdsUJ
// > bump: 252
```

variable "token_account" {

## create_token_account_instruction

`svm::create_token_account_instruction` створює байти необробленої інструкції
для створення associated token account.

### Вхідні дані

| Name                 | Required | Type   | Description                 |
| -------------------- | -------- | ------ | --------------------------- |
| `funding_address`    | required | string | Адреса фінансуючого акаунту |
| `wallet_address`     | required | string | Адреса гаманця              |
| `token_mint_address` | optional | string | Адреса токен-мінту          |
| `token_program_id`   | optional | string | Адреса Token Program        |

```hcl
action "call" "svm::process_instructions" {
    signers = [signer.caller]

    instruction {
        raw_bytes = svm::create_token_account_instruction(
            signer.caller.address, // funding address
            signer.caller.address, // wallet address
            variable.token_mint, // token mint address
            variable.token_program // token program id
        )
    }
}
```

## u64

`svm::u64` створює байтове представлення цілого числа u64, придатне для
використання як seed у деривації PDA.

### Вхідні дані

| Name    | Required | Type    | Description                                       |
| ------- | -------- | ------- | ------------------------------------------------- |
| `value` | required | integer | Ціле число u64 для перетворення на байтовий масив |

```hcl
variable "pda" {
    value = svm::find_pda("3bv3j4GvMPjvvBX9QdoX27pVoWhDSXpwKZipFF1QiVr6", [svm::u64(1000000000)])
}
```
