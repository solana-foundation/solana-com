Ці підписувачі доступні при використанні SVM-аддону для підписання транзакцій.

## secret_key

Підписувач `svm::secret_key` можна використовувати для синхронного підписання
транзакції за допомогою секретного ключа, мнемонічної фрази або файлу keypair.

### Вхідні дані

| Name              | Required | Type   | Description                                                                 |
| ----------------- | -------- | ------ | --------------------------------------------------------------------------- |
| `secret_key`      | optional | string | Секретний ключ, що використовується для підписання повідомлень і транзакцій |
| `mnemonic`        | optional | string | Мнемонічна фраза, що використовується для генерації секретного ключа        |
| `derivation_path` | optional | string | Шлях виведення, що використовується для генерації секретного ключа          |
| `keypair_json`    | optional | string | Шлях до файлу keypair.json, що містить секретний ключ                       |

### Вихідні дані

| Name         | Type   | Description                           |
| ------------ | ------ | ------------------------------------- |
| `public_key` | string | Публічний ключ облікового запису      |
| `address`    | string | SVM-адреса (псевдонім для public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Також можна використовувати мнемонічну фразу:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Або файл keypair:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

Підписувач `svm::web_wallet` дозволить оператору Runbook підписати транзакцію за
допомогою браузерного підписувача на їхній вибір.

### Вхідні дані

| Name               | Required | Type   | Description                                                                     |
| ------------------ | -------- | ------ | ------------------------------------------------------------------------------- |
| `expected_address` | optional | string | Очікувана для підключення SVM-адреса (залиште порожнім, щоб дозволити будь-яку) |

### Вихідні дані

| Name         | Type   | Description                                   |
| ------------ | ------ | --------------------------------------------- |
| `address`    | string | Адреса підключеного облікового запису         |
| `public_key` | string | Публічний ключ підключеного облікового запису |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Цей підписант запропонує користувачеві у веб-інтерфейсі:

1. Підключити гаманець
2. Надати pubkey через підпис повідомлення
3. Підписати всі транзакції, що вимагають цього підписанта

## squads

Підписант `svm::squads` можна використовувати для підписання транзакцій за
допомогою мультипідпису Squads vault.

### Вхідні дані

| Назва     | Обов'язковий   | Тип    | Опис                |
| --------- | -------------- | ------ | ------------------- |
| `address` | необов'язковий | string | Адреса vault Squads |

### Вихідні дані

| Назва        | Тип    | Опис                                  |
| ------------ | ------ | ------------------------------------- |
| `public_key` | string | pubkey vault Squads                   |
| `address`    | string | SVM-адреса (псевдонім для public_key) |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Використання мультипідпису Squads забезпечує безпечне підписання транзакцій
кількома сторонами без розкриття індивідуальних приватних ключів.
