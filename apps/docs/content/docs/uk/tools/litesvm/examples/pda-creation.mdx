## Повний тест

```rust title="tests/pda_test.rs"
use litesvm::LiteSVM;
use solana_account::Account;
use solana_sdk::pubkey::Pubkey;

#[test]
fn test_pda_creation() {
    let mut svm = LiteSVM::new();

    // Program that owns the PDA
    let program_id = Pubkey::new_unique();

    // Derive PDA address
    let seed = b"my_pda";
    let (pda, bump) = Pubkey::find_program_address(
        &[seed],
        &program_id
    );

    println!("PDA: {}", pda);
    println!("Bump: {}", bump);

    // Create PDA account manually (for testing)
    svm.set_account(pda, Account {
        lamports: 1_000_000,
        data: vec![bump], // Store bump seed in data
        owner: program_id,
        executable: false,
        rent_epoch: 0,
    }).unwrap();

    // Verify PDA was created
    let account = svm.get_account(&pda).unwrap();
    assert_eq!(account.owner, program_id);
    assert_eq!(account.data[0], bump);

    println!("PDA created successfully!");
}
```

## Ключові моменти

1. **Деривація PDA**: Використовуйте `Pubkey::find_program_address()` для
   деривації адрес PDA
2. **Seeds**: PDA деривуються з seeds та ідентифікатора програми
3. **Bump seed**: bump seed гарантує, що адреса знаходиться поза кривою ed25519
4. **Створення акаунту**: Використовуйте `set_account()` для ручного створення
   акаунтів під час тестування
5. **Власність**: PDA повинні належати програмі, яка їх деривувала
