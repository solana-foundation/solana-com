# LiteSVM для TypeScript

Тестові набори Solana, що запускають `solana-test-validator`, витрачають секунди
на кожен тест — ви чекаєте на запуск процесу, RPC-з'єднання та мережеві запити.
LiteSVM виконує ті самі тести за мілісекунди. Без зовнішніх процесів, без
мережі, усе в пам'яті всередині вашого тестового середовища.

<Callout type="info">
  **Протестовано з:** `@solana/kit@6.8.0`, `@solana/kit-plugin-litesvm@0.10.0`,
  `@solana/kit-plugin-signer@0.10.0`, `@solana-program/system@0.12.0`,
  `@solana-program/token@0.13.0`. Ці пакети плагінів є новими і швидко
  розвиваються — якщо ви зіткнетеся з невідповідністю, перевірте
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  щодо точних версій, що використовуються в документації та тестовому наборі.
</Callout>

Ви збираєте Kit-клієнт з `@solana/kit`, встановлюючи плагіни за допомогою
`.use()`: плагін підписувача
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
плагін LiteSVM
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm)),
та будь-які необхідні вам плагіни програм. Результат надає доступ до
`client.sendTransaction()`, `client.rpc` та прямий доступ до стану акаунтів,
годинника та sysvar — усе без виходу за межі вашого процесу.

### Створення клієнта

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` повертає порожню оболонку — кожен `.use(plugin)` додає до неї
  нові можливості. Плагін `signer()` має стояти **перед** `litesvm()`, оскільки
  транспортному плагіну потрібен платник для прив'язки. `signer(mySigner)` —
  скорочення для встановлення одного keypair одночасно як `payer` і `identity` —
  використовуйте `payer()` або `identity()` з того самого пакету, якщо хочете
  розділити ці ролі. Такий самий порядок застосовується для `solanaRpc()` при
  роботі з реальною мережею.
</Callout>

<Callout type="info">
  Хочете швидке однорядкове налаштування для експериментів? Пакет `litesvm()`
  включає airdrop, тому ви можете використати ланцюжок
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` з
  `@solana/kit-plugin-signer` для автоматичного створення та поповнення рахунку
  платника.
</Callout>

</Step>

</Steps>

## Розуміння клієнта

Після `.use(signer(...)).use(litesvm())` клієнт надає доступ до:

| Властивість                 | Опис                                                                                                                                                                                                          |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `client.payer`              | `TransactionSigner`, який ви встановили за допомогою плагіна `signer()` (також задається як `client.identity`)                                                                                                |
| `client.identity`           | Підписувач гаманця, якому належать акаунти, токени та права доступу                                                                                                                                           |
| `client.svm`                | Прямий екземпляр [`LiteSVM`](https://www.npmjs.com/package/litesvm) для керування акаунтами, транзакціями та конфігурацією                                                                                    |
| `client.rpc`                | RPC-інтерфейс, сумісний з Kit (підмножина: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Планування, підписання та надсилання інструкцій як транзакції                                                                                                                                                 |
| `client.airdrop`            | Запит SOL із крана LiteSVM                                                                                                                                                                                    |
| `client.getMinimumBalance`  | Обчислення мінімальної кількості lamport для звільнення від rent                                                                                                                                              |

### Використання client.svm

Прямий доступ до LiteSVM для:

- Керування акаунтами (`setAccount`, `getAccount`, `airdrop`)
- Виконання транзакцій (`sendTransaction`, `simulateTransaction`)
- Конфігурація (`withSigverify`, `withBlockhashCheck` тощо)
- Маніпуляції з часом (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### Використання client.rpc

RPC-інтерфейс, сумісний з Kit, для стандартних операцій:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  RPC-шар підтримує лише **кодування base64**. Запит інших кодувань призведе до
  помилки.
</Callout>

## Конфігурація

Налаштуйте LiteSVM за допомогою методів-будівників на `client.svm`:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## Надсилання транзакцій

Додайте плагіни програм після транспорту — вони прикріплюють будівники
інструкцій до клієнта, тож вам ніколи не доведеться безпосередньо працювати з
`TransactionMessage`:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## Потік SVM → RPC

Ви можете встановлювати стан через SVM і зчитувати його через RPC:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```

## Що далі

Тут описано основний клієнт — створення акаунтів, надсилання інструкцій та
читання стану. Якщо ваші тести передбачають використання SPL-токенів (mints,
ATA, перекази), у
[посібнику з тестування токенів](/docs/tools/litesvm/typescript/tokens)
показано, як плагін `tokenProgram()` скорочує ~40 рядків налаштування до 3
викликів. Повний довідник API див. у
[документації API](/docs/tools/litesvm/typescript/api-reference).
