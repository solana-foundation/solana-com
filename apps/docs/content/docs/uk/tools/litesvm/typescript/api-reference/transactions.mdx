# Транзакції

Методи для надсилання, симуляції та керування транзакціями.

## sendTransaction

```typescript
sendTransaction(tx: Transaction): TransactionMetadata | FailedTransactionMetadata
```

Виконати транзакцію. Повертає метадані в разі успіху або деталі помилки в разі
збою.

```typescript
const result = client.svm.sendTransaction(signedTx);

// Check for errors
if ("err" in result && result.err()) {
  console.error("Transaction failed:", result.err());
  return;
}

// Access result properties (note: these are getter functions)
console.log("Signature:", result.signature());
console.log("Compute units:", result.computeUnitsConsumed());
console.log("Logs:", result.logs());
```

<Callout type="warn">
  Властивості результату транзакції, такі як `err()`, `computeUnitsConsumed()`,
  `logs()` та `signature()`, є **функціями-гетерами**, а не властивостями.
  Завжди викликайте їх із дужками.
</Callout>

### Методи TransactionMetadata

| Метод                    | Повертає                | Опис                               |
| ------------------------ | ----------------------- | ---------------------------------- |
| `signature()`            | `Uint8Array`            | Байти підпису транзакції           |
| `computeUnitsConsumed()` | `bigint`                | Використані обчислювальні одиниці  |
| `logs()`                 | `string[]`              | Журнали програми                   |
| `innerInstructions()`    | `InnerInstruction[]`    | Внутрішні (CPI) інструкції         |
| `returnData()`           | `TransactionReturnData` | Повернуті дані програм             |
| `prettyLogs()`           | `string`                | Відформатовані журнали з кольорами |

## simulateTransaction

```typescript
simulateTransaction(tx: Transaction): SimulatedTransactionInfo | FailedTransactionMetadata
```

Симулювати транзакцію без зміни стану.

```typescript
const simResult = client.svm.simulateTransaction(signedTx);

if ("err" in simResult && simResult.err()) {
  console.error("Simulation failed:", simResult.err());
  return;
}

// Use .meta() to get TransactionMetadata
const meta = simResult.meta();
console.log("Would use", meta.computeUnitsConsumed(), "compute units");
console.log("Logs:", meta.logs());

// Get post-execution account states
const postAccounts = simResult.postAccounts();
console.log("Post-state accounts:", postAccounts);
```

<Callout type="info">
  Симуляція не змінює стан. Використовуйте її для перевірки транзакцій перед
  надсиланням.
</Callout>

## Керування блокгешем

### latestBlockhash

```typescript
latestBlockhash(): Blockhash
```

Отримати поточний блокгеш.

```typescript
const blockhash = client.svm.latestBlockhash();
console.log("Blockhash:", blockhash);
```

### getLatestBlockhash (через client.rpc)

Отримати блокгеш разом з інформацією про час дії для ручного формування
транзакцій. Використовуйте RPC-помічник, оскільки він повертає як `blockhash`,
так і `lastValidBlockHeight`.

```typescript
import {
  createTransactionMessage,
  setTransactionMessageLifetimeUsingBlockhash
} from "@solana/kit";

const { value: blockhashLifetime } = await client.rpc
  .getLatestBlockhash()
  .send();

const tx = setTransactionMessageLifetimeUsingBlockhash(
  blockhashLifetime,
  createTransactionMessage({ version: 0 })
);
```

<Callout type="info">
  При використанні програмних плагінів, таких як `systemProgram()` або
  `tokenProgram()`, вам ніколи не потрібно отримувати блокгеш самостійно —
  `client.sendTransaction()` (а також помічник `.sendTransaction()` у ланцюжках
  інструкцій) зробить це за вас.
</Callout>

### expireBlockhash

```typescript
expireBlockhash(): LiteSVM
```

Перехід до нового blockhash. Корисно для тестування сценаріїв із закінченням
терміну дії blockhash.

```typescript
const oldBlockhash = client.svm.latestBlockhash();
client.svm.expireBlockhash();
const newBlockhash = client.svm.latestBlockhash();

console.log("Old:", oldBlockhash);
console.log("New:", newBlockhash);
// Blockhashes will be different
```

## Історія транзакцій

Увімкніть історію транзакцій для отримання транзакцій після надсилання:

```typescript
// Enable history storage (stores last N transactions)
client.svm.withTransactionHistory(100n);

// Send a transaction
const result = client.svm.sendTransaction(signedTx);

// Later, retrieve it by signature (base58 string)
// Note: getTransaction expects a base58-encoded signature string
```
