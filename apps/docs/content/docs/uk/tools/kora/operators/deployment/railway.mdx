---
title: Розгортання на Railway
description: Розгортання на Railway одним кліком
---

[Railway](https://railway.com/) пропонує зручний досвід розгортання для вузлів
Kora з автоматичним SSL, керуванням доменами та вбудованим моніторингом. Цей
посібник допоможе налаштувати вузол Kora на Railway за допомогою простого
Docker-образу та Railway CLI.

## Передумови

- [Обліковий запис Railway](https://railway.app)
- Встановлений [Railway CLI](https://docs.railway.com/guides/cli)
- [Solana CLI](https://solana.com/docs/intro/installation) для створення
  приватного ключа
- [Точка доступу Solana RPC](https://solana.com/rpc)

## Крок 1: Підготовка файлів для розгортання

Створіть новий каталог для вашого вузла Kora:

```bash
mkdir my-kora-node
cd my-kora-node
```

Додайте приклади файлів до вашого каталогу:

- [kora.toml](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/kora.toml)
- [signers.toml](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/signers.toml)
- [Dockerfile](https://github.com/solana-foundation/kora/blob/v2.0.5/examples/deploy/Dockerfile)

Тепер ваш каталог повинен виглядати так:

```bash
my-kora-node/
├── kora.toml
├── signers.toml
└── Dockerfile
```

## Крок 2: Розгортання на Railway

З каталогу вашого проєкту увійдіть до Railway

```bash
# Login to Railway
railway login
```

Дотримуйтесь інструкцій на екрані для входу у ваш обліковий запис.

Ініціалізуйте проєкт Railway у вашому каталозі.

```bash
railway init
```

Railway запропонує вам:

1. Створити новий проєкт або обрати існуючий
2. Вибрати назву проєкту (наприклад, "my-kora-node")

### Розгорніть ваш застосунок

```bash
# Deploy the application
railway up
```

Це дозволить:

- Завантажити ваші файли на Railway
- Зібрати Docker-образ
- Розгорнути ваш застосунок

**Очікуваний результат:**

```
Build time: 121.81 seconds
Deploy complete
Starting Container
```

Чудово! Ми все налаштували! Ймовірно, ви помітите помилку у ваших логах ("error:
a value is required for `--rpc-url <RPC_URL>` but none was supplied "). Це тому,
що ми ще не налаштували наші змінні оточення — тож давайте зробимо це далі.

## Крок 3: Налаштування змінних оточення

Відкрийте панель керування Railway за адресою [railway.app](https://railway.app)
і перейдіть до вашого проєкту. У вашому проєкті натисніть "Settings" >
"Variables"

![Railway Environment Variables for Kora](/assets/docs/tools/kora/deploy/railway-variables.jpg)

Додайте ці змінні оточення:

| Змінна             | Значення                              | Опис                                      |
| ------------------ | ------------------------------------- | ----------------------------------------- |
| `RPC_URL`          | `https://api.mainnet-beta.solana.com` | Кінцева точка Solana RPC                  |
| `KORA_PRIVATE_KEY` | `your-base58-private-key`             | Ваш приватний ключ Solana (формат base58) |
| `RUST_LOG`         | `info`                                | Рівень логування для налагодження         |

> **Попередження про безпеку**: Ваш приватний ключ має доступ до SOL для сплати
> комісій. Використовуйте виділений keypair для вашого вузла Kora і поповнюйте
> його лише тим SOL, який ви готові витратити на комісії за транзакції. Для
> отримання додаткової інформації про підписувачів дивіться
> [Посібник з підписувачів](/docs/tools/kora/operators/signers).

### Отримання вашого приватного ключа

Якщо вам потрібно створити новий keypair:

```bash
# Generate new keypair
solana-keygen new --outfile ~/.config/solana/kora-keypair.json
```

Переконайтеся, що поповнили достатньою кількістю SOL для тестування/роботи
вашого вузла.

## Крок 4: Повторне розгортання

Після збереження ваших змінних вам буде запропоновано повторно розгорнути ваш
сервер — зробіть це.

## Крок 5: Тестування вашого розгортання

### Генерація публічної кінцевої точки

Щоб надсилати запити до вашої кінцевої точки, вам потрібно згенерувати публічний
домен. З вашого розгортання перейдіть до "Settings" > "Generate domain".

![Railway Domain Setup](/assets/docs/tools/kora/deploy/railway-domain.jpg)

Використовуйте типовий порт "8080" і натисніть "Generate Domain".

Вам слід отримати домен на зразок `https://my-kora-node.railway.app/`

### Тестування кінцевої точки конфігурації

```bash
curl -X POST https://my-kora-node.railway.app \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"getConfig","params":[]}' | jq
```

**Очікувана відповідь:**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "fee_payer": "YourPublicKeyHere...",
    "validation_config": {
      "max_allowed_lamports": 1000000,
      "allowed_programs": [...],
      "allowed_spl_paid_tokens": [...]
      // ...
    }
  }
}
```

Чудова робота! Ваш вузол Kora тепер працює.
