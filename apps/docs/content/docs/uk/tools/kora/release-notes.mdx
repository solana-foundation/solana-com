## Kora v2.2.0-beta.6

_23.03.2026_ |
[Реліз на GitHub](https://github.com/solana-foundation/kora/releases/tag/v2.2.0-beta.6)
| `cargo install kora-cli@2.2.0-beta.6`

Попередній реліз з підтримкою пакетів Jito, новими RPC-методами та розширеними
функціями безпеки.

- Додано підтримку пакетів Jito з методами `signBundle`, `signAndSendBundle` та
  `estimateBundleFee`
- Додано RPC-метод `getVersion`
- Додано захист платника комісій Lighthouse
- Додано захист від ботів reCAPTCHA v3 для кінцевих точок підпису
- Додано обмеження використання для кожного гаманця з підтримкою Redis
- Додано офіційні Docker-образи на GHCR
- Додано деталізовані правила обмеження використання
- SDK: `KoraClient` тепер приймає об'єкт параметрів; новий `koraPlugin` для
  `@solana/kit`

Див. [Бета-документація](/docs/tools/kora/beta) для деталей.

## Kora TypeScript SDK v0.2.0

_09.03.2026_ |
[Реліз на GitHub](https://github.com/solana-foundation/kora/releases/tag/ts-sdk-v0.2.0)
| `pnpm add @solana/kora@0.2.0`

Архітектура плагінів Kit-клієнта для композиційних транзакцій без газу.

- Додано `createKitKoraClient()` з композицією плагінів Kit (плагіни
  планувальника, виконавця, платника)
- Забезпечує композиційність з плагінами програм Kit, такими як `tokenProgram()`
- Додано підтримку Token-2022 через опцію конфігурації `tokenProgramId`
- Додано підтримку інструкцій бюджету обчислень з оцінкою CU на основі симуляції
- Додано відсутнє поле `signature` до відповіді `signAndSendTransaction`
- Повертає `NoopSigner` з `getPaymentInstruction` для дедуплікації підписувачів
  Kit v6
- Оновлено залежність `@solana/kit` до `^6.1.0`

Див. [Посібник з Kit-клієнта](/docs/tools/kora/guides/kit-client).

## Kora v2.0.0

_19.11.2025_ |
[Реліз на GitHub](https://github.com/solana-foundation/kora/releases/tag/v2.0.0)
| `cargo install kora-cli@2.0.0`

Мажорний реліз з новою архітектурою.

- Представлено нову архітектуру для вузла Kora
- Опубліковано TypeScript SDK `@solana/kora` v0.1.0
- Виконано міграцію на Solana SDK v3.0.x та оновлено всі залежності
- Замінено власний Kora Signer на крейт solana-signers
- Видалено методи `sign_transaction_if_paid` та `get_all_signers`
- Змінено типи відповідей: видалено підписи з `signAndSendTransaction` та
  `signTransaction`
- Покращено досвід розробників, безпеку та продуктивність
