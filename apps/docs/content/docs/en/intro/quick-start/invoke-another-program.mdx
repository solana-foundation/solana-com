h1: Invoke Another Program

A cross-program invocation (CPI) is one program calling another program during
the same transaction. The starter program performs a CPI from its `initialize`
instruction to the System Program.

This page continues from
[Create Program Derived Accounts](/docs/intro/quick-start/create-program-derived-accounts).

## Read `initialize.rs`

The `initialize.rs` file has two jobs. The `Initialize` struct validates the
accounts passed into the instruction. The `handle_initialize` function contains
the logic that runs after validation succeeds.

<SideBySide>

### !left

```rust title="Account validation" -w
use anchor_lang::prelude::*;

use crate::{constants::*, state::Counter};

#[derive(Accounts)]
pub struct Initialize<'info> {
    #[account(mut)]
    pub payer: Signer<'info>,
    #[account(
        init,
        payer = payer,
        space = 8 + Counter::INIT_SPACE,
        seeds = [COUNTER_SEED],
        bump
    )]
    pub counter: Account<'info, Counter>,
    pub system_program: Program<'info, System>,
}
```

### !right

```rust title="Instruction logic" -w
pub fn handle_initialize(ctx: Context<Initialize>) -> Result<()> {
    // !mark(1:2)
    ctx.accounts.counter.count = 0;
    ctx.accounts.counter.authority = ctx.accounts.payer.key();

    // !mark(1:8)
    let cpi_accounts = anchor_lang::system_program::Transfer {
        from: ctx.accounts.payer.to_account_info(),
        to: ctx.accounts.counter.to_account_info(),
    };
    let cpi_ctx = CpiContext::new(anchor_lang::system_program::ID, cpi_accounts);
    anchor_lang::system_program::transfer(cpi_ctx, HELLO_WORLD_LAMPORTS)?;

    msg!("Hello, world! Counter initialized");
    Ok(())
}
```

</SideBySide>

## Where the CPI happens

The `initialize` instruction creates the counter account, writes the initial
counter data, then transfers one lamport from the payer to the counter account
with a System Program CPI.

<ScrollyCoding>

## !!steps Write State

The handler first writes program-owned state into the counter account. The
counter starts at `0`, and the payer becomes the authority allowed to increment
it later.

<CodePlaceholder title="initialize.rs" />

## !!steps Invoke Transfer

The `transfer` helper invokes the System Program. The `?` operator returns early
if the CPI fails, so the transaction fails instead of continuing with partial
state.

## CPI accounts and program account

The account struct includes the System Program because account creation and the
explicit transfer both depend on System Program instructions.

<SideBySide>

### !left

```rust title="Initialize accounts" -w
#[derive(Accounts)]
pub struct Initialize<'info> {
    #[account(mut)]
    pub payer: Signer<'info>,
    #[account(
        init,
        payer = payer,
        space = 8 + Counter::INIT_SPACE,
        seeds = [COUNTER_SEED],
        bump
    )]
    pub counter: Account<'info, Counter>,
    // !mark
    pub system_program: Program<'info, System>,
}
```

### !right

```rust title="System Program CPI" -w
let cpi_accounts = anchor_lang::system_program::Transfer {
    // !mark(1:2)
    from: ctx.accounts.payer.to_account_info(),
    to: ctx.accounts.counter.to_account_info(),
};

let cpi_ctx = CpiContext::new(anchor_lang::system_program::ID, cpi_accounts);
anchor_lang::system_program::transfer(cpi_ctx, HELLO_WORLD_LAMPORTS)?;
```

</SideBySide>

The left side validates the accounts passed into `initialize`. The right side
repackages the payer and counter accounts for the System Program transfer
instruction.

## Test the CPI path

The starter test sends the `initialize` instruction and asserts that it
succeeds. That instruction includes the CPI, so a failed transfer would cause
the transaction result to be an error.

```rust title="programs/my-program/tests/test_initialize.rs"
let instruction = Instruction::new_with_bytes(
    program_id,
    &my_program::instruction::Initialize {}.data(),
    my_program::accounts::Initialize {
        payer: payer.pubkey(),
        counter,
        system_program: system_program::ID,
    }
    .to_account_metas(None),
);

let blockhash = svm.latest_blockhash();
let msg = Message::new_with_blockhash(&[instruction], Some(&payer.pubkey()), &blockhash);
let tx = VersionedTransaction::try_new(VersionedMessage::Legacy(msg), &[&payer]).unwrap();

let res = svm.send_transaction(tx);
assert!(res.is_ok());
```
