Solana produces blocks roughly every 400ms; any application that reports on,
reacts to, or reconciles against onchain activity needs an indexing strategy.
This is the canonical indexing documentation - the
[tokenization](/docs/tokenization/index-token-activity) and
[DeFi](/docs/defi/index-program-activity) sections describe what to track for
those use cases and link back here for how.

## The four approaches

| Approach                   | Latency                | Completeness                                               | Ops burden               |
| -------------------------- | ---------------------- | ---------------------------------------------------------- | ------------------------ |
| RPC polling                | Seconds                | Miss nothing if you page carefully                         | Low                      |
| Websocket subscriptions    | Sub-second             | Can drop events on disconnect - pair with polling backfill | Low-medium               |
| Webhooks (provider-hosted) | ~1s                    | Provider-dependent                                         | Low                      |
| Geyser / streaming         | Milliseconds, firehose | Complete                                                   | High (or use a provider) |

### RPC polling

Walk an address's history with
[getSignaturesForAddress](/docs/rpc/http/getsignaturesforaddress) and fetch
details with [getTransaction](/docs/rpc/http/gettransaction); snapshot state
with [getProgramAccounts](/docs/rpc/http/getprogramaccounts). Reliable and
simple - the right default for reconciliation jobs.

### Websocket subscriptions

Subscribe to [account changes](/docs/rpc/websocket/accountsubscribe),
[program accounts](/docs/rpc/websocket/programsubscribe), and
[logs](/docs/rpc/websocket/logssubscribe) for push updates. Always build a
polling backfill for reconnects - subscriptions are best-effort.

### Webhooks and hosted indexers

Providers such as Helius (webhooks, DAS API), Triton, and QuickNode deliver
filtered events to your endpoint or expose enhanced query APIs, trading some
control for a lot less infrastructure. The bootcamp has a hands-on webhook
tutorial:
[using webhooks to fetch transactions](/developers/bootcamp/depin/using-webhooks-to-fetch-solana-transactions).

### Geyser and streaming pipelines

Validators can stream every account update and transaction through the Geyser
plugin interface (commonly consumed via
[Yellowstone gRPC](https://github.com/rpcpool/yellowstone-grpc)). This is the
millisecond-latency, full-firehose option used by trading systems and analytics
platforms - run your own node or buy the stream from an RPC provider.

## Choosing

- Reporting / reconciliation → polling
- User-facing live updates → websockets + polling backfill
- Event-driven backends → webhooks
- Trading, analytics at scale → Geyser/streaming
