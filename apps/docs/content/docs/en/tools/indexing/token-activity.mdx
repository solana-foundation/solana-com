title: Index token activity

  Track transfers, balances, holders, and supply for a token mint using RPC
  methods, subscriptions, and hosted APIs.
---

Token activity means four datasets: transfers, current balances, the holder set,
and supply changes. Each maps to specific [RPC methods](/docs/rpc/http) or
provider APIs.

## Transfers

Poll [getSignaturesForAddress](/docs/rpc/http/getsignaturesforaddress) for the
accounts you care about, then [getTransaction](/docs/rpc/http/gettransaction)
with `jsonParsed` encoding — parsed token instructions include mint, source,
destination, and amount. Token-2022 mints with the
[confidential transfer extension](/docs/tokens/extensions/confidential-transfer)
emit transfer events without amounts; amount-level reporting then requires the
auditor key.

For push delivery, subscribe to
[logs mentioning your mint](/docs/rpc/websocket/logssubscribe) or use a provider
webhook filtered to token transfers.

The wider surface:

- Single account:
  [getTokenAccountBalance](/docs/rpc/http/gettokenaccountbalance)
- All accounts for an owner:
  [getTokenAccountsByOwner](/docs/rpc/http/gettokenaccountsbyowner)
- **All holders of a mint**:
  [getProgramAccounts](/docs/rpc/http/getprogramaccounts) against the token
  program, filtered on the mint — expensive on public RPC; hosted APIs (e.g. DAS
  `getTokenAccounts`) or a Geyser-fed database are the production answer.
  Largest holders only:
  [getTokenLargestAccounts](/docs/rpc/http/gettokenlargestaccounts)

Live balance updates: [programSubscribe](/docs/rpc/websocket/programsubscribe)
with a mint filter.

## Supply

[getTokenSupply](/docs/rpc/http/gettokensupply) for the current number; index
mint/burn instructions from transfer history for the audit trail — issuers
should reconcile both continuously
([why](/docs/tokenization/index-token-activity)).

## Compliance events

Freeze/thaw, pause, and permanent-delegate actions are ordinary instructions
against the token program — capture them in the same transfer-history pipeline
and alert on them separately.
