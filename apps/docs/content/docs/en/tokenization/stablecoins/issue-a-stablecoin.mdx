  Create a fiat-backed stablecoin mint on Solana with Token-2022 — metadata,

This guide walks through creating a stablecoin mint with the controls a
regulated issuer needs. Each step links to the extension reference with full
code; the goal here is the end-to-end sequence and the decisions at each step.

<Callout type="warn">
  All extensions must be enabled when the mint account is created — you cannot
  add them to an existing mint. Test the full configuration on devnet before
  mainnet deployment.
</Callout>

## 1. Create the mint with extensions

Create a Token-2022 mint ([create a mint](/docs/tokens/basics/create-mint)) and
enable your chosen extensions in the same transaction. A typical stablecoin
configuration:

- [Metadata](/docs/tokens/extensions/metadata) — name, symbol, and a URI
  pointing to issuer documentation and reserve attestations
- [Pausable](/docs/tokens/extensions/pausable) — halts all transfers, mints, and
  burns during an incident
- [Permanent delegate](/docs/tokens/extensions/permanent-delegate) — optional;
  grants an authority the power to transfer or burn from any account for seizure
  under legal order
- [Default account state](/docs/tokens/extensions/default-state) — optional; new
  token accounts start frozen until your compliance process thaws them. For
  allowlist/blocklist policies managed by a dedicated program, use
  [Token ACL](/docs/tokenization/token-acl) instead.

Set decimals to match your unit conventions (USDC uses 6).

## 2. Set authorities to multisigs

Before any real value touches the mint, move every authority — mint, freeze,
pause, permanent delegate, metadata update — to multisigs or program-controlled
PDAs. See [set authority](/docs/tokens/basics/set-authority) and the authority
guidance in
[Choose a token design](/docs/tokenization/token-design#authority-structure).

## 3. Mint against deposits, burn against redemptions

Issuance is [minting tokens](/docs/tokens/basics/mint-tokens) to a customer's
token account when fiat clears; redemption is
[burning](/docs/tokens/basics/burn-tokens) when fiat is returned. Run both
through a service that:

- verifies the fiat leg before signing,
- enforces per-transaction and daily limits,
- writes an audit log keyed by transaction signature.

## 4. Wire up compliance operations

- **Freeze / thaw** individual accounts:
  [freeze](/docs/tokens/basics/freeze-account) /
  [thaw](/docs/tokens/basics/thaw-account)
- **Pause everything**: the pause authority flips the
  [pausable](/docs/tokens/extensions/pausable) state
- **Screen counterparties** before thawing accounts or processing redemptions

## 5. Monitor from day one

Reconciliation and reporting need an indexed view of supply and transfers — see
[Index token activity](/docs/tokenization/index-token-activity). At minimum,
track total supply vs reserves, large transfers, and frozen-account events.

## Going to production

Review [production readiness](/docs/tokenization/production-readiness) — key
ceremonies, runbooks for freeze/pause events, and monitoring — before launch.
