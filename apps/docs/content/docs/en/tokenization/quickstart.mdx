You'll end up with a Token-2022 mint on a local Solana network with four
compliance extensions enabled, and you'll know how to point the same code at
devnet. You need Node.js 20+ and about ten minutes.

## 1. Start a local network

[Surfpool](/docs/intro/installation/surfpool-cli-basics) runs a local Solana
network that mirrors mainnet accounts, so everything here works without touching
a public cluster or holding SOL:

```bash
surfpool start
```

Leave it running. Your RPC endpoint is `http://localhost:8899`.

## 2. Set up the project

```bash
mkdir token-quickstart && cd token-quickstart
npm init -y && npm pkg set type=module
npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-signer \
  @solana-program/token-2022 @solana-program/system typescript tsx
```

## 3. Create the mint

Save this as `create-mint.ts`. It creates a mint with onchain metadata, pausable
transfers, a permanent delegate, and token accounts that start frozen - the
configuration walked through in
[Issue a stablecoin](/docs/tokenization/stablecoins/issue-a-stablecoin):

<CodeTabs storage="cookbook" flags="r">

```ts !! title="create-mint.ts" file=packages/docs-examples/cookbook/tokens/create-mint-with-extensions/kit.ts#region=create

```

</CodeTabs>

Run it:

```bash
npx tsx create-mint.ts
```

You should see the transaction signature, the mint address, and the list of
enabled extensions.

## 4. Point it at devnet

Swap the RPC URLs for devnet and fund the payer from a
[faucet](/developers/cookbook/development/airdrops-and-faucets):

```ts
solanaRpc({
  rpcUrl: "https://api.devnet.solana.com",
  rpcSubscriptionsUrl: "wss://api.devnet.solana.com"
});
```

Everything else is unchanged. Mainnet is the same swap again - but read
[production readiness](/docs/tokenization/production-readiness) first: real
issuance means multisig authorities, compliance runbooks, and monitoring.
