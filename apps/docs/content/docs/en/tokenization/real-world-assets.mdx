title: Tokenize a real-world asset
description:
  Patterns for bringing funds, treasuries, and other offchain assets onchain -
  eligibility controls, NAV, privacy, and settlement.
---

Tokenizing an offchain asset means representing claims on it as tokens whose
transfer rules match the asset's legal constraints. On Solana those rules are
expressed as token extensions and policy programs, not bespoke smart contracts,
which keeps tokenized assets compatible with the whole ecosystem - wallets, DEXs
where permitted, and indexers.

## The core pattern

1. **Restrict holders.** Most securities can only be held by eligible (KYC'd,
   accredited, or jurisdiction-screened) investors. Enforce this with
   [Token ACL](/docs/tokenization/token-acl) allowlists, or
   [default frozen accounts](/docs/tokens/extensions/default-state) thawed by
   your transfer agent.
2. **Reflect value.** For funds, use
   [scaled UI amount](/docs/tokens/extensions/scaled-ui-amount) or
   [interest-bearing](/docs/tokens/extensions/interest-bearing-tokens)
   extensions so a share's displayed value tracks NAV - see
   [NAV strikes](/docs/tokenization/nav-strikes) for the update workflow.
3. **Settle atomically.** Exchange the asset against payment in a single
   transaction so neither side takes settlement risk:
   [Delivery vs Payment](/docs/tokenization/dvp).
4. **Preserve confidentiality where required.** Position sizes and transfer
   amounts can be encrypted with
   [confidential transfers](/docs/tokenization/private-transfers).
5. **Report.** Transfer agents and fund administrators need a complete record of
   holders and movements:
   [Index token activity](/docs/tokenization/index-token-activity).
