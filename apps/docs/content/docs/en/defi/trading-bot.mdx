By the end of this page you know the four-stage anatomy of a Solana trading bot
and which primitive serves each stage. You should already be able to
[swap tokens](/docs/defi/swaps); everything here builds on that.

A Solana trading bot is a loop: observe state, decide, build a transaction, land
it fast. This page maps each stage to the primitives you'll use.

## 1. Observe

Match the data channel to your latency needs:

| Latency need | Approach                                                                    |
| ------------ | --------------------------------------------------------------------------- |
| Seconds      | Poll [RPC methods](/docs/rpc/http) or aggregator price APIs                 |
| Sub-second   | [Websocket subscriptions](/docs/rpc/websocket) on accounts and program logs |
| Milliseconds | Geyser/streaming infrastructure — see [Indexing](/docs/tools/indexing)      |

Keep a local view of the accounts you trade against and reconcile on every slot
rather than re-fetching everything.

## 2. Decide and build

Keep signal logic offchain (compute is free there) and produce a fully-built
transaction: instructions, compute budget, and priority fee. For multi-venue or
multi-step strategies, put every leg in one atomic transaction — see
[Compose atomic transactions](/docs/defi/compose-transactions) — so a partial
fill of your strategy is impossible.

The rest of the landing checklist:

- **Compute budget** trimmed to actual usage:
  [optimize compute](/developers/cookbook/transactions/optimize-compute)
- **Blockhash and retry discipline**:
  [retrying transactions](/developers/cookbook/transactions/retry) and
  [confirmation & expiration](/developers/cookbook/transactions/confirmation)
- **MEV protection** for any transaction that reveals intent:
  [MEV protection](/docs/defi/mev-protection)
- For sustained high-priority flows, consider
  [stake-weighted QoS](/docs/defi/stake-weighted-qos) with your RPC provider

## 4. Operate safely

- Trade from a dedicated hot wallet holding only working capital; sweep profits
  to cold storage.
- Enforce position and per-transaction limits in code, not convention.
- Simulate before sending
  ([simulateTransaction](/docs/rpc/http/simulatetransaction)) and alert on
  simulation-vs-execution divergence.
- Track your own fills:
  [Index program activity](/docs/defi/index-program-activity).

See [production readiness](/docs/defi/production-readiness) for the full
operational checklist.
