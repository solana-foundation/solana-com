By the end of this page you can quote and execute a token swap from your app,
with slippage protection and a landing strategy. You need a funded keypair or
connected wallet; to try quoting first without one, start with the
[quickstart](/docs/defi/quickstart).

The standard way to swap on Solana is through an aggregator that routes across
all major DEXs for best execution. [Jupiter](https://dev.jup.ag) routes the
large majority of aggregated swap volume; DEXs like Orca and Raydium also offer
direct SDKs when you want a specific venue.

## The swap flow

1. **Quote** - request a quote for an input mint, output mint, and amount. The
   response includes the route, expected output, and price impact.
2. **Set slippage** - the maximum deviation from the quoted output you'll
   accept. Too tight and transactions fail during volatility; too loose and
   you're exposed to [MEV](/docs/defi/mev-protection).
3. **Build and sign** - the aggregator API returns a serialized transaction for
   your wallet or keypair to sign. Inspect it before signing: verify the program
   IDs and that the destination is your account.
4. **Send and confirm** - use a landing strategy appropriate for trading:
   [priority fees](/developers/cookbook/transactions/add-priority-fees),
   [retries](/developers/cookbook/transactions/retry), and
   [confirmation handling](/developers/cookbook/transactions/confirmation).

## Composing swaps with other instructions

Because Solana transactions are atomic across programs, a swap can be one
instruction among several - swap then deposit, claim then swap then repay - with
the guarantee that either everything executes or nothing does. See
[Compose atomic transactions](/docs/defi/compose-transactions). Large
multi-program transactions often need
[address lookup tables](/developers/cookbook/transactions/lookup-tables) to stay
under the transaction size limit.

## In-program swaps (CPI)

Programs can invoke DEXs directly via [cross-program invocation](/docs/core/cpi)

- this is how vaults and strategies execute trades. Prefer venue-direct CPIs
  (single AMM) over routed swaps inside programs; routing is better done
  client-side where compute is free.
