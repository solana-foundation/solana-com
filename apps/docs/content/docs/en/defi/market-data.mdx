---
title: Read market data
description:
  Get asset prices into your app or program - oracle networks, onchain orderbook
  data, and when to use each.
---

Almost every financial application starts with a price. On Solana you have three
sources, with different trust and latency profiles.

## Oracles

Oracle networks publish externally-sourced prices (CEX + DEX aggregated)
onchain, for use inside programs:

- **[Pyth](https://docs.pyth.network/price-feeds)** - pull-based price feeds
  updated at high frequency, with confidence intervals. The most widely used
  oracle in Solana DeFi.
- **[Switchboard](https://docs.switchboard.xyz)** - customizable feeds; you can
  define your own data sources and update logic.

Inside a program, you read the oracle's price account like any other account and
validate its staleness and confidence before acting on it. Client-side, both
networks offer SDKs and streaming APIs.

<Callout type="warn">
  Always check the feed's last-update slot and confidence interval. Using a
  stale or wide-confidence price is the root cause of most oracle-related
  exploits.
</Callout>

## DEX and aggregator APIs

For client-side quotes (what you'd actually get if you traded now), query
aggregators like [Jupiter](https://dev.jup.ag) - see
[Swap tokens](/docs/defi/swaps). Quotes reflect live routed liquidity, which is
different from an oracle's index price; use quotes for execution, oracles for
valuation and risk.

## Raw onchain data

For orderbook or AMM-pool state, read the venue's accounts directly via
[getProgramAccounts](/docs/rpc/http/getprogramaccounts) or subscribe with
[websockets](/docs/rpc/websocket) - this is the lowest-latency option but
couples you to each venue's account layout. For sustained consumption, see
[Index program activity](/docs/defi/index-program-activity).
