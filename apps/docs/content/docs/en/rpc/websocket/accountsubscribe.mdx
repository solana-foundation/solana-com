Subscribe to notifications when an account's lamports or data change.

<Callout type="info" title="Source">
  [`account_subscribe`](https://github.com/anza-xyz/agave/blob/v3.1.8/rpc/src/rpc_pubsub.rs#L425)
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "accountSubscribe",
  "params": [
    // !hover 0
    "CM78CPUeXjn8o3yroDHxUtKsZZgoy4GPkPPXfouKNH12",
    // !hover(1:4) 1
    {
      // !hover encoding
      "encoding": "jsonParsed",
      // !hover commitment
      "commitment": "finalized"
    }
  ]
}
```

### !params

#### !! 0

!type string  
!required

Account Pubkey, as base-58 encoded string

#### !! 1

!type object

Optional configuration object.

##### !! commitment

!type string  
!values processed confirmed finalized  
!default finalized

The commitment describes how finalized a block is at that point in time. See
[Configuring State Commitment](/docs/rpc#configuring-state-commitment).

##### !! encoding

!type string  
!values base58 base64 base64+zstd binary jsonParsed  
!default binary

Encoding format for account data.

- `base58` is slow.
- `jsonParsed` attempts to use program-specific parsers to return more
  human-readable account state data.
- If `jsonParsed` is requested but no parser is available, Agave falls back to
  `[data, "base64"]`.
- `binary` is a deprecated legacy alias for base58 data returned as a plain
  string.

##### !! dataSlice

!type object

Request a slice of the account's data.

| Field    | Type    | Description                                           |
| -------- | ------- | ----------------------------------------------------- |
| `offset` | `usize` | Byte offset from which to start reading account data. |
| `length` | `usize` | Number of bytes to return.                            |

```json title="Example"
{ "offset": 0, "length": 32 }
```

<Callout type="info">
  Data slicing is only available for `base58`, `base64`, `base64+zstd`, and
  `binary` encodings.
</Callout>

<Callout type="info">
  `base58` and `binary` use base-58 encoding and fail if the selected payload is
  more than 128 bytes after slicing. Use `base64` for larger accounts.
</Callout>

<Callout type="info">
  `accountSubscribe` shares the `RpcAccountInfoConfig` type with
  [getAccountInfo](/docs/rpc/http/getaccountinfo), but Agave currently ignores
  `minContextSlot` for PubSub subscriptions.
</Callout>

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 23784,
  "id": 1
}
```

!type number

Subscription id. Pass this to
[accountUnsubscribe](/docs/rpc/websocket/accountunsubscribe).

</APIMethod>

### Notification format

Notifications are delivered as `accountNotification`. The `params.result` value
uses the same `RpcResponse<UiAccount>` wrapper documented for
[getAccountInfo](/docs/rpc/http/getaccountinfo), except PubSub notifications
only include `context.slot`.

When `encoding` is `jsonParsed`, the `data` field uses the same stable wrapper
documented in [Account Data](/docs/rpc/json-structures#account-data). If no
parser is available, Agave falls back to `[data, "base64"]`.

```json
{
  "jsonrpc": "2.0",
  "method": "accountNotification",
  "params": {
    "result": {
      "context": {
        "slot": 5199307
      },
      "value": {
        "data": [
          "11116bv5nS2h3y12kD1yUKeMZvGcKLSjQgX6BeV7u1FrjeJcKfsHPXHRDEHrBesJhZyqnnq9qJeUuF7WHxiuLuL5twc38w2TXNLxnDbjmuR",
          "base58"
        ],
        "executable": false,
        "lamports": 33594,
        "owner": "11111111111111111111111111111111",
        "rentEpoch": 635,
        "space": 80
      }
    },
    "subscription": 23784
  }
}
```
