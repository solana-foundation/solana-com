`pay` is a CLI and MCP server for making paid HTTP requests with a Solana
wallet. You can use it from the terminal with tools such as `curl`, `fetch`,
`codex`, and `claude`. When an API returns an MPP or x402 payment challenge,
`pay` signs the payment with your wallet.

| Resource    | Link                                                              |
| ----------- | ----------------------------------------------------------------- |
| Source code | [solana-foundation/pay](https://github.com/solana-foundation/pay) |

| MPP docs    | [Stripe MPP docs](https://docs.stripe.com/payments/machine/mpp)   |

## Install and Set Up Pay

<Steps>
<Step>

### Installation

<Tabs items={["Homebrew", "From source"]} defaultValue="Homebrew">

<Tab value="Homebrew">

```terminal
$ brew install pay
$ pay --version
```

</Tab>
<Tab value="From source">

```terminal
$ git clone https://github.com/solana-foundation/pay.git
$ cd pay
$ just install-pay
$ pay --version
```

</Tab>
</Tabs>

</Step>
<Step>

### List Pay Services

```terminal
$ pay skills list
```

Use this to browse the providers currently available through `pay`.

</Step>
<Step>

### Set Up Pay Wallet

```terminal
$ pay setup
```

`pay setup` creates a new wallet and stores it in a supported secure backend:

- macOS Keychain
- GNOME Keyring
- Windows Hello
- 1Password

<Callout type="info" title="Fund the wallet with USDC">
  As part of `pay setup`, `pay` prompts you to fund the new wallet with USDC.
  That USDC balance is what `pay` uses for paid requests.

You do not need SOL in this wallet to send paid requests because the supported
providers included in `pay` can sponsor the transaction fee. Sandbox mode is
auto-funded for you.

</Callout>

</Step>
<Step>

### Use Codex or Claude Code

<Tabs items={["Codex", "Claude Code"]} defaultValue="Codex">

<Tab value="Codex">

```terminal
$ pay codex
```

</Tab>
<Tab value="Claude Code">

```terminal
$ pay claude
```

</Tab>
</Tabs>

The wrapper injects the `pay` MCP server automatically so Codex or Claude Code
can make paid requests through the same `pay` wallet flow.

This is useful when you want Codex or Claude Code to use a paid service on your
behalf. You can ask it to use a service supported by `pay`, and `pay` handles
the payment step when the request requires it.

For example, you can ask Codex or Claude Code to:

- list the services available through `pay`
- find a service for a task
- use a supported service and pay with your `pay` wallet

</Step>
<Step>

### Add Funds to Pay Wallet

```terminal
$ pay topup
```

To add additional funds later, use the `pay topup` command.

</Step>
</Steps>

## Local Testing with Sandbox Mode

<Steps>
<Step>

### Send a test request

```terminal
$ pay --sandbox curl https://payment-debugger.vercel.app/mpp/quote/AAPL
```

Sandbox mode forces `localnet`, points the client at the hosted Surfpool RPC,
and auto creates a funded ephemeral wallet on first use.

</Step>
<Step>

### Use Codex or Claude Code with Sandbox Mode

<Tabs items={["Codex", "Claude Code"]} defaultValue="Codex">

<Tab value="Codex">

```terminal
$ pay --sandbox codex
```

</Tab>
<Tab value="Claude Code">

```terminal
$ pay --sandbox claude
```

</Tab>
</Tabs>

The wrapper injects the `pay` MCP server automatically so Codex or Claude Code
can make paid requests through the same sandbox wallet flow.

</Step>
</Steps>
