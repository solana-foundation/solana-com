A transaction is the fundamental unit of state change on Solana. Each
transaction bundles one or more [instructions](/docs/core/instructions) together
with cryptographic signatures and a recent blockhash, then submits them to the
network as a single atomic operation. If any instruction fails, all state
changes in the transaction revert.

## Key facts

- **Atomic execution**: All instructions succeed or all revert. Fees are still charged on failure.
- **Size limit**: 1,232 bytes maximum, derived from the IPv6 minimum MTU (1,280 bytes) minus 48 bytes for network headers.
- **Signatures**: Each signer provides one 64-byte Ed25519 signature.
- **Blockhash expiry**: A transaction's recent blockhash is valid for approximately 150 slots (~60 seconds at ~400ms/slot).

## Constraints

| Constraint | Value | Source |
|---|---|---|
| Max transaction size | 1,232 bytes | [`PACKET_DATA_SIZE`](https://github.com/anza-xyz/solana-sdk/blob/clock%40v2.2.3/packet/src/lib.rs#L32) |
| Max accounts per transaction | 256 | [`MAX_ACCOUNTS_PER_TRANSACTION`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L29) |
| Blockhash expiry | 150 slots (~60s at ~400ms/slot) | [`MAX_PROCESSING_AGE`](https://github.com/anza-xyz/solana-sdk/blob/clock%40v2.2.3/clock/src/lib.rs#L133) |
| Signature size | 64 bytes (Ed25519) | -- |
| Base fee per signature | 5,000 lamports | [Fees](/docs/core/fees/fee-structure#base-fee) |
| Max signatures per packet | 12 | [`MAX_SIGNATURES_PER_PACKET`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-view/src/signature_frame.rs#L18) |
| BlockhashQueue size | 300 entries | [`MAX_RECENT_BLOCKHASHES`](https://github.com/anza-xyz/solana-sdk/blob/clock%40v2.2.3/clock/src/lib.rs#L127). The queue holds 300 entries, but only hashes with age `<= MAX_PROCESSING_AGE` are accepted for transactions. |
| ALT loaded size overhead | 8,248 bytes per ALT | [`ADDRESS_LOOKUP_TABLE_BASE_SIZE`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/account_loader.rs#L49) |
| MessageHeader size | 3 bytes | fixed: 3 x u8 |

See [Fees](/docs/core/fees) for compute unit and loaded data size limits. See [Instructions](/docs/core/instructions) for per-instruction constraints.
