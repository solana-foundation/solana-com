A Solana program is an [account](/docs/core/accounts/account-types#program-accounts) that
contains executable sBPF bytecode and has its `executable` flag set to `true`.
Programs are stateless: they store no mutable data in their own account. All
mutable state lives in separate data accounts passed via
[instructions](/docs/core/instructions).

## Key facts

- **Compiled to sBPF**: Programs are compiled to Solana Bytecode Format (sBPF) via LLVM and stored in executable accounts.
- **Stateless**: All mutable state lives in separate data accounts, not in the program account.
- **Upgradeable**: Programs deployed with loader-v3 (BPF Loader Upgradeable) can be upgraded when an upgrade authority is set; revoking that authority makes the program immutable.
- **System Program**: The System Program address is `11111111111111111111111111111111`.
