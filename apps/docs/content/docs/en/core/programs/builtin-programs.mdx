<Callout type="info" title="Summary">
  The System Program creates accounts and transfers SOL. Loader programs manage
  program deployment and execution. Core programs handle voting, staking,
  configuration, and compute budgets.
</Callout>

## The System Program

The System Program (`11111111111111111111111111111111`) is the only program that
can create new accounts. All new accounts are initially owned by the System
Program, though ownership is typically reassigned upon creation. The System
Program consumes
[`DEFAULT_COMPUTE_UNITS`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/system/src/system_processor.rs#L282)
(150 CUs) per instruction.

## Loader programs

Every program is owned by a loader program. Loaders handle deployment,
redeployment, upgrade, closure, and authority transfers for the programs they
own.

<Callout>Loader programs are sometimes referred to as 'BPF Loaders'.</Callout>

There are five loader programs:

| Loader | Program ID                                    | Status     | Notes                                                                                                     | Instructions Link                                                                                                                               |
| ------ | --------------------------------------------- | ---------- | --------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| native | `NativeLoader1111111111111111111111111111111` | Active     | Owns the other four loaders                                                                               | --                                                                                                                                              |
| v1     | `BPFLoader1111111111111111111111111111111111` | Deprecated | Management instructions are disabled, but programs still execute                                          | --                                                                                                                                              |
| v2     | `BPFLoader2111111111111111111111111111111111` | Deprecated | Management instructions are disabled, but programs still execute                                          | [Instructions](https://docs.rs/solana-loader-v2-interface/latest/solana_loader_v2_interface/enum.LoaderInstruction.html)                        |
| v3     | `BPFLoaderUpgradeab1e11111111111111111111111` | Active     | Programs can be updated after deployment. Program executable is stored in a separate program data account | [Instructions](https://docs.rs/solana-loader-v3-interface/latest/solana_loader_v3_interface/instruction/enum.UpgradeableLoaderInstruction.html) |
| v4     | `LoaderV411111111111111111111111111111111111` | Unreleased | In development (unreleased)                                                                               | [Instructions](https://docs.rs/solana-loader-v4-interface/latest/solana_loader_v4_interface/instruction/enum.LoaderV4Instruction.html)          |

Programs deployed with loader-v3 or loader-v4 may be modifiable after
deployment, as determined by its upgrade authority.

<Callout>
  When a new program is deployed, the latest loader version will be used by
  default.
</Callout>

### Loader compute costs

Each loader charges a fixed base cost before executing the program it loads:

| Loader                        | Base CU cost | Source                                                                                                                 |
| ----------------------------- | ------------ | ---------------------------------------------------------------------------------------------------------------------- |
| `bpf_loader_deprecated` (v1)  | 1,140        | [`DEPRECATED_LOADER_COMPUTE_UNITS`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L59)  |
| `bpf_loader` (v2)             | 570          | [`DEFAULT_LOADER_COMPUTE_UNITS`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L57)     |
| `bpf_loader_upgradeable` (v3) | 2,370        | [`UPGRADEABLE_LOADER_COMPUTE_UNITS`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L61) |

Note: v1 and v2 loader management instructions are no longer supported. Attempts
to use them return
[`UnsupportedProgramId`](https://github.com/anza-xyz/agave/blob/v3.1.8/programs/bpf_loader/src/lib.rs#L400).
Existing programs under these loaders still execute normally.

See [Precompiled Programs](/docs/core/programs/precompiles) for signature
verification programs (Ed25519, Secp256k1, Secp256r1).
