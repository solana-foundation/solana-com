_Hinweis vor dem Lesen: Alle Verweise auf Wertsteigerungen beziehen sich auf
absolute Werte in Bezug auf den SOL-Saldo. Dieses Dokument macht keine Aussagen
zum monetären Wert von SOL zu irgendeinem Zeitpunkt._

Durch das Staking Ihrer SOL-Token tragen Sie zur Sicherung des Netzwerks bei und
[verdienen dabei Belohnungen](https://docs.anza.xyz/implemented-proposals/staking-rewards).

Sie können staken, indem Sie Ihre Token an Validatoren delegieren, die
Transaktionen verarbeiten und das Netzwerk betreiben.

Das Delegieren von Stake ist ein Finanzmodell mit geteiltem Risiko und geteilter
Belohnung, das Token-Inhabern, die ihre Token für einen längeren Zeitraum
delegieren, Erträge bieten kann. Dies wird erreicht, indem die finanziellen
Anreize der Token-Inhaber (Delegatoren) und der Validatoren, an die sie
delegieren, aufeinander abgestimmt werden.

Je mehr Stake an einen Validator delegiert wird, desto häufiger wird dieser
Validator ausgewählt, um neue Transaktionen in das Ledger zu schreiben. Je mehr
Transaktionen der Validator schreibt, desto mehr Belohnungen verdienen der
Validator und seine Delegatoren. Validatoren, die ihre Systeme so konfigurieren,
dass sie mehr Transaktionen verarbeiten können, verdienen proportional mehr
Belohnungen, da sie das Netzwerk so schnell und reibungslos wie möglich am
Laufen halten.

Validatoren entstehen Kosten durch den Betrieb und die Wartung ihrer Systeme,
und diese werden an die Delegatoren in Form einer Gebühr weitergegeben, die als
Prozentsatz der verdienten Belohnungen erhoben wird. Diese Gebühr wird als
_Provision_ bezeichnet. Da Validatoren mehr Belohnungen verdienen, je mehr Stake
an sie delegiert wird, können sie miteinander konkurrieren, um die niedrigste
Provision für ihre Dienste anzubieten.

Obwohl dies heute noch nicht im Solana-Protokoll implementiert ist, könnten
Delegatoren in Zukunft durch einen Prozess namens _Slashing_ Token verlieren.
Slashing beinhaltet die Entfernung und Vernichtung eines Teils der SOL eines
Validators als Reaktion auf vorsätzliches böswilliges Verhalten, wie das
Erstellen ungültiger Transaktionen oder die Zensur bestimmter Arten von
Transaktionen oder Netzwerk- Teilnehmern.

Es gibt derzeit keine protokollinterne Implementierung von Slashing. Für weitere
Informationen zu Slashing siehe die
[Slashing-Roadmap](https://docs.anza.xyz/proposals/optimistic-confirmation-and-slashing#slashing-roadmap).

## Wie stake ich meine SOL-Token?

Sie können SOL staken, indem Sie Ihre Token in eine Wallet übertragen, die
Staking unterstützt. Die Wallet bietet Schritte zum Erstellen eines Stake-Kontos
und zur Durchführung der Delegation.

### Unterstützte Wallets

Viele Web- und Mobile-Wallets unterstützen Solana-Staking-Operationen. Bitte
erkundigen Sie sich bei den Entwicklern Ihrer bevorzugten Wallet nach dem
aktuellen Status

### Solana-Befehlszeilentools

- Solana-Befehlszeilentools können alle Stake-Operationen in Verbindung mit
  einer CLI-generierten Keypair-Datei-Wallet, einer Paper-Wallet oder mit einem
  verbundenen Ledger Nano durchführen.
  [Staking-Befehle mit den Solana-Befehlszeilentools](https://docs.anza.xyz/cli/examples/delegate-stake).

### Ein Stake-Konto erstellen

Folgen Sie den Anweisungen der Wallet zum Erstellen eines Staking-Kontos. Dieses
Konto wird von einem anderen Typ sein als eines, das einfach zum Senden und
Empfangen von Token verwendet wird.

### Einen Validator auswählen

Folgen Sie den Anweisungen der Wallet zur Auswahl eines Validators.
Informationen über potenziell leistungsstarke Validatoren erhalten Sie über die
unten stehenden Links. Die Solana Foundation empfiehlt keinen bestimmten
Validator.

Die Website solanabeach.io wird von einem unserer Validatoren, Staking
Facilities, erstellt und gepflegt. Sie bietet einige allgemeine grafische
Informationen über das Netzwerk als Ganzes sowie eine Liste aller Validatoren
und einige aktuelle Leistungsstatistiken zu jedem einzelnen.

- https://solanabeach.io

Um Blockproduktionsstatistiken anzuzeigen, verwenden Sie die
Solana-Befehlszeilentools:

- `solana validators`
- `solana block-production`

Das Solana-Team gibt keine Empfehlungen zur Interpretation dieser Informationen.
Führen Sie Ihre eigene Sorgfaltsprüfung durch.

### Delegieren Sie Ihren Stake

Folgen Sie den Anweisungen der Wallet, um Ihren Stake an den von Ihnen gewählten
Validator zu delegieren.
