Jede Solana-[Transaktion](/docs/core/transactions) erfordert eine in SOL zu
zahlende Gebühr. Die Gebühr besteht aus zwei Komponenten: einer **Basisgebühr**
und einer optionalen **Priorisierungsgebühr**. Die Basisgebühr entschädigt
Validatoren für die kryptografische Arbeit der Signaturverifizierung. Die
Priorisierungsgebühr erhöht die Wahrscheinlichkeit, dass der aktuelle Leader
Ihre Transaktion vor konkurrierenden Transaktionen einplant.

## Wichtige Fakten

- **Basisgebühr**: pro Signatur, aufgeteilt in 50 % verbrannt / 50 % an den
  Validator.
- **Priorisierungsgebühr**:
  `ceil(compute_unit_price * compute_unit_limit / 1,000,000)` Lamports. 100 % an
  den Validator.
