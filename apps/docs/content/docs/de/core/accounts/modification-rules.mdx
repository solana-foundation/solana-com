<Callout type="info" title="Zusammenfassung">
  Laufzeitprüfungen nach jeder Anweisung: Nur der Eigentümer kann Lamports
  abbuchen oder Daten ändern, Daten können maximal 10 KiB pro Anweisung wachsen,
  Eigentümerwechsel erfordern nullinitialisierte Daten, und das ausführbare Flag
  ist irreversibel.
</Callout>

Die Solana-Laufzeitumgebung setzt diese Regeln nach jeder Anweisungsausführung
über
[`BorrowedInstructionAccount`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L722-L727)
Methoden durch. Jede Regel wird zum Zeitpunkt der Änderung geprüft, und die
Transaktion wird zurückgesetzt, wenn eine Prüfung fehlschlägt.

## Lamports-Regeln

| Regel                                                                | Durchsetzung                                                                                                                                                                                                                                    | Fehler                        |
| -------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Nur der Eigentümer kann Lamports abbuchen                            | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L785): prüft `is_owned_by_current_program()` wenn `lamports < current`                                                                     | `ExternalAccountLamportSpend` |
| Schreibgeschützte Konten können keine Lamports-Änderungen haben      | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L788-L789): prüft `is_writable()`                                                                                                               | `ReadonlyLamportChange`       |
| Jedes Programm kann Lamports einem beschreibbaren Konto gutschreiben | [`set_lamports()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L782-L789): die Eigentümerprüfung gilt nur, wenn der neue Saldo kleiner als der aktuelle Saldo ist; die Beschreibbarkeitsprüfung gilt weiterhin | `ReadonlyLamportChange`       |
| Lamports müssen über eine Anweisung hinweg ausgeglichen sein         | [`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L366-L398): verifiziert `get_lamports_delta() == 0`                                                                                  | `UnbalancedInstruction`       |

## Mietstatusübergänge

Konten existieren in einem von drei
[`RentState`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L19-L30)
Werten: _rs`Uninitialized`_ (0 Lamports), _rs`RentPaying`_ (über 0, aber unter
dem Mietbefreiungsminimum) und _rs`RentExempt`_ (am oder über dem Minimum).
Nicht zulässige Übergänge erzeugen
_rs`TransactionError::InsufficientFundsForRent`_.

Die Runtime setzt diese Regeln über
[`transition_allowed()`](https://github.com/anza-xyz/agave/blob/v3.1.8/svm/src/rent_calculator.rs#L107-L126)
durch:

- Jedes Konto kann zu _rs`Uninitialized`_ (schließen) oder _rs`RentExempt`_
  übergehen.
- Kein Konto kann von _rs`Uninitialized`_ oder _rs`RentExempt`_ in
  _rs`RentPaying`_ eintreten. Alle neuen Konten müssen mietbefreit sein.

## Regeln für Kontoausleihe

Während der Anweisungsausführung setzt die Runtime Single-Writer-Ausleihsemantik
für Konten durch. Ein Programm kann entweder eine veränderbare Referenz oder
mehrere unveränderbare Referenzen auf ein Konto erhalten, aber nicht beides
gleichzeitig. Wenn ein Programm versucht, ein Konto auszuleihen, das bereits
veränderbar ausgeliehen ist (oder veränderbar ein Konto auszuleihen, das bereits
unveränderbar ausgeliehen ist), schlägt die Anweisung mit
_rs`AccountBorrowFailed`_ über
[`try_borrow()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L498-L508)
und
[`try_borrow_mut()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/transaction_accounts.rs#L512-L520)
fehl. Wenn eine Anweisung abgeschlossen wird, während eine Ausleihe noch
aussteht, gibt die Runtime _rs`AccountBorrowOutstanding`_ in
[`TransactionContext::pop()`](https://github.com/anza-xyz/agave/blob/v3.1.8/transaction-context/src/lib.rs#L374-L385)
zurück.
