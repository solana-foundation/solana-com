Ein Solana-Programm ist ein
[Konto](/docs/core/accounts/account-types#program-accounts), das ausführbaren
sBPF-Bytecode enthält und dessen `executable`-Flag auf `true` gesetzt ist.
Programme sind zustandslos. Alle veränderbaren Zustände befinden sich in
separaten Datenkonten, die über [Anweisungen](/docs/core/instructions) übergeben
werden.

## Wichtige Fakten

- **Kompiliert zu sBPF**: Programme werden über LLVM zu Solana Bytecode Format
  (sBPF) kompiliert und in ausführbaren Konten gespeichert.
- **Zustandslos**: Alle veränderbaren Zustände befinden sich in separaten
  Datenkonten, nicht im Programmkonto.
- **Upgradefähig**: Programme, die mit loader-v3 (BPF Loader Upgradeable)
  deployed wurden, können aktualisiert werden, wenn eine Upgrade-Authority
  gesetzt ist; das Widerrufen dieser Authority macht das Programm
  unveränderlich.
