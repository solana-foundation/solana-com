Wenn Sie von `@solana/web3.js` (v1), `@solana/wallet-adapter-*` oder den
früheren Framework-Kit-Paketen (`@solana/client`, `@solana/react-hooks`,
`@solana/web3-compat`) kommen: Diese Ansätze sind überholt. Neue Anwendungen
bauen direkt auf `@solana/kit` und den Kit-Plugins auf. Diese Seite ordnet die
Ihnen bereits bekannten Konzepte ihren Kit-Entsprechungen zu.

## Konzept-Mapping

| web3.js v1 / wallet-adapter            | Kit                                                                               |
| -------------------------------------- | --------------------------------------------------------------------------------- |
| `new Connection(url)`                  | `createSolanaRpc(url)`, oder `.use(solanaRpc({ rpcUrl }))` auf einem Client       |
| `PublicKey`                            | `Address` — String umwandeln mit `address("...")`                                 |
| `Keypair`                              | `generateKeyPairSigner()` oder die `@solana/kit-plugin-signer`-Plugins            |
| `SystemProgram.transfer(...)`          | `getTransferSolInstruction(...)` aus `@solana-program/system`                     |
| `LAMPORTS_PER_SOL`-Arithmetik          | `lamports(...)` aus `@solana/kit`                                                 |
| `Transaction` / `VersionedTransaction` | Transaktionsnachrichten, die vom Transaction Planner erstellt und verpackt werden |
| `sendAndConfirmTransaction(...)`       | `client.sendTransaction([...])` (Planner signiert, sendet und bestätigt)          |
| `@solana/wallet-adapter-*`             | `@solana/kit-plugin-wallet` + Wallet Standard Discovery                           |

## Von web3.js migrieren?

Wenn Sie eine umfangreiche `@solana/web3.js`-v1-Codebasis betreuen und eine
vollständige Umstellung auf Kit noch nicht praktikabel ist, stellt web3.js v3
die Brücke dar. Es wird derzeit als `@solana/web3.js@rc` (die `3.0.0-rc`-Linie)
ausgeliefert und baut die klassische klassenbasierte API — `Connection`,
`Keypair`, `Transaction` — auf `@solana/kit`- Internas auf. Da es auf Kit
aufbaut, ist die Interoperabilität eng: `PublicKey` ist ein veralteter Alias für
`Address`, und eine v3-`Keypair` erfüllt strukturell Kits `KeyPairSigner`,
sodass sie direkt an Kit-APIs, Plugins und Codama-generierte Clients übergeben
werden kann.

v3 ist noch ein **Release Candidate** — pinnen Sie genaue Versionen und rechnen
Sie mit gewissen API-Änderungen zwischen RCs. Es ist ein Legacy-Interop-Pfad,
kein Standard für neue Projekte; neue Anwendungen sollten direkt auf Kit
aufbauen. Für die Migration selbst siehe den offiziellen
[web3.js v1 → v3 Migrationsleitfaden](https://github.com/solana-foundation/solana-web3.js/blob/v3.x/docs/web3js-v1-to-v3-migration.md).

## Wallets

Kit verwendet keinen wallet-adapter. `@solana/kit-plugin-wallet` erkennt Wallet
Standard-Wallets und stellt sie über `client.wallet` sowie die
[React-Hooks](/docs/frontend/react-hooks) bereit. Moderne Wallets (Phantom,
Solflare, Backpack und andere) melden sich selbst über den Wallet Standard an,
sodass kein wallet-spezifisches Adapter-Paket erforderlich ist.

Sie können auch vorgefertigte Tools wie
[@solana/connector](https://www.npmjs.com/package/@solana/connector) verwenden,
das Kit-Primitive direkt zur Verbindung mit Wallets nutzt.

<Callout type="info" title="Noch auf wallet-adapter?">
  Wenn Sie eine App pflegen, die auf `@solana/wallet-adapter-*` basiert und noch
  nicht zur Migration bereit sind, bleibt das [wallet-adapter
  Repository](https://github.com/anza-xyz/wallet-adapter) die Referenz für
  diesen Stack. Für alle Neuentwicklungen empfehlen wir das Kit-Wallet-Plugin.
</Callout>
