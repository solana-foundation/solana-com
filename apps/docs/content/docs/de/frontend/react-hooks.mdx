  Teile einen Kit-Client in deiner React-App mit @solana/react und den

`@solana/react` veröffentlicht einen Kit-Client über einen einzigen Provider im
Komponentenbaum. Hooks lesen daraus RPC-Daten, Abonnements und den
Wallet-Status. Die Browser-Wallet-Hooks befinden sich in
`@solana/kit-plugin-wallet/react` und lesen denselben Provider – es ist kein
separater Wallet-Provider erforderlich.

## Installation

```terminal
$ npm install @solana/kit @solana/kit-plugin-rpc @solana/kit-plugin-wallet @solana/react
```

## Den Client erstellen

Erstelle den Client einmalig auf Modulebene, außerhalb des React-Baums. Im
Browser macht `walletSigner` das verbundene Wallet sowohl zum Fee-Payer als auch
zur Identität.

```tsx title="client.ts"
import { createClient } from "@solana/kit";
import { solanaRpc } from "@solana/kit-plugin-rpc";
import { walletSigner } from "@solana/kit-plugin-wallet";

export const client = createClient()
  .use(walletSigner({ chain: "solana:devnet" }))
  .use(solanaRpc({ rpcUrl: "https://api.devnet.solana.com" }));

export type AppClient = Awaited<typeof client>;
```

`walletSigner` fügt `client.wallet` hinzu (Wallet Standard-Erkennung und
-Verbindung), und `solanaRpc` fügt `client.rpc` sowie den
`client.sendTransaction`-Helper hinzu. Exportiere den Client-Typ, eingebettet in
`Awaited<>`, damit er korrekt bleibt, falls du später ein Plugin hinzufügst, das
asynchron auflöst, und übergib diesen `AppClient` an jeden `useClient`-Aufruf.

## Den Baum einmalig einwickeln

`ClientProvider` veröffentlicht den Client. Alle nachfolgenden Hooks lesen
daraus.

```tsx title="providers.tsx"
"use client";

import { ClientProvider } from "@solana/react";
import { client } from "./client";

export function Providers({ children }: { children: React.ReactNode }) {
  return <ClientProvider client={client}>{children}</ClientProvider>;
}
```

## Wallet-Hooks

Die Wallet-Hooks lesen die `wallet`-Fähigkeit des Clients und funktionieren
daher nur innerhalb eines `ClientProvider`, dessen Client ein Wallet-Plugin
installiert hat.

- `useWallets()` — Über Wallet Standard erkannte Wallets für die Chain des
  Clients.
- `useWalletStatus()` —
  `"pending" | "disconnected" | "connecting" | "connected" | "disconnecting" | "reconnecting"`.
- `useConnectedWallet()` — `{ account, signer, wallet }` oder `null`. `signer`
  ist `null` für schreibgeschützte Wallets.
- `useConnect()` / `useDisconnect()` — gibt ein `ActionResult` zurück, dessen
  `dispatch` (und `dispatchAsync`) die Aktion ausführt und dessen `isRunning`,
  `status`, `data`, `error` und `reset` das Ergebnis verfolgen.

```tsx
"use client";

import {
  useConnect,
  useConnectedWallet,
  useDisconnect,
  useWallets,
  useWalletStatus
} from "@solana/kit-plugin-wallet/react";

function WalletPanel() {
  const status = useWalletStatus();
  const wallets = useWallets();
  const connected = useConnectedWallet();
  const connect = useConnect();
  const disconnect = useDisconnect();

  if (status === "pending") return null; // wait out auto-reconnect

  if (connected) {
    return (
      <div>
        <p>{connected.account.address}</p>
        <button onClick={() => disconnect.dispatch()}>Disconnect</button>
      </div>
    );
  }

  return (
    <div>
      {wallets.map((wallet) => (
        <button
          key={wallet.name}
          disabled={connect.isRunning}
          onClick={() => connect.dispatch(wallet)}
        >
          Connect {wallet.name}
        </button>
      ))}
    </div>
  );
}
```

## Eine Transaktion senden

`useClient` gibt denselben Client zurück, den du veröffentlicht hast. Erstelle
Anweisungen mit einem generierten Programm-Client und sende sie über
`client.sendTransaction`.

```tsx
"use client";

import { address, lamports } from "@solana/kit";
import { getTransferSolInstruction } from "@solana-program/system";
import { useClient } from "@solana/react";
import { useConnectedWallet } from "@solana/kit-plugin-wallet/react";
import type { AppClient } from "./client";

function SendSol({ destination }: { destination: string }) {
  const client = useClient<AppClient>();
  const connected = useConnectedWallet();

  async function handleSend() {
    if (!connected?.signer) return;
    const transfer = getTransferSolInstruction({
      source: connected.signer,
      destination: address(destination),
      amount: lamports(10_000_000n) // 0.01 SOL
    });
    const result = await client.sendTransaction([transfer]);
    console.log("sent", result.context.signature);
  }

  return (
    <button disabled={!connected?.signer} onClick={handleSend}>
      Send 0.01 SOL
    </button>
  );
}
```

Die Übergabe des exportierten Typs `AppClient` an `useClient` gibt dem Hook den
vollständigen plugin-erweiterten Client, sodass `client.rpc` und
`client.sendTransaction` typisiert sind.

## Daten lesen

`@solana/react` stellt Hooks bereit, die über den Client lesen:

- `useRequest(source)` — einmaliger RPC-Lesevorgang; gibt
  `{ data, error, status, refresh }` zurück.
- `useSubscription(source)` — ein Live-Abonnement; gibt
  `{ data, error, reconnect, status }` zurück.
- `useTrackedData(spec)` — ein per RPC initialisierter Wert, der durch ein
  Abonnement aktuell gehalten wird.

```tsx
"use client";

import { address } from "@solana/kit";
import { useClient, useRequest } from "@solana/react";
import { useMemo } from "react";
import type { AppClient } from "./client";

function Balance({ owner }: { owner: string }) {
  const client = useClient<AppClient>();
  const source = useMemo(
    () => client.rpc.getBalance(address(owner)),
    [client, owner]
  );
  const { data, status, refresh } = useRequest(source);

  if (status === "fetching") return <p>Loading…</p>;
  if (status === "error") return <p role="alert">RPC error</p>;

  return (
    <div>
      <p>Lamports: {data?.value.toString()}</p>
      <button onClick={() => refresh()}>Refresh</button>
    </div>
  );
}
```

Memoize die Quelle (mit `useMemo` oder `useCallback` für eine Funktion), damit
der Hook nur bei Änderungen seiner Eingaben neu abruft und nicht bei jedem
Render-Vorgang.

Für Caching und Revalidierung umschließen die Unterpfade `@solana/react/swr` und
`@solana/react/query` dieselben Hooks für [SWR](https://swr.vercel.app) und
[TanStack Query](https://tanstack.com/query).

## Häufige Muster für Solana-Entwickler

- **Ein Provider**: `ClientProvider` ist der einzige Provider, den Sie
  benötigen; Wallet-Hooks lesen ihn über die `wallet`-Fähigkeit des Clients.
- **Server-Komponenten-bewusst**: Markieren Sie nur Blattkomponenten, die Hooks
  aufrufen, mit `"use client"`; Server-Lesevorgänge können einen einfachen
  Kit-RPC-Client ohne Wallet verwenden.
- **Testing**: Veröffentlichen Sie einen gemockten Client über `ClientProvider`,
  um Wallets und RPC-Antworten zu simulieren.

<Callout type="warn" title="Zwei Hook-Familien, gleiche Namen">
  `useSignIn` und `useSignMessage` existieren sowohl in `@solana/react` (sie
  nehmen ein `UiWalletAccount`-Argument entgegen) als auch in
  `@solana/kit-plugin-wallet/react` (clientbasiert, kein Konten-Argument, geben
  ein `ActionResult` zurück). Verwenden Sie die
  `@solana/kit-plugin-wallet/react`-Versionen mit dem oben beschriebenen
  Client-Muster; die älteren `@solana/react`-Wallet-Hooks werden abgelöst.
</Callout>

Ergänzen Sie diese Anleitung mit der
[Kit-Client-Übersicht](/docs/frontend/client), um zu verstehen, von welchem
Client jeder Hook liest.
