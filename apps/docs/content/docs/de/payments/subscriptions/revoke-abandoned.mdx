---
title: Aufgabe von verlassenen Delegierungen und Abonnements widerrufen
description:
  Leitfaden zur Rückforderung von rent aus einer Delegierung oder einem
  Abonnement, deren Subscription Authority nicht mehr vorhanden ist.
---

Delegierungen und Abonnements halten rent, die bei einem Widerruf normalerweise
an den eingetragenen Zahler zurückfließt. Wenn ein Sponsor das Konten finanziert
hat und der Benutzer später seine Subscription Authority schließt oder ersetzt,
ist das Konten verwaist und sein rent steckt fest. `RevokeAbandonedDelegation`
und `RevokeAbandonedSubscription` ermöglichen es dem eingetragenen Zahler,
diesen zurückzufordern.

## Wann es gilt

Nur zulässig, wenn die Subscription Authority des Kontos nachweislich terminal
ist – geschlossen oder geschlossen und neu erstellt (sodass `init_id` nicht mehr
übereinstimmt). Eine aktive Delegierung oder ein aktives Abonnement kann auf
diese Weise niemals geschlossen werden.

|                                               | `RevokeAbandonedDelegation`                                                                                                         | `RevokeAbandonedSubscription`                                                                                                                                                          |
| --------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Schließt                                      | Feste oder wiederkehrende Delegierungs-PDA                                                                                          | Abonnement-PDA                                                                                                                                                                         |
| Signer                                        | Eingetragener Zahler                                                                                                                | Eingetragener Zahler                                                                                                                                                                   |
| Konten                                        | Delegierung + eingetragene Authority                                                                                                | Abonnement + eingetragene Authority + Plan                                                                                                                                             |
| Berechtigt wenn                               | Authority geschlossen oder neu erstellt (`init_id` stimmt nicht überein)                                                            | Dasselbe gilt für den Mint des Plans – der Mint wird aus dem gebundenen Plan ausgelesen, sodass eine unabhängige Authority nicht eingesetzt werden kann, um ein Schließen zu erzwingen |
| Stattdessen normalen Widerruf verwenden, wenn | Delegierung ist [abgelaufen oder vollständig aufgebraucht](/docs/payments/subscriptions/recurring-delegation#revoke-the-delegation) | Abonnement ist [storniert und abgelaufen](/docs/payments/subscriptions/subscription-plan#cancel-and-revoke)                                                                            |

`RevokeAbandonedDelegation` ist außerdem der einzige Wiederherstellungsweg des
eingetragenen Zahlers für eine niemals ablaufende Delegierung
(`expiry_ts == 0`), die nicht vollständig aufgebraucht ist: Der normale Widerruf
greift für den Zahler nur bei Ablauf oder – bei einer festen Delegierung – bei
vollständigem Verbrauch.

## Eine verlassene Delegierung widerrufen

Der eingetragene Zahler unterschreibt. Übergeben Sie die inaktive
Delegierungs-PDA und ihre eingetragene Subscription Authority (die
möglicherweise bereits geschlossen ist). Der rent wird an den Zahler
zurückgegeben.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedDelegationInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedDelegationInstruction({
      payer: payerSigner,
      delegationAccount: delegationPda,
      subscriptionAuthority: subscriptionAuthorityPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let delegation_pda: Address = address!("DELEGATION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");

    let revoke_abandoned_ix = RevokeAbandonedDelegationBuilder::new()
        .payer(payer)
        .delegation_account(delegation_pda)
        .subscription_authority(subscription_authority)
        .instruction();
    ```

  </Tab>
</Tabs>

## Ein aufgegebenes Abonnement widerrufen

Der eingetragene Zahler unterzeichnet. Übergeben Sie die PDA des aufgegebenen
Abonnements, die eingetragene Abonnement-Autorität (die möglicherweise bereits
geschlossen ist) und den Plan, zu dem das Abonnement gehört. rent wird an den
Zahler zurückgegeben.

<Tabs items={["TypeScript", "Rust"]}>
  <Tab value="TypeScript">

    ```ts
    import { getRevokeAbandonedSubscriptionInstruction } from '@solana/subscriptions';

    // Build the instruction and send it in a transaction like any other.
    const revokeAbandonedIx = getRevokeAbandonedSubscriptionInstruction({
      payer: payerSigner,
      subscriptionAccount: subscriptionPda,
      subscriptionAuthority: subscriptionAuthorityPda,
      planPda,
    });
    ```

  </Tab>

  <Tab value="Rust">

    ```rust
    use solana_address::{address, Address};
    use subscriptions::generated::instructions::*;

    let payer: Address = address!("PAYER_THAT_FUNDED_RENT_HERE");
    let subscription_pda: Address = address!("SUBSCRIPTION_PDA_ADDRESS_HERE");
    let subscription_authority: Address = address!("RECORDED_SUBSCRIPTION_AUTHORITY_HERE");
    let plan_pda: Address = address!("PLAN_PDA_ADDRESS_HERE");

    let revoke_abandoned_ix = RevokeAbandonedSubscriptionBuilder::new()
        .payer(payer)
        .subscription_account(subscription_pda)
        .subscription_authority(subscription_authority)
        .plan_pda(plan_pda)
        .instruction();
    ```

  </Tab>
</Tabs>

## Hinweise

- Der eingetragene Zahler unterzeichnet, nicht der Delegierende, der Delegierte
  oder der Abonnent.
- Beide Anweisungen schlagen fehl, wenn die Abonnement-Autorität für das Konten
  noch aktiv ist – verwenden Sie stattdessen den normalen Widerruf.
- `RevokeAbandonedDelegation` funktioniert sowohl für feste als auch für
  wiederkehrende Delegierungen.
