---

Sie können Zahlungen für API-Anfragen über Pay.sh akzeptieren, indem Sie einen
Pay-Server einrichten. Installieren Sie Pay mit:

```bash
brew install pay
```

oder

```bash
npm install -g @solana/pay
```

Überprüfen Sie die Installation mit:

```bash
pay --version
```

Die Toolchain richtet Ihre Adresse automatisch ein. Führen Sie Folgendes aus, um
Ihr Wallet einzurichten und aufzuladen:

```bash
pay setup
```

## Einrichten des Demo-Servers

Sie können mit der Annahme von Zahlungen beginnen, indem Sie Ihren Server
starten. Sie können einen Demo-Server in Aktion sehen mit:

```bash
pay server demo
```

Dies erfüllt zwei Aufgaben: Es startet einen aktiven Pay-Server, den Sie per
HTTP-Anfrage abfragen können, und es generiert eine `pay-demo.yml`-Datei in
Ihrem aktuellen Verzeichnis.

Alle API-Routen werden über die `endpoints` YAML-Konfiguration eingerichtet. Um
Zahlungen zu akzeptieren, müssen Sie ein `metering:`-Attribut hinzufügen. Die
Route `/api/v1/reports/usage` wurde bereits als Beispiel für Sie eingerichtet:

```yaml
endpoints:
  - method: GET
    path: "api/v1/reports/usage"
    resource: "reports"
    description: "Usage report — flat fee, no splits."
    metering:
      dimensions:
        - direction: usage
          unit: requests
          scale: 1
          tiers:
            - price_usd: 0.01
```

Sie können testen, ob die Zahlungen funktionieren, indem Sie den folgenden
Befehl als Client in einem anderen Terminal ausführen:

```bash
pay --sandbox curl http://127.0.0.1:1402/api/v1/reports/usage
```

Erfahren Sie mehr über die Einrichtung von Pay.sh im Produktionsbetrieb in der
