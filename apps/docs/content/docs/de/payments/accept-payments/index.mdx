Akzeptieren Sie Zahlungen auf Solana mithilfe von Drop-in-Komponenten, dem
Solana Pay-Protokoll oder direkter RPC-Integration.

## Konvention für Empfangsadressen

Auf Solana **teilen Sie Ihre Wallet-Adresse, nicht Ihr token account**. Es ist
üblich, dass der Absender das entsprechende
[Associated Token Account (ATA)](/docs/payments/how-payments-work#token-accounts)
für den jeweiligen Token ableitet, den er sendet (siehe
[Zahlungen senden](/docs/payments/send-payments) für weitere Details).

**Beispiel:** Teilen Sie `7EcDhS...` (Ihre Wallet), nicht `9xKpQ...` (Ihr USDC
token account).

<Callout>
  Für Point-of-Sale und E-Commerce siehe [Solana
  Pay](/docs/payments/accept-payments/solana-pay) für QR-Codes und
  Zahlungs-URLs, oder [Payment
  Button](/docs/payments/accept-payments/payment-button) für Drop-in
  React-Komponenten.
</Callout>

## Integrationsoptionen

| Ansatz                                                          | Am besten geeignet für                | Komplexität |
| --------------------------------------------------------------- | ------------------------------------- | ----------- |
| [Payment Button](/docs/payments/accept-payments/payment-button) | React-Apps, schnelle Integration      | Niedrig     |
| [Solana Pay](/docs/payments/accept-payments/solana-pay)         | QR-Codes, Zahlungslinks, POS          | Mittel      |
| [Direct RPC](/docs/payments/interacting-with-solana)            | Individuelle Abläufe, volle Kontrolle | Höher       |

## Verifizierung

Überprüfen Sie nach Erhalt einer Zahlung, ob diese eingegangen ist, mithilfe der
[Verification Tools](/docs/payments/accept-payments/verification-tools). Für
Anwendungen mit hohem Volumen siehe
[Indexing](/docs/payments/accept-payments/indexing) für produktionsreife
Transaktionsüberwachung.
