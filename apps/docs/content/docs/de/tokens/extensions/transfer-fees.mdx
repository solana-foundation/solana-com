## Was sind Transfergebühren?

Die Mint-Erweiterung _rs`TransferFeeConfig`_ des Token Extensions Program wendet
eine Gebühr auf jede Übertragung für diese Mint an.

Zum Zeitpunkt der Übertragung:

- Der übertragene Betrag wird um die konfigurierte Gebühr reduziert
- Die einbehaltene Gebühr wird auf dem Ziel-Token-Konto verfolgt
- Einbehaltene Gebühren können direkt von Token-Konten abgehoben oder zur Mint
  gesammelt werden
- Der `withdraw_withheld_authority` kann gesammelte Gebühren auf ein
  Gebührenempfänger-Token-Konto verschieben

<Callout type="info" title="Token-Konto-Erweiterungen">
  Wenn ein Token-Konto für eine Mint mit _rs`TransferFeeConfig`_ initialisiert
  wird, wird das Token-Konto mit _rs`TransferFeeAmount`_ initialisiert. Wenn das
  Token-Konto über das [Associated Token
  Program](/docs/tokens/basics/create-token-account#how-to-create-an-associated-token-account)
  erstellt wird, wird die erforderliche Kontogröße berechnet und das Token-Konto
  mit der erforderlichen Größe und den rent-exempt Lamports erstellt.
</Callout>

## So erstellen Sie eine Transfer-Fee-Mint

Um eine Transfer-Fee-Mint zu erstellen:

1. Berechnen Sie die Mint-Kontogröße und die benötigte Miete für die Mint und
   die _rs`TransferFeeConfig`_-Erweiterung.
2. Erstellen Sie das Mint-Konto mit _rs`CreateAccount`_, initialisieren Sie
   _rs`TransferFeeConfig`_ und initialisieren Sie die Mint mit
   _rs`InitializeMint`_.
3. Erstellen Sie Token-Konten für die Mint. _rs`TransferFeeAmount`_ wird
   automatisch für Token-Konten aktiviert.
4. Verwenden Sie _rs`TransferCheckedWithFee`_, um Token zu übertragen und die
   erwartete Gebühr zu überprüfen. _rs`TransferChecked`_ funktioniert ebenfalls
   und behält automatisch die konfigurierte Gebühr auf dem Ziel-Token-Konto ein.
5. Verwenden Sie _rs`WithdrawWithheldTokensFromAccounts`_, um einbehaltene
   Gebühren direkt von Token-Konten auf ein Gebührenempfänger-Token-Konto zu
   verschieben, signiert vom `withdraw_withheld_authority` der Mint.
6. Verwenden Sie das erlaubnisfreie _rs`HarvestWithheldTokensToMint`_, um
   einbehaltene Gebühren von Token-Konten zum Mint-Konto zu verschieben, und
   verwenden Sie dann _rs`WithdrawWithheldTokensFromMint`_, um die einbehaltenen
   Gebühren des Mint-Kontos auf ein Gebührenempfänger-Token-Konto abzuheben,
   signiert vom `withdraw_withheld_authority` der Mint.
7. Verwenden Sie _rs`SetTransferFee`_, um die nächste
   Transfergebührenkonfiguration zu aktualisieren, die ab zwei Epochen später in
   Kraft tritt.

## !!steps Kontengröße berechnen

Berechnen Sie die Größe des mint account für die Basis-Mint plus die
_rs`TransferFeeConfig`_ Erweiterung. Dies ist die Größe, die in
_rs`CreateAccount`_ verwendet wird.

<CodePlaceholder title="Beispiel" />

```ts !! title="Example"
const client = await createLocalClient();
const mint = await generateKeyPairSigner();

// !focus(1:14)
const transferFeeConfigExtension = extension("TransferFeeConfig", {
  transferFeeConfigAuthority: client.payer.address,
  withdrawWithheldAuthority: client.payer.address,
  withheldAmount: 0n,
  olderTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  },
  newerTransferFee: {
    epoch: 0n,
    maximumFee: 10n,
    transferFeeBasisPoints: 150
  }
});

// !focus(1:1)
const mintSpace = BigInt(getMintSize([transferFeeConfigExtension]));
```
