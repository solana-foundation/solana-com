## Was bewirkt „Überschüssige Lamports abheben“?

_rs`WithdrawExcessLamports`_ verschiebt SOL, das von einem Token Program-Konten
über seinem mietbefreiten Minimum gehalten wird, auf ein Zielkonto.
Token-Guthaben und Mint-Angebot bleiben unberührt – nur die überschüssigen
lamports werden bewegt.

Dies stellt SOL wieder her, das zuvor nicht zugänglich war: lamports, die an ein
token account, einen Mint oder ein Multisig (alles im Besitz des Token Program)
gesendet wurden, konnten andernfalls nicht verschoben werden.

## Konten

| Konten        | Beschreibung                                                                         |
| ------------- | ------------------------------------------------------------------------------------ |
| `source`      | Beschreibbar. Das token account, der Mint oder das Multisig, von dem abgehoben wird. |
| `destination` | Beschreibbar. Empfängt die überschüssigen lamports.                                  |
| `authority`   | Unterzeichnet basierend auf dem Typ des Quellkontos (siehe unten).                   |

## Wer kann unterzeichnen

Der erforderliche Signer hängt vom Typ des `source`-Kontos ab:

| Quellkonto                    | Signer                                                            |
| ----------------------------- | ----------------------------------------------------------------- |
| Token account                 | Der Eigentümer des token account (oder seine Multisig-Signer).    |
| Mint mit einer Mint-Autorität | Die Mint-Autorität.                                               |
| Mint ohne eine Mint-Autorität | Das mint account selbst muss unterzeichnen (siehe Hinweis unten). |
| Multisig                      | Die konfigurierten M-von-N-Signer.                                |

<Callout type="info">

Der Fall „Mint ohne Mint-Autorität“ ist ungewöhnlich. Wenn er zutrifft, ist das
Mint-Konten der Signer: Ein programmeigenes (off-curve) Mint signiert per CPI
aus seinem besitzenden Programm, und ein keypair (on-curve) Mint signiert mit
seinem Schlüssel.

</Callout>

<Callout type="warn">

Native (wrapped SOL) token accounts werden nicht unterstützt und geben
_rs`NativeNotSupported`_ zurück. Die Auszahlung darf das Quell-Konten nicht
unter sein Rent-Exempt-Minimum senken.

</Callout>

## So heben Sie überschüssige Lamports ab

Das Beispiel verwendet einen konfigurierten
[`@solana/kit`](/docs/clients/official/javascript#solana-kit) Client — siehe
[Token übertragen](/docs/tokens/basics/transfer-tokens) für die
Client-Einrichtung.

```ts title="Withdraw excess lamports (Kit)"
import { getWithdrawExcessLamportsInstruction } from "@solana-program/token";

// source/destination: addresses of the account to drain and the lamport recipient.
// authority: a TransactionSigner that signs per the source account type.
const instruction = getWithdrawExcessLamportsInstruction({
  source,
  destination,
  authority
});

await client.sendTransaction([instruction]);
```

Für eine Multisig-Autorität übergeben Sie die Mitsignierer mit
_ts`multiSigners`_.
