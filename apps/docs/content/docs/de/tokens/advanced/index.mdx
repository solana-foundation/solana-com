Neben den gängigen Erstellungs-, Mint- und Transfer-Operationen wird das Token
Program auch von On-Chain-Programmen über CPI aufgerufen und enthält Anweisungen
zur Wiederherstellung von lamport:

<Cards>
  <Card
    title="Das Token Program via CPI aufrufen"
    href="/docs/tokens/advanced/cpi"
  >
    Rufen Sie das Token Program aus einem On-Chain-Programm auf – übertragen, minen und verbrennen
    Sie Token, signieren Sie mit einer PDA-Autorität und bündeln Sie Anweisungen in einem einzigen CPI.
  </Card>

<Card
  title="Überschüssige Lamports abheben"
  href="/docs/tokens/advanced/withdraw-excess-lamports"
>
  Stellen Sie SOL oberhalb des mietbefreiten Minimums aus token account, Mints
  und Multisig-Konten wieder her, ohne die Token-Guthaben zu verändern.
</Card>

  <Card title="Lamports entpacken" href="/docs/tokens/advanced/unwrap-lamports">
    Konvertieren Sie eingepacktes SOL ganz oder teilweise zurück in lamport, ohne
    den token account zu schließen.
  </Card>
</Cards>

<Callout type="info">

_rs`WithdrawExcessLamports`_ und _rs`UnwrapLamports`_ sind über den
[`@solana-program/token`](https://www.npmjs.com/package/@solana-program/token)
Client verfügbar. Der Aufruf des Token Program via CPI gilt für
On-Chain-Programme.

</Callout>

## Lamport-Wiederherstellung

_rs`WithdrawExcessLamports`_ stellt SOL oberhalb des mietbefreiten Minimums aus
token account, Mints und Multisig-Konten wieder her. _rs`UnwrapLamports`_
konvertiert eingepacktes SOL in einem nativen token account zurück in lamport.
