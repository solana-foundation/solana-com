  Einheitliche Solana-Signierung über mehrere Schlüsselverwaltungs-Backends

**Solana Keychain bietet eine einheitliche Schnittstelle zum Signieren von
Solana-Transaktionen über mehrere Schlüsselverwaltungs-Backends hinweg.**
Entwickeln Sie lokal mit einem In-Memory-Schlüssel und wechseln Sie in der
Produktion zu Enterprise-Schlüsselverwaltung, Cloud-HSMs oder verwalteten
Wallet-Diensten – ohne Ihren Signiercode neu schreiben zu müssen.

### Warum Keychain?

- **Einheitliche Schnittstelle**: Ein `SolanaSigner`-Trait funktioniert mit
  allen Backends
- **Backends austauschen**: Wechseln Sie den Schlüsselverwaltungsanbieter per
  Konfiguration, nicht per Code
- **Abstraktion ohne Mehrkosten**: Feature-Flags binden nur das ein, was Sie
  benötigen
- **Geprüft**: Sicherheitsüberprüft von [Accretion](https://accretion.xyz)

### Architektur

- **Sprachen**: Rust + TypeScript, mit vollständiger Parität
- **Trait**: Einheitliche `SolanaSigner`-Schnittstelle
- **Backends**: Memory, Vault, AWS KMS, GCP KMS, Privy, Turnkey, Fireblocks,
  CDP, Crossmint, Dfns, Openfort, Para, Utila
- **Kompatibilität**: `@solana/kit` und `@solana/signers` kompatibel
  (TypeScript) | `solana-sdk` und `solana-sdk-v3` kompatibel (Rust)

### Unterstützte Backends

Die Backends umfassen lokale Entwicklung, selbst gehostete und cloudbasierte
Schlüsselverwaltung sowie verwaltete oder MPC-Wallet-Dienste. Siehe
[Backend auswählen](/docs/tools/keychain/choosing-a-backend), um
Verwahrungsmodelle zu vergleichen und das passende auszuwählen.

| Backend         | Anwendungsfall                       | Rust | TypeScript |
| --------------- | ------------------------------------ | ---- | ---------- |
| Memory          | Entwicklung, Tests                   | ✓    | ✓          |
| HashiCorp Vault | Selbst gehostete Schlüsselverwaltung | ✓    | ✓          |
| AWS KMS         | Cloud-nativ (AWS)                    | ✓    | ✓          |
| GCP KMS         | Cloud-nativ (GCP)                    | ✓    | ✓          |
| Privy           | Eingebettete Wallets                 | ✓    | ✓          |
| Turnkey         | Non-custodiale Schlüsselverwaltung   | ✓    | ✓          |
| Fireblocks      | Institutionelle Verwahrung           | ✓    | ✓          |
| CDP             | Coinbase Developer Platform          | ✓    | ✓          |
| Crossmint       | Verwaltete Crossmint-Wallets         | ✓    | ✓          |
| Dfns            | Dfns-Wallet-Infrastruktur            | ✓    | ✓          |
| Openfort        | Openfort-Backend-Wallets             | ✓    | ✓          |
| Para            | Para-MPC-Wallets                     | ✓    | ✓          |
| Utila           | Utila-MPC-Verwahrung                 | ✓    | ✓          |

## Schnellstart

### Rust

Installiere das Rust-Crate:

```bash
cargo add solana-keychain
```

Grundlegende Verwendung:

```rust
use solana_keychain::{Signer, SolanaSigner};

// Create a signer from any backend
let signer = Signer::from_memory("base58_private_key")?;

// All signers share the same interface
let pubkey = signer.pubkey();
let signature = signer.sign_transaction(&mut tx).await?;
```

- [Rust-Leitfaden](/docs/tools/keychain/getting-started/rust) - Vollständige
  Installation und Backend-Konfiguration
- [Crates.io](https://crates.io/crates/solana-keychain) - Rust-Crate

### TypeScript

Installiere das TypeScript-Paket:

```bash
pnpm add @solana/keychain
```

Grundlegende Verwendung:

```typescript
import { createKeychainSigner } from "@solana/keychain";
import { signTransactionWithSigners } from "@solana/signers";

// Create any signer via the unified factory
const signer = await createKeychainSigner({
  backend: "vault",
  vaultAddr: "https://vault.example.com:8200",
  vaultToken: "hvs.xxxxx",
  keyName: "my-solana-key",
  publicKey: "base58_public_key"
});

// Sign an already-compiled transaction
const signedTx = await signTransactionWithSigners(
  [signer],
  compiledTransaction
);
```

- [TypeScript-Leitfaden](/docs/tools/keychain/getting-started/typescript) -
  `@solana/keychain` Pakete
- [npm-Paket](https://www.npmjs.com/package/@solana/keychain) - npm-Paket
