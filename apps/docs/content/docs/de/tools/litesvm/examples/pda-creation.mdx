## Vollständiger Test

```rust title="tests/pda_test.rs"
use litesvm::LiteSVM;
use solana_account::Account;
use solana_sdk::pubkey::Pubkey;

#[test]
fn test_pda_creation() {
    let mut svm = LiteSVM::new();

    // Program that owns the PDA
    let program_id = Pubkey::new_unique();

    // Derive PDA address
    let seed = b"my_pda";
    let (pda, bump) = Pubkey::find_program_address(
        &[seed],
        &program_id
    );

    println!("PDA: {}", pda);
    println!("Bump: {}", bump);

    // Create PDA account manually (for testing)
    svm.set_account(pda, Account {
        lamports: 1_000_000,
        data: vec![bump], // Store bump seed in data
        owner: program_id,
        executable: false,
        rent_epoch: 0,
    }).unwrap();

    // Verify PDA was created
    let account = svm.get_account(&pda).unwrap();
    assert_eq!(account.owner, program_id);
    assert_eq!(account.data[0], bump);

    println!("PDA created successfully!");
}
```

## Wichtige Punkte

1. **PDA-Ableitung**: Verwende `Pubkey::find_program_address()`, um PDA-Adressen
   abzuleiten
2. **Seeds**: PDAs werden aus seeds und der Programm-ID abgeleitet
3. **Bump Seed**: Der bump seed stellt sicher, dass die Adresse außerhalb der
   ed25519-Kurve liegt
4. **Konten-Erstellung**: Verwende `set_account()`, um Konten manuell für Tests
   zu erstellen
5. **Eigentümerschaft**: PDAs müssen dem Programm gehören, das sie abgeleitet
   hat
