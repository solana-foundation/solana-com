# LiteSVM für TypeScript

Solana-Test-Suites, die `solana-test-validator` hochfahren, benötigen Sekunden
pro Test — du wartest auf den Prozessstart, RPC-Verbindungen und
Netzwerk-Roundtrips. LiteSVM führt dieselben Tests in Millisekunden aus. Kein
externer Prozess, kein Netzwerk, alles im Arbeitsspeicher innerhalb deines
Test-Runners.

<Callout type="info">
  **Getestet mit:** `@solana/kit@6.8.0`, `@solana/kit-plugin-litesvm@0.10.0`,
  `@solana/kit-plugin-signer@0.10.0`, `@solana-program/system@0.12.0`,
  `@solana-program/token@0.13.0`. Diese Plugin-Pakete sind neu und entwickeln
  sich schnell weiter — falls du auf eine Inkompatibilität stößt, prüfe
  [`testing/package.json`](https://github.com/solana-foundation/litesvm-docs/blob/main/testing/package.json)
  für die genauen Versionen, gegen die die Dokumentation und die Test-Suite
  laufen.
</Callout>

Du stellst einen Kit-Client aus `@solana/kit` zusammen, indem du Plugins mit
`.use()` installierst: ein Signer-Plugin
([`@solana/kit-plugin-signer`](https://www.npmjs.com/package/@solana/kit-plugin-signer)),
das LiteSVM-Plugin
([`@solana/kit-plugin-litesvm`](https://www.npmjs.com/package/@solana/kit-plugin-litesvm)),
und alle weiteren Programm-Plugins, die du benötigst. Das Ergebnis stellt
`client.sendTransaction()`, `client.rpc` sowie direkten Zugriff auf
Konten-Status, Clock und Sysvars bereit — alles ohne deinen Prozess zu
verlassen.

### Deinen Client erstellen

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";

// Payer first, then the LiteSVM transport.
const mySigner = await generateKeyPairSigner();
const client = createClient().use(signer(mySigner)).use(litesvm());

// Fund the payer
client.svm.airdrop(client.payer.address, lamports(5_000_000_000n));

// Check the balance
const balance = client.svm.getBalance(client.payer.address);
console.log("Balance:", balance, "lamports");
```

<Callout type="info">
  `createClient()` gibt eine leere Hülle zurück — jedes `.use(plugin)` fügt
  Funktionen hinzu. Das `signer()`-Plugin muss **vor** `litesvm()` kommen, da
  das Transport-Plugin einen Payer benötigt, an den es sich anhängen kann.
  `signer(mySigner)` ist eine Kurzschreibweise, um dasselbe keypair sowohl als
  `payer` als auch als `identity` zu installieren — verwende `payer()` oder
  `identity()` aus demselben Paket, wenn du diese Rollen aufteilen möchtest.
  Diese Reihenfolge gilt auch für `solanaRpc()`, wenn du ein echtes Netzwerk
  verwendest.
</Callout>

<Callout type="info">
  Bevorzugst du ein einzeiliges Setup für schnelle Experimente? Das
  `litesvm()`-Bundle enthält einen Airdrop, sodass du
  `.use(generatedPayerWithSol(lamports(10_000_000_000n)))` von
  `@solana/kit-plugin-signer` verketten kannst, um automatisch einen Payer zu
  generieren und zu finanzieren.
</Callout>

</Step>

</Steps>

## Den Client verstehen

Nach `.use(signer(...)).use(litesvm())` stellt der Client folgendes bereit:

| Property                    | Beschreibung                                                                                                                                                                                                 |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `client.payer`              | Das `TransactionSigner`, das Sie mit dem `signer()`-Plugin installiert haben (auch als `client.identity` gesetzt)                                                                                            |
| `client.identity`           | Der Wallet-Signer, dem Konten, Token und Berechtigungen gehören                                                                                                                                              |
| `client.svm`                | Direktes [`LiteSVM`](https://www.npmjs.com/package/litesvm)-Instance für Kontenverwaltung, Transaktionen und Konfiguration                                                                                   |
| `client.rpc`                | Kit-kompatibles RPC-Interface (Teilmenge: `getAccountInfo`, `getBalance`, `getEpochSchedule`, `getLatestBlockhash`, `getMinimumBalanceForRentExemption`, `getMultipleAccounts`, `getSlot`, `requestAirdrop`) |
| `client.sendTransaction(s)` | Anweisungen planen, signieren und als Transaktion senden                                                                                                                                                     |
| `client.airdrop`            | SOL vom LiteSVM-Faucet anfordern                                                                                                                                                                             |
| `client.getMinimumBalance`  | Mindestanzahl an lamport für rent-Befreiung berechnen                                                                                                                                                        |

### client.svm verwenden

Direktzugriff auf LiteSVM für:

- Kontenverwaltung (`setAccount`, `getAccount`, `airdrop`)
- Transaktionsausführung (`sendTransaction`, `simulateTransaction`)
- Konfiguration (`withSigverify`, `withBlockhashCheck`, usw.)
- Zeitmanipulation (`warpToSlot`, `getClock`)

```typescript
import { address, lamports } from "@solana/kit";

// Fund an account
client.svm.airdrop(address, lamports(1_000_000_000n));

// Set account state directly
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([1, 2, 3]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 3n
});

// Get the latest blockhash
const blockhash = client.svm.latestBlockhash();
```

### client.rpc verwenden

Kit-kompatibles RPC-Interface für Standardoperationen:

```typescript
// Get account info (returns base64-encoded data)
const { value: account } = await client.rpc.getAccountInfo(address).send();

// Get multiple accounts in one call
const { value: accounts } = await client.rpc
  .getMultipleAccounts([addr1, addr2, addr3])
  .send();

// Get latest blockhash
const { value: blockhash } = await client.rpc.getLatestBlockhash().send();
```

<Callout type="warn">
  Die RPC-Schicht unterstützt nur **Base64-Kodierung**. Anfragen mit anderen
  Kodierungen führen zu einem Fehler.
</Callout>

## Konfiguration

LiteSVM mithilfe von Builder-Methoden auf `client.svm` konfigurieren:

```typescript
const client = createClient().use(signer(mySigner)).use(litesvm());

// Configure for testing
client.svm
  .withSigverify(false) // Skip signature verification
  .withBlockhashCheck(false) // Skip blockhash validation
  .withSysvars() // Enable Clock, Rent, etc.
  .withBuiltins() // Enable ed25519, secp256k1
  .withTransactionHistory(100n); // Store transaction history
```

## Transaktionen senden

Programm-Plugins nach dem Transport einbinden — sie hängen Anweisungen-Builder
an den Client, sodass Sie nie direkt mit einem rohen `TransactionMessage`
arbeiten müssen:

```typescript
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { litesvm } from "@solana/kit-plugin-litesvm";
import { signer } from "@solana/kit-plugin-signer";
import { systemProgram } from "@solana-program/system";

const mySigner = await generateKeyPairSigner();
const client = createClient()
  .use(signer(mySigner))
  .use(litesvm())
  .use(systemProgram());

client.svm.airdrop(client.payer.address, lamports(10_000_000_000n));

const recipient = await generateKeyPairSigner();

// Build and send in one call
await client.system.instructions
  .transferSol({
    source: client.payer,
    destination: recipient.address,
    amount: lamports(1_000_000_000n)
  })
  .sendTransaction();
```

## SVM → RPC-Ablauf

Sie können den Zustand über SVM setzen und ihn über RPC auslesen:

```typescript
import { address, lamports } from "@solana/kit";

// Set account directly via SVM
client.svm.setAccount({
  address: myAddress,
  data: new Uint8Array([0xde, 0xad, 0xbe, 0xef]),
  executable: false,
  lamports: lamports(1_000_000n),
  programAddress: address("11111111111111111111111111111111"),
  space: 4n
});

// Read via RPC (returns base64-encoded data)
const { value } = await client.rpc.getAccountInfo(myAddress).send();
console.log("Data:", value?.data); // ['3q2+7w==', 'base64']
```
