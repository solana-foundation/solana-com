Starten Sie mit Solana Attestations in nur wenigen Schritten. Dieser Leitfaden
führt Sie durch die Kernkonzepte und hilft Ihnen, Ihre erste Attestierung zu
erstellen.

## Was Sie lernen werden

- Installieren und Einrichten der Attestations-Bibliothek
- Erstellen Ihrer ersten Credential (Attestierungsinstanz)
- Definieren eines Schemas für Ihre Attestierungen
- Ausstellen Ihrer ersten Attestierung
- Abrufen und Verifizieren von Attestierungen

## Voraussetzungen

- Grundkenntnisse in der Solana-Entwicklung
- Node.js 16+ oder Rust-Entwicklungsumgebung
- Eine Solana-Wallet mit etwas SOL für Transaktionsgebühren

## Schnellstart-Schritte

### 1. Installation

Installieren Sie die erforderlichen Pakete und richten Sie Ihre
Entwicklungsumgebung ein.

**[→ Installationsleitfaden](/docs/tools/attestations/quickstart/installation)**

### 2. Credential erstellen

Richten Sie eine Attestierungsinstanz ein, die Attestierungen ausstellen kann.

**[→ Leitfaden zum Erstellen von Credentials](/docs/tools/attestations/instructions/create-credential)**

### 3. Schema definieren

Erstellen Sie ein Schema, das die Struktur Ihrer Attestierungen definiert.

**[→ Leitfaden zum Erstellen von Schemas](/docs/tools/attestations/instructions/create-schema)**

### 4. Attestierung ausstellen

Erstellen Sie Ihren ersten verifizierbaren Nachweis unter Verwendung der
Credential und des Schemas.

**[→ Leitfaden zum Erstellen von Attestierungen](/docs/tools/attestations/instructions/create-attestation)**

## Kernkonzepte

### Credentials

Eine Credential stellt eine Attestierungsinstanz dar. Sie definiert, wer
Attestierungen ausstellen kann, und enthält eine Liste autorisierter Signer.

### Schemas

Schemas definieren die Struktur und Validierungsregeln für Attestierungen. Sie
legen fest, welche Felder eine Attestierung enthält und welche Datentypen diese
haben.

### Attestierungen

Attestierungen sind verifizierbare Aussagen oder Erklärungen, die von
autorisierten Signern im Rahmen einer Credential gemäß einer spezifischen
Schemastruktur erstellt werden.

## Anwendungsfälle

Attestierungen können verwendet werden für:

- **Identitätsverifizierung**: Überprüfung von Benutzeridentitäten und
  Compliance
- **Reputationssysteme**: Aufbau von Vertrauenswerten und Reputationsmetriken
- **Zertifizierungen**: Ausstellung beruflicher Zertifikate und Qualifikationen
- **Compliance**: Sicherstellung der Einhaltung regulatorischer Vorschriften für
  Finanzdienstleistungen
- **Lieferkette**: Nachverfolgung von Authentizität und Herkunft von Waren
