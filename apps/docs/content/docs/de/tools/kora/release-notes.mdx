## Kora v2.2.0-beta.6

_23.03.2026_ |
[GitHub-Release](https://github.com/solana-foundation/kora/releases/tag/v2.2.0-beta.6)
| `cargo install kora-cli@2.2.0-beta.6`

Vorabversion mit Jito-Bundle-Unterstützung, neuen RPC-Methoden und erweiterten
Sicherheitsfunktionen.

- Jito-Bundle-Unterstützung mit den Methoden `signBundle`, `signAndSendBundle`
  und `estimateBundleFee` hinzugefügt
- RPC-Methode `getVersion` hinzugefügt
- Lighthouse-Gebührenzahler-Schutz hinzugefügt
- reCAPTCHA v3 Bot-Schutz für Signatur-Endpunkte hinzugefügt
- Nutzungslimits pro Wallet mit Redis-Unterstützung hinzugefügt
- Offizielle Docker-Images auf GHCR hinzugefügt
- Granulare Regeln für Nutzungslimits hinzugefügt
- SDK: `KoraClient` nimmt jetzt ein Optionsobjekt entgegen; neuer `koraPlugin`
  für `@solana/kit`

Siehe [Beta-Dokumentation](/docs/tools/kora/beta) für Details.

## Kora TypeScript SDK v0.2.0

_09.03.2026_ |
[GitHub-Release](https://github.com/solana-foundation/kora/releases/tag/ts-sdk-v0.2.0)
| `pnpm add @solana/kora@0.2.0`

Kit-Client-Plugin-Architektur für komponierbare gaslose Transaktionen.

- `createKitKoraClient()` mit Kit-Plugin-Kompositionsfähigkeit (Planer-,
  Executor-, Zahler-Plugins) hinzugefügt
- Ermöglicht Kompositionsfähigkeit mit Kit-Programm-Plugins wie `tokenProgram()`
- Token-2022-Unterstützung über die Konfigurationsoption `tokenProgramId`
  hinzugefügt
- Compute-Budget- Anweisungen-Unterstützung mit simulationsbasierter
  CU-Schätzung hinzugefügt
- Fehlendes Feld `signature` zur Antwort `signAndSendTransaction` hinzugefügt
- Gibt `NoopSigner` von `getPaymentInstruction` für Kit v6 Signer-Deduplizierung
  zurück
- `@solana/kit` Peer-Abhängigkeit auf `^6.1.0` erhöht

Siehe [Kit-Client-Leitfaden](/docs/tools/kora/guides/kit-client).

## Kora v2.0.0

_19.11.2025_ |
[GitHub-Release](https://github.com/solana-foundation/kora/releases/tag/v2.0.0)
| `cargo install kora-cli@2.0.0`

Major-Release mit neuer Architektur.

- Neue Architektur für den Kora-Knoten eingeführt
- TypeScript SDK `@solana/kora` v0.1.0 veröffentlicht
- Migration zu Solana SDK v3.0.x und Aktualisierung aller Abhängigkeiten
- Ersetzung des benutzerdefinierten Kora-Signers durch solana-signers-Crate
- `sign_transaction_if_paid`- und `get_all_signers`-Methoden entfernt
- Geänderte Antworttypen: Signaturen aus `signAndSendTransaction` und
  `signTransaction` entfernt
- Verbesserte Entwicklererfahrung, Sicherheit und Leistung
