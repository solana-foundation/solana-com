_Verfügbar auf GitHub:
[`v2.2.0-beta.7`](https://github.com/solana-foundation/kora/releases/tag/v2.2.0-beta.7)_

Kora v2.2.0-beta führt Jito-Bundle-Unterstützung, neue RPC-Methoden,
Docker-Images, erweiterte Sicherheitsfunktionen und SDK-Verbesserungen ein.

> **Dies ist eine Vorabversion.** Die hier dokumentierten Funktionen sind in
> `v2.2.0-beta.7` verfügbar und können sich vor der stabilen Veröffentlichung
> ändern. Für die stabile Dokumentation siehe die
> [Haupt-Kora-Dokumentation](/docs/tools/kora).

## Highlights

### Jito-Bundle-Unterstützung

Senden Sie bis zu 5 Transaktionen atomar über Jitos Block-Engine. Neue
RPC-Methoden:

- [`signBundle`](/docs/tools/kora/beta/bundle-methods#signbundle) — Bundle
  signieren ohne zu senden
- [`signAndSendBundle`](/docs/tools/kora/beta/bundle-methods#signandsendbundle)
  — Signieren und an Jito übermitteln
- [`estimateBundleFee`](/docs/tools/kora/beta/bundle-methods#estimatebundlefee)
  — Gesamte Bundle-Fee schätzen

Siehe den [Jito-Bundle-Leitfaden](/docs/tools/kora/beta/jito-guide) für eine
vollständige Anleitung.

### Docker-Images

Offizielle Multiplattform-Images (amd64/arm64), veröffentlicht in der GitHub
Container Registry. Siehe [Installation](/docs/tools/kora/beta/installation).

### Transaktions-Plugins

Ein neues Plugin-System ermöglicht es Kora, Transaktionsstrukturen während der
Signiervorgänge zu validieren. Das erste integrierte Plugin, `gas_swap`,
erzwingt strikte Regeln für gaslose Token-gegen-SOL-Swap-Transaktionen. Plugins
werden in `kora.toml` konfiguriert und laufen automatisch während
`signTransaction`, `signAndSendTransaction`, `signBundle` und
`signAndSendBundle`.

Siehe
[Konfiguration — Transaktions-Plugins](/docs/tools/kora/beta/configuration#transaction-plugins).

### Neue Konfigurationsoptionen

- **Transaktions-Plugins** zur Validierung von Transaktionsstrukturen
  (`gas_swap`)
- **reCAPTCHA v3** Bot-Schutz für Signierungsendpunkte
- **Lighthouse** Schutz vor Fee-Payer-Drainage
- **Nutzungsbeschränkungen** pro Wallet-Transaktionslimits
- **Bundle-Konfiguration** für Jito-Integration

Siehe [Konfiguration](/docs/tools/kora/beta/configuration).

### SDK-Änderungen

- Unterstützung für Bundle-Methoden (`signBundle`, `signAndSendBundle`,
  `estimateBundleFee`)

Siehe [SDK-Änderungen](/docs/tools/kora/beta/sdk-changes).

### Zusätzliche RPC-Methoden

- [`getVersion`](/docs/tools/kora/beta/bundle-methods#getversion) —
  Server-Versionsinformationen abrufen

## Installation

```bash
cargo install kora-cli@2.2.0-beta.7
```

Oder über Docker:

```bash
docker pull ghcr.io/solana-foundation/kora:v2.2.0-beta.7
```
