title: SVM Signer
description: Signer für Solana und SVM-kompatible Blockchains
---

Diese Signer sind verfügbar, wenn das SVM-Addon für die Transaktionssignierung
verwendet wird.

## secret_key

Der `svm::secret_key` Signer kann verwendet werden, um eine Transaktion synchron
mit einem geheimen Schlüssel, einer Mnemonic oder einer keypair-Datei zu
signieren.

### Eingaben

| Name              | Erforderlich | Typ    | Beschreibung                                                             |
| ----------------- | ------------ | ------ | ------------------------------------------------------------------------ |
| `secret_key`      | optional     | string | Der geheime Schlüssel zum Signieren von Nachrichten und Transaktionen    |
| `mnemonic`        | optional     | string | Die Mnemonic-Phrase zur Generierung des geheimen Schlüssels              |
| `derivation_path` | optional     | string | Der Ableitungspfad zur Generierung des geheimen Schlüssels               |
| `keypair_json`    | optional     | string | Ein Pfad zu einer keypair.json-Datei, die den geheimen Schlüssel enthält |

### Ausgaben

| Name         | Typ    | Beschreibung                           |
| ------------ | ------ | -------------------------------------- |
| `public_key` | string | Der öffentliche Schlüssel des Konten   |
| `address`    | string | Die SVM-Adresse (Alias für public_key) |

```hcl
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

Sie können auch eine Mnemonic verwenden:

```hcl
signer "deployer" "svm::secret_key" {
    mnemonic = input.mnemonic
    derivation_path = "m/44'/501'/0'/0'"
}
```

Oder eine keypair-Datei:

```hcl
signer "deployer" "svm::secret_key" {
    keypair_json = "~/.config/solana/id.json"
}
```

## web_wallet

Der `svm::web_wallet` Signer ermöglicht es einem Runbook-Operator, die
Transaktion mit dem Browser-Signer seiner Wahl zu signieren.

### Eingaben

| Name               | Erforderlich | Typ    | Beschreibung                                                                      |
| ------------------ | ------------ | ------ | --------------------------------------------------------------------------------- |
| `expected_address` | optional     | string | Die erwartete SVM-Adresse für die Verbindung (weglassen, um beliebige zuzulassen) |

### Ausgaben

| Name         | Typ    | Beschreibung                                     |
| ------------ | ------ | ------------------------------------------------ |
| `address`    | string | Die Adresse des verbundenen Konten               |
| `public_key` | string | Der öffentliche Schlüssel des verbundenen Konten |

```hcl
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}
```

Dieser Signer fordert den Benutzer in der Web-Benutzeroberfläche auf, Folgendes
zu tun:

1. Wallet verbinden
2. Einen öffentlichen Schlüssel über eine Nachrichtensignatur bereitstellen
3. Alle Transaktionen signieren, die diesen Signer erfordern

## squads

Der Signer `svm::squads` kann verwendet werden, um Transaktionen mit einem
Squads-Multisig-Tresor zu signieren.

### Eingaben

| Name      | Erforderlich | Typ    | Beschreibung                  |
| --------- | ------------ | ------ | ----------------------------- |
| `address` | optional     | string | Die Adresse des Squad-Tresors |

### Ausgaben

| Name         | Typ    | Beschreibung                                |
| ------------ | ------ | ------------------------------------------- |
| `public_key` | string | Der öffentliche Schlüssel des Squad-Tresors |
| `address`    | string | Die SVM-Adresse (Alias für public_key)      |

```hcl
signer "deployer" "svm::squads" {
    address = input.address
}
```

Die Verwendung von Squads-Multisig ermöglicht eine sichere
Transaktionssignierung durch mehrere Parteien, ohne einzelne private Schlüssel
preiszugeben.
