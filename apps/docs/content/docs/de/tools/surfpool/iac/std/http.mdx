## send_http_request

`std::send_http_request` sendet eine HTTP-Anfrage an die angegebene URL und gibt
die Antwort zurück.

### Eingaben

| Name             | Erforderlich | Typ     | Beschreibung                                                              |
| ---------------- | ------------ | ------- | ------------------------------------------------------------------------- |
| `url`            | erforderlich | string  | Die URL für die Anfrage. Unterstützte Schemas: http und https             |
| `body`           | optional     | string  | Der Anfrage-Body als Zeichenkette oder JSON-Objekt                        |
| `method`         | optional     | string  | Die HTTP-Methode für die Anfrage. Erlaubte Methoden: GET, HEAD, POST      |
| `timeout_ms`     | optional     | integer | Das Anfrage-Timeout in Millisekunden                                      |
| `headers`        | optional     | object  | Eine Zuordnung von Anfrage-Header-Feldnamen und -Werten                   |
| `pre_condition`  | optional     | map     | Vorbedingungen, die vor der Ausführung eines Befehls ausgewertet werden   |
| `post_condition` | optional     | map     | Nachbedingungen, die nach der Ausführung eines Befehls ausgewertet werden |

### Vorbedingungen

Vorbedingungen sind Assertions, die vor der Ausführung eines Befehls ausgewertet
werden. Sie können verwendet werden, um zu bestimmen, ob der Befehl ausgeführt
werden soll oder ob ein bestimmtes Verhalten basierend auf dem Ergebnis der
Assertion ausgeführt werden soll.

| Eigenschaft | Beschreibung                                                                                                            |
| ----------- | ----------------------------------------------------------------------------------------------------------------------- |
| `behavior`  | Das Verhalten, wenn die Vorbedingungs-Assertion nicht erfüllt wird. Optionen: `halt` (Standard), `log`, `skip`          |
| `assertion` | Die zu prüfende Assertion. Sollte einen booleschen Wert ergeben oder `std::assert_eq` und ähnliche Funktionen verwenden |

### Nachbedingungen

Nachbedingungen sind Aussagen, die nach der Ausführung eines Befehls ausgewertet
werden. Sie können verwendet werden, um zu bestimmen, ob der Befehl erneut
ausgeführt werden soll.

| Eigenschaft  | Beschreibung                                                                                                     |
| ------------ | ---------------------------------------------------------------------------------------------------------------- |
| `retries`    | Anzahl der Wiederholungen des Befehls, wenn die Aussage fehlschlägt. Standard: 0                                 |
| `backoff_ms` | Millisekunden, die vor der erneuten Ausführung gewartet werden sollen. Standard: 1000                            |
| `behavior`   | Das Verhalten, wenn die Nachbedingung nicht erfüllt wird. Optionen: `halt` (Standard), `log`, `skip`, `continue` |
| `assertion`  | Die zu prüfende Aussage                                                                                          |

### Ausgaben

Wenn die Aktion `send_http_request` erfolgreich ausgeführt wurde, werden der
Aktion die folgenden Ausgaben hinzugefügt:

| Name            | Typ     | Beschreibung                                     |
| --------------- | ------- | ------------------------------------------------ |
| `response_body` | string  | Der zurückgegebene Antwort-Body als Zeichenkette |
| `status_code`   | integer | Der HTTP-Antwortstatuscode                       |

### Beispiel

```hcl
action "example" "std::send_http_request" {
  url = "https://example.com"
}

output "status" {
  value = action.example.status_code
}
// > status: 200
```

### POST-Anfrage-Beispiel

```hcl
action "create_resource" "std::send_http_request" {
    description = "Create a new resource"
    url = "https://api.example.com/resources"
    method = "POST"
    headers = {
        "Content-Type" = "application/json"
    }
    body = {
        "name" = variable.name,
        "value" = variable.value
    }
}
```

### Anfrage mit Authentifizierung

```hcl
action "api_call" "std::send_http_request" {
    description = "Fetch data from API"
    url = "https://api.example.com/data"
    method = "GET"
    headers = {
        "Authorization" = "Bearer ${variable.token}"
    }
}

output "response" {
    value = action.api_call.response_body
}

output "status" {
    value = action.api_call.status_code
}
```
