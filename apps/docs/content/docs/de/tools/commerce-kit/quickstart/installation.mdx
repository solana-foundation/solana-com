---

Wählen Sie die Commerce Kit-Pakete aus, die Sie benötigen, je nachdem, was Sie
entwickeln. Jedes Paket kann unabhängig installiert und einzeln oder in
Kombination mit anderen verwendet werden.

## Systemanforderungen

- **Node.js**: Version 18 oder höher (LTS empfohlen)
- **Paketmanager**: pnpm, npm oder yarn
- **TypeScript**: Version 4.5+ (empfohlen, aber nicht erforderlich)

## Gängige Peer-Abhängigkeiten

Solana Commerce Kit basiert auf der Solana Kit-Bibliothek und ist daher
kompatibel mit Solana Kit, Gill, Solana Programs
[Clients](https://github.com/solana-program) und
[Codama-generierten JS-Clients](https://github.com/codama-idl/renderers-js).

```bash
pnpm add gill # or @solana/kit
pnpm add @solana-program/token # or @solana-program/target-program
pnpm add @my-program/codama-js-client
```

## TypeScript-Konfiguration

Wenn Sie TypeScript verwenden, stellen Sie sicher, dass Ihre `tsconfig.json`
Folgendes enthält:

```json
{
  "compilerOptions": {
    "module": "ESNext",
    "target": "ES2020",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "jsx": "react-jsx",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  }
}
```

## Next.js-Konfiguration

Wenn Sie Next.js verwenden, aktualisieren Sie `next.config.js`, um Commerce
Kit-Pakete zu transpilieren:

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  transpilePackages: [
    "@solana-commerce/react",
    "@solana-commerce/connector",
    "@solana-commerce/headless",
    "@solana-commerce/solana-pay",
    "@solana-commerce/sdk"
  ]
};

export default nextConfig;
```
