Im vorherigen Abschnitt haben Sie gelernt, wie Sie
[Daten aus dem Solana-Netzwerk lesen](/docs/intro/quick-start/reading-from-network)
indem Sie Konten abrufen. Das Schreiben von Daten in das Solana-Netzwerk
erfordert eine [Transaktion](/docs/core/transactions). Eine Transaktion enthält
eine oder mehrere [ Anweisungen](/docs/core/instructions), und jede Anweisung
ruft ein Programm auf.

Programme definieren die Geschäftslogik für jede Anweisung. Wenn Sie eine
transaktion senden, führt die Solana-Laufzeitumgebung die Anweisungen der
Transaktion der Reihe nach aus. Transaktionen sind atomar. Entweder gelingt jede
Anweisung in der Transaktion, oder die gesamte Transaktion schlägt fehl.

Die Beispiele in diesem Abschnitt zeigen, wie man:

1. SOL zwischen Konten überträgt
2. Einen neuen Token-Mint erstellt

## SOL übertragen

Das folgende Beispiel überträgt SOL von einem Konto auf ein anderes. Nur das
Programm, das als Eigentümer eines Kontos festgelegt ist, kann die Daten des
Kontos ändern oder lamports von dessen Guthaben abziehen. Wallet-Konten gehören
dem [System Program](/docs/core/programs/builtin-programs#the-system-program),
daher erfordert die Übertragung von SOL zwischen Wallet-Konten eine Anweisung,
die das System Program aufruft
[transfer](https://github.com/anza-xyz/agave/blob/v2.1.11/programs/system/src/system_processor.rs#L183-L213)
Anweisung. Das Quellkonto muss die Transaktion ebenfalls signieren.

<WithNotes>

<CodeTabs flags="r">

```ts !! title="Transfer SOL"
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { solanaRpc, rpcAirdrop } from "@solana/kit-plugin-rpc";
import { generatedPayer, airdropPayer } from "@solana/kit-plugin-signer";
import { getTransferSolInstruction } from "@solana-program/system";

// !tooltip[/createClient/] client
const client = await createClient()
  .use(generatedPayer())
  .use(
    solanaRpc({
      rpcUrl: "http://localhost:8899",
      rpcSubscriptionsUrl: "ws://localhost:8900"
    })
  )
  .use(rpcAirdrop())
  .use(airdropPayer(lamports(1_000_000_000n)));

// !tooltip[/receiver/] receiver
const receiver = await generateKeyPairSigner();

// !tooltip[/transferInstruction/] transferInstruction
const transferInstruction = getTransferSolInstruction({
  source: client.payer,
  destination: receiver.address,
  amount: lamports(10_000_000n)
});

// !tooltip[/sendTransaction/] sendTransaction
const result = await client.sendTransaction([transferInstruction]);

console.log("Transaction Signature:", result.context.signature);

// !tooltip[/balances/] balances
const { value: senderBalance } = await client.rpc
  .getBalance(client.payer.address)
  .send();
const { value: receiverBalance } = await client.rpc
  .getBalance(receiver.address)
  .send();

console.log("Sender Balance:", senderBalance);
console.log("Receiver Balance:", receiverBalance);
```

</CodeTabs>

### !client

Erstellen Sie einen Kit-Client für den lokalen Validatoren. `generatedPayer()`
fügt einen Zahler- Signer hinzu, `solanaRpc()` fügt RPC- und
Transaktionssendefähigkeiten hinzu, `rpcAirdrop()` fügt `client.airdrop` hinzu,
und `airdropPayer()` versorgt den Zahler mit Test-SOL.

### !receiver

Generieren Sie einen Signer für das Konto, das SOL empfängt. Der Empfänger muss
diese Übertragung nicht signieren. Wenn die Empfängeradresse noch kein Konto
besitzt, erstellt das Empfangen von SOL ein Systemkonto an dieser Adresse.

### !transferInstruction

Erstelle die System Program- Anweisung, die lamports vom Zahler des Clients an
den Empfänger überträgt.

### !sendTransaction

Sende die Anweisung als Transaktion. Der Kit-Client ruft einen aktuellen
Blockhash ab, legt den Gebührenzahler fest, signiert mit den erforderlichen
Signern, sendet die Transaktion und wartet auf die Bestätigung.

### !balances

Rufe die Guthaben von Sender und Empfänger ab, nachdem die Transaktion bestätigt
wurde.

</WithNotes>

## Ein Token erstellen

Das folgende Beispiel erstellt eine neue Token-Prägung mithilfe des
[Token Extensions Program](/docs/tokens/extensions). Ein mint account ist das
Konten, das die globalen Einstellungen eines Tokens definiert, wie z. B.
Dezimalstellen, Angebot, Prägeberechtigung und Einfrierungsberechtigung.

Das Erstellen eines mint account erfordert zwei Anweisungen:

1. Rufen Sie das System Program auf, um ein neues Konten zu erstellen, das dem
   Token Extensions Program gehört.
2. Rufen Sie das Token Extensions Program auf, um dieses Konten als Prägung zu
   initialisieren.

<WithNotes>

<CodeTabs flags="r">

```ts !! title="Create mint account"
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { solanaRpc, rpcAirdrop } from "@solana/kit-plugin-rpc";
import { generatedPayer, airdropPayer } from "@solana/kit-plugin-signer";
import { getCreateAccountInstruction } from "@solana-program/system";
import {
  fetchMint,
  getInitializeMintInstruction,
  getMintSize,
  TOKEN_2022_PROGRAM_ADDRESS
} from "@solana-program/token-2022";

// !tooltip[/createClient/] client
const client = await createClient()
  .use(generatedPayer())
  .use(
    solanaRpc({
      rpcUrl: "http://localhost:8899",
      rpcSubscriptionsUrl: "ws://localhost:8900"
    })
  )
  .use(rpcAirdrop())
  .use(airdropPayer(lamports(1_000_000_000n)));

// !tooltip[/mint/] mint
const mint = await generateKeyPairSigner();

// !tooltip[/space/] space
const space = BigInt(getMintSize());

// !tooltip[/rent/] rent
const rent = await client.rpc.getMinimumBalanceForRentExemption(space).send();

// !tooltip[/createAccountInstruction/] createAccountInstruction
const createAccountInstruction = getCreateAccountInstruction({
  payer: client.payer,
  newAccount: mint,
  space,
  lamports: rent,
  programAddress: TOKEN_2022_PROGRAM_ADDRESS
});

// !tooltip[/initializeMintInstruction/] initializeMintInstruction
const initializeMintInstruction = getInitializeMintInstruction({
  mint: mint.address,
  decimals: 2,
  mintAuthority: client.payer.address,
  freezeAuthority: client.payer.address,
  tokenProgram: TOKEN_2022_PROGRAM_ADDRESS
});

// !tooltip[/sendTransaction/] sendTransaction
const result = await client.sendTransaction([
  createAccountInstruction,
  initializeMintInstruction
]);

console.log("Mint Address:", mint.address);
console.log("Transaction Signature:", result.context.signature);

// !tooltip[/mintAccount/] mintAccount
const mintAccount = await fetchMint(client.rpc, mint.address);
console.log("Mint Account:", mintAccount);
```

</CodeTabs>

### !client

Erstellen und finanzieren Sie einen Kit-Client für den lokalen validator. Der
finanzierte Zahler übernimmt die Konten-rent und Transaktions-Fee.

### !mint

Generieren Sie einen Signer als Adresse des neuen mint account. Der Signer
autorisiert die Erstellung dieses Konten.

### !space

Berechnen Sie die Anzahl der Bytes, die für ein mint account ohne Erweiterungen
erforderlich sind.

### !rent

Berechnen Sie die lamport, die erforderlich sind, um das mint account rent-frei
zu machen.

### !createAccountInstruction

Erstellen Sie die System Program- Anweisungen, die das mint account anlegen und
das Konten dem Token Extensions Program zuweisen.

### !initializeMintInstruction

Erstellen Sie die Token Extensions Program- Anweisungen, die die anfänglichen
Prägungsdaten in das neue Konten schreiben.

### !sendTransaction

Sende beide Anweisungen als eine Transaktion. Die Reihenfolge der Anweisungen
ist wichtig: Das Konten muss erstellt werden, bevor es initialisiert werden
kann.

### !mintAccount

Rufe das mint account nach der Bestätigung ab. Der Helfer liest das Konten und
deserialisiert das Datenfeld des mint account in den Mint-Typ.

</WithNotes>

```ts !! title="Create mint account"
import { createClient, generateKeyPairSigner, lamports } from "@solana/kit";
import { solanaRpc, rpcAirdrop } from "@solana/kit-plugin-rpc";
import { generatedPayer, airdropPayer } from "@solana/kit-plugin-signer";
import { getCreateAccountInstruction } from "@solana-program/system";
import {
  fetchMint,
  getInitializeMintInstruction,
  getMintSize,
  TOKEN_2022_PROGRAM_ADDRESS
} from "@solana-program/token-2022";

const client = await createClient()
  .use(generatedPayer())
  .use(
    solanaRpc({
      rpcUrl: "http://localhost:8899",
      rpcSubscriptionsUrl: "ws://localhost:8900"
    })
  )
  .use(rpcAirdrop())
  .use(airdropPayer(lamports(1_000_000_000n)));

const mint = await generateKeyPairSigner();

const space = BigInt(getMintSize());

const rent = await client.rpc.getMinimumBalanceForRentExemption(space).send();

const createAccountInstruction = getCreateAccountInstruction({
  payer: client.payer,
  newAccount: mint,
  space,
  lamports: rent,
  programAddress: TOKEN_2022_PROGRAM_ADDRESS
});

const initializeMintInstruction = getInitializeMintInstruction({
  mint: mint.address,
  decimals: 2,
  mintAuthority: client.payer.address,
  freezeAuthority: client.payer.address,
  tokenProgram: TOKEN_2022_PROGRAM_ADDRESS
});

const result = await client.sendTransaction([
  createAccountInstruction,
  initializeMintInstruction
]);

console.log("Mint Address:", mint.address);
console.log("Transaction Signature:", result.context.signature);

// !focus(1:2)
const mintAccount = await fetchMint(client.rpc, mint.address);
console.log("Mint Account:", mintAccount);
```

<Callout type="info">
  Diese Beispiele verwenden `generatedPayer()`, um ein temporäres keypair für
  lokale Tests zu erstellen. Produktionsanwendungen sollten niemals rohe private
  Schlüssel im Code speichern — delegieren Sie das Signieren an ein
  Schlüsselverwaltungs-Backend. Siehe [Signieren in der
  Produktion](/docs/core/transactions/signing-in-production).
</Callout>
