Eine Anleitung zum Einrichten Ihrer lokalen Solana-Entwicklungsumgebung.
Erfahren Sie, wie Sie Rust, die Solana CLI und das Anchor Framework unter
Windows (WSL), Linux und Mac installieren. Verwenden Sie diese Anleitung, wenn
Sie jede Abhängigkeit einzeln installieren möchten oder wenn die
[Schnellinstallation](/docs/intro/installation/) aus irgendeinem Grund
fehlschlägt.

### Voraussetzungen

<Accordions>
<Accordion title="Windows">

Um Solana-Programme unter Windows zu entwickeln, **müssen** Sie das
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL) verwenden. Nach der Installation von WSL werden alle anderen
Abhängigkeiten über das Linux-Terminal installiert.

Um WSL zu installieren, führen Sie den folgenden Befehl in Windows PowerShell
aus:

```terminal
$ wsl --install
```

Der Installationsprozess fordert Sie auf, ein Standard-Benutzerkonto zu
erstellen.

![WSL Install](/assets/docs/intro/installation/wsl-install.png)

Standardmäßig installiert WSL Ubuntu. Sie können ein Linux-Terminal öffnen,
indem Sie "Ubuntu" in der Suchleiste suchen.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Wenn Ihr Ubuntu-Terminal wie das Bild unten aussieht, kann ein Problem
auftreten, bei dem `ctrl + v` (Tastenkombination zum Einfügen) im Terminal nicht
funktioniert.

![Ubuntu Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Wenn Sie auf dieses Problem stoßen, öffnen Sie das Windows Terminal, indem Sie
in der Suchleiste nach "Terminal" suchen.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Schließen Sie als Nächstes das Windows Terminal und öffnen Sie erneut ein
Linux-Terminal, indem Sie wieder nach Ubuntu suchen. Das Terminal sollte nun wie
das Bild unten aussehen, wo `ctrl + v` (Tastenkombination zum Einfügen)
funktioniert.

![Ubuntu Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Wenn Sie VS Code verwenden, ermöglicht Ihnen die
[WSL-Erweiterung](https://code.visualstudio.com/docs/remote/wsl-tutorial), WSL
und VS Code zusammen zu verwenden.

![WSL-Einrichtung in VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Sie sehen Folgendes in der VS Code-Statusleiste:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Sobald Sie WSL eingerichtet haben, können Sie entweder:

- Die [Schnellinstallation](/docs/intro/installation) über das Ubuntu
  (Linux)-Terminal durchführen
- [Rust](#install-rust), die [Solana CLI](#install-solana-cli) und die
  [Anchor CLI](#install-anchor-cli) einzeln installieren, wie unten beschrieben

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Wenn Sie Debian oder Debian-basierte Alternativen wie Ubuntu oder Mint verwenden, folgen Sie den nachstehenden Schritten:

1. Aktualisieren Sie die Paketlisten, um sicherzustellen, dass alle Pakete auf
   dem neuesten Stand sind:

```terminal
$ sudo apt-get update
```

2. Installieren Sie die folgenden Abhängigkeiten:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Wenn Sie beim Installieren von `protobuf-compiler` den folgenden Fehler erhalten, stellen Sie
sicher, dass Sie zuerst _shell`sudo apt-get update` ausführen

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Für RHEL-Derivate wie Fedora oder Red Hat Linux folgen Sie den nachstehenden Schritten.

Aktualisieren Sie die Paketlisten, um neuere Versionen der Pakete zu erhalten,
indem Sie den Befehl ausführen.

```terminal
$ sudo dnf upgrade --refresh
```

Installieren Sie anschließend die folgenden Abhängigkeiten:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Rust installieren

Entwickler erstellen Solana-Programme mit der Programmiersprache
[Rust](https://www.rust-lang.org/).

1. Installieren Sie Rust mit [rustup](https://www.rust-lang.org/tools/install),
   indem Sie den folgenden Befehl in Ihrem Terminal eingeben:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Nach einer erfolgreichen Installation sehen Sie die folgende Meldung:

<Accordions>
<Accordion title="Erfolgreiche Rust-Installationsmeldung">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Laden Sie Ihre PATH-Umgebungsvariable neu, um das bin-Verzeichnis von
   [Cargo](https://github.com/rust-lang/cargo) einzuschließen:

```terminal
$ . "$HOME/.cargo/env"
```

3. Überprüfen Sie, ob die Installation erfolgreich war.

```terminal
$ rustc --version
```

Sie werden eine Ausgabe wie die folgende sehen:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Solana CLI installieren

Die Solana CLI bietet alle Tools, die zum Erstellen und Bereitstellen von
Solana-Programmen erforderlich sind.

1. Installieren Sie die Solana CLI Tool Suite mit dem offiziellen
   Installationsbefehl:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Sie können `stable` durch das Release-Tag ersetzen, das der Softwareversion
  Ihres gewünschten Release entspricht (z. B. `v2.0.3`), oder einen der drei
  symbolischen Kanalnamen verwenden: `stable`, `beta` oder `edge`.
</Callout>

2. Fügen Sie eine PATH-Umgebungsvariable hinzu

Eine Erstinstallation der Solana CLI kann Sie auffordern, eine `PATH`
Umgebungsvariable hinzuzufügen. Schließen und öffnen Sie dazu Ihr Terminal
erneut oder führen Sie Folgendes in Ihrer Shell aus:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Aktualisieren Sie Ihre PATH-Umgebungsvariable

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Wenn Sie **Linux** oder **WSL** verwenden, müssen Sie die Solana CLI-Binärdatei
zu Ihrem PATH hinzufügen, damit der Befehl in Ihrem Terminal verfügbar ist.
Befolgen Sie dazu die folgenden Schritte:

**a.** Überprüfen Sie, welche Shell Sie verwenden:

```terminal
$ echo $SHELL
```

- Wenn die Ausgabe `/bash` enthält, verwenden Sie `.bashrc`.
- Wenn die Ausgabe `/zsh` enthält, verwenden Sie `.zshrc`.

**b.** Führen Sie den entsprechenden Befehl basierend auf Ihrer Shell aus.

Für Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Für Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Wenn Sie einen Mac mit `zsh` verwenden, bleibt der standardmäßige
`export PATH`-Befehl, der bei der Installation protokolliert wurde, nicht
bestehen, sobald Sie Ihr Terminal schließen. Fügen Sie stattdessen den PATH zu
Ihrer Shell-Konfigurationsdatei hinzu, indem Sie folgenden Befehl ausführen:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Die obigen Schritte gelten nicht für `bash`-Benutzer, da die Änderungen an der
  PATH-Variable standardmäßig bestehen bleiben.
</Callout>

</Tab>
</Tabs>

4. Starten Sie Ihr Terminal neu oder führen Sie folgenden Befehl aus, um die
   Terminal-Sitzung zu aktualisieren:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Überprüfen Sie, ob die Installation erfolgreich war, indem Sie die Solana
   CLI-Version prüfen:

```terminal
$ solana --version
```

Sie werden eine Ausgabe wie die folgende sehen:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Sie können alle verfügbaren Versionen im
[Agave Github-Repository](https://github.com/anza-xyz/agave/releases) einsehen.
Agave ist der Validatoren-Client von [Anza](https://www.anza.xyz/), früher
bekannt als Solana Labs Validatoren-Client.

</Callout>

5. Aktualisieren Sie die Solana CLI bei Bedarf auf die neueste Version
   _(Optional)_

```terminal
$ agave-install update
```

</Step>

### Anchor CLI installieren

[Anchor](https://www.anchor-lang.com/) ist ein Framework zur Entwicklung von
Solana-Programmen. Das Anchor-Framework nutzt Rust-Makros, um den Prozess des
Schreibens von Solana-Programmen zu vereinfachen.

#### Voraussetzungen

Die Standard-Testdatei (TypeScript) des Anchor-Projekts, die mit dem Befehl
_shell`anchor init`_ erstellt wird, erfordert Node.js und Yarn. (Die
Rust-Testvorlage ist mit _shell`anchor init --test-template rust`_ verfügbar)

<Accordions>
<Accordion title="Node-Installation">

1. Installieren Sie Node mit dem
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm), indem Sie den
   folgenden Befehl ausführen:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Starten Sie Ihr Terminal neu und bestätigen Sie, dass der Befehl `nvm`
   erfolgreich ausgeführt wird:

```terminal
$ command -v nvm
```

3. Verwenden Sie `nvm`, um Node zu installieren:

```terminal
$ nvm install node
```

4. Überprüfen Sie, ob die Installation erfolgreich war, indem Sie die
   Node-Version prüfen:

```terminal
$ node --version
```

Sie werden eine Ausgabe wie die folgende sehen:

```
v23.7.0
```

</Accordion>
<Accordion title="Yarn-Installation">

1. Installieren Sie Yarn:

```terminal
$ npm install --global yarn
```

2. Überprüfen Sie, ob die Installation erfolgreich war, indem Sie die
   Yarn-Version prüfen:

```terminal
$ yarn --version
```

Sie werden die folgende Ausgabe sehen:

```
1.22.1
```

</Accordion>
</Accordions>

#### Installation

Sie können die Anchor-CLI und die Tools auf zwei Arten installieren:

- Anchor Version Manager (AVM) – empfohlene Installationsmethode
- Ohne AVM – direkt von GitHub installieren

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Der Anchor Version Manager (AVM) ermöglicht es Ihnen, verschiedene
Anchor-Versionen auf Ihrem System zu installieren und zu verwalten sowie
Anchor-Versionen in Zukunft einfach zu aktualisieren. Um Anchor mit AVM zu
installieren, folgen Sie den nachstehenden Schritten:

1. Installieren Sie AVM mit folgendem Befehl:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Bestätigen Sie, dass AVM erfolgreich installiert wurde:

```terminal
$ avm --version
```

3. Installieren Sie die Anchor CLI mit AVM:

Um die neueste Version zu installieren:

```terminal
$ avm install latest
$ avm use latest
```

Um eine bestimmte Version zu installieren, geben Sie die Versionsnummer an:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Bei der Installation der Anchor CLI unter Linux oder WSL kann dieser Fehler
auftreten:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Wenn Sie diese Fehlermeldung sehen, befolgen Sie diese Schritte:

1. Installieren Sie die Abhängigkeiten, die im Abschnitt
   [Linux](/docs/intro/installation#install-dependencies) am Anfang dieser Seite
   aufgeführt sind.
2. Versuchen Sie erneut, die Anchor CLI zu installieren.

</Callout>

4. Überprüfen Sie, ob die Installation erfolgreich war, indem Sie die Anchor
   CLI-Version kontrollieren:

```terminal
$ anchor --version
```

Sie werden eine Ausgabe wie die folgende sehen:

```
anchor-cli 0.31.1
```

<Callout type="info">

Vergessen Sie nicht, den Befehl _shell`avm use`_ auszuführen, um die Anchor
CLI-Version zu deklarieren, die auf Ihrem System ausgeführt werden soll.

- Wenn Sie die Version `latest` installiert haben, führen Sie
  _shell`avm use latest`_ aus.
- Wenn Sie die Version `0.30.1` installiert haben, führen Sie
  _shell`avm use 0.30.1`_ aus.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Installieren Sie eine bestimmte Version der Anchor CLI mit folgendem Befehl:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Überprüfen Sie, ob die Installation erfolgreich war, indem Sie die Anchor
   CLI-Version prüfen:

```terminal
$ anchor --version
```

Sie werden eine Ausgabe wie die folgende sehen:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>

### Surfpool CLI installieren

[Surfpool](https://surfpool.run/) ist ein Tool für die lokale Entwicklung und
ein verbesserter Ersatz für `solana-test-validator`. Erfahren Sie mehr über die
Funktionen von Surfpool in der
[Surfpool-Dokumentation](https://docs.surfpool.run/).

1. Installieren Sie Surfpool mit folgendem Befehl:

```terminal
$ curl -sL https://run.surfpool.run/ | bash
```

2. Überprüfen Sie, ob die Installation erfolgreich war, indem Sie die
   Surfpool-CLI-Version prüfen:

```terminal
$ surfpool --version
```

Sie werden eine Ausgabe wie die folgende sehen:

```
surfpool 0.12.0
```
