Dieser Abschnitt bietet einige gängige Befehle und Beispiele, um Ihnen den
Einstieg in die Verwendung der [Anchor CLI](https://www.anchor-lang.com/docs) zu
erleichtern.

<Steps>
<Step>

### Projekt initialisieren

Erstellen Sie ein neues Anchor-Projekt, indem Sie den unten gezeigten Befehl
ausführen. Es wird ein neues Verzeichnis mit dem Projektnamen erstellt und darin
ein neues Anchor-Projekt initialisiert.

```terminal
$ anchor init <project-name>
```

Zum Beispiel erstellt der folgende Befehl ein Projekt namens `my-project`. Das
`my-project`-Verzeichnis enthält ein einfaches Rust-Programm und eine
TypeScript-Testvorlage.

```terminal
$ anchor init my-project
```

Navigieren Sie dann zum Projektverzeichnis:

```terminal
$ cd <project-name>
```

Siehe Anchors
[Projektdateistruktur](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure).

</Step>
<Step>

### Programm erstellen

Um Ihr Projekt zu erstellen, führen Sie folgenden Befehl aus:

```terminal
$ anchor build
```

Das kompilierte Programm finden Sie im `/target/deploy`-Verzeichnis.

<Callout type="warn">

Wenn Sie beim Ausführen von `anchor build` auf folgende Fehler stoßen:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Probieren Sie diese Lösungen:

1. Erzwingen Sie die Installation mit folgendem Befehl:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Wenn die vorherige Lösung nicht funktioniert, löschen Sie den Solana-Cache:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Sie können dies beheben, indem Sie das Versionsfeld der `Cargo.lock`-Datei von 4 auf 3 ändern:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Siehe [dieses Issue](https://github.com/solana-foundation/anchor/issues/3392)
für weitere Informationen.

</Accordion>

</Accordions>

Nachdem Sie die vorherige Lösung angewendet haben, versuchen Sie erneut,
_shell`anchor build`_ auszuführen.

</Callout>

</Step>
<Step>

### Programm bereitstellen

Um Ihr Projekt bereitzustellen, führen Sie den folgenden Befehl aus:

```terminal
$ anchor deploy
```

Dieser Befehl stellt Ihr Programm auf dem `cluster` bereit, der in der
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml)-Datei
angegeben ist.

</Step>
<Step>

### Programm testen

Um Ihr Projekt zu testen, führen Sie den folgenden Befehl aus:

```terminal
$ anchor test
```

Dieser Befehl baut, stellt bereit und führt die Tests für Ihr Projekt aus.

<Callout>
  Bei Verwendung des `localnet`-Clusters startet Anchor automatisch einen
  lokalen Validator, stellt das Programm bereit, führt Tests aus und stoppt dann
  den Validator.
</Callout>

<Callout type="warn">

Einer der folgenden Fehler kann darauf hinweisen, dass Sie Node.js oder Yarn
nicht installiert haben:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
