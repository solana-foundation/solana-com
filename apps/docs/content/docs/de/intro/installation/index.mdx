Eine Schnellinstallation, um Ihre lokale Solana-Entwicklungsumgebung mit einem
einzigen Befehl einzurichten. Installieren Sie Rust, die Solana CLI und das
Anchor Framework unter Windows (WSL), Linux und Mac. (Falls bevorzugt, können
Sie auch
[jede Abhängigkeit einzeln installieren](/docs/intro/installation/dependencies).)

## Voraussetzungen

<Accordions>
<Accordion title="Windows">

Um Solana-Programme unter Windows zu entwickeln, **müssen** Sie das
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL) verwenden. Nach der Installation von WSL werden alle weiteren
Abhängigkeiten über das Linux-Terminal installiert.

Um WSL zu installieren, führen Sie den folgenden Befehl in der Windows
PowerShell aus:

```terminal
$ wsl --install
```

Der Installationsprozess fordert Sie auf, ein Standard-Benutzerkonto zu
erstellen.

![WSL-Installation](/assets/docs/intro/installation/wsl-install.png)

Standardmäßig installiert WSL Ubuntu. Sie können ein Linux-Terminal öffnen,
indem Sie in der Suchleiste nach "Ubuntu" suchen.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Wenn Ihr Ubuntu-Terminal wie das Bild unten aussieht, kann es vorkommen, dass
`ctrl + v` (Tastenkombination zum Einfügen) im Terminal nicht funktioniert.

![Ubuntu-Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Wenn dieses Problem auftritt, öffnen Sie das Windows-Terminal, indem Sie in der
Suchleiste nach "Terminal" suchen.

![Windows-Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Schließen Sie anschließend das Windows-Terminal und öffnen Sie erneut ein
Linux-Terminal, indem Sie wieder nach Ubuntu suchen. Das Terminal sollte nun wie
das Bild unten aussehen, wo `ctrl + v` (Tastenkombination zum Einfügen)
funktioniert.

![Ubuntu-Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Wenn Sie VS Code verwenden, ermöglicht Ihnen die
[WSL-Erweiterung](https://code.visualstudio.com/docs/remote/wsl-tutorial), WSL
und VS Code zusammen zu nutzen.

![WSL-Einrichtung in VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Sie sehen Folgendes in der VS Code-Statusleiste:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Sobald Sie WSL eingerichtet haben, können Sie entweder:

- Die [Schnellinstallation](#installation) über das Ubuntu (Linux)-Terminal
  abschließen
- [Rust](/docs/intro/installation/dependencies#install-rust), die
  [Solana CLI](/docs/intro/installation/dependencies#install-solana-cli) und die
  [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli) einzeln
  installieren, wie unten beschrieben

</Accordion>

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Wenn Sie Debian oder Debian-basierte Alternativen wie Ubuntu oder Mint verwenden, folgen Sie den nachstehenden Schritten:

1. Aktualisieren Sie die Paketlisten, um sicherzustellen, dass alle Pakete auf
   dem neuesten Stand sind:

```terminal
$ sudo apt-get update
```

2. Installieren Sie die folgenden Abhängigkeiten:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Wenn beim Installieren von `protobuf-compiler` der folgende Fehler auftritt, stellen Sie
sicher, dass Sie zuerst _shell`sudo apt-get update` ausführen

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

    <Tab value=".rpm" >
    Für RHEL-Derivate wie Fedora oder Red Hat Linux folgen Sie den nachstehenden Schritten.

Aktualisieren Sie die Paketlisten, um neuere Versionen der Pakete zu erhalten,
indem Sie den Befehl ausführen.

```terminal
$ sudo dnf upgrade --refresh
```

Installieren Sie anschließend die folgenden Abhängigkeiten:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## Installation

1. Geben Sie den folgenden Befehl in Ihr Terminal ein, um alle erforderlichen
   Abhängigkeiten zu installieren:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Eine erfolgreiche Installation gibt eine Ausgabe wie die folgende zurück:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.91.1 (ed61e7d7e 2025-11-07)
Solana CLI: solana-cli 3.0.10 (src:96c3a851; feat:3604001754, client:Agave)
Anchor CLI: anchor-cli 0.32.1
Surfpool CLI: surfpool 0.12.0
Node.js: v24.10.0
Yarn: 1.22.1
```

2. Überprüfen Sie eine erfolgreiche Installation, indem Sie die Version jeder
   installierten Abhängigkeit prüfen.

```terminal
$ rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

<Callout>
  Wenn der Schnellinstallationsbefehl fehlschlägt, lesen Sie bitte den Abschnitt
  [Abhängigkeiten installieren](/docs/intro/installation/dependencies) unten für
  Anweisungen zur individuellen Installation jeder Abhängigkeit.
</Callout>
