Dieser Abschnitt führt durch einige gängige Anchor CLI-Befehle, um Ihnen den
Einstieg zu erleichtern. Für weitere Informationen zur Anchor CLI, siehe die
[Anchor-Dokumentation](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Projekt initialisieren

Um ein neues Anchor-Projekt zu erstellen, führen Sie folgenden Befehl aus:

```terminal
$ anchor init <project-name>
```

Um beispielsweise ein Projekt namens `my-project` zu erstellen, führen Sie aus:

```terminal
$ anchor init my-project
```

Dieser Befehl erstellt ein neues Verzeichnis mit dem Projektnamen und
initialisiert ein neues Anchor-Projekt mit einem grundlegenden Rust-Programm und
einer TypeScript-Test-Vorlage.

Navigieren Sie zum Projektverzeichnis:

```terminal
$ cd <project-name>
```

Sehen Sie sich die
[Dateistruktur](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
des Anchor-Projekts an.

</Step>

<Step>
### Programm bauen

Um Ihr Projekt zu bauen, führen Sie folgenden Befehl aus:

```terminal
$ anchor build
```

Sie finden das kompilierte Programm im Verzeichnis `/target/deploy`.

<Callout type="warn">

Wenn Sie beim Ausführen von `anchor build` folgende Fehler erhalten:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Versuchen Sie diese Lösungen:

1. Erzwingen Sie die Installation mit folgendem Befehl:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Wenn die vorherige Lösung nicht funktioniert, löschen Sie den Solana-Cache:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Sie können dieses Problem beheben, indem Sie das Versionsfeld der `Cargo.lock` Datei von 4 auf 3 ändern:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Weitere Informationen finden Sie in
[diesem Issue](https://github.com/solana-foundation/anchor/issues/3392).

</Accordion>

</Accordions>

Nachdem Sie die vorherige Lösung angewendet haben, versuchen Sie erneut,
_shell`anchor build`_ auszuführen.

</Callout>

</Step>
<Step>

### Programm bereitstellen

Um Ihr Projekt bereitzustellen, führen Sie den folgenden Befehl aus:

```terminal
$ anchor deploy
```

Dieser Befehl stellt Ihr Programm auf dem `cluster` bereit, der in der
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml) Datei
angegeben ist.

</Step>
<Step>

### Programm testen

Um Ihr Projekt zu testen, führen Sie den folgenden Befehl aus:

```terminal
$ anchor test
```

<Callout>

Dieser Befehl erstellt, stellt bereit und führt die Tests für Ihr Projekt aus.

Wenn Sie `localnet` als `cluster` in `Anchor.toml` verwenden, startet Anchor
automatisch einen lokalen Validator, stellt Ihr Programm bereit, führt Tests aus
und stoppt dann den Validator.

</Callout>

<Callout type="warn">

Wenn du _shell`anchor test`_ nach dem Erstellen eines neuen Anchor-Projekts
unter Linux oder WSL ausführst, könntest du auf folgende Fehler stoßen, wenn
Node.js oder Yarn nicht installiert sind:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
