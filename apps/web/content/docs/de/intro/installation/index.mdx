Eine Schnellinstallation zur Einrichtung Ihrer lokalen
Solana-Entwicklungsumgebung mit einem Befehl. Installieren Sie Rust, die Solana
CLI und das Anchor Framework unter Windows (WSL), Linux und Mac. (Falls
gewünscht, können Sie
[jede Abhängigkeit einzeln installieren](/docs/intro/installation/dependencies).)

## Voraussetzungen

<Accordions>
<Accordion title="Windows">

Um Solana-Programme unter Windows zu entwickeln, **müssen** Sie das
[Windows Subsystem for Linux](https://learn.microsoft.com/de-de/windows/wsl/install)
(WSL) verwenden. Nach der Installation von WSL werden alle anderen
Abhängigkeiten über das Linux-Terminal installiert.

Um WSL zu installieren, führen Sie den folgenden Befehl in der Windows
PowerShell aus:

```terminal
$ wsl --install
```

Der Installationsprozess fordert Sie zur Erstellung eines
Standard-Benutzerkontos auf.

![WSL Installation](/assets/docs/intro/installation/wsl-install.png)

Standardmäßig installiert WSL Ubuntu. Sie können ein Linux-Terminal öffnen,
indem Sie "Ubuntu" in der Suchleiste suchen.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Wenn Ihr Ubuntu-Terminal wie das Bild unten aussieht, könnten Sie auf ein
Problem stoßen, bei dem `ctrl + v` (Tastenkombination zum Einfügen) im Terminal
nicht funktioniert.

![Ubuntu Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Wenn Sie auf dieses Problem stoßen, öffnen Sie das Windows Terminal, indem Sie
nach "Terminal" in der Suchleiste suchen.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Schließen Sie als Nächstes das Windows Terminal und öffnen Sie erneut ein
Linux-Terminal, indem Sie wieder nach Ubuntu suchen. Das Terminal sollte jetzt
wie das Bild unten aussehen, wo `ctrl + v` (Tastenkombination zum Einfügen)
funktioniert.

![Ubuntu Terminal](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Wenn Sie VS Code verwenden, ermöglicht die
[WSL-Erweiterung](https://code.visualstudio.com/docs/remote/wsl-tutorial) die
gemeinsame Nutzung von WSL und VS Code.

![WSL-Einrichtung in VS Code](/assets/docs/intro/installation/wsl-vscode.png)

In der VS Code-Statusleiste wird Folgendes angezeigt:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Sobald Sie WSL eingerichtet haben, können Sie entweder:

- Die [Schnellinstallation](#installation) über das Ubuntu (Linux) Terminal
  durchführen
- [Rust](/docs/intro/installation/dependencies#install-rust), die
  [Solana CLI](/docs/intro/installation/dependencies#install-solana-cli) und die
  [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli) einzeln
  installieren, wie unten beschrieben

</Accordion>

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Wenn Sie Debian oder Debian-basierte Alternativen wie Ubuntu oder Mint verwenden, folgen Sie den nachstehenden Schritten:

1. Aktualisieren Sie die Paketlisten, um sicherzustellen, dass alle Pakete auf
   dem neuesten Stand sind:

```terminal
$ sudo apt-get update
```

2. Installieren Sie die folgenden Abhängigkeiten:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Wenn beim Installieren von `protobuf-compiler` der folgende Fehler auftritt, führen
Sie zuerst _shell`sudo apt-get update` aus

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

Aktualisieren Sie die Paketlisten, um die neueren Versionen der Pakete zu

```terminal

&& sudo dnf install pkg-config openssl-devel
```

## Installation

1. Geben Sie den folgenden Befehl in Ihr Terminal ein, um alle notwendigen
   Abhängigkeiten zu installieren:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Eine erfolgreiche Installation gibt eine Ausgabe wie die folgende zurück:

```
Installed Versions:
Rust: rustc 1.86.0 (05f9846f8 2025-03-31)
Solana CLI: solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
Anchor CLI: anchor-cli 0.31.1
Node.js: v23.11.0
Yarn: 1.22.1
```

2. Überprüfen Sie die erfolgreiche Installation, indem Sie die Version jeder
   installierten Abhängigkeit kontrollieren.

```terminal
$ rustc --version && solana --version && anchor --version && node --version && yarn --version
```

<Callout>
  Falls der schnelle Installationsbefehl fehlschlägt, lesen Sie bitte den
  Abschnitt [Abhängigkeiten installieren](/docs/intro/installation/dependencies)
  für Anweisungen zur individuellen Installation jeder Abhängigkeit.
</Callout>

### Nächste Schritte

Sobald Sie alle notwendigen Abhängigkeiten installiert haben, sind Sie bereit,
die [Solana CLI Grundlagen](/docs/intro/installation/solana-cli-basics) und
[Anchor CLI Grundlagen](/docs/intro/installation/anchor-cli-basics) zu
überprüfen.
