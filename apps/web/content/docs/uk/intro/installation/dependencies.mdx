Посібник з налаштування локального середовища розробки Solana. Дізнайтеся, як
встановити Rust, Solana CLI та Anchor Framework на Windows (WSL), Linux та Mac.
Використовуйте цей посібник, якщо ви бажаєте встановити кожну залежність окремо,
або якщо [швидке встановлення](/docs/intro/installation/) з якоїсь причини не
вдалося.

### Передумови

<Accordions>
<Accordion title="Windows">

Для розробки програм Solana на Windows ви **повинні** використовувати
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Після встановлення WSL, усі інші залежності будуть встановлені через
термінал Linux.

Щоб встановити WSL, виконайте наступну команду в Windows PowerShell:

```terminal
$ wsl --install
```

Процес встановлення запропонує вам створити обліковий запис користувача за
замовчуванням.

![Встановлення WSL](/assets/docs/intro/installation/wsl-install.png)

За замовчуванням WSL встановлює Ubuntu. Ви можете відкрити термінал Linux,
шукаючи "Ubuntu" в рядку пошуку.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Якщо ваш термінал Ubuntu виглядає як на зображенні нижче, ви можете зіткнутися з
проблемою, коли `ctrl + v` (комбінація клавіш для вставки) не працює в
терміналі.

![Термінал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Якщо ви зіткнулися з цією проблемою, відкрийте Windows Terminal, шукаючи
"Terminal" в рядку пошуку.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Далі закрийте Windows Terminal і знову відкрийте термінал Linux, шукаючи Ubuntu
знову. Тепер термінал повинен виглядати як на зображенні нижче, де `ctrl + v`
(комбінація клавіш для вставки) працює.

![Термінал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Якщо ви використовуєте VS Code,
[розширення WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
дозволяє вам використовувати WSL та VS Code разом.

![Налаштування WSL у VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Ви побачите наступне в рядку стану VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Після налаштування WSL ви готові до:

- Завершення [швидкого встановлення](/docs/intro/installation) за допомогою
  терміналу Ubuntu (Linux)
- Окремого встановлення [Rust](#install-rust), [Solana CLI](#install-solana-cli)
  та [Anchor CLI](#install-anchor-cli), як описано нижче

</Accordion>

Якщо ви використовуєте Debian або похідні від Debian дистрибутиви, такі як Ubuntu або Mint, виконайте наступні кроки:

1. Оновіть списки пакетів, щоб переконатися, що всі пакети актуальні:

```terminal
$ sudo apt-get update
```

2. Встановіть наступні залежності:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Якщо ви зіткнулися з наступною помилкою під час встановлення `protobuf-compiler`, спочатку
виконайте _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

```terminal
$ sudo dnf upgrade --refresh
```

Далі встановіть наступні залежності:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Встановлення Rust

Розробники створюють програми Solana за допомогою мови програмування
[Rust](https://www.rust-lang.org/).

1. Встановіть Rust за допомогою
   [rustup](https://www.rust-lang.org/tools/install), ввівши наступну команду у
   вашому терміналі:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Після успішного встановлення ви побачите наступне повідомлення:

<Accordions>
<Accordion title="Повідомлення про успішне встановлення Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Перезавантажте змінну середовища PATH, щоб включити каталог bin
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Перевірте, що встановлення пройшло успішно.

```terminal
$ rustc --version
```

Ви побачите вивід, схожий на наступний:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Встановлення Solana CLI

Solana CLI надає всі інструменти, необхідні для створення та розгортання програм
Solana.

1. Встановіть набір інструментів Solana CLI, використовуючи офіційну команду
   встановлення:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Ви можете замінити `stable` на тег релізу, що відповідає версії програмного
  забезпечення вашого бажаного випуску (наприклад, `v2.0.3`), або
  використовувати одну з трьох символічних назв каналів: `stable`, `beta` або
  `edge`.
</Callout>

2. Додайте змінну середовища PATH

При першому встановленні Solana CLI вам може бути запропоновано додати змінну
середовища `PATH`. Щоб це зробити, закрийте та знову відкрийте термінал або
виконайте наступну команду у вашій оболонці:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Оновіть вашу змінну середовища PATH

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Якщо ви використовуєте **Linux** або **WSL**, вам потрібно додати бінарний файл
Solana CLI до вашого PATH, щоб команда була доступна у вашому терміналі. Для
цього виконайте кроки нижче:

**a.** Перевірте, яку оболонку ви використовуєте:

```terminal
$ echo $SHELL
```

- Якщо вивід містить `/bash`, використовуйте `.bashrc`.
- Якщо вивід містить `/zsh`, використовуйте `.zshrc`.

**б.** Запустіть відповідну команду, залежно від вашої оболонки.

Для Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Для Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Якщо ви використовуєте Mac з `zsh`, запуск стандартної команди `export PATH`,
зареєстрованої під час встановлення, не зберігається після закриття терміналу.
Натомість додайте PATH до файлу конфігурації вашої оболонки, виконавши таку
команду:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Вищезазначені кроки не стосуються користувачів `bash`, оскільки зміни змінної
  PATH зберігатимуться за замовчуванням.
</Callout>

</Tab>
</Tabs>

4. Перезапустіть термінал або виконайте таку команду, щоб оновити сеанс
   терміналу:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Перевірте успішність встановлення, перевіривши версію Solana CLI:

```terminal
$ solana --version
```

Ви побачите вивід, схожий на такий:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Ви можете переглянути всі доступні версії в
[репозиторії Agave на Github](https://github.com/anza-xyz/agave/releases). Agave
— це клієнт validator від [Anza](https://www.anza.xyz/), раніше відомий як
клієнт validator від Solana Labs.

</Callout>

5. Оновіть Solana CLI до найновішої версії за потреби _(Необов'язково)_

```terminal
$ agave-install update
```

</Step>

### Встановлення Anchor CLI

[Anchor](https://www.anchor-lang.com/) — це фреймворк для розробки програм
Solana. Фреймворк Anchor використовує макроси Rust для спрощення процесу
написання програм Solana.

#### Передумови

Типовий тестовий файл проєкту Anchor (TypeScript), створений за допомогою
команди _shell`anchor init`_, потребує Node.js та Yarn. (Шаблон тесту Rust
доступний за допомогою _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Встановлення Node">

1. Встановіть Node за допомогою
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm), виконавши таку
   команду:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Перезапустіть термінал і переконайтеся, що команда `nvm` виконується успішно:

```terminal
$ command -v nvm
```

3. Використовуйте `nvm` для встановлення Node:

```terminal
$ nvm install node
```

4. Перевірте, чи встановлення пройшло успішно, перевіривши версію Node:

```terminal
$ node --version
```

Ви побачите такий результат:

```
v23.7.0
```

</Accordion>
<Accordion title="Встановлення Yarn">

1. Встановіть Yarn:

```terminal
$ npm install --global yarn
```

2. Перевірте, чи встановлення пройшло успішно, перевіривши версію Yarn:

```terminal
$ yarn --version
```

Ви побачите такий результат:

```
1.22.1
```

</Accordion>
</Accordions>

#### Встановлення

Ви можете встановити Anchor CLI та інструменти двома способами:

- Anchor Version Manager (AVM) — рекомендований спосіб встановлення
- Без AVM — встановлення безпосередньо з GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Менеджер версій Anchor (AVM) дозволяє встановлювати та керувати різними версіями
Anchor у вашій системі та легко оновлювати версії Anchor у майбутньому. Щоб
встановити Anchor за допомогою AVM, виконайте наступні кроки:

1. Встановіть AVM за допомогою наступної команди:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Підтвердіть, що AVM встановлено успішно:

```terminal
$ avm --version
```

3. Встановіть Anchor CLI за допомогою AVM:

Щоб встановити найновішу версію:

```terminal
$ avm install latest
$ avm use latest
```

Щоб встановити конкретну версію, вкажіть номер версії:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Під час встановлення Anchor CLI на Linux або WSL ви можете зіткнутися з такою
помилкою:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Якщо ви бачите це повідомлення про помилку, виконайте такі кроки:

1. Встановіть залежності, перелічені в розділі
   [Linux](/docs/intro/installation#install-dependencies) на початку цієї
   сторінки.
2. Спробуйте встановити Anchor CLI знову.

</Callout>

4. Перевірте, що встановлення пройшло успішно, перевіривши версію Anchor CLI:

```terminal
$ anchor --version
```

Ви побачите вивід, схожий на такий:

```
anchor-cli 0.31.1
```

<Callout type="info">

Не забудьте виконати команду _shell`avm use`_, щоб вказати версію Anchor CLI,
яка буде використовуватися у вашій системі.

- Якщо ви встановили версію `latest`, виконайте _shell`avm use latest`_.
- Якщо ви встановили версію `0.30.1`, виконайте _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Встановіть конкретну версію Anchor CLI за допомогою наступної команди:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Перевірте успішність встановлення, перевіривши версію Anchor CLI:

```terminal
$ anchor --version
```

Ви побачите вивід, схожий на такий:

```
anchor-cli 0.31.1
```

</Tab>
