Цей розділ надає деякі поширені команди та приклади, щоб допомогти вам почати
використовувати [Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Ініціалізація проєкту

Створіть новий проєкт Anchor, виконавши команду, показану нижче. Вона створить
новий каталог з назвою проєкту та використає його для ініціалізації нового
проєкту Anchor.

```terminal
$ anchor init <project-name>
```

Наприклад, команда нижче створить проєкт під назвою `my-project`. Каталог
`my-project` міститиме базову програму на Rust і шаблон тесту на TypeScript.

```terminal
$ anchor init my-project
```

Потім перейдіть до каталогу проєкту:

```terminal
$ cd <project-name>
```

Дивіться
[структуру файлів проєкту](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
Anchor.

</Step>
<Step>

### Збірка програми

Щоб зібрати ваш проєкт, виконайте таку команду:

```terminal
$ anchor build
```

Ви можете знайти скомпільовану програму в каталозі `/target/deploy`.

<Callout type="warn">

Під час виконання `anchor build`, якщо ви зіткнетеся з такими помилками:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Спробуйте ці рішення:

1. Примусово встановіть, використовуючи таку команду:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Якщо попереднє рішення не працює, очистіть кеш Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Ви можете виправити це, змінивши поле версії файлу `Cargo.lock` з 4 на 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Дивіться [це питання](https://github.com/solana-foundation/anchor/issues/3392)
для додаткової інформації.

</Accordion>

</Accordions>

Після застосування вищезазначеного рішення спробуйте знову запустити
_shell`anchor build`_.

</Callout>

</Step>
<Step>

### Розгортання програми

Щоб розгорнути ваш проєкт, виконайте наступну команду:

```terminal
$ anchor deploy
```

Ця команда розгортає вашу програму в `cluster`, вказаному у файлі
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Тестування програми

Щоб протестувати ваш проєкт, виконайте наступну команду:

```terminal
$ anchor test
```

Ця команда збирає, розгортає та запускає тести для вашого проєкту.

<Callout>
  При використанні кластера `localnet`, Anchor автоматично запускає локальний
  validator, розгортає програму, запускає тести, а потім зупиняє validator.
</Callout>

<Callout type="warn">

Будь-яка з наступних помилок може вказувати на те, що у вас не встановлено
Node.js або Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
