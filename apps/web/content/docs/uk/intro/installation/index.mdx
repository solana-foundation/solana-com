Швидке встановлення для налаштування локального середовища розробки Solana за
допомогою однієї команди. Встановіть Rust, Solana CLI та фреймворк Anchor на
Windows (WSL), Linux та Mac. (За бажанням, ви можете
[встановити кожну залежність окремо](/docs/intro/installation/dependencies).

## Передумови

<Accordions>
<Accordion title="Windows">

Для розробки програм Solana на Windows ви **повинні** використовувати
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Після встановлення WSL, всі інші залежності будуть встановлені через
термінал Linux.

Щоб встановити WSL, виконайте наступну команду в Windows PowerShell:

```terminal
$ wsl --install
```

Процес встановлення запропонує вам створити обліковий запис користувача за
замовчуванням.

![Встановлення WSL](/assets/docs/intro/installation/wsl-install.png)

За замовчуванням WSL встановлює Ubuntu. Ви можете відкрити термінал Linux,
шукаючи "Ubuntu" в рядку пошуку.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Якщо ваш термінал Ubuntu виглядає як на зображенні нижче, ви можете зіткнутися з
проблемою, коли `ctrl + v` (комбінація клавіш для вставки) не працює в
терміналі.

![Термінал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Якщо ви зіткнулися з цією проблемою, відкрийте Windows Terminal, шукаючи
"Terminal" в рядку пошуку.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Далі закрийте Windows Terminal і знову відкрийте термінал Linux, шукаючи Ubuntu
знову. Тепер термінал повинен виглядати як на зображенні нижче, де `ctrl + v`
(комбінація клавіш для вставки) працює.

![Термінал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Якщо ви використовуєте VS Code, розширення
[WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) дозволяє вам
використовувати WSL та VS Code разом.

![Налаштування WSL у VS Code](/assets/docs/intro/installation/wsl-vscode.png)

У рядку стану VS Code ви побачите наступне:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Після налаштування WSL ви готові до:

- Завершення [швидкого встановлення](#installation) за допомогою терміналу
  Ubuntu (Linux)
- Окремого встановлення
  [Rust](/docs/intro/installation/dependencies#install-rust),
  [Solana CLI](/docs/intro/installation/dependencies#install-solana-cli) та
  [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli), як
  описано нижче

Якщо ви використовуєте Debian або похідні від Debian дистрибутиви, такі як Ubuntu або Mint, виконайте наступні кроки:

1. Оновіть списки пакетів, щоб переконатися, що всі пакети актуальні:

```terminal
$ sudo apt-get update
```

2. Встановіть наступні залежності:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

## Встановлення

1. Введіть наступну команду в термінал для встановлення всіх необхідних
   залежностей:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Успішне встановлення поверне вивід, схожий на наступний:

```
Installed Versions:
Rust: rustc 1.86.0 (05f9846f8 2025-03-31)
Solana CLI: solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
Anchor CLI: anchor-cli 0.31.1
Node.js: v23.11.0
Yarn: 1.22.1
```

2. Перевірте успішність встановлення, перевіривши версію кожної встановленої
   залежності.

```terminal
$ rustc --version && solana --version && anchor --version && node --version && yarn --version
```

<Callout>
  Якщо команда швидкого встановлення не виконується, зверніться до розділу
  [Встановлення залежностей](/docs/intro/installation/dependencies) нижче для
  інструкцій щодо встановлення кожної залежності окремо.
</Callout>
