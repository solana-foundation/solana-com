Цей функціональний шлюз зменшує стандартне виділення CU для інструкцій
вбудованих програм з 200 000 до 3 000 CU для покращення продуктивності
валідатора.

## Вплив

Менше ніж 0,00001% транзакцій будуть порушені цією зміною.

**Розробники** завжди повинні використовувати ліміти обчислювального бюджету для
явного виділення CU для своїх транзакцій. Ця зміна не вплине на транзакції, які
явно виділяють CU.
