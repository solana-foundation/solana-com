Un guide pour configurer votre environnement de développement Solana local.
Apprenez à installer Rust, la CLI Solana et le framework Anchor sur Windows
(WSL), Linux et Mac. Utilisez ce guide si vous préférez installer chaque
dépendance individuellement, ou si
l'[installation rapide](/docs/intro/installation/) échoue pour une raison
quelconque.

### Prérequis

<Accordions>
<Accordion title="Windows">

Pour développer des programmes Solana sur Windows, vous **devez** utiliser le
[Sous-système Windows pour Linux](https://learn.microsoft.com/fr-fr/windows/wsl/install)
(WSL). Après l'installation de WSL, toutes les autres dépendances seront
installées via le terminal Linux.

Pour installer WSL, exécutez la commande suivante dans Windows PowerShell :

```terminal
$ wsl --install
```

Le processus d'installation vous invitera à créer un compte utilisateur par
défaut.

![Installation WSL](/assets/docs/intro/installation/wsl-install.png)

Par défaut, WSL installe Ubuntu. Vous pouvez ouvrir un terminal Linux en
recherchant "Ubuntu" dans la barre de recherche.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Si votre terminal Ubuntu ressemble à l'image ci-dessous, vous pourriez
rencontrer un problème où `ctrl + v` (raccourci clavier pour coller) ne
fonctionne pas dans le terminal.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Si vous rencontrez ce problème, ouvrez Windows Terminal en recherchant
"Terminal" dans la barre de recherche.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Ensuite, fermez Windows Terminal et rouvrez un terminal Linux en recherchant
Ubuntu à nouveau. Le terminal devrait maintenant ressembler à l'image
ci-dessous, où `ctrl + v` (raccourci clavier pour coller) fonctionne.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Si vous utilisez VS Code,
l'[extension WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) vous
permet d'utiliser WSL et VS Code ensemble.

![Configuration WSL dans VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Vous verrez ce qui suit dans la barre d'état de VS Code :

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Une fois que vous avez configuré WSL, vous êtes prêt à :

- Effectuer l'[installation rapide](/docs/intro/installation) en utilisant le
  terminal Ubuntu (Linux)
- Installer [Rust](#install-rust), la [CLI Solana](#install-solana-cli) et la
  [CLI Anchor](#install-anchor-cli) individuellement, comme décrit ci-dessous

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Si vous utilisez Debian ou des alternatives dérivées de Debian comme Ubuntu ou Mint, suivez les étapes ci-dessous :

1. Mettez à jour la liste des paquets pour vous assurer que tous les paquets
   sont à jour :

```terminal
$ sudo apt-get update
```

2. Installez les dépendances suivantes :

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Si vous rencontrez l'erreur suivante lors de l'installation de `protobuf-compiler`, assurez-vous
d'exécuter d'abord _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Pour les dérivés RHEL comme Fedora ou Red Hat Linux, suivez les étapes suivantes.

Mettez à jour la liste des paquets pour obtenir les versions plus récentes des
paquets en exécutant la commande.

```terminal
$ sudo dnf upgrade --refresh
```

Ensuite, installez les dépendances suivantes :

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Installer Rust

Les développeurs créent des programmes Solana en utilisant le langage de
programmation [Rust](https://www.rust-lang.org/).

1. Installez Rust en utilisant [rustup](https://www.rust-lang.org/tools/install)
   en saisissant la commande suivante dans votre terminal :

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Après une installation réussie, vous verrez le message suivant :

<Accordions>
<Accordion title="Message d'installation réussie de Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Rechargez votre variable d'environnement PATH pour inclure le répertoire bin
   de [Cargo](https://github.com/rust-lang/cargo) :

```terminal
$ . "$HOME/.cargo/env"
```

3. Vérifiez que l'installation a réussi.

```terminal
$ rustc --version
```

Vous verrez une sortie comme celle-ci :

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Installer Solana CLI

Solana CLI fournit tous les outils nécessaires pour créer et déployer des
programmes Solana.

1. Installez la suite d'outils Solana CLI en utilisant la commande
   d'installation officielle :

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Vous pouvez remplacer `stable` par le tag de version correspondant à la
  version du logiciel de votre version souhaitée (par exemple `v2.0.3`), ou
  utiliser l'un des trois noms de canaux symboliques : `stable`, `beta`, ou
  `edge`.
</Callout>

2. Ajoutez une variable d'environnement PATH

Une première installation de Solana CLI peut vous inviter à ajouter une variable
d'environnement `PATH`. Pour ce faire, fermez et rouvrez votre terminal ou
exécutez la commande suivante dans votre shell :

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Mettez à jour votre variable d'environnement PATH

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Si vous utilisez **Linux** ou **WSL**, vous devez ajouter le binaire Solana CLI
à votre PATH pour que la commande soit disponible dans votre terminal. Pour ce
faire, suivez les étapes ci-dessous :

**a.** Vérifiez quel shell vous utilisez :

```terminal
$ echo $SHELL
```

- Si la sortie contient `/bash`, utilisez `.bashrc`.
- Si la sortie contient `/zsh`, utilisez `.zshrc`.

**b.** Exécutez la commande appropriée, selon votre shell.

Pour Bash (`bashrc`) :

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Pour Zsh (`zshrc`) :

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Si vous utilisez un Mac avec `zsh`, l'exécution de la commande `export PATH` par
défaut enregistrée lors de l'installation ne persiste pas une fois que vous
fermez votre terminal. À la place, ajoutez le PATH à votre fichier de
configuration shell en exécutant la commande suivante :

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Les étapes ci-dessus ne s'appliquent pas aux utilisateurs de `bash`, car les
  modifications de la variable PATH persisteront par défaut.
</Callout>

</Tab>
</Tabs>

4. Redémarrez votre terminal ou exécutez la commande suivante pour actualiser la
   session du terminal :

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Vérifiez que l'installation a réussi en vérifiant la version de la CLI Solana
   :

```terminal
$ solana --version
```

Vous verrez une sortie comme celle-ci :

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Vous pouvez consulter toutes les versions disponibles sur le
[dépôt Github Agave](https://github.com/anza-xyz/agave/releases). Agave est le
client validator d'[Anza](https://www.anza.xyz/), anciennement connu sous le nom
de client validator Solana Labs.

</Callout>

5. Mettez à jour la CLI Solana vers la dernière version, si nécessaire
   _(Facultatif)_

```terminal
$ agave-install update
```

</Step>

### Installer la CLI Anchor

[Anchor](https://www.anchor-lang.com/) est un framework pour développer des
programmes Solana. Le framework Anchor utilise des macros Rust pour simplifier
le processus d'écriture des programmes Solana.

#### Prérequis

Le fichier de test par défaut du projet Anchor (TypeScript) créé avec la
commande _shell`anchor init`_ nécessite Node.js et Yarn. (Le modèle de test Rust
est disponible en utilisant _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Installation de Node">

1. Installez Node en utilisant
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) en exécutant la
   commande suivante :

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Redémarrez votre terminal et confirmez que la commande `nvm` s'exécute avec
   succès :

```terminal
$ command -v nvm
```

3. Utilisez `nvm` pour installer Node :

```terminal
$ nvm install node
```

4. Vérifiez que l'installation a réussi en vérifiant la version de Node :

```terminal
$ node --version
```

Vous verrez une sortie comme celle-ci :

```
v23.7.0
```

</Accordion>
<Accordion title="Installation de Yarn">

1. Installez Yarn :

```terminal
$ npm install --global yarn
```

2. Vérifiez que l'installation a réussi en vérifiant la version de Yarn :

```terminal
$ yarn --version
```

Vous verrez la sortie suivante :

```
1.22.1
```

</Accordion>
</Accordions>

#### Installation

Vous pouvez installer l'interface en ligne de commande (CLI) Anchor et les
outils de deux façons :

- Anchor Version Manager (AVM) — Méthode d'installation recommandée
- Sans AVM — Installation directe depuis GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Le gestionnaire de versions d'Anchor (AVM) vous permet d'installer et de gérer
différentes versions d'Anchor sur votre système et de mettre à jour facilement
les versions d'Anchor à l'avenir. Pour installer Anchor en utilisant AVM, suivez
les étapes ci-dessous :

1. Installez AVM avec la commande suivante :

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Confirmez que l'installation d'AVM a réussi :

```terminal
$ avm --version
```

3. Installez l'interface de ligne de commande (CLI) Anchor en utilisant AVM :

Pour installer la dernière version :

```terminal
$ avm install latest
$ avm use latest
```

Pour installer une version spécifique, précisez le numéro de version :

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Lors de l'installation du CLI Anchor sur Linux ou WSL, vous pourriez rencontrer
cette erreur :

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Si vous voyez ce message d'erreur, suivez ces étapes :

1. Installez les dépendances listées dans la section
   [Linux](/docs/intro/installation#install-dependencies) en haut de cette page.
2. Réessayez d'installer le CLI Anchor.

</Callout>

4. Vérifiez que l'installation a réussi en contrôlant la version du CLI Anchor :

```terminal
$ anchor --version
```

Vous verrez une sortie comme celle-ci :

```
anchor-cli 0.31.1
```

<Callout type="info">

N'oubliez pas d'exécuter la commande _shell`avm use`_ pour déclarer la version
du CLI Anchor à utiliser sur votre système.

- Si vous avez installé la version `latest`, exécutez _shell`avm use latest`_.
- Si vous avez installé la version `0.30.1`, exécutez _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Installez une version spécifique de l'interface de ligne de commande Anchor
   avec la commande suivante :

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Vérifiez que l'installation a réussi en contrôlant la version de l'interface
   de ligne de commande Anchor :

```terminal
$ anchor --version
```

Vous verrez une sortie comme celle-ci :

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>
