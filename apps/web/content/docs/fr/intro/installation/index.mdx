Une installation rapide pour configurer votre environnement de développement
Solana local avec une seule commande. Installez Rust, la CLI Solana et le
framework Anchor sur Windows (WSL), Linux et Mac. (Si vous préférez, vous pouvez
[installer chaque dépendance individuellement](/docs/intro/installation/dependencies).)

## Prérequis

<Accordions>
<Accordion title="Windows">

Pour développer des programmes Solana sur Windows, vous **devez** utiliser
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Après l'installation de WSL, toutes les autres dépendances seront
installées via le terminal Linux.

Pour installer WSL, exécutez la commande suivante dans Windows PowerShell :

```terminal
$ wsl --install
```

Le processus d'installation vous invitera à créer un compte utilisateur par
défaut.

![Installation WSL](/assets/docs/intro/installation/wsl-install.png)

Par défaut, WSL installe Ubuntu. Vous pouvez ouvrir un terminal Linux en
recherchant "Ubuntu" dans la barre de recherche.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Si votre terminal Ubuntu ressemble à l'image ci-dessous, vous pourriez
rencontrer un problème où `ctrl + v` (raccourci clavier pour coller) ne
fonctionne pas dans le terminal.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Si vous rencontrez ce problème, ouvrez Windows Terminal en recherchant
"Terminal" dans la barre de recherche.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Ensuite, fermez Windows Terminal et rouvrez un terminal Linux en recherchant à
nouveau Ubuntu. Le terminal devrait maintenant ressembler à l'image ci-dessous,
où `ctrl + v` (raccourci clavier pour coller) fonctionne.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Si vous utilisez VS Code, l'extension
[WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) vous permet
d'utiliser WSL et VS Code ensemble.

![Configuration WSL dans VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Vous verrez ce qui suit dans la barre d'état de VS Code :

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Une fois que vous avez configuré WSL, vous êtes prêt à :

- Effectuer l'[installation rapide](#installation) en utilisant le terminal
  Ubuntu (Linux)
- Installer [Rust](/docs/intro/installation/dependencies#install-rust), le
  [CLI Solana](/docs/intro/installation/dependencies#install-solana-cli), et le
  [CLI Anchor](/docs/intro/installation/dependencies#install-anchor-cli)
  individuellement, comme décrit ci-dessous

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Si vous utilisez Debian ou des alternatives dérivées de Debian comme Ubuntu ou Mint, suivez les étapes ci-dessous :

1. Mettez à jour la liste des paquets pour vous assurer que tous les paquets
   sont à jour :

```terminal
$ sudo apt-get update
```

2. Installez les dépendances suivantes :

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Si vous rencontrez l'erreur suivante lors de l'installation de `protobuf-compiler`, assurez-vous
d'abord d'exécuter _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Pour les dérivés de RHEL comme Fedora ou Red Hat Linux, suivez les étapes suivantes.

Mettez à jour la liste des paquets pour obtenir les versions plus récentes des
paquets en exécutant la commande.

```terminal
$ sudo dnf upgrade --refresh
```

Ensuite, installez les dépendances suivantes :

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## Installation

1. Saisissez la commande suivante dans votre terminal pour installer toutes les
   dépendances nécessaires :

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Une installation réussie affichera une sortie comme celle-ci :

```
Installed Versions:
Rust: rustc 1.86.0 (05f9846f8 2025-03-31)
Solana CLI: solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
Anchor CLI: anchor-cli 0.31.1
Node.js: v23.11.0
Yarn: 1.22.1
```

2. Vérifiez que l'installation a réussi en contrôlant la version de chaque
   dépendance installée.

```terminal
$ rustc --version && solana --version && anchor --version && node --version && yarn --version
```

<Callout>
  Si la commande d'installation rapide échoue, veuillez consulter la section
  [Installer les dépendances](/docs/intro/installation/dependencies) ci-dessous
  pour obtenir des instructions sur l'installation individuelle de chaque
  dépendance.
</Callout>
