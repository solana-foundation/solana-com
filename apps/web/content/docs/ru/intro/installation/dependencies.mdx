Руководство по настройке локальной среды разработки Solana. Узнайте, как
установить Rust, Solana CLI и Anchor Framework на Windows (WSL), Linux и Mac.
Используйте это руководство, если вы предпочитаете устанавливать каждую
зависимость отдельно или если [быстрая установка](/docs/intro/installation/) не
удалась по какой-либо причине.

### Предварительные требования

<Accordions>
<Accordion title="Windows">

Для разработки программ Solana на Windows вы **должны** использовать
[Подсистему Windows для Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). После установки WSL все остальные зависимости будут установлены через
терминал Linux.

Чтобы установить WSL, выполните следующую команду в Windows PowerShell:

```terminal
$ wsl --install
```

В процессе установки вам будет предложено создать учетную запись пользователя по
умолчанию.

![Установка WSL](/assets/docs/intro/installation/wsl-install.png)

По умолчанию WSL устанавливает Ubuntu. Вы можете открыть терминал Linux, найдя
"Ubuntu" в строке поиска.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Если ваш терминал Ubuntu выглядит как на изображении ниже, вы можете столкнуться
с проблемой, когда `ctrl + v` (сочетание клавиш для вставки) не работает в
терминале.

![Терминал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Если вы столкнулись с этой проблемой, откройте Windows Terminal, найдя
"Terminal" в строке поиска.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Затем закройте Windows Terminal и снова откройте терминал Linux, найдя Ubuntu.
Терминал теперь должен выглядеть как на изображении ниже, где `ctrl + v`
(сочетание клавиш для вставки) работает.

![Терминал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Если вы используете VS Code, то
[расширение WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
позволяет использовать WSL и VS Code вместе.

![Настройка WSL в VS Code](/assets/docs/intro/installation/wsl-vscode.png)

В строке состояния VS Code вы увидите следующее:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

После настройки WSL вы готовы к одному из следующих шагов:

- Завершить [быструю установку](/docs/intro/installation) с использованием
  терминала Ubuntu (Linux)
- Установить [Rust](#install-rust), [Solana CLI](#install-solana-cli) и
  [Anchor CLI](#install-anchor-cli) по отдельности, как описано ниже

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Если вы используете Debian или производные от него дистрибутивы, такие как Ubuntu или Mint, выполните следующие шаги:

1. Обновите списки пакетов, чтобы убедиться, что все пакеты актуальны:

```terminal
$ sudo apt-get update
```

2. Установите следующие зависимости:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Если вы столкнулись с ошибкой при установке `protobuf-compiler`, убедитесь, что вы сначала выполнили _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >

Обновите списки пакетов, чтобы получить более новые версии пакетов, выполнив
команду.

```terminal
$ sudo dnf upgrade --refresh
```

Далее установите следующие зависимости:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Установите Rust

Разработчики создают программы для Solana, используя язык программирования
[Rust](https://www.rust-lang.org/).

1. Установите Rust с помощью [rustup](https://www.rust-lang.org/tools/install),
   введя следующую команду в терминале:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

После успешной установки вы увидите следующее сообщение:

<Accordions>
<Accordion title="Сообщение об успешной установке Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Перезагрузите переменную окружения PATH, чтобы включить каталог bin
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Убедитесь, что установка прошла успешно.

```terminal
$ rustc --version
```

Вы увидите вывод, похожий на следующий:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Установите Solana CLI

Solana CLI предоставляет все инструменты, необходимые для создания и
развертывания программ Solana.

1. Установите набор инструментов Solana CLI, используя официальную команду
   установки:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Вы можете заменить `stable` на тег релиза, соответствующий версии программного
  обеспечения вашего желаемого релиза (например, `v2.0.3`), или использовать
  одно из трёх символических названий каналов: `stable`, `beta`, или `edge`.
</Callout>

2. Добавьте переменную окружения PATH

При первой установке Solana CLI вам может быть предложено добавить переменную
окружения `PATH`. Для этого закройте и снова откройте терминал или выполните
следующую команду в оболочке:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Обновите переменную окружения PATH

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Если вы используете **Linux** или **WSL**, вам необходимо добавить бинарный файл
Solana CLI в PATH, чтобы команда была доступна в вашем терминале. Для этого
выполните следующие шаги:

**a.** Проверьте, какую оболочку вы используете:

```terminal
$ echo $SHELL
```

- Если вывод содержит `/bash`, используйте `.bashrc`.
- Если вывод содержит `/zsh`, используйте `.zshrc`.

**б.** Выполните соответствующую команду в зависимости от используемой оболочки.

Для Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Для Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Если вы используете Mac с `zsh`, выполнение стандартной команды `export PATH`,
записанной при установке, не сохраняется после закрытия терминала. Вместо этого
добавьте PATH в файл конфигурации оболочки, выполнив следующую команду:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Указанные выше шаги не применяются к пользователям `bash`, так как изменения
  переменной PATH сохраняются по умолчанию.
</Callout>

</Tab>
</Tabs>

4. Перезапустите терминал или выполните следующую команду, чтобы обновить сеанс
   терминала:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Убедитесь, что установка прошла успешно, проверив версию Solana CLI:

```terminal
$ solana --version
```

Вы увидите вывод, похожий на следующий:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Вы можете просмотреть все доступные версии в
[репозитории Agave на Github](https://github.com/anza-xyz/agave/releases). Agave
— это клиент валидатора от [Anza](https://www.anza.xyz/), ранее известного как
клиент валидатора Solana Labs.

</Callout>

5. Обновите Solana CLI до последней версии, если необходимо _(опционально)_

```terminal
$ agave-install update
```

</Step>

### Установите Anchor CLI

[Anchor](https://www.anchor-lang.com/) — это фреймворк для разработки программ
на Solana. Фреймворк Anchor использует макросы Rust, чтобы упростить процесс
написания программ для Solana.

#### Предварительные требования

Тестовый файл проекта Anchor по умолчанию (TypeScript), созданный с помощью
команды _shell`anchor init`_, требует наличия Node.js и Yarn. (Шаблон теста на
Rust доступен с использованием _shell`anchor init --test-template rust`_.)

<Accordions>
<Accordion title="Установка Node">

1. Установите Node с помощью
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm), выполнив
   следующую команду:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Перезапустите терминал и убедитесь, что команда `nvm` выполняется успешно:

```terminal
$ command -v nvm
```

3. Используйте `nvm` для установки Node:

```terminal
$ nvm install node
```

4. Убедитесь, что установка прошла успешно, проверив версию Node:

```terminal
$ node --version
```

Вы увидите вывод, похожий на следующий:

```
v23.7.0
```

</Accordion>
<Accordion title="Установка Yarn">

1. Установите Yarn:

```terminal
$ npm install --global yarn
```

2. Убедитесь, что установка прошла успешно, проверив версию Yarn:

```terminal
$ yarn --version
```

Вы увидите следующий вывод:

```
1.22.1
```

</Accordion>
</Accordions>

#### Установка

Вы можете установить Anchor CLI и инструменты двумя способами:

- Anchor Version Manager (AVM) — рекомендуемый способ установки
- Без AVM — установка напрямую из GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Anchor Version Manager (AVM) позволяет устанавливать и управлять различными
версиями Anchor на вашей системе, а также легко обновлять версии Anchor в
будущем. Чтобы установить Anchor с помощью AVM, выполните следующие шаги:

1. Установите AVM с помощью следующей команды:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Убедитесь, что AVM успешно установлен:

```terminal
$ avm --version
```

3. Установите Anchor CLI с помощью AVM:

Чтобы установить последнюю версию:

```terminal
$ avm install latest
$ avm use latest
```

Чтобы установить конкретную версию, укажите номер версии:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

При установке Anchor CLI на Linux или WSL вы можете столкнуться с этой ошибкой:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Если вы видите это сообщение об ошибке, выполните следующие действия:

1. Установите зависимости, перечисленные в разделе
   [Linux](/docs/intro/installation#install-dependencies) в начале этой
   страницы.
2. Повторите попытку установки Anchor CLI.

</Callout>

4. Убедитесь, что установка прошла успешно, проверив версию Anchor CLI:

```terminal
$ anchor --version
```

Вы увидите вывод, похожий на следующий:

```
anchor-cli 0.31.1
```

<Callout type="info">

Не забудьте выполнить команду _shell`avm use`_, чтобы указать версию Anchor CLI,
которая будет использоваться на вашей системе.

- Если вы установили версию `latest`, выполните _shell`avm use latest`_.
- Если вы установили версию `0.30.1`, выполните _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Без AVM">

1. Установите определённую версию Anchor CLI с помощью следующей команды:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Убедитесь, что установка прошла успешно, проверив версию Anchor CLI:

```terminal
$ anchor --version
```

Вы увидите вывод, похожий на следующий:

```
anchor-cli 0.31.1
```

</Tab>
