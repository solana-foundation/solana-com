В этом разделе представлены некоторые распространенные команды и примеры,
которые помогут вам начать использовать
[Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Инициализация проекта

Создайте новый проект Anchor, выполнив команду, показанную ниже. Она создаст
новую директорию с именем проекта и использует её для инициализации нового
проекта Anchor.

```terminal
$ anchor init <project-name>
```

Например, команда ниже создаст проект с именем `my-project`. Директория
`my-project` будет содержать базовую программу на Rust и шаблон теста на
TypeScript.

```terminal
$ anchor init my-project
```

Затем перейдите в директорию проекта:

```terminal
$ cd <project-name>
```

См.
[структуру файлов проекта](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
Anchor.

</Step>
<Step>

### Сборка программы

Чтобы собрать ваш проект, выполните следующую команду:

```terminal
$ anchor build
```

Вы можете найти скомпилированную программу в директории `/target/deploy`.

<Callout type="warn">

При выполнении `anchor build`, если вы столкнулись со следующими ошибками:

<Accordions>
<Accordion title="ошибка: не является директорией">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Попробуйте следующие решения:

1. Принудительно установите, используя следующую команду:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Если предыдущее решение не сработало, очистите кэш Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="версия файла блокировки 4 требует `-Znext-lockfile-bump">
Вы можете исправить это, изменив поле версии файла `Cargo.lock` с 4 на 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

См. [эту задачу](https://github.com/solana-foundation/anchor/issues/3392) для
получения дополнительной информации.

</Accordion>

</Accordions>

После применения предыдущего решения попробуйте снова выполнить
_shell`anchor build`_.

</Callout>

</Step>
<Step>

### Развертывание программы

Чтобы развернуть ваш проект, выполните следующую команду:

```terminal
$ anchor deploy
```

Эта команда развертывает вашу программу в `cluster`, указанном в
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml) файле.

</Step>
<Step>

### Тестирование программы

Чтобы протестировать ваш проект, выполните следующую команду:

```terminal
$ anchor test
```

Эта команда собирает, развертывает и запускает тесты для вашего проекта.

<Callout>
  При использовании кластера `localnet` Anchor автоматически запускает локальный
  валидатор, развертывает программу, выполняет тесты, а затем останавливает
  валидатор.
</Callout>

<Callout type="warn">

Любая из следующих ошибок может указывать на то, что у вас не установлен Node.js
или Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
