Każda transakcja w sieci Solana wymaga opłaty transakcyjnej, płatnej w SOL.
Opłaty transakcyjne są podzielone na dwie części: opłatę podstawową i opłatę
priorytetową. Opłata podstawowa rekompensuje validatorom przetwarzanie
transakcji. Opłata priorytetowa jest opcjonalna i zwiększa szansę, że obecny
lider przetworzy Twoją transakcję.

## Opłata podstawowa

Każda transakcja kosztuje 5000 [lamportów](/docs/references/terminology#lamport)
za każdy dołączony podpis. Opłata ta jest płacona przez pierwszego podpisującego
transakcję i musi być uiszczona z konta należącego do System Program. Opłata
podstawowa jest podzielona w następujący sposób:

- **50% spalane:** Połowa jest
  [spalana](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/fee-calculator/src/lib.rs#L70)
  (usuwana z obiegu SOL).
- **50% dystrybuowane:** Połowa jest
  [wypłacana validatorowi](https://github.com/anza-xyz/agave/blob/e621336acad4f5d6e5b860eaa1b074b01c99253c/runtime/src/bank/fee_distribution.rs#L58-L62),
  który przetworzył transakcję.

## Opłata priorytetowa

[Opłata priorytetowa](https://github.com/anza-xyz/agave/blob/v2.2.14/compute-budget/src/compute_budget_limits.rs#L47-L48)
jest opcjonalną opłatą, która zwiększa szansę, że obecny lider (validator)
przetworzy Twoją transakcję. Validator otrzymuje
[100% opłaty priorytetowej](https://github.com/solana-foundation/solana-improvement-documents/blob/main/proposals/0096-reward-collected-priority-fee-in-entirety.md).
Opłaty priorytetowe można ustawić, dostosowując cenę jednostki obliczeniowej
[computation unit](/docs/references/terminology#compute-units) (CU) oraz limit
CU. (Zobacz
[przewodnik dotyczący korzystania z opłat priorytetowych](/developers/guides/advanced/how-to-use-priority-fees)
po więcej szczegółów na temat opłat priorytetowych.)

Opłata priorytetowa jest obliczana w następujący sposób:

```text title="Prioritization fee formula"
Prioritization fee = CU limit * CU price
```

Opłata priorytetowa jest używana do określenia
[priorytetu transakcji](https://github.com/anza-xyz/agave/blob/v2.2.14/core/src/banking_stage/transaction_scheduler/receive_and_buffer.rs#L646)
względem innych transakcji. Jest obliczana za pomocą następującego wzoru:

```text title="Transaction priority formula"
Priority = (Prioritization fee + Base fee) / (1 + CU limit + Signature CUs + Write lock CUs)
```

#### Limit jednostek obliczeniowych

Domyślnie
[default](https://github.com/anza-xyz/agave/blob/v2.1.13/compute-budget/src/compute_budget.rs#L149-L197),
każda instrukcja ma przydzielone
[200 000 CU](https://github.com/anza-xyz/agave/blob/v2.1.13/compute-budget/src/compute_budget_limits.rs#L10),
a każda transakcja ma przydzielone
[1,4 miliona CU](https://github.com/anza-xyz/agave/blob/v2.1.13/compute-budget/src/compute_budget_limits.rs#L14).
Możesz zmienić te wartości domyślne, dołączając instrukcję
[`SetComputeUnitLimit`](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/src/compute_budget.rs#L42-L44)
do swojej transakcji.

Aby obliczyć odpowiedni limit jednostek obliczeniowych (CU) dla swojej
transakcji, zalecamy wykonanie następujących kroków:

1. Oszacuj wymagane jednostki CU poprzez
   [symulację](/developers/guides/advanced/how-to-request-optimal-compute)
   transakcji
2. Dodaj 10% marginesu bezpieczeństwa do tego oszacowania

<Callout type="warn">
  Opłata priorytetowa jest określana na podstawie żądanego limitu jednostek
  obliczeniowych transakcji, *a nie* rzeczywistej liczby użytych jednostek
  obliczeniowych. Jeśli ustawisz zbyt wysoki limit jednostek obliczeniowych lub
  użyjesz domyślnej wartości, możesz zapłacić za niewykorzystane jednostki
  obliczeniowe.
</Callout>

#### Cena jednostki obliczeniowej

Cena jednostki obliczeniowej to opcjonalna kwota w
[mikro-lamportach](/docs/references/terminology#micro-lamports) płacona za każdą
żądaną jednostkę CU. Możesz traktować cenę jednostki CU jako napiwek, który
zachęca validatora do priorytetyzacji Twojej transakcji. Aby ustawić cenę
jednostki CU, dołącz instrukcję
[`SetComputeUnitPrice`](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/src/compute_budget.rs#L48-L50)
do swojej transakcji.

<Callout>
  Domyślna cena jednostki CU wynosi 0, co oznacza, że domyślna opłata
  priorytetowa również wynosi 0.
</Callout>

Aby uzyskać pomoc w określeniu najlepszej ceny jednostki CU dla swojej
transakcji, zapoznaj się z rekomendacją ceny jednostki CU w czasie rzeczywistym
podaną w tabeli poniżej.

```ts !! title="Typescript"
import {
  LAMPORTS_PER_SOL,
  SystemProgram,
  Transaction,
  Keypair,
  Connection,
  ComputeBudgetProgram,
  sendAndConfirmTransaction
} from "@solana/web3.js";

const connection = new Connection("http://localhost:8899", "confirmed");

const sender = Keypair.generate();
const recipient = new Keypair();

const airdropSignature = await connection.requestAirdrop(
  sender.publicKey,
  LAMPORTS_PER_SOL
);
await connection.confirmTransaction(airdropSignature, "confirmed");

// Create compute budget instructions
const limitInstruction = ComputeBudgetProgram.setComputeUnitLimit({
  units: 300_000
});
const priceInstruction = ComputeBudgetProgram.setComputeUnitPrice({
  microLamports: 1
});

const transferInstruction = SystemProgram.transfer({
  fromPubkey: sender.publicKey,
  toPubkey: recipient.publicKey,
  lamports: 0.01 * LAMPORTS_PER_SOL
});

// Add the compute budget and transfer instructions to a new transaction
const transaction = new Transaction()
  .add(limitInstruction)
  .add(priceInstruction)
  .add(transferInstruction);

const signature = await sendAndConfirmTransaction(connection, transaction, [
  sender
]);

console.log("Transaction Signature:", signature);
```

```rust !! title="Rust"
use solana_client::nonblocking::rpc_client::RpcClient;
use solana_commitment_config::CommitmentConfig;
use solana_compute_budget_interface::ComputeBudgetInstruction;
use solana_sdk::{
    native_token::LAMPORTS_PER_SOL,
    signature::{Keypair, Signer},
    transaction::Transaction,
};
use solana_system_interface::instruction::transfer;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let client = RpcClient::new_with_commitment(
        String::from("http://localhost:8899"),
        CommitmentConfig::confirmed(),
    );

    let sender = Keypair::new();
    let recipient = Keypair::new();

    let airdrop_signature = client
        .request_airdrop(&sender.pubkey(), LAMPORTS_PER_SOL)
        .await?;

    loop {
        let confirmed = client.confirm_transaction(&airdrop_signature).await?;
        if confirmed {
            break;
        }
    }

    // Create compute budget instructions
    let limit_instruction = ComputeBudgetInstruction::set_compute_unit_limit(300_000);
    let price_instruction = ComputeBudgetInstruction::set_compute_unit_price(1);

    let transfer_amount = LAMPORTS_PER_SOL / 100;
    let transfer_instruction = transfer(&sender.pubkey(), &recipient.pubkey(), transfer_amount);

    let recent_blockhash = client.get_latest_blockhash().await?;

    let mut transaction = Transaction::new_with_payer(
        &[limit_instruction, price_instruction, transfer_instruction],
        Some(&sender.pubkey()),
    );
    transaction.sign(&[&sender], recent_blockhash);

    let signature = client.send_and_confirm_transaction(&transaction).await?;
    println!("Transaction Signature: {}", signature);

    Ok(())
}
```
