Szybka instalacja, aby skonfigurować lokalne środowisko programistyczne Solana
za pomocą jednego polecenia. Zainstaluj Rust, Solana CLI i Anchor Framework na
Windows (WSL), Linux i Mac. (Jeśli wolisz, możesz
[zainstalować każdą zależność osobno](/docs/intro/installation/dependencies).)

## Wymagania wstępne

<Accordions>
<Accordion title="Windows">

Aby tworzyć programy Solana na Windows, **musisz** używać
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Po zainstalowaniu WSL wszystkie inne zależności będą instalowane za
pomocą terminala Linux.

Aby zainstalować WSL, uruchom następujące polecenie w Windows PowerShell:

```terminal
$ wsl --install
```

Proces instalacji poprosi Cię o utworzenie domyślnego konta użytkownika.

![Instalacja WSL](/assets/docs/intro/installation/wsl-install.png)

Domyślnie WSL instaluje Ubuntu. Możesz otworzyć terminal Linux, wyszukując
"Ubuntu" w pasku wyszukiwania.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Jeśli Twój terminal Ubuntu wygląda jak na poniższym obrazku, możesz napotkać
problem, w którym `ctrl + v` (skrót klawiaturowy do wklejania) nie działa w
terminalu.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Jeśli napotkasz ten problem, otwórz Windows Terminal, wyszukując "Terminal" w
pasku wyszukiwania.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Następnie zamknij Windows Terminal i ponownie otwórz terminal Linux, wyszukując
ponownie Ubuntu. Terminal powinien teraz wyglądać jak na poniższym obrazku,
gdzie `ctrl + v` (skrót klawiaturowy do wklejania) działa.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Jeśli używasz VS Code, rozszerzenie
[WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) umożliwia
korzystanie z WSL i VS Code razem.

![Konfiguracja WSL w VS Code](/assets/docs/intro/installation/wsl-vscode.png)

W pasku stanu VS Code zobaczysz następujące informacje:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Po skonfigurowaniu WSL możesz przejść do jednej z następujących opcji:

- Wykonaj [szybką instalację](#installation) za pomocą terminala Ubuntu (Linux)
- Zainstaluj indywidualnie
  [Rust](/docs/intro/installation/dependencies#install-rust),
  [Solana CLI](/docs/intro/installation/dependencies#install-solana-cli) oraz
  [Anchor CLI](/docs/intro/installation/dependencies#install-anchor-cli),
  zgodnie z poniższym opisem

</Accordion>

<Tab value=".deb" >
Jeśli używasz Debiana lub jego pochodnych, takich jak Ubuntu czy Mint, wykonaj poniższe kroki:

1. Zaktualizuj listy pakietów, aby upewnić się, że wszystkie pakiety są
   aktualne:

```terminal
$ sudo apt-get update
```

2. Zainstaluj następujące zależności:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Jeśli napotkasz następujący błąd podczas instalacji `protobuf-compiler`, upewnij się, że najpierw uruchomiłeś _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

## Instalacja

1. Wpisz następujące polecenie w terminalu, aby zainstalować wszystkie niezbędne
   zależności:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Pomyślna instalacja zwróci wynik podobny do poniższego:

```
Installed Versions:
Rust: rustc 1.86.0 (05f9846f8 2025-03-31)
Solana CLI: solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
Anchor CLI: anchor-cli 0.31.1
Node.js: v23.11.0
Yarn: 1.22.1
```

2. Zweryfikuj pomyślną instalację, sprawdzając wersję każdej zainstalowanej
   zależności.

```terminal
$ rustc --version && solana --version && anchor --version && node --version && yarn --version
```

<Callout>
  Jeśli polecenie szybkiej instalacji nie powiedzie się, zapoznaj się z sekcją
  [Instalacja zależności](/docs/intro/installation/dependencies) poniżej, aby
  uzyskać instrukcje dotyczące instalacji każdej zależności osobno.
</Callout>
