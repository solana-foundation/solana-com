Przewodnik po konfiguracji lokalnego środowiska programistycznego Solana.
Dowiedz się, jak zainstalować Rust, Solana CLI i Anchor Framework na Windows
(WSL), Linuxie i Macu. Skorzystaj z tego przewodnika, jeśli wolisz instalować
każdą zależność osobno lub jeśli [szybka instalacja](/docs/intro/installation/)
nie powiodła się z jakiegokolwiek powodu.

### Wymagania wstępne

<Accordions>
<Accordion title="Windows">

Aby tworzyć programy Solana na Windows, **musisz** używać
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Po zainstalowaniu WSL wszystkie inne zależności będą instalowane za
pomocą terminala Linux.

Aby zainstalować WSL, uruchom następujące polecenie w Windows PowerShell:

```terminal
$ wsl --install
```

Proces instalacji poprosi Cię o utworzenie domyślnego konta użytkownika.

![Instalacja WSL](/assets/docs/intro/installation/wsl-install.png)

Domyślnie WSL instaluje Ubuntu. Możesz otworzyć terminal Linux, wyszukując
"Ubuntu" w pasku wyszukiwania.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Jeśli Twój terminal Ubuntu wygląda jak na poniższym obrazku, możesz napotkać
problem, w którym `ctrl + v` (skrót klawiaturowy do wklejania) nie działa w
terminalu.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Jeśli napotkasz ten problem, otwórz Windows Terminal, wyszukując "Terminal" w
pasku wyszukiwania.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Następnie zamknij Windows Terminal i ponownie otwórz terminal Linux, wyszukując
ponownie Ubuntu. Terminal powinien teraz wyglądać jak na poniższym obrazku,
gdzie `ctrl + v` (skrót klawiaturowy do wklejania) działa.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Jeśli używasz VS Code, rozszerzenie
[WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) umożliwia
korzystanie z WSL i VS Code razem.

![Konfiguracja WSL w VS Code](/assets/docs/intro/installation/wsl-vscode.png)

W pasku stanu VS Code zobaczysz następujące informacje:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Po skonfigurowaniu WSL możesz:

- Wykonać [szybką instalację](/docs/intro/installation) za pomocą terminala
  Ubuntu (Linux)
- Zainstalować indywidualnie [Rust](#install-rust),
  [Solana CLI](#install-solana-cli) oraz [Anchor CLI](#install-anchor-cli), jak
  opisano poniżej

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Jeśli używasz Debiana lub jego pochodnych, takich jak Ubuntu czy Mint, wykonaj poniższe kroki:

1. Zaktualizuj listy pakietów, aby upewnić się, że wszystkie pakiety są
   aktualne:

```terminal
$ sudo apt-get update
```

2. Zainstaluj następujące zależności:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Jeśli napotkasz następujący błąd podczas instalacji `protobuf-compiler`, upewnij się, że najpierw uruchomiłeś _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    W przypadku dystrybucji opartych na RHEL, takich jak Fedora czy Red Hat Linux, wykonaj następujące kroki.

Zaktualizuj listy pakietów, aby pobrać nowsze wersje pakietów, uruchamiając
polecenie.

```terminal
$ sudo dnf upgrade --refresh
```

Następnie zainstaluj następujące zależności:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Instalacja Rust

Deweloperzy tworzą programy Solana, używając języka programowania
[Rust](https://www.rust-lang.org/).

1. Zainstaluj Rust za pomocą [rustup](https://www.rust-lang.org/tools/install),
   wpisując następujące polecenie w terminalu:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Po pomyślnej instalacji zobaczysz następujący komunikat:

<Accordions>
<Accordion title="Komunikat o pomyślnej instalacji Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Załaduj ponownie zmienną środowiskową PATH, aby uwzględnić katalog bin
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Zweryfikuj, czy instalacja zakończyła się pomyślnie.

```terminal
$ rustc --version
```

Zobaczysz wynik podobny do poniższego:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Instalacja Solana CLI

Solana CLI dostarcza wszystkie narzędzia potrzebne do tworzenia i wdrażania
programów Solana.

1. Zainstaluj zestaw narzędzi Solana CLI, używając oficjalnego polecenia
   instalacyjnego:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Możesz zastąpić `stable` tagiem wydania odpowiadającym wersji oprogramowania
  Twojego pożądanego wydania (np. `v2.0.3`), lub użyć jednej z trzech
  symbolicznych nazw kanałów: `stable`, `beta`, lub `edge`.
</Callout>

2. Dodaj zmienną środowiskową PATH

Podczas pierwszej instalacji Solana CLI możesz zostać poproszony o dodanie
zmiennej środowiskowej `PATH`. Aby to zrobić, zamknij i ponownie otwórz terminal
lub uruchom następujące polecenie w swojej powłoce:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Zaktualizuj zmienną środowiskową PATH

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Jeśli używasz **Linuxa** lub **WSL**, musisz dodać plik binarny Solana CLI do
swojej zmiennej PATH, aby polecenie było dostępne w terminalu. Aby to zrobić,
wykonaj poniższe kroki:

**a.** Sprawdź, której powłoki używasz:

```terminal
$ echo $SHELL
```

- Jeśli wynik zawiera `/bash`, użyj `.bashrc`.
- Jeśli wynik zawiera `/zsh`, użyj `.zshrc`.

**b.** Uruchom odpowiednie polecenie w zależności od używanej powłoki.

Dla Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Dla Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Jeśli używasz Maca z `zsh`, uruchomienie domyślnego polecenia `export PATH`
zalogowanego podczas instalacji nie będzie trwałe po zamknięciu terminala.
Zamiast tego dodaj ścieżkę PATH do pliku konfiguracyjnego powłoki, uruchamiając
następujące polecenie:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Powyższe kroki nie dotyczą użytkowników `bash`, ponieważ zmiany w zmiennej
  PATH będą trwałe domyślnie.
</Callout>

</Tab>
</Tabs>

4. Uruchom ponownie terminal lub wykonaj następujące polecenie, aby odświeżyć
   sesję terminala:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Zweryfikuj, czy instalacja zakończyła się sukcesem, sprawdzając wersję Solana
   CLI:

```terminal
$ solana --version
```

Zobaczysz wynik podobny do poniższego:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Możesz zobaczyć wszystkie dostępne wersje w
[repozytorium Agave na Githubie](https://github.com/anza-xyz/agave/releases).
Agave to klient walidatora od [Anza](https://www.anza.xyz/), wcześniej znany
jako klient walidatora Solana Labs.

</Callout>

5. Zaktualizuj Solana CLI do najnowszej wersji, jeśli to konieczne
   _(Opcjonalne)_

```terminal
$ agave-install update
```

</Step>

### Zainstaluj Anchor CLI

[Anchor](https://www.anchor-lang.com/) to framework do tworzenia programów na
platformie Solana. Framework Anchor wykorzystuje makra języka Rust, aby uprościć
proces pisania programów na Solanę.

#### Wymagania wstępne

Domyślny plik testowy projektu Anchor (TypeScript) utworzony za pomocą polecenia
_shell`anchor init`_ wymaga Node.js i Yarn. (Szablon testowy w języku Rust jest
dostępny za pomocą _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Instalacja Node">

1. Zainstaluj Node za pomocą
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm), uruchamiając
   następujące polecenie:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Uruchom ponownie terminal i potwierdź, że polecenie `nvm` działa poprawnie:

```terminal
$ command -v nvm
```

3. Użyj `nvm`, aby zainstalować Node:

```terminal
$ nvm install node
```

4. Zweryfikuj, czy instalacja zakończyła się sukcesem, sprawdzając wersję Node:

```terminal
$ node --version
```

Zobaczysz wynik podobny do poniższego:

```
v23.7.0
```

</Accordion>
<Accordion title="Instalacja Yarn">

1. Zainstaluj Yarn:

```terminal
$ npm install --global yarn
```

2. Zweryfikuj, czy instalacja zakończyła się sukcesem, sprawdzając wersję Yarn:

```terminal
$ yarn --version
```

Zobaczysz następujący wynik:

```
1.22.1
```

</Accordion>
</Accordions>

#### Instalacja

Możesz zainstalować Anchor CLI i narzędzia na dwa sposoby:

- Anchor Version Manager (AVM) — zalecana metoda instalacji
- Bez AVM — instalacja bezpośrednio z GitHuba

<Tabs groupId="anchor" items={["AVM", "Bez AVM"]}>
<Tab value="AVM">

Anchor Version Manager (AVM) umożliwia instalację i zarządzanie różnymi wersjami
Anchor na Twoim systemie oraz łatwą aktualizację wersji Anchor w przyszłości.
Aby zainstalować Anchor za pomocą AVM, wykonaj poniższe kroki:

1. Zainstaluj AVM za pomocą następującego polecenia:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Potwierdź, że AVM został pomyślnie zainstalowany:

```terminal
$ avm --version
```

3. Zainstaluj Anchor CLI za pomocą AVM:

Aby zainstalować najnowszą wersję:

```terminal
$ avm install latest
$ avm use latest
```

Aby zainstalować konkretną wersję, podaj numer wersji:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Podczas instalacji Anchor CLI na Linuxie lub WSL możesz napotkać ten błąd:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Jeśli zobaczysz ten komunikat o błędzie, wykonaj następujące kroki:

1. Zainstaluj zależności wymienione w sekcji
   [Linux](/docs/intro/installation#install-dependencies) na początku tej
   strony.
2. Ponownie spróbuj zainstalować Anchor CLI.

</Callout>

4. Zweryfikuj, czy instalacja zakończyła się sukcesem, sprawdzając wersję Anchor
   CLI:

```terminal
$ anchor --version
```

Zobaczysz wynik podobny do poniższego:

```
anchor-cli 0.31.1
```

<Callout type="info">

Nie zapomnij uruchomić polecenia _shell`avm use`_, aby zadeklarować wersję
Anchor CLI uruchamianą na Twoim systemie.

- Jeśli zainstalowałeś wersję `latest`, uruchom _shell`avm use latest`_.
- Jeśli zainstalowałeś wersję `0.30.1`, uruchom _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Bez AVM">

1. Zainstaluj określoną wersję Anchor CLI za pomocą następującego polecenia:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Zweryfikuj, czy instalacja zakończyła się pomyślnie, sprawdzając wersję
   Anchor CLI:

```terminal
$ anchor --version
```

Zobaczysz wynik podobny do poniższego:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>
