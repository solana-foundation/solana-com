Ta bramka funkcji zmniejsza domyślną alokację CU dla instrukcji wbudowanych
programów z 200 000 do 3 000 CU, aby poprawić wydajność walidatorów.

## Wpływ

Mniej niż 0,00001% transakcji zostanie dotkniętych tą zmianą.

**Deweloperzy** powinni zawsze używać limitów budżetu obliczeniowego, aby jawnie
alokować CU dla swoich transakcji. Ta zmiana nie wpłynie na transakcje, które
jawnie alokują CU.
