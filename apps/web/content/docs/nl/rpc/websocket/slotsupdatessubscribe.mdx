h1: slotsUpdatesSubscribe RPC-methode
---

Abonneer je om meldingen van de validator te ontvangen over verschillende
updates bij elke slot

<Callout type="warn">
  Dit abonnement is onstabiel. Het formaat van dit abonnement kan in de toekomst
  veranderen en wordt mogelijk niet altijd ondersteund.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "slotsUpdatesSubscribe"
}
```

### !params

**Geen**

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

Abonnement-id (nodig om het abonnement op te zeggen)

</APIMethod>

### Meldingsformaat

De melding is een object met de volgende velden:

- `err: <string|undefined>` - Het foutbericht. Alleen aanwezig als het
  updatetype "dead" is.
- `parent: <u64|undefined>` - De bovenliggende slot. Alleen aanwezig als het
  updatetype "createdBank" is.
- `slot: <u64>` - De nieuw bijgewerkte slot
- `stats: <object|undefined>` - Het foutbericht. Alleen aanwezig als het
  updatetype "frozen" is. Een object met de volgende velden:
  - `maxTransactionsPerEntry: <u64>`,
  - `numFailedTransactions: <u64>`,
  - `numSuccessfulTransactions: <u64>`,
  - `numTransactionEntries: <u64>`,
- `timestamp: <i64>` - De Unix-tijdstempel van de update in milliseconden
- `type: <string>` - Het updatetype, een van:
  - "firstShredReceived"
  - "completed"
  - "createdBank"
  - "frozen"
  - "dead"
  - "optimisticConfirmation"
  - "root"

```shell
{
  "jsonrpc": "2.0",
  "method": "slotsUpdatesNotification",
  "params": {
    "result": {
      "parent": 75,
      "slot": 76,
      "timestamp": 1625081266243,
      "type": "optimisticConfirmation"
    },
    "subscription": 0
  }
}
```
