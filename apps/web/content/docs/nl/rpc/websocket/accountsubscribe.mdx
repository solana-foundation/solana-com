h1: accountSubscribe RPC-methode
---

Abonneer je op een account om meldingen te ontvangen wanneer de lamports of
gegevens voor een bepaalde account public key veranderen

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "accountSubscribe",
  "params": [
    // !hover 0
    "CM78CPUeXjn8o3yroDHxUtKsZZgoy4GPkPPXfouKNH12",
    // !hover(1:4) 1
    {
      // !hover encoding
      "encoding": "jsonParsed",
      // !hover commitment
      "commitment": "finalized"
    }
  ]
}
```

### !params

#### !! 0

!type string  
!required

Account pubkey, als base-58 gecodeerde string

#### !! 1

!type object

Configuratieobject met de volgende velden:

##### !! commitment

!type string

De commitment beschrijft hoe definitief een blok is op dat moment. Zie
[Configureren van state commitment](/docs/rpc#configuring-state-commitment).

##### !! encoding

!type string  
!values base58 base64 base64+zstd jsonParsed

Coderingsformaat voor accountgegevens

- `base58` is traag.
- `jsonParsed` codering probeert programmaspecifieke state parsers te gebruiken
  om meer leesbare en expliciete accountstatusgegevens te retourneren
- Als `jsonParsed` wordt aangevraagd maar er geen parser kan worden gevonden,
  valt het veld terug naar binaire codering, te herkennen wanneer het `data`
  veld van het type `string` is.

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 23784,
  "id": 1
}
```

!type number

Abonnement-id (nodig om het abonnement op te zeggen)

</APIMethod>

[getAccountInfo](/docs/rpc/http/getaccountinfo) RPC HTTP-methode.
