Ένας οδηγός για τη ρύθμιση του τοπικού περιβάλλοντος ανάπτυξης Solana. Μάθετε
πώς να εγκαταστήσετε τη Rust, το Solana CLI και το Anchor Framework σε Windows
(WSL), Linux και Mac. Χρησιμοποιήστε αυτόν τον οδηγό αν προτιμάτε να
εγκαταστήσετε κάθε εξάρτηση ξεχωριστά, ή αν η
[γρήγορη εγκατάσταση](/docs/intro/installation/) αποτύχει για οποιονδήποτε λόγο.

### Προαπαιτούμενα

<Accordions>
<Accordion title="Windows">

Για να αναπτύξετε προγράμματα Solana στα Windows **πρέπει** να χρησιμοποιήσετε
το
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Μετά την εγκατάσταση του WSL, όλες οι άλλες εξαρτήσεις θα εγκατασταθούν
μέσω του τερματικού Linux.

Για να εγκαταστήσετε το WSL, εκτελέστε την ακόλουθη εντολή στο Windows
PowerShell:

```terminal
$ wsl --install
```

Η διαδικασία εγκατάστασης θα σας ζητήσει να δημιουργήσετε έναν προεπιλεγμένο
λογαριασμό χρήστη.

![Εγκατάσταση WSL](/assets/docs/intro/installation/wsl-install.png)

Από προεπιλογή, το WSL εγκαθιστά το Ubuntu. Μπορείτε να ανοίξετε ένα τερματικό
Linux αναζητώντας "Ubuntu" στη γραμμή αναζήτησης.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Αν το τερματικό Ubuntu σας μοιάζει με την παρακάτω εικόνα, μπορεί να
αντιμετωπίσετε ένα πρόβλημα όπου το `ctrl + v` (συντόμευση πληκτρολογίου για
επικόλληση) δεν λειτουργεί στο τερματικό.

![Τερματικό Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Αν αντιμετωπίσετε αυτό το πρόβλημα, ανοίξτε το Windows Terminal αναζητώντας
"Terminal" στη γραμμή αναζήτησης.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Στη συνέχεια, κλείστε το Windows Terminal και ανοίξτε ξανά ένα τερματικό Linux
αναζητώντας ξανά το Ubuntu. Το τερματικό θα πρέπει τώρα να μοιάζει με την
παρακάτω εικόνα, όπου το `ctrl + v` (συντόμευση πληκτρολογίου για επικόλληση)
λειτουργεί.

![Τερματικό Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Αν χρησιμοποιείτε το VS Code, η
[επέκταση WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) σας
επιτρέπει να χρησιμοποιείτε το WSL και το VS Code μαζί.

![Ρύθμιση WSL στο VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Θα δείτε τα ακόλουθα στη γραμμή κατάστασης του VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Μόλις ρυθμίσετε το WSL, είστε έτοιμοι είτε να:

- Ολοκληρώσετε την [γρήγορη εγκατάσταση](/docs/intro/installation)
  χρησιμοποιώντας το τερματικό Ubuntu (Linux)
- Εγκαταστήσετε ξεχωριστά τη [Rust](#install-rust), το
  [Solana CLI](#install-solana-cli) και το [Anchor CLI](#install-anchor-cli),
  όπως περιγράφεται παρακάτω

</Accordion>
<Accordion title="Linux">

<Tab value=".deb" >
Αν χρησιμοποιείτε Debian ή παράγωγα του Debian όπως Ubuntu ή Mint, ακολουθήστε τα παρακάτω βήματα:

1. Ενημερώστε τις λίστες πακέτων για να διασφαλίσετε ότι όλα τα πακέτα είναι
   ενημερωμένα:

```terminal
$ sudo apt-get update
```

2. Εγκαταστήστε τις ακόλουθες εξαρτήσεις:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Αν αντιμετωπίσετε το ακόλουθο σφάλμα κατά την εγκατάσταση του `protobuf-compiler`, βεβαιωθείτε
ότι πρώτα εκτελείτε την εντολή _shell`sudo apt-get update`

```

This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

εκτελώντας την εντολή.

```terminal
$ sudo dnf upgrade --refresh
```

Στη συνέχεια, εγκαταστήστε τις ακόλουθες εξαρτήσεις:

```terminal
$ sudo dnf group install c-development development-tools \

```

### Εγκατάσταση Rust

Οι προγραμματιστές δημιουργούν προγράμματα Solana χρησιμοποιώντας τη γλώσσα
προγραμματισμού [Rust](https://www.rust-lang.org/).

1. Εγκαταστήστε τη Rust χρησιμοποιώντας το
   [rustup](https://www.rust-lang.org/tools/install) εισάγοντας την ακόλουθη
   εντολή στο τερματικό σας:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Μετά από μια επιτυχημένη εγκατάσταση θα δείτε το ακόλουθο μήνυμα:

<Accordions>
<Accordion title="Μήνυμα επιτυχούς εγκατάστασης της Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Επαναφορτώστε τη μεταβλητή περιβάλλοντος PATH για να συμπεριλάβετε τον
   κατάλογο bin του [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Επαληθεύστε ότι η εγκατάσταση ήταν επιτυχής.

```terminal
$ rustc --version
```

Θα δείτε μια έξοδο όπως η ακόλουθη:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Εγκατάσταση του Solana CLI

Το Solana CLI παρέχει όλα τα εργαλεία που απαιτούνται για την ανάπτυξη και την
ανάπτυξη προγραμμάτων Solana.

1. Εγκαταστήστε τη σουίτα εργαλείων Solana CLI χρησιμοποιώντας την επίσημη
   εντολή εγκατάστασης:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Μπορείτε να αντικαταστήσετε το `stable` με την ετικέτα έκδοσης που αντιστοιχεί
  στην έκδοση λογισμικού που επιθυμείτε (π.χ. `v2.0.3`), ή να χρησιμοποιήσετε
  ένα από τα τρία συμβολικά ονόματα καναλιών: `stable`, `beta`, ή `edge`.
</Callout>

2. Προσθέστε μια μεταβλητή περιβάλλοντος PATH

Κατά την πρώτη εγκατάσταση του Solana CLI, μπορεί να σας ζητηθεί να προσθέσετε
μια μεταβλητή περιβάλλοντος `PATH`. Για να το κάνετε αυτό, κλείστε και ανοίξτε
ξανά το τερματικό σας ή εκτελέστε το ακόλουθο στο κέλυφός σας:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Ενημερώστε τη μεταβλητή περιβάλλοντος PATH

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Εάν χρησιμοποιείτε **Linux** ή **WSL**, πρέπει να προσθέσετε το εκτελέσιμο
αρχείο Solana CLI στο PATH σας ώστε η εντολή να είναι διαθέσιμη στο τερματικό
σας. Για να το κάνετε αυτό, ακολουθήστε τα παρακάτω βήματα:

**α.** Ελέγξτε ποιο κέλυφος χρησιμοποιείτε:

```terminal
$ echo $SHELL
```

- Αν το αποτέλεσμα περιέχει `/bash`, χρησιμοποιήστε `.bashrc`.
- Αν το αποτέλεσμα περιέχει `/zsh`, χρησιμοποιήστε `.zshrc`.

**β.** Εκτελέστε την κατάλληλη εντολή, ανάλογα με το κέλυφός σας.

Για Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Για Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Αν χρησιμοποιείτε Mac με `zsh`, η εκτέλεση της προεπιλεγμένης εντολής
`export PATH` που καταγράφεται από την εγκατάσταση δεν διατηρείται μόλις
κλείσετε το τερματικό σας. Αντ' αυτού, προσθέστε τη διαδρομή PATH στο αρχείο
ρύθμισης του κελύφους σας εκτελώντας την ακόλουθη εντολή:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Τα παραπάνω βήματα δεν ισχύουν για τους χρήστες `bash`, καθώς οι αλλαγές στη
  μεταβλητή PATH θα διατηρηθούν από προεπιλογή.
</Callout>

</Tab>
</Tabs>

4. Επανεκκινήστε το τερματικό σας ή εκτελέστε την ακόλουθη εντολή για να
   ανανεώσετε τη συνεδρία του τερματικού:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Επαληθεύστε ότι η εγκατάσταση ήταν επιτυχής ελέγχοντας την έκδοση του Solana
   CLI:

```terminal
$ solana --version
```

Θα δείτε αποτέλεσμα όπως το ακόλουθο:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Μπορείτε να δείτε όλες τις διαθέσιμες εκδόσεις στο
[Agave Github repo](https://github.com/anza-xyz/agave/releases). Το Agave είναι
ο πελάτης validator από την [Anza](https://www.anza.xyz/), παλαιότερα γνωστός ως
πελάτης validator της Solana Labs.

</Callout>

5. Ενημερώστε το Solana CLI στην τελευταία έκδοση, όπως χρειάζεται
   _(Προαιρετικό)_

```terminal
$ agave-install update
```

</Step>

### Εγκατάσταση του Anchor CLI

Το [Anchor](https://www.anchor-lang.com/) είναι ένα πλαίσιο για την ανάπτυξη
προγραμμάτων Solana. Το πλαίσιο Anchor αξιοποιεί τα macros της Rust για να
απλοποιήσει τη διαδικασία συγγραφής προγραμμάτων Solana.

#### Προαπαιτούμενα

Το προεπιλεγμένο αρχείο δοκιμών του έργου Anchor (TypeScript) που δημιουργείται
με την εντολή _shell`anchor init`_ απαιτεί Node.js και Yarn. (Το πρότυπο δοκιμών
Rust είναι διαθέσιμο χρησιμοποιώντας _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Εγκατάσταση Node">

1. Εγκαταστήστε το Node χρησιμοποιώντας το
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) εκτελώντας την
   ακόλουθη εντολή:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Επανεκκινήστε το τερματικό σας και επιβεβαιώστε ότι η εντολή `nvm` εκτελείται
   με επιτυχία:

```terminal
$ command -v nvm
```

3. Χρησιμοποιήστε το `nvm` για να εγκαταστήσετε το Node:

```terminal
$ nvm install node
```

4. Επαληθεύστε ότι η εγκατάσταση ήταν επιτυχής, ελέγχοντας την έκδοση του Node:

```terminal
$ node --version
```

Θα δείτε μια έξοδο όπως η ακόλουθη:

```
v23.7.0
```

</Accordion>
<Accordion title="Εγκατάσταση Yarn">

1. Εγκαταστήστε το Yarn:

```terminal
$ npm install --global yarn
```

2. Επαληθεύστε ότι η εγκατάσταση ήταν επιτυχής, ελέγχοντας την έκδοση του Yarn:

```terminal
$ yarn --version
```

Θα δείτε την ακόλουθη έξοδο:

```
1.22.1
```

</Accordion>
</Accordions>

#### Εγκατάσταση

Μπορείτε να εγκαταστήσετε το CLI και τα εργαλεία του Anchor με δύο τρόπους:

- Anchor Version Manager (AVM) — Συνιστώμενη μέθοδος εγκατάστασης
- Χωρίς AVM — Εγκατάσταση απευθείας από το GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Ο Διαχειριστής Εκδόσεων Anchor (AVM) σας επιτρέπει να εγκαταστήσετε και να
διαχειριστείτε διαφορετικές εκδόσεις του Anchor στο σύστημά σας και να
ενημερώνετε εύκολα τις εκδόσεις του Anchor στο μέλλον. Για να εγκαταστήσετε το
Anchor χρησιμοποιώντας το AVM, ακολουθήστε τα παρακάτω βήματα:

1. Εγκαταστήστε το AVM με την ακόλουθη εντολή:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Επιβεβαιώστε ότι το AVM εγκαταστάθηκε με επιτυχία:

```terminal
$ avm --version
```

3. Εγκαταστήστε το Anchor CLI χρησιμοποιώντας το AVM:

Για να εγκαταστήσετε την πιο πρόσφατη έκδοση:

```terminal
$ avm install latest
$ avm use latest
```

Για να εγκαταστήσετε μια συγκεκριμένη έκδοση, καθορίστε τον αριθμό έκδοσης:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Κατά την εγκατάσταση του Anchor CLI σε Linux ή WSL, ενδέχεται να αντιμετωπίσετε
αυτό το σφάλμα:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Αν δείτε αυτό το μήνυμα σφάλματος, ακολουθήστε αυτά τα βήματα:

1. Εγκαταστήστε τις εξαρτήσεις που αναφέρονται στην ενότητα
   [Linux](/docs/intro/installation#install-dependencies) στην αρχή αυτής της
   σελίδας.
2. Δοκιμάστε ξανά να εγκαταστήσετε το Anchor CLI.

</Callout>

4. Επαληθεύστε ότι η εγκατάσταση ολοκληρώθηκε με επιτυχία, ελέγχοντας την έκδοση
   του Anchor CLI:

```terminal
$ anchor --version
```

Θα δείτε μια έξοδο όπως η παρακάτω:

```
anchor-cli 0.31.1
```

<Callout type="info">

Μην ξεχάσετε να εκτελέσετε την εντολή _shell`avm use`_ για να δηλώσετε την
έκδοση του Anchor CLI που θα εκτελείται στο σύστημά σας.

- Αν εγκαταστήσατε την έκδοση `latest`, εκτελέστε _shell`avm use latest`_.
- Αν εγκαταστήσατε την έκδοση `0.30.1`, εκτελέστε _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Εγκαταστήστε μια συγκεκριμένη έκδοση του Anchor CLI με την ακόλουθη εντολή:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Επαληθεύστε ότι η εγκατάσταση ολοκληρώθηκε με επιτυχία, ελέγχοντας την έκδοση
   του Anchor CLI:

```terminal
$ anchor --version
```

Θα δείτε μια έξοδο όπως η παρακάτω:

```
anchor-cli 0.31.1
```
