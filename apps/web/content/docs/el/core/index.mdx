Για να αναπτύξετε στο Solana, είναι απαραίτητο να κατανοήσετε βασικές έννοιες
που είναι μοναδικές στην ανάπτυξη Solana. Αυτή η ενότητα καλύπτει τις βασικές
έννοιες που πρέπει να κατανοήσετε καθώς ξεκινάτε να αναπτύσσετε στο Solana,
συμπεριλαμβανομένων των λογαριασμών, των συναλλαγών, των προγραμμάτων και άλλων.

## Λογαριασμοί

Όλα τα δεδομένα στο blockchain του Solana αποθηκεύονται ως Λογαριασμός. Η δομή
του blockchain του Solana μοιάζει με αποθήκευση κλειδιού-τιμής, όπου το κλειδί
είναι η διεύθυνση στην αλυσίδα και η τιμή είναι ένας Λογαριασμός.

Μάθετε περισσότερα για τους [Λογαριασμούς](/docs/core/accounts).

## Συναλλαγές και οδηγίες

Για να αλληλεπιδράσει με το δίκτυο Solana, ένας χρήστης πρέπει να στείλει μια
συναλλαγή. Μια συναλλαγή είναι ένα σύνολο μίας ή περισσότερων οδηγιών—μια οδηγία
είναι μια συγκεκριμένη λειτουργία που πρέπει να επεξεργαστεί. Η λογική εκτέλεσης
για μια οδηγία αποθηκεύεται σε ένα πρόγραμμα που έχει αναπτυχθεί στο δίκτυο
Solana.

Μάθετε περισσότερα για τις [Συναλλαγές](/docs/core/transactions) και τις
[Οδηγίες](/docs/core/transactions#instruction).

## Χρεώσεις

Υπάρχουν τρεις διαφορετικοί τύποι χρεώσεων που μπορεί να προκύψουν κατά τη χρήση
του blockchain του Solana:

- **Χρεώσεις Συναλλαγών**: Μια χρέωση για να επεξεργαστούν οι validators τις
  συναλλαγές και τις οδηγίες
- **Χρεώσεις Προτεραιότητας**: Μια προαιρετική χρέωση για να ενισχύσετε τη σειρά
  επεξεργασίας μιας συναλλαγής
- **Rent**: Ένα παρακρατούμενο υπόλοιπο για να διατηρηθούν τα δεδομένα
  αποθηκευμένα στην αλυσίδα

Μάθετε περισσότερα για τις [Χρεώσεις στο Solana](/docs/core/fees).

## Προγράμματα στο Solana

Στο Solana, τα έξυπνα συμβόλαια ονομάζονται Προγράμματα. Κάθε πρόγραμμα
αποθηκεύεται σε έναν λογαριασμό στην αλυσίδα και περιέχει εκτελέσιμο κώδικα με
συγκεκριμένες οδηγίες. Αυτές οι οδηγίες αντιπροσωπεύουν τη λειτουργικότητα του
προγράμματος και καλούνται στέλνοντας μια συναλλαγή στο δίκτυο.

Μάθετε περισσότερα για τα [Προγράμματα στο Solana](/docs/core/programs).

## Program Derived Address

Οι διευθύνσεις που προέρχονται από προγράμματα (PDAs) επιτρέπουν στους
προγραμματιστές να δημιουργούν δομές τύπου hashmap στην αλυσίδα χρησιμοποιώντας
ένα προκαθορισμένο σύνολο εισόδων (π.χ. συμβολοσειρές, αριθμούς και άλλες
διευθύνσεις λογαριασμών). Τα PDAs επιτρέπουν δύο σημαντικές περιπτώσεις χρήσης:

- **Ντετερμινιστικές Διευθύνσεις Λογαριασμών**: Προκαθορισμένες είσοδοι (seeds)
  και ένα αναγνωριστικό προγράμματος συνδυάζονται για να παράγουν
  ντετερμινιστικά μια διεύθυνση.
- **Υπογραφή Προγράμματος**: Τα προγράμματα Solana μπορούν να υπογράψουν για
  PDAs, που προέρχονται από το αναγνωριστικό του προγράμματός τους.

Μάθετε περισσότερα για τις
[Διευθύνσεις Παραγόμενες από Πρόγραμμα](/docs/core/pda).

## Cross-program Invocation

Η Cross-program invocation (CPI) συμβαίνει όταν ένα πρόγραμμα επικαλείται μια
εντολή από ένα άλλο. Αυτό επιτρέπει τη συνθεσιμότητα των προγραμμάτων Solana.
Για να κατανοήσετε την έννοια, φανταστείτε το σύνολο των εντολών ενός
προγράμματος ως ένα σύνολο τελικών σημείων API. Ένα CPI συμβαίνει όταν ένα
τελικό σημείο επικαλείται ένα άλλο.

Μάθετε περισσότερα για το [Cross Program Invocation](/docs/core/cpi).

## Tokens

Τα tokens είναι ψηφιακά περιουσιακά στοιχεία που αντιπροσωπεύουν την ιδιοκτησία
διαφόρων κατηγοριών περιουσιακών στοιχείων. Η τοκενοποίηση επιτρέπει την
ψηφιοποίηση των δικαιωμάτων ιδιοκτησίας—ένα θεμελιώδες συστατικό για τη
διαχείριση ανταλλάξιμων και μη ανταλλάξιμων περιουσιακών στοιχείων.

- **Ανταλλάξιμα Tokens**: Εναλλάξιμα και διαιρετά περιουσιακά στοιχεία του ίδιου
  τύπου και αξίας (π.χ. USDC).
- **Μη Ανταλλάξιμα Tokens (NFT):** Ιδιοκτησία αδιαίρετων περιουσιακών στοιχείων
  (π.χ. έργα τέχνης).

Μάθετε περισσότερα για τα [Tokens στο Solana](/docs/core/tokens).

## Συστάδες και Τελικά Σημεία

Το blockchain του Solana έχει διάφορες ομάδες επικυρωτών, γνωστές ως
[Συστάδες](/docs/references/clusters). Κάθε συστάδα εξυπηρετεί διαφορετικό σκοπό
και περιέχει αφιερωμένους κόμβους για την εκπλήρωση αιτημάτων
[JSON-RPC](/docs/rpc/). Υπάρχουν τρεις κύριες συστάδες στο δίκτυο Solana, που
φαίνονται στον παρακάτω πίνακα.

| Όνομα   | Τελικό Σημείο                         | Χρήση                         |
| ------- | ------------------------------------- | ----------------------------- |
| Mainnet | `https://api.mainnet-beta.solana.com` | Παραγωγή                      |
| Devnet  | `https://api.devnet.solana.com`       | Πειραματισμός προγραμματιστών |
| Testnet | `https://api.testnet.solana.com`      | Δοκιμή επικυρωτών             |

Μάθετε περισσότερα για τις
[Συστάδες και τα Τελικά Σημεία](/docs/references/clusters).
