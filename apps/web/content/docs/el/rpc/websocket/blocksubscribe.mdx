h1: Μέθοδος RPC blockSubscribe
---

Εγγραφείτε για να λαμβάνετε ειδοποιήσεις κάθε φορά που ένα νέο μπλοκ είναι
`confirmed` ή `finalized`.

<Callout type="warn" title={"Ασταθής Μέθοδος"}>
  Αυτή η συνδρομή θεωρείται **ασταθής** και είναι διαθέσιμη μόνο αν ο validator
  ξεκίνησε με τη σημαία `--rpc-pubsub-enable-block-subscription`. Η μορφή αυτής
  της συνδρομής μπορεί να αλλάξει στο μέλλον.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    // !hover(1:3) 0
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    // !hover(1:6) 1
    {
      // !hover commitment
      "commitment": "confirmed",
      // !hover encoding
      "encoding": "base64",
      // !hover transactionDetails
      "transactionDetails": "full",
      // !hover maxSupportedTransactionVersion
      "maxSupportedTransactionVersion": 0,
      // !hover showRewards
      "showRewards": true
    }
  ]
}
```

### !params

#### !! 0

!type string | object  
!required

κριτήρια φιλτραρίσματος για τα αρχεία καταγραφής ώστε να λαμβάνετε αποτελέσματα
ανά τύπο λογαριασμού· προς το παρόν υποστηρίζονται:

- `all` - συμπεριλαμβάνει όλες τις συναλλαγές στο μπλοκ
- Ένα αντικείμενο JSON με το ακόλουθο πεδίο:
  - `mentionsAccountOrProgram: <string>` - επιστρέφει μόνο συναλλαγές που
    αναφέρουν το παρεχόμενο δημόσιο κλειδί (ως συμβολοσειρά κωδικοποιημένη σε
    base-58). Αν δεν υπάρχουν αναφορές σε ένα συγκεκριμένο μπλοκ, τότε δεν θα
    σταλεί ειδοποίηση.

#### !! 1

!type object

Αντικείμενο διαμόρφωσης που περιέχει τα ακόλουθα πεδία:

##### !! commitment

!type string
!values confirmed finalized
!default finalized

Το commitment περιγράφει πόσο οριστικοποιημένο είναι ένα μπλοκ σε εκείνο το
χρονικό σημείο. Δείτε
[Διαμόρφωση Δέσμευσης Κατάστασης](/docs/rpc#configuring-state-commitment).

- Το `processed` δεν υποστηρίζεται.

##### !! encoding

!type string  
!values json jsonParsed base58 base64  
!default json

μορφή κωδικοποίησης για κάθε επιστρεφόμενη Συναλλαγή

- Το `jsonParsed` προσπαθεί να χρησιμοποιήσει αναλυτές οδηγιών συγκεκριμένων
  προγραμμάτων για να επιστρέψει πιο αναγνώσιμα και σαφή δεδομένα στη λίστα
  `transaction.message.instructions`.
- Αν ζητηθεί το `jsonParsed` αλλά δεν βρεθεί αναλυτής, η οδηγία επιστρέφει στην
  κανονική κωδικοποίηση JSON (πεδία `accounts`, `data` και `programIdIndex`).

##### !! transactionDetails

!type string  
!values full accounts signatures none  
!default full

επίπεδο λεπτομέρειας συναλλαγής προς επιστροφή

- Εάν ζητηθούν `accounts`, οι λεπτομέρειες συναλλαγής περιλαμβάνουν μόνο
  υπογραφές και μια σχολιασμένη λίστα λογαριασμών σε κάθε συναλλαγή.
- Τα μεταδεδομένα συναλλαγής περιορίζονται μόνο σε: fee, err, pre_balances,
  post_balances, pre_token_balances και post_token_balances.

##### !! maxSupportedTransactionVersion

!type number
!values 0
!default 0

Προς το παρόν, η μόνη έγκυρη τιμή για αυτήν την παράμετρο είναι `0`. Ορίζοντάς
την σε `0` σας επιτρέπει να ανακτήσετε όλες τις συναλλαγές, συμπεριλαμβανομένων
τόσο των συναλλαγών με έκδοση (Versioned) όσο και των παλαιού τύπου (legacy)
συναλλαγών.

Αυτή η παράμετρος καθορίζει τη μέγιστη έκδοση συναλλαγής που θα επιστραφεί στην
απάντηση. Εάν ζητήσετε μια συναλλαγή με υψηλότερη έκδοση από αυτήν την τιμή, θα
επιστραφεί σφάλμα. Εάν παραλείψετε αυτήν την παράμετρο, θα επιστραφούν μόνο
παλαιού τύπου συναλλαγές—οποιαδήποτε συναλλαγή με έκδοση θα οδηγήσει σε σφάλμα.

##### !! showRewards

!type bool

εάν θα συμπληρωθεί ο πίνακας `rewards`. Εάν η παράμετρος δεν παρέχεται, η
προεπιλογή περιλαμβάνει ανταμοιβές.

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

αναγνωριστικό συνδρομής (απαιτείται για την κατάργηση της εγγραφής)

</APIMethod>
