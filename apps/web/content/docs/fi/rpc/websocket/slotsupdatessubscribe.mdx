h1: slotsUpdatesSubscribe RPC-metodi
---

Tilaa ilmoituksia validatorilta erilaisista päivityksistä jokaisessa slotissa

<Callout type="warn">
  Tämä tilaus on epävakaa. Tämän tilauksen muoto saattaa muuttua
  tulevaisuudessa, eikä sitä välttämättä aina tueta.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "slotsUpdatesSubscribe"
}
```

### !params

**Ei mitään**

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

Tilauksen tunnus (tarvitaan tilauksen peruuttamiseen)

</APIMethod>

### Ilmoitusmuoto

Ilmoitus on objekti, jossa on seuraavat kentät:

- `err: <string|undefined>` - Virheviesti. Näkyy vain, jos päivityksen tyyppi on
  "dead".
- `parent: <u64|undefined>` - Vanhempi slot. Näkyy vain, jos päivityksen tyyppi
  on "createdBank".
- `slot: <u64>` - Juuri päivitetty slot
- `stats: <object|undefined>` - Virheviesti. Näkyy vain, jos päivityksen tyyppi
  on "frozen". Objekti, jossa on seuraavat kentät:
  - `maxTransactionsPerEntry: <u64>`,
  - `numFailedTransactions: <u64>`,
  - `numSuccessfulTransactions: <u64>`,
  - `numTransactionEntries: <u64>`,
- `timestamp: <i64>` - Päivityksen Unix-aikaleima millisekunteina
- `type: <string>` - Päivityksen tyyppi, yksi seuraavista:
  - "firstShredReceived"
  - "completed"
  - "createdBank"
  - "frozen"
  - "dead"
  - "optimisticConfirmation"
  - "root"

```shell
{
  "jsonrpc": "2.0",
  "method": "slotsUpdatesNotification",
  "params": {
    "result": {
      "parent": 75,
      "slot": 76,
      "timestamp": 1625081266243,
      "type": "optimisticConfirmation"
    },
    "subscription": 0
  }
}
```
