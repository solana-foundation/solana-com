Tämä osio käy läpi yleisiä Anchor CLI -komentoja, joilla pääset alkuun.
Lisätietoja Anchor CLI:stä löydät
[Anchor-dokumentaatiosta](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Projektin alustaminen

Luodaksesi uuden Anchor-projektin, suorita seuraava komento:

```terminal
$ anchor init <project-name>
```

Esimerkiksi luodaksesi projektin nimeltä `my-project`, suorita:

```terminal
$ anchor init my-project
```

Tämä komento luo uuden hakemiston projektin nimellä ja alustaa uuden
Anchor-projektin, joka sisältää perus-Rust-ohjelman ja TypeScript-testauspohjan.

Siirry projektin hakemistoon:

```terminal
$ cd <project-name>
```

Tutustu Anchor-projektin
[tiedostorakenteeseen](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure).

</Step>

<Step>
### Ohjelman kääntäminen

Kääntääksesi projektisi, suorita seuraava komento:

```terminal
$ anchor build
```

Löydät käännetyn ohjelman `/target/deploy` hakemistosta.

<Callout type="warn">

Kun suoritat komentoa `anchor build`, jos kohtaat seuraavia virheitä:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Kokeile näitä ratkaisuja:

1. Pakota asennus seuraavalla komennolla:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Jos edellinen ratkaisu ei toimi, tyhjennä Solana-välimuisti:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Voit korjata tämän muuttamalla `Cargo.lock` tiedoston version-kentän arvosta 4 arvoon 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Katso lisätietoja
[tästä ongelmasta](https://github.com/solana-foundation/anchor/issues/3392).

</Accordion>

</Accordions>

Kun olet soveltanut edellisen ratkaisun, yritä suorittaa _shell`anchor build`_
uudelleen.

</Callout>

</Step>
<Step>

### Ohjelman käyttöönotto

Ottaaksesi projektisi käyttöön, suorita seuraava komento:

```terminal
$ anchor deploy
```

Tämä komento ottaa ohjelmasi käyttöön `cluster`:ssa, joka on määritetty
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml)
tiedostossa.

</Step>
<Step>

### Ohjelman testaus

Testataksesi projektiasi, suorita seuraava komento:

```terminal
$ anchor test
```

<Callout>

Tämä komento kääntää, ottaa käyttöön ja suorittaa testit projektillesi.

Kun käytät `localnet`:a `cluster`:na `Anchor.toml`:ssa, Anchor käynnistää
automaattisesti paikallisen validatorin, ottaa ohjelmasi käyttöön, suorittaa
testit ja sitten pysäyttää validatorin.

</Callout>

<Callout type="warn">

Kun suoritat komentoa _shell`anchor test`_ uuden Anchor-projektin luomisen
jälkeen Linuxissa tai WSL:ssä, saatat kohdata seuraavia virheitä, jos sinulla ei
ole Node.js:ää tai Yarnia asennettuna:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
