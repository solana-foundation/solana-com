Tämä osio tarjoaa yleisiä komentoja ja esimerkkejä, jotka auttavat sinua
pääsemään alkuun [Anchor CLI:n](https://www.anchor-lang.com/docs) käytössä.

<Steps>
<Step>

### Projektin alustaminen

Luo uusi Anchor-projekti suorittamalla alla näytetty komento. Se luo uuden
hakemiston projektin nimellä ja käyttää sitä uuden Anchor- projektin
alustamiseen.

```terminal
$ anchor init <project-name>
```

Esimerkiksi alla oleva komento luo projektin nimeltä `my-project`.
`my-project`-hakemisto sisältää perus-Rust-ohjelman ja TypeScript-testauksen
mallin.

```terminal
$ anchor init my-project
```

Siirry sitten projektihakemistoon:

```terminal
$ cd <project-name>
```

Katso Anchorin
[projektin tiedostorakenne](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure).

</Step>
<Step>

### Ohjelman kääntäminen

Kääntääksesi projektisi, suorita seuraava komento:

```terminal
$ anchor build
```

Löydät käännetyn ohjelman `/target/deploy`-hakemistosta.

<Callout type="warn">

Kun suoritat komentoa `anchor build`, saatat kohdata seuraavia virheitä:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Kokeile näitä ratkaisuja:

1. Pakota asennus käyttämällä seuraavaa komentoa:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Jos edellinen ratkaisu ei toimi, tyhjennä Solana-välimuisti:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Voit korjata tämän muuttamalla `Cargo.lock`-tiedoston version-kentän arvosta 4 arvoon 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Katso lisätietoja
[tästä ongelmasta](https://github.com/solana-foundation/anchor/issues/3392).

</Accordion>

</Accordions>

Kun olet soveltanut edellä mainittua ratkaisua, yritä suorittaa
_shell`anchor build`_ uudelleen.

</Callout>

</Step>
<Step>

### Ohjelman käyttöönotto

Ottaaksesi projektisi käyttöön, suorita seuraava komento:

```terminal
$ anchor deploy
```

Tämä komento ottaa ohjelmasi käyttöön `cluster`ssa, joka on määritetty
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml)
tiedostossa.

</Step>
<Step>

### Ohjelman testaaminen

Testataksesi projektiasi, suorita seuraava komento:

```terminal
$ anchor test
```

Tämä komento kääntää, ottaa käyttöön ja suorittaa testit projektillesi.

<Callout>
  Kun käytät `localnet` klusteria, Anchor käynnistää automaattisesti paikallisen
  validaattorin, ottaa ohjelman käyttöön, suorittaa testit ja sitten pysäyttää
  validaattorin.
</Callout>

<Callout type="warn">

Jompikumpi seuraavista virheistä voi osoittaa, että sinulla ei ole Node.js tai
Yarn asennettuna:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
