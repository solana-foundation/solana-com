Una guida per configurare il tuo ambiente di sviluppo locale Solana. Impara come
installare Rust, la CLI di Solana e il framework Anchor su Windows (WSL), Linux
e Mac. Usa questa guida se preferisci installare ogni dipendenza
individualmente, o se l'[installazione rapida](/docs/intro/installation/)
fallisce per qualsiasi motivo.

### Prerequisiti

<Accordions>
<Accordion title="Windows">

Per sviluppare programmi Solana su Windows **devi** utilizzare
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Dopo aver installato WSL, tutte le altre dipendenze saranno installate
tramite il terminale Linux.

Per installare WSL, esegui il seguente comando in Windows PowerShell:

```terminal
$ wsl --install
```

Il processo di installazione ti chiederà di creare un account utente
predefinito.

![Installazione WSL](/assets/docs/intro/installation/wsl-install.png)

Per impostazione predefinita, WSL installa Ubuntu. Puoi aprire un terminale
Linux cercando "Ubuntu" nella barra di ricerca.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Se il tuo terminale Ubuntu appare come nell'immagine sottostante, potresti
riscontrare un problema dove `ctrl + v` (scorciatoia da tastiera per incollare)
non funziona nel terminale.

![Terminale Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Se riscontri questo problema, apri Windows Terminal cercando "Terminal" nella
barra di ricerca.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Successivamente, chiudi Windows Terminal e riapri un terminale Linux cercando
Ubuntu di nuovo. Il terminale dovrebbe ora apparire come nell'immagine
sottostante, dove `ctrl + v` (scorciatoia da tastiera per incollare) funziona.

![Terminale Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Se utilizzi VS Code,
l'[estensione WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) ti
permette di utilizzare WSL e VS Code insieme.

![Configurazione WSL in VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Vedrai quanto segue nella barra di stato di VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Una volta configurato WSL, sei pronto per:

- Completare l'[installazione rapida](/docs/intro/installation) utilizzando il
  terminale Ubuntu (Linux)
- Installare [Rust](#install-rust), la [CLI di Solana](#install-solana-cli) e la
  [CLI di Anchor](#install-anchor-cli) individualmente, come descritto di
  seguito

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Se utilizzi Debian o distribuzioni derivate da Debian come Ubuntu o Mint, segui i passaggi seguenti:

1. Aggiorna gli elenchi dei pacchetti per assicurarti che tutti i pacchetti
   siano aggiornati:

```terminal
$ sudo apt-get update
```

2. Installa le seguenti dipendenze:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Se riscontri il seguente errore durante l'installazione di `protobuf-compiler`, assicurati
di eseguire prima _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Per le derivate RHEL come Fedora o Red Hat Linux, segui i seguenti passaggi.

Aggiorna gli elenchi dei pacchetti per ottenere le versioni più recenti dei
pacchetti eseguendo il comando.

```terminal
$ sudo dnf upgrade --refresh
```

Successivamente, installa le seguenti dipendenze:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

  </Tab>
</Tabs>

### Installa Rust

Gli sviluppatori creano programmi Solana utilizzando il linguaggio di
programmazione [Rust](https://www.rust-lang.org/).

1. Installa Rust utilizzando [rustup](https://www.rust-lang.org/tools/install)
   inserendo il seguente comando nel tuo terminale:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Dopo un'installazione riuscita vedrai il seguente messaggio:

<Accordions>
<Accordion title="Messaggio di installazione Rust riuscita">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Ricarica la variabile d'ambiente PATH per includere la directory bin di
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Verifica che l'installazione sia avvenuta con successo.

```terminal
$ rustc --version
```

Vedrai un output simile al seguente:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Installa Solana CLI

La Solana CLI fornisce tutti gli strumenti necessari per sviluppare e
distribuire programmi Solana.

1. Installa la suite di strumenti Solana CLI utilizzando il comando di
   installazione ufficiale:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Puoi sostituire `stable` con il tag di rilascio corrispondente alla versione
  del software che desideri (ad esempio `v2.0.3`), o utilizzare uno dei tre nomi
  simbolici dei canali: `stable`, `beta`, o `edge`.
</Callout>

2. Aggiungi una variabile d'ambiente PATH

La prima installazione della Solana CLI potrebbe chiederti di aggiungere una
variabile d'ambiente `PATH`. Per farlo, chiudi e riapri il terminale o esegui il
seguente comando nella tua shell:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Aggiorna la tua variabile d'ambiente PATH

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Se stai utilizzando **Linux** o **WSL**, devi aggiungere il binario della Solana
CLI al tuo PATH affinché il comando sia disponibile nel tuo terminale. Per
farlo, segui i passaggi seguenti:

**a.** Controlla quale shell stai utilizzando:

```terminal
$ echo $SHELL
```

- Se l'output contiene `/bash`, usa `.bashrc`.
- Se l'output contiene `/zsh`, usa `.zshrc`.

**b.** Esegui il comando appropriato, in base alla tua shell.

Per Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Per Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Se utilizzi un Mac con `zsh`, l'esecuzione del comando predefinito `export PATH`
registrato dall'installazione non persiste una volta chiuso il terminale.
Invece, aggiungi il PATH al file di configurazione della shell eseguendo il
seguente comando:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  I passaggi sopra descritti non si applicano agli utenti di `bash`, poiché le
  modifiche alla variabile PATH persisteranno di default.
</Callout>

</Tab>
</Tabs>

4. Riavvia il terminale o esegui il seguente comando per aggiornare la sessione
   del terminale:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Verifica che l'installazione sia riuscita controllando la versione della CLI
   di Solana:

```terminal
$ solana --version
```

Vedrai un output simile al seguente:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Puoi visualizzare tutte le versioni disponibili nel
[repository Github di Agave](https://github.com/anza-xyz/agave/releases). Agave
è il client validator di [Anza](https://www.anza.xyz/), precedentemente noto
come client validator di Solana Labs.

</Callout>

5. Aggiorna la CLI di Solana all'ultima versione, se necessario _(Opzionale)_

```terminal
$ agave-install update
```

</Step>

### Installa Anchor CLI

[Anchor](https://www.anchor-lang.com/) è un framework per lo sviluppo di
programmi Solana. Il framework Anchor sfrutta le macro di Rust per semplificare
il processo di scrittura dei programmi Solana.

#### Prerequisiti

Il file di test predefinito del progetto Anchor (TypeScript) creato con il
comando _shell`anchor init`_ richiede Node.js e Yarn. (Il modello di test Rust è
disponibile utilizzando _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Installazione di Node">

1. Installa Node utilizzando
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) eseguendo il
   seguente comando:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Riavvia il terminale e conferma che il comando `nvm` venga eseguito
   correttamente:

```terminal
$ command -v nvm
```

3. Usa `nvm` per installare Node:

```terminal
$ nvm install node
```

4. Verifica che l'installazione sia riuscita, controllando la versione di Node:

```terminal
$ node --version
```

Vedrai un output simile al seguente:

```
v23.7.0
```

</Accordion>
<Accordion title="Installazione di Yarn">

1. Installa Yarn:

```terminal
$ npm install --global yarn
```

2. Verifica che l'installazione sia riuscita, controllando la versione di Yarn:

```terminal
$ yarn --version
```

Vedrai il seguente output:

```
1.22.1
```

</Accordion>
</Accordions>

#### Installazione

Puoi installare la CLI di Anchor e gli strumenti in due modi:

- Anchor Version Manager (AVM) — Metodo di installazione consigliato
- Senza AVM — Installa direttamente da GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

L'Anchor Version Manager (AVM) ti permette di installare e gestire diverse
versioni di Anchor sul tuo sistema e aggiornare facilmente le versioni di Anchor
in futuro. Per installare Anchor utilizzando AVM, segui i passaggi seguenti:

1. Installa AVM con il seguente comando:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Conferma che AVM sia stato installato correttamente:

```terminal
$ avm --version
```

3. Installa Anchor CLI utilizzando AVM:

Per installare l'ultima versione:

```terminal
$ avm install latest
$ avm use latest
```

Per installare una versione specifica, indica il numero di versione:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Durante l'installazione di Anchor CLI su Linux o WSL, potresti incontrare questo
errore:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Se vedi questo messaggio di errore, segui questi passaggi:

1. Installa le dipendenze elencate nella sezione
   [Linux](/docs/intro/installation#install-dependencies) all'inizio di questa
   pagina.
2. Riprova a installare Anchor CLI.

</Callout>

4. Verifica che l'installazione sia riuscita, controllando la versione di Anchor
   CLI:

```terminal
$ anchor --version
```

Vedrai un output simile al seguente:

```
anchor-cli 0.31.1
```

<Callout type="info">

Non dimenticare di eseguire il comando _shell`avm use`_ per dichiarare la
versione di Anchor CLI da eseguire sul tuo sistema.

- Se hai installato la versione `latest`, esegui _shell`avm use latest`_.
- Se hai installato la versione `0.30.1`, esegui _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Installa una versione specifica della CLI di Anchor con il seguente comando:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Verifica che l'installazione sia riuscita, controllando la versione della CLI
   di Anchor:

```terminal
$ anchor --version
```

Vedrai un output simile al seguente:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>
