Hướng dẫn thiết lập môi trường phát triển Solana trên máy tính của bạn. Tìm hiểu
cách cài đặt Rust, Solana CLI và Anchor Framework trên Windows (WSL), Linux và
Mac. Sử dụng hướng dẫn này nếu bạn muốn cài đặt từng phụ thuộc riêng lẻ, hoặc
nếu [cài đặt nhanh](/docs/intro/installation/) không thành công vì bất kỳ lý do
nào.

### Điều kiện tiên quyết

<Accordions>
<Accordion title="Windows">

Để phát triển các chương trình Solana trên Windows, bạn **phải** sử dụng
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Sau khi cài đặt WSL, tất cả các phụ thuộc khác sẽ được cài đặt thông qua
terminal Linux.

Để cài đặt WSL, chạy lệnh sau trong Windows PowerShell:

```terminal
$ wsl --install
```

Quá trình cài đặt sẽ yêu cầu bạn tạo một tài khoản người dùng mặc định.

![Cài đặt WSL](/assets/docs/intro/installation/wsl-install.png)

Theo mặc định, WSL cài đặt Ubuntu. Bạn có thể mở terminal Linux bằng cách tìm
kiếm "Ubuntu" trong thanh Tìm kiếm.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Nếu terminal Ubuntu của bạn trông giống như hình ảnh dưới đây, bạn có thể gặp
vấn đề khi `ctrl + v` (phím tắt dán) không hoạt động trong terminal.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Nếu bạn gặp vấn đề này, hãy mở Windows Terminal bằng cách tìm kiếm "Terminal"
trong thanh Tìm kiếm.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Tiếp theo, đóng Windows Terminal và mở lại terminal Linux bằng cách tìm kiếm
Ubuntu một lần nữa. Terminal bây giờ sẽ trông giống như hình ảnh dưới đây, nơi
`ctrl + v` (phím tắt dán) hoạt động.

![Terminal Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Nếu bạn sử dụng VS Code,
[tiện ích mở rộng WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
cho phép bạn sử dụng WSL và VS Code cùng nhau.

![Thiết lập WSL trong VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Bạn sẽ thấy nội dung sau trong thanh trạng thái của VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Sau khi thiết lập WSL, bạn đã sẵn sàng để:

- Hoàn thành [cài đặt nhanh](/docs/intro/installation) bằng terminal Ubuntu
  (Linux)
- Cài đặt [Rust](#install-rust), [Solana CLI](#install-solana-cli), và
  [Anchor CLI](#install-anchor-cli) riêng lẻ, như được mô tả bên dưới

</Accordion>

<Tab value=".deb" >
Nếu bạn đang sử dụng Debian hoặc các hệ điều hành dựa trên Debian như Ubuntu hoặc Mint, hãy làm theo các bước dưới đây:

1. Cập nhật danh sách gói để đảm bảo tất cả các gói đều được cập nhật:

```terminal
$ sudo apt-get update
```

2. Cài đặt các phụ thuộc sau:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Nếu bạn gặp lỗi sau khi cài đặt `protobuf-compiler`, hãy
đảm bảo bạn chạy lệnh _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Đối với các hệ điều hành dựa trên RHEL như Fedora hoặc Red Hat Linux, hãy làm theo các bước sau.

Cập nhật danh sách gói để nhận các phiên bản mới hơn của các gói bằng cách chạy
lệnh.

```terminal
$ sudo dnf upgrade --refresh
```

Tiếp theo, cài đặt các phụ thuộc sau:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### Cài đặt Rust

Các nhà phát triển xây dựng chương trình Solana bằng ngôn ngữ lập trình
[Rust](https://www.rust-lang.org/).

1. Cài đặt Rust bằng cách sử dụng
   [rustup](https://www.rust-lang.org/tools/install) bằng cách nhập lệnh sau vào
   terminal của bạn:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Sau khi cài đặt thành công, bạn sẽ thấy thông báo sau:

<Accordions>
<Accordion title="Thông báo cài đặt Rust thành công">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Tải lại biến môi trường PATH của bạn để bao gồm thư mục bin của
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Xác minh rằng việc cài đặt đã thành công.

```terminal
$ rustc --version
```

Bạn sẽ thấy kết quả như sau:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Cài đặt Solana CLI

Solana CLI cung cấp tất cả các công cụ cần thiết để xây dựng và triển khai các
chương trình Solana.

1. Cài đặt bộ công cụ Solana CLI bằng cách sử dụng lệnh cài đặt chính thức:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Bạn có thể thay thế `stable` bằng thẻ phiên bản phù hợp với phiên bản phần mềm
  của bản phát hành mong muốn (ví dụ: `v2.0.3`), hoặc sử dụng một trong ba tên
  kênh tượng trưng: `stable`, `beta`, hoặc `edge`.
</Callout>

2. Thêm biến môi trường PATH

Lần đầu cài đặt Solana CLI, có thể bạn sẽ được nhắc thêm biến môi trường `PATH`.
Để làm điều này, hãy đóng và mở lại terminal của bạn hoặc chạy lệnh sau trong
shell của bạn:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Cập nhật biến môi trường PATH của bạn

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Nếu bạn đang sử dụng **Linux** hoặc **WSL**, bạn phải thêm Solana CLI binary vào
PATH của bạn để lệnh có sẵn trong terminal. Để làm điều này, hãy làm theo các
bước dưới đây:

**a.** Kiểm tra shell bạn đang sử dụng:

```terminal
$ echo $SHELL
```

- Nếu đầu ra chứa `/bash`, hãy sử dụng `.bashrc`.
- Nếu đầu ra chứa `/zsh`, hãy sử dụng `.zshrc`.

**b.** Chạy lệnh thích hợp, dựa trên shell của bạn.

Đối với Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Đối với Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Nếu bạn sử dụng Mac với `zsh`, việc chạy lệnh `export PATH` mặc định được ghi
lại từ quá trình cài đặt sẽ không duy trì khi bạn đóng terminal. Thay vào đó,
hãy thêm PATH vào tệp cấu hình shell của bạn bằng cách chạy lệnh sau:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Các bước trên không áp dụng cho người dùng `bash`, vì các thay đổi đối với
  biến PATH sẽ được duy trì theo mặc định.
</Callout>

</Tab>
</Tabs>

4. Khởi động lại terminal của bạn hoặc chạy lệnh sau để làm mới phiên terminal:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Xác minh rằng việc cài đặt đã thành công bằng cách kiểm tra phiên bản Solana
   CLI:

```terminal
$ solana --version
```

Bạn sẽ thấy kết quả như sau:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Bạn có thể xem tất cả các phiên bản có sẵn trên
[Agave Github repo](https://github.com/anza-xyz/agave/releases). Agave là
validator client từ [Anza](https://www.anza.xyz/), trước đây được biết đến là
Solana Labs validator client.

</Callout>

5. Cập nhật Solana CLI lên phiên bản mới nhất, khi cần _(Tùy chọn)_

```terminal
$ agave-install update
```

</Step>

### Cài đặt Anchor CLI

[Anchor](https://www.anchor-lang.com/) là một framework để phát triển các chương
trình Solana. Framework Anchor sử dụng các macro Rust để đơn giản hóa quá trình
viết các chương trình Solana.

#### Điều kiện tiên quyết

Tệp kiểm thử mặc định của dự án Anchor (TypeScript) được tạo bằng lệnh
_shell`anchor init`_ yêu cầu Node.js và Yarn. (Mẫu kiểm thử Rust có sẵn bằng
cách sử dụng _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Cài đặt Node">

1. Cài đặt Node bằng cách sử dụng
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) bằng cách chạy
   lệnh sau:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Khởi động lại terminal và xác nhận rằng lệnh `nvm` chạy thành công:

```terminal
$ command -v nvm
```

3. Sử dụng `nvm` để cài đặt Node:

```terminal
$ nvm install node
```

4. Xác minh rằng việc cài đặt đã thành công bằng cách kiểm tra phiên bản Node:

```terminal
$ node --version
```

Bạn sẽ thấy kết quả như sau:

```
v23.7.0
```

</Accordion>
<Accordion title="Cài đặt Yarn">

1. Cài đặt Yarn:

```terminal
$ npm install --global yarn
```

2. Xác minh rằng việc cài đặt đã thành công bằng cách kiểm tra phiên bản Yarn:

```terminal
$ yarn --version
```

Bạn sẽ thấy kết quả sau:

```
1.22.1
```

</Accordion>
</Accordions>

#### Cài đặt

Bạn có thể cài đặt Anchor CLI và các công cụ theo hai cách:

- Anchor Version Manager (AVM) — Phương pháp cài đặt được khuyến nghị
- Không dùng AVM — Cài đặt trực tiếp từ GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Anchor Version Manager (AVM) cho phép bạn cài đặt và quản lý các phiên bản
Anchor khác nhau trên hệ thống của bạn và dễ dàng cập nhật phiên bản Anchor
trong tương lai. Để cài đặt Anchor sử dụng AVM, hãy làm theo các bước sau:

1. Cài đặt AVM với lệnh sau:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Xác nhận rằng AVM đã được cài đặt thành công:

```terminal
$ avm --version
```

3. Cài đặt Anchor CLI sử dụng AVM:

Để cài đặt phiên bản mới nhất:

```terminal
$ avm install latest
$ avm use latest
```

Để cài đặt một phiên bản cụ thể, hãy chỉ định số phiên bản:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Khi cài đặt Anchor CLI trên Linux hoặc WSL, bạn có thể gặp lỗi này:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Nếu bạn thấy thông báo lỗi này, hãy làm theo các bước sau:

1. Cài đặt các phụ thuộc được liệt kê trong phần
   [Linux](/docs/intro/installation#install-dependencies) ở đầu trang này.
2. Thử cài đặt lại Anchor CLI.

</Callout>

4. Xác minh rằng việc cài đặt đã thành công bằng cách kiểm tra phiên bản Anchor
   CLI:

```terminal
$ anchor --version
```

Bạn sẽ thấy kết quả như sau:

```
anchor-cli 0.31.1
```

<Callout type="info">

Đừng quên chạy lệnh _shell`avm use`_ để khai báo phiên bản Anchor CLI sẽ chạy
trên hệ thống của bạn.

- Nếu bạn đã cài đặt phiên bản `latest`, hãy chạy _shell`avm use latest`_.
- Nếu bạn đã cài đặt phiên bản `0.30.1`, hãy chạy _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Cài đặt phiên bản cụ thể của Anchor CLI bằng lệnh sau:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Xác minh rằng việc cài đặt đã thành công bằng cách kiểm tra phiên bản Anchor
   CLI:

```terminal
$ anchor --version
```

Bạn sẽ thấy kết quả như sau:

```
anchor-cli 0.31.1
```

</Tab>
