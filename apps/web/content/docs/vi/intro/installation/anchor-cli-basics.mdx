description:
  Phần này hướng dẫn một số lệnh Anchor CLI phổ biến để giúp bạn bắt đầu.
---

Phần này cung cấp một số lệnh và ví dụ phổ biến để giúp bạn bắt đầu sử dụng
[Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Khởi tạo dự án

Tạo một dự án Anchor mới bằng cách chạy lệnh được hiển thị bên dưới. Lệnh này sẽ
tạo một thư mục mới với tên dự án và sử dụng nó để khởi tạo một dự án Anchor
mới.

```terminal
$ anchor init <project-name>
```

Ví dụ, lệnh bên dưới sẽ tạo một dự án có tên `my-project`. Thư mục `my-project`
sẽ chứa một chương trình Rust cơ bản và mẫu kiểm thử TypeScript.

```terminal
$ anchor init my-project
```

Sau đó di chuyển đến thư mục dự án:

```terminal
$ cd <project-name>
```

Xem
[cấu trúc tệp dự án](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
của Anchor.

</Step>
<Step>

### Biên dịch chương trình

Để biên dịch dự án của bạn, hãy chạy lệnh sau:

```terminal
$ anchor build
```

Bạn có thể tìm thấy chương trình đã được biên dịch trong thư mục
`/target/deploy`.

<Callout type="warn">

Khi chạy `anchor build`, nếu bạn gặp các lỗi sau:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Hãy thử các giải pháp sau:

1. Cài đặt bắt buộc bằng lệnh sau:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Nếu giải pháp trên không hiệu quả, hãy xóa bộ nhớ đệm của Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Bạn có thể sửa lỗi này bằng cách thay đổi trường version của tệp `Cargo.lock` từ 4 thành 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Xem [vấn đề này](https://github.com/solana-foundation/anchor/issues/3392) để
biết thêm thông tin.

</Accordion>

</Accordions>

Sau khi áp dụng giải pháp trên, hãy thử chạy lại _shell`anchor build`_.

</Callout>

</Step>
<Step>

### Triển khai chương trình

Để triển khai dự án của bạn, chạy lệnh sau:

```terminal
$ anchor deploy
```

Lệnh này triển khai chương trình của bạn đến `cluster` được chỉ định trong tệp
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Kiểm thử chương trình

Để kiểm thử dự án của bạn, chạy lệnh sau:

```terminal
$ anchor test
```

Lệnh này sẽ biên dịch, triển khai và chạy các bài kiểm thử cho dự án của bạn.

<Callout>
  Khi sử dụng cụm `localnet`, Anchor tự động khởi động validator cục bộ, triển
  khai chương trình, chạy kiểm thử, sau đó dừng validator.
</Callout>

<Callout type="warn">

Một trong các lỗi sau có thể cho thấy bạn chưa cài đặt Node.js hoặc Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
