Anchor 프레임워크를 사용하여 첫 번째 [프로그램](/docs/core/programs)을 Solana
블록체인에 빌드, 배포 및 테스트해 보세요. 이 가이드는 Solana Playground에 더
익숙해지고 Program Derived Address(PDA) 및 Cross Program Invocation(CPI)을
사용할 준비를 하는 데 도움이 됩니다.

<Steps>

<Step>

### Anchor 프로젝트 생성하기

1. 웹 브라우저에서 [Solana Playground](https://beta.solpg.io)를 엽니다
2. 왼쪽 패널에서 **+ Create a new project**를 클릭합니다
3. **Project name**을 입력합니다
4. 프레임워크로 **Anchor (Rust)**를 선택합니다
5. **Create**를 클릭합니다

![새 프로젝트](/assets/docs/intro/quickstart/pg-new-project.gif)

`src/lib.rs` 소스 파일이 열립니다. 이 파일에는 `Initialize`라는 하나의 구조체가
포함되어 있으며, 다음과 같은 작업을 수행합니다:

- *rs`data: u64`*라는 필수 매개변수를 받습니다
- 새 계정을 생성합니다
- _rs`data: u64`_ 값을 새 계정에 저장합니다
- 트랜잭션의 프로그램 로그에 메시지를 추가합니다

<WithNotes>

```rust title="lib.rs"
use anchor_lang::prelude::*;

// This is your program's public key and it will update
// automatically when you build the project.
// !tooltip[/declare_id/] declare_id
declare_id!("11111111111111111111111111111111");

// !tooltip[/program/] program
#[program]
mod hello_anchor {
    use super::*;
    // !tooltip[/data: u64/] parameter
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !tooltip[/ctx.accounts.new_account.data = data/] initialize
        ctx.accounts.new_account.data = data;
        // !tooltip[/msg!/] msg
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}

// !tooltip[/Accounts/] accounts
#[derive(Accounts)]
pub struct Initialize<'info> {
    // We must specify the space in order to initialize an account.
    // First 8 bytes are default account discriminator,
    // next 8 bytes come from NewAccount.data being type u64.
    // (u64 = 64 bits unsigned integer = 8 bytes)
    // !tooltip[/account/] constraints
    #[account(
      // !tooltip[/init/] init
      init,
      // !tooltip[/payer/] payer
      payer = signer,
      // !tooltip[/space/] space
      space = 8 + 8
    )]
    pub new_account: Account<'info, NewAccount>,
    // !tooltip[/account/] constraints
    #[account(mut)]
    pub signer: Signer<'info>,
    pub system_program: Program<'info, System>,
}

// !tooltip[/account/] account
#[account]
pub struct NewAccount {
    data: u64
}
```

### !declare_id

프로그램의 온체인 주소입니다. 프로젝트를 빌드할 때 업데이트됩니다

### !program

프로그램의 명령어를 포함하는 모듈입니다

### !accounts

프로그램의 계정을 포함하는 구조체에 주석을 답니다

### !account

이 프로그램이 생성하고 소유할 수 있는 계정 데이터 유형을 나타내는 구조체입니다

### !constraints

계정 제약 조건입니다. 예를 들어, 프로그램 보안을 강화하는 일반적인 검사 또는
기능입니다

### !parameter

명령어가 호출될 때 전달되는 값입니다

### !initialize

`new_account` 계정의 `data` 필드에 접근하여 `data: u64` 값으로 설정합니다

### !msg

_rs`msg!`_ 매크로는 트랜잭션의 프로그램 로그에 메시지를 기록합니다

### !init

System Program을 호출하여 새 계정을 생성하고 이 프로그램을 소유자로 만듭니다.
프로그램 소유자로서, 이 프로그램은 계정 데이터에 쓸 수 있습니다

### !payer

새 계정 비용을 지불하는 계정

### !space

계정에 할당할 바이트 수

</WithNotes>

<Accordion title="설명">

<WithMentions>

- [`declare_id!`](mention:one) 매크로는 프로그램의 온체인 주소를 지정합니다.
  Solana Playground는 프로그램을 빌드할 때 이 주소를 자동으로 업데이트합니다.

```rs
// !mention one
declare_id!("11111111111111111111111111111111");
```

</WithMentions>

<WithMentions>

- [`#[program]`](mention:one) 속성은 프로그램의 명령어를 나타내는 함수가 포함된
  모듈에 주석을 답니다.

```rs
// !mention one
#[program]
mod hello_anchor {
    use super::*;
    // !mention two
    pub fn initialize(ctx: Context<Initialize>, data: u64) -> Result<()> {
        // !mention three
        ctx.accounts.new_account.data = data;
        msg!("Changed data to: {}!", data); // Message will show up in the tx logs
        Ok(())
    }
}
```

이 예제에서 [`initialize`](mention:two) 명령어는 두 가지 매개변수를 받습니다:

- `ctx: Context<Initialize>`: `Initialize` 구조체에 지정된 대로 이 명령어에
  필요한 계정을 함수에 전달합니다.
- `data: u64`: 명령어를 호출할 때 제공해야 하는 사용자 정의 매개변수입니다.

함수 본문은
[제공된 `data` 인수로 `new_account`의 `data` 필드를 설정](mention:three)한 다음
프로그램 로그에 메시지를 출력합니다.

</WithMentions>

<WithMentions>

- [`#[derive(Accounts)]`](mention:one) 속성은 특정 명령어에 필요한 계정을
  지정하는 구조체를 정의합니다. 각 필드는 별도의 계정을 나타냅니다.

  Anchor는 필드 유형(예: `Signer<'info>`)과 제약 조건(예: `#[account(mut)]`)을
  통해 계정 검증과 관련된 일반적인 보안 검사를 자동으로 처리합니다.

  ```rs
  // !mention one
  #[derive(Accounts)]
  pub struct Initialize<'info> {
      #[account(init, payer = signer, space = 8 + 8)]
      pub new_account: Account<'info, NewAccount>,
      #[account(mut)]
      pub signer: Signer<'info>,
      pub system_program: Program<'info, System>,
  }
  ```

</WithMentions>

<WithMentions>

- [`#[account]`](mention:one) 속성은 이 프로그램이 생성하고 소유하는 계정의
  데이터 유형을 나타내는 구조체를 정의합니다.

```rs
// !mention one
#[account]
pub struct NewAccount {
  data: u64
}
```

</WithMentions>

</Accordion>
</Accordions>

<Step>

### 프로그램 빌드 및 배포하기

1. 프로그램 코드 컴파일하기

```terminal
$ build
Building...
Build successful. Completed in 1.46s.
```

<Callout>
  _rs`declare_id!()`_에 정의된 [프로그램
  ID](/docs/intro/quick-start/deploying-programs)가 이제 업데이트됩니다.
</Callout>

2. 지갑에 SOL 추가하기 _(선택 사항)_

프로그램을 배포하려면 프로그램의 온체인 계정에 SOL이 필요합니다. 필요한 경우,
3단계로 진행하기 전에
[지갑에 SOL 추가하기](/docs/intro/quick-start#add-sol-to-your-wallet)를
수행하세요.

3. 네트워크에 프로그램 배포하기

```terminal
$ deploy
Deploying... This could take a while depending on the program size and network conditions.
Warning: 1 transaction not confirmed, retrying...
Deployment successful. Completed in 19s.
```

<Callout type="info">
  배포되면 프로그램의 바이트코드는 실행 가능한 program account에 저장됩니다.
</Callout>

**참고:** 원하는 경우, 아래 스크린샷과 같이 왼쪽 패널의 **Build** 및 **Deploy**
버튼을 사용할 수 있습니다.

![빌드 및 배포](/assets/docs/intro/quickstart/pg-build-deploy.png)

</Step>

### 프로그램 테스트하기

<WithMentions>

1. `tests/anchor.test.ts` 테스트 파일을 엽니다. 이 파일은 클라이언트에서
   프로그램의 [`initialize`](mention:one) 구조체를 호출하는 방법을 보여줍니다.

```ts title="anchor.test.ts"
// No imports needed: web3, anchor, pg and more are globally available

describe("Test", () => {
  it("initialize", async () => {
    // Generate keypair for the new account
    const newAccountKp = new web3.Keypair();

    // Send transaction
    const data = new BN(42);
    const txHash = await pg.program.methods
      // !mention one
      .initialize(data)
      .accounts({
        newAccount: newAccountKp.publicKey,
        signer: pg.wallet.publicKey,
        systemProgram: web3.SystemProgram.programId
      })
      .signers([newAccountKp])
      .rpc();
    console.log(`Use 'solana confirm -v ${txHash}' to see the logs`);

    // Confirm transaction
    await pg.connection.confirmTransaction(txHash);

    // Fetch the created account
    const newAccount = await pg.program.account.newAccount.fetch(
      newAccountKp.publicKey
    );

    console.log("On-chain data is:", newAccount.data.toString());

    // Check whether the data on-chain is equal to local 'data'
    assert(data.eq(newAccount.data));
  });
});
```

</WithMentions>

<Step>

### 프로그램 종료하기

온체인 프로그램에 할당된 SOL을 회수하기 위해 프로그램을 종료하세요.
_rs`declare_id!()`_ 매크로에 저장된 프로그램 주소를 사용하여
`solana program close [ProgramID]`를 실행하세요.

```terminal
$ solana program close 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r
Closed Program Id 2VvQ11q8xrn5tkPNyeraRsPaATdiPx8weLAD8aD4dn2r, 2.79511512 SOL reclaimed
```

<Callout type="warn">
  프로그램은 업그레이드 권한이 있는 계정에 의해서만 종료되거나 업데이트될 수
  있습니다. 프로그램이 배포될 때 Playground 지갑이 자동으로 각 프로그램의
  업그레이드 권한을 갖게 됩니다. 업그레이드 권한이 취소되면 프로그램은 변경
  불가능하게 됩니다.
</Callout>

축하합니다! Anchor를 사용하여 첫 번째 Solana 프로그램을 빌드하고 배포했습니다.

</Step>
