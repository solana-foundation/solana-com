로컬 Solana 개발 환경 설정 가이드입니다. Windows(WSL), Linux 및 Mac에서 Rust,
Solana CLI 및 Anchor 프레임워크를 설치하는 방법을 알아보세요. 각 의존성을
개별적으로 설치하려는 경우나 [빠른 설치](/docs/intro/installation/)가 어떤
이유로든 실패한 경우 이 가이드를 사용하세요.

### 사전 요구사항

<Accordions>
<Accordion title="Windows">

Windows에서 Solana 프로그램을 개발하려면 **반드시**
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)(WSL)를
사용해야 합니다. WSL을 설치한 후, 다른 모든 의존성은 Linux 터미널을 통해
설치됩니다.

WSL을 설치하려면 Windows PowerShell에서 다음 명령어를 실행하세요:

```terminal
$ wsl --install
```

설치 과정에서 기본 사용자 계정을 생성하라는 메시지가 표시됩니다.

![WSL 설치](/assets/docs/intro/installation/wsl-install.png)

기본적으로 WSL은 Ubuntu를 설치합니다. 검색 창에서 "Ubuntu"를 검색하여 Linux
터미널을 열 수 있습니다.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Ubuntu 터미널이 아래 이미지와 같이 보인다면, 터미널에서 `ctrl + v` (붙여넣기
단축키)가 작동하지 않는 문제가 발생할 수 있습니다.

![Ubuntu 터미널](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

이 문제가 발생하면 검색 창에서 "Terminal"을 검색하여 Windows Terminal을 엽니다.

![Windows 터미널](/assets/docs/intro/installation/wsl-windows-terminal.png)

다음으로, Windows Terminal을 닫고 다시 Ubuntu를 검색하여 Linux 터미널을 엽니다.
이제 터미널은 아래 이미지와 같이 보이며, `ctrl + v` (붙여넣기 단축키)가
작동합니다.

![Ubuntu 터미널](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

VS Code를 사용한다면,
[WSL 확장 프로그램](https://code.visualstudio.com/docs/remote/wsl-tutorial)을
통해 WSL과 VS Code를 함께 사용할 수 있습니다.

![VS Code에서 WSL 설정](/assets/docs/intro/installation/wsl-vscode.png)

VS Code 상태 표시줄에 다음과 같이 표시됩니다:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

WSL 설정이 완료되면 다음 작업을 수행할 준비가 된 것입니다:

- Ubuntu(Linux) 터미널을 사용하여 [빠른 설치](/docs/intro/installation) 완료하기
- 아래 설명된 대로 [Rust](#install-rust), [Solana CLI](#install-solana-cli) 및
  [Anchor CLI](#install-anchor-cli)를 개별적으로 설치하기

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Debian이나 Ubuntu 또는 Mint와 같은 Debian 기반 배포판을 사용하는 경우 아래 단계를 따르세요:

1. 모든 패키지가 최신 상태인지 확인하기 위해 패키지 목록을 업데이트합니다:

```terminal
$ sudo apt-get update
```

2. 다음 종속성을 설치합니다:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
`protobuf-compiler` 설치 중에 오류가 발생하면
먼저 _shell`sudo apt-get update`를 실행하세요

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Fedora나 Red Hat Linux와 같은 RHEL 파생 배포판의 경우 다음 단계를 따르세요.

다음 명령을 실행하여 최신 버전의 패키지를 얻기 위해 패키지 목록을
업데이트합니다.

```terminal
$ sudo dnf upgrade --refresh
```

다음으로, 아래 종속성을 설치합니다:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

  </Tab>
</Tabs>

### Rust 설치하기

개발자들은 [Rust](https://www.rust-lang.org/) 프로그래밍 언어를 사용하여 Solana
프로그램을 구축합니다.

1. 터미널에 다음 명령어를 입력하여
   [rustup](https://www.rust-lang.org/tools/install)을 사용해 Rust를 설치하세요:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

설치가 성공적으로 완료되면 다음과 같은 메시지가 표시됩니다:

<Accordions>
<Accordion title="Rust 설치 성공 메시지">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. [Cargo](https://github.com/rust-lang/cargo)의 bin 디렉토리를 포함하도록 PATH
   환경 변수를 다시 로드하세요:

```terminal
$ . "$HOME/.cargo/env"
```

3. 설치가 성공적으로 완료되었는지 확인하세요.

```terminal
$ rustc --version
```

다음과 같은 출력이 표시됩니다:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Solana CLI 설치하기

Solana CLI는 Solana 프로그램을 빌드하고 배포하는 데 필요한 모든 도구를
제공합니다.

1. 공식 설치 명령어를 사용하여 Solana CLI 도구 모음을 설치하세요:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  `stable`를 원하는 릴리스의 소프트웨어 버전과 일치하는 릴리스 태그(예:
  `v2.0.3`)로 대체하거나, 세 가지 심볼릭 채널 이름 중 하나를 사용할 수 있습니다:
  `stable`, `beta` 또는 `edge`.
</Callout>

2. PATH 환경 변수 추가하기

Solana CLI를 처음 설치하면 `PATH` 환경 변수를 추가하라는 메시지가 표시될 수
있습니다. 이를 위해 터미널을 닫고 다시 열거나 쉘에서 다음 명령어를 실행하세요:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. PATH 환경 변수 업데이트하기

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

**Linux** 또는 **WSL**을 사용하는 경우, 터미널에서 명령어를 사용할 수 있도록
Solana CLI 바이너리를 PATH에 추가해야 합니다. 이를 위해 아래 단계를 따르세요:

**a.** 현재 사용 중인 쉘 확인하기:

```terminal
$ echo $SHELL
```

- 출력에 `/bash`가 포함되어 있으면, `.bashrc`를 사용하세요.
- 출력에 `/zsh`가 포함되어 있으면, `.zshrc`를 사용하세요.

**b.** 사용 중인 셸에 맞는 명령어를 실행하세요.

Bash(`bashrc`)의 경우:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Zsh(`zshrc`)의 경우:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Mac에서 `zsh`를 사용하는 경우, 설치 과정에서 로그된 기본 `export PATH` 명령어를
실행하면 터미널을 닫을 때 설정이 유지되지 않습니다. 대신, 다음 명령어를 실행하여
셸 구성 파일에 PATH를 추가하세요:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  위의 단계는 `bash` 사용자에게는 적용되지 않습니다. PATH 변수에 대한 변경
  사항이 기본적으로 유지됩니다.
</Callout>

</Tab>
</Tabs>

4. 터미널을 다시 시작하거나 다음 명령어를 실행하여 터미널 세션을 새로고침하세요:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Solana CLI 버전을 확인하여 설치가 성공했는지 확인하세요:

```terminal
$ solana --version
```

다음과 같은 출력이 표시됩니다:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

[Agave Github 저장소](https://github.com/anza-xyz/agave/releases)에서 사용
가능한 모든 버전을 확인할 수 있습니다. Agave는 이전에 Solana Labs validator
클라이언트로 알려졌던 [Anza](https://www.anza.xyz/)의 validator
클라이언트입니다.

</Callout>

5. 필요에 따라 Solana CLI를 최신 버전으로 업데이트하세요 _(선택 사항)_

```terminal
$ agave-install update
```

</Step>

### Anchor CLI 설치하기

[Anchor](https://www.anchor-lang.com/)는 Solana 프로그램을 개발하기 위한
프레임워크입니다. Anchor 프레임워크는 Rust 매크로를 활용하여 Solana 프로그램
작성 과정을 단순화합니다.

#### 사전 요구사항

_shell`anchor init`_ 명령으로 생성된 기본 Anchor 프로젝트 테스트
파일(TypeScript)은 Node.js와 Yarn이 필요합니다. (Rust 테스트 템플릿은
_shell`anchor init --test-template rust`_ 를 사용하여 이용 가능합니다)

<Accordions>
<Accordion title="Node 설치">

1. 다음 명령을 실행하여 [Node 버전 관리자(nvm)](https://github.com/nvm-sh/nvm)를
   사용하여 Node를 설치하세요:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. 터미널을 다시 시작하고 `nvm` 명령이 성공적으로 실행되는지 확인하세요:

```terminal
$ command -v nvm
```

3. `nvm`를 사용하여 Node를 설치하세요:

```terminal
$ nvm install node
```

4. Node 버전을 확인하여 설치가 성공했는지 확인하세요:

```terminal
$ node --version
```

다음과 같은 출력이 표시됩니다:

```
v23.7.0
```

</Accordion>
<Accordion title="Yarn 설치">

1. Yarn 설치:

```terminal
$ npm install --global yarn
```

2. Yarn 버전을 확인하여 설치가 성공했는지 확인하세요:

```terminal
$ yarn --version
```

다음과 같은 출력이 표시됩니다:

```
1.22.1
```

</Accordion>
</Accordions>

#### 설치

Anchor CLI와 도구를 설치하는 두 가지 방법이 있습니다:

- Anchor 버전 관리자(AVM) — 권장 설치 방법
- AVM 없이 — GitHub에서 직접 설치

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Anchor 버전 관리자(AVM)를 사용하면 시스템에 다양한 Anchor 버전을 설치하고 관리할
수 있으며, 향후 Anchor 버전을 쉽게 업데이트할 수 있습니다. AVM을 사용하여
Anchor를 설치하려면 아래 단계를 따르세요:

1. 다음 명령어로 AVM을 설치하세요:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. AVM이 성공적으로 설치되었는지 확인하세요:

```terminal
$ avm --version
```

3. AVM을 사용하여 Anchor CLI를 설치하세요:

최신 버전을 설치하려면:

```terminal
$ avm install latest
$ avm use latest
```

특정 버전을 설치하려면 버전 번호를 지정하세요:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Linux나 WSL에서 Anchor CLI를 설치할 때 다음과 같은 오류가 발생할 수 있습니다:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

이 오류 메시지가 표시되면 다음 단계를 따르세요:

1. 이 페이지 상단의 [Linux](/docs/intro/installation#install-dependencies)
   섹션에 나열된 종속성을 설치하세요.
2. Anchor CLI 설치를 다시 시도하세요.

</Callout>

4. 다음과 같이 Anchor CLI 버전을 확인하여 설치가 성공했는지 확인하세요:

```terminal
$ anchor --version
```

다음과 같은 출력이 표시됩니다:

```
anchor-cli 0.31.1
```

<Callout type="info">

시스템에서 실행할 Anchor CLI 버전을 선언하기 위해 _shell`avm use`_ 명령어를
실행하는 것을 잊지 마세요.

- `latest` 버전을 설치했다면, *shell`avm use latest`*를 실행하세요.
- `0.30.1` 버전을 설치했다면, *shell`avm use 0.30.1`*를 실행하세요.

</Callout>

</Tab>

<Tab value="Without AVM">

1. 다음 명령어로 특정 버전의 Anchor CLI를 설치하세요:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Anchor CLI 버전을 확인하여 설치가 성공했는지 확인하세요:

```terminal
$ anchor --version
```

다음과 같은 출력이 표시됩니다:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>
