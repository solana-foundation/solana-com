Yerel Solana geliştirme ortamınızı kurma rehberi. Windows (WSL), Linux ve Mac
üzerinde Rust, Solana CLI ve Anchor Framework'ü nasıl yükleyeceğinizi öğrenin.
Her bir bağımlılığı ayrı ayrı yüklemeyi tercih ediyorsanız veya
[hızlı kurulum](/docs/intro/installation/) herhangi bir nedenle başarısız olursa
bu rehberi kullanın.

### Ön koşullar

<Accordions>
<Accordion title="Windows">

Windows'ta Solana programları geliştirmek için **mutlaka**
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL) kullanmalısınız. WSL'yi kurduktan sonra, diğer tüm bağımlılıklar Linux
terminali aracılığıyla yüklenecektir.

WSL'yi yüklemek için Windows PowerShell'de aşağıdaki komutu çalıştırın:

```terminal
$ wsl --install
```

Kurulum işlemi sırasında bir varsayılan kullanıcı hesabı oluşturmanız
istenecektir.

![WSL Kurulumu](/assets/docs/intro/installation/wsl-install.png)

Varsayılan olarak, WSL Ubuntu'yu yükler. Arama çubuğunda "Ubuntu"yu arayarak
Linux terminalini açabilirsiniz.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Ubuntu terminaliniz aşağıdaki görüntüye benziyorsa, terminalde `ctrl + v`
(yapıştırma klavye kısayolu) çalışmaması sorunu ile karşılaşabilirsiniz.

![Ubuntu Terminali](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Bu sorunla karşılaşırsanız, Arama çubuğunda "Terminal"i arayarak Windows
Terminal'i açın.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Ardından, Windows Terminal'i kapatın ve tekrar Ubuntu'yu arayarak Linux
terminalini yeniden açın. Terminal artık aşağıdaki görüntüye benzemeli ve
`ctrl + v` (yapıştırma klavye kısayolu) çalışmalıdır.

![Ubuntu Terminali](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

VS Code kullanıyorsanız,
[WSL uzantısı](https://code.visualstudio.com/docs/remote/wsl-tutorial) WSL ve VS
Code'u birlikte kullanmanızı sağlar.

![VS Code'da WSL Kurulumu](/assets/docs/intro/installation/wsl-vscode.png)

VS Code durum çubuğunda aşağıdakini göreceksiniz:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

WSL'yi kurduktan sonra, aşağıdakilerden birini yapmaya hazırsınız:

- Ubuntu (Linux) terminalini kullanarak
  [hızlı kurulumu](/docs/intro/installation) tamamlayın
- Aşağıda açıklandığı gibi [Rust](#install-rust),
  [Solana CLI](#install-solana-cli) ve [Anchor CLI](#install-anchor-cli)
  bileşenlerini ayrı ayrı kurun

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Debian veya Ubuntu ya da Mint gibi Debian türevi sistemlerde, aşağıdaki adımları izleyin:

1. Tüm paketlerin güncel olduğundan emin olmak için paket listelerini
   güncelleyin:

```terminal
$ sudo apt-get update
```

2. Aşağıdaki bağımlılıkları yükleyin:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
`protobuf-compiler` yüklerken aşağıdaki hatayla karşılaşırsanız,
önce _shell`sudo apt-get update` komutunu çalıştırdığınızdan emin olun

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Fedora veya Red Hat Linux gibi RHEL türevleri için aşağıdaki adımları izleyin.

Paketlerin daha yeni sürümlerini almak için paket listelerini aşağıdaki komutu
çalıştırarak güncelleyin.

```terminal
$ sudo dnf upgrade --refresh
```

Ardından, aşağıdaki bağımlılıkları yükleyin:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

  </Tab>
</Tabs>

</Accordion>

### Rust'ı Yükleyin

Geliştiriciler, Solana programlarını [Rust](https://www.rust-lang.org/)
programlama dilini kullanarak oluştururlar.

1. Terminalinizde aşağıdaki komutu girerek
   [rustup](https://www.rust-lang.org/tools/install) aracılığıyla Rust'ı
   yükleyin:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Başarılı bir kurulumdan sonra aşağıdaki mesajı göreceksiniz:

<Accordions>
<Accordion title="Başarılı Rust Kurulum Mesajı">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. [Cargo'nun](https://github.com/rust-lang/cargo) bin dizinini içerecek şekilde
   PATH ortam değişkeninizi yeniden yükleyin:

```terminal
$ . "$HOME/.cargo/env"
```

3. Kurulumun başarılı olduğunu doğrulayın.

```terminal
$ rustc --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Solana CLI'yi Yükleyin

Solana CLI, Solana programlarını oluşturmak ve dağıtmak için gereken tüm
araçları sağlar.

1. Resmi kurulum komutunu kullanarak Solana CLI araç setini yükleyin:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  `stable` ifadesini istediğiniz sürümün yazılım sürümüyle eşleşen sürüm
  etiketiyle değiştirebilirsiniz (örneğin `v2.0.3`) veya üç sembolik kanal
  adından birini kullanabilirsiniz: `stable`, `beta` veya `edge`.
</Callout>

2. PATH ortam değişkeni ekleyin

Solana CLI'nin ilk kez kurulumu, bir `PATH` ortam değişkeni eklemenizi
isteyebilir. Bunu yapmak için terminalinizi kapatıp yeniden açın veya
kabuğunuzda aşağıdaki komutu çalıştırın:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. PATH ortam değişkeninizi güncelleyin

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

**Linux** veya **WSL** kullanıyorsanız, komutun terminalinizde kullanılabilir
olması için Solana CLI ikili dosyasını PATH'inize eklemelisiniz. Bunu yapmak
için aşağıdaki adımları izleyin:

**a.** Hangi kabuğu kullandığınızı kontrol edin:

```terminal
$ echo $SHELL
```

- Eğer çıktı `/bash` içeriyorsa, `.bashrc` kullanın.
- Eğer çıktı `/zsh` içeriyorsa, `.zshrc` kullanın.

**b.** Kullandığınız kabuğa (shell) göre uygun komutu çalıştırın.

Bash için (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Zsh için (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Eğer `zsh` ile Mac kullanıyorsanız, kurulumdan sonra varsayılan `export PATH`
komutunu çalıştırmak, terminalinizi kapattığınızda kalıcı olmaz. Bunun yerine,
aşağıdaki komutu çalıştırarak PATH'i kabuk yapılandırma dosyanıza ekleyin:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Yukarıdaki adımlar `bash` kullanıcıları için geçerli değildir, çünkü PATH
  değişkenindeki değişiklikler varsayılan olarak kalıcı olacaktır.
</Callout>

</Tab>
</Tabs>

4. Terminal oturumunuzu yenilemek için terminalinizi yeniden başlatın veya
   aşağıdaki komutu çalıştırın:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Solana CLI sürümünü kontrol ederek kurulumun başarılı olduğunu doğrulayın:

```terminal
$ solana --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Mevcut tüm sürümleri
[Agave Github deposunda](https://github.com/anza-xyz/agave/releases)
görüntüleyebilirsiniz. Agave, eskiden Solana Labs validator istemcisi olarak
bilinen [Anza](https://www.anza.xyz/) tarafından geliştirilen validator
istemcisidir.

</Callout>

5. Gerektiğinde Solana CLI'yi en son sürüme güncelleyin _(İsteğe bağlı)_

```terminal
$ agave-install update
```

</Step>

### Anchor CLI'yi yükleyin

[Anchor](https://www.anchor-lang.com/), Solana programları geliştirmek için bir
çerçevedir. Anchor çerçevesi, Solana programları yazma sürecini basitleştirmek
için Rust makrolarından yararlanır.

#### Ön koşullar

_shell`anchor init`_ komutuyla oluşturulan varsayılan Anchor proje test dosyası
(TypeScript), Node.js ve Yarn gerektirir. (Rust test şablonu
_shell`anchor init --test-template rust`_ kullanılarak mevcuttur)

<Accordions>
<Accordion title="Node Kurulumu">

1. Aşağıdaki komutu çalıştırarak
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) kullanarak Node'u
   kurun:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Terminalinizi yeniden başlatın ve `nvm` komutunun başarıyla çalıştığını
   doğrulayın:

```terminal
$ command -v nvm
```

3. Node'u kurmak için `nvm` kullanın:

```terminal
$ nvm install node
```

4. Node sürümünü kontrol ederek kurulumun başarılı olduğunu doğrulayın:

```terminal
$ node --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
v23.7.0
```

</Accordion>
<Accordion title="Yarn Kurulumu">

1. Yarn'ı kurun:

```terminal
$ npm install --global yarn
```

2. Yarn sürümünü kontrol ederek kurulumun başarılı olduğunu doğrulayın:

```terminal
$ yarn --version
```

Aşağıdaki çıktıyı göreceksiniz:

```
1.22.1
```

</Accordion>
</Accordions>

#### Kurulum

Anchor CLI ve araçlarını iki şekilde kurabilirsiniz:

- Anchor Version Manager (AVM) — Önerilen kurulum yöntemi
- AVM olmadan — Doğrudan GitHub'dan kurulum

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Anchor Versiyon Yöneticisi (AVM), sisteminizde farklı Anchor sürümlerini
kurmanıza ve yönetmenize olanak tanır ve gelecekte Anchor sürümlerini kolayca
güncellemenizi sağlar. AVM kullanarak Anchor'u kurmak için aşağıdaki adımları
izleyin:

1. AVM'yi aşağıdaki komutla kurun:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. AVM'nin başarıyla kurulduğunu doğrulayın:

```terminal
$ avm --version
```

3. AVM kullanarak Anchor CLI'yi kurun:

En son sürümü kurmak için:

```terminal
$ avm install latest
$ avm use latest
```

Belirli bir sürümü kurmak için, sürüm numarasını belirtin:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Linux veya WSL üzerinde Anchor CLI'yi kurarken şu hatayı alabilirsiniz:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Bu hata mesajını görürseniz, şu adımları izleyin:

1. Bu sayfanın üst kısmındaki
   [Linux](/docs/intro/installation#install-dependencies) bölümünde listelenen
   bağımlılıkları kurun.
2. Anchor CLI'yi yeniden kurmayı deneyin.

</Callout>

4. Kurulumun başarılı olduğunu, Anchor CLI sürümünü kontrol ederek doğrulayın:

```terminal
$ anchor --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
anchor-cli 0.31.1
```

<Callout type="info">

Sisteminizde çalıştırılacak Anchor CLI sürümünü belirtmek için _shell`avm use`_
komutunu çalıştırmayı unutmayın.

- Eğer `latest` sürümünü kurduysanız, _shell`avm use latest`_ komutunu
  çalıştırın.
- Eğer `0.30.1` sürümünü kurduysanız, _shell`avm use 0.30.1`_ komutunu
  çalıştırın.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Aşağıdaki komutla Anchor CLI'nin belirli bir sürümünü yükleyin:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Anchor CLI sürümünü kontrol ederek kurulumun başarılı olduğunu doğrulayın:

```terminal
$ anchor --version
```

Aşağıdakine benzer bir çıktı göreceksiniz:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>

### Solana geliştirme için yapay zeka araçlarının kurulumu

Bu bölüm, Solana geliştirme sürecinizi hızlandırmak için kullanabileceğiniz
isteğe bağlı yapay zeka araçlarının kurulumunu detaylandırır.

| Araç     | Açıklama                                                                                                               | Bağlantı                    |
| -------- | ---------------------------------------------------------------------------------------------------------------------- | --------------------------- |
| MCP      | Cursor ile bağlanabileceğiniz ve Solana yapay zeka destekli geliştirmeyi iyileştiren MCP sunucusu.                     | https://mcp.solana.com/     |
| LLMs.txt | Solana dokümanları üzerinde yapay zeka modellerini eğitmek için kullanabileceğiniz LLM optimize edilmiş dokümantasyon. | https://solana.com/llms.txt |
