h1: slotsUpdatesSubscribe RPC Metodu
---

Her slot üzerinde çeşitli güncellemeler hakkında validator'dan bildirim almak
için abone olun

<Callout type="warn">
  Bu abonelik kararsızdır. Bu aboneliğin formatı gelecekte değişebilir ve her
  zaman desteklenmeyebilir.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "slotsUpdatesSubscribe"
}
```

### !params

**Yok**

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

Abonelik kimliği (abonelikten çıkmak için gereklidir)

</APIMethod>

### Bildirim Formatı

Bildirim, aşağıdaki alanları içeren bir nesne olacaktır:

- `err: <string|undefined>` - Hata mesajı. Yalnızca güncelleme türü "dead"
  olduğunda mevcuttur.
- `parent: <u64|undefined>` - Üst slot. Yalnızca güncelleme türü "createdBank"
  olduğunda mevcuttur.
- `slot: <u64>` - Yeni güncellenen slot
- `stats: <object|undefined>` - Hata mesajı. Yalnızca güncelleme türü "frozen"
  olduğunda mevcuttur. Aşağıdaki alanları içeren bir nesne:
  - `maxTransactionsPerEntry: <u64>`,
  - `numFailedTransactions: <u64>`,
  - `numSuccessfulTransactions: <u64>`,
  - `numTransactionEntries: <u64>`,
- `timestamp: <i64>` - Güncellemenin milisaniye cinsinden Unix zaman damgası
- `type: <string>` - Güncelleme türü, aşağıdakilerden biri:
  - "firstShredReceived"
  - "completed"
  - "createdBank"
  - "frozen"
  - "dead"
  - "optimisticConfirmation"
  - "root"

```shell
{
  "jsonrpc": "2.0",
  "method": "slotsUpdatesNotification",
  "params": {
    "result": {
      "parent": 75,
      "slot": 76,
      "timestamp": 1625081266243,
      "type": "optimisticConfirmation"
    },
    "subscription": 0
  }
}
```
