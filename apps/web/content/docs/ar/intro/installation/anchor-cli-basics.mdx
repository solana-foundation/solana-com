يقدم هذا القسم بعض الأوامر والأمثلة الشائعة لمساعدتك على البدء باستخدام
[واجهة سطر أوامر Anchor](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### تهيئة المشروع

قم بإنشاء مشروع Anchor جديد عن طريق تشغيل الأمر الموضح أدناه. سيقوم بإنشاء دليل
جديد باسم المشروع واستخدامه لتهيئة مشروع Anchor جديد.

```terminal
$ anchor init <project-name>
```

على سبيل المثال، الأمر أدناه سينشئ مشروعًا يسمى `my-project`. سيحتوي دليل
`my-project` على برنامج Rust أساسي وقالب اختبار TypeScript.

```terminal
$ anchor init my-project
```

ثم انتقل إلى دليل المشروع:

```terminal
$ cd <project-name>
```

راجع
[هيكل ملفات المشروع](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
الخاص بـ Anchor.

</Step>
<Step>

### بناء البرنامج

لبناء مشروعك، قم بتشغيل الأمر التالي:

```terminal
$ anchor build
```

يمكنك العثور على البرنامج المُجمّع في دليل `/target/deploy`.

<Callout type="warn">

عند تشغيل `anchor build`، إذا واجهت الأخطاء التالية:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

جرب هذه الحلول:

1. فرض التثبيت باستخدام الأمر التالي:

```terminal
$ cargo build-sbf --force-tools-install
```

2. إذا لم يعمل الحل السابق، قم بمسح ذاكرة التخزين المؤقت لـ سولانا:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
يمكنك إصلاح هذا عن طريق تغيير حقل الإصدار في ملف `Cargo.lock` من 4 إلى 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

راجع [هذه المشكلة](https://github.com/solana-foundation/anchor/issues/3392)
لمزيد من المعلومات.

</Accordion>

</Accordions>

بعد تطبيق الحل السابق، حاول تشغيل _shell`anchor build`_ مرة أخرى.

</Callout>

</Step>
<Step>

### نشر البرنامج

لنشر مشروعك، قم بتشغيل الأمر التالي:

```terminal
$ anchor deploy
```

يقوم هذا الأمر بنشر برنامجك على `cluster` المحدد في ملف
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### اختبار البرنامج

لاختبار مشروعك، قم بتشغيل الأمر التالي:

```terminal
$ anchor test
```

يقوم هذا الأمر ببناء ونشر وتشغيل الاختبارات لمشروعك.

<Callout>
  عند استخدام مجموعة `localnet`، يقوم Anchor تلقائيًا ببدء validator محلي، ونشر
  البرنامج، وتشغيل الاختبارات، ثم إيقاف الـvalidator.
</Callout>

<Callout type="warn">

قد يشير أي من الأخطاء التالية إلى أنك لم تقم بتثبيت Node.js أو Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
