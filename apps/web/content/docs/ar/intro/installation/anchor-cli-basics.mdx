يستعرض هذا القسم بعض أوامر واجهة سطر أوامر Anchor الشائعة لمساعدتك على البدء.
لمزيد من المعلومات حول واجهة سطر أوامر Anchor، راجع
[وثائق Anchor](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### تهيئة المشروع

لإنشاء مشروع Anchor جديد، قم بتشغيل الأمر التالي:

```terminal
$ anchor init <project-name>
```

على سبيل المثال، لإنشاء مشروع يسمى `my-project`، قم بتشغيل:

```terminal
$ anchor init my-project
```

ينشئ هذا الأمر دليلاً جديدًا باسم المشروع ويقوم بتهيئة مشروع Anchor جديد مع
برنامج Rust أساسي وقالب اختبار TypeScript.

انتقل إلى دليل المشروع:

```terminal
$ cd <project-name>
```

اطلع على
[هيكل ملفات](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
مشروع Anchor.

</Step>

<Step>
### بناء البرنامج

لبناء مشروعك، قم بتشغيل الأمر التالي:

```terminal
$ anchor build
```

يمكنك العثور على البرنامج المُجمّع في دليل `/target/deploy`.

<Callout type="warn">

عند تشغيل `anchor build`، إذا واجهت الأخطاء التالية:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

جرب هذه الحلول:

1. فرض التثبيت باستخدام الأمر التالي:

```terminal
$ cargo build-sbf --force-tools-install
```

2. إذا لم يعمل الحل السابق، قم بمسح ذاكرة التخزين المؤقت لسولانا:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="إصدار ملف القفل 4 يتطلب `-Znext-lockfile-bump`">
يمكنك إصلاح هذا بتغيير حقل الإصدار في ملف `Cargo.lock` من 4 إلى 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

راجع [هذه المشكلة](https://github.com/solana-foundation/anchor/issues/3392)
لمزيد من المعلومات.

</Accordion>

</Accordions>

بعد تطبيق الحل السابق، حاول تشغيل _shell`anchor build`_ مرة أخرى.

</Callout>

</Step>
<Step>

### نشر البرنامج

لنشر مشروعك، قم بتشغيل الأمر التالي:

```terminal
$ anchor deploy
```

يقوم هذا الأمر بنشر برنامجك على `cluster` المحدد في ملف
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### اختبار البرنامج

لاختبار مشروعك، قم بتشغيل الأمر التالي:

```terminal
$ anchor test
```

<Callout>

يقوم هذا الأمر ببناء ونشر وتشغيل الاختبارات لمشروعك.

عند استخدام `localnet` كـ `cluster` في `Anchor.toml`، يقوم Anchor تلقائيًا ببدء
مصادق محلي، ونشر برنامجك، وتشغيل الاختبارات، ثم إيقاف المصادق.

</Callout>

<Callout type="warn">

عند تشغيل _shell`anchor test`_ بعد إنشاء مشروع Anchor جديد على نظام لينكس أو
WSL، قد تواجه الأخطاء التالية إذا لم يكن لديك Node.js أو Yarn مثبتين:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
