دليل لإعداد بيئة تطوير سولانا المحلية الخاصة بك. تعلم كيفية تثبيت Rust وواجهة
سطر أوامر سولانا وإطار عمل Anchor على نظام ويندوز (WSL) ولينكس وماك. استخدم هذا
الدليل إذا كنت تفضل تثبيت كل تبعية بشكل فردي، أو إذا فشل
[التثبيت السريع](/docs/intro/installation/) لأي سبب.

### المتطلبات الأساسية

<Accordions>
<Accordion title="ويندوز">

لتطوير برامج سولانا على نظام ويندوز **يجب** عليك استخدام
[نظام ويندوز الفرعي للينكس](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). بعد تثبيت WSL، سيتم تثبيت جميع التبعيات الأخرى من خلال طرفية لينكس.

لتثبيت WSL، قم بتشغيل الأمر التالي في PowerShell لويندوز:

```terminal
$ wsl --install
```

ستطلب منك عملية التثبيت إنشاء حساب مستخدم افتراضي.

![تثبيت WSL](/assets/docs/intro/installation/wsl-install.png)

بشكل افتراضي، يقوم WSL بتثبيت أوبونتو. يمكنك فتح طرفية لينكس عن طريق البحث عن
"Ubuntu" في شريط البحث.

![أوبونتو WSL](/assets/docs/intro/installation/wsl-ubuntu-search.png)

إذا كانت طرفية أوبونتو الخاصة بك تبدو مثل الصورة أدناه، فقد تواجه مشكلة حيث لا
يعمل اختصار لوحة المفاتيح `ctrl + v` (لصق) في الطرفية.

![طرفية أوبونتو](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

إذا واجهت هذه المشكلة، افتح طرفية ويندوز عن طريق البحث عن "Terminal" في شريط
البحث.

![طرفية ويندوز](/assets/docs/intro/installation/wsl-windows-terminal.png)

بعد ذلك، أغلق طرفية ويندوز وأعد فتح طرفية لينكس عن طريق البحث عن أوبونتو مرة
أخرى. يجب أن تبدو الطرفية الآن مثل الصورة أدناه، حيث يعمل اختصار لوحة المفاتيح
`ctrl + v` (لصق).

![طرفية أوبونتو](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

إذا كنت تستخدم VS Code، فإن
[امتداد WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) يتيح لك
استخدام WSL وVS Code معًا.

![إعداد WSL في VS Code](/assets/docs/intro/installation/wsl-vscode.png)

سترى ما يلي في شريط الحالة الخاص بـ VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

بمجرد إعداد WSL، ستكون جاهزًا إما لـ:

- إكمال [التثبيت السريع](/docs/intro/installation) باستخدام طرفية Ubuntu (Linux)
- تثبيت [Rust](#install-rust) و[واجهة سطر أوامر سولانا](#install-solana-cli) و
  [واجهة سطر أوامر Anchor](#install-anchor-cli) بشكل فردي، كما هو موضح أدناه

</Accordion>
<Accordion title="لينكس">

<Tab value=".deb" >
إذا كنت تستخدم نظام ديبيان أو الأنظمة المشتقة منه مثل أوبونتو أو مينت، اتبع الخطوات التالية:

1. قم بتحديث قوائم الحزم للتأكد من أن جميع الحزم محدثة:

```terminal
$ sudo apt-get update
```

2. قم بتثبيت التبعيات التالية:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \

```

من تشغيل _shell`sudo apt-get update` أولاً

```
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

```terminal
$ sudo dnf upgrade --refresh
```

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

### تثبيت Rust

يقوم المطورون ببناء برامج سولانا باستخدام لغة البرمجة
[Rust](https://www.rust-lang.org/).

1. قم بتثبيت Rust باستخدام [rustup](https://www.rust-lang.org/tools/install) عن
   طريق إدخال الأمر التالي في نافذة الطرفية الخاصة بك:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

بعد التثبيت الناجح ستظهر لك الرسالة التالية:

<Accordions>
<Accordion title="رسالة تثبيت Rust الناجحة">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. أعد تحميل متغير بيئة PATH الخاص بك ليشمل دليل bin الخاص بـ
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. تحقق من نجاح عملية التثبيت.

```terminal
$ rustc --version
```

سترى مخرجات مشابهة لما يلي:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### تثبيت واجهة سطر أوامر سولانا

توفر واجهة سطر أوامر سولانا جميع الأدوات المطلوبة لبناء ونشر برامج سولانا.

1. قم بتثبيت مجموعة أدوات واجهة سطر أوامر سولانا باستخدام أمر التثبيت الرسمي:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  يمكنك استبدال `stable` بعلامة الإصدار المطابقة لإصدار البرنامج الذي ترغب فيه
  (مثل `v2.0.3`)، أو استخدام أحد أسماء القنوات الرمزية الثلاثة: `stable`،
  `beta`، أو `edge`.
</Callout>

2. إضافة متغير بيئة PATH

قد يطالبك التثبيت الأول لواجهة سطر أوامر سولانا بإضافة متغير بيئة `PATH`. للقيام
بذلك، أغلق وأعد فتح نافذة الطرفية أو قم بتنفيذ الأمر التالي في الشل الخاص بك:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. تحديث متغير بيئة PATH الخاص بك

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

إذا كنت تستخدم **Linux** أو **WSL**، يجب عليك إضافة الملف الثنائي لواجهة سطر
أوامر سولانا إلى متغير PATH الخاص بك حتى يكون الأمر متاحًا في نافذة الطرفية.
للقيام بذلك، اتبع الخطوات أدناه:

**أ.** تحقق من الشل الذي تستخدمه:

```terminal
$ echo $SHELL
```

- إذا كان الناتج يحتوي على `/bash`، استخدم `.bashrc`.
- إذا كان الناتج يحتوي على `/zsh`، استخدم `.zshrc`.

**ب.** قم بتشغيل الأمر المناسب، بناءً على الشل الخاص بك.

لـ Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

لـ Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

إذا كنت تستخدم جهاز Mac مع `zsh`، فإن تشغيل أمر `export PATH` الافتراضي المسجل
من التثبيت لا يستمر بمجرد إغلاق الطرفية. بدلاً من ذلك، أضف المسار إلى ملف تكوين
الشل الخاص بك عن طريق تشغيل الأمر التالي:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  الخطوات المذكورة أعلاه لا تنطبق على مستخدمي `bash`، حيث ستستمر التغييرات على
  متغير PATH بشكل افتراضي.
</Callout>

</Tab>
</Tabs>

4. أعد تشغيل الطرفية أو قم بتشغيل الأمر التالي لتحديث جلسة الطرفية:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. تحقق من نجاح التثبيت عن طريق التحقق من إصدار واجهة سطر أوامر سولانا:

```terminal
$ solana --version
```

سترى مخرجات مثل ما يلي:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

يمكنك عرض جميع الإصدارات المتاحة على
[مستودع Agave على Github](https://github.com/anza-xyz/agave/releases). Agave هو
عميل validator من [Anza](https://www.anza.xyz/)، المعروف سابقًا باسم عميل
validator من سولانا Labs.

</Callout>

5. قم بتحديث واجهة سطر أوامر سولانا إلى أحدث إصدار، حسب الحاجة _(اختياري)_

```terminal
$ agave-install update
```

</Step>

### تثبيت واجهة سطر أوامر Anchor

[Anchor](https://www.anchor-lang.com/) هو إطار عمل لتطوير برامج سولانا. يستخدم
إطار عمل Anchor وحدات ماكرو Rust لتبسيط عملية كتابة برامج سولانا.

#### المتطلبات الأساسية

ملف اختبار مشروع Anchor الافتراضي (TypeScript) الذي تم إنشاؤه باستخدام الأمر
_shell`anchor init`_ يتطلب Node.js و Yarn. (قالب اختبار Rust متاح باستخدام
_shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="تثبيت Node">

1. قم بتثبيت Node باستخدام
   [مدير إصدارات Node (nvm)](https://github.com/nvm-sh/nvm) عن طريق تشغيل الأمر
   التالي:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. أعد تشغيل نافذة الطرفية وتأكد من أن الأمر `nvm` يعمل بنجاح:

```terminal
$ command -v nvm
```

3. استخدم `nvm` لتثبيت Node:

```terminal
$ nvm install node
```

4. تحقق من نجاح التثبيت، من خلال التحقق من إصدار Node:

```terminal
$ node --version
```

سترى مخرجات مثل ما يلي:

```
v23.7.0
```

</Accordion>
<Accordion title="تثبيت Yarn">

1. قم بتثبيت Yarn:

```terminal
$ npm install --global yarn
```

2. تحقق من نجاح التثبيت، من خلال التحقق من إصدار Yarn:

```terminal
$ yarn --version
```

سترى المخرجات التالية:

```
1.22.1
```

</Accordion>
</Accordions>

#### التثبيت

يمكنك تثبيت واجهة سطر أوامر Anchor والأدوات بطريقتين:

- مدير إصدارات Anchor (AVM) — طريقة التثبيت الموصى بها
- بدون AVM — التثبيت مباشرة من GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

يتيح لك مدير إصدارات Anchor (AVM) تثبيت وإدارة إصدارات مختلفة من Anchor على
نظامك وتحديث إصدارات Anchor بسهولة في المستقبل. لتثبيت Anchor باستخدام AVM، اتبع
الخطوات التالية:

1. قم بتثبيت AVM باستخدام الأمر التالي:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. تأكد من نجاح تثبيت AVM:

```terminal
$ avm --version
```

3. قم بتثبيت واجهة سطر أوامر Anchor باستخدام AVM:

لتثبيت أحدث إصدار:

```terminal
$ avm install latest
$ avm use latest
```

لتثبيت إصدار محدد، حدد رقم الإصدار:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

عند تثبيت واجهة سطر أوامر Anchor على Linux أو WSL، قد تواجه هذا الخطأ:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

إذا رأيت رسالة الخطأ هذه، اتبع هذه الخطوات:

1. قم بتثبيت التبعيات المذكورة في قسم
   [Linux](/docs/intro/installation#install-dependencies) في أعلى هذه الصفحة.
2. أعد محاولة تثبيت واجهة سطر أوامر Anchor.

</Callout>

4. تحقق من نجاح التثبيت، من خلال التحقق من إصدار واجهة سطر أوامر Anchor:

```terminal
$ anchor --version
```

سترى مخرجات مثل ما يلي:

```
anchor-cli 0.31.1
```

<Callout type="info">

لا تنس تشغيل أمر _shell`avm use`_ لتحديد إصدار واجهة سطر أوامر Anchor الذي سيتم
تشغيله على نظامك.

- إذا قمت بتثبيت الإصدار `latest`، قم بتشغيل _shell`avm use latest`_.
- إذا قمت بتثبيت الإصدار `0.30.1`، قم بتشغيل _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. قم بتثبيت إصدار محدد من واجهة سطر أوامر Anchor باستخدام الأمر التالي:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. تحقق من نجاح التثبيت، من خلال التحقق من إصدار واجهة سطر أوامر Anchor:

```terminal
$ anchor --version
```

سترى مخرجات مثل ما يلي:

```
anchor-cli 0.31.1
```

</Tab>
