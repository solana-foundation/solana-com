## SDK Python da Solana

Existem dois pacotes que são úteis para desenvolver com Python:

[`@kevinheavy/solana.py`](https://github.com/michaelhly/solana-py)

- Solana.py é a biblioteca Python base para interagir com a Solana.
- Você pode usá-la para construir transações e interagir com a
  [API JSON RPC da Solana](/docs/rpc), assim como você faria com
  [solana-web3.js](https://github.com/solana-labs/solana-web3.js/)

- Também abrange o [Token Program](/docs/rpc).

- [Documentação mais recente](https://michaelhly.github.io/solana-py/).

[`@kevinheavey/solders`](https://github.com/kevinheavey/solders)

- `solders` é um kit de ferramentas Python de alto desempenho para Solana,
  escrito em Rust. Ele fornece soluções robustas nas seguintes áreas:

- Funcionalidade principal do SDK: pares de chaves, chaves públicas, assinatura
  e serialização de transações.
- RPC: construção de solicitações e análise de respostas (para redes e APIs
  assíncronas,
  [solana-py](https://michaelhly.github.io/solana-py/rpc/async_api/) é a opção
  recomendada).
- Testes de integração: o módulo `solders.litesvm` é uma alternativa ao
  `solana-test-validator` que é muito mais conveniente e **muito** mais rápido.
  É baseado em
  [solana-program-test](https://crates.io/crates/solana-program-test).

## SDK Python do Cliente Solana

Se você deseja interagir com contratos Solana com Python, pode usar SDKs de
cliente para renderizar contratos Rust da Solana em clientes Python, desde que
você consiga obter o arquivo IDL do contrato.

[`Codama-py`](https://www.npmjs.com/package/codama-py)

- Padroniza qualquer programa de contrato on-chain da Solana para
  [`Codama IDL`](https://github.com/codama-idl/codama) e gera um cliente Python
  interativo.
- Suporta tanto programas Anchor quanto personalizados.

[`Anchor-py`](https://github.com/kevinheavey/anchorpy)

- Anchor-py é um cliente Python para programas Anchor. Ele pode gerar clientes a
  partir de arquivos IDL do Anchor e interagir com programas Anchor.
