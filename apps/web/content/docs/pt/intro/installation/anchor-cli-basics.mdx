Esta seção fornece alguns comandos e exemplos comuns para ajudar você a começar
a usar a [CLI do Anchor](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Inicializar o projeto

Crie um novo projeto Anchor executando o comando mostrado abaixo. Ele criará um
novo diretório com o nome do projeto e o usará para inicializar um novo projeto
Anchor.

```terminal
$ anchor init <project-name>
```

Por exemplo, o comando abaixo criará um projeto chamado `my-project`. O
diretório `my-project` conterá um programa básico em Rust e um modelo de teste
em TypeScript.

```terminal
$ anchor init my-project
```

Em seguida, navegue até o diretório do projeto:

```terminal
$ cd <project-name>
```

Veja a
[estrutura de arquivos do projeto](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
do Anchor.

</Step>
<Step>

### Compilar o programa

Para compilar seu projeto, execute o seguinte comando:

```terminal
$ anchor build
```

Você pode encontrar o programa compilado no diretório `/target/deploy`.

<Callout type="warn">

Ao executar `anchor build`, se você encontrar os seguintes erros:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Tente estas soluções:

1. Force a instalação usando o seguinte comando:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Se a solução anterior não funcionar, limpe o cache do Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Você pode corrigir isso alterando o campo de versão do arquivo `Cargo.lock` de 4 para 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Veja [este problema](https://github.com/solana-foundation/anchor/issues/3392)
para mais informações.

</Accordion>

</Accordions>

Após aplicar a solução anterior, tente executar _shell`anchor build`_ novamente.

</Callout>

</Step>
<Step>

### Implantar o programa

Para implantar seu projeto, execute o seguinte comando:

```terminal
$ anchor deploy
```

Este comando implanta seu programa no `cluster` especificado no arquivo
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Testar o programa

Para testar seu projeto, execute o seguinte comando:

```terminal
$ anchor test
```

Este comando compila, implanta e executa os testes para o seu projeto.

<Callout>
  Ao usar o cluster `localnet`, o Anchor inicia automaticamente um validator
  local, implanta o programa, executa testes e depois interrompe o validator.
</Callout>

<Callout type="warn">

Qualquer um dos seguintes erros pode indicar que você não tem Node.js ou Yarn
instalado:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
