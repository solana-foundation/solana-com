Una instalación rápida para configurar tu entorno de desarrollo local de Solana
con un solo comando. Instala Rust, la CLI de Solana y el Framework Anchor en
Windows (WSL), Linux y Mac. (Si lo prefieres, puedes
[instalar cada dependencia individualmente](/docs/intro/installation/dependencies).)

## Requisitos previos

<Accordions>
<Accordion title="Windows">

Para desarrollar programas de Solana en Windows **debes** usar
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Después de instalar WSL, todas las demás dependencias se instalarán a
través del terminal de Linux.

Para instalar WSL, ejecuta el siguiente comando en Windows PowerShell:

```terminal
$ wsl --install
```

El proceso de instalación te pedirá que crees una cuenta de usuario
predeterminada.

![Instalación de WSL](/assets/docs/intro/installation/wsl-install.png)

Por defecto, WSL instala Ubuntu. Puedes abrir un terminal de Linux buscando
"Ubuntu" en la barra de búsqueda.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Si tu terminal de Ubuntu se parece a la imagen de abajo, es posible que
encuentres un problema donde `ctrl + v` (atajo de teclado para pegar) no
funciona en el terminal.

![Terminal de Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Si encuentras este problema, abre Windows Terminal buscando "Terminal" en la
barra de búsqueda.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

A continuación, cierra Windows Terminal y vuelve a abrir un terminal de Linux
buscando Ubuntu de nuevo. El terminal ahora debería verse como la imagen de
abajo, donde `ctrl + v` (atajo de teclado para pegar) funciona.

![Terminal de Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Si usas VS Code, la
[extensión WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) te
permite usar WSL y VS Code juntos.

![Configuración de WSL en VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Verás lo siguiente en la barra de estado de VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Una vez que hayas configurado WSL, estarás listo para:

- Completar la [instalación rápida](#installation) usando la terminal de Ubuntu
  (Linux)
- Instalar [Rust](/docs/intro/installation/dependencies#install-rust), la
  [CLI de Solana](/docs/intro/installation/dependencies#install-solana-cli), y
  la [CLI de Anchor](/docs/intro/installation/dependencies#install-anchor-cli)
  individualmente, como se describe a continuación

</Accordion>

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Si usas Debian o alternativas derivadas de Debian como Ubuntu o Mint, sigue los pasos a continuación:

1. Actualiza las listas de paquetes para asegurar que todos los paquetes estén
   actualizados:

```terminal
$ sudo apt-get update
```

2. Instala las siguientes dependencias:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Si encuentras el siguiente error al instalar `protobuf-compiler`, asegúrate
de ejecutar primero _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Para derivados de RHEL como Fedora o Red Hat Linux, sigue los siguientes pasos.

Actualiza las listas de paquetes para obtener las versiones más recientes de los
paquetes ejecutando el comando.

```terminal
$ sudo dnf upgrade --refresh
```

A continuación, instala las siguientes dependencias:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

## Instalación

1. Introduce el siguiente comando en tu terminal para instalar todas las
   dependencias necesarias:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

Una instalación exitosa mostrará un resultado como el siguiente:

```txt title="Example output"
Installed Versions:
Rust: rustc 1.90.0 (1159e78c4 2025-09-14)
Solana CLI: solana-cli 2.3.13 (src:5466f459; feat:2142755730, client:Agave)
Anchor CLI: 0.32.1
Node.js: v24.10.0
Yarn: 1.22.22
```

2. Verifica que la instalación se ha realizado correctamente comprobando la
   versión de cada dependencia instalada.

```terminal
$ rustc --version && solana --version && anchor --version && node --version && yarn --version
```

<Callout>
  Si el comando de instalación rápida falla, consulta la sección [Instalar
  dependencias](/docs/intro/installation/dependencies) a continuación para
  obtener instrucciones sobre cómo instalar cada dependencia individualmente.
</Callout>
