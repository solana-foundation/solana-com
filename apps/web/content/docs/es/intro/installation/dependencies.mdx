Una guía para configurar tu entorno de desarrollo local de Solana. Aprende cómo
instalar Rust, la CLI de Solana y Anchor Framework en Windows (WSL), Linux y
Mac. Usa esta guía si prefieres instalar cada dependencia individualmente, o si
la [instalación rápida](/docs/intro/installation/) falla por alguna razón.

### Requisitos previos

<Accordions>
<Accordion title="Windows">

Para desarrollar programas de Solana en Windows **debes** usar
[Windows Subsystem for Linux](https://learn.microsoft.com/en-us/windows/wsl/install)
(WSL). Después de instalar WSL, todas las demás dependencias se instalarán a
través de la terminal de Linux.

Para instalar WSL, ejecuta el siguiente comando en Windows PowerShell:

```terminal
$ wsl --install
```

El proceso de instalación te pedirá que crees una cuenta de usuario
predeterminada.

![Instalación de WSL](/assets/docs/intro/installation/wsl-install.png)

Por defecto, WSL instala Ubuntu. Puedes abrir una terminal de Linux buscando
"Ubuntu" en la barra de búsqueda.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Si tu terminal de Ubuntu se parece a la imagen de abajo, es posible que
encuentres un problema donde `ctrl + v` (atajo de teclado para pegar) no
funciona en la terminal.

![Terminal de Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Si encuentras este problema, abre Windows Terminal buscando "Terminal" en la
barra de búsqueda.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

A continuación, cierra Windows Terminal y vuelve a abrir una terminal de Linux
buscando Ubuntu nuevamente. La terminal ahora debería verse como la imagen de
abajo, donde `ctrl + v` (atajo de teclado para pegar) funciona.

![Terminal de Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Si usas VS Code, la
[extensión WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) te
permite usar WSL y VS Code juntos.

![Configuración de WSL en VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Verás lo siguiente en la barra de estado de VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Una vez que hayas configurado WSL, estarás listo para:

- Completar la [instalación rápida](/docs/intro/installation) usando la terminal
  de Ubuntu (Linux)
- Instalar [Rust](#install-rust), la [CLI de Solana](#install-solana-cli) y la
  [CLI de Anchor](#install-anchor-cli) individualmente, como se describe a
  continuación

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
<Tab value=".deb" >
Si usas Debian o alternativas derivadas de Debian como Ubuntu o Mint, sigue los pasos a continuación:

1. Actualiza las listas de paquetes para asegurarte de que todos los paquetes
   estén actualizados:

```terminal
$ sudo apt-get update
```

2. Instala las siguientes dependencias:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

<Callout>
Si encuentras el siguiente error al instalar `protobuf-compiler`, asegúrate
de ejecutar primero _shell`sudo apt-get update`

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

</Callout>

  </Tab>

    <Tab value=".rpm" >
    Para derivados de RHEL como Fedora o Red Hat Linux, sigue los siguientes pasos.

Actualiza las listas de paquetes para obtener las versiones más recientes de los
paquetes ejecutando el comando.

```terminal
$ sudo dnf upgrade --refresh
```

A continuación, instala las siguientes dependencias:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

  </Tab>

### Instalar Rust

Los desarrolladores construyen programas de Solana utilizando el lenguaje de
programación [Rust](https://www.rust-lang.org/).

1. Instala Rust usando [rustup](https://www.rust-lang.org/tools/install)
   introduciendo el siguiente comando en tu terminal:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Después de una instalación exitosa verás el siguiente mensaje:

<Accordions>
<Accordion title="Mensaje de instalación exitosa de Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

2. Recarga tu variable de entorno PATH para incluir el directorio bin de
   [Cargo](https://github.com/rust-lang/cargo):

```terminal
$ . "$HOME/.cargo/env"
```

3. Verifica que la instalación se realizó correctamente.

```terminal
$ rustc --version
```

Verás una salida como la siguiente:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Instalar Solana CLI

Solana CLI proporciona todas las herramientas necesarias para construir y
desplegar programas de Solana.

1. Instala el conjunto de herramientas de Solana CLI usando el comando de
   instalación oficial:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

<Callout>
  Puedes reemplazar `stable` con la etiqueta de versión que coincida con la
  versión de software de tu versión deseada (por ejemplo, `v2.0.3`), o usar uno
  de los tres nombres de canal simbólicos: `stable`, `beta`, o `edge`.
</Callout>

2. Añade una variable de entorno PATH

Una instalación por primera vez de Solana CLI puede pedirte que añadas una
variable de entorno `PATH`. Para hacerlo, cierra y vuelve a abrir tu terminal o
ejecuta lo siguiente en tu shell:

```
export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

3. Actualiza tu variable de entorno PATH

<Tabs groupId="language" items={["Windows & Linux", "Mac"]}>
<Tab value="Windows & Linux">

Si estás usando **Linux** o **WSL**, debes añadir el binario de Solana CLI a tu
PATH para que el comando esté disponible en tu terminal. Para hacerlo, sigue los
pasos a continuación:

**a.** Comprueba qué shell estás utilizando:

```terminal
$ echo $SHELL
```

- Si la salida contiene `/bash`, usa `.bashrc`.
- Si la salida contiene `/zsh`, usa `.zshrc`.

**b.** Ejecuta el comando apropiado, según tu shell.

Para Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Para Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

</Tab>
<Tab value="Mac">

Si usas un Mac con `zsh`, ejecutar el comando predeterminado `export PATH`
registrado desde la instalación no persiste una vez que cierras tu terminal. En
su lugar, añade el PATH a tu archivo de configuración del shell ejecutando el
siguiente comando:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

<Callout>
  Los pasos anteriores no aplican a los usuarios de `bash`, ya que los cambios
  en la variable PATH persistirán por defecto.
</Callout>

</Tab>
</Tabs>

4. Reinicia tu terminal o ejecuta el siguiente comando para actualizar la sesión
   del terminal:

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

5. Verifica que la instalación se realizó correctamente comprobando la versión
   de Solana CLI:

```terminal
$ solana --version
```

Verás una salida como la siguiente:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

<Callout>

Puedes ver todas las versiones disponibles en el
[repositorio de Github de Agave](https://github.com/anza-xyz/agave/releases).
Agave es el cliente validator de [Anza](https://www.anza.xyz/), anteriormente
conocido como cliente validator de Solana Labs.

</Callout>

5. Actualiza Solana CLI a la última versión, según sea necesario _(Opcional)_

```terminal
$ agave-install update
```

</Step>

### Instalar Anchor CLI

[Anchor](https://www.anchor-lang.com/) es un framework para desarrollar
programas en Solana. El framework Anchor utiliza macros de Rust para simplificar
el proceso de escribir programas en Solana.

#### Requisitos previos

El archivo de prueba predeterminado del proyecto Anchor (TypeScript) creado con
el comando _shell`anchor init`_ requiere Node.js y Yarn. (La plantilla de prueba
de Rust está disponible usando _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Instalación de Node">

1. Instala Node usando
   [Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) ejecutando el
   siguiente comando:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

2. Reinicia tu terminal y confirma que el comando `nvm` se ejecuta
   correctamente:

```terminal
$ command -v nvm
```

3. Usa `nvm` para instalar Node:

```terminal
$ nvm install node
```

4. Verifica que la instalación se realizó correctamente, comprobando la versión
   de Node:

```terminal
$ node --version
```

Verás un resultado como el siguiente:

```
v23.7.0
```

</Accordion>
<Accordion title="Instalación de Yarn">

1. Instala Yarn:

```terminal
$ npm install --global yarn
```

2. Verifica que la instalación se realizó correctamente, comprobando la versión
   de Yarn:

```terminal
$ yarn --version
```

Verás el siguiente resultado:

```
1.22.1
```

</Accordion>
</Accordions>

#### Instalación

Puedes instalar la CLI de Anchor y sus herramientas de dos maneras:

- Anchor Version Manager (AVM) — Método de instalación recomendado
- Sin AVM — Instalar directamente desde GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

El Administrador de Versiones de Anchor (AVM) te permite instalar y gestionar
diferentes versiones de Anchor en tu sistema y actualizar fácilmente las
versiones de Anchor en el futuro. Para instalar Anchor usando AVM, sigue los
pasos a continuación:

1. Instala AVM con el siguiente comando:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor avm --force
```

2. Confirma que AVM se instaló correctamente:

```terminal
$ avm --version
```

3. Instala Anchor CLI usando AVM:

Para instalar la última versión:

```terminal
$ avm install latest
$ avm use latest
```

Para instalar una versión específica, especifica el número de versión:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="warn">

Al instalar Anchor CLI en Linux o WSL, es posible que encuentres este error:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Si ves este mensaje de error, sigue estos pasos:

1. Instala las dependencias listadas en la sección
   [Linux](/docs/intro/installation#install-dependencies) al principio de esta
   página.
2. Intenta instalar Anchor CLI nuevamente.

</Callout>

4. Verifica que la instalación se realizó correctamente, comprobando la versión
   de Anchor CLI:

```terminal
$ anchor --version
```

Verás un resultado como el siguiente:

```
anchor-cli 0.31.1
```

<Callout type="info">

No olvides ejecutar el comando _shell`avm use`_ para declarar la versión de
Anchor CLI que se ejecutará en tu sistema.

- Si instalaste la versión `latest`, ejecuta _shell`avm use latest`_.
- Si instalaste la versión `0.30.1`, ejecuta _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

1. Instala una versión específica de Anchor CLI con el siguiente comando:

```terminal
$ cargo install --git https://github.com/solana-foundation/anchor --tag v0.30.1 anchor-cli
```

2. Verifica que la instalación se realizó correctamente, comprobando la versión
   de Anchor CLI:

```terminal
$ anchor --version
```

Verás un resultado como el siguiente:

```
anchor-cli 0.31.1
```

</Tab>
</Tabs>
