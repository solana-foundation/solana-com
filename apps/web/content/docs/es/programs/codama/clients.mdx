  Guía completa para generar clientes para programas de Solana utilizando Codama

[Codama](https://github.com/codama-idl/codama) genera clientes en TypeScript y
Rust a partir del archivo IDL de un programa. Los clientes generados incluyen
funciones para llamar a instrucciones y obtener cuentas, eliminando la necesidad
de serializar y deserializar manualmente las cuentas del programa o instruction
data.

<Callout type="warn">
  Codama está en desarrollo activo y sujeto a cambios. Consulta los repositorios
  enlazados en la referencia rápida para obtener la información más reciente.
</Callout>

## Referencia rápida

### Comandos CLI

| Comando       | Descripción                                                    |
| ------------- | -------------------------------------------------------------- |
| `codama init` | Generar un archivo de configuración de codama                  |
| `codama run`  | Generar clientes de programa desde el archivo de configuración |

### Generadores de clientes

| Paquete                                                                                                  | Descripción                                                                                 |
| -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| [`@codama/renderers-js`](https://github.com/codama-idl/renderers-js)                                     | Genera clientes TypeScript compatibles con [`@solana/kit`](https://www.solanakit.com)       |
| [`@codama/renderers-rust`](https://github.com/codama-idl/renderers-rust)                                 | Genera clientes Rust compatibles con [`solana_sdk`](https://github.com/anza-xyz/solana-sdk) |
| [`@codama/nodes-from-anchor`](https://github.com/codama-idl/codama/tree/main/packages/nodes-from-anchor) | Convierte IDLs de Anchor a IDLs de Codama para generar clientes                             |

### Recursos adicionales

| Recurso                                                                                                    | Descripción                                                                                        |
| ---------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| [Codama](https://github.com/codama-idl/codama)                                                             | GitHub de Codama                                                                                   |
| [Codama CLI](https://github.com/codama-idl/codama/blob/main/packages/cli/README.md)                        | GitHub de Codama CLI                                                                               |

## !!steps Crear archivo de configuración de Codama

Codama utiliza un archivo de configuración para especificar dónde se encuentra
el archivo IDL, qué lenguajes de cliente generar y dónde guardar el código
generado. Crea esta configuración ejecutando:

```terminal
$ npx codama init
```

Esto te mostrará preguntas para crear un archivo `codama.json` en la raíz de tu
proyecto e instalar las dependencias necesarias de Codama.

## !!steps Generar clientes

Una vez creado el archivo de configuración, ejecuta el comando de generación
para generar los clientes:

```terminal
$ npx codama run --all
```

<WithMentions>

Esto generará los clientes del programa en las [rutas de archivo](mention:path)
especificadas en el archivo de configuración de codama.

```json title="codama.json"
{
  "idl": "idl.json",
  "before": [],
  "scripts": {
    "js": {
      "from": "@codama/renderers-js",
      // !mark
      // !mention path
      "args": ["clients/js/src/generated"]
    },
    "rust": {
      "from": "@codama/renderers-rust",
      "args": [
        // !mark
        // !mention path
        "clients/rust/src/generated",
        {
          "crateFolder": "clients/rust",
          "formatCode": true
        }
      ]
    }
  }
}
```

</WithMentions>

</Callout>

## Uso de los clientes generados

Los clientes generados incluyen código fuente para interactuar con tu programa
pero no incluyen archivos `package.json` (para TypeScript) o `Cargo.toml` (para
Rust).

Necesitarás crear nuevos archivos `package.json` y `Cargo.toml` o agregar las
dependencias requeridas de los clientes generados a los archivos existentes de
tu proyecto.

### Integración en tu aplicación

Genera el cliente directamente en el directorio de código fuente de tu
aplicación y luego agrega las dependencias requeridas a tus archivos
`package.json` o `Cargo.toml` existentes.

Utiliza este enfoque cuando construyas una aplicación donde el código del
cliente no necesita ser compartido o publicado.

### Paquetes o crates independientes

Crea paquetes de TypeScript o crates de Rust separados para cada cliente. Por
ejemplo, agrega `package.json` en `clients/js/` y `Cargo.toml` en
`clients/rust/`.

Utiliza este enfoque para publicar el cliente como una biblioteca reutilizable
(npm o crates.io) o compartir el cliente entre múltiples aplicaciones en un
monorepo.
