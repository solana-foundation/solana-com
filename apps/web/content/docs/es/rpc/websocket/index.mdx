Después de conectarse al websocket RPC PubSub en `ws://<ADDRESS>/`:

- Envía solicitudes de suscripción al websocket utilizando los métodos a
  continuación
- Pueden estar activas múltiples suscripciones a la vez
- Muchas suscripciones aceptan el parámetro opcional
  [`commitment` parameter](/docs/rpc/#configuring-state-commitment), que define
  cuán finalizado debe estar un cambio para activar una notificación. Para
  suscripciones, si no se especifica commitment, el valor predeterminado es
  `finalized`.

## Punto de conexión WebSocket RPC PubSub

Puerto predeterminado: `8900`

- ws://localhost:8900
- http://192.168.1.88:8900
