h1: Metode RPC getHealth
---

Mengembalikan kesehatan node saat ini. Node yang sehat adalah node yang berada
dalam `HEALTH_CHECK_SLOT_DISTANCE` slot dari slot terkonfirmasi terbaru klaster.

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getHealth"
}
```

```ts !!request title="Kit"
import { createSolanaRpc } from "@solana/kit";

const rpc_url = "https://api.devnet.solana.com";
const rpc = createSolanaRpc(rpc_url);

let health = await rpc.getHealth().send();

console.log(health);
```

```rs !!request title="Rust"
use anyhow::Result;
use solana_client::nonblocking::rpc_client::RpcClient;
use solana_sdk::commitment_config::CommitmentConfig;

#[tokio::main]
async fn main() -> Result<()> {
    let client = RpcClient::new_with_commitment(
        String::from("https://api.devnet.solana.com"),
        CommitmentConfig::confirmed(),
    );

    let health = client.get_health().await?;

    println!("{:#?}", health);

    Ok(())
}
```

### !params

### !!result Sehat

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": "ok",
  "id": 1
}
```

!type string

Jika node sehat: "ok"

### !!result Tidak Sehat

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:5) result
  "error": {
    "code": -32005,
    "message": "Node is unhealthy",
    "data": { "numSlotsBehind": 42 }
  },
  "id": 1
}
```

!type object

Jika node tidak sehat, respons error JSON RPC akan dikembalikan. Spesifikasi
dari respons error tersebut **TIDAK STABIL** dan dapat berubah di masa mendatang

</APIMethod>
