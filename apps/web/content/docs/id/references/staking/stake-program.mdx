---
title: Pemrograman Staking
---

Untuk memaksimalkan distribusi stake, desentralisasi, dan ketahanan terhadap
sensor pada jaringan Solana, staking dapat dilakukan secara terprogram. Tim dan
komunitas telah mengembangkan beberapa program on-chain dan offchain untuk
membuat stake lebih mudah dikelola.

### Program Delegasi Solana Foundation

[Program Delegasi Solana Foundation](https://solana.org/delegation-program)
dirancang untuk melayani kebutuhan jaringan dengan menargetkan tujuan-tujuan
berikut:

1. Memaksimalkan desentralisasi, keandalan, dan performa Solana
2. Memaksimalkan jumlah validator yang memiliki sumber stake yang beragam
3. Mempertahankan testnet yang besar dan representatif

### Stake Pools

Program on-chain ini menggabungkan SOL untuk di-stake oleh seorang manajer,
memungkinkan pemegang SOL untuk melakukan staking dan mendapatkan rewards tanpa
perlu mengelola stake. Pengguna menyetor SOL sebagai pertukaran untuk token SPL
(derivatif staking) yang mewakili kepemilikan mereka di stake pool. Manajer pool
melakukan staking SOL yang disetor sesuai dengan strategi mereka, mungkin
menggunakan varian bot auto-delegasi seperti yang dijelaskan di atas. Saat stake
menghasilkan rewards, pool dan token pool bertambah nilainya secara
proporsional. Akhirnya, pemegang token pool dapat mengirim token SPL kembali ke
stake pool untuk menukarkan SOL, sehingga berpartisipasi dalam desentralisasi
dengan usaha yang jauh lebih sedikit. Informasi lebih lanjut dapat ditemukan di
[dokumentasi SPL stake pool](https://spl.solana.com/stake-pool).
