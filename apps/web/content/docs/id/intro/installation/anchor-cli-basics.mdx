Bagian ini menyediakan beberapa perintah umum dan contoh untuk membantu Anda
memulai menggunakan [Anchor CLI](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Inisialisasi proyek

Buat proyek Anchor baru dengan menjalankan perintah yang ditunjukkan di bawah
ini. Perintah ini akan membuat direktori baru dengan nama proyek dan
menggunakannya untuk menginisialisasi proyek Anchor baru.

```terminal
$ anchor init <project-name>
```

Misalnya, perintah di bawah ini akan membuat proyek bernama `my-project`.
Direktori `my-project` akan berisi program Rust dasar dan template pengujian
TypeScript.

```terminal
$ anchor init my-project
```

Kemudian navigasikan ke direktori proyek:

```terminal
$ cd <project-name>
```

Lihat
[struktur file proyek](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
Anchor.

</Step>
<Step>

### Build program

Untuk mem-build proyek Anda, jalankan perintah berikut:

```terminal
$ anchor build
```

Anda dapat menemukan program yang telah dikompilasi di direktori
`/target/deploy`.

<Callout type="warn">

Ketika menjalankan `anchor build`, jika Anda mengalami kesalahan berikut:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Coba solusi berikut:

1. Paksa instalasi menggunakan perintah berikut:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Jika solusi sebelumnya tidak berhasil, bersihkan cache Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Anda dapat memperbaiki ini dengan mengubah bidang versi file `Cargo.lock` dari 4 menjadi 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Lihat [masalah ini](https://github.com/solana-foundation/anchor/issues/3392)
untuk informasi lebih lanjut.

</Accordion>

</Accordions>

Setelah menerapkan solusi di atas, coba jalankan _shell`anchor build`_ lagi.

</Callout>

</Step>
<Step>

### Deploy program

Untuk men-deploy proyek Anda, jalankan perintah berikut:

```terminal
$ anchor deploy
```

Perintah ini men-deploy program Anda ke `cluster` yang ditentukan dalam file
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Uji program

Untuk menguji proyek Anda, jalankan perintah berikut:

```terminal
$ anchor test
```

Perintah ini membangun, men-deploy, dan menjalankan pengujian untuk proyek Anda.

<Callout>
  Ketika menggunakan cluster `localnet`, Anchor secara otomatis memulai
  validator lokal, men-deploy program, menjalankan pengujian, kemudian
  menghentikan validator.
</Callout>

<Callout type="warn">

Salah satu dari kesalahan berikut mungkin menunjukkan bahwa Anda belum
menginstal Node.js atau Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>
</Steps>
