솔라나에서 "스마트 계약"은 프로그램이라고 불립니다.
[프로그램](/docs/core/accounts#program-account)은 프로그램의 컴파일된 실행 가능한 바이너리를 포함하는 계정에 체인 상에서 배포됩니다. 사용자는 프로그램에
[instructions](/docs/core/transactions#instruction)를 포함한 트랜잭션을 보내어 프로그램이 수행할 작업을 지시합니다.

## 주요 사항

- 프로그램은 **실행 가능한 코드**를 포함하는 계정으로, **지침**이라고 불리는 함수로 구성됩니다.
- 프로그램은 **무상태**이지만, 데이터를 저장하기 위해 다른 계정을 생성하고 업데이트하는 지침을 포함할 수 있습니다.
- **업그레이드 권한**이 프로그램을 업데이트할 수 있습니다. 이 권한이 제거되면 프로그램은 변경할 수 없습니다.
- 사용자는 검증 가능한 빌드를 통해 체인 상의 프로그램 계정 데이터가 공개 소스 코드와 일치하는지 확인할 수 있습니다.

## 솔라나 프로그램 작성

솔라나 프로그램은 주로 [Rust](https://rust-book.cs.brown.edu/title-page.html) 프로그래밍 언어로 작성되며,
개발을 위한 두 가지 일반적인 접근 방식이 있습니다:

- [Anchor](https://www.anchor-lang.com/docs): 솔라나 프로그램 개발을 위해 설계된 프레임워크입니다.
  Rust 매크로를 사용하여 보일러플레이트 코드를 크게 줄여 프로그램을 더 빠르고 간단하게 작성할 수 있도록 합니다. 초보자에게는 Anchor 프레임워크로 시작하는 것이 권장됩니다.

- [네이티브 Rust](/docs/programs/rust): 이 접근 방식은 프레임워크를 사용하지 않고 Rust로 솔라나 프로그램을 작성하는 것입니다. 더 많은 유연성을 제공하지만 복잡성이 증가합니다.

## 솔라나 프로그램 업데이트

프로그램 배포 및 업그레이드에 대한 자세한 내용은
[프로그램 배포](/docs/programs/deploying) 페이지를 참조하세요.

체인 상의 프로그램은 "업그레이드 권한"으로 지정된 계정에 의해
[직접 수정](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/bpf_loader/src/lib.rs#L704)될 수 있으며,
이는 일반적으로 프로그램을 처음 배포한 계정입니다. 만약
[업그레이드 권한](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/bpf_loader/src/lib.rs#L894)이 취소되고 `None`으로 설정되면,
프로그램은 변경할 수 없게 됩니다.

## 검증 가능한 프로그램

검증 가능한 빌드를 통해 누구나 프로그램의 체인 상 코드가 공개 소스 코드와 일치하는지 확인할 수 있으며,
소스와 배포된 버전 간의 불일치를 감지할 수 있습니다.

솔라나 개발자 커뮤니티는 검증 가능한 빌드를 지원하는 도구를 도입하여,
개발자와 사용자가 체인 상 프로그램이 공개적으로 공유된 소스 코드를 정확하게 반영하는지 확인할 수 있도록 합니다.

- **검증된 프로그램 검색**: 검증된 프로그램을 빠르게 확인하려면,
  사용자는 [SolanaFM](https://solana.fm/) 탐색기에서 프로그램 주소를 검색하고 "검증" 탭으로 이동할 수 있습니다. 검증된 프로그램의 예시는
  [여기](https://solana.fm/address/PhoeNiXZ8ByJGLkxNfZRnkUfjvmuYqLR89jjFHGqdXY)에서 확인할 수 있습니다.

- **검증 도구**: Ellipsis Labs의
  [Solana Verifiable Build CLI](https://github.com/Ellipsis-Labs/solana-verifiable-build)는 사용자가 공개된 소스 코드에 대해 체인 상 프로그램을 독립적으로 검증할 수 있도록 합니다.

- **Anchor의 검증 가능한 빌드 지원**: Anchor는 검증 가능한 빌드에 대한 기본 지원을 제공합니다. 자세한 내용은
  [Anchor 문서](https://www.anchor-lang.com/docs/verifiable-builds)에서 확인할 수 있습니다.

## 버클리 패킷 필터 (BPF)

솔라나는 [LLVM](https://llvm.org/)을 사용하여 프로그램을
[ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format) 파일로 컴파일합니다. 이 파일은 솔라나의 커스텀 버전인
[eBPF](https://en.wikipedia.org/wiki/EBPF) 바이트코드, 즉 "솔라나 바이트코드 형식"(sBPF)을 포함합니다. ELF 파일은 프로그램의 바이너리를 포함하며,
프로그램이 배포될 때 실행 가능한 계정에 체인 상에 저장됩니다.

## 내장 프로그램

### 로더 프로그램

모든 프로그램은 다른 프로그램에 의해 소유되며, 이를 로더라고 합니다. 현재 다섯 개의 로더가 있습니다:

1. 네이티브 로더:

  - 프로그램 ID: `NativeLoader1111111111111111111111111111111`
  - 나머지 네 개의 로더를 소유합니다.

2. 로더-v1:

  - 프로그램 ID: `BPFLoader1111111111111111111111111111111111`
  - 관리 지침이 비활성화되어 있지만 프로그램은 여전히 실행됩니다.

3. 로더-v2:

  - 프로그램 ID: `BPFLoader2111111111111111111111111111111111`
  - [지침](https://docs.rs/solana-loader-v2-interface/latest/solana_loader_v2_interface/enum.LoaderInstruction.html)
  - 관리 지침이 비활성화되어 있지만 프로그램은 여전히 실행됩니다.

4. 로더-v3:

  - 프로그램 ID: `BPFLoaderUpgradeab1e11111111111111111111111`
  - [지침](https://docs.rs/solana-loader-v3-interface/latest/solana_loader_v3_interface/instruction/enum.UpgradeableLoaderInstruction.html)
  - 단계적으로 폐기되고 있습니다.

5. 로더-v4:

  - 프로그램 ID: `LoaderV411111111111111111111111111111111111`
  - [지침](https://docs.rs/solana-loader-v4-interface/latest/solana_loader_v4_interface/instruction/enum.LoaderV4Instruction.html)
  - 표준 로더가 될 것입니다.

이 로더는 사용자 정의 프로그램을 생성하고 관리하는 데 필요합니다:

- 새로운 프로그램 또는 버퍼 배포
- 프로그램 또는 버퍼 종료
- 기존 프로그램 재배포 / 업그레이드
- 프로그램에 대한 권한 이전
- 프로그램 완료

로더-v3 및 로더-v4는 초기 배포 후 프로그램 수정 지원을 제공합니다. 이를 수행할 권한은 프로그램의 권한에 의해 규제되며,
각 프로그램의 계정 소유권은 로더에 있습니다.

### 프리컴파일 프로그램

#### Ed25519 프로그램

ed25519 서명을 검증하는 프로그램입니다. ed25519 서명, 공개 키 및 메시지를 입력으로 받습니다.
여러 서명을 검증할 수 있습니다. 서명 중 하나라도 검증에 실패하면 오류가 반환됩니다.

- 프로그램 ID: `Ed25519SigVerify111111111111111111111111111`
- [지침](https://docs.rs/solana-ed25519-program/latest/solana_ed25519_program/index.html)

ed25519 프로그램은 지침을 처리합니다. 첫 번째 `u8`는 확인할 서명의 수를 나타내며, 그 뒤에 단일 바이트 패딩이 옵니다. 그 후, 다음 구조체가 직렬화되며, 확인할 각 서명마다 하나씩 존재합니다.

```
struct Ed25519SignatureOffsets {
    signature_offset: u16,             // 64바이트의 ed25519 서명에 대한 오프셋
    signature_instruction_index: u16,  // 서명을 찾기 위한 지침 인덱스
    public_key_offset: u16,            // 32바이트의 공개 키에 대한 오프셋
    public_key_instruction_index: u16, // 공개 키를 찾기 위한 지침 인덱스
    message_data_offset: u16,          // 메시지 데이터 시작에 대한 오프셋
    message_data_size: u16,            // 메시지 데이터의 크기
    message_instruction_index: u16,    // 메시지 데이터를 가져오기 위한 지침 데이터 인덱스
}
```

서명 검증의 의사 코드:

```
process_instruction() {
    for i in 0..count {
        // i번째 인덱스 값 참조:
        instructions = &transaction.message().instructions
        instruction_index = ed25519_signature_instruction_index != u16::MAX ? ed25519_signature_instruction_index : current_instruction;
        signature = instructions[instruction_index].data[ed25519_signature_offset..ed25519_signature_offset + 64]
        instruction_index = ed25519_pubkey_instruction_index != u16::MAX ? ed25519_pubkey_instruction_index : current_instruction;
        pubkey = instructions[instruction_index].data[ed25519_pubkey_offset..ed25519_pubkey_offset + 32]
        instruction_index = ed25519_message_instruction_index != u16::MAX ? ed25519_message_instruction_index : current_instruction;
        message = instructions[instruction_index].data[ed25519_message_data_offset..ed25519_message_data_offset + ed25519_message_data_size]
        if pubkey.verify(signature, message) != Success {
            return Error
        }
    }
    return Success
}
```

#### Secp256k1 프로그램

secp256k1 공개 키 복구 작업(ecrecover)을 검증합니다.

- 프로그램 ID: `KeccakSecp256k11111111111111111111111111111`
- [지침](https://docs.rs/solana-secp256k1-program/latest/solana_secp256k1_program/index.html)

secp256k1 프로그램은 지침을 처리하며, 첫 번째 바이트로 다음 구조체의 수를 입력으로 받습니다:

```
struct Secp256k1SignatureOffsets {
    secp_signature_offset: u16,            // 64+1바이트의 [서명, recovery_id]에 대한 오프셋
    secp_signature_instruction_index: u8,  // 서명을 찾기 위한 지침 인덱스
    secp_pubkey_offset: u16,               // 20바이트의 ethereum_address 공개 키에 대한 오프셋
    secp_pubkey_instruction_index: u8,     // 공개 키를 찾기 위한 지침 인덱스
    secp_message_data_offset: u16,         // 메시지 데이터 시작에 대한 오프셋
    secp_message_data_size: u16,           // 메시지 데이터의 크기
    secp_message_instruction_index: u8,    // 메시지 데이터를 찾기 위한 지침 인덱스
}
```

복구 검증의 의사 코드:

```
process_instruction() {
  for i in 0..count {
      // i번째 인덱스 값 참조:
      instructions = &transaction.message().instructions
      signature = instructions[secp_signature_instruction_index].data[secp_signature_offset..secp_signature_offset + 64]
      recovery_id = instructions[secp_signature_instruction_index].data[secp_signature_offset + 64]
      ref_eth_pubkey = instructions[secp_pubkey_instruction_index].data[secp_pubkey_offset..secp_pubkey_offset + 20]
      message_hash = keccak256(instructions[secp_message_instruction_index].data[secp_message_data_offset..secp_message_data_offset + secp_message_data_size])
      pubkey = ecrecover(signature, recovery_id, message_hash)
      eth_pubkey = keccak256(pubkey[1..])[12..]
      if eth_pubkey != ref_eth_pubkey {
          return Error
      }
  }
  return Success
}
```

이것은 사용자가 서명 및 메시지 데이터에 대한 지침 데이터를 트랜잭션에 지정할 수 있도록 합니다. 특별한 지침 sysvar를 지정하면,
트랜잭션 자체에서 데이터를 받을 수도 있습니다.

트랜잭션 비용은 검증할 서명의 수에 서명 비용 검증 배수를 곱한 값으로 계산됩니다.

#### Secp256r1 프로그램

secp256r1 서명을 검증하는 프로그램입니다. secp256r1 서명, 공개 키 및 메시지를 입력으로 받습니다. 최대 8개의 서명을 검증할 수 있습니다. 서명 중 하나라도 검증에 실패하면 오류가 반환됩니다.

- 프로그램 ID: `Secp256r1SigVerify1111111111111111111111111`
- [지침](https://docs.rs/solana-secp256k1-recover/latest/solana_secp256k1_recover/index.html)

secp256r1 프로그램은 지침을 처리합니다. 첫 번째 `u8`는 확인할 서명의 수를 나타내며, 그 뒤에 단일 바이트 패딩이 옵니다. 그 후, 다음 구조체가 직렬화되며, 확인할 각 서명마다 하나씩 존재합니다:

```rust
struct Secp256r1SignatureOffsets {
    signature_offset: u16,             // 64바이트의 압축된 secp256r1 서명에 대한 오프셋
    signature_instruction_index: u16,  // 서명을 찾기 위한 지침 인덱스
    public_key_offset: u16,            // 33바이트의 압축된 공개 키에 대한 오프셋
    public_key_instruction_index: u16, // 공개 키를 찾기 위한 지침 인덱스
    message_data_offset: u16,          // 메시지 데이터 시작에 대한 오프셋
    message_data_size: u16,            // 메시지 데이터의 크기
    message_instruction_index: u16,    // 메시지 데이터를 가져오기 위한 지침 데이터 인덱스
}

```

서명 검증의 의사 코드:
```
process_instruction() {
    if data.len() < SIGNATURE_OFFSETS_START {
        return Error
    }

    num_signatures = data[0] as usize
    if num_signatures == 0 || num_signatures > 8 {
        return Error
    }

    expected_data_size = num_signatures * SIGNATURE_OFFSETS_SERIALIZED_SIZE + SIGNATURE_OFFSETS_START
    if data.len() < expected_data_size {
        return Error
    }

    for i in 0..num_signatures {
        offsets = parse_signature_offsets(data, i)

        signature = get_data_slice(data, instruction_datas, offsets.signature_instruction_index, offsets.signature_offset, SIGNATURE_SERIALIZED_SIZE)

        if s > half_curve_order {
            return Error
        }

        pubkey = get_data_slice(data, instruction_datas, offsets.public_key_instruction_index, offsets.public_key_offset, COMPRESSED_PUBKEY_SERIALIZED_SIZE)

        message = get_data_slice(data, instruction_datas, offsets.message_instruction_index, offsets.message_data_offset, offsets.message_data_size)

        if !verify_signature(signature, pubkey, message) {
            return Error
        }
    }

    return Success
}
```
노트: 모든 서명에 대해 낮은 S 값이 적용되어 우발적인 서명 변형을 방지합니다.

### 코어 프로그램

솔라나 클러스터 제네시스에는 네트워크의 다양한 핵심 기능을 제공하는 특별 프로그램 목록이 포함되어 있습니다. 역사적으로 이들은 "네이티브" 프로그램이라고 불렸으며,
검증자 코드와 함께 배포되었습니다.

1. 시스템 프로그램

- 프로그램 ID: `11111111111111111111111111111111`
- [지침](https://docs.rs/solana-program/latest/solana_program/system_instruction/enum.SystemInstruction.html)
- 새로운 계정을 생성하고, 계정 데이터를 할당하며, 계정을 소유 프로그램에 할당하고,
시스템 프로그램 소유 계정에서 램포트를 전송하고 트랜잭션 수수료를 지불합니다.

2. 투표 프로그램

- 프로그램 ID: `Vote111111111111111111111111111111111111111`
- [지침](https://docs.rs/solana-vote-program/latest/solana_vote_program/vote_instruction/enum.VoteInstruction.html)
- 검증자 투표 상태 및 보상을 추적하는 계정을 생성하고 관리합니다.

3. 스테이크 프로그램

- 프로그램 ID: `Stake11111111111111111111111111111111111111`
- [지침](https://docs.rs/solana-sdk/latest/solana_sdk/stake/instruction/enum.StakeInstruction.html)
- 검증자에게 위임된 스테이크 및 보상을 나타내는 계정을 생성하고 관리합니다.

4. 구성 프로그램
