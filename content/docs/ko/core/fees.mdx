모든 솔라나 트랜잭션은 검증자가 트랜잭션을 처리하는 데 대한 보상으로 기본
수수료(SOL)가 필요합니다. 현재 리더(검증자)가 트랜잭션을 처리할 확률을 높이기
위해 선택적으로 우선순위 수수료를 추가할 수 있습니다.

## 주요 포인트

- 트랜잭션의 **기본 수수료**는 트랜잭션의 서명당 최소 5000 램포트입니다.
- **우선순위 수수료**(선택 사항)는 트랜잭션이 처리될 확률을 높이기 위해
  검증자에게 지불하는 추가 수수료입니다.
- 우선순위 수수료는 다음과 같이 계산됩니다: **(컴퓨트 유닛 한도 \* 컴퓨트 유닛
  가격)**.
- **컴퓨트 유닛 한도**는 트랜잭션이 소비할 수 있는 최대 컴퓨트 유닛 수입니다.
- **컴퓨트 유닛 가격**은 마이크로 램포트 단위의 컴퓨트 유닛당 가격입니다.
- 1,000,000 마이크로 램포트 = 1 램포트
- 트랜잭션 수수료 지불자는 시스템 프로그램이 소유한 계정이어야 합니다.

## 기본 트랜잭션 수수료

기본 수수료는 트랜잭션을 보내는 최소 비용입니다. 비용은 트랜잭션에 포함된 서명당
5000 램포트입니다.

기본 수수료는 트랜잭션의 첫 번째 서명자인 트랜잭션 수수료 지불자가 자동으로
지불합니다. 수수료 지불자는 시스템 프로그램이 소유한 계정이어야 합니다.

- [50% 소각](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/fee-calculator/src/lib.rs#L70):
  기본 수수료의 절반은 소각됩니다.
- [50% 분배](https://github.com/anza-xyz/agave/blob/e621336acad4f5d6e5b860eaa1b074b01c99253c/runtime/src/bank/fee_distribution.rs#L58-L62):
  나머지 절반은 트랜잭션을 처리한 검증자에게 지급됩니다.

## 우선순위 수수료

우선순위 수수료는 트랜잭션이 처리될 확률을 높이기 위해 검증자에게 지불하는
선택적 수수료입니다.

- [SIMD-0096](https://github.com/solana-foundation/solana-improvement-documents/blob/main/proposals/0096-reward-collected-priority-fee-in-entirety.md):
  우선순위 수수료의 100%는 트랜잭션을 처리하는 검증자에게 직접 지급됩니다.

### 컴퓨트 유닛 및 제한

트랜잭션이 실행될 때, 컴퓨트 유닛(CU)으로 측정되는 계산 리소스를 소비합니다. 각
명령어는 트랜잭션의 컴퓨트 유닛 예산에서 차감됩니다.

- 최대 제한: 트랜잭션은 최대
  [140만 컴퓨트 유닛](https://github.com/anza-xyz/agave/blob/v2.1.13/compute-budget/src/compute_budget_limits.rs#L14)까지
  사용할 수 있습니다.
- 기본 제한: 기본적으로 각 명령어는
  [200,000 컴퓨트 유닛](https://github.com/anza-xyz/agave/blob/v2.1.13/compute-budget/src/compute_budget_limits.rs#L10)으로
  제한됩니다.
- 사용자 지정 제한: 트랜잭션에 `SetComputeUnitLimit` 명령어를 포함하여 특정
  컴퓨트 유닛 제한을 요청할 수 있습니다.

### 컴퓨트 유닛 가격

마이크로 램포트(micro lamports)로 표시되는 컴퓨트 유닛 가격은 우선순위 수수료를
계산하기 위해 컴퓨트 유닛당 지불하는 선택적 가격입니다.

<Callout>1,000,000 마이크로 램포트 = 1 램포트</Callout>

현재 컴퓨트 유닛 가격에 대한 실시간 추천을 얻으려면 다음 리소스를 사용하세요:

- Helius의
  [우선순위 수수료 API](https://docs.helius.dev/solana-apis/priority-fee-api)
- Triton의
  [글로벌 우선순위 수수료 트래커](https://triton.one/solana-prioritization-fees/)

다음을 참조하세요

[우선순위 수수료 사용 방법](/developers/guides/advanced/how-to-use-priority-fees)

우선순위 수수료에 대한 자세한 내용은 가이드를 참조하세요.

### 우선순위 수수료 계산

우선순위 수수료는 다음과 같이 계산됩니다:

**우선순위 수수료 = 컴퓨트 유닛 제한 × 컴퓨트 유닛 가격**

트랜잭션에서 컴퓨트 유닛 제한과 가격을 설정하려면 다음 명령어를 사용하세요:

- [`SetComputeUnitLimit`](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/src/compute_budget.rs#L42-L44)로
  특정 유닛 제한을 설정합니다.
- [`SetComputeUnitPrice`](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/src/compute_budget.rs#L48-L50)로
  컴퓨트 유닛당 가격을 정의합니다.

<Callout>

이러한 명령어가 제공되지 않으면, 트랜잭션은 자동으로 기본 컴퓨트 유닛 제한을
사용하며 컴퓨트 유닛 가격은 0(우선순위 수수료 없음)이 됩니다.

우선순위 수수료는 트랜잭션이 실제로 사용한 컴퓨트 유닛이 아니라 트랜잭션에
요청된 컴퓨트 유닛을 기준으로 합니다. 너무 높은 컴퓨트 유닛 제한을 설정하거나
기본 양을 사용하는 경우, 사용하지 않은 컴퓨트 유닛에 대해 비용을 지불할 수
있습니다.

</Callout>

### 예시

아래는 트랜잭션의 컴퓨팅 유닛 한도와 가격을 설정하는 데 사용되는 명령어의
예시입니다.

#### Rust

`solana-sdk`는 다음을 통해 메서드를 제공합니다

[`ComputeBudgetInstruction`](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/src/compute_budget.rs#L17-L31)

타입:

```rust
let limit_instruction = ComputeBudgetInstruction::set_compute_unit_limit(300_000);

let price_instruction = ComputeBudgetInstruction::set_compute_unit_price(1);
```

#### JavaScript

`@solana/web3.js` 라이브러리(v1)는 다음을 통해 메서드를 제공합니다

[`ComputeBudgetProgram`](https://solana-labs.github.io/solana-web3.js/v1.x/classes/ComputeBudgetProgram.html)

클래스:

```js
const limitInstruction = ComputeBudgetProgram.setComputeUnitLimit({
  units: 300_000
});

const priceInstruction = ComputeBudgetProgram.setComputeUnitPrice({
  microLamports: 1
});
```
