h1: getSupply RPC 메소드
---

현재 공급량에 대한 정보를 반환합니다.

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getSupply",
  "params": [
    // !hover(1:3) 0
    {
      // !hover commitment
      "commitment": "finalized"
    }
  ]
}
```

```ts !!request title="Kit"
import { createSolanaRpc } from "@solana/kit";

const rpc_url = "https://api.devnet.solana.com";
const rpc = createSolanaRpc(rpc_url);

let supply = await rpc.getSupply().send();

console.log(supply);
```

```ts !!request title="web3.js"
import { Connection, clusterApiUrl } from "@solana/web3.js";

const connection = new Connection(clusterApiUrl("devnet"), "confirmed");

let supply = await connection.getSupply();

console.log(supply);
```

### !params

#### !! 0

!type object

다음 필드를 포함하는 구성 객체:

##### !! commitment

!type string

커밋먼트는 해당 시점에서 블록이 얼마나 확정되었는지를 설명합니다.
[상태 커밋먼트 구성](/docs/rpc#configuring-state-commitment)을 참조하세요.

##### !! excludeNonCirculatingAccountsList

!type bool

응답에서 비유통 계정 목록 제외

!type object

결과는 다음을 포함하는 JSON 객체입니다:

##### !! circulating

!type u64

lamport 단위의 유통 공급량

##### !! nonCirculating

!type u64

lamport 단위의 비유통 공급량

##### !! nonCirculatingAccounts

!type array

비유통 계정의 주소 배열(문자열 형식). `excludeNonCirculatingAccountsList`가
활성화된 경우, 반환되는 배열은 비어 있습니다.

##### !! total

!type u64

lamport 단위의 총 공급량

</APIMethod>
