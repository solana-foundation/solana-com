h1: blockSubscribe RPC 메소드
---

새 블록이 `confirmed` 되거나 `finalized` 될 때마다 알림을 받기 위한 구독입니다.

<Callout type="warn" title={"불안정한 메소드"}>
  이 구독은 **불안정**한 것으로 간주되며 validator가
  `--rpc-pubsub-enable-block-subscription` 플래그와 함께 시작된 경우에만 사용할
  수 있습니다. 이 구독의 형식은 향후 변경될 수 있습니다.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    // !hover(1:3) 0
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    // !hover(1:6) 1
    {
      // !hover commitment
      "commitment": "confirmed",
      // !hover encoding
      "encoding": "base64",
      // !hover transactionDetails
      "transactionDetails": "full",
      // !hover maxSupportedTransactionVersion
      "maxSupportedTransactionVersion": 0,
      // !hover showRewards
      "showRewards": true
    }
  ]
}
```

### !params

#### !! 0

!type string | object  
!required

계정 유형별로 결과를 받기 위한 로그의 필터 기준; 현재 지원되는 항목:

- `all` - 블록의 모든 트랜잭션 포함
- 다음 필드가 있는 JSON 객체:
  - `mentionsAccountOrProgram: <string>` - 제공된 공개 키(base-58로 인코딩된
    문자열)를 언급하는 트랜잭션만 반환합니다. 주어진 블록에 언급이 없으면 알림이
    전송되지 않습니다.

#### !! 1

!type object

다음 필드를 포함하는 구성 객체:

##### !! commitment

!type string
!values confirmed finalized
!default finalized

commitment는 해당 시점에서 블록이 얼마나 확정되었는지를 설명합니다.
[상태 커밋 구성](/docs/rpc#configuring-state-commitment)을 참조하세요.

- `processed` 는 지원되지 않습니다.

##### !! encoding

!type string  
!values json jsonParsed base58 base64  
!default json

반환된 각 트랜잭션의 인코딩 형식

- `jsonParsed` 는 프로그램별 명령어 파서를 사용하여
  `transaction.message.instructions` 목록에서 더 읽기 쉽고 명시적인 데이터를
  반환하려고 시도합니다.
- `jsonParsed` 가 요청되었지만 파서를 찾을 수 없는 경우, 명령어는 일반 JSON
  인코딩(`accounts`, `data`, 및 `programIdIndex` 필드)으로 대체됩니다.

##### !! transactionDetails

!type string  
!values full accounts signatures none  
!default full

반환할 트랜잭션 세부 정보 수준

- `accounts`가 요청되면, 트랜잭션 세부 정보에는 서명과 각 트랜잭션의 주석이 달린
  계정 목록만 포함됩니다.
- 트랜잭션 메타데이터는 다음으로만 제한됩니다: fee, err, pre_balances,
  post_balances, pre_token_balances, post_token_balances.

##### !! maxSupportedTransactionVersion

!type number
!values 0
!default 0

현재, 이 매개변수에 유효한 값은 `0`뿐입니다. 이를 `0`으로 설정하면 버전이 지정된
트랜잭션과 레거시 트랜잭션을 포함한 모든 트랜잭션을 가져올 수 있습니다.

이 매개변수는 응답에서 반환될 최대 트랜잭션 버전을 결정합니다. 이 값보다 높은
버전의 트랜잭션을 요청하면 오류가 반환됩니다. 이 매개변수를 생략하면 레거시
트랜잭션만 반환되며, 버전이 지정된 트랜잭션은 오류를 발생시킵니다.

##### !! showRewards

!type bool

`rewards` 배열을 채울지 여부. 매개변수가 제공되지 않으면 기본적으로 보상이
포함됩니다.

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

구독 ID (구독 취소에 필요)

</APIMethod>
