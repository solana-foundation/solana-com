सोलाना पर, "स्मार्ट कॉन्ट्रैक्ट्स" को प्रोग्राम्स कहा जाता है।
[प्रोग्राम्स](/docs/core/accounts#program-account) को ऐसे अकाउंट्स पर ऑन-चेन
डिप्लॉय किया जाता है जिनमें प्रोग्राम का कंपाइल किया हुआ एक्जीक्यूटेबल बाइनरी
होता है। उपयोगकर्ता प्रोग्राम्स के साथ
[इंस्ट्रक्शन्स](/docs/core/transactions#instruction) वाले ट्रांजैक्शन्स भेजकर
इंटरैक्ट करते हैं जो प्रोग्राम को बताते हैं कि क्या करना है।

## मुख्य बिंदु

- प्रोग्राम्स **एक्जीक्यूटेबल कोड** वाले अकाउंट्स हैं, जिन्हें **इंस्ट्रक्शन्स**
  नामक फंक्शन्स में व्यवस्थित किया गया है।
- जबकि प्रोग्राम्स **स्टेटलेस** होते हैं, वे ऐसे इंस्ट्रक्शन्स शामिल कर सकते हैं
  जो डेटा स्टोर करने के लिए अन्य अकाउंट्स बनाते और अपडेट करते हैं।
- एक **अपग्रेड अथॉरिटी** प्रोग्राम्स को अपडेट कर सकती है। एक बार यह अथॉरिटी हटा
  दी जाए, तो प्रोग्राम अपरिवर्तनीय हो जाता है।
- उपयोगकर्ता वेरिफायबल बिल्ड्स के माध्यम से यह सत्यापित कर सकते हैं कि ऑन-चेन
  प्रोग्राम अकाउंट का डेटा उसके पब्लिक सोर्स कोड से मेल खाता है।

## सोलाना प्रोग्राम्स लिखना

सोलाना प्रोग्राम्स मुख्य रूप से
[रस्ट](https://rust-book.cs.brown.edu/title-page.html) प्रोग्रामिंग लैंग्वेज में
लिखे जाते हैं, विकास के लिए दो सामान्य दृष्टिकोण हैं:

- [Anchor](https://www.anchor-lang.com/docs): सोलाना प्रोग्राम डेवलपमेंट के लिए
  डिज़ाइन किया गया एक फ्रेमवर्क। यह रस्ट मैक्रोज़ का उपयोग करके बॉयलरप्लेट कोड
  को कम करके प्रोग्राम लिखने का एक तेज़ और सरल तरीका प्रदान करता है। शुरुआती
  लोगों के लिए, Anchor फ्रेमवर्क से शुरुआत करने की सिफारिश की जाती है।

- [नेटिव रस्ट](/docs/programs/rust): इस दृष्टिकोण में किसी भी फ्रेमवर्क का उपयोग
  किए बिना रस्ट में सोलाना प्रोग्राम लिखना शामिल है। यह अधिक लचीलापन प्रदान करता
  है लेकिन इसमें अधिक जटिलता भी आती है।

## सोलाना प्रोग्राम्स को अपडेट करना

प्रोग्राम्स को डिप्लॉय और अपग्रेड करने के बारे में अधिक जानने के लिए,
[प्रोग्राम्स डिप्लॉय करना](/docs/programs/deploying) पेज देखें।

प्रोग्राम्स को "अपग्रेड अथॉरिटी" के रूप में नामित अकाउंट द्वारा
[सीधे संशोधित](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/bpf_loader/src/lib.rs#L704)
किया जा सकता है, जो आमतौर पर वह अकाउंट होता है जिसने मूल रूप से प्रोग्राम को
डिप्लॉय किया था। यदि
[अपग्रेड अथॉरिटी](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/bpf_loader/src/lib.rs#L894)
को रद्द कर दिया जाता है और `None` पर सेट किया जाता है, तो प्रोग्राम अपरिवर्तनीय
हो जाता है और इसे अब अपडेट नहीं किया जा सकता है।

## सत्यापन योग्य प्रोग्राम

सत्यापन योग्य बिल्ड किसी को भी यह जांचने की अनुमति देते हैं कि क्या प्रोग्राम का
ऑन-चेन कोड उसके सार्वजनिक सोर्स कोड से मेल खाता है, जिससे स्रोत और तैनात
संस्करणों के बीच विसंगतियों का पता लगाना संभव हो जाता है।

Solana डेवलपर समुदाय ने सत्यापन योग्य बिल्ड का समर्थन करने के लिए उपकरण पेश किए
हैं, जो डेवलपर्स और उपयोगकर्ताओं दोनों को यह सत्यापित करने में सक्षम बनाते हैं
कि ऑन-चेन प्रोग्राम उनके सार्वजनिक रूप से साझा किए गए सोर्स कोड को सही ढंग से
दर्शाते हैं।

- **सत्यापित प्रोग्राम खोजना**: सत्यापित प्रोग्रामों की त्वरित जांच के लिए,
  उपयोगकर्ता [Solana Explorer](https://explorer.solana.com/) पर प्रोग्राम एड्रेस
  खोज सकते हैं। सत्यापित प्रोग्राम का एक उदाहरण
  [यहां](https://explorer.solana.com/address/PhoeNiXZ8ByJGLkxNfZRnkUfjvmuYqLR89jjFHGqdXY)
  देखें।

- **सत्यापन उपकरण**: Ellipsis Labs द्वारा
  [Solana Verifiable Build CLI](https://github.com/Ellipsis-Labs/solana-verifiable-build)
  उपयोगकर्ताओं को प्रकाशित सोर्स कोड के खिलाफ स्वतंत्र रूप से ऑन-चेन प्रोग्राम
  सत्यापित करने की अनुमति देता है।

- **Anchor में सत्यापन योग्य बिल्ड के लिए समर्थन**: Anchor सत्यापन योग्य बिल्ड
  के लिए अंतर्निहित समर्थन प्रदान करता है। विवरण
  [Anchor दस्तावेज़ीकरण](https://www.anchor-lang.com/docs/verifiable-builds) में
  पाया जा सकता है।

## बर्कले पैकेट फिल्टर (BPF)

Solana प्रोग्राम को
[ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format) (Executable
and Linkable Format) फाइलों में संकलित करने के लिए [LLVM](https://llvm.org/)
(Low Level Virtual Machine) का उपयोग करता है। ये फाइलें Solana के कस्टम वर्जन
[eBPF](https://en.wikipedia.org/wiki/EBPF) बाइटकोड को शामिल करती हैं, जिसे
"Solana Bytecode Format" (sBPF) कहा जाता है। ELF फाइल में प्रोग्राम का बाइनरी
होता है और जब प्रोग्राम को डिप्लॉय किया जाता है तो इसे ऑन-चेन एक एक्जीक्यूटेबल
अकाउंट में स्टोर किया जाता है।

## अंतर्निहित प्रोग्राम

### लोडर प्रोग्राम

हर प्रोग्राम स्वयं किसी अन्य प्रोग्राम के स्वामित्व में होता है, जो उसका लोडर
है। वर्तमान में, पांच लोडर प्रोग्राम मौजूद हैं:

| लोडर   | प्रोग्राम ID                                  | नोट्स                                                                | निर्देश लिंक                                                                                                                               |
| ------ | --------------------------------------------- | -------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| native | `NativeLoader1111111111111111111111111111111` | अन्य चार लोडर्स का स्वामित्व रखता है                                 | —                                                                                                                                          |
| v1     | `BPFLoader1111111111111111111111111111111111` | प्रबंधन निर्देश अक्षम हैं, लेकिन प्रोग्राम अभी भी निष्पादित होते हैं | —                                                                                                                                          |
| v2     | `BPFLoader2111111111111111111111111111111111` | प्रबंधन निर्देश अक्षम हैं, लेकिन प्रोग्राम अभी भी निष्पादित होते हैं | [निर्देश](https://docs.rs/solana-loader-v2-interface/latest/solana_loader_v2_interface/enum.LoaderInstruction.html)                        |
| v3     | `BPFLoaderUpgradeab1e11111111111111111111111` | इसे चरणबद्ध तरीके से समाप्त किया जा रहा है                           | [निर्देश](https://docs.rs/solana-loader-v3-interface/latest/solana_loader_v3_interface/instruction/enum.UpgradeableLoaderInstruction.html) |
| v4     | `LoaderV411111111111111111111111111111111111` | v4 के मानक लोडर बनने की उम्मीद है                                    | [निर्देश](https://docs.rs/solana-loader-v4-interface/latest/solana_loader_v4_interface/instruction/enum.LoaderV4Instruction.html)          |

ये लोडर कस्टम प्रोग्राम बनाने और प्रबंधित करने के लिए आवश्यक हैं:

- नया प्रोग्राम या बफर डिप्लॉय करें
- प्रोग्राम या बफर बंद करें
- मौजूदा प्रोग्राम को रीडिप्लॉय / अपग्रेड करें
- प्रोग्राम पर अधिकार हस्तांतरित करें
- प्रोग्राम को फाइनलाइज़ करें

लोडर-v3 और लोडर-v4 प्रारंभिक डिप्लॉयमेंट के बाद प्रोग्राम में संशोधन का समर्थन
करते हैं। ऐसा करने की अनुमति प्रोग्राम के अधिकार द्वारा नियंत्रित होती है
क्योंकि प्रत्येक प्रोग्राम का खाता स्वामित्व लोडर के पास रहता है।

### प्रीकंपाइल्ड प्रोग्राम

#### Ed25519 प्रोग्राम

| प्रोग्राम         | प्रोग्राम ID                                | विवरण                                                                                                | निर्देश                                                                                    |
| ----------------- | ------------------------------------------- | ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| Ed25519 प्रोग्राम | Ed25519SigVerify111111111111111111111111111 | ed25519 हस्ताक्षरों को सत्यापित करता है। यदि कोई हस्ताक्षर विफल होता है, तो एक त्रुटि लौटाई जाती है। | [निर्देश](https://docs.rs/solana-ed25519-program/latest/solana_ed25519_program/index.html) |

ed25519 प्रोग्राम एक निर्देश को संसाधित करता है। पहला u8 जांच करने के लिए
हस्ताक्षरों की संख्या की गणना है, जिसके बाद एक बाइट पैडिंग होती है। उसके बाद,
निम्न संरचना सीरियलाइज़ की जाती है, प्रत्येक हस्ताक्षर के लिए एक जिसे जांचना है।

```rust
struct Ed25519SignatureOffsets {
    signature_offset: u16,             // offset to ed25519 signature of 64 bytes
    signature_instruction_index: u16,  // instruction index to find signature
    public_key_offset: u16,            // offset to public key of 32 bytes
    public_key_instruction_index: u16, // instruction index to find public key
    message_offset: u16,               // offset to start of message data
    message_size: u16,                 // size of message data
    message_instruction_index: u16,    // instruction index to find message data
}
```

हस्ताक्षर सत्यापन का स्यूडो कोड:

for i in 0..signatures_count { let Ed25519SignatureOffsets { signature_offset,
signature_instruction_index, public_key_offset, public_key_instruction_index,
message_offset, message_size, message_instruction_index, } =
deserialize(instruction_datas[i]);

let signature =
instruction_datas[signature_instruction_index][signature_offset..signature_offset + 64];
let public_key =
instruction_datas[public_key_instruction_index][public_key_offset..public_key_offset + 32];
let message =
instruction_datas[message_instruction_index][message_offset..message_offset + message_size];

if !ed25519_verify(signature, public_key, message) { return Error; } }

#### Secp256k1 प्रोग्राम

| प्रोग्राम           | प्रोग्राम ID                                | विवरण                                                              | निर्देश                                                                                        |
| ------------------- | ------------------------------------------- | ------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------- |
| Secp256k1 प्रोग्राम | KeccakSecp256k11111111111111111111111111111 | secp256k1 पब्लिक की रिकवरी ऑपरेशन (ecrecover) को सत्यापित करता है। | [निर्देश](https://docs.rs/solana-secp256k1-program/latest/solana_secp256k1_program/index.html) |

secp256k1 प्रोग्राम एक निर्देश को संसाधित करता है जो पहले बाइट के रूप में निम्न
संरचना की गणना लेता है जो निर्देश डेटा में सीरियलाइज़ की गई है:

```rust
struct Secp256k1SignatureOffsets {
    signature_offset: u16,             // offset to secp256k1 signature
    signature_instruction_index: u16,  // instruction index to find signature
    eth_address_offset: u16,           // offset to eth_address
    eth_address_instruction_index: u16, // instruction index to find eth_address
    message_offset: u16,               // offset to start of message data
    message_size: u16,                 // size of message data
    message_instruction_index: u16,    // instruction index to find message data
}
```

रिकवरी सत्यापन का स्यूडो कोड:

for i in 0..signatures_count { let Secp256k1SignatureOffsets { signature_offset,
signature_instruction_index, eth_address_offset, eth_address_instruction_index,
message_offset, message_size, message_instruction_index, } =
deserialize(instruction_datas[i]);

let signature =
instruction_datas[signature_instruction_index][signature_offset..signature_offset + 64];
let eth_address =
instruction_datas[eth_address_instruction_index][eth_address_offset..eth_address_offset + 20];
let message =
instruction_datas[message_instruction_index][message_offset..message_offset + message_size];

let pubkey = ecrecover(signature, message); let eth_address_computed =
keccak256(pubkey)[12..32];

if eth_address_computed != eth_address { return Error; } }

यह उपयोगकर्ता को हस्ताक्षर और संदेश डेटा के लिए लेनदेन में किसी भी निर्देश डेटा
को निर्दिष्ट करने की अनुमति देता है। एक विशेष निर्देश sysvar निर्दिष्ट करके, कोई
भी लेनदेन से स्वयं डेटा प्राप्त कर सकता है।

लेनदेन की लागत सत्यापित करने के लिए हस्ताक्षरों की संख्या को हस्ताक्षर लागत
सत्यापन गुणक से गुणा करके गिनी जाएगी।

#### Secp256r1 प्रोग्राम

| प्रोग्राम           | प्रोग्राम ID                                  | विवरण                                                                                                                                  | निर्देश                                                                                        |
| ------------------- | --------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| Secp256r1 प्रोग्राम | `Secp256r1SigVerify1111111111111111111111111` | 8 तक secp256r1 हस्ताक्षरों को सत्यापित करता है। हस्ताक्षर, सार्वजनिक कुंजी और संदेश लेता है। यदि कोई विफल होता है तो त्रुटि लौटाता है। | [निर्देश](https://docs.rs/solana-secp256k1-recover/latest/solana_secp256k1_recover/index.html) |

secp256r1 प्रोग्राम एक निर्देश को संसाधित करता है। पहला `u8` जांच करने के लिए
हस्ताक्षरों की संख्या की गणना है, उसके बाद एक बाइट पैडिंग है। इसके बाद,
निम्नलिखित संरचना सीरियलाइज़ की जाती है, प्रत्येक हस्ताक्षर के लिए एक जांच करने
के लिए:

```rs title="Secp256r1SignatureOffsets"
struct Secp256r1SignatureOffsets {
    signature_offset: u16,             // offset to compact secp256r1 signature of 64 bytes
    signature_instruction_index: u16,  // instruction index to find signature
    public_key_offset: u16,            // offset to compressed public key of 33 bytes
    public_key_instruction_index: u16, // instruction index to find public key
    message_data_offset: u16,          // offset to start of message data
    message_data_size: u16,            // size of message data
    message_instruction_index: u16,    // index of instruction data to get message data
}

```

हस्ताक्षर सत्यापन का सूडो कोड:

```
process_instruction() {
    if data.len() < SIGNATURE_OFFSETS_START {
        return Error
    }

    num_signatures = data[0] as usize
    if num_signatures == 0 || num_signatures > 8 {
        return Error
    }

    expected_data_size = num_signatures * SIGNATURE_OFFSETS_SERIALIZED_SIZE + SIGNATURE_OFFSETS_START
    if data.len() < expected_data_size {
        return Error
    }

    for i in 0..num_signatures {
        offsets = parse_signature_offsets(data, i)

        signature = get_data_slice(data, instruction_datas, offsets.signature_instruction_index, offsets.signature_offset, SIGNATURE_SERIALIZED_SIZE)

        if s > half_curve_order {
            return Error
        }

        pubkey = get_data_slice(data, instruction_datas, offsets.public_key_instruction_index, offsets.public_key_offset, COMPRESSED_PUBKEY_SERIALIZED_SIZE)

        message = get_data_slice(data, instruction_datas, offsets.message_instruction_index, offsets.message_data_offset, offsets.message_data_size)

        if !verify_signature(signature, pubkey, message) {
            return Error
        }
    }

    return Success
}
```

नोट: सभी हस्ताक्षरों के लिए कम S मान लागू किए जाते हैं ताकि आकस्मिक हस्ताक्षर
मैलिएबिलिटी से बचा जा सके।
