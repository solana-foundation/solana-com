सोलाना पर, "स्मार्ट अनुबंध" को कार्यक्रम कहा जाता है।
[कार्यक्रम](/docs/core/accounts#program-account) ऑन-चेन तैनात होते हैं
उन खातों पर जो कार्यक्रम के संकलित निष्पादन योग्य बाइनरी को रखते हैं। उपयोगकर्ता कार्यक्रमों के साथ इंटरैक्ट करते हैं
लेनदेन भेजकर जिसमें
[निर्देश](/docs/core/transactions#instruction) होते हैं जो कार्यक्रम को बताते हैं कि
क्या करना है।

## मुख्य बिंदु

- कार्यक्रम **निष्पादन योग्य कोड** वाले खाते होते हैं, जिन्हें
  **निर्देशों** के रूप में संगठित किया जाता है।
- जबकि कार्यक्रम **राज्यहीन** होते हैं, वे अन्य खातों को बनाने और अपडेट करने के लिए
  निर्देश शामिल कर सकते हैं ताकि डेटा संग्रहीत किया जा सके।
- एक **उन्नयन प्राधिकरण** कार्यक्रमों को अपडेट कर सकता है। एक बार जब यह प्राधिकरण हटा दिया जाता है,
  कार्यक्रम अपरिवर्तनीय हो जाता है।
- उपयोगकर्ता सत्यापित निर्माण के माध्यम से एक ऑन-चेन कार्यक्रम खाते के डेटा को उसके सार्वजनिक स्रोत
  कोड से मेल खाने की पुष्टि कर सकते हैं।

## सोलाना कार्यक्रम लिखना

सोलाना कार्यक्रम मुख्य रूप से
[Rust](https://rust-book.cs.brown.edu/title-page.html) प्रोग्रामिंग भाषा में लिखे जाते हैं,
विकास के लिए दो सामान्य दृष्टिकोण के साथ:

- [एंकर](https://www.anchor-lang.com/docs): सोलाना के लिए डिज़ाइन किया गया एक ढांचा
  कार्यक्रम विकास। यह कार्यक्रम लिखने के लिए एक तेज़ और सरल तरीका प्रदान करता है,
  रस्ट मैक्रोज़ का उपयोग करके बायलरप्लेट कोड को काफी कम करता है। शुरुआती लोगों के लिए,
  एंकर ढांचे के साथ शुरू करने की सिफारिश की जाती है।

- [नेटिव रस्ट](/docs/programs/rust): यह दृष्टिकोण सोलाना
  कार्यक्रमों को रस्ट में लिखने में शामिल है बिना किसी ढांचे का उपयोग किए। यह अधिक लचीलापन प्रदान करता है
  लेकिन इसके साथ जटिलता बढ़ जाती है।

## सोलाना कार्यक्रमों को अपडेट करना

कार्यक्रमों की तैनाती और उन्नयन के बारे में अधिक जानने के लिए,
[कार्यक्रमों की तैनाती](/docs/programs/deploying) पृष्ठ देखें।

ऑन-चेन कार्यक्रमों को
[प्रत्यक्ष रूप से संशोधित](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/bpf_loader/src/lib.rs#L704)
किया जा सकता है एक खाते द्वारा जिसे "उन्नयन प्राधिकरण" के रूप में नामित किया गया है, जो आमतौर पर
वह खाता होता है जिसने मूल रूप से कार्यक्रम को तैनात किया था। यदि
[उन्नयन प्राधिकरण](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/bpf_loader/src/lib.rs#L894)
हटा दिया जाता है और `None` पर सेट किया जाता है, तो कार्यक्रम अपरिवर्तनीय हो जाता है और इसे फिर से
अपडेट नहीं किया जा सकता।

## सत्यापित कार्यक्रम

सत्यापित निर्माण किसी को भी यह जांचने की अनुमति देते हैं कि एक कार्यक्रम का ऑन-चेन कोड उसके
सार्वजनिक स्रोत कोड से मेल खाता है, जिससे स्रोत और तैनात संस्करणों के बीच विसंगतियों का पता लगाना संभव हो जाता है।

सोलाना डेवलपर समुदाय ने सत्यापित
निर्माण का समर्थन करने के लिए उपकरण पेश किए हैं, जिससे डेवलपर्स और उपयोगकर्ता दोनों यह सत्यापित कर सकते हैं कि ऑनचेन कार्यक्रम
सटीक रूप से उनके सार्वजनिक रूप से साझा किए गए स्रोत कोड को दर्शाते हैं।

- **सत्यापित कार्यक्रमों की खोज**: सत्यापित कार्यक्रमों की त्वरित जांच के लिए,
  उपयोगकर्ता [SolanaFM](https://solana.fm/)
  एक्सप्लोरर पर एक कार्यक्रम पते की खोज कर सकते हैं और "सत्यापन" टैब पर जा सकते हैं। एक सत्यापित कार्यक्रम का उदाहरण देखें
  [यहाँ](https://solana.fm/address/PhoeNiXZ8ByJGLkxNfZRnkUfjvmuYqLR89jjFHGqdXY)।

- **सत्यापन उपकरण**: 
  [सोलाना सत्यापित निर्माण CLI](https://github.com/Ellipsis-Labs/solana-verifiable-build)
  द्वारा एलीप्सिस लैब्स उपयोगकर्ताओं को प्रकाशित स्रोत कोड के खिलाफ ऑनचेन कार्यक्रमों को स्वतंत्र रूप से सत्यापित करने में सक्षम बनाता है।

- **एंकर में सत्यापित निर्माण के लिए समर्थन**: एंकर सत्यापित निर्माण के लिए अंतर्निहित समर्थन प्रदान करता है। विवरण
  [एंकर दस्तावेज़ीकरण](https://www.anchor-lang.com/docs/verifiable-builds) में पाया जा सकता है।

## बर्कले पैकेट फ़िल्टर (BPF)

सोलाना [LLVM](https://llvm.org/) का उपयोग करता है कार्यक्रमों को
[ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format) फ़ाइलों में संकलित करने के लिए। ये
फ़ाइलें सोलाना के कस्टम संस्करण को शामिल करती हैं
[eBPF](https://en.wikipedia.org/wiki/EBPF) बाइटकोड, जिसे "सोलाना बाइटकोड
फॉर्मेट" (sBPF) कहा जाता है। ELF फ़ाइल कार्यक्रम के बाइनरी को शामिल करती है और इसे
तैनाती के समय एक निष्पादन योग्य खाते में ऑन-चेन संग्रहीत किया जाता है।

## अंतर्निहित कार्यक्रम

### लोडर कार्यक्रम

हर कार्यक्रम स्वयं एक अन्य कार्यक्रम द्वारा स्वामित्व में होता है, जो इसका लोडर होता है। वर्तमान में पाँच लोडर हैं:

1. नेटिव लोडर:

  - कार्यक्रम आईडी: `NativeLoader1111111111111111111111111111111`
  - अन्य चार लोडरों का स्वामित्व करता है

2. लोडर-v1:

  - कार्यक्रम आईडी: `BPFLoader1111111111111111111111111111111111`
  - प्रबंधन निर्देश निष्क्रिय हैं, लेकिन कार्यक्रम अभी भी निष्पादित होते हैं

3. लोडर-v2:

  - कार्यक्रम आईडी: `BPFLoader2111111111111111111111111111111111`
  - [निर्देश](https://docs.rs/solana-loader-v2-interface/latest/solana_loader_v2_interface/enum.LoaderInstruction.html)
  - प्रबंधन निर्देश निष्क्रिय हैं, लेकिन कार्यक्रम अभी भी निष्पादित होते हैं

4. लोडर-v3:

  - कार्यक्रम आईडी: `BPFLoaderUpgradeab1e11111111111111111111111`
  - [निर्देश](https://docs.rs/solana-loader-v3-interface/latest/solana_loader_v3_interface/instruction/enum.UpgradeableLoaderInstruction.html)
  - इसे चरणबद्ध तरीके से समाप्त किया जा रहा है

5. लोडर-v4:

  - कार्यक्रम आईडी: `LoaderV411111111111111111111111111111111111`
  - [निर्देश](https://docs.rs/solana-loader-v4-interface/latest/solana_loader_v4_interface/instruction/enum.LoaderV4Instruction.html)
  - यह मानक लोडर बन जाएगा

इन लोडरों की आवश्यकता होती है कस्टम कार्यक्रमों को बनाने और प्रबंधित करने के लिए:

- एक नया कार्यक्रम या बफर तैनात करें
- एक कार्यक्रम या बफर बंद करें
- एक मौजूदा कार्यक्रम को फिर से तैनात / उन्नत करें
- एक कार्यक्रम पर प्राधिकरण स्थानांतरित करें
- एक कार्यक्रम को अंतिम रूप दें

लोडर-v3 और लोडर-v4 प्रारंभिक तैनाती के बाद कार्यक्रमों में संशोधन का समर्थन करते हैं। ऐसा करने की अनुमति कार्यक्रम के प्राधिकरण द्वारा नियंत्रित की जाती है
क्योंकि प्रत्येक कार्यक्रम का खाता स्वामित्व लोडर के पास होता है।

### प्री-कंपाइल कार्यक्रम

#### Ed25519 कार्यक्रम

Ed25519 हस्ताक्षरों को सत्यापित करने के लिए कार्यक्रम। यह एक ed25519 हस्ताक्षर, एक सार्वजनिक कुंजी और एक संदेश लेता है।
एकाधिक हस्ताक्षरों को सत्यापित किया जा सकता है। यदि कोई भी हस्ताक्षर सत्यापित करने में विफल रहता है, तो एक त्रुटि लौटाई जाती है।

- कार्यक्रम आईडी: `Ed25519SigVerify111111111111111111111111111`
- [निर्देश](https://docs.rs/solana-ed25519-program/latest/solana_ed25519_program/index.html)

Ed25519 कार्यक्रम एक निर्देश को संसाधित करता है। पहला `u8` सत्यापित करने के लिए हस्ताक्षरों की संख्या है, जिसके बाद एक एकल बाइट पैडिंग होती है। इसके बाद, निम्नलिखित संरचना को अनुक्रमित किया जाता है, प्रत्येक हस्ताक्षर के लिए एक।

```
struct Ed25519SignatureOffsets {
    signature_offset: u16,             // 64 बाइट के ed25519 हस्ताक्षर के लिए ऑफसेट
    signature_instruction_index: u16,  // हस्ताक्षर खोजने के लिए निर्देशांक
    public_key_offset: u16,            // 32 बाइट के सार्वजनिक कुंजी के लिए ऑफसेट
    public_key_instruction_index: u16, // सार्वजनिक कुंजी खोजने के लिए निर्देशांक
    message_data_offset: u16,          // संदेश डेटा की शुरुआत के लिए ऑफसेट
    message_data_size: u16,            // संदेश डेटा का आकार
    message_instruction_index: u16,    // संदेश डेटा प्राप्त करने के लिए निर्देश डेटा का अनुक्रमांक
}
```

हस्ताक्षर सत्यापन का छद्म कोड:

```
process_instruction() {
    for i in 0..count {
        // i'th अनुक्रमांक के संदर्भित मान:
        instructions = &transaction.message().instructions
        instruction_index = ed25519_signature_instruction_index != u16::MAX ? ed25519_signature_instruction_index : current_instruction;
        signature = instructions[instruction_index].data[ed25519_signature_offset..ed25519_signature_offset + 64]
        instruction_index = ed25519_pubkey_instruction_index != u16::MAX ? ed25519_pubkey_instruction_index : current_instruction;
        pubkey = instructions[instruction_index].data[ed25519_pubkey_offset..ed25519_pubkey_offset + 32]
        instruction_index = ed25519_message_instruction_index != u16::MAX ? ed25519_message_instruction_index : current_instruction;
        message = instructions[instruction_index].data[ed25519_message_data_offset..ed25519_message_data_offset + ed25519_message_data_size]
        if pubkey.verify(signature, message) != Success {
            return Error
        }
    }
    return Success
}
```

#### Secp256k1 कार्यक्रम

Secp256k1 सार्वजनिक कुंजी पुनर्प्राप्ति संचालन (ecrecover) को सत्यापित करें।

- कार्यक्रम आईडी: `KeccakSecp256k11111111111111111111111111111`
- [निर्देश](https://docs.rs/solana-secp256k1-program/latest/solana_secp256k1_program/index.html)

Secp256k1 कार्यक्रम एक निर्देश को संसाधित करता है जो पहले बाइट के रूप में लेता है
निर्देश डेटा में अनुक्रमित निम्नलिखित संरचना की संख्या:

```
struct Secp256k1SignatureOffsets {
    secp_signature_offset: u16,            // 64+1 बाइट के [हस्ताक्षर, पुनर्प्राप्ति_id] के लिए ऑफसेट
    secp_signature_instruction_index: u8,  // हस्ताक्षर खोजने के लिए निर्देशांक
    secp_pubkey_offset: u16,               // 20 बाइट के ethereum_address सार्वजनिक कुंजी के लिए ऑफसेट
    secp_pubkey_instruction_index: u8,     // सार्वजनिक कुंजी खोजने के लिए निर्देशांक
    secp_message_data_offset: u16,         // संदेश डेटा की शुरुआत के लिए ऑफसेट
    secp_message_data_size: u16,           // संदेश डेटा का आकार
    secp_message_instruction_index: u8,    // संदेश डेटा प्राप्त करने के लिए निर्देशांक
}
```

पुनर्प्राप्ति सत्यापन का छद्म कोड:

```
process_instruction() {
  for i in 0..count {
      // i'th अनुक्रमांक के संदर्भित मान:
      instructions = &transaction.message().instructions
      signature = instructions[secp_signature_instruction_index].data[secp_signature_offset..secp_signature_offset + 64]
      recovery_id = instructions[secp_signature_instruction_index].data[secp_signature_offset + 64]
      ref_eth_pubkey = instructions[secp_pubkey_instruction_index].data[secp_pubkey_offset..secp_pubkey_offset + 20]
      message_hash = keccak256(instructions[secp_message_instruction_index].data[secp_message_data_offset..secp_message_data_offset + secp_message_data_size])
      pubkey = ecrecover(signature, recovery_id, message_hash)
      eth_pubkey = keccak256(pubkey[1..])[12..]
      if eth_pubkey != ref_eth_pubkey {
          return Error
      }
  }
  return Success
}
```

यह उपयोगकर्ता को लेनदेन में हस्ताक्षर और संदेश डेटा के लिए किसी भी निर्देश डेटा को निर्दिष्ट करने की अनुमति देता है। एक विशेष निर्देश sysvar निर्दिष्ट करके, कोई लेनदेन से डेटा भी प्राप्त कर सकता है।

लेनदेन की लागत सत्यापित करने के लिए हस्ताक्षरों की संख्या को हस्ताक्षर लागत सत्यापन गुणांक से गुणा करके गिनी जाएगी।

#### Secp256r1 कार्यक्रम

Secp256r1 हस्ताक्षरों को सत्यापित करने के लिए कार्यक्रम। यह एक secp256r1 हस्ताक्षर,
एक सार्वजनिक कुंजी, और एक संदेश लेता है। अधिकतम 8 हस्ताक्षरों को सत्यापित किया जा सकता है। यदि कोई भी हस्ताक्षर सत्यापित करने में विफल रहता है, तो एक त्रुटि लौटाई जाती है।

- कार्यक्रम आईडी: `Secp256r1SigVerify1111111111111111111111111`
- [निर्देश](https://docs.rs/solana-secp256k1-recover/latest/solana_secp256k1_recover/index.html)

Secp256r1 कार्यक्रम एक निर्देश को संसाधित करता है। पहला `u8` सत्यापित करने के लिए हस्ताक्षरों की संख्या है, जिसके बाद एक एकल बाइट पैडिंग होती है। इसके बाद, निम्नलिखित संरचना को अनुक्रमित किया जाता है, प्रत्येक हस्ताक्षर के लिए एक:

```rust
struct Secp256r1SignatureOffsets {
    signature_offset: u16,             // 64 बाइट के संकुचित secp256r1 हस्ताक्षर के लिए ऑफसेट
    signature_instruction_index: u16,  // हस्ताक्षर खोजने के लिए निर्देशांक
    public_key_offset: u16,            // 33 बाइट के संकुचित सार्वजनिक कुंजी के लिए ऑफसेट
    public_key_instruction_index: u16, // सार्वजनिक कुंजी खोजने के लिए निर्देशांक
    message_data_offset: u16,          // संदेश डेटा की शुरुआत के लिए ऑफसेट
    message_data_size: u16,            // संदेश डेटा का आकार
    message_instruction_index: u16,    // संदेश डेटा प्राप्त करने के लिए निर्देश डेटा का अनुक्रमांक
}

```

हस्ताक्षर सत्यापन का छद्म कोड:
```
process_instruction() {
    if data.len() < SIGNATURE_OFFSETS_START {
        return Error
    }

    num_signatures = data[0] as usize
    if num_signatures == 0 || num_signatures > 8 {
        return Error
    }

    expected_data_size = num_signatures * SIGNATURE_OFFSETS_SERIALIZED_SIZE + SIGNATURE_OFFSETS_START
    if data.len() < expected_data_size {
        return Error
    }

    for i in 0..num_signatures {
        offsets = parse_signature_offsets(data, i)

        signature = get_data_slice(data, instruction_datas, offsets.signature_instruction_index, offsets.signature_offset, SIGNATURE_SERIALIZED_SIZE)

        if s > half_curve_order {
            return Error
        }

        pubkey = get_data_slice(data, instruction_datas, offsets.public_key_instruction_index, offsets.public_key_offset, COMPRESSED_PUBKEY_SERIALIZED_SIZE)

        message = get_data_slice(data, instruction_datas, offsets.message_instruction_index, offsets.message_data_offset, offsets.message_data_size)

        if !verify_signature(signature, pubkey, message) {
            return Error
        }
    }

    return Success
}
```
नोट: सभी हस्ताक्षरों के लिए कम S मान लागू किए जाते हैं ताकि आकस्मिक हस्ताक्षर
मैलियाबिलिटी से बचा जा सके।

### कोर कार्यक्रम

सोलाना क्लस्टर जेनसिस में विशेष कार्यक्रमों की एक सूची शामिल है जो
नेटवर्क के लिए विभिन्न कोर कार्यक्षमताएँ प्रदान करती हैं। ऐतिहासिक रूप से इन्हें "नेटिव" कार्यक्रमों के रूप में संदर्भित किया जाता था और इन्हें
मान्यकर्ता कोड के साथ वितरित किया जाता था।

1. सिस्टम कार्यक्रम

- कार्यक्रम आईडी: `11111111111111111111111111111111`
- [निर्देश](https://docs.rs/solana-program/latest/solana_program/system_instruction/enum.SystemInstruction.html)
- नए खातों का निर्माण करें, खाता डेटा आवंटित करें, खातों को स्वामित्व वाले
कार्यक्रमों को सौंपें, सिस्टम कार्यक्रम स्वामित्व वाले खातों से लमपोर्ट्स स्थानांतरित करें और लेनदेन शुल्क का भुगतान करें।
