यह खंड Solana विकास के लिए अपने स्थानीय वातावरण को सेट करने के चरणों को कवर करता
है।

## त्वरित इंस्टॉलेशन

Mac और Linux पर, सभी डिपेंडेंसीज़ को इंस्टॉल करने के लिए यह एक कमांड चलाएँ।

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

<Callout type="warn">
  Windows उपयोगकर्ता: आपको पहले WSL इंस्टॉल करना होगा (देखें [डिपेंडेंसीज़
  इंस्टॉल करें](#install-dependencies))। फिर Ubuntu (Linux) टर्मिनल में उपरोक्त
  कमांड चलाएँ।
</Callout>

इंस्टॉलेशन के बाद, आपको निम्न जैसा आउटपुट दिखाई देना चाहिए:

```
Installed Versions:
Rust: rustc 1.86.0 (05f9846f8 2025-03-31)
Solana CLI: solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
Anchor CLI: anchor-cli 0.31.1
Node.js: v23.11.0
Yarn: 1.22.1
```

<Callout>

यदि त्वरित इंस्टॉलेशन कमांड काम नहीं करता है, तो कृपया प्रत्येक डिपेंडेंसी को
अलग-अलग इंस्टॉल करने के निर्देशों के लिए नीचे दिए गए
[डिपेंडेंसीज़ इंस्टॉल करें](#install-dependencies) खंड को देखें।

यदि त्वरित इंस्टॉल कमांड सफलतापूर्वक चलता है, तो नीचे दिए गए
[Solana CLI बेसिक्स](#solana-cli-basics) और
[Anchor CLI बेसिक्स](#anchor-cli-basics) खंडों पर जाएँ।

</Callout>

## डिपेंडेंसीज़ इंस्टॉल करें

नीचे दिए गए निर्देश आपको प्रत्येक डिपेंडेंसी को अलग-अलग इंस्टॉल करने के लिए
मार्गदर्शन करते हैं।

- Windows उपयोगकर्ताओं को पहले WSL (Windows subsystem for Linux) इंस्टॉल करना
  होगा और फिर नीचे Linux खंड में निर्दिष्ट डिपेंडेंसीज़ को इंस्टॉल करना होगा।
- Linux उपयोगकर्ताओं को पहले नीचे Linux खंड में निर्दिष्ट डिपेंडेंसीज़ को
  इंस्टॉल करना चाहिए।
- Mac उपयोगकर्ताओं को नीचे दिए गए Rust इंस्टॉलेशन निर्देशों से शुरू करना चाहिए।

<Accordions>
<Accordion title="Windows Subsystem for Linux (WSL)">

Windows पर Solana प्रोग्राम विकसित करने के लिए **आपको
[WSL](https://learn.microsoft.com/en-us/windows/wsl/install) (Windows subsystem
for Linux) का उपयोग करना होगा**। अन्य सभी डिपेंडेंसीज़ को Linux टर्मिनल के
माध्यम से इंस्टॉल करें।

WSL इंस्टॉल करने के बाद, Rust, Solana CLI, और Anchor CLI को इंस्टॉल करने से पहले
नीचे Linux खंड में निर्दिष्ट डिपेंडेंसीज़ को इंस्टॉल करें।

WSL इंस्टॉल करने के लिए, Windows PowerShell में निम्न कमांड चलाएँ:

```terminal
$ wsl --install
```

इंस्टॉल प्रक्रिया आपको एक डिफ़ॉल्ट उपयोगकर्ता खाता बनाने के लिए प्रॉम्प्ट करेगी।

![WSL इंस्टॉल](/assets/docs/intro/installation/wsl-install.png)

डिफ़ॉल्ट रूप से, WSL Ubuntu इंस्टॉल करता है। आप सर्च बार में "Ubuntu" खोजकर
Linux टर्मिनल खोल सकते हैं।

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

यदि आपका Ubuntu टर्मिनल नीचे दिए गए चित्र जैसा दिखता है, तो आपको एक समस्या का
सामना करना पड़ सकता है जहां `ctrl + v` (पेस्ट कीबोर्ड शॉर्टकट) टर्मिनल में काम
नहीं करता है।

![Ubuntu टर्मिनल](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

यदि आपको यह समस्या आती है, तो सर्च बार में "Terminal" खोजकर Windows Terminal
खोलें।

![Windows टर्मिनल](/assets/docs/intro/installation/wsl-windows-terminal.png)

इसके बाद, Windows Terminal बंद करें और फिर से Ubuntu खोजकर Linux टर्मिनल फिर से
खोलें। अब टर्मिनल नीचे दिए गए चित्र की तरह दिखना चाहिए, जहां `ctrl + v` (पेस्ट
कीबोर्ड शॉर्टकट) काम करता है।

![Ubuntu टर्मिनल](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

यदि आप VS Code का उपयोग करते हैं, तो
[WSL एक्सटेंशन](https://code.visualstudio.com/docs/remote/wsl-tutorial) आपको WSL
और VS Code को एक साथ उपयोग करने की सुविधा देता है।

![VS Code में WSL सेटअप](/assets/docs/intro/installation/wsl-vscode.png)

तब आपको VS Code स्टेटस बार में निम्नलिखित दिखाई देना चाहिए:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

एक बार जब आप WSL सेट कर लेते हैं, तो Ubuntu (Linux) टर्मिनल के माध्यम से सभी
अन्य निर्भरताओं को इंस्टॉल करें।

आवश्यक Solana निर्भरताओं को इंस्टॉल करने के लिए, पहले Ubuntu (Linux) टर्मिनल में
[त्वरित इंस्टॉलेशन](#quick-installation) कमांड चलाने का प्रयास करें।

<Callout>

यदि त्वरित इंस्टॉल कमांड काम नहीं करता है, तो Rust, Solana CLI, और Anchor CLI को
इंस्टॉल करने के व्यक्तिगत चरणों पर जाने से पहले नीचे दिए गए Linux अनुभाग में
निर्दिष्ट निर्भरताओं को इंस्टॉल करें।

यदि त्वरित इंस्टॉल कमांड सफलतापूर्वक चलता है, तो नीचे दिए गए
[Solana CLI बेसिक्स](#solana-cli-basics) और
[Anchor CLI बेसिक्स](#anchor-cli-basics) अनुभागों पर जाएं।

</Callout>

</Accordion>

### Rust इंस्टॉल करें

डेवलपर्स [Rust प्रोग्रामिंग भाषा](https://www.rust-lang.org/) का उपयोग करके
Solana प्रोग्राम बनाते हैं।

[rustup](https://www.rust-lang.org/tools/install) के साथ Rust इंस्टॉल करें।

Rust इंस्टॉल करने के लिए निम्न कमांड चलाएँ:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

इंस्टॉलेशन पूरा होने के बाद आपको निम्न संदेश दिखाई देना चाहिए:

<Accordions>
<Accordion title="सफल Rust इंस्टॉल संदेश">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

Cargo के bin डायरेक्टरी को शामिल करने के लिए अपने PATH एनवायरनमेंट वेरिएबल को
रीलोड करने के लिए निम्न कमांड चलाएँ:

```terminal
$ . "$HOME/.cargo/env"
```

इंस्टॉलेशन सफल हुआ है या नहीं, यह सत्यापित करने के लिए, Rust का वर्जन जांचें:

```terminal
$ rustc --version
```

आपको निम्न जैसा आउटपुट दिखाई देना चाहिए:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Solana CLI इंस्टॉल करें

Solana CLI Solana प्रोग्राम बनाने और डिप्लॉय करने के लिए आवश्यक सभी टूल प्रदान
करता है।

आधिकारिक इंस्टॉल कमांड का उपयोग करके Solana CLI टूल सूट इंस्टॉल करें:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

आप `stable` को अपने वांछित रिलीज के सॉफ्टवेयर वर्जन से मेल खाने वाले रिलीज टैग
से बदल सकते हैं (जैसे `v2.0.3`), या तीन प्रतीकात्मक चैनल नामों में से एक का
उपयोग कर सकते हैं: `stable`, `beta`, या `edge`।

Solana CLI के पहली बार इंस्टॉलेशन के लिए, आपको निम्न संदेश दिखाई दे सकता है जो
आपको `PATH` पर्यावरण वेरिएबल जोड़ने के लिए प्रेरित करता है:

```
Close and reopen your terminal to apply the PATH changes or run the following in your existing shell:

export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

<Tabs groupId="language" items={["Linux", "Mac"]}>
<Tab value="Linux">

यदि आप **Linux** या **WSL** का उपयोग कर रहे हैं, तो आपको Solana CLI बाइनरी को
अपने `PATH` में जोड़ना होगा ताकि कमांड आपके टर्मिनल में उपलब्ध हो।

सबसे पहले, यह जांचने के लिए निम्न कमांड चलाएं कि आप कौन सा शेल उपयोग कर रहे हैं:

```terminal
$ echo $SHELL
```

- यदि आउटपुट में `/bash` शामिल है, तो `.bashrc` का उपयोग करें।
- यदि आउटपुट में `/zsh` शामिल है, तो `.zshrc` का उपयोग करें।

अपने शेल के आधार पर, उपयुक्त कमांड चलाएं।

Bash के लिए (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Zsh के लिए (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

फिर टर्मिनल सेशन को रिफ्रेश करने या अपने टर्मिनल को पुनः आरंभ करने के लिए निम्न
कमांड चलाएं।

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

</Tab>
<Tab value="Mac">

यदि आप `zsh` वाले Mac का उपयोग करते हैं, तो इंस्टॉलेशन से लॉग किया गया डिफ़ॉल्ट
`export PATH` कमांड आपके टर्मिनल को बंद करने के बाद बना नहीं रहता है।

इसके बजाय, आप निम्न कमांड चलाकर अपनी शेल कॉन्फ़िगरेशन फ़ाइल में PATH जोड़ सकते
हैं:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

फिर टर्मिनल सेशन को रिफ्रेश करने या अपने टर्मिनल को रीस्टार्ट करने के लिए निम्न
कमांड चलाएं।

```terminal
$ source ~/.zshrc
```

</Tab>
</Tabs>

इंस्टॉलेशन सफल हुआ है या नहीं, यह जांचने के लिए Solana CLI वर्जन देखें:

```terminal
$ solana --version
```

आपको निम्न जैसा आउटपुट दिखाई देना चाहिए:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

आप सभी उपलब्ध वर्जन
[Agave Github repo](https://github.com/anza-xyz/agave/releases) पर देख सकते हैं।

<Callout>

Agave [Anza](https://www.anza.xyz/) से validator क्लाइंट है, जिसे पहले Solana
Labs validator क्लाइंट के नाम से जाना जाता था।

</Callout>

बाद में Solana CLI को नवीनतम वर्जन में अपडेट करने के लिए, आप निम्न कमांड का
उपयोग कर सकते हैं:

```terminal
$ agave-install update
```

### Anchor CLI इंस्टॉल करें

[Anchor](https://www.anchor-lang.com/) Solana प्रोग्राम विकसित करने के लिए एक
फ्रेमवर्क है। Anchor फ्रेमवर्क Solana प्रोग्राम लिखने की प्रक्रिया को सरल बनाने
के लिए Rust मैक्रोज़ का उपयोग करता है।

आप Anchor CLI और टूलिंग को दो तरीकों से इंस्टॉल कर सकते हैं:

1. Anchor Version Manager (AVM) - अनुशंसित इंस्टॉलेशन विधि
2. AVM के बिना - सीधे GitHub से इंस्टॉल करें

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Anchor वर्जन मैनेजर (AVM) आपको अपने सिस्टम पर विभिन्न Anchor वर्जन इंस्टॉल और
प्रबंधित करने तथा भविष्य में Anchor वर्जन को आसानी से अपडेट करने की अनुमति देता
है।

निम्न कमांड से AVM इंस्टॉल करें:

```terminal
$ cargo install --git https://github.com/coral-xyz/anchor avm --force
```

पुष्टि करें कि AVM सफलतापूर्वक इंस्टॉल हो गया है:

```terminal
$ avm --version
```

AVM का उपयोग करके Anchor CLI का नवीनतम संस्करण इंस्टॉल करें:

```terminal
$ avm install latest
$ avm use latest
```

आप संस्करण संख्या निर्दिष्ट करके Anchor CLI का कोई विशिष्ट संस्करण इंस्टॉल कर
सकते हैं:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="info">

अपने सिस्टम पर चलाने के लिए Anchor CLI संस्करण घोषित करने के लिए
_shell`avm use`_ कमांड चलाना न भूलें।

- यदि आपने `latest` संस्करण इंस्टॉल किया है, तो _shell`avm use latest`_ चलाएँ।
- यदि आपने संस्करण `0.30.1` इंस्टॉल किया है, तो _shell`avm use 0.30.1`_ चलाएँ।

</Callout>

</Tab>

<Tab value="Without AVM">

निम्न कमांड के साथ Anchor CLI का एक विशिष्ट संस्करण इंस्टॉल करें:

```terminal
$ cargo install --git https://github.com/coral-xyz/anchor --tag v0.30.1 anchor-cli
```

</Tab>
</Tabs>

यह सत्यापित करने के लिए कि इंस्टॉलेशन सफल हुआ है, Anchor CLI संस्करण की जाँच
करें:

```terminal
$ anchor --version
```

आपको निम्न जैसा आउटपुट दिखाई देना चाहिए:

```
anchor-cli 0.31.1
```

<Callout type="warn">

Linux या WSL पर Anchor CLI इंस्टॉल करते समय, आपको यह त्रुटि हो सकती है:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

यदि आप यह त्रुटि संदेश देखते हैं:

1. इस पृष्ठ के शीर्ष पर [Linux](/docs/intro/installation#install-dependencies)
   अनुभाग में सूचीबद्ध निर्भरताओं को इंस्टॉल करें।
2. Anchor CLI इंस्टॉल करने का पुनः प्रयास करें।

</Callout>

#### Node.js और Yarn

_shell`anchor init`_ कमांड से बनाई गई डिफ़ॉल्ट Anchor प्रोजेक्ट टेस्ट फ़ाइल
(TypeScript) के लिए Node.js और Yarn की आवश्यकता होती है। (Rust टेस्ट टेम्पलेट
_shell`anchor init --test-template rust`_ का उपयोग करके उपलब्ध है)

<Accordions>
<Accordion title="Node इंस्टालेशन">

[Node Version Manager (nvm)](https://github.com/nvm-sh/nvm) का उपयोग करके node
इंस्टॉल करें।

निम्न कमांड का उपयोग करके nvm इंस्टॉल करें:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

अपने टर्मिनल को रीस्टार्ट करें और पुष्टि करें कि nvm कमांड सफलतापूर्वक चलता है:

```terminal
$ command -v nvm
```

फिर, node इंस्टॉल करने के लिए `nvm` का उपयोग करें:

```terminal
$ nvm install node
```

इंस्टालेशन सफल हुआ है, यह सत्यापित करने के लिए, Node वर्जन की जांच करें:

```terminal
$ node --version
```

आपको निम्न जैसा आउटपुट दिखाई देना चाहिए:

```
v23.7.0
```

</Accordion>
<Accordion title="Yarn इंस्टालेशन">

Yarn इंस्टॉल करें:

```terminal
$ npm install --global yarn
```

इंस्टालेशन सफल हुआ है, यह सत्यापित करने के लिए, Yarn वर्जन की जांच करें:

```terminal
$ yarn --version
```

आपको निम्न आउटपुट दिखाई देना चाहिए:

```
1.22.1
```

</Accordion>
</Accordions>

<Callout type="warn">

जब `anchor build` चलाते हैं, तो अगर आपको निम्न त्रुटियां मिलती हैं:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

इन समाधानों को आज़माएँ:

1. निम्न कमांड का उपयोग करके फोर्स इंस्टॉल करें:

```terminal
$ cargo build-sbf --force-tools-install
```

2. यदि पिछला समाधान काम नहीं करता है, तो Solana कैश को साफ़ करें:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
आप `Cargo.lock` फ़ाइल के version फ़ील्ड को 4 से 3 में बदलकर इसे ठीक कर सकते हैं:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

अधिक जानकारी के लिए [इस मुद्दे](https://github.com/coral-xyz/anchor/issues/3392)
को देखें।

</Accordion>

</Accordions>

उपरोक्त समाधान लागू करने के बाद, _shell`anchor build`_ को फिर से चलाने का प्रयास
करें।

</Callout>

<Callout type="warn">

Linux या WSL पर एक नया Anchor प्रोजेक्ट बनाने के बाद _shell`anchor test`_ चलाते
समय, यदि आपके पास Node.js या Yarn इंस्टॉल नहीं है तो आपको निम्न त्रुटियाँ मिल
सकती हैं:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

</Step>

## Solana CLI मूल बातें

यह खंड आपको शुरू करने के लिए कुछ सामान्य Solana CLI कमांड के बारे में बताता है।

<Steps>
<Step>

### Solana कॉन्फिग

अपने वर्तमान कॉन्फ़िगरेशन को देखने के लिए:

```terminal
$ solana config get
```

आपको निम्न जैसा आउटपुट दिखाई देना चाहिए:

```
Config File: /Users/test/.config/solana/cli/config.yml
RPC URL: https://api.mainnet-beta.solana.com
WebSocket URL: wss://api.mainnet-beta.solana.com/ (computed)
Keypair Path: /Users/test/.config/solana/id.json
Commitment: confirmed
```

RPC URL और Websocket URL उस Solana क्लस्टर को निर्दिष्ट करते हैं जिससे CLI
अनुरोध करता है।

आप निम्न कमांड का उपयोग करके Solana CLI क्लस्टर को अपडेट कर सकते हैं:

```terminal
$ solana config set --url mainnet-beta
$ solana config set --url devnet
$ solana config set --url localhost
$ solana config set --url testnet
```

आप निम्न शॉर्ट ऑप्शन्स का भी उपयोग कर सकते हैं:

```terminal
$ solana config set -um    # For mainnet-beta
$ solana config set -ud    # For devnet
$ solana config set -ul    # For localhost
$ solana config set -ut    # For testnet
```

Keypair Path डिफ़ॉल्ट Solana वॉलेट (keypair) को इंगित करता है जिसका उपयोग Solana
CLI द्वारा ट्रांजैक्शन शुल्क का भुगतान करने और प्रोग्राम डिप्लॉय करने के लिए
किया जाता है। डिफ़ॉल्ट रूप से, यह फ़ाइल `~/.config/solana/id.json` पर संग्रहित
की जाती है।

</Step>
<Step>

### वॉलेट बनाएँ

Solana CLI का उपयोग करके ट्रांजैक्शन भेजने के लिए, आपको SOL से फंडेड Solana
वॉलेट की आवश्यकता होती है।

डिफ़ॉल्ट Keypair Path पर एक keypair जनरेट करने के लिए, निम्न कमांड चलाएँ:

```terminal
$ solana-keygen new
```

आपको निम्न जैसा आउटपुट दिखाई देना चाहिए:

```
Generating a new keypair

For added security, enter a BIP39 passphrase

NOTE! This passphrase improves security of the recovery seed phrase NOT the
keypair file itself, which is stored as insecure plain text

BIP39 Passphrase (empty for none):

Wrote new keypair to /Users/test/.config/solana/id.json
===========================================================================
pubkey: 8dBTPrjnkXyuQK3KDt9wrZBfizEZijmmUQXVHpFbVwGT
===========================================================================
Save this seed phrase and your BIP39 passphrase to recover your new keypair:
cream bleak tortoise ocean nasty game gift forget fancy salon mimic amazing
===========================================================================
```

<Callout type="info">

यदि आपके पास पहले से ही डिफ़ॉल्ट स्थान पर सहेजा गया फ़ाइल सिस्टम वॉलेट है, तो यह
कमांड उसे ओवरराइड नहीं करेगा जब तक कि आप स्पष्ट रूप से `--force` फ्लैग का उपयोग
करके ओवरराइड को फोर्स न करें।

</Callout>

अपने वॉलेट का पता (पब्लिक की) देखने के लिए, चलाएँ:

```terminal
$ solana address
```

</Step>
<Step>

### SOL एयरड्रॉप

ट्रांजैक्शन और प्रोग्राम डिप्लॉयमेंट के लिए भुगतान करने के लिए अपने वॉलेट में
SOL का एयरड्रॉप अनुरोध करें।

अपने क्लस्टर को devnet पर सेट करें:

```terminal
$ solana config set -ud
```

फिर devnet SOL का एयरड्रॉप अनुरोध करें:

```terminal
$ solana airdrop 2
```

<Callout>

Devnet एयरड्रॉप प्रति अनुरोध 5 SOL तक सीमित हैं। यदि आप रेट लिमिट से टकराते हैं
या त्रुटियों का सामना करते हैं, तो इसके बजाय
[वेब फॉसेट](https://faucet.solana.com) का उपयोग करने का प्रयास करें।

</Callout>

अपने वॉलेट के SOL बैलेंस की जांच करने के लिए, निम्न कमांड चलाएं:

```terminal
$ solana balance
```

</Step>
<Step>

### लोकल validator चलाएं

Solana CLI में स्थानीय विकास के लिए एक अंतर्निहित
[टेस्ट validator](https://docs.anza.xyz/cli/examples/test-validator) शामिल है।

एक अलग टर्मिनल में, लोकल validator शुरू करने के लिए निम्न कमांड चलाएं:

```terminal
$ solana-test-validator
```

<Callout>

Solana CLI कमांड चलाने से पहले अपने CLI को localhost का उपयोग करने के लिए अपडेट
करना याद रखें:

```terminal
$ solana config set -ul
```

</Callout>

</Step>
</Steps>

## Anchor CLI मूल बातें

इस खंड में आपको शुरू करने के लिए कुछ सामान्य Anchor CLI कमांड के बारे में बताया
गया है। Anchor CLI के बारे में अधिक जानकारी के लिए,
[Anchor दस्तावेज़](https://www.anchor-lang.com/docs) देखें।

<Steps>
<Step>

### प्रोजेक्ट प्रारंभ करें

एक नया Anchor प्रोजेक्ट बनाने के लिए, निम्न कमांड चलाएं:

```terminal
$ anchor init <project-name>
```

उदाहरण के लिए, `my-project` नामक प्रोजेक्ट बनाने के लिए, चलाएँ:

```terminal
$ anchor init my-project
```

यह कमांड प्रोजेक्ट नाम के साथ एक नई डायरेक्टरी बनाती है और एक बेसिक Rust
प्रोग्राम और TypeScript टेस्ट टेम्पलेट के साथ एक नया Anchor प्रोजेक्ट
इनिशियलाइज़ करती है।

प्रोजेक्ट डायरेक्टरी में जाएँ:

```terminal
$ cd <project-name>
```

Anchor प्रोजेक्ट की
[फाइल संरचना](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
देखें।

</Step>

<Step>
### प्रोग्राम बिल्ड करें

अपने प्रोजेक्ट को बिल्ड करने के लिए, निम्न कमांड चलाएँ:

```terminal
$ anchor build
```

आप कंपाइल किए गए प्रोग्राम को `/target/deploy` डायरेक्टरी में पा सकते हैं।

</Step>
<Step>

### प्रोग्राम डिप्लॉय करें

अपने प्रोजेक्ट को डिप्लॉय करने के लिए, निम्न कमांड चलाएँ:

```terminal
$ anchor deploy
```

यह कमांड आपके प्रोग्राम को
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml) फाइल
में निर्दिष्ट `cluster` पर डिप्लॉय करती है।

</Step>
<Step>

### प्रोग्राम टेस्ट करें

अपने प्रोजेक्ट का टेस्ट करने के लिए, निम्न कमांड चलाएँ:

```terminal
$ anchor test
```

<Callout>

यह कमांड आपके प्रोजेक्ट को बिल्ड करती है, डिप्लॉय करती है, और टेस्ट चलाती है।

जब `Anchor.toml` में `cluster` के रूप में `localnet` का उपयोग करते हैं, तो
Anchor स्वचालित रूप से एक लोकल validator शुरू करता है, आपके प्रोग्राम को डिप्लॉय
करता है, टेस्ट चलाता है, और फिर validator को बंद कर देता है।

</Callout>

</Step>
</Steps>
