---
title: नेटवर्क से पढ़ना
description:
  सोलाना ब्लॉकचेन नेटवर्क से डेटा पढ़ने का तरीका सीखें। यह गाइड
  जावास्क्रिप्ट/टाइपस्क्रिप्ट का उपयोग करके वॉलेट अकाउंट्स, प्रोग्राम अकाउंट्स
  और token mint अकाउंट्स को प्राप्त करने के बारे में बताती है, जिसमें सोलाना
  web3.js लाइब्रेरी के साथ व्यावहारिक उदाहरण शामिल हैं।
---

यह खंड सोलाना नेटवर्क से डेटा पढ़ने के तरीके का अन्वेषण करता है, जिसमें विभिन्न
अकाउंट्स को प्राप्त करके सोलाना अकाउंट की संरचना को समझना शामिल है।

सोलाना पर, सभी डेटा "accounts" में मौजूद होता है। आप सोलाना पर डेटा को एक
सार्वजनिक डेटाबेस के रूप में सोच सकते हैं जिसमें एक ही "Accounts" टेबल है, जहां
प्रत्येक प्रविष्टि एक अकाउंट है जिसमें समान बेस
[Account type](https://github.com/anza-xyz/agave/blob/v2.1.11/sdk/account/src/lib.rs#L48-L60)
है।

![Accounts](/assets/docs/core/accounts/accounts.png)

सोलाना पर अकाउंट्स "state" या "executable" प्रोग्राम स्टोर कर सकते हैं। प्रत्येक
अकाउंट में एक "address" (पब्लिक की) होती है जो इसकी अनूठी आईडी के रूप में काम
करती है और इसके संबंधित ऑन-चेन डेटा को खोजने के लिए उपयोग की जाती है।

सोलाना अकाउंट्स में निम्न में से कोई एक होता है:

- **State**: डेटा जिसे पढ़ने और संग्रहित करने के लिए बनाया गया है। उदाहरण के
  लिए, टोकन के बारे में जानकारी, उपयोगकर्ता डेटा, या प्रोग्राम के भीतर परिभाषित
  अन्य डेटा।
- **Executable Programs**: अकाउंट्स जिनमें सोलाना प्रोग्राम का वास्तविक कोड होता
  है। ये अकाउंट्स निर्देश स्टोर करते हैं जिन्हें उपयोगकर्ता इनवोक कर सकते हैं।

प्रोग्राम कोड और प्रोग्राम स्टेट का यह अलगाव सोलाना के अकाउंट मॉडल की एक प्रमुख
विशेषता है। अधिक जानकारी के लिए, [सोलाना अकाउंट मॉडल](/docs/core/accounts) पेज
देखें।

## वॉलेट अकाउंट प्राप्त करें

यह उदाहरण दिखाता है कि कैसे:

1. एक नया keypair (पब्लिक/प्राइवेट की पेयर) जनरेट करें।
2. नए एड्रेस को फंड करने के लिए SOL का एयरड्रॉप अनुरोध करें।
3. फंडेड एड्रेस के लिए अकाउंट डेटा प्राप्त करें।

सोलाना पर, SOL के साथ एक नए एड्रेस को फंड करने से स्वचालित रूप से सिस्टम
प्रोग्राम के स्वामित्व वाला एक अकाउंट बनता है। सभी "wallet" अकाउंट्स बस सिस्टम
प्रोग्राम के स्वामित्व वाले अकाउंट्स हैं जो SOL रखते हैं और ट्रांजैक्शन पर
हस्ताक्षर कर सकते हैं।

<CodeTabs flags="r">

```ts !! title="Fetch Account"
import { Keypair, Connection, LAMPORTS_PER_SOL } from "@solana/web3.js";

// !mark
const keypair = Keypair.generate();
console.log(`Public Key: ${keypair.publicKey}`);

const connection = new Connection("http://localhost:8899", "confirmed");

// Funding an address with SOL automatically creates an account
const signature = await connection.requestAirdrop(
  keypair.publicKey,
  LAMPORTS_PER_SOL
);
await connection.confirmTransaction(signature, "confirmed");

// !mark
const accountInfo = await connection.getAccountInfo(keypair.publicKey);
console.log(JSON.stringify(accountInfo, null, 2));
```

</CodeTabs>

सभी Solana खाते एक ही
[खाता प्रकार](https://github.com/anza-xyz/agave/blob/v2.1.11/sdk/account/src/lib.rs#L48-L60)

Solana पर एक "वॉलेट"
[System Program](https://github.com/anza-xyz/agave/tree/v2.1.11/programs/system)
के स्वामित्व वाला एक खाता है, जो Solana के अंतर्निहित प्रोग्रामों में से एक है।
वॉलेट खातों का मुख्य रूप से SOL रखने (`lamports` फ़ील्ड में ट्रैक किया जाता है)
और लेनदेन पर हस्ताक्षर करने के लिए उपयोग किया जाता है।

जब आप एक वॉलेट खाता प्राप्त करते हैं, तो प्रतिक्रिया में ये फ़ील्ड शामिल होते
हैं:

| फ़ील्ड       | विवरण                                                                                                                             |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| `data`       | खाते का डेटा बफर। वॉलेट खातों के लिए, यह खाली (0 बाइट्स) होता है। अन्य खाते डेटा स्टोर करने के लिए इस फ़ील्ड का उपयोग करते हैं।   |
| `executable` | इंगित करता है कि क्या खाते में निष्पादन योग्य प्रोग्राम कोड है। वॉलेट और स्टेट खातों के लिए, यह `false` होता है।                  |
| `owner`      | वह प्रोग्राम जो खाते का मालिक है। वॉलेट के लिए, यह हमेशा System Program होता है, जिसका पता `11111111111111111111111111111111` है। |
| `lamports`   | खाते का SOL बैलेंस, lamports में दर्शाया गया (1 SOL = 1,000,000,000 lamports)।                                                    |
| `rentEpoch`  | Solana के पुराने rent तंत्र से संबंधित एक विरासत फ़ील्ड (वर्तमान में अप्रयुक्त)।                                                  |
| `space`      | `data` फ़ील्ड का आकार, बाइट्स में (नोट: यह _rs`Account`_ प्रकार में स्वयं एक फ़ील्ड नहीं है)।                                     |

## Token Program प्राप्त करें

यह उदाहरण वॉलेट और प्रोग्राम खातों के बीच अंतर दिखाने के लिए Token Program को
प्राप्त करता है।

<CodeTabs flags="r">

```ts !! title="Fetch Program Account"
import { Connection, PublicKey } from "@solana/web3.js";

const connection = new Connection(
  "https://api.mainnet-beta.solana.com",
  "confirmed"
);
// !mark(1:2)
const address = new PublicKey("TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA");
const accountInfo = await connection.getAccountInfo(address);

// !collapse(1:17) collapsed
console.log(
  JSON.stringify(
    accountInfo,
    (key, value) => {
      if (key === "data" && value && value.length > 1) {
        return [
          value[0],
          "...truncated, total bytes: " + value.length + "...",
          value[value.length - 1]
        ];
      }
      return value;
    },
    2
  )
);
```

</CodeTabs>

## मिंट अकाउंट प्राप्त करें

यह उदाहरण USD Coin (USDC) मिंट अकाउंट को प्राप्त करता है ताकि यह दिखाया जा सके
कि Solana पर प्रोग्राम अलग-अलग अकाउंट्स में स्टेट कैसे स्टोर करते हैं।

एक
[Mint](https://github.com/solana-program/token/blob/program%40v8.0.0/program/src/state.rs#L16-L30)
अकाउंट Token Program के स्वामित्व वाला अकाउंट है। यह एक विशिष्ट टोकन के लिए
ग्लोबल मेटाडेटा स्टोर करता है, जिसमें कुल सप्लाई, डेसिमल की संख्या, और टोकन को
मिंट या फ्रीज करने के लिए अधिकृत अकाउंट्स शामिल हैं। मिंट अकाउंट का एड्रेस
Solana नेटवर्क पर एक टोकन को विशिष्ट रूप से पहचानता है।

<CodeTabs flags="r">

```ts !! title="Fetch Program Account"
import { Connection, PublicKey } from "@solana/web3.js";

const connection = new Connection(
  "https://api.mainnet-beta.solana.com",
  "confirmed"
);

// !mark(1:2)
const address = new PublicKey("EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v");
const accountInfo = await connection.getAccountInfo(address);

// !collapse(1:17) collapsed
console.log(
  JSON.stringify(
    accountInfo,
    (key, value) => {
      if (key === "data" && value && value.length > 1) {
        return [
          value[0],
          "...truncated, total bytes: " + value.length + "...",
          value[value.length - 1]
        ];
      }
      return value;
    },
    2
  )
);
```

## मिंट अकाउंट को डीसीरियलाइज़ करें

Solana अकाउंट का `data` फील्ड रॉ बाइट्स को कंटेन करता है। इस डेटा को सार्थक रूप
से समझने के लिए, आपको इसे उस उपयुक्त डेटा टाइप में डीसीरियलाइज़ करना होगा जो
अकाउंट के मालिक प्रोग्राम द्वारा परिभाषित किया गया है।

अधिकांश Solana प्रोग्राम क्लाइंट लाइब्रेरीज़ प्रदान करते हैं जिनमें हेल्पर
फंक्शन्स होते हैं जो डीसीरियलाइजेशन प्रक्रिया को अमूर्त बनाते हैं। ये फंक्शन्स
रॉ अकाउंट बाइट्स को संरचित डेटा टाइप्स में कन्वर्ट करते हैं, जिससे अकाउंट डेटा
के साथ काम करना आसान हो जाता है।

<WithMentions>

उदाहरण के लिए, _shell`@solana/spl-token`_ में [_ts`getMint()`_](mention:one)
फ़ंक्शन शामिल है जो Mint अकाउंट के `data` फ़ील्ड को
[Mint](https://github.com/solana-program/token/blob/program%40v8.0.0/program/src/state.rs#L16-L30)
डेटा टाइप में डीसीरियलाइज़ करने में मदद करता है।

<CodeTabs flags="r">

```ts !! title="Deserialize Mint Account Data"
import { PublicKey, Connection } from "@solana/web3.js";
import { getMint } from "@solana/spl-token";

const connection = new Connection(
  "https://api.mainnet-beta.solana.com",
  "confirmed"
);

const address = new PublicKey("EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v");
// !mention one
const mintData = await getMint(connection, address, "confirmed");

// !collapse(1:17) collapsed
console.log(
  JSON.stringify(
    mintData,
    (key, value) => {
      // Convert BigInt to String
      if (typeof value === "bigint") {
        return value.toString();
      }
      // Handle Buffer objects
      if (Buffer.isBuffer(value)) {
        return `<Buffer ${value.toString("hex")}>`;
      }
      return value;
    },
    2
  )
);
```

[Mint](https://github.com/solana-program/token/blob/program%40v8.0.0/program/src/state.rs#L16-L30)

| फ़ील्ड            | विवरण                                                                     |
| ----------------- | ------------------------------------------------------------------------- |
| `address`         | Mint अकाउंट का एड्रेस                                                     |
| `mintAuthority`   | नए टोकन मिंट करने के लिए अधिकृत अथॉरिटी                                   |
| `supply`          | टोकन की कुल सप्लाई                                                        |
| `decimals`        | टोकन के लिए दशमलव स्थानों की संख्या                                       |
| `isInitialized`   | इंगित करता है कि Mint अकाउंट इनिशियलाइज़ किया गया है या नहीं              |
| `freezeAuthority` | टोकन अकाउंट को फ्रीज करने के लिए अधिकृत अथॉरिटी                           |
| `tlvData`         | Token Extensions के लिए अतिरिक्त डेटा (आगे डीसीरियलाइजेशन की आवश्यकता है) |
