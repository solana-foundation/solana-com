टोकन डिजिटल संपत्तियां हैं जो विविध श्रेणियों की संपत्तियों के स्वामित्व का
प्रतिनिधित्व करती हैं। टोकनाइजेशन संपत्ति अधिकारों के डिजिटलीकरण को सक्षम बनाता
है। सोलाना पर टोकन को SPL
([सोलाना प्रोग्राम लाइब्रेरी](https://github.com/solana-program)) टोकन के रूप
में जाना जाता है।

इस खंड में सोलाना पर टोकन के प्रतिनिधित्व के मूल अवधारणाओं को शामिल किया गया है।
कोड उदाहरणों के लिए [SPL Token Basics](/docs/tokens/basics) खंड देखें।

## प्रमुख बिंदु

- [Token Program](#token-program) में नेटवर्क पर टोकन (फंजिबल और नॉन-फंजिबल
  दोनों) के साथ इंटरैक्ट करने के लिए सभी निर्देश लॉजिक शामिल हैं।

- एक [Mint Account](#mint-account) एक विशिष्ट टोकन का प्रतिनिधित्व करता है और
  टोकन के बारे में वैश्विक मेटाडेटा जैसे कुल आपूर्ति और मिंट अथॉरिटी (टोकन की नई
  इकाइयां बनाने के लिए अधिकृत पता) को स्टोर करता है।

- एक [Token Account](#token-account) एक विशिष्ट स्वामी के लिए एक विशिष्ट mint
  account के टोकन के व्यक्तिगत स्वामित्व को ट्रैक करता है।

- एक [Associated Token Account](#associated-token-account) एक Token Account है
  जिसका पता स्वामी और mint account पते से व्युत्पन्न किया गया है।

## Token Programs

सोलाना इकोसिस्टम में दो मुख्य Token Programs हैं। दोनों प्रोग्रामों का सोर्स कोड
नीचे दिया गया है।

<Cards>
  <Card title="Token Program (मूल)" href="https://github.com/solana-program/token">
   - बेसिक टोकन क्षमता (मिंट, ट्रांसफर, आदि)
   - अपरिवर्तनीय और व्यापक रूप से उपयोग किया जाता है
  </Card>

  <Card title="Token Extension Program (Token 2022)" href="https://github.com/solana-program/token-2022">
   - मूल Token Program की सभी सुविधाएँ शामिल हैं
   - "एक्सटेंशन" के माध्यम से सुविधाएँ जोड़ता है
  </Card>
</Cards>

Token Programs में नेटवर्क पर टोकन (फंजिबल और नॉन-फंजिबल दोनों) के साथ इंटरैक्ट
करने के लिए सभी निर्देश लॉजिक शामिल हैं। सोलाना पर सभी टोकन वास्तव में
[डेटा अकाउंट्स](/docs/core/accounts#data-account) हैं जो Token Program के
स्वामित्व में हैं।

![Token Program](/assets/docs/core/tokens/token-program.svg)

### Mint Account

सोलाना पर टोकन को विशिष्ट रूप से Token Program के स्वामित्व वाले
[Mint Account](https://github.com/solana-program/token/blob/6d18ff73b1dd30703a30b1ca941cb0f1d18c2b2a/program/src/state.rs#L16-L30)
के पते द्वारा पहचाना जाता है। यह अकाउंट एक विशिष्ट टोकन के लिए वैश्विक काउंटर के
रूप में कार्य करता है और निम्न जैसे डेटा स्टोर करता है:

- **आपूर्ति (Supply)**: टोकन की कुल आपूर्ति
- **दशमलव (Decimals)**: टोकन की दशमलव परिशुद्धता
- **मिंट अथॉरिटी (Mint authority)**: वह खाता जो टोकन की नई इकाइयां बनाने के लिए
  अधिकृत है, जिससे आपूर्ति बढ़ती है
- **फ्रीज अथॉरिटी (Freeze authority)**: वह खाता जो token account में टोकन को
  फ्रीज करने के लिए अधिकृत है, जिससे उन्हें ट्रांसफर या बर्न होने से रोका जा सके

![Mint Account](/assets/docs/core/tokens/mint-account.svg)

प्रत्येक mint account पर संग्रहित पूर्ण विवरण में निम्नलिखित शामिल हैं:

```rust title="Mint Account State"
pub struct Mint {
    /// Optional authority used to mint new tokens. The mint authority may only
    /// be provided during mint creation. If no mint authority is present
    /// then the mint has a fixed supply and no further tokens may be
    /// minted.
    pub mint_authority: COption<Pubkey>,
    /// Total supply of tokens.
    pub supply: u64,
    /// Number of base 10 digits to the right of the decimal place.
    pub decimals: u8,
    /// Is `true` if this structure has been initialized
    pub is_initialized: bool,
    /// Optional authority to freeze token accounts.
    pub freeze_authority: COption<Pubkey>,
}
```

संदर्भ के लिए, यहां
[USDC Mint Account](https://explorer.solana.com/address/EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v)
का सोलाना एक्सप्लोरर लिंक है।

### Token Account

Token Program
[Token Accounts](https://github.com/solana-program/token/blob/6d18ff73b1dd30703a30b1ca941cb0f1d18c2b2a/program/src/state.rs#L87-L108)
बनाता है जो प्रत्येक टोकन इकाई के व्यक्तिगत स्वामित्व को ट्रैक करते हैं। एक
token account निम्न जैसे डेटा स्टोर करता है:

- **मिंट (Mint)**: वह टोकन जिसकी इकाइयां token account रखता है
- **मालिक (Owner)**: वह खाता जो token account से टोकन ट्रांसफर करने के लिए
  अधिकृत है
- **राशि (Amount)**: token account में वर्तमान में मौजूद टोकन की संख्या

![Token Account](/assets/docs/core/tokens/token-account.svg)

प्रत्येक token account पर संग्रहित पूर्ण विवरण में निम्नलिखित शामिल हैं:

```rust title="Token Account State"
pub struct Account {
    /// The mint associated with this account
    pub mint: Pubkey,
    /// The owner of this account.
    pub owner: Pubkey,
    /// The amount of tokens this account holds.
    pub amount: u64,
    /// If `delegate` is `Some` then `delegated_amount` represents
    /// the amount authorized by the delegate
    pub delegate: COption<Pubkey>,
    /// The account's state
    pub state: AccountState,
    /// If is_native.is_some, this is a native token, and the value logs the
    /// rent-exempt reserve. An Account is required to be rent-exempt, so
    /// the value is used by the Processor to ensure that wrapped SOL
    /// accounts do not drop below this threshold.
    pub is_native: COption<u64>,
    /// The amount delegated
    pub delegated_amount: u64,
    /// Optional authority to close the account.
    pub close_authority: COption<Pubkey>,
}
```

एक वॉलेट को प्रत्येक टोकन (mint) के लिए एक token account की आवश्यकता होती है
जिसे वह रखना चाहता है, जिसमें वॉलेट एड्रेस को token account के मालिक के रूप में
सेट किया जाता है। प्रत्येक वॉलेट एक ही टोकन (mint) के लिए कई token account रख
सकता है, लेकिन एक token account का केवल एक ही मालिक हो सकता है और एक ही टोकन
(mint) की इकाइयां रख सकता है।

![Account Relationship](/assets/docs/core/tokens/token-account-relationship.svg)

<Callout type="info">

ध्यान दें कि प्रत्येक token account के डेटा में एक `owner` फ़ील्ड शामिल है जो
बताता है कि token account पर किसका अधिकार है। यह बेस
[Account](/docs/core/accounts#account-type) प्रकार में निर्दिष्ट प्रोग्राम मालिक
से भिन्न है, जो सभी token account के लिए Token Program है।

</Callout>

### Associated Token Account

Associated token account विशिष्ट mint और मालिक के लिए token account का पता खोजने
की प्रक्रिया को सरल बनाते हैं। Associated token account को एक विशिष्ट mint और
मालिक के लिए "डिफ़ॉल्ट" token account के रूप में समझें।

एक एसोसिएटेड टोकन अकाउंट (Associated Token Account) मालिक के पते और मिंट अकाउंट
के पते से व्युत्पन्न एक पते के साथ बनाया जाता है। यह समझना महत्वपूर्ण है कि
एसोसिएटेड टोकन अकाउंट केवल एक विशिष्ट पते वाला टोकन अकाउंट है।

यह सोलाना डेवलपमेंट में एक महत्वपूर्ण अवधारणा प्रस्तुत करता है:
[प्रोग्राम डिराइव्ड एड्रेस (PDA)](/docs/core/pda)। एक PDA पूर्वनिर्धारित इनपुट
का उपयोग करके निश्चित रूप से एक पता व्युत्पन्न करता है, जिससे किसी अकाउंट का पता
खोजना आसान हो जाता है।

![एसोसिएटेड टोकन अकाउंट](/assets/docs/core/tokens/associated-token-account.svg)

ध्यान दें कि प्रत्येक वॉलेट को एक ही मिंट से टोकन रखने के लिए अपना स्वयं का टोकन
अकाउंट चाहिए।

![अकाउंट्स रिलेशनशिप एक्सपैंडेड](/assets/docs/core/tokens/token-account-relationship-ata.svg)

## टोकन CLI उदाहरण

_spl-token_ CLI आपको SPL टोकन के साथ प्रयोग करने में मदद करता है। नीचे दिए गए
उदाहरण स्थानीय इंस्टॉलेशन के बिना सीधे ब्राउज़र में CLI कमांड चलाने के लिए
[सोलाना प्लेग्राउंड](https://beta.solpg.io/) टर्मिनल का उपयोग करते हैं।

टोकन और अकाउंट बनाने के लिए अकाउंट रेंट डिपॉजिट और ट्रांजैक्शन फीस के लिए SOL की
आवश्यकता होती है। पहली बार सोलाना प्लेग्राउंड उपयोगकर्ताओं के लिए, एक
प्लेग्राउंड वॉलेट बनाएं और प्लेग्राउंड टर्मिनल में _solana airdrop 1_ कमांड
चलाएं। आप सार्वजनिक [वेब फॉसेट](https://faucet.solana.com/) से भी डेवनेट SOL
प्राप्त कर सकते हैं।

```bash
solana airdrop 1
```

सभी उपलब्ध कमांड देखने के लिए _spl-token --help_ चलाएं।

```bash
spl-token --help
```

स्थानीय रूप से _spl-token_ CLI इंस्टॉल करने के लिए, चलाएं:

```bash
cargo install spl-token-cli
```

_spl-token_ [डॉक्यूमेंटेशन](https://www.solana-program.com/docs/token) में अधिक
उदाहरण खोजें।

<Callout type="info">

आपके टर्मिनल आउटपुट में दिखाए गए अकाउंट पते नीचे दिए गए उदाहरणों से भिन्न हैं।
अनुसरण करते समय अपने प्लेग्राउंड टर्मिनल के पतों का उपयोग करें। उदाहरण के लिए,
_spl-token create-token_ कमांड आपके प्लेग्राउंड वॉलेट को मिंट अथॉरिटी के रूप में
एक मिंट अकाउंट पता आउटपुट करता है।

</Callout>

### एक नया टोकन बनाएं

एक नया टोकन (मिंट अकाउंट) बनाने के लिए, चलाएं:

```bash
spl-token create-token
```

आउटपुट इस प्रकार दिखता है:

```bash
Creating token 7EYQDdNKNJbQwJN8sGgCpuEHnXLZGEYpYeJsA9BMmzYW

Address:  7EYQDdNKNJbQwJN8sGgCpuEHnXLZGEYpYeJsA9BMmzYW
Decimals:  9

Signature: 5f4hyRQfSDyQB5ZDGTcLwZ9wnvLwxRk1K44rYD1xCRBL3phKiR3zFWxRKAz7gNtHKJjdLGbxseMgDMwkpgDRQJpz
```

एक नया mint account शून्य सप्लाई से शुरू होता है। वर्तमान सप्लाई की जांच करें:

```terminal
$ spl-token supply <TOKEN_ADDRESS>
```

नए टोकन के लिए _sh`supply`_ कमांड `0` लौटाता है:

```sh title="Example"
spl-token supply 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
```

एक नया Mint Account बनाने के लिए दो निर्देशों वाले ट्रांजैक्शन की आवश्यकता होती
है। यहां [Solana Playground](https://beta.solpg.io/660ce32ecffcf4b13384d00f) पर
एक जावास्क्रिप्ट उदाहरण है।

1. सिस्टम प्रोग्राम Mint Account डेटा के लिए स्थान के साथ एक नया अकाउंट बनाता है
   और स्वामित्व Token Program को स्थानांतरित करता है।

2. Token Program नए अकाउंट के डेटा को Mint Account के रूप में इनिशियलाइज़ करता
   है

### टोकन अकाउंट बनाएं

किसी विशिष्ट mint के टोकन रखने के लिए, एक token account बनाएं:

```terminal
$ spl-token create-account <TOKEN_ADDRESS>
```

उदाहरण कमांड:

```sh title="Example"
spl-token create-account 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
```

आउटपुट:

```sh title="Terminal Output"
Creating account AfB7uwBEsGtrrBqPTVqEgzWed5XdYfM1psPNLmf7EeX9

Signature: 2BtrynuCLX9CNofFiaw6Yzbx6hit66pup9Sk7aFjwU2NEbFz7NCHD9w9sWhrCfEd73XveAGK1DxFpJoQZPXU9tS1
```

_sh`create-account`_ कमांड आपके वॉलेट एड्रेस को मालिक के रूप में एक associated
token account बनाता है।

अलग मालिक के साथ एक token account बनाने के लिए:

```terminal
$ spl-token create-account --owner <OWNER_ADDRESS> <TOKEN_ADDRESS>
```

नोट: `<TOKEN_ADDRESS>` token account के लिए mint account निर्दिष्ट करता है।

उदाहरण कमांड:

```sh title="Example"
spl-token create-account --owner 2i3KvjDCZWxBsqcxBHpdEaZYQwQSYE6LXUMx5VjY5XrR 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
```

आउटपुट:

```shell title="Terminal Output" /Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt/
Creating account Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt

Signature: 44vqKdfzspT592REDPY4goaRJH3uJ3Ce13G4BCuUHg35dVUbHuGTHvqn4ZjYF9BGe9QrjMfe9GmuLkQhSZCBQuEt
```

एक Associated Token Account बनाने के लिए एक निर्देश की आवश्यकता होती है जो
[Associated Token Program](https://github.com/solana-program/associated-token-account)
को इनवोक करता है। यहां
[Solana Playground](https://beta.solpg.io/660ce868cffcf4b13384d011) पर एक
जावास्क्रिप्ट उदाहरण है।

Associated Token Program [क्रॉस प्रोग्राम इनवोकेशन्स](/docs/core/cpi) का उपयोग
करता है:

- प्रदान किए गए PDA को एड्रेस के रूप में उपयोग करके नया अकाउंट बनाने के लिए
  [सिस्टम प्रोग्राम को इनवोक करें](https://github.com/solana-program/associated-token-account/blob/9d94201e8158f06015ff80ad47fefac62a2ec450/program/src/tools/account.rs#L19)
- Token Account डेटा को इनिशियलाइज़ करने के लिए
  [Token Program को इनवोक करें](https://github.com/solana-program/associated-token-account/blob/9d94201e8158f06015ff80ad47fefac62a2ec450/program/src/processor.rs#L138-L161)

Associated Token Account के पते के बजाय नए keypair के साथ एक नया Token Account
बनाने के लिए, दो निर्देशों के साथ एक लेनदेन भेजें। यहां
[Solana Playground](https://beta.solpg.io/660ce716cffcf4b13384d010) पर एक
Javascript उदाहरण है।

1. System Program Token Account डेटा के लिए स्थान के साथ एक नया खाता बनाता है और
   Token Program को स्वामित्व हस्तांतरित करता है।

2. Token Program डेटा को Token Account के रूप में प्रारंभ करता है

### टोकन मिंट करें

टोकन की नई इकाइयां बनाने के लिए, Token Account में टोकन मिंट करें:

```terminal
$ spl-token mint [OPTIONS] <TOKEN_ADDRESS> <TOKEN_AMOUNT> [--] [RECIPIENT_TOKEN_ACCOUNT_ADDRESS]
```

उदाहरण कमांड:

```sh title="Example"
spl-token mint 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg 100
```

आउटपुट:

```shell title="Terminal Output"
Minting 100 tokens
  Token: 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg
  Recipient: AfB7uwBEsGtrrBqPTVqEgzWed5XdYfM1psPNLmf7EeX9
```

### टोकन ट्रांसफर करें

token accounts के बीच टोकन ट्रांसफर करने के लिए:

```sh title="Terminal"
spl-token transfer [OPTIONS] <TOKEN_ADDRESS> <TOKEN_AMOUNT> <RECIPIENT_ADDRESS or RECIPIENT_TOKEN_ACCOUNT_ADDRESS>
```

उदाहरण कमांड:

```sh title="Example"
spl-token transfer 99zqUzQGohamfYxyo8ykTEbi91iom3CLmwCA75FK5zTg 100 Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt
```

आउटपुट:

```shell title="Terminal Output"
Transfer 100 tokens
  Sender: AfB7uwBEsGtrrBqPTVqEgzWed5XdYfM1psPNLmf7EeX9
  Recipient: Hmyk3FSw4cfsuAes7sanp2oxSkE9ivaH6pMzDzbacqmt

Signature: 5y6HVwV8V2hHGLTVmTmdySRiEUCZnWmkasAvJ7J6m7JR46obbGKCBqUFgLpZu5zQGwM4Xy6GZ4M5LKd1h6Padx3o
```

Token Program पर _rs`Transfer`_ निर्देश टोकन ट्रांसफर को संभालता है। प्रेषक के
Token Account के मालिक को लेनदेन पर हस्ताक्षर करना होगा। यह निर्देश Token
Accounts के बीच टोकन स्थानांतरित करता है। यहां
[Solana Playground](https://beta.solpg.io/660ced84cffcf4b13384d013) पर एक
Javascript उदाहरण है।

प्रेषक और प्राप्तकर्ता दोनों को विशिष्ट टोकन (mint) के लिए token accounts की
आवश्यकता होती है। प्रेषक उसी लेनदेन में प्राप्तकर्ता का token account बनाने के
लिए निर्देश शामिल कर सकता है।

### टोकन मेटाडेटा बनाएं

टोकन एक्सटेंशन्स प्रोग्राम आपको मेटाडेटा (नाम, प्रतीक, छवि लिंक) सीधे मिंट
अकाउंट पर स्टोर करने की अनुमति देता है।

मेटाडेटा एक्सटेंशन के साथ टोकन बनाने के लिए:

```sh title="Example"
spl-token create-token --program-id TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb --enable-metadata
```

आउटपुट:

```shell title="Terminal Output"
Creating token BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP under program TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb
To initialize metadata inside the mint, please run `spl-token initialize-metadata BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP <YOUR_TOKEN_NAME> <YOUR_TOKEN_SYMBOL> <YOUR_TOKEN_URI>`, and sign with the mint authority.

Address:  BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP
Decimals:  9

Signature: 5iQofFeXdYhMi9uTzZghcq8stAaa6CY6saUwcdnELST13eNSifiuLbvR5DnRt311frkCTUh5oecj8YEvZSB3wfai
```

मेटाडेटा को इनिशियलाइज़ करने के लिए:

```sh title="Terminal"
spl-token initialize-metadata <TOKEN_MINT_ADDRESS> <YOUR_TOKEN_NAME> <YOUR_TOKEN_SYMBOL> <YOUR_TOKEN_URI>
```

टोकन URI ऑफ-चेन मेटाडेटा से लिंक करता है। JSON फॉर्मेट का एक उदाहरण
[यहां](https://raw.githubusercontent.com/solana-developers/opos-asset/main/assets/DeveloperPortal/metadata.json)
देखें।

उदाहरण कमांड:

```sh title="Example"
spl-token initialize-metadata BdhzpzhTD1MFqBiwNdrRy4jFo2FHFufw3n9e8sVjJczP "TokenName" "TokenSymbol" "https://raw.githubusercontent.com/solana-developers/opos-asset/main/assets/DeveloperPortal/metadata.json"
```

पर मेटाडेटा देखें।
