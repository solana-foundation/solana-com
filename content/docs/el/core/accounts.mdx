Στο Solana, όλα τα δεδομένα αποθηκεύονται σε αυτό που ονομάζεται "accounts"
(λογαριασμοί). Μπορείτε να σκεφτείτε τα δεδομένα στο Solana ως μια δημόσια βάση
δεδομένων με έναν μόνο πίνακα "Accounts", όπου κάθε εγγραφή σε αυτόν τον πίνακα
είναι ένας "account". Κάθε λογαριασμός Solana μοιράζεται τον ίδιο βασικό
[τύπο Account](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/account/src/lib.rs#L48-L60).

![Λογαριασμοί](/assets/docs/core/accounts/accounts.png)

## Βασικά Σημεία

- Οι λογαριασμοί μπορούν να αποθηκεύσουν έως και
  [10MiB](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/program/src/system_instruction.rs#L85)
  δεδομένων, τα οποία περιέχουν είτε εκτελέσιμο κώδικα προγράμματος είτε
  κατάσταση προγράμματος.
- Οι λογαριασμοί απαιτούν μια
  [κατάθεση rent](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/rent/src/lib.rs#L93-L97)
  σε lamports (SOL) που είναι ανάλογη με την ποσότητα των αποθηκευμένων
  δεδομένων, και μπορείτε να την ανακτήσετε πλήρως όταν κλείσετε τον λογαριασμό.
- Κάθε λογαριασμός έχει έναν
  [ιδιοκτήτη](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/account/src/lib.rs#L55)
  προγράμματος. Μόνο το πρόγραμμα που κατέχει έναν λογαριασμό μπορεί να αλλάξει
  τα δεδομένα του ή να αφαιρέσει το υπόλοιπο lamport. Αλλά ο καθένας μπορεί να
  αυξήσει το υπόλοιπο.
- **Λογαριασμοί Sysvar** είναι ειδικοί λογαριασμοί που αποθηκεύουν την κατάσταση
  του δικτύου cluster.
- **Λογαριασμοί προγράμματος** αποθηκεύουν τον εκτελέσιμο κώδικα των έξυπνων
  συμβολαίων.
- **Λογαριασμοί δεδομένων** δημιουργούνται από προγράμματα για την αποθήκευση
  και διαχείριση της κατάστασης του προγράμματος.

## Λογαριασμός

Κάθε λογαριασμός στο Solana έχει μια μοναδική διεύθυνση 32-byte, που συχνά
εμφανίζεται ως κωδικοποιημένη συμβολοσειρά base58 (π.χ.
`vines1vzrYbzLMRdu58ou5XTby4qAqVRLmqo36NKPTg`).

Η σχέση μεταξύ του λογαριασμού και της διεύθυνσής του λειτουργεί σαν ζεύγος
κλειδιού-τιμής, όπου η διεύθυνση είναι το κλειδί για τον εντοπισμό των
αντίστοιχων on-chain δεδομένων του λογαριασμού. Η διεύθυνση του λογαριασμού
λειτουργεί ως το "μοναδικό αναγνωριστικό" για κάθε εγγραφή στον πίνακα
"Accounts".

![Διεύθυνση Λογαριασμού](/assets/docs/core/accounts/account-address.svg)

Οι περισσότεροι λογαριασμοί Solana χρησιμοποιούν ένα δημόσιο κλειδί
[Ed25519](https://ed25519.cr.yp.to/) ως τη διεύθυνσή τους.

<CodeTabs storage="accounts" flags="r">

```ts !! title="Kit"
import { generateKeyPairSigner } from "@solana/kit";

// Kit does not enable extractable private keys
const keypairSigner = await generateKeyPairSigner();
console.log(keypairSigner);
```

```ts !! title="Legacy"
import { Keypair } from "@solana/web3.js";

const keypair = Keypair.generate();
console.log(`Public Key: ${keypair.publicKey}`);
console.log(`Secret Key: ${keypair.secretKey}`);
```

```rs !! title="Rust"
use solana_sdk::signer::{keypair::Keypair, Signer};

#[tokio::main]
async fn main() {
    let keypair = Keypair::new();
    println!("Public Key: {}", keypair.pubkey());
    println!("Secret Key: {:?}", keypair.to_bytes());
}
```

</CodeTabs>

Ενώ τα δημόσια κλειδιά χρησιμοποιούνται συνήθως ως διευθύνσεις λογαριασμών, το
Solana υποστηρίζει επίσης ένα χαρακτηριστικό που ονομάζεται
[Διευθύνσεις Παραγόμενες από Προγράμματα](/docs/core/pda) (PDAs). Οι PDAs είναι
ειδικές διευθύνσεις που μπορείτε να παράγετε ντετερμινιστικά από ένα
αναγνωριστικό προγράμματος και προαιρετικές εισόδους (seeds).

<CodeTabs storage="accounts" flags="r">

```ts !! title="Kit"
import { Address, getProgramDerivedAddress } from "@solana/kit";

const programAddress = "11111111111111111111111111111111" as Address;

const seeds = ["helloWorld"];
const [pda, bump] = await getProgramDerivedAddress({
  programAddress,
  seeds
});

console.log(`PDA: ${pda}`);
console.log(`Bump: ${bump}`);
```

```ts !! title="Legacy"
import { PublicKey } from "@solana/web3.js";

const programAddress = new PublicKey("11111111111111111111111111111111");

const seeds = [Buffer.from("helloWorld")];
const [pda, bump] = await PublicKey.findProgramAddressSync(
  seeds,
  programAddress
);

console.log(`PDA: ${pda}`);
console.log(`Bump: ${bump}`);
```

```rs !! title="Rust"
use solana_sdk::pubkey; // macro
use solana_sdk::pubkey::Pubkey;

#[tokio::main]
async fn main() {
    let program_address = pubkey!("11111111111111111111111111111111");
    let seeds = [b"helloWorld".as_ref()];
    let (pda, bump) = Pubkey::find_program_address(&seeds, &program_address);
    println!("PDA: {}", pda);
    println!("Bump: {}", bump);
}
```

</CodeTabs>

### Τύπος Λογαριασμού

Οι λογαριασμοί έχουν μέγιστο μέγεθος
[10MiB](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/program/src/system_instruction.rs#L85)
και κάθε λογαριασμός στο Solana μοιράζεται τον ίδιο βασικό τύπο
[Account](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/account/src/lib.rs#L48-L60).

![Τύπος Λογαριασμού](/assets/docs/core/accounts/account-type.svg)

Κάθε λογαριασμός στο Solana έχει τα ακόλουθα πεδία:

- `data`: Ένας πίνακας byte που αποθηκεύει αυθαίρετα δεδομένα για έναν
  λογαριασμό. Για μη εκτελέσιμους λογαριασμούς, αυτό συχνά αποθηκεύει την
  κατάσταση που προορίζεται για ανάγνωση. Για λογαριασμούς προγραμμάτων (έξυπνα
  συμβόλαια), αυτό περιέχει τον εκτελέσιμο κώδικα προγράμματος. Το πεδίο
  δεδομένων συνήθως ονομάζεται "δεδομένα λογαριασμού".
- `executable`: Αυτή η σημαία δείχνει αν ένας λογαριασμός είναι πρόγραμμα.
- `lamports`: Το υπόλοιπο του λογαριασμού σε lamports, τη μικρότερη μονάδα του
  SOL (1 SOL = 1 δισεκατομμύριο lamports).
- `owner`: Το αναγνωριστικό προγράμματος (δημόσιο κλειδί) του προγράμματος που
  κατέχει αυτόν τον λογαριασμό. Μόνο το πρόγραμμα ιδιοκτήτης μπορεί να αλλάξει
  τα δεδομένα του λογαριασμού ή να αφαιρέσει το υπόλοιπο των lamports του.
- `rent_epoch`: Ένα παλαιό πεδίο από την εποχή που το Solana είχε έναν μηχανισμό
  που αφαιρούσε περιοδικά lamports από τους λογαριασμούς. Ενώ αυτό το πεδίο
  εξακολουθεί να υπάρχει στον τύπο Account, δεν χρησιμοποιείται πλέον αφού η
  συλλογή rent έχει καταργηθεί.

```rust title="Base Account Type"
pub struct Account {
    /// lamports in the account
    pub lamports: u64,
    /// data held in this account
    #[cfg_attr(feature = "serde", serde(with = "serde_bytes"))]
    pub data: Vec<u8>,
    /// the program that owns this account. If executable, the program that loads this account.
    pub owner: Pubkey,
    /// this account's data contains a loaded program (and is now read-only)
    pub executable: bool,
    /// the epoch at which this account will next owe rent
    pub rent_epoch: Epoch,
}
```

<CodeTabs storage="accounts" flags="r">

```ts !! title="Kit"
import {
  airdropFactory,
  createSolanaRpc,
  createSolanaRpcSubscriptions,
  generateKeyPairSigner,
  lamports
} from "@solana/kit";

// Create a connection to Solana cluster
const rpc = createSolanaRpc("http://localhost:8899");
const rpcSubscriptions = createSolanaRpcSubscriptions("ws://localhost:8900");

// Generate a new keypair
const keypair = await generateKeyPairSigner();
console.log(`Public Key: ${keypair.address}`);

// Funding an address with SOL automatically creates an account
const signature = await airdropFactory({ rpc, rpcSubscriptions })({
  recipientAddress: keypair.address,
  lamports: lamports(1_000_000_000n),
  commitment: "confirmed"
});

const accountInfo = await rpc.getAccountInfo(keypair.address).send();
console.log(accountInfo);
```

```ts !! title="Legacy"
import { Keypair, Connection, LAMPORTS_PER_SOL } from "@solana/web3.js";

// Generate a new keypair
const keypair = Keypair.generate();
console.log(`Public Key: ${keypair.publicKey}`);

// Create a connection to the Solana cluster
const connection = new Connection("http://localhost:8899", "confirmed");

// Funding an address with SOL automatically creates an account
const signature = await connection.requestAirdrop(
  keypair.publicKey,
  LAMPORTS_PER_SOL
);
await connection.confirmTransaction(signature, "confirmed");

const accountInfo = await connection.getAccountInfo(keypair.publicKey);
console.log(JSON.stringify(accountInfo, null, 2));
```

```rs !! title="Rust"
use anyhow::Result;
use solana_client::nonblocking::rpc_client::RpcClient;
use solana_sdk::{
    commitment_config::CommitmentConfig,
    native_token::LAMPORTS_PER_SOL,
    signer::{keypair::Keypair, Signer},
};

#[tokio::main]
async fn main() -> Result<()> {
    // Generate a new keypair
    let keypair = Keypair::new();
    println!("Public Key: {}", keypair.pubkey());

    // Create a connection to Solana cluster
    let connection = RpcClient::new_with_commitment(
        "http://localhost:8899".to_string(),
        CommitmentConfig::confirmed(),
    );

    // Funding an address with SOL automatically creates an account
    let signature = connection
        .request_airdrop(&keypair.pubkey(), LAMPORTS_PER_SOL)
        .await?;
    connection.confirm_transaction(&signature).await?;

    let account_info = connection.get_account(&keypair.pubkey()).await?;
    println!("{:#?}", account_info);

    Ok(())
}
```

### Rent

Για την αποθήκευση δεδομένων στην αλυσίδα, οι λογαριασμοί πρέπει επίσης να
διατηρούν ένα υπόλοιπο lamport (SOL) που είναι ανάλογο με την ποσότητα των
δεδομένων που αποθηκεύονται στον λογαριασμό (σε bytes). Αυτό το υπόλοιπο
ονομάζεται "rent", αλλά λειτουργεί περισσότερο σαν κατάθεση επειδή μπορείτε να
ανακτήσετε το πλήρες ποσό όταν κλείσετε έναν λογαριασμό. Μπορείτε να βρείτε τον
υπολογισμό
[εδώ](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/rent/src/lib.rs#L93-L97)
χρησιμοποιώντας αυτές τις
[σταθερές](https://github.com/anza-xyz/agave/blob/v2.1.13/sdk/rent/src/lib.rs#L47-L70).

Ο όρος "rent" προέρχεται από έναν παρωχημένο μηχανισμό που αφαιρούσε τακτικά
lamports από λογαριασμούς που έπεφταν κάτω από το όριο του rent. Αυτός ο
μηχανισμός δεν είναι πλέον ενεργός.

### Ιδιοκτήτης προγράμματος

Στο Solana, τα "έξυπνα συμβόλαια" ονομάζονται
[προγράμματα](/docs/core/programs). Η ιδιοκτησία προγράμματος είναι ένα βασικό
μέρος του Μοντέλου Λογαριασμών Solana. Κάθε λογαριασμός έχει έναν καθορισμένο
πρόγραμμα ως ιδιοκτήτη του. Μόνο το πρόγραμμα ιδιοκτήτης μπορεί:

- Να αλλάξει το πεδίο `data` του λογαριασμού
- Να αφαιρέσει lamports από το υπόλοιπο του λογαριασμού

## System Program

Από προεπιλογή, όλοι οι νέοι λογαριασμοί ανήκουν στο
[System Program](https://github.com/anza-xyz/agave/tree/v2.1.13/programs/system/src).
Το System Program κάνει μερικά βασικά πράγματα:

- [Δημιουργία νέου λογαριασμού](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/system/src/system_processor.rs#L146):
  Μόνο το System Program μπορεί να δημιουργήσει νέους λογαριασμούς.
- [Κατανομή χώρου](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/system/src/system_processor.rs#L71):
  Ορίζει τη χωρητικότητα σε bytes για το πεδίο δεδομένων κάθε λογαριασμού.
- [Μεταφορά / Ανάθεση ιδιοκτησίας προγράμματος](https://github.com/anza-xyz/agave/blob/v2.1.13/programs/system/src/system_processor.rs#L113):
  Μόλις το System Program δημιουργήσει έναν λογαριασμό, μπορεί να επανεκχωρήσει
  τον καθορισμένο ιδιοκτήτη προγράμματος σε διαφορετικό λογαριασμό προγράμματος.
  Έτσι τα προσαρμοσμένα προγράμματα αποκτούν ιδιοκτησία νέων λογαριασμών που
  δημιουργούνται από το System Program.

Όλοι οι λογαριασμοί "πορτοφολιού" στο Solana είναι απλώς λογαριασμοί που ανήκουν
στο System Program. Το υπόλοιπο lamport σε αυτούς τους λογαριασμούς δείχνει την
ποσότητα SOL που κατέχει το πορτοφόλι. Μόνο οι λογαριασμοί που ανήκουν στο
System Program μπορούν να πληρώσουν τέλη συναλλαγών.

![Λογαριασμός Συστήματος](/assets/docs/core/accounts/system-account.svg)

## Λογαριασμοί Sysvar

Οι λογαριασμοί Sysvar είναι ειδικοί λογαριασμοί σε προκαθορισμένες διευθύνσεις
που παρέχουν πρόσβαση σε δεδομένα κατάστασης του συστάδας. Αυτοί οι λογαριασμοί
ενημερώνονται δυναμικά με δεδομένα σχετικά με το δίκτυο της συστάδας. Μπορείτε
να βρείτε την πλήρη λίστα των Λογαριασμών Sysvar
[εδώ](https://docs.anza.xyz/runtime/sysvars).

<CodeTabs storage="accounts" flags="r">

```ts !! title="Kit"
import { Address, createSolanaRpc } from "@solana/kit";

const rpc = createSolanaRpc("https://api.mainnet-beta.solana.com");

const SYSVAR_CLOCK_ADDRESS =
  "SysvarC1ock11111111111111111111111111111111" as Address;

const accountInfo = await rpc
  .getAccountInfo(SYSVAR_CLOCK_ADDRESS, { encoding: "base64" })
  .send();
console.log(accountInfo);
```

```ts !! title="Legacy"
import { Connection, SYSVAR_CLOCK_PUBKEY } from "@solana/web3.js";

const connection = new Connection(
  "https://api.mainnet-beta.solana.com",
  "confirmed"
);

const accountInfo = await connection.getAccountInfo(SYSVAR_CLOCK_PUBKEY);
// !collapse(1:17) collapsed
console.log(
  JSON.stringify(
    accountInfo,
    (key, value) => {
      if (key === "data" && value && value.length > 1) {
        return [
          value[0],
          "...truncated, total bytes: " + value.length + "...",
          value[value.length - 1]
        ];
      }
      return value;
    },
    2
  )
);
```

```rs !! title="Rust"
        CommitmentConfig::confirmed(),
    );

    let account_info = connection.get_account(&sysvar::clock::ID).await?;
    println!("{:#?}", account_info);

    Ok(())
}
```

## Λογαριασμός Προγράμματος

Η ανάπτυξη ενός προγράμματος Solana δημιουργεί έναν εκτελέσιμο λογαριασμό
προγράμματος. Ο λογαριασμός προγράμματος αποθηκεύει τον εκτελέσιμο κώδικα του
προγράμματος.

Οι λογαριασμοί προγράμματος ανήκουν σε ένα
[Πρόγραμμα Φορτωτή](/docs/core/programs#loader-programs).

![Λογαριασμός Προγράμματος](/assets/docs/core/accounts/program-account-simple.svg)

Για απλότητα, μπορείτε να αντιμετωπίσετε τον λογαριασμό προγράμματος ως το ίδιο
το πρόγραμμα. Όταν επικαλείστε τις οδηγίες ενός προγράμματος, καθορίζετε τη
διεύθυνση του λογαριασμού προγράμματος (συνήθως αποκαλείται "Program ID").

<CodeTabs storage="accounts" flags="r">

```ts !! title="Kit"
import { Address, createSolanaRpc } from "@solana/kit";

const rpc = createSolanaRpc("https://api.mainnet-beta.solana.com");

const programId = "TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA" as Address;

const accountInfo = await rpc
  .getAccountInfo(programId, { encoding: "base64" })
  .send();

```

<Callout type="info">

Όταν αναπτύσσετε ένα πρόγραμμα Solana, αποθηκεύεται σε έναν λογαριασμό
προγράμματος. Οι λογαριασμοί προγράμματος ανήκουν σε ένα
[Πρόγραμμα Φορτωτή](/docs/core/programs#loader-programs). Υπάρχουν διάφορες
εκδόσεις του φορτωτή, αλλά όλες εκτός από τον loader-v3 αποθηκεύουν τον
εκτελέσιμο κώδικα απευθείας στον λογαριασμό προγράμματος. Ο loader-v3 αποθηκεύει
τον εκτελέσιμο κώδικα σε έναν ξεχωριστό "λογαριασμό δεδομένων προγράμματος" και
ο λογαριασμός προγράμματος απλώς δείχνει σε αυτόν. Όταν αναπτύσσετε ένα νέο
πρόγραμμα, το Solana CLI χρησιμοποιεί την τελευταία έκδοση φορτωτή ως
προεπιλογή.

</Callout>

### Λογαριασμός Buffer

Ο loader-v3 έχει έναν ειδικό τύπο λογαριασμού για την προσωρινή σταδιοποίηση της
μεταφόρτωσης ενός προγράμματος κατά την ανάπτυξη ή επανανάπτυξη/αναβαθμίσεις.
Στον loader-v4, εξακολουθούν να υπάρχουν buffers, αλλά είναι απλώς κανονικοί
λογαριασμοί προγράμματος.

### Λογαριασμός Δεδομένων Προγράμματος

Ο loader-v3 λειτουργεί διαφορετικά από όλα τα άλλα προγράμματα BPF Loader. Ο
λογαριασμός προγράμματος περιέχει μόνο τη διεύθυνση ενός λογαριασμού δεδομένων

![Λογαριασμός Δεδομένων Προγράμματος](/assets/docs/core/accounts/program-account-expanded.svg)

Μην συγχέετε αυτούς τους λογαριασμούς δεδομένων προγράμματος με τους
λογαριασμούς δεδομένων των προγραμμάτων (δείτε παρακάτω).

## Λογαριασμός Δεδομένων

Στο Solana, ο εκτελέσιμος κώδικας ενός προγράμματος αποθηκεύεται σε διαφορετικό
λογαριασμό από την κατάσταση του προγράμματος. Αυτό είναι παρόμοιο με τον τρόπο
που τα λειτουργικά συστήματα συνήθως έχουν ξεχωριστά αρχεία για τα προγράμματα
και τα δεδομένα τους.

Για να διατηρήσουν την κατάσταση, τα προγράμματα ορίζουν οδηγίες για τη
δημιουργία ξεχωριστών λογαριασμών που τους ανήκουν. Κάθε ένας από αυτούς τους
λογαριασμούς έχει τη δική του μοναδική διεύθυνση και μπορεί να αποθηκεύσει
οποιαδήποτε αυθαίρετα δεδομένα που ορίζονται από το πρόγραμμα.

![Λογαριασμός Δεδομένων](/assets/docs/core/accounts/data-account.svg)

Σημειώστε ότι μόνο το System Program μπορεί να δημιουργήσει νέους λογαριασμούς.
Μόλις το System Program δημιουργήσει έναν λογαριασμό, μπορεί στη συνέχεια να
μεταφέρει ή να αναθέσει την ιδιοκτησία του νέου λογαριασμού σε άλλο πρόγραμμα.

Με άλλα λόγια, η δημιουργία ενός λογαριασμού δεδομένων για ένα προσαρμοσμένο
πρόγραμμα απαιτεί δύο βήματα:

1. Επίκληση του System Program για τη δημιουργία ενός λογαριασμού και στη
   συνέχεια μεταφορά της ιδιοκτησίας στο προσαρμοσμένο πρόγραμμα
2. Επίκληση του προσαρμοσμένου προγράμματος, το οποίο τώρα κατέχει τον
   λογαριασμό, για την αρχικοποίηση των δεδομένων του λογαριασμού όπως ορίζεται
   από την οδηγία του προγράμματος

Αυτή η διαδικασία δημιουργίας λογαριασμού συχνά αφαιρείται ως ένα μόνο βήμα,
αλλά είναι χρήσιμο να κατανοήσετε την υποκείμενη διαδικασία.

```ts !! title="Kit"
import {
  airdropFactory,
  appendTransactionMessageInstructions,
  createSolanaRpc,
  createSolanaRpcSubscriptions,
  createTransactionMessage,
  generateKeyPairSigner,
  getSignatureFromTransaction,
  lamports,
  pipe,
  sendAndConfirmTransactionFactory,
  setTransactionMessageFeePayerSigner,
  setTransactionMessageLifetimeUsingBlockhash,
  signTransactionMessageWithSigners
} from "@solana/kit";
import { getCreateAccountInstruction } from "@solana-program/system";
import {
  getInitializeMintInstruction,
  getMintSize,
  TOKEN_2022_PROGRAM_ADDRESS
} from "@solana-program/token-2022";

// Create Connection, local validator in this example
const rpc = createSolanaRpc("http://127.0.0.1:8899");
const rpcSubscriptions = createSolanaRpcSubscriptions("ws://localhost:8900");

// Generate keypairs for fee payer
const feePayer = await generateKeyPairSigner();

// Fund fee payer
await airdropFactory({ rpc, rpcSubscriptions })({
  recipientAddress: feePayer.address,
  lamports: lamports(1_000_000_000n),
  commitment: "confirmed"
});

// Generate keypair to use as address of mint
const mint = await generateKeyPairSigner();

// Get default mint account size (in bytes), no extensions enabled
const space = BigInt(getMintSize());

// Get minimum balance for rent exemption
const rent = await rpc.getMinimumBalanceForRentExemption(space).send();

// Instruction to create new account for mint (token 2022 program)
// Invokes the system program
const createAccountInstruction = getCreateAccountInstruction({
  payer: feePayer,
  newAccount: mint,
  lamports: rent,
  space,
  programAddress: TOKEN_2022_PROGRAM_ADDRESS
});

// Instruction to initialize mint account data
// Invokes the token 2022 program
const initializeMintInstruction = getInitializeMintInstruction({
  mint: mint.address,
  decimals: 9,
  mintAuthority: feePayer.address
});

const instructions = [createAccountInstruction, initializeMintInstruction];

// Get latest blockhash to include in transaction
const { value: latestBlockhash } = await rpc.getLatestBlockhash().send();

// Create transaction message
const transactionMessage = pipe(
  createTransactionMessage({ version: 0 }), // Create transaction message
  (tx) => setTransactionMessageFeePayerSigner(feePayer, tx), // Set fee payer
  (tx) => setTransactionMessageLifetimeUsingBlockhash(latestBlockhash, tx), // Set transaction blockhash
  (tx) => appendTransactionMessageInstructions(instructions, tx) // Append instructions
);

// Sign transaction message with required signers (fee payer and mint keypair)
const signedTransaction =
  await signTransactionMessageWithSigners(transactionMessage);

// Send and confirm transaction
await sendAndConfirmTransactionFactory({ rpc, rpcSubscriptions })(
  signedTransaction,
  { commitment: "confirmed" }
);

// Get transaction signature
const transactionSignature = getSignatureFromTransaction(signedTransaction);

console.log("Mint Address:", mint.address);
console.log("Transaction Signature:", transactionSignature);

const accountInfo = await rpc.getAccountInfo(mint.address).send();
console.log(accountInfo);
```

```ts !! title="Legacy"
import {
  Connection,
  Keypair,
  sendAndConfirmTransaction,
  SystemProgram,
  Transaction,
  LAMPORTS_PER_SOL
} from "@solana/web3.js";
import {
  createInitializeMintInstruction,
  TOKEN_2022_PROGRAM_ID,
  MINT_SIZE,
  getMinimumBalanceForRentExemptMint
} from "@solana/spl-token";

// Create connection to local validator
const connection = new Connection("http://localhost:8899", "confirmed");
const recentBlockhash = await connection.getLatestBlockhash();

// Generate a new keypair for the fee payer
const feePayer = Keypair.generate();

// Airdrop 1 SOL to fee payer
const airdropSignature = await connection.requestAirdrop(
  feePayer.publicKey,
  LAMPORTS_PER_SOL
);
await connection.confirmTransaction({
  blockhash: recentBlockhash.blockhash,
  lastValidBlockHeight: recentBlockhash.lastValidBlockHeight,
  signature: airdropSignature
});

// Generate keypair to use as address of mint
const mint = Keypair.generate();

const createAccountInstruction = SystemProgram.createAccount({
  fromPubkey: feePayer.publicKey,
  newAccountPubkey: mint.publicKey,
  space: MINT_SIZE,
  lamports: await getMinimumBalanceForRentExemptMint(connection),
  programId: TOKEN_2022_PROGRAM_ID
});

const initializeMintInstruction = createInitializeMintInstruction(
  mint.publicKey, // mint pubkey
  9, // decimals
  feePayer.publicKey, // mint authority
  feePayer.publicKey, // freeze authority
  TOKEN_2022_PROGRAM_ID
);

const transaction = new Transaction().add(
  createAccountInstruction,
  initializeMintInstruction
);

const transactionSignature = await sendAndConfirmTransaction(
  connection,
  transaction,
  [feePayer, mint] // Signers
);

console.log("Mint Address: ", mint.publicKey.toBase58());
console.log("Transaction Signature: ", transactionSignature);

const accountInfo = await connection.getAccountInfo(mint.publicKey);

      }
      return value;
    },
    2
  )
);
```

```rs !! title="Rust"
use anyhow::Result;
use solana_client::nonblocking::rpc_client::RpcClient;
use solana_sdk::{
    commitment_config::CommitmentConfig,
    program_pack::Pack,
    signature::{Keypair, Signer},
    system_instruction::create_account,
    transaction::Transaction,
};
use spl_token_2022::{id as token_2022_program_id, instruction::initialize_mint, state::Mint};

#[tokio::main]
async fn main() -> Result<()> {
    // Create connection to local validator
    let client = RpcClient::new_with_commitment(
        String::from("http://127.0.0.1:8899"),
        CommitmentConfig::confirmed(),
    );
    let recent_blockhash = client.get_latest_blockhash().await?;

    // Generate a new keypair for the fee payer
    let fee_payer = Keypair::new();

    // Airdrop 1 SOL to fee payer
    let airdrop_signature = client
        .request_airdrop(&fee_payer.pubkey(), 1_000_000_000)
        .await?;
    client.confirm_transaction(&airdrop_signature).await?;

    loop {
        let confirmed = client.confirm_transaction(&airdrop_signature).await?;
        if confirmed {
            break;
        }
    }

    // Generate keypair to use as address of mint
    let mint = Keypair::new();

    let space = Mint::LEN;
    let rent = client.get_minimum_balance_for_rent_exemption(space).await?;

    // Create account instruction
    let create_account_instruction = create_account(
        &fee_payer.pubkey(),      // fee payer
        &mint.pubkey(),           // mint address
        rent,                     // rent
        space as u64,             // space
        &token_2022_program_id(), // program id
    );

    // Initialize mint instruction
    let initialize_mint_instruction = initialize_mint(
        &token_2022_program_id(),
        &mint.pubkey(),            // mint address
        &fee_payer.pubkey(),       // mint authority
        Some(&fee_payer.pubkey()), // freeze authority
        9,                         // decimals
    )?;

    // Create transaction and add instructions
    let transaction = Transaction::new_signed_with_payer(
        &[create_account_instruction, initialize_mint_instruction],
        Some(&fee_payer.pubkey()),
        &[&fee_payer, &mint],
        recent_blockhash,
    );

    // Send and confirm transaction
    let transaction_signature = client.send_and_confirm_transaction(&transaction).await?;

    println!("Mint Address: {}", mint.pubkey());
    println!("Transaction Signature: {}", transaction_signature);

    let account_info = client.get_account(&mint.pubkey()).await?;
    println!("{:#?}", account_info);

    Ok(())
}
```
