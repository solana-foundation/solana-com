h1: 程序派生地址
---

在本节中，您将学习如何构建一个基本的创建、读取、更新、删除（CRUD）程序。

本指南演示了一个简单的程序，用户可以创建、更新和删除消息。每条消息都存在于一个由程序本身派生的确定性地址（程序派生地址或 PDA）中的账户中。

本指南将带您逐步构建和测试一个使用 Anchor 框架的 Solana 程序，同时演示程序派生地址（PDAs）。有关更多详细信息，请参阅
[程序派生地址](/docs/core/pda) 页面。

作为参考，您可以在完成 PDA 和跨程序调用（CPI）部分后查看
[最终代码](https://beta.solpg.io/668304cfcffcf4b13384d20a)。

### 定义消息账户类型

首先，定义程序创建的消息账户的结构。此结构定义了程序创建的账户中要存储的数据。

<WithNotes>

在 `lib.rs` 中，更新
_rs`MessageAccount`_
结构体，内容如下：

```rs title="lib.rs"
// !tooltip[/account/] account
#[account]
pub struct MessageAccount {
    // !tooltip[/user/] user
    pub user: Pubkey,
    // !tooltip[/message/] message
    pub message: String,
    // !tooltip[/bump/] bump
    pub bump: u8,
}
```

### !account

Anchor 程序中的
_rs`#[account]`_
属性用于注解表示账户数据的结构体（存储在账户数据字段中的数据类型）。

### !user

_rs`user`_ 字段包含一个
_rs`Pubkey`_，用于标识创建消息账户的用户。

### !message

_rs`message`_ 字段保存一个
_rs`String`_，其中包含用户的消息。

### !bump

_rs`bump`_ 字段存储一个
_rs`u8`_
["bump" 种子](/docs/core/pda#canonical-bump)，用于派生程序派生地址（PDA）。存储此值可以节省计算资源，因为无需在后续指令中重新计算。

### 添加创建指令

现在，添加 _rs`create`_
指令，该指令创建并初始化
_rs`MessageAccount`_。

首先，通过更新
_rs`Create`_
结构体定义指令所需的账户，如下所示：

<WithNotes>

```rs title="lib.rs"
#[derive(Accounts)]
// !tooltip[/instruction/] instruction
#[instruction(message: String)]
pub struct Create<'info> {
    // !tooltip[/mut/] mut
    #[account(mut)]
    // !tooltip[/Signer<'info>/] signer
    pub user: Signer<'info>,

    #[account(
        // !tooltip[/init/] init
        init,
        // !tooltip[/seeds/] seeds
        seeds = [b"message", user.key().as_ref()],
        // !tooltip[/bump/] bump
        bump,
        // !tooltip[/payer/] payer
        payer = user,
        // !tooltip[/space/] space
        space = 8 + 32 + 4 + message.len() + 1
    )]
    // !tooltip[/Account<'info, MessageAccount>/] account
    pub message_account: Account<'info, MessageAccount>,
    // !tooltip[/Program<'info, System>/] program
    pub system_program: Program<'info, System>,
}
```

### !mut

_rs`mut`_
约束声明该账户为可变账户。

### !init

_rs`init`_
约束创建一个新账户。

### !seeds

_rs`seeds`_
约束定义了用于派生 PDA 的可选输入。

### !bump

_rs`bump`_
约束声明了 PDA 的 bump seed。

如果您未指定值，Anchor 会自动计算。

### !payer

_rs`payer`_
约束指定哪个账户支付新账户的创建费用。

### !space

_rs`space`_
约束指定为新账户的数据字段分配的字节数。

### !signer

_rs`Signer<'info>`_
类型要求账户对交易进行签名。

### !account

_rs`Account<'info, T>`_
类型要求账户与指定类型匹配。

在这种情况下，该账户必须匹配自定义的
_rs`MessageAccount`_ 类型。

### !program

_rs`Program<'info, T>`_
类型要求账户匹配一个程序。

在这种情况下，该账户必须匹配
_rs`System`_
类型，该类型指的是系统程序（System Program）。

### !instruction

_rs`#[instruction(message: String)]`_ 注解允许
_rs`Create`_ 结构体访问
_rs`message`_
参数，该参数来自 `create` 指令。

</WithNotes>

<Accordion title="Explanation">

Anchor 程序中的
_rs`#[derive(Accounts)]`_
属性用于注解定义指令所需账户的结构体。

结构体中的每个字段表示一个账户，并通过以下两种方式进行验证：

1. 账户类型（例如
   _rs`Signer<'info>`_ 或
   _rs`Account<'info, T>`_），指定程序期望的账户类型
2. 可选约束条件（例如
   _rs`#[account(mut)]`_ 或
   _rs`#[account(init)]`_），定义额外的要求

通过这些方式，Anchor 能够自动验证传递给指令的账户并确保程序的安全性。

结构体中的字段名称用于在程序代码中访问账户，但不会影响验证。为了清晰起见，您应使用描述性名称。

在此示例中， _rs`Create`_
结构体定义了 _rs`create`_
指令所需的账户。

1. _rs`user: Signer<'info>`_

   - 表示创建消息账户的用户
   - 需要可变状态 (_rs`#[account(mut)]`_)，因为它为新账户支付费用
   - 必须签署交易以批准从该账户扣除 lamport

2. _rs`message_account: Account<'info, MessageAccount>`_

   - 存储用户消息的新账户
   - `init` 约束在指令执行期间创建账户
   - `seeds` 和 `bump` 约束将账户地址派生为程序派生地址（PDA）
   - `payer = user` 确定谁为新账户的创建支付费用
   - `space` 分配账户数据字段所需的字节数

3. _rs`system_program: Program<'info, System>`_

   - 必需用于创建账户
   - 在后台，`init` 约束调用系统程序，创建一个具有指定 `space` 的新账户，并将所有者更改为当前程序。

---

_rs`#[instruction(message: String)]`_ 注解允许
_rs`Create`_
结构体从 `create` 指令中访问
_rs`message`_ 参数。

---

`seeds` 和 `bump` 约束共同定义了账户地址为程序派生地址 (PDA)。

```rs title="lib.rs"
seeds = [b"message", user.key().as_ref()],
bump,
```

`seeds` 约束定义了用于派生 PDA 的可选输入。

- _rs`b"message"`_ - 一个固定字符串作为第一个 seed。
- _rs`user.key().as_ref()`_ - 公钥
  _rs`user`_
  账户作为第二个 seed。

`bump` 约束指示 Anchor 自动查找并使用正确的 bump
seed。Anchor 使用 `seeds` 和 `bump` 来派生 PDA。

---

`space` 计算
_rs`(8 + 32 + 4 + message.len() + 1)`_
为 `MessageAccount` 数据类型分配空间：

- Anchor 账户标识符 (discriminator)：8 字节
- 用户地址 (_rs`Pubkey`_)：32 字节
- 用户消息 (_rs`String`_)：4 字节用于长度 + 可变消息长度
- PDA bump seed
  (_rs`u8`_)：1 字节

```rs title="lib.rs"
#[account]
pub struct MessageAccount {
    pub user: Pubkey,
    pub message: String,
    pub bump: u8,
}
```

通过 Anchor 程序创建的所有账户需要 8 字节的账户标识符 (discriminator)，它作为账户类型的标识符，Anchor 在创建账户时会自动生成。

_rs`String`_
类型需要 4 字节来存储字符串的长度，其余长度包含实际数据。

</Accordions>

```rs title="lib.rs"
    msg!("Create Message: {}", message);
    let account_data = &mut ctx.accounts.message_account;
    account_data.user = ctx.accounts.user.key();
    account_data.message = message;
    account_data.bump = ctx.bumps.message_account;
    Ok(())
}
```

### 添加更新指令

接下来，添加 `update` 指令以更改 `MessageAccount` 为新消息。

使用以下内容更新 `Update` 结构：

<WithNotes>

```rs title="lib.rs"
#[derive(Accounts)]
#[instruction(message: String)]
pub struct Update<'info> {
    #[account(mut)]
    pub user: Signer<'info>,

    #[account(
        mut,
        seeds = [b"message", user.key().as_ref()],
```

### 添加删除指令

接下来，添加 _rs`delete`_
指令以关闭 _rs`MessageAccount`_。

使用以下内容更新
_rs`Delete`_ 结构体：

<WithNotes>

```rs title="lib.rs"
#[derive(Accounts)]
pub struct Delete<'info> {
    #[account(mut)]
    pub user: Signer<'info>,

    #[account(
        mut,
        // !tooltip[/seeds/] seeds

        // !tooltip[/bump/] bump

        // !tooltip[/close/] close
```

### !seeds

_rs`seeds`_
约束指定了用于派生 PDA 的种子。

### !bump

_rs`bump`_
约束指定了 PDA 的 bump seed。

在此情况下，程序使用存储在
_rs`message_account`_ 上的现有 bump
seed。

### !close

_rs`close`_
约束关闭账户。

在此情况下，_rs`user`_
账户接收来自关闭的
_rs`message_account`_ 的 lamports。
