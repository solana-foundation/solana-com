通常，Solana 智能合约（也称为
[程序](/docs/core/programs)）的工作区会具有以下文件结构：

```shell
.
├── app
├── migrations
├── node_modules
├── programs
├── target
└── tests
```

主程序是 `lib.rs` 文件，位于 `programs` 目录中，如下所示：

```shell
.
├── app
├── migrations
├── node_modules
├── programs
    ├── src
        ├── lib.rs
├── target
└── tests
```

随着程序变得更加复杂，通常需要将逻辑分离到多个文件中，如下所示：

```shell
├── programs
    ├── src
        ├── state.rs
        ├── instructions
            ├── instruction_1.rs
            ├── instruction_2.rs
            ├── instruction_3.rs
        ├── lib.rs
        ├── constants.rs
        ├── error.rs
        ├── mod.rs
```

对于
[原生 Rust 程序开发](/docs/programs/rust/)，您需要显式地为程序编写入口点和处理器，因此需要更多的文件：

```shell
├── program.rs
│   ├── src.rs
│   │   ├──assertions.rs
│   │   ├──entrypoint.rs
│   │   ├──error.rs
│   │   ├──instruction.rs
│   │   ├──lib.rs
│   │   ├──processor.rs
│   │   ├──state.rs
│   │   ├──utils.rs
│   ├── Cargo.toml
│   ├── keypair.json
│   ├── README.md
```
