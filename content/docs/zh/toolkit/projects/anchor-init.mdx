```shell
anchor init <project_name>
```

## 概述

这将初始化一个用于 Anchor 程序开发的简单工作区设置，其结构如下：

- `Anchor.toml`:
  Anchor 配置文件。
- `Cargo.toml`:
  Rust 工作区配置文件。
- `package.json`:
  JavaScript 依赖文件。
- `programs/`:
  Solana 程序 crates 目录。
- `app/`: 应用程序前端目录。
- `tests/`:
  JavaScript 集成测试目录。
- `migrations/deploy.js`: 部署脚本。

Anchor 框架抽象了许多复杂性，从而加快了程序开发。

## 构建和测试

在进行任何修改之前，可以先构建和测试此项目：

```shell
anchor build
```

```shell
anchor test
```

要开始编写自己的 Anchor 程序，请导航到 `programs/src/lib.rs`。

## 文件结构模板

对于更复杂的程序，使用更结构化的项目模板是最佳实践。这可以通过以下命令生成：

```shell
anchor init --template multiple
```

这将在 `programs/src` 内创建以下布局：

```shell
├── constants.rs
├── error.rs
├── instructions
│   ├── initialize.rs
│   └── mod.rs
├── lib.rs
└── state
    └── mod.rs
```

有关项目文件结构的最佳实践，请查看此
[文档](/docs/toolkit/projects/project-layout)。
