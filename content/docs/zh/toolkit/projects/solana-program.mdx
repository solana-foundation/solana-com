> 这是 [Solana 工具包](/docs/toolkit/)
> 的测试版本，目前仍在开发中。请将所有反馈作为 GitHub 问题提交到
> [这里](https://github.com/solana-foundation/mucho/issues/new?title=%5Btoolkit%5D%20)。

```shell
pnpm create solana-program
```

[`create-solana-program`](https://github.com/solana-program/create-solana-program)
初始化了一个全面的工作区，包含了进行 Solana 程序开发所需的一切。此模板允许您编写原生 Rust 程序或 Anchor 程序。

## 程序框架

运行此命令后，您可以选择 Shank 或 Anchor 作为程序框架：

- **Shank**
  创建一个基础的 Solana 智能合约，并使用 Shank 宏生成 IDL。有关 Shank 的更多信息，请阅读其
  [README](https://github.com/metaplex-foundation/shank)。

- **Anchor**
  使用 Anchor 框架创建智能合约，该框架抽象了许多复杂性，从而加快了程序开发速度。有关 Anchor 框架的更多信息，请阅读
  [Anchor 文档](https://www.anchor-lang.com/)。

### Anchor 框架

对于 **Anchor
Rust 开发**，在选择程序框架时请选择 Anchor。这将创建一个基础的 Anchor 计数器程序，项目结构如下：

```shell
├── program.rs
│   ├── src.rs
│   │   ├── lib.rs
│   ├── Cargo.toml
│   ├── keypair.json
│   ├── README.md
```

### 原生 Rust

对于
**原生 Rust 开发**，请确保在选择程序框架时选择 Shank。这将创建一个基础的计数器程序，项目结构如下：

```shell
├── program.rs
│   ├── src.rs
│   │   ├── assertions.rs
│   │   ├──entrypoint.rs
│   │   ├──error.rs
│   │   ├──instruction.rs
│   │   ├──lib.rs
│   │   ├──processor.rs
│   │   ├──state.rs
│   │   ├──utils.rs
│   ├── Cargo.toml
│   ├── keypair.json
│   ├── README.md
```

## 生成的客户端

接下来，您可以选择 JavaScript 客户端、Rust 客户端，或两者兼选。

- **JavaScript 客户端** 创建一个与
  [web3.js](https://solana-labs.github.io/solana-web3.js/)
  兼容的 TypeScript 库。

- **Rust 客户端** 创建一个 Rust crate，允许用户与程序交互。

有关工作区的进一步自定义和更多信息，请查看 `create-solana-program`
[README](https://github.com/solana-program/create-solana-program?tab=readme-ov-file)。

## 构建

在回答上述提示后，工作区将被生成。要开始，请通过运行以下命令构建您的程序和客户端：

```shell
cd <my-program-name>
npm install
npm run generate
```

要将现有的 Anchor 项目更新为此模板，请阅读此
[指南](/docs/toolkit/projects/existing-project)。
