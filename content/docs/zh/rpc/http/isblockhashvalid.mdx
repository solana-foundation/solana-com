h1: isBlockhashValid RPC 方法
---

返回一个区块哈希是否仍然有效

<Callout type="warn" title={"版本限制"}>
  此方法仅适用于 `solana-core`
  v1.9 或更新版本。 请使用
  [getFeeCalculatorForBlockhash](/docs/rpc/http/getfeecalculatorforblockhash)
  适用于 `solana-core` v1.8
  及以下版本。
</Callout>

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 45,
  "method": "isBlockhashValid",
  "params": [
    // !hover blockhash
    "J7rBdM6AecPDEZp8aPq5iPSNKVkU5Q76F3oAV4eW5wsW",
    // !hover(1:3) config
    {
      // !hover commitment
      "commitment": "processed"
    }
  ]
}
```

### !参数

#### !! blockhash

!类型 string  
!必需

要评估的区块的区块哈希，作为 base-58 编码的字符串

#### !! config

!类型 object

包含以下字段的配置对象：

##### !! commitment

!类型 string

承诺描述了区块在某一时间点的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! minContextSlot

!类型 number

请求可以评估的最小 slot

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  "result": {
    "context": { "slot": 2483 },
    // !hover result
    "value": false
  },
  "id": 45
}
```

!类型 bool

区块哈希是否仍然有效

</APIMethod>
