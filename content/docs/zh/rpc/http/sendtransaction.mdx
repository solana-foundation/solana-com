h1: sendTransaction RPC 方法
---

提交已签名的交易到集群进行处理。

此方法不会以任何方式更改交易；它将客户端创建的交易原样传递给节点。

如果节点的 RPC 服务接收到交易，此方法会立即成功，而无需等待任何确认。此方法的成功响应并不保证交易已被集群处理或确认。

尽管 RPC 服务会合理地重试提交，但如果交易的 `recent_blockhash` 在落地之前过期，交易可能会被拒绝。

使用
[`getSignatureStatuses`](/docs/rpc/http/getsignaturestatuses)
确保交易已被处理并确认。

在提交之前，将执行以下预检：

1. 验证交易签名
2. 根据预检承诺指定的银行 slot 模拟交易。如果失败，将返回错误。如果需要，可以禁用预检。建议指定相同的承诺和预检承诺以避免混淆行为。

返回的签名是交易中的第一个签名，用于标识交易 ([交易 ID](/docs/references/terminology#transaction-id))。此标识符可以在提交之前轻松从交易数据中提取。

### !参数

#### !! transaction

!类型 string  
!必需

完全签名的交易，作为编码字符串。

#### !! config

!类型 object

包含以下字段的配置对象：

##### !! encoding

!类型 string  
!值 base58 base64  
!默认 base58

用于交易数据的编码。值：`base58`（_慢_，
**已弃用**），或 `base64`。

##### !! skipPreflight

!类型 bool

当 `true` 时，跳过预检交易检查。默认值：`false`。

##### !! preflightCommitment

!type string

用于预检的承诺级别。请参阅
[配置状态承诺](/docs/rpc/index.mdx#configuring-state-commitment)。默认值 `finalized`。

##### !! maxRetries

!type usize

RPC 节点重试将交易发送到领导者的最大次数。如果未提供此参数，RPC 节点将重试交易，直到交易被最终确认或区块哈希过期。

##### !! minContextSlot

!type number  
!optional

设置执行预检交易检查的最小 slot
