h1: getLargestAccounts RPC 方法
---

返回按 lamport 余额排序的 20 个最大账户（结果可能会缓存长达两小时）

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getLargestAccounts",
  "params": []
}
```

### !参数

#### !! 0

!类型 object

包含以下字段的配置对象：

##### !! commitment

!类型 string

commitment 描述了区块在某一时间点的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! filter

!类型 string  
!值 circulating nonCirculating

按账户类型筛选结果

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  "result": {
    "context": { "slot": 54 },
    "value": [
      {
        // !hover address
        "address": "99P8ZgtJYe1buSK8JXkvpLh8xPsCFuLYhz9hQFNw93WJ",
        // !hover lamports
        "lamports": 999974
      },
      {
        "address": "uPwWLo16MVehpyWqsLkK3Ka8nLowWvAHbBChqv2FZeL",
        "lamports": 42
      }
    ]
  },
  "id": 1
}
```

!类型 array

结果将是一个 RpcResponse
JSON 对象，其中 `value` 等于一个包含以下内容的对象数组：

##### !! address

!类型 string

账户的 Base-58 编码地址

##### !! lamports

!类型 u64

账户中的 lamport 数量

</APIMethod>
