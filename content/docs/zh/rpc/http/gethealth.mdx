h1: getHealth RPC 方法
---

返回节点的当前健康状态。健康的节点是指在 `HEALTH_CHECK_SLOT_DISTANCE` 个 slot 内与最新的集群确认 slot 保持同步的节点。

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getHealth"
}
```

### !参数

### !!结果 健康

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": "ok",
  "id": 1
}
```

!类型 字符串

如果节点健康："ok"

### !!结果 不健康

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:5) result
  "error": {
    "code": -32005,
    "message": "Node is unhealthy",
    "data": { "numSlotsBehind": 42 }
  },
  "id": 1
}
```

!类型 对象

如果节点不健康，将返回一个 JSON RPC 错误响应。错误响应的具体内容是
**不稳定的**，未来可能会发生变化。

</APIMethod>
