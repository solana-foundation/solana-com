h1: getProgramAccounts RPC 方法
---

返回由提供的程序 Pubkey 所拥有的所有账户

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getProgramAccounts",
  "params": [
    // !hover pubkey
    "4Nd1mBQtrMJVYVfKf2PJy9NZUZdTAsp7D4xWLs4gDB4T",
    // !hover(1:11) 1
    {
      // !hover(1:9) filters
      "filters": [
        { "dataSize": 17 },
        {
          "memcmp": {
            "offset": 4,
            "bytes": "3Mc6vR"
          }
        }
      ]
    }
  ]
}
```

### !参数

#### !! pubkey

!类型 string  
!必需

程序的 Pubkey，以 base-58 编码的字符串形式

#### !! 1

!类型 object

包含以下字段的配置对象：

##### !! commitment

!类型 string

承诺描述了在某一时间点区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! minContextSlot

!类型 number

请求可以被评估的最小 slot

##### !! withContext

!类型 bool

将结果包装在一个 RpcResponse JSON 对象中

##### !! encoding

!类型 string  
!值 jsonParsed base58 base64 base64+zstd  
!默认值 json

返回的账户数据的编码格式

- `base58` 速度较慢，且限制账户数据小于 129 字节。
- `base64` 将返回任何大小账户数据的 base64 编码数据。
- `base64+zstd` 使用
  [Zstandard](https://facebook.github.io/zstd/)
  压缩账户数据，并对结果进行 base64 编码。
- `jsonParsed` 编码尝试使用程序特定的状态解析器返回更具可读性和明确的账户状态数据。
- 如果请求了 `jsonParsed`，但找不到解析器，则字段会回退到 `base64` 编码，可通过 `data` 字段类型 `<string>` 检测到。

##### !! dataSlice

!类型 object

请求账户数据的一个切片。

- `length: <usize>` - 要返回的字节数
- `offset: <usize>` - 开始读取的字节偏移量

<Callout type="info">
  数据切片仅适用于
  `base58`、
  `base64` 或
  `base64+zstd` 编码。
</Callout>

##### !! filters

!类型 array

使用最多 4 个过滤对象来过滤结果。

<Callout type="info">
  返回的账户必须满足 **所有** 过滤条件才能包含在结果中。
</Callout>

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:12) result
  "result": [
    {
      // !hover pubkey
      "pubkey": "CxELquR1gPP8wHe33gZ4QxqGB3sZ9RSwsJ2KshVewkFY",
      // !hover(1:8) account
      "account": {
        // !hover data
        "data": "2R9jLfiAQ9bgdcw6h8s44439",
        // !hover executable
        "executable": false,
        // !hover lamports
        "lamports": 15298080,
        // !hover owner
        "owner": "4Nd1mBQtrMJVYVfKf2PJy9NZUZdTAsp7D4xWLs4gDB4T",
        // !hover rentEpoch
        "rentEpoch": 28,
        // !hover space
        "space": 42
      }
    }
  ],
  "id": 1
}
```

!类型 array

默认情况下，返回一个 JSON 对象数组。如果设置了 `withContext` 标志，数组将被包装在一个 RpcResponse
JSON 对象中。

每个对象包含：

##### !! pubkey

!类型 string

账户的 Pubkey，作为 base-58 编码的字符串

##### !! account

!类型 object

一个包含以下内容的 JSON 对象：

- `lamports: <u64>` - 分配给此账户的 lamports 数量，类型为 u64
- `owner: <string>` - 分配给此账户的程序的 base-58 编码 Pubkey
- `data: <[string,encoding]|object>` - 与账户关联的数据，可以是编码的二进制数据或 JSON 格式，取决于编码参数 `{<program>: <state>}`
- `executable: <bool>` - 一个布尔值，指示账户是否包含程序（并且是严格只读的）
- `rentEpoch: <u64>` - 此账户下一次需要支付租金的 epoch，类型为 u64
- `space: <u64>` - 账户的数据大小

</APIMethod>
