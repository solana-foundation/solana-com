h1: getBlockProduction RPC 方法
---

返回当前或前一个 epoch 的最近区块生产信息。

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getBlockProduction"
}
```

### !参数

#### !! 0

!类型 对象

配置对象

##### !! commitment

!类型 字符串

commitment 描述了在某一时间点区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! identity

!类型 字符串

仅返回此验证者身份（base-58 编码）的结果。

##### !! range

!类型 对象

返回区块生产的 slot 范围。如果未提供参数，则默认为当前 epoch。

- `firstSlot: <u64>` - 第一个 slot，返回区块生产信息（包含）
- （可选）`lastSlot: <u64>` - 最后一个 slot，返回区块生产信息（包含）。如果未提供参数，默认为最高 slot

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  "result": {
    "context": {
      "slot": 9887
    },
    // !hover(1:9) result
    "value": {
      // !hover(1:3) byIdentity
      "byIdentity": {
        "85iYT5RuzRTDgjyRa3cP8SYhM2j21fj7NhfJ3peu1DPr": [9888, 9886]
      },
      // !hover(1:4) range
      "range": {
        "firstSlot": 0,
        "lastSlot": 9887
      }
    }
  },
  "id": 1
}
```

!类型 对象

结果将是一个 RpcResponse JSON 对象，其值等于：

##### !! byIdentity

!类型 对象

一个以 base-58 编码字符串表示的验证者身份字典。值是一个包含两个元素的数组，分别是领导者 slot 的数量和生成的区块数量。

##### !! range

!类型 object

区块生产 slot 范围

- `firstSlot: <u64>` - 区块生产信息的第一个 slot（包含）
- `lastSlot: <u64>` - 区块生产信息的最后一个 slot（包含）

</APIMethod>
