请求向 Pubkey 空投 lamports

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "requestAirdrop",
  "params": [
    // !hover pubkey
    "83astBRguLMdt2h5U1Tpdq5tjFoJ6noeGwaY3mDLVcri",
    // !hover lamports
    1000000000
  ]
}
```

### !参数

#### !! pubkey

!类型 string  
!必填

接收 lamports 的账户的 Pubkey，作为 base-58 编码的字符串

#### !! lamports

!类型 u64  
!必填

要空投的 lamports 数量

#### !! config

!类型 object

包含以下字段的配置对象：

##### !! commitment

!类型 string

承诺描述了在该时间点区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": "5VERv8NMvzbJMEkV8xnrLkEaWRtSz9CosKDYjCJjBRnbJLgp8uirBgmQpjKhoR4tjF3ZpRzrFmBV6UjKdiSZkQUW",
  "id": 1
}
```

!类型 string

空投的交易签名，作为 base-58 编码的字符串
