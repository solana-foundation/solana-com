---
title: getRecentPrioritizationFees
hideTableOfContents: true
h1: getRecentPrioritizationFees RPC 方法
---

返回最近区块的优先费用列表。

<Callout type="info">
  当前，节点的优先费用缓存最多存储 150 个区块的数据。
</Callout>

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getRecentPrioritizationFees",
  "params": [
    // !hover address
    ["CxELquR1gPP8wHe33gZ4QxqGB3sZ9RSwsJ2KshVewkFY"]
  ]
}
```

### !params

#### !! address

!type array

一个账户地址数组（最多 128 个地址），以 base-58 编码的字符串形式表示

<Callout type="info">
  如果提供了此参数，响应将反映出锁定所有提供账户为可写状态的交易费用。
</Callout>

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:22) result
  "result": [
    {
      // !hover slot
      "slot": 348125,
      // !hover prioritizationFee
      "prioritizationFee": 0
    },
    {
      "slot": 348126,
      "prioritizationFee": 1000
    },
    {
      "slot": 348127,
      "prioritizationFee": 500
    },
    {
      "slot": 348128,
      "prioritizationFee": 0
    },
    {
      "slot": 348129,
      "prioritizationFee": 1234
    }
  ],
  "id": 1
}
```

!type array

一个包含优先费用对象的数组，包含以下内容：

##### !! slot

!type u64

观察到费用的 slot

##### !! prioritizationFee

!type u64

至少一个成功落地的交易所支付的每计算单元费用，以微 lamports（0.000001
lamports）为单位表示

</APIMethod>
