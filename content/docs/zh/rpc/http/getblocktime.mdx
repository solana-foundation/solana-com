h1: getBlockTime RPC 方法
---

返回区块的预计生成时间。

<Callout type="info">
  每个 validator 会定期将其 UTC 时间报告到分类账中，
  通过间歇性地为特定区块的投票添加时间戳。
  请求区块的时间是根据分类账中记录的一组最近区块的投票时间戳的
  权重平均值计算得出的。
</Callout>

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getBlockTime",
  "params": [
    // !hover slot number
    5
  ]
}
```

### !参数

#### !! slot 编号

!类型 u64  
!必需

区块编号，由 Slot 标识

### !!结果可用

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 1574721591,
  "id": 1
}
```

!类型 i64

预计生成时间，以 Unix 时间戳表示（自 Unix 纪元以来的秒数）

### !!结果不可用

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:4) result
  "error": {
    "code": -32004,
    "message": "Block not available for slot 150"
  },
  "id": 1
}
```

!类型 对象

错误对象

</APIMethod>
