h1: getEpochInfo RPC 方法
---

返回当前 epoch 的信息

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getEpochInfo"
}
```

### !params

#### !! 0

!type object

配置对象。

##### !! commitment

!type string

commitment 描述了在某一时刻区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! minContextSlot

!type number

请求可以评估的最小 slot。

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:8) result
  "result": {
    // !hover absoluteSlot
    "absoluteSlot": 166598,
    // !hover blockHeight
    "blockHeight": 166500,
    // !hover epoch
    "epoch": 27,
    // !hover slotIndex
    "slotIndex": 2790,
    // !hover slotsInEpoch
    "slotsInEpoch": 8192,
    // !hover transactionCount
    "transactionCount": 22661093
  },
  "id": 1
}
```

!type object

result 字段将是一个包含以下字段的对象：

##### !! absoluteSlot

!type u64

当前的 slot

##### !! blockHeight

!type u64

当前的区块高度

##### !! epoch

!类型 u64

当前 epoch

##### !! slotIndex

!类型 u64

当前 slot 相对于当前 epoch 开始的索引

##### !! slotsInEpoch

!类型 u64

此 epoch 中的 slot 数量

##### !! transactionCount

!类型 u64 | null

自创世以来无错误处理的交易总数

</APIMethod>
