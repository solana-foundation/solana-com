h1: getTokenAccountsByOwner RPC 方法
---

返回指定 Token 所有者的所有 SPL Token 账户。

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getTokenAccountsByOwner",
  "params": [
    // !hover pubkey
    "A1TMhSGzQxMr1TboBKtgixKz1sS6REASMxPo1qsyTSJd",
    // !hover(1:3) 1
    {
      // !hover programId
      "programId": "TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA"
    },
    // !hover(1:3) 2
    {
      // !hover encoding
      "encoding": "jsonParsed"
    }
  ]
}
```

### !参数

#### !! pubkey

!type string  
!required

要查询的账户所有者的公钥，使用 base-58 编码的字符串

#### !! 1

!type object  
!required

一个包含以下字段之一的 JSON 对象：

##### !! mint

!type string

特定 Token Mint 的公钥，用于限制账户范围，使用 base-58 编码的字符串

##### !! programId

!type string

拥有这些账户的 Token 程序的公钥，使用 base-58 编码的字符串

#### !! 2

!type object

包含以下字段的配置对象：

##### !! commitment

!type string

承诺值描述了区块在某一时间点的最终确定程度。详见[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! minContextSlot

!类型 number

请求可以评估的最小 slot

##### !! dataSlice

!类型 object

请求账户数据的切片。

- `length: <usize>` - 要返回的字节数
- `offset: <usize>` - 开始读取的字节偏移量

<Callout type="info">
  数据切片仅适用于
  `base58`、
  `base64` 或
  `base64+zstd` 编码。
</Callout>

##### !! encoding

!类型 string  
!值 base58 base64 base64+zstd jsonParsed

账户数据的编码格式

- `base58` 速度较慢，且限制为小于 129 字节的账户数据。
- `base64` 将返回任何大小账户数据的 base64 编码数据。
- `base64+zstd` 使用
  [Zstandard](https://facebook.github.io/zstd/)
  压缩账户数据，并对结果进行 base64 编码。
- `jsonParsed` 编码尝试使用特定于程序的状态解析器，以返回更易于理解和更明确的账户状态数据。
- 如果请求了 `jsonParsed`，但找不到解析器，则字段会回退到 `base64` 编码，可以通过 `data` 字段为类型 `string` 来检测。

!类型 array

包含以下内容的 JSON 对象数组：

##### !! pubkey

!类型 string

账户 Pubkey，作为 base-58 编码字符串

##### !! account

!类型 object

包含以下内容的 JSON 对象：

- `lamports: <u64>` - 分配给此账户的 lamports 数量，类型为 u64
- `owner: <string>` - 分配给此账户的程序的 base-58 编码 Pubkey
- `data: <object>` - 与账户关联的 Token 状态数据，可以是编码的二进制数据或 JSON 格式 `{<program>: <state>}`
- `executable: <bool>` - 布尔值，指示账户是否包含程序（并且是严格只读的）
- `rentEpoch: <u64>` - 此账户下一次需要支付租金的 epoch，类型为 u64
- `space: <u64>` - 账户的数据大小

当使用 `jsonParsed` 编码请求数据时，结构中可以预期会出现类似于
[Token Balances Structure](/docs/rpc/json-structures#token-balances)
的格式，适用于 `tokenAmount` 和 `delegatedAmount`，后者为可选对象。
