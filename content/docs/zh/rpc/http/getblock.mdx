h1: getBlock RPC 方法
---

返回账本中已确认区块的身份和交易信息。

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getBlock",
  "params": [
    // !hover slot number
    430,
    // !hover(1:6) 1
    {
      // !hover encoding
      "encoding": "json",
      // !hover maxSupportedTransactionVersion
      "maxSupportedTransactionVersion": 0,
      // !hover transactionDetails
      "transactionDetails": "full",
      // !hover rewards
      "rewards": false
    }
  ]
}
```

### !参数

#### !! slot 编号

!type u64  
!required

Slot 编号。

#### !! 1

!type object

配置对象。

##### !! commitment

!type string

承诺值描述了区块在某一时间点的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

- `processed` 不支持。

##### !! encoding

!type string  
!values json jsonParsed base58 base64  
!default json

每个返回交易的编码格式。详见 [解析响应](/docs/rpc#parsed-responses)。

- `jsonParsed` 尝试使用程序特定的指令解析器，以返回更易于理解和更明确的数据，`transaction.message.instructions` 列表中。
- 如果请求了 `jsonParsed` 但未找到解析器，则指令将回退到常规 JSON 编码 (`accounts`、`data` 和 `programIdIndex` 字段)。

##### !! transactionDetails

!type string  
!values full accounts signatures none  
!default full

返回的交易详情级别。

- 如果请求了 `accounts`，交易详情仅包括签名和每笔交易中的账户注释列表。
- 交易元数据仅限于：fee、err、pre_balances、post_balances、pre_token_balances 和 post_token_balances。

##### !! maxSupportedTransactionVersion

!type number

返回响应中最大交易版本。

- 如果请求的区块包含更高版本的交易，将返回错误。
- 如果未提供此参数，则仅返回传统交易，包含任何版本化交易的区块将触发错误。

##### !! 奖励

!type bool

是否填充奖励数组。如果未提供此参数，默认包含奖励。

!type object | null

如果指定的区块未确认，结果将为 `null`。否则，将返回一个包含以下内容的对象：

##### !! 区块哈希

!type string

此区块的区块哈希，作为 base-58 编码的字符串

##### !! 上一区块哈希

!type string

此区块父区块的区块哈希，作为 base-58 编码的字符串；如果由于账本清理无法获取父区块，此字段将返回 "11111111111111111111111111111111"

##### !! 父槽位

!type u64

此区块父区块的槽位索引

##### !! 交易

!type array

如果请求了“完整”交易详情，则存在；为包含以下内容的 JSON 对象数组：

- `transaction: <object|[string,encoding]>` -
  [交易](/docs/rpc/json-structures#transactions)
  对象，格式为 JSON 或编码的二进制数据，取决于编码参数。
- `meta: <object>` - 交易状态
  [元数据对象](/docs/rpc/json-structures#transaction-status-metadata)
  或 `null`。

##### !! 区块时间

!type i64 | null

预计生产时间，以 Unix 时间戳表示（自 Unix
epoch 起的秒数）。`null` 如果不可用。

##### !! blockHeight

!type u64 | null

此区块下方的区块数量。

</APIMethod>
