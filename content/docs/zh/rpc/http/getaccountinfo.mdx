```rs !!request title="Rust"
use solana_client::nonblocking::rpc_client::RpcClient;
use solana_sdk::{commitment_config::CommitmentConfig, pubkey::Pubkey};
use anyhow::Result;
use std::str::FromStr;

#[tokio::main]
async fn main() -> Result<()> {
    let client = RpcClient::new_with_commitment(
        String::from("https://api.devnet.solana.com"),
        CommitmentConfig::confirmed()
    );
    // !hover pubkey
    let pubkey = Pubkey::from_str("vines1vzrYbzLMRdu58ou5XTby4qAqVRLmqo36NKPTg")?;
    let account = client.get_account(&pubkey).await?;

    println!("{:#?}", account);

    Ok(())
}
```

### !参数

#### !! pubkey

!类型 string  
!必需

要查询账户的 Pubkey，作为 base-58 编码的字符串。

#### !! 1

!类型 object

配置对象。

##### !! commitment

!类型 string

commitment 描述了区块在某一时间点的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! encoding

!类型 string  
!值 base58 base64 base64+zstd jsonParsed

账户数据的编码格式。详见 [解析响应](/docs/rpc#parsed-responses)。

- `base58` 速度较慢，且限制为小于 129 字节的账户数据。
- `base64` 将返回任何大小账户数据的 base64 编码数据。
- `base64+zstd` 使用
  [Zstandard](https://facebook.github.io/zstd/)
  压缩账户数据，并对结果进行 base64 编码。
- `jsonParsed` 编码尝试使用特定程序的状态解析器，以返回更具可读性和更明确的账户状态数据。
- 如果请求了 `jsonParsed`，但找不到解析器，则字段会回退到 `base64` 编码，可通过 `data` 字段为类型 `string` 检测到。

##### !! dataSlice

!类型 object

请求账户数据的一个切片。

- `length: <usize>` - 要返回的字节数
- `offset: <usize>` - 开始读取的字节偏移量

<Callout type="info">
  数据切片仅适用于
  `base58`、
  `base64` 或
  `base64+zstd` 编码。
</Callout>

##### !! minContextSlot

!类型 string

请求可以被评估的最小 slot。

!类型 object | null

如果请求的账户不存在，结果将为 `null`。否则，将返回一个包含以下内容的对象：

##### !! lamports

!类型 u64

分配给此账户的 lamports 数量

#### !! owner

!类型 string

分配给此账户的程序的 base-58 编码 Pubkey

#### !! data

!类型 \[string,encoding\] | object

与账户关联的数据，可以是编码的二进制数据或 JSON 格式 `{<program>: <state>}` - 取决于编码参数

#### !! executable

!类型 bool

布尔值，指示账户是否包含程序（并且是严格只读的）

#### !! rentEpoch

!类型 u64

此账户下一次需要支付租金的 epoch，以 u64 表示

#### !! space

!类型 u64

账户的数据大小
