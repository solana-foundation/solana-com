h1: getSignatureStatuses RPC 方法
---

返回签名列表的状态。每个签名必须是一个
[txid](/docs/references/terminology#transaction-id)，即交易的第一个签名。

<Callout type="info">
  除非包含 `searchTransactionHistory`
  配置参数，否则此方法仅搜索最近的签名状态缓存，该缓存保留所有活动 slot
  的状态以及 `MAX_RECENT_BLOCKHASHES` 已根
  slot 的状态。
</Callout>

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getSignatureStatuses",
  "params": [
    // !hover(1:3) 0
    [
      "5VERv8NMvzbJMEkV8xnrLkEaWRtSz9CosKDYjCJjBRnbJLgp8uirBgmQpjKhoR4tjF3ZpRzrFmBV6UjKdiSZkQUW"
    ],
    // !hover(1:3) 1
    {
      // !hover searchTransactionHistory
      "searchTransactionHistory": true
    }
  ]
}
```

### !参数

#### !! 0

!类型 array  
!必需

一个包含交易签名的数组，签名以 base-58 编码的字符串形式表示（最多 256 个）

#### !! 1

!类型 object

包含以下字段的配置对象：

##### !! searchTransactionHistory

!类型 bool

如果 `true` -
Solana 节点将搜索其分类账缓存，以查找在最近状态缓存中未找到的任何签名

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  "result": {
    "context": {
      "slot": 82
    },
    // !hover(1:12) result
    "value": [
      {
        // !hover slot
        "slot": 48,
        // !hover confirmations
        "confirmations": null,
        // !hover err
        "err": null,
        // !hover status
        "status": {
          "Ok": null
        },
        // !hover confirmationStatus
        "confirmationStatus": "finalized"
      },
      null
    ]
  },
  "id": 1
}
```

!类型 array

一个由 `RpcResponse<object>` 组成的数组，包含 `null` 或包含以下字段的对象：

##### !! slot

!类型 u64

交易被处理的 slot

##### !! confirmations

!类型 usize | null

自签名确认以来的区块数量，`null` 如果已根植，并且已被集群的超级多数确认。

##### !! 错误

!type object | null

如果交易失败的错误，`null` 如果交易成功。详见
[TransactionError 定义](https://github.com/solana-labs/solana/blob/c0c60386544ec9a9ec7119229f37386d9f070523/sdk/src/transaction/error.rs#L13)。

##### !! 确认状态

!type string | null

交易的集群确认状态；可能是 `processed`、`confirmed` 或 `finalized`。详见
[Commitment](/docs/rpc/#configuring-state-commitment)
了解更多关于乐观确认的信息。

##### !! 状态

!type object

**已弃用** 交易状态

- `"Ok": <null>` - 交易成功
- `"Err": <ERR>` - 交易因 TransactionError 失败

</APIMethod>
