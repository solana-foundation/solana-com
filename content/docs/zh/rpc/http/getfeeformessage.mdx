h1: getFeeForMessage RPC 方法
---

获取网络对特定消息收取的费用

<Callout type="warn" title={"版本限制"}>
  此方法仅适用于 `solana-core`
  v1.9 或更新版本。 请使用 [getFees](/docs/rpc/deprecated/getfees) 适用于
  `solana-core` v1.8
  及以下版本。
</Callout>

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getFeeForMessage",
  "params": [
    // !hover message
    "AQABAgIAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAQAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAEBAQAA",
    // !hover(1:3) commitment
    {
      // !hover commitment
      "commitment": "processed"
    }
  ]
}
```

### !参数

#### !! message

!类型 string  
!必需

Base-64 编码的消息

#### !! 1

!类型 object

配置对象。

##### !! commitment

!类型 string

承诺描述了在特定时间点区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! minContextSlot

!类型 number

请求可以评估的最小 slot。

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  "result": {
    "context": { "slot": 5068 },
    // !hover result
    "value": 5000
  },
  "id": 1
}
```

!类型 u64 | null

指定区块哈希中与消息对应的费用

</APIMethod>
