h1: getVoteAccounts RPC 方法
---

返回当前银行中所有投票账户的账户信息及相关的质押信息。

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getVoteAccounts",
  "params": [
    // !hover(1:3) 0
    {
      // !hover votePubkey
      "votePubkey": "3ZT31jkAGhUaw8jsy4bTknwBMP8i4Eueh52By4zXcsVw"
    }
  ]
}
```

### !params

#### !! 0

!type object  
!optional

包含以下字段的配置对象：

##### !! commitment

!type string

承诺值描述了在某一时刻区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! votePubkey

!type string

仅返回此验证者投票地址（base-58 编码）的结果

##### !! keepUnstakedDelinquents

!type bool

不要过滤掉没有质押的失责验证者

##### !! delinquentSlotDistance

!type u64

指定验证者被视为失责所需的与最新区块的槽位差距。**注意：**
为了在生态系统产品之间保持一致，_**不**建议指定此参数。_

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:17) result
  "result": {
    "current": [
      {
        // !hover votePubkey
        "votePubkey": "3ZT31jkAGhUaw8jsy4bTknwBMP8i4Eueh52By4zXcsVw",
        // !hover nodePubkey
        "nodePubkey": "B97CCUW3AEZFGy6uUg6zUdnNYvnVq5VG8PUtb2HayTDD",
        // !hover activatedStake
        "activatedStake": 42,
        // !hover epochVoteAccount
        "epochVoteAccount": true,
        // !hover commission
        "commission": 0,
        // !hover lastVote
        "lastVote": 147,
        // !hover(1:4) epochCredits
        "epochCredits": [
          [1, 64, 0],
          [2, 192, 64]
        ]
      }
    ],
    "delinquent": []
  },
  "id": 1
}
```

!type object

结果字段将是一个 JSON 对象，包含 `current` 和 `delinquent` 账户，每个账户包含一个带有以下子字段的 JSON 对象数组：

##### !! votePubkey

!类型 string

投票账户地址，使用 base-58 编码的字符串

##### !! nodePubkey

!类型 string

验证者身份，使用 base-58 编码的字符串

##### !! activatedStake

!类型 u64

在本 epoch 中委托给此投票账户并处于激活状态的质押量（以 lamports 为单位）

##### !! epochVoteAccount

!类型 bool

此投票账户是否在本 epoch 中被质押

##### !! commission

!类型 number

投票账户应得奖励支付的百分比（0-100）

##### !! lastVote

!类型 u64

此投票账户最近投票的 slot

##### !! epochCredits

!类型 array

最近五个 epoch 的奖励历史记录，作为包含以下内容的数组的数组：`[epoch, credits, previousCredits]`

##### !! rootSlot

!类型 u64

此投票账户的当前根 slot

</APIMethod>
