h1: getInflationReward RPC 方法
---

返回指定 epoch 中一组地址的通胀/质押奖励

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getInflationReward",
  "params": [
    // !hover(1:4) 0
    [
      "6dmNQ5jwLeLk5REvio1JcMshcbvkYMwy26sJ8pbkvStu",
      "BGsqMegLpV6n6Ve146sSX2dTjUMj3M92HnU8BbNRMhF2"
    ],
    // !hover(1:3) 1
    {
      // !hover epoch
      "epoch": 2
    }
  ]
}
```

### !params

#### !! 0

!type array

一个包含地址的数组，地址以 base-58 编码的字符串表示

#### !! 1

!type object

包含以下字段的配置对象：

##### !! commitment

!type string

commitment 描述了某个时间点区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! epoch

!type u64

奖励发生的 epoch。如果未指定，将使用前一个 epoch

##### !! minContextSlot

!type number

请求可以评估的最小 slot

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:9) result
  "result": [
    {
      // !hover epoch
      "epoch": 2,
      // !hover effectiveSlot
      "effectiveSlot": 224,
      // !hover amount
      "amount": 2500,
      // !hover postBalance
      "postBalance": 499999442500
    },
    null
  ],
  "id": 1
}
```

!type array

result 字段将是一个包含以下内容的 JSON 对象数组：

##### !! epoch

!类型 u64

奖励发生的 epoch

##### !! effectiveSlot

!类型 u64

奖励生效的 slot

##### !! amount

!类型 u64

以 lamports 为单位的奖励金额

##### !! postBalance

!类型 u64

账户奖励后的余额（以 lamports 为单位）

##### !! commission

!类型 u8 | undefined

奖励记入时的投票账户佣金

</APIMethod>
