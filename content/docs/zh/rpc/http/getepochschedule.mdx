返回此集群创世配置中的 epoch 调度信息

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getEpochSchedule"
}
```

### !参数

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:7) result
  "result": {
    // !hover firstNormalEpoch
    "firstNormalEpoch": 8,
    // !hover firstNormalSlot
    "firstNormalSlot": 8160,
    // !hover leaderScheduleSlotOffset
    "leaderScheduleSlotOffset": 8192,
    // !hover slotsPerEpoch
    "slotsPerEpoch": 8192,
    // !hover warmup
    "warmup": true
  },
  "id": 1
}
```

!类型 对象

结果字段将是一个包含以下字段的对象：

##### !! slotsPerEpoch

!类型 u64

每个 epoch 中的最大 slot 数量

##### !! leaderScheduleSlotOffset

!类型 u64

在 epoch 开始之前，用于计算该 epoch 的领导者调度的 slot 数量

##### !! warmup

!类型 bool

epoch 是否从短开始并逐渐增长

##### !! firstNormalEpoch

!类型 u64

第一个正常长度的 epoch，`log2(slotsPerEpoch) - log2(MINIMUM_SLOTS_PER_EPOCH)`

##### !! firstNormalSlot

!类型 u64

epoch 中的最小 slot 数量，`MINIMUM_SLOTS_PER_EPOCH * (2.pow(firstNormalEpoch) - 1)`

</APIMethod>
