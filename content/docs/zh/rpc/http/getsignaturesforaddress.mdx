h1: getSignaturesForAddress RPC 方法
---

返回包含指定地址的已确认交易的签名，这些地址位于 `accountKeys` 列表中。从提供的签名或最近的已确认区块开始，按时间倒序返回签名。

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getSignaturesForAddress",
  "params": [
    // !hover address
    "Vote111111111111111111111111111111111111111",
    // !hover(1:6) 1
    {
      // !hover limit
      "limit": 1
    }
  ]
}
```

### !参数

#### !! address

!类型 字符串  
!必需

账户地址，使用 base-58 编码的字符串

#### !! 1

!类型 对象

配置对象，包含以下字段：

##### !! commitment

!类型 字符串

承诺值描述了区块在该时间点的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! minContextSlot

!类型 数字

请求可以被评估的最小 slot

##### !! limit

!类型 数字

返回的最大交易签名数量（介于 1 到 1,000 之间）。

默认值：`1000`

##### !! before

!类型 字符串

从此交易签名开始向后搜索。如果未提供，则从最高的最大已确认区块的顶部开始搜索。

##### !! 直到

!类型 string

在达到限制之前，如果找到，则搜索直到此交易签名

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover(1:10) result
  "result": [
    {
      // !hover signature
      "signature": "5h6xBEauJ3PK6SWCZ1PGjBvj8vDdWG3KpwATGy1ARAXFSDwt8GFXM7W5Ncn16wmqokgpiKRLuS83KUxyZyv2sUYv",
      // !hover slot
      "slot": 114,
      // !hover err
      "err": null,
      // !hover memo
      "memo": null,
      // !hover blockTime
      "blockTime": null,
      // !hover confirmationStatus
      "confirmationStatus": "finalized"
    }
  ],
  "id": 1
}
```

!类型 array

一个交易签名信息对象的数组，从**最新**到**最旧**的交易排序，包含：

##### !! 签名

!类型 string

以 base-58 编码的交易签名字符串

##### !! slot

!类型 u64

包含该交易的区块的 slot

##### !! 错误

!类型 object | null

如果交易失败则显示错误信息，如果交易成功则为 null。更多信息请参见
[TransactionError 定义](https://github.com/solana-labs/solana/blob/c0c60386544ec9a9ec7119229f37386d9f070523/sdk/src/transaction/error.rs#L13)。

##### !! 备注

!类型 string | null

与交易相关的备注，如果没有备注则为 null

##### !! 区块时间

!类型 i64 | null

估算的生产时间，以 Unix 时间戳（自 Unix 纪元以来的秒数）表示交易被处理的时间。如果不可用则为 null。

##### !! 确认状态

!类型 string | null

交易的集群确认状态；可能为 `processed`、`confirmed` 或 `finalized`。有关乐观确认的更多信息，请参见
[Commitment](/docs/rpc/#configuring-state-commitment)。

</APIMethod>
