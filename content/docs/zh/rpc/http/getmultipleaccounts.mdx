h1: getMultipleAccounts RPC 方法
---

返回一组 Pubkeys 的账户信息。

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getMultipleAccounts",
  "params": [
    // !hover(1:4) 0
    [
      "vines1vzrYbzLMRdu58ou5XTby4qAqVRLmqo36NKPTg",
      "4fYNw3dojWmQ4dXtSGE9epjRGy9pFSx62YypT7avPYvA"
    ],
    // !hover(1:3) 1
    {
      // !hover encoding
      "encoding": "base58"
    }
  ]
}
```

### !参数

#### !! 0

!类型 数组  
!必填

一个要查询的 Pubkeys 数组，作为 base-58 编码的字符串（最多 100 个）

#### !! 1

!类型 对象

包含以下字段的配置对象：

##### !! commitment

!类型 字符串

commitment 描述了在某一时间点区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! minContextSlot

!类型 数字

请求可以被评估的最小 slot

##### !! dataSlice

!类型 对象

请求账户数据的一个切片。

- `length: <usize>` - 要返回的字节数
- `offset: <usize>` - 开始读取的字节偏移量

<Callout type="info">
  数据切片仅适用于
  `base58`、
  `base64` 或
  `base64+zstd` 编码。
</Callout>

##### !! encoding

!类型 字符串  
!值 jsonParsed base58 base64 base64+zstd  
!默认值 base64

返回账户数据的编码格式

- `base58` 速度较慢，且限制为小于 129 字节的账户数据。
- `base64` 将返回任意大小账户数据的 base64 编码数据。
- `base64+zstd` 使用
  [Zstandard](https://facebook.github.io/zstd/)
  压缩账户数据，并对结果进行 base64 编码。
- `jsonParsed` 编码尝试使用特定程序的状态解析器，以返回更具可读性和更明确的账户状态数据。
- 如果请求了 `jsonParsed` 但未找到解析器，则字段将回退到 `base64` 编码，可通过 `data` 字段为类型 `<string>` 检测到。

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  "result": {
    "context": { "apiVersion": "2.0.15", "slot": 341197247 },
    // !hover(1:18) result
    "value": [
      {
        // !hover data
        "data": ["", "base58"],
        // !hover executable
        "executable": false,
        // !hover lamports
        "lamports": 88849814690250,
        // !hover owner
        "owner": "11111111111111111111111111111111",
        // !hover rentEpoch
        "rentEpoch": 18446744073709551615,
        // !hover space
        "space": 0
      },
      {
        "data": ["", "base58"],
        "executable": false,
        "lamports": 998763433,

        "rentEpoch": 18446744073709551615,
        "space": 0
      }
    ]
  },
  "id": 1

```

!类型 数组

结果将是一个包含以下内容的数组：

- `null` - 如果该 Pubkey 的账户不存在，或者
- 包含以下字段的账户对象：

##### !! lamports

!类型 u64

分配给此账户的 lamports 数量

##### !! owner

!类型 字符串

分配给此账户的程序的 Base-58 编码 Pubkey

##### !! data

!类型 [字符串, 编码] | 对象

与账户相关的数据，可以是编码的二进制数据或 JSON 格式 `{<program>: <state>}` - 取决于编码参数

##### !! executable

!类型 布尔值

布尔值，指示账户是否包含程序（并且严格为只读）

##### !! rentEpoch

!类型 u64

此账户下一次需要支付租金的 epoch

##### !! space

!类型 u64

账户的数据大小

</APIMethod>
