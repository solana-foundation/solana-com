h1: getTransaction RPC 方法
---

返回已确认交易的详细信息

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getTransaction",
  "params": [
    // !hover signature
    "2nBhEBYYvfaAe16UMNqRHre4YNSskvuYgx3M6E4JP1oDYvZEJHvoPzyUidNgNX5r9sTyN1J9UxtbCXy2rqYcuyuv",
    "json"
  ]
}
```

### !参数

#### !! signature

!类型 string  
!必填

交易签名，作为 base-58 编码的字符串

#### !! 1

!类型 object

包含以下字段的配置对象：

##### !! commitment

!类型 string

承诺描述了在某一时刻区块的最终确定程度。详见
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

- `processed` 不支持。

##### !! maxSupportedTransactionVersion

!类型 number

设置响应中返回的最大交易版本。如果请求的交易版本更高，将返回错误。如果省略此参数，则仅返回传统交易，任何版本化交易将提示错误。

##### !! encoding

!类型 string  
!值 json jsonParsed base64 base58  
!默认值 json

返回交易的编码方式

- `jsonParsed` 编码尝试使用程序特定的状态解析器，以返回更具可读性和更明确的数据，位于 `transaction.message.instructions` 列表中。
- 如果请求了 `jsonParsed` 但找不到解析器，指令将回退到常规 JSON 编码（`accounts`、`data` 和 `programIdIndex` 字段）。

!type object | null

如果未找到交易或交易未确认，则返回 `null`，否则返回包含以下内容的对象：

##### !! slot

!type u64

此交易被处理的 slot

##### !! blockTime

!type i64 | null

估算的生产时间，以 Unix 时间戳（自 Unix 纪元以来的秒数）表示交易被处理的时间。如果不可用，则为 null

##### !! meta

!type object | null

交易状态 [元数据对象](/docs/rpc/json-structures#transaction-status-metadata)
或 `null`。

##### !! transaction

!type object | [string,encoding]

[交易](/docs/rpc/json-structures#transactions)
对象，可以是 JSON 格式或根据编码参数的二进制数据。

##### !! version

!type "legacy" | number | undefined

交易版本。如果请求参数中未设置 `maxSupportedTransactionVersion`，则为 undefined。

</APIMethod>
