h1: blockSubscribe RPC 方法
---

订阅以在每次有新区块 `confirmed` 或 `finalized` 时接收通知。

<Callout type="warn" title={"不稳定的方法"}>
  此订阅被认为是**不稳定的**，仅在验证器以
  `--rpc-pubsub-enable-block-subscription`
  标志启动时可用。 此订阅的格式可能在未来发生变化。
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    // !hover(1:3) 0
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    // !hover(1:6) 1
    {
      // !hover commitment
      "commitment": "confirmed",
      // !hover encoding
      "encoding": "base64",
      // !hover showRewards
      "showRewards": true,
      // !hover transactionDetails
      "transactionDetails": "full"
    }
  ]
}
```

### !参数

#### !! 0

!类型 string | object  
!必需

用于根据账户类型的过滤条件接收日志结果；当前支持：

- `all` - 包括区块中的所有交易
- 一个包含以下字段的 JSON 对象：
  - `mentionsAccountOrProgram: <string>` - 仅返回提到提供的公钥（以 base-58 编码的字符串）的交易。如果在某个区块中没有提到，则不会发送通知。

#### !! 1

!类型 object

包含以下字段的配置对象：

##### !! commitment

!类型 string  
!默认值 finalized

承诺描述了区块在某一时间点的最终确定程度。请参阅[配置状态承诺](/docs/rpc#configuring-state-commitment)。

- `processed` 不支持。

##### !! encoding

!类型 string  
!值 json jsonParsed base58 base64  
!默认值 json

每个返回交易的编码格式

- `jsonParsed` 尝试使用程序特定的指令解析器，以在 `transaction.message.instructions` 列表中返回更易于理解和更明确的数据。
- 如果请求了 `jsonParsed` 但未找到解析器，则指令将回退到常规 JSON 编码（`accounts`、`data` 和 `programIdIndex` 字段）。

##### !! transactionDetails

!类型 string  
!值 full accounts signatures none  
!默认值 full

返回的交易详情级别

- 如果请求了 `accounts`，交易详情仅包括签名和每笔交易中的账户注释列表。
- 交易元数据仅限于：fee、err、pre_balances、post_balances、pre_token_balances 和 post_token_balances。

##### !! maxSupportedTransactionVersion

!type number

响应中返回的最大交易版本。

- 如果请求的区块包含更高版本的交易，将返回错误。
- 如果省略此参数，则仅返回传统交易，且包含任何版本化交易的区块将触发错误。

##### !! showRewards

!type bool

是否填充 `rewards` 数组。如果未提供参数，默认包含奖励。

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

订阅 ID（用于取消订阅）

</APIMethod>

#### 通知格式：

通知将是一个包含以下字段的对象：

- `slot: <u64>` - 对应的 slot。
- `err: <object|null>` - 如果发布通知时出错，则为错误信息，否则为 null。
- `block: <object|null>` - 一个区块对象，如
  [getBlock](/docs/rpc/http/getblock) RPC HTTP 方法中所示。
