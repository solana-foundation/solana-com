h1: programSubscribe RPC 方法
---

订阅一个程序，以便在由该程序拥有的账户的 lamports 或数据发生更改时接收通知。

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "programSubscribe",
  "params": [
    // !hover 0
    "11111111111111111111111111111111",
    // !hover(1:4) 1
    {
      // !hover encoding
      "encoding": "base64",
      // !hover filters
      "filters": [{ "dataSize": 80 }]
    }
  ]
}
```

### !参数

#### !! 0

!类型 string  
!必需

以 base-58 编码的字符串形式表示的 `program_id` 的公钥。

#### !! 1

!类型 object

包含以下字段的配置对象：

##### !! commitment

!类型 string

commitment 描述了在某一时刻区块的最终确定程度。请参阅[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! filters

!类型 array

使用各种过滤对象筛选结果。请参阅[过滤](/docs/rpc#filter-criteria)。

<Callout type="info">
  返回的账户必须满足**所有**过滤条件才能包含在返回结果中。
</Callout>

##### !! encoding

!类型 string  
!值 base58 base64 base64+zstd jsonParsed

账户数据的编码格式

- `base58` 较慢。
- `jsonParsed` 编码尝试使用特定于程序的状态解析器，以返回更具可读性和更明确的账户状态数据。
- 如果请求了 `jsonParsed` 但找不到解析器，则该字段会回退到 `base64` 编码，可以通过 `data` 字段为类型 `string` 检测到。

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 24040,
  "id": 1
}
```

!类型 整数

订阅 ID（取消订阅时需要）

</APIMethod>

#### 通知格式

通知格式是一个<b>单一的</b>程序账户对象，如
[getProgramAccounts](/docs/rpc/http/getprogramaccounts) RPC HTTP 方法中所示。

Base58 编码：

```json
{
  "jsonrpc": "2.0",
  "method": "programNotification",
  "params": {
    "result": {
      "context": {
        "slot": 5208469
      },
      "value": {
        "pubkey": "H4vnBqifaSACnKa7acsxstsY1iV1bvJNxsCY7enrd1hq",
        "account": {
          "data": [
            "11116bv5nS2h3y12kD1yUKeMZvGcKLSjQgX6BeV7u1FrjeJcKfsHPXHRDEHrBesJhZyqnnq9qJeUuF7WHxiuLuL5twc38w2TXNLxnDbjmuR",
            "base58"
          ],
          "executable": false,
          "lamports": 33594,
          "owner": "11111111111111111111111111111111",
          "rentEpoch": 636,
          "space": 80
        }
      }
    },
    "subscription": 24040
  }
}
```
