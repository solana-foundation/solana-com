h1: accountSubscribe RPC 方法
---

订阅一个账户，以便在给定账户公钥的 lamports 或数据发生变化时接收通知。

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "accountSubscribe",
  "params": [
    // !hover 0
    "CM78CPUeXjn8o3yroDHxUtKsZZgoy4GPkPPXfouKNH12",
    // !hover(1:4) 1
    {
      // !hover encoding
      "encoding": "jsonParsed",
      // !hover commitment
      "commitment": "finalized"
    }
  ]
}
```

### !参数

#### !! 0

!类型 string  
!必需

账户公钥，使用 base-58 编码的字符串

#### !! 1

!类型 object

包含以下字段的配置对象：

##### !! commitment

!类型 string

承诺描述了在某一时间点区块的最终确定程度。请参阅
[配置状态承诺](/docs/rpc#configuring-state-commitment)。

##### !! encoding

!类型 string  
!值 base58 base64 base64+zstd jsonParsed

账户数据的编码格式

- `base58` 较慢。
- `jsonParsed` 编码尝试使用特定程序的状态解析器，以返回更易于理解和更明确的账户状态数据。
- 如果请求了 `jsonParsed` 但找不到解析器，则字段会回退到二进制编码，可通过 `data` 字段的类型 `string` 检测到。

### !!结果

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 23784,
  "id": 1
}
```

!类型 number

订阅 ID（用于取消订阅）

</APIMethod>

#### 通知格式：

HTTP 方法中所示的格式相同。
