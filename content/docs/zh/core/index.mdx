要在Solana上构建应用，理解几个Solana开发特有的关键概念至关重要。本节涵盖了开始在Solana上构建时需要了解的核心概念，包括账户、交易、程序等。

## Solana账户模型

在Solana上，所有数据都存储在被称为"账户"的结构中。Solana区块链上的数据组织方式类似于键值存储，其中数据库中的每个条目都被称为"账户"。

在[这里](/docs/core/accounts)了解更多关于账户的信息。

## 交易和指令

在Solana上，我们发送交易来与网络交互。交易包含一个或多个指令，每个指令代表一个待处理的特定操作。指令的执行逻辑存储在部署到Solana网络的程序中，每个程序定义了自己的指令集。

在[这里](/docs/core/transactions)了解更多关于交易和[指令](/docs/core/transactions#instruction)的信息。

## Solana上的费用

Solana区块链有几种不同类型的费用和成本，这些费用在使用网络时产生。它们可以分为以下几种具体类型：

* 交易费用 - 验证者处理交易/指令的费用
* 优先级费用 - 可选费用，用于提升交易处理顺序
* rent - 保留的余额，用于在链上保存数据

在[这里](/docs/core/fees)了解更多关于Solana上的费用的信息。

## Solana上的程序

在Solana上，"智能合约"被称为程序。每个程序都存储在链上账户中，包含定义特定指令的可执行代码。这些指令代表程序的功能，可以通过向网络发送交易来调用。

在[这里](/docs/core/programs)了解更多关于Solana上的程序的信息。

## Program Derived Address

Program Derived Addresses (PDAs) 为 Solana 上的开发者提供了两个主要用例：

* **确定性账户地址**：PDAs 提供了一种机制，通过组合可选的"seeds"（预定义输入）和特定的程序 ID 来确定性地派生地址。
* **启用程序签名**：Solana 运行时使程序能够为从其程序 ID 派生的 PDAs "签名"。

你可以将 PDAs 视为一种方式，通过预定义的输入集（例如字符串、数字和其他账户地址）在链上创建类似哈希映射的结构。

在[这里](/docs/core/pda)了解更多关于 Program Derived Address 的信息。

## Cross Program Invocation

Cross Program Invocation
(CPI) 是指一个程序调用另一个程序的指令。这种机制允许 Solana 程序的可组合性。

你可以将指令视为程序向网络公开的 API 端点，而 CPI 则是一个 API 内部调用另一个 API。

在[这里](/docs/core/cpi)了解更多关于 Cross Program Invocation 的信息。

## Solana 上的代币

代币是代表各种资产所有权的数字资产。代币化使财产权的数字化成为可能，作为管理同质和非同质资产的基础组件。

* 同质化代币代表同类型和价值的可互换和可分割资产（例如 USDC）。
* 非同质化代币（NFT）代表不可分割资产的所有权（例如艺术品）。

在[这里](/docs/core/tokens)了解更多关于 Solana 上的代币信息。

## 集群和端点

Solana 区块链有几个不同的验证者组，称为[集群](/docs/references/clusters)。每个集群服务于不同的目的，并包含专用节点来满足
[JSON-RPC](/docs/rpc/) 请求。

Solana 网络上有三个主要集群，具有以下公共端点：

* 主网 - `https://api.mainnet-beta.solana.com`（生产环境）
* 开发网 - `https://api.devnet.solana.com`（开发者实验）
* 测试网 - `https://api.testnet.solana.com`（validator 测试）

在[这里](/docs/references/clusters)了解更多关于集群和端点的信息。
