要在 Solana 上进行开发，理解几个独特的 Solana 开发关键概念是至关重要的。
本节涵盖了您在开始在 Solana 上构建时需要理解的核心概念，包括账户、交易、程序等。

## Solana 账户模型

在 Solana 上，所有数据都存储在被称为“账户”的地方。数据在 Solana 区块链上的组织方式类似于键值存储，
数据库中的每个条目称为“账户”。

在这里了解更多关于 [账户](/docs/core/accounts) 的信息。

## 交易和指令

在 Solana 上，我们发送交易以与网络进行交互。交易包括一个或多个指令，每个指令代表一个特定的操作。
指令的执行逻辑存储在部署到 Solana 网络的程序中，每个程序定义自己的一组指令。

在这里了解更多关于 [交易](/docs/core/transactions) 和 [指令](/docs/core/transactions#instruction) 的信息。

## Solana 上的费用

Solana 区块链有几种不同类型的费用和成本，这些费用是在使用网络时产生的。
这些费用可以分为几种特定类型：

- 交易费用 - 让验证者处理交易/指令的费用
- 优先费用 - 提高交易处理顺序的可选费用
- 租金 - 保留余额以保持数据存储在链上

在这里了解更多关于 [Solana 上的费用](/docs/core/fees) 的信息。

## Solana 上的程序

在 Solana 上，“智能合约”被称为程序。每个程序存储在链上账户中，包含定义特定指令的可执行代码。
这些指令代表程序的功能，可以通过向网络发送交易来调用。

在这里了解更多关于 [Solana 上的程序](/docs/core/programs) 的信息。

## 程序派生地址

程序派生地址（PDA）为 Solana 上的开发者提供了两个主要用例：

- **确定性账户地址**：PDA 提供了一种机制，通过组合可选的“种子”（预定义输入）和特定的程序 ID 来确定性地派生地址。
- **启用程序签名**：Solana 运行时允许程序为从其程序 ID 派生的 PDA “签名”。

您可以将 PDA 视为一种从预定义输入集（例如字符串、数字和其他账户地址）在链上创建类似哈希映射结构的方法。

在这里了解更多关于 [程序派生地址](/docs/core/pda) 的信息。

## 跨程序调用

跨程序调用（CPI）是指一个程序调用另一个程序的指令。这种机制允许 Solana 程序的组合性。

您可以将指令视为程序向网络暴露的 API 端点，而 CPI 则是一个 API 内部调用另一个 API。

在这里了解更多关于 [跨程序调用](/docs/core/cpi) 的信息。

## Solana 上的代币

代币是代表对多种资产类别的所有权的数字资产。代币化使财产权的数字化成为可能，
是管理可替代和不可替代资产的基本组成部分。

- 可替代代币代表同类型和同价值的可互换和可分割资产（例如 USDC）。
- 不可替代代币（NFT）代表不可分割资产的所有权（例如艺术品）。

在这里了解更多关于 [Solana 上的代币](/docs/core/tokens) 的信息。

## 集群和端点

Solana 区块链有几个不同的验证者组，称为 [集群](/docs/references/clusters)。每个集群服务于不同的目的，
并包含专用节点以满足 [JSON-RPC](/docs/rpc/) 请求。

在 Solana 网络上有三个主要集群，具有以下公共端点：

- 主网 - `https://api.mainnet-beta.solana.com`（生产环境）
- 开发网 - `https://api.devnet.solana.com`（开发者实验）
- 测试网 - `https://api.testnet.solana.com`（验证者测试）

在这里了解更多关于 [集群和端点](/docs/references/clusters) 的信息。
