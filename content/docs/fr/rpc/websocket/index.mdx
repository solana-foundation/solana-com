Après vous être connecté au websocket RPC PubSub à `ws://<ADDRESS>/` :

- Soumettez des demandes d'abonnement au websocket en utilisant les méthodes
  ci-dessous
- Plusieurs abonnements peuvent être actifs simultanément
- De nombreux abonnements acceptent le paramètre optionnel
  [`commitment`](/docs/rpc/#configuring-state-commitment), définissant le niveau
  de finalisation qu'un changement doit atteindre pour déclencher une
  notification. Pour les abonnements, si l'engagement n'est pas spécifié, la
  valeur par défaut est `finalized`.

## Point de terminaison WebSocket RPC PubSub

Port par défaut : `8900`

- ws://localhost:8900
- http://192.168.1.88:8900
