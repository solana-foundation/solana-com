يقدم مستودع
[أمثلة برامج سولانا](https://github.com/solana-developers/program-examples) على
GitHub العديد من المجلدات الفرعية، كل منها يحتوي على أمثلة برمجية لمساعدة
المطورين على التعلم والتجربة في تطوير بلوكتشين سولانا.

يمكنك العثور على الأمثلة في `solana-developers/program-examples` مع ملفات README
التي تشرح لك كيفية تشغيل الأمثلة المختلفة. معظم الأمثلة مستقلة بذاتها ومتوفرة
بلغة Rust الأصلية (أي بدون إطار عمل)
و[Anchor](https://www.anchor-lang.com/docs/installation).

داخل المستودع ستجد المجلدات الفرعية التالية، كل منها يحتوي على برامج متنوعة
كأمثلة:

- [أساسيات](#basics)
- [الرموز](#tokens)
- [Token 2022 (ملحقات الرمز)](#token-2022-token-extensions)

## أساسيات

باستخدام مكتبات Rust الأصلية. تم تصميم هذه الأمثلة لمساعدة المطورين على فهم
المفاهيم الأساسية لبرمجة سولانا.

## الرموز

تستخدم معظم الرموز على سولانا معيار رمز مكتبة برامج سولانا (SPL). هنا يمكنك
العثور على العديد من الأمثلة حول كيفية سك الرموز ونقلها وحرقها وحتى كيفية
التفاعل معها في البرامج.

| اسم المثال                                                                                               | الوصف                                                                   | اللغة          |
| -------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- | -------------- |
| [إنشاء رمز](https://github.com/solana-developers/program-examples/tree/main/tokens/create-token)         | كيفية إنشاء رمز وإضافة بيانات وصفية من metaplex إليه.                   | Anchor, Native |
| [صانع NFT](https://github.com/solana-developers/program-examples/tree/main/tokens/nft-minter)            | سك كمية واحدة فقط من الرمز ثم إزالة سلطة السك.                          | Anchor, Native |
| [سلطة سك PDA](https://github.com/solana-developers/program-examples/tree/main/tokens/pda-mint-authority) | يوضح لك كيفية تغيير سلطة السك لعملية سك، لسك الرموز من داخل برنامج.     | Anchor, Native |
| [صانع رمز SPL](https://github.com/solana-developers/program-examples/tree/main/tokens/spl-token-minter)  | يشرح كيفية استخدام حسابات الرموز المرتبطة لتتمكن من تتبع حسابات الرموز. | Anchor, Native |
| [مبادلة الرموز](https://github.com/solana-developers/program-examples/tree/main/tokens/token-swap)       | مثال شامل يوضح لك كيفية بناء مجمع AMM (صانع السوق الآلي) لرموز SPL.     | Anchor         |
| [نقل الرموز](https://github.com/solana-developers/program-examples/tree/main/tokens/transfer-tokens)     | يوضح كيفية نقل رمز SPL باستخدام CPIs في برنامج الرمز.                   | Anchor, Native |
| [Token-2022](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022)          | انظر Token 2022 (Token Extensions).                                     | Anchor, Native |

## Token 2022 (Token Extensions)

Token 2022 هو معيار جديد للرموز على سولانا. وهو أكثر مرونة ويتيح لك إضافة 16
امتدادًا مختلفًا إلى عملية سك الرمز لإضافة المزيد من الوظائف إليه. يمكن العثور
على قائمة كاملة بالامتدادات في
[دليل البدء](/developers/guides/token-extensions/getting-started)

| اسم المثال                                                                                                                               | الوصف                                                                                            | اللغة          |
| ---------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ | -------------- |
| [أساسيات](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022/basics/anchor)                               | كيفية إنشاء رمز وسكه ونقله.                                                                      | Anchor         |
| [حالة الحساب الافتراضية](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022/default-account-state)        | يتيح لك هذا الامتداد إنشاء حسابات رمز بحالة معينة، على سبيل المثال مجمدة.                        | Anchor, Native |
| [سلطة إغلاق السك](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022/mint-close-authority)                | مع برنامج الرمز القديم، لم يكن من الممكن إغلاق عملية سك. الآن أصبح ذلك ممكنًا.                   | Anchor, Native |
| [امتدادات متعددة](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022/multiple-extensions)                 | يوضح لك كيفية إضافة امتدادات متعددة إلى عملية سك واحدة                                           | Native         |
| [مؤشر البيانات الوصفية للـ NFT](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022/nft-meta-data-pointer) | من الممكن استخدام امتداد البيانات الوصفية لإنشاء NFTs وإضافة بيانات وصفية ديناميكية على السلسلة. | Anchor         |
| [غير قابل للتحويل](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022/non-transferable)                   | مفيد على سبيل المثال للإنجازات أو برامج الإحالة أو أي رموز مرتبطة بالروح.                        | Anchor, Native |
| [رسوم التحويل](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022/transfer-fee)                           | كل عملية تحويل للرموز تحتفظ ببعض الرموز في حساب الرمز والتي يمكن جمعها بعد ذلك.                  | Anchor, Native |
| [خطاف التحويل](https://github.com/solana-developers/program-examples/tree/main/tokens/token-2022/transfer-hook)                          | أربعة أمثلة لإضافة وظائف إضافية إلى الرمز الخاص بك باستخدام CPI من برنامج الرمز إلى برنامجك.     | Anchor         |
