h1: Metoda RPC blockSubscribe
---

Subskrybuj, aby otrzymywać powiadomienia za każdym razem, gdy nowy blok zostanie
`confirmed` lub `finalized`.

<Callout type="warn" title={"Niestabilna metoda"}>
  Ta subskrypcja jest uznawana za **niestabilną** i jest dostępna tylko wtedy,
  gdy validator został uruchomiony z flagą
  `--rpc-pubsub-enable-block-subscription`. Format tej subskrypcji może ulec
  zmianie w przyszłości.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    // !hover(1:3) 0
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    // !hover(1:6) 1
    {
      // !hover commitment
      "commitment": "confirmed",
      // !hover encoding
      "encoding": "base64",
      // !hover transactionDetails
      "transactionDetails": "full",
      // !hover maxSupportedTransactionVersion
      "maxSupportedTransactionVersion": 0,
      // !hover showRewards
      "showRewards": true
    }
  ]
}
```

### !params

#### !! 0

!type string | object  
!required

kryteria filtrowania logów w celu otrzymywania wyników według typu konta;
obecnie obsługiwane:

- `all` - uwzględnij wszystkie transakcje w bloku
- Obiekt JSON z następującym polem:
  - `mentionsAccountOrProgram: <string>` - zwróć tylko transakcje, które
    wspominają podany klucz publiczny (jako ciąg zakodowany w base-58). Jeśli w
    danym bloku nie ma wspomnień, powiadomienie nie zostanie wysłane.

#### !! 1

!type object

Obiekt konfiguracyjny zawierający następujące pola:

##### !! commitment

!type string !values confirmed finalized !default finalized

Commitment opisuje, jak bardzo blok jest sfinalizowany w danym momencie. Zobacz
[Konfigurowanie zaangażowania stanu](/docs/rpc#configuring-state-commitment).

- `processed` nie jest obsługiwany.

##### !! encoding

!type string  
!values json jsonParsed base58 base64  
!default json

format kodowania dla każdej zwróconej transakcji

- `jsonParsed` próbuje użyć parserów instrukcji specyficznych dla programu, aby
  zwrócić bardziej czytelne i precyzyjne dane w liście
  `transaction.message.instructions`.
- Jeśli `jsonParsed` jest żądane, ale parser nie może zostać znaleziony,
  instrukcja wraca do regularnego kodowania JSON (`accounts`, `data` i
  `programIdIndex` pola).

##### !! transactionDetails

!type string  
!values full accounts signatures none  
!default full

poziom szczegółowości transakcji do zwrócenia

- Jeśli `accounts` są żądane, szczegóły transakcji zawierają tylko podpisy oraz
  adnotowaną listę kont w każdej transakcji.
- Metadane transakcji są ograniczone tylko do: fee, err, pre_balances,
  post_balances, pre_token_balances i post_token_balances.

##### !! maxSupportedTransactionVersion

!type number !values 0 !default 0

Obecnie jedyną prawidłową wartością dla tego parametru jest `0`. Ustawienie go
na `0` pozwala na pobieranie wszystkich transakcji, w tym zarówno
wersjonowanych, jak i starszych transakcji.

Ten parametr określa maksymalną wersję transakcji, która zostanie zwrócona w
odpowiedzi. Jeśli zażądasz transakcji z wyższą wersją niż ta wartość, zostanie
zwrócony błąd. Jeśli pominiesz ten parametr, zostaną zwrócone tylko starsze
transakcje — każda wersjonowana transakcja spowoduje błąd.

##### !! showRewards

!type bool

czy wypełnić tablicę `rewards`. Jeśli parametr nie zostanie podany, domyślnie
zostaną uwzględnione nagrody.

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

id subskrypcji (potrzebne do anulowania subskrypcji)

</APIMethod>

#### Format powiadomienia:

Powiadomienie będzie obiektem z następującymi polami:

- `slot: <u64>` - Odpowiadający slot.
- `err: <object|null>` - Błąd, jeśli coś poszło nie tak podczas publikowania
  powiadomienia, w przeciwnym razie null.
- `block: <object|null>` - Obiekt bloku, jak w metodzie RPC HTTP
  [getBlock](/docs/rpc/http/getblock).
