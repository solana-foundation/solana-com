Retorna detalhes da transação para uma transação confirmada

<APIMethod>

```rs !!request title="Rust"
use anyhow::Result;
use solana_client::{nonblocking::rpc_client::RpcClient, rpc_config::RpcTransactionConfig};
use solana_sdk::{commitment_config::CommitmentConfig, signature::Signature};
use solana_transaction_status_client_types::UiTransactionEncoding;
use std::str::FromStr;

#[tokio::main]
async fn main() -> Result<()> {
    let client = RpcClient::new_with_commitment(
        String::from("https://api.devnet.solana.com"),
        CommitmentConfig::confirmed(),
    );

    // !hover(1:3) signature
    let tx_sig = Signature::from_str(
        "5zSQuTcWsPy2cVAshBXWuJJXLwMD1GbgMpz3iq4xgwiV1s6mxYRbYb7qBiRGZd1xvDcYhQQRBKoNcnW8eKtcyZWg",
    )?;

    // !hover(1:5) 1
    let config = RpcTransactionConfig {
        // !hover commitment
        commitment: CommitmentConfig::finalized().into(),
        // !hover encoding
        encoding: UiTransactionEncoding::Base64.into(),
        // !hover maxSupportedTransactionVersion
        max_supported_transaction_version: Some(0),
    };

    let transaction = client.get_transaction_with_config(&tx_sig, config).await?;

    println!("{:#?}", transaction);

    Ok(())
}
```

### !params

#### !! signature

!type string !required

Assinatura da transação, como string codificada em base-58

#### !! config

!type object

Objeto de configuração contendo os seguintes campos:

##### !! commitment

!type string !values confirmed finalized !default finalized

O commitment descreve quão finalizado está um bloco naquele momento. Veja
[Configurando State Commitment](/docs/rpc#configuring-state-commitment).

- `processed` não é suportado.

##### !! maxSupportedTransactionVersion

!type number !values 0 !default 0

Atualmente, o único valor válido para este parâmetro é `0`. Configurá-lo como
`0` permite buscar todas as transações, incluindo tanto as transações
versionadas quanto as legadas.

Este parâmetro determina a versão máxima de transação que será retornada na
resposta. Se você solicitar uma transação com uma versão superior a este valor,
um erro será retornado. Se você omitir este parâmetro, apenas as transações
legadas serão retornadas—qualquer transação versionada resultará em um erro.

##### !! encoding

!type string !values json jsonParsed base64 base58 !default json

Codificação para a Transaction retornada

- A codificação `jsonParsed` tenta usar analisadores de estado específicos do
  programa para retornar dados mais legíveis e explícitos na lista
  `transaction.message.instructions`.
- Se `jsonParsed` for solicitado, mas um analisador não puder ser encontrado, a
  instrução volta para a codificação JSON regular (campos `accounts`, `data` e
  `programIdIndex`).

!type object | null

Retorna `null` se a transação não for encontrada ou não estiver confirmada, caso
contrário, retorna um objeto contendo:

##### !! blockTime

!type i64 | null

Tempo estimado de produção, como timestamp Unix (segundos desde a época Unix) de
quando a transação foi processada. null se não estiver disponível

##### !! meta

!type object | null

Status da transação
[objeto de metadados](/docs/rpc/json-structures#transaction-status-metadata) ou
`null`.

##### !! slot

!type u64

O slot em que esta transação foi processada

##### !! transaction

!type object | [string,encoding]

Objeto [Transaction](/docs/rpc/json-structures#transactions), seja em formato
JSON ou dados binários codificados, dependendo do parâmetro de codificação

##### !! version

!type "legacy" | number | undefined

Versão da transação. Indefinido se `maxSupportedTransactionVersion` não estiver
definido nos parâmetros da solicitação.
