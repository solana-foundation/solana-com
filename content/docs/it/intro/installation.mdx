Questa sezione copre i passaggi per configurare il tuo ambiente locale per lo
sviluppo su Solana.

## Installazione rapida

Su Mac e Linux, esegui questo singolo comando per installare tutte le
dipendenze.

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

<Callout type="warn">
  Utenti Windows: dovete prima installare WSL (vedi [Installa
  dipendenze](#install-dependencies)). Poi eseguite il comando precedente nel
  terminale Ubuntu (Linux).
</Callout>

Dopo l'installazione, dovresti vedere un output simile al seguente:

```
Installed Versions:
Rust: rustc 1.86.0 (05f9846f8 2025-03-31)
Solana CLI: solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
Anchor CLI: anchor-cli 0.31.1
Node.js: v23.11.0
Yarn: 1.22.1
```

<Callout>

Se il comando di installazione rapida non funziona, fai riferimento alla sezione
[Installa dipendenze](#install-dependencies) qui sotto per le istruzioni su come
installare ogni dipendenza individualmente.

Se il comando di installazione rapida viene eseguito con successo, passa alle
sezioni [Solana CLI Basics](#solana-cli-basics) e
[Anchor CLI Basics](#anchor-cli-basics) qui sotto.

</Callout>

## Installa dipendenze

Le istruzioni qui sotto ti guidano attraverso l'installazione di ogni dipendenza
individualmente.

- Gli utenti Windows devono prima installare WSL (Windows subsystem for Linux) e
  poi installare le dipendenze specificate nella sezione Linux qui sotto.
- Gli utenti Linux dovrebbero prima installare le dipendenze specificate nella
  sezione Linux qui sotto.
- Gli utenti Mac dovrebbero iniziare con le istruzioni di installazione di Rust
  qui sotto.

<Accordions>
<Accordion title="Windows Subsystem for Linux (WSL)">

Per sviluppare programmi Solana su Windows **devi usare
[WSL](https://learn.microsoft.com/en-us/windows/wsl/install)** (Windows
subsystem for Linux). Installa tutte le altre dipendenze attraverso il terminale
Linux.

Dopo aver installato WSL, installa le dipendenze specificate nella sezione Linux
qui sotto prima di procedere all'installazione di Rust, CLI di Solana e CLI di
Anchor.

Per installare WSL, esegui il seguente comando in Windows PowerShell:

```terminal
$ wsl --install
```

Il processo di installazione ti chiederà di creare un account utente
predefinito.

![Installazione WSL](/assets/docs/intro/installation/wsl-install.png)

Per impostazione predefinita, WSL installa Ubuntu. Puoi aprire un terminale
Linux cercando "Ubuntu" nella barra di ricerca.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Se il tuo terminale Ubuntu appare come nell'immagine qui sotto, potresti
riscontrare un problema dove `ctrl + v` (scorciatoia da tastiera per incollare)
non funziona nel terminale.

![Terminale Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Se riscontri questo problema, apri Windows Terminal cercando "Terminal" nella
barra di ricerca.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Successivamente, chiudi Windows Terminal e riapri un terminale Linux cercando
Ubuntu di nuovo. Il terminale dovrebbe ora apparire come nell'immagine qui
sotto, dove `ctrl + v` (scorciatoia da tastiera per incollare) funziona.

![Terminale Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Se utilizzi VS Code,
l'[estensione WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial) ti
permette di usare WSL e VS Code insieme.

![Configurazione WSL in VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Dovresti quindi vedere quanto segue nella barra di stato di VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Una volta configurato WSL, installa tutte le altre dipendenze attraverso il
terminale Ubuntu (Linux).

Per installare le dipendenze richieste da Solana, prima prova a eseguire il
comando di [installazione rapida](#quick-installation) nel terminale Ubuntu
(Linux).

<Callout>

Se il comando di installazione rapida non funziona, installa le dipendenze
specificate nella sezione Linux qui sotto prima di procedere con i singoli
passaggi per l'installazione di Rust, Solana CLI e Anchor CLI.

Se il comando di installazione rapida viene eseguito con successo, passa alle
sezioni [Nozioni di base della CLI di Solana](#solana-cli-basics) e
[Nozioni di base della CLI di Anchor](#anchor-cli-basics) qui sotto.

</Callout>

</Accordion>
<Accordion title="Linux">

<Tabs items={[".deb", ".rpm"]}>
  <Tab value=".deb" >
  Se utilizzi Debian o alternative derivate da Debian come Ubuntu o Mint, segui i seguenti passaggi.

Aggiorna gli elenchi dei pacchetti per ottenere le versioni più recenti dei
pacchetti eseguendo il comando.

```terminal
$ sudo apt-get update
```

Successivamente, installa le seguenti dipendenze:

```terminal
$ sudo apt-get install -y \
    build-essential \
    pkg-config \
    libudev-dev llvm libclang-dev \
    protobuf-compiler libssl-dev
```

Se riscontri il seguente errore durante l'installazione di `protobuf-compiler`,
assicurati di eseguire prima _shell`sudo apt-get update`_:

```
Package protobuf-compiler is not available, but is referred to by another package.
This may mean that the package is missing, has been obsoleted, or
is only available from another source
```

  </Tab>

    <Tab value=".rpm" >
    Per le derivate RHEL come Fedora o Red Hat Linux, segui i seguenti passaggi.

Aggiorna gli elenchi dei pacchetti per ottenere le versioni più recenti dei
pacchetti eseguendo il comando.

```terminal
$ sudo dnf upgrade --refresh
```

Successivamente, installa le seguenti dipendenze:

```terminal
$ sudo dnf group install c-development development-tools \
&& sudo dnf install pkg-config openssl-devel
```

  </Tab>

### Installa Rust

Gli sviluppatori creano programmi Solana utilizzando il
[linguaggio di programmazione Rust](https://www.rust-lang.org/).

Installa Rust con [rustup](https://www.rust-lang.org/tools/install).

Esegui il seguente comando per installare Rust:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Dovresti vedere il seguente messaggio dopo il completamento dell'installazione:

<Accordions>
<Accordion title="Messaggio di installazione Rust riuscita">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

Esegui il seguente comando per ricaricare la tua variabile di ambiente PATH per
includere la directory bin di Cargo:

```terminal
$ . "$HOME/.cargo/env"
```

Per verificare che l'installazione sia riuscita, controlla la versione di Rust:

```terminal
$ rustc --version
```

Dovresti vedere un output simile al seguente:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>

### Installare la CLI di Solana

La CLI di Solana fornisce tutti gli strumenti necessari per creare e distribuire
programmi Solana.

Installa la suite di strumenti CLI di Solana utilizzando il comando di
installazione ufficiale:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

Puoi sostituire `stable` con il tag di rilascio corrispondente alla versione del
software della release desiderata (ad esempio `v2.0.3`), o utilizzare uno dei
tre nomi simbolici dei canali: `stable`, `beta`, o `edge`.

Per una prima installazione della CLI di Solana, potresti vedere il seguente
messaggio che ti chiede di aggiungere una variabile d'ambiente `PATH`:

```
Close and reopen your terminal to apply the PATH changes or run the following in your existing shell:

export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

<Tabs groupId="language" items={["Linux", "Mac"]}>
<Tab value="Linux">

Se stai utilizzando **Linux** o **WSL**, devi aggiungere il binario della CLI di
Solana al tuo `PATH` in modo che il comando sia disponibile nel tuo terminale.

Prima, esegui il seguente comando per verificare quale shell stai utilizzando:

```terminal
$ echo $SHELL
```

- Se l'output contiene `/bash`, usa `.bashrc`.
- Se l'output contiene `/zsh`, usa `.zshrc`.

A seconda della tua shell, esegui il comando appropriato.

Per Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Per Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

Poi esegui il seguente comando per aggiornare la sessione del terminale o
riavvia il terminale.

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

</Tab>
<Tab value="Mac">

Se utilizzi un Mac con `zsh`, l'esecuzione del comando predefinito `export PATH`
registrato dall'installazione non persiste una volta chiuso il terminale.

Invece, puoi aggiungere il PATH al tuo file di configurazione della shell
eseguendo il seguente comando:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

Quindi esegui il seguente comando per aggiornare la sessione del terminale o
riavvia il terminale.

```terminal
$ source ~/.zshrc
```

</Tab>
</Tabs>

Per verificare che l'installazione sia riuscita, controlla la versione della CLI
di Solana:

```terminal
$ solana --version
```

Dovresti vedere un output simile al seguente:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

Puoi visualizzare tutte le versioni disponibili nel
[repository Github di Agave](https://github.com/anza-xyz/agave/releases).

<Callout>

Agave è il client validator di [Anza](https://www.anza.xyz/), precedentemente
noto come client validator di Solana Labs.

</Callout>

Per aggiornare successivamente la CLI di Solana all'ultima versione, puoi
utilizzare il seguente comando:

```terminal
$ agave-install update
```

</Step>

### Installare la CLI di Anchor

[Anchor](https://www.anchor-lang.com/) è un framework per lo sviluppo di
programmi Solana. Il framework Anchor sfrutta le macro di Rust per semplificare
il processo di scrittura dei programmi Solana.

Puoi installare la CLI di Anchor e gli strumenti in due modi:

1. Anchor Version Manager (AVM) - Metodo di installazione consigliato
2. Senza AVM - Installazione diretta da GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

L'Anchor version manager (AVM) ti permette di installare e gestire diverse
versioni di Anchor sul tuo sistema e aggiornare facilmente le versioni di Anchor
in futuro.

Installa AVM con il seguente comando:

```terminal
$ cargo install --git https://github.com/coral-xyz/anchor avm --force
```

Conferma che AVM sia stato installato correttamente:

```terminal
$ avm --version
```

Installa l'ultima versione di Anchor CLI utilizzando AVM:

```terminal
$ avm install latest
$ avm use latest
```

Puoi installare una versione specifica di Anchor CLI specificando il numero di
versione:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="info">

Non dimenticare di eseguire il comando _shell`avm use`_ per dichiarare la
versione di Anchor CLI da eseguire sul tuo sistema.

- Se hai installato la versione `latest`, esegui _shell`avm use latest`_.
- Se hai installato la versione `0.30.1`, esegui _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

Installa una versione specifica di Anchor CLI con il seguente comando:

```terminal
$ cargo install --git https://github.com/coral-xyz/anchor --tag v0.30.1 anchor-cli
```

</Tab>
</Tabs>

Per verificare che l'installazione sia riuscita, controlla la versione di Anchor
CLI:

```terminal
$ anchor --version
```

Dovresti vedere un output simile al seguente:

```
anchor-cli 0.31.1
```

<Callout type="warn">

Quando installi Anchor CLI su Linux o WSL, potresti incontrare questo errore:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Se vedi questo messaggio di errore:

1. Installa le dipendenze elencate nella sezione
   [Linux](/docs/intro/installation#install-dependencies) all'inizio di questa
   pagina.
2. Riprova a installare Anchor CLI.

</Callout>

#### Node.js e Yarn

Il file di test predefinito del progetto Anchor (TypeScript) creato con il
comando _shell`anchor init`_ richiede Node.js e Yarn. (Il template di test Rust
è disponibile usando _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Installazione di Node">

Installa node usando
[Node Version Manager (nvm)](https://github.com/nvm-sh/nvm).

Installa nvm usando il seguente comando:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

Riavvia il terminale e conferma che il comando nvm funzioni correttamente:

```terminal
$ command -v nvm
```

Successivamente, usa `nvm` per installare node:

```terminal
$ nvm install node
```

Per verificare che l'installazione sia riuscita, controlla la versione di Node:

```terminal
$ node --version
```

Dovresti vedere un output simile al seguente:

```
v23.7.0
```

</Accordion>
<Accordion title="Installazione di Yarn">

Installa Yarn:

```terminal
$ npm install --global yarn
```

Per verificare che l'installazione sia riuscita, controlla la versione di Yarn:

```terminal
$ yarn --version
```

Dovresti vedere il seguente output:

```
1.22.1
```

</Accordion>
</Accordions>

<Callout type="warn">

Quando esegui `anchor build`, se riscontri i seguenti errori:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Prova queste soluzioni:

1. Forza l'installazione usando il seguente comando:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Se la soluzione precedente non funziona, cancella la cache di Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Puoi risolvere questo problema cambiando il campo version del file `Cargo.lock` da 4 a 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Vedi [questo issue](https://github.com/coral-xyz/anchor/issues/3392) per
maggiori informazioni.

</Accordion>

</Accordions>

Dopo aver applicato la soluzione precedente, prova a eseguire nuovamente
_shell`anchor build`_.

</Callout>

<Callout type="warn">

Quando esegui _shell`anchor test`_ dopo aver creato un nuovo progetto Anchor su
Linux o WSL, potresti incontrare i seguenti errori se non hai Node.js o Yarn
installati:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

## Nozioni di base sulla CLI Solana

Questa sezione illustra alcuni comandi comuni della CLI di Solana per aiutarti a
iniziare.

<Steps>
<Step>

### Configurazione di Solana

Per vedere la tua configurazione attuale:

```terminal
$ solana config get
```

Dovresti vedere un output simile al seguente:

```
Config File: /Users/test/.config/solana/cli/config.yml
RPC URL: https://api.mainnet-beta.solana.com
WebSocket URL: wss://api.mainnet-beta.solana.com/ (computed)
Keypair Path: /Users/test/.config/solana/id.json
Commitment: confirmed
```

L'URL RPC e l'URL Websocket specificano il cluster Solana a cui la CLI invia le
richieste.

Puoi aggiornare il cluster della CLI Solana utilizzando i seguenti comandi:

```terminal
$ solana config set --url mainnet-beta
$ solana config set --url devnet
$ solana config set --url localhost
$ solana config set --url testnet
```

Puoi anche utilizzare le seguenti opzioni abbreviate:

```terminal
$ solana config set -um    # For mainnet-beta
$ solana config set -ud    # For devnet
$ solana config set -ul    # For localhost
$ solana config set -ut    # For testnet
```

Il percorso del keypair indica il portafoglio Solana predefinito (keypair)
utilizzato dalla CLI di Solana per pagare le commissioni di transazione e
distribuire i programmi. Per impostazione predefinita, questo file è memorizzato
in `~/.config/solana/id.json`.

</Step>
<Step>

### Creazione del portafoglio

Per inviare transazioni utilizzando la CLI di Solana, hai bisogno di un
portafoglio Solana finanziato con SOL.

Per generare un keypair nel percorso predefinito, esegui il seguente comando:

```terminal
$ solana-keygen new
```

Dovresti vedere un output simile al seguente:

```
Generating a new keypair

For added security, enter a BIP39 passphrase

NOTE! This passphrase improves security of the recovery seed phrase NOT the
keypair file itself, which is stored as insecure plain text

BIP39 Passphrase (empty for none):

Wrote new keypair to /Users/test/.config/solana/id.json
===========================================================================
pubkey: 8dBTPrjnkXyuQK3KDt9wrZBfizEZijmmUQXVHpFbVwGT
===========================================================================
Save this seed phrase and your BIP39 passphrase to recover your new keypair:
cream bleak tortoise ocean nasty game gift forget fancy salon mimic amazing
===========================================================================
```

<Callout type="info">

Se hai già un portafoglio salvato nel file system nella posizione predefinita,
questo comando non lo sovrascrive a meno che non forzi esplicitamente la
sovrascrittura utilizzando il flag `--force`.

</Callout>

Per visualizzare l'indirizzo del tuo portafoglio (chiave pubblica), esegui:

```terminal
$ solana address
```

</Step>
<Step>

### Airdrop SOL

Richiedi un airdrop di SOL nel tuo wallet per pagare le transazioni e i
deployment dei programmi.

Imposta il tuo cluster su devnet:

```terminal
$ solana config set -ud
```

Poi richiedi un airdrop di SOL devnet:

```terminal
$ solana airdrop 2
```

<Callout>

Gli airdrop su devnet limitano le richieste a 5 SOL per volta. Se raggiungi i
limiti di frequenza o incontri errori, prova a utilizzare il
[Web Faucet](https://faucet.solana.com) invece.

</Callout>

Per verificare il saldo SOL del tuo wallet, esegui il seguente comando:

```terminal
$ solana balance
```

</Step>
<Step>

### Esegui un validator locale

La CLI di Solana include un
[test validator](https://docs.anza.xyz/cli/examples/test-validator) integrato
per lo sviluppo locale.

In un terminale separato, esegui il seguente comando per avviare un validator
locale:

```terminal
$ solana-test-validator
```

<Callout>

Ricordati anche di aggiornare la tua CLI per utilizzare localhost prima di
eseguire i comandi della CLI di Solana:

```terminal
$ solana config set -ul
```

</Callout>

</Step>
</Steps>

## Nozioni di base della CLI di Anchor

Questa sezione illustra alcuni comandi comuni della CLI di Anchor per aiutarti a
iniziare. Per maggiori informazioni sulla CLI di Anchor, consulta la
[documentazione di Anchor](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Inizializza il progetto

Per creare un nuovo progetto Anchor, esegui il seguente comando:

```terminal
$ anchor init <project-name>
```

Per esempio, per creare un progetto chiamato `my-project`, esegui:

```terminal
$ anchor init my-project
```

Questo comando crea una nuova directory con il nome del progetto e inizializza
un nuovo progetto Anchor con un programma Rust di base e un template di test in
TypeScript.

Naviga nella directory del progetto:

```terminal
$ cd <project-name>
```

Consulta la
[struttura dei file](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
del progetto Anchor.

</Step>

<Step>
### Compila il programma

Per compilare il tuo progetto, esegui il seguente comando:

```terminal
$ anchor build
```

Puoi trovare il programma compilato nella directory `/target/deploy`.

</Step>
<Step>

### Distribuisci il programma

Per distribuire il tuo progetto, esegui il seguente comando:

```terminal
$ anchor deploy
```

Questo comando distribuisce il tuo programma sul `cluster` specificato nel file
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Testa il programma

Per testare il tuo progetto, esegui il seguente comando:

```terminal
$ anchor test
```

<Callout>

Questo comando compila, distribuisce ed esegue i test per il tuo progetto.

Quando si utilizza `localnet` come `cluster` in `Anchor.toml`, Anchor avvia
automaticamente un validator locale, distribuisce il tuo programma, esegue i
test e poi arresta il validator.

</Callout>

</Step>
</Steps>
