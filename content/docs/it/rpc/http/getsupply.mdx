h1: Metodo RPC getSupply
---

Restituisce informazioni sulla fornitura attuale.

<APIMethod>

```jsonc !!request curl
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getSupply",
  "params": [
    // !hover(1:3) 0
    {
      // !hover commitment
      "commitment": "finalized"
    }
  ]
}
```

```ts !!request title="Kit"
import { createSolanaRpc } from "@solana/kit";

const rpc_url = "https://api.devnet.solana.com";
const rpc = createSolanaRpc(rpc_url);

let supply = await rpc.getSupply().send();

console.log(supply);
```

```ts !!request title="web3.js"
import { Connection, clusterApiUrl } from "@solana/web3.js";

const connection = new Connection(clusterApiUrl("devnet"), "confirmed");

let supply = await connection.getSupply();

console.log(supply);
```

### !params

#### !! 0

!type object

Oggetto di configurazione contenente i seguenti campi:

##### !! commitment

!type string

Il commitment descrive quanto è finalizzato un blocco in quel momento. Vedi
[Configurazione del commitment di stato](/docs/rpc#configuring-state-commitment).

##### !! excludeNonCirculatingAccountsList

!type bool

Esclude l'elenco degli account non circolanti dalla risposta

!type object

Il risultato sarà un oggetto JSON contenente:

##### !! circulating

!type u64

Fornitura circolante in lamport

##### !! nonCirculating

!type u64

Fornitura non circolante in lamport

##### !! nonCirculatingAccounts

!type array

Un array di indirizzi di account non circolanti, come stringhe. Se
`excludeNonCirculatingAccountsList` è abilitato, l'array restituito sarà vuoto.

##### !! total

!type u64

Fornitura totale in lamport

</APIMethod>
