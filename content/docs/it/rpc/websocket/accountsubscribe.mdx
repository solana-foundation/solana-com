h1: Metodo RPC accountSubscribe
---

Sottoscrivi un account per ricevere notifiche quando i lamport o i dati di una
chiave pubblica di un account specifico cambiano

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "accountSubscribe",
  "params": [
    // !hover 0
    "CM78CPUeXjn8o3yroDHxUtKsZZgoy4GPkPPXfouKNH12",
    // !hover(1:4) 1
    {
      // !hover encoding
      "encoding": "jsonParsed",
      // !hover commitment
      "commitment": "finalized"
    }
  ]
}
```

### !params

#### !! 0

!type string  
!required

Pubkey dell'account, come stringa codificata in base-58

#### !! 1

!type object

Oggetto di configurazione contenente i seguenti campi:

##### !! commitment

!type string

Il commitment descrive quanto è finalizzato un blocco in quel momento. Vedi
[Configurazione del commitment di stato](/docs/rpc#configuring-state-commitment).

##### !! encoding

!type string  
!values base58 base64 base64+zstd jsonParsed

Formato di codifica per i dati dell'account

- `base58` è lento.
- La codifica `jsonParsed` tenta di utilizzare parser di stato specifici del
  programma per restituire dati di stato dell'account più leggibili ed espliciti
- Se viene richiesto `jsonParsed` ma non viene trovato un parser, il campo torna
  alla codifica binaria, rilevabile quando il campo `data` è di tipo `string`.

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 23784,
  "id": 1
}
```

!type number

Id di sottoscrizione (necessario per annullare la sottoscrizione)

</APIMethod>

#### Formato di notifica:

Il formato della notifica è lo stesso visto nel metodo RPC HTTP
[getAccountInfo](/docs/rpc/http/getaccountinfo).
