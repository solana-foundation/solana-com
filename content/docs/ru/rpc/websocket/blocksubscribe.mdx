h1: Метод RPC blockSubscribe
---

Подпишитесь, чтобы получать уведомления каждый раз, когда новый блок `confirmed`
или `finalized`.

<Callout type="warn" title={"Нестабильный метод"}>
  Эта подписка считается **нестабильной** и доступна только в том случае, если
  валидатор был запущен с флагом `--rpc-pubsub-enable-block-subscription`.
  Формат этой подписки может измениться в будущем.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    // !hover(1:3) 0
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    // !hover(1:6) 1
    {
      // !hover commitment
      "commitment": "confirmed",
      // !hover encoding
      "encoding": "base64",
      // !hover transactionDetails
      "transactionDetails": "full",
      // !hover maxSupportedTransactionVersion
      "maxSupportedTransactionVersion": 0,
      // !hover showRewards
      "showRewards": true
    }
  ]
}
```

### !params

#### !! 0

!type string | object  
!required

критерии фильтрации для получения результатов по типу аккаунта; в настоящее
время поддерживаются:

- `all` - включить все транзакции в блоке
- JSON-объект со следующим полем:
  - `mentionsAccountOrProgram: <string>` - возвращать только транзакции, которые
    упоминают предоставленный публичный ключ (в виде строки, закодированной в
    base-58). Если в данном блоке нет упоминаний, уведомление отправлено не
    будет.

#### !! 1

!type object

Объект конфигурации, содержащий следующие поля:

##### !! commitment

!type string !values confirmed finalized !default finalized

Commitment описывает, насколько завершён блок в данный момент времени. См.
[Настройка состояния завершённости](/docs/rpc#configuring-state-commitment).

- `processed` не поддерживается.

##### !! encoding

!type string  
!values json jsonParsed base58 base64  
!default json

формат кодирования для каждой возвращаемой транзакции

- `jsonParsed` пытается использовать парсеры инструкций, специфичных для
  программы, чтобы возвращать более читаемые и явные данные в списке
  `transaction.message.instructions`.
- Если запрошен `jsonParsed`, но парсер не найден, инструкция возвращается к
  обычному JSON-кодированию (`accounts`, `data` и `programIdIndex` поля).

##### !! transactionDetails

!type string  
!values full accounts signatures none  
!default full

уровень детализации транзакции для возврата

- Если запрашиваются `accounts`, детали транзакции включают только подписи и
  аннотированный список аккаунтов в каждой транзакции.
- Метаданные транзакции ограничиваются только следующими параметрами: fee, err,
  pre_balances, post_balances, pre_token_balances и post_token_balances.

##### !! maxSupportedTransactionVersion

!type number !values 0 !default 0

В настоящее время единственное допустимое значение для этого параметра — `0`.
Установка этого значения на `0` позволяет вам получать все транзакции, включая
как версионные, так и устаревшие транзакции.

Этот параметр определяет максимальную версию транзакции, которая будет
возвращена в ответе. Если вы запросите транзакцию с версией выше этого значения,
будет возвращена ошибка. Если вы не укажете этот параметр, будут возвращены
только устаревшие транзакции — любая версионная транзакция приведет к ошибке.

##### !! showRewards

!type bool

указывает, заполнять ли массив `rewards`. Если параметр не указан, по умолчанию
включаются вознаграждения.

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

id подписки (необходим для отмены подписки)

</APIMethod>

#### Формат уведомления:

Уведомление будет объектом со следующими полями:

- `slot: <u64>` - Соответствующий слот.
- `err: <object|null>` - Ошибка, если что-то пошло не так при публикации
  уведомления, иначе null.
- `block: <object|null>` - Объект блока, как указано в
  [getBlock](/docs/rpc/http/getblock) методе RPC HTTP.
