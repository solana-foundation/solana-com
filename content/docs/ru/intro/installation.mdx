В этом разделе описаны шаги по настройке локальной среды для разработки на
Solana.

## Быстрая установка

На Mac и Linux выполните эту единственную команду для установки всех
зависимостей.

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

<Callout type="warn">
  Пользователи Windows: сначала вы должны установить WSL (см. [Установка
  зависимостей](#install-dependencies)). Затем выполните предыдущую команду в
  терминале Ubuntu (Linux).
</Callout>

После установки вы должны увидеть вывод, подобный следующему:

```
Installed Versions:
Rust: rustc 1.86.0 (05f9846f8 2025-03-31)
Solana CLI: solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
Anchor CLI: anchor-cli 0.31.1
Node.js: v23.11.0
Yarn: 1.22.1
```

<Callout>

Если команда быстрой установки не работает, обратитесь к разделу
[Установка зависимостей](#install-dependencies) ниже для получения инструкций по
установке каждой зависимости по отдельности.

Если команда быстрой установки выполняется успешно, перейдите к разделам
[Основы Solana CLI](#solana-cli-basics) и
[Основы Anchor CLI](#anchor-cli-basics) ниже.

</Callout>

## Установка зависимостей

Инструкции ниже проведут вас через процесс установки каждой зависимости по
отдельности.

- Пользователи Windows должны сначала установить WSL (подсистему Windows для
  Linux), а затем установить зависимости, указанные в разделе Linux ниже.
- Пользователи Linux должны сначала установить зависимости, указанные в разделе
  Linux ниже.
- Пользователи Mac должны начать с инструкций по установке Rust ниже.

<Accordions>
<Accordion title="Windows Subsystem for Linux (WSL)">

Для разработки программ Solana на Windows **вы должны использовать
[WSL](https://learn.microsoft.com/en-us/windows/wsl/install)** (подсистему
Windows для Linux). Устанавливайте все остальные зависимости через терминал
Linux.

После установки WSL установите зависимости, указанные в разделе Linux ниже,
прежде чем приступить к установке Rust, Solana CLI и Anchor CLI.

Для установки WSL выполните следующую команду в Windows PowerShell:

```terminal
$ wsl --install
```

В процессе установки вам будет предложено создать учетную запись пользователя по
умолчанию.

"Ubuntu" в строке поиска.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Если ваш терминал Ubuntu выглядит как на изображении ниже, вы можете столкнуться
с проблемой, когда `ctrl + v` (сочетание клавиш для вставки) не работает в
терминале.

![Терминал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Если вы столкнулись с этой проблемой, откройте Windows Terminal, найдя
"Terminal" в строке поиска.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Затем закройте Windows Terminal и снова откройте терминал Linux, найдя Ubuntu.
Теперь терминал должен выглядеть как на изображении ниже, где `ctrl + v`
(сочетание клавиш для вставки) работает.

![Терминал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

команду [быстрой установки](#quick-installation) в терминале Ubuntu (Linux).

### Установка Rust

Разработчики создают программы Solana, используя
[язык программирования Rust](https://www.rust-lang.org/).

Установите Rust с помощью [rustup](https://www.rust-lang.org/tools/install).

Выполните следующую команду для установки Rust:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

После завершения установки вы должны увидеть следующее сообщение:

<Accordions>
<Accordion title="Сообщение об успешной установке Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

Выполните следующую команду, чтобы перезагрузить переменную среды PATH, включив
в нее каталог bin Cargo:

```terminal
$ . "$HOME/.cargo/env"
```

Чтобы проверить успешность установки, проверьте версию Rust:

```terminal
$ rustc --version
```

Вы должны увидеть вывод, подобный следующему:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

</Step>
<Step>

### Установка Solana CLI

Solana CLI предоставляет все инструменты, необходимые для создания и
развертывания программ Solana.

Установите набор инструментов Solana CLI, используя официальную команду
установки:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

Вы можете заменить `stable` на тег релиза, соответствующий версии программного
обеспечения желаемого выпуска (например, `v2.0.3`), или использовать одно из
трех символических имен каналов: `stable`, `beta` или `edge`.

При первой установке Solana CLI вы можете увидеть следующее сообщение,
предлагающее добавить переменную окружения `PATH`:

```
Close and reopen your terminal to apply the PATH changes or run the following in your existing shell:

export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

<Tabs groupId="language" items={["Linux", "Mac"]}>
<Tab value="Linux">

Если вы используете **Linux** или **WSL**, вам необходимо добавить бинарный файл
Solana CLI в ваш `PATH`, чтобы команда была доступна в вашем терминале.

Сначала выполните следующую команду, чтобы проверить, какую оболочку вы
используете:

```terminal
$ echo $SHELL
```

- Если вывод содержит `/bash`, используйте `.bashrc`.
- Если вывод содержит `/zsh`, используйте `.zshrc`.

В зависимости от вашей оболочки выполните соответствующую команду.

Для Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Для Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

Затем выполните следующую команду, чтобы обновить сессию терминала или
перезапустите терминал.

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

</Tab>
<Tab value="Mac">

Если вы используете Mac с `zsh`, выполнение стандартной команды `export PATH`,
записанной при установке, не сохраняется после закрытия терминала.

Вместо этого вы можете добавить PATH в файл конфигурации оболочки, выполнив
следующую команду:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

Затем выполните следующую команду, чтобы обновить сессию терминала или
перезапустите терминал.

```terminal
$ source ~/.zshrc
```

</Tab>
</Tabs>

Чтобы проверить успешность установки, проверьте версию Solana CLI:

```terminal
$ solana --version
```

Вы должны увидеть вывод, подобный следующему:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

Вы можете просмотреть все доступные версии в
[репозитории Agave на Github](https://github.com/anza-xyz/agave/releases).

<Callout>

Agave — это клиент validator от [Anza](https://www.anza.xyz/), ранее известный
как клиент validator от Solana Labs.

</Callout>

Чтобы позже обновить Solana CLI до последней версии, вы можете использовать
следующую команду:

```terminal
$ agave-install update
```

</Step>

### Установка Anchor CLI

[Anchor](https://www.anchor-lang.com/) — это фреймворк для разработки программ
Solana. Фреймворк Anchor использует макросы Rust для упрощения процесса
написания программ Solana.

Вы можете установить Anchor CLI и инструменты двумя способами:

1. Anchor Version Manager (AVM) - Рекомендуемый метод установки
2. Без AVM - Установка напрямую из GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Менеджер версий Anchor (AVM) позволяет устанавливать и управлять различными
версиями Anchor в вашей системе и легко обновлять версии Anchor в будущем.

Установите AVM с помощью следующей команды:

```terminal
$ cargo install --git https://github.com/coral-xyz/anchor avm --force
```

Подтвердите, что AVM установлен успешно:

```terminal
$ avm --version
```

Установите последнюю версию Anchor CLI с помощью AVM:

```terminal
$ avm install latest
$ avm use latest
```

Вы можете установить определенную версию Anchor CLI, указав номер версии:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="info">

Не забудьте выполнить команду _shell`avm use`_, чтобы указать версию Anchor CLI
для запуска в вашей системе.

- Если вы установили версию `latest`, выполните _shell`avm use latest`_.
- Если вы установили версию `0.30.1`, выполните _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

Установите определенную версию Anchor CLI с помощью следующей команды:

```terminal
$ cargo install --git https://github.com/coral-xyz/anchor --tag v0.30.1 anchor-cli
```

</Tab>
</Tabs>

Чтобы проверить успешность установки, проверьте версию Anchor CLI:

```terminal
$ anchor --version
```

Вы должны увидеть вывод, подобный следующему:

```
anchor-cli 0.31.1
```

<Callout type="warn">

При установке Anchor CLI на Linux или WSL вы можете столкнуться с этой ошибкой:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Если вы видите это сообщение об ошибке:

1. Установите зависимости, перечисленные в разделе
   [Linux](/docs/intro/installation#install-dependencies) в начале этой
   страницы.
2. Повторите попытку установки Anchor CLI.

</Callout>

#### Node.js и Yarn

Файл тестирования проекта Anchor по умолчанию (TypeScript), созданный с помощью
команды _shell`anchor init`_, требует Node.js и Yarn. (Шаблон теста на Rust
доступен с помощью _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Установка Node">

Установите node с помощью
[Node Version Manager (nvm)](https://github.com/nvm-sh/nvm).

Установите nvm с помощью следующей команды:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

Перезапустите терминал и убедитесь, что команда nvm успешно выполняется:

```terminal
$ command -v nvm
```

Затем используйте `nvm` для установки node:

```terminal
$ nvm install node
```

Чтобы проверить успешность установки, проверьте версию Node:

```terminal
$ node --version
```

Вы должны увидеть вывод, подобный следующему:

```
v23.7.0
```

</Accordion>
<Accordion title="Установка Yarn">

Установите Yarn:

```terminal
$ npm install --global yarn
```

Чтобы проверить успешность установки, проверьте версию Yarn:

```terminal
$ yarn --version
```

Вы должны увидеть следующий вывод:

```
1.22.1
```

</Accordion>
</Accordions>

<Callout type="warn">

При запуске `anchor build`, если вы столкнетесь со следующими ошибками:

<Accordions>
<Accordion title="error: not a directory">

```
error: not a directory: '.../solana-release/bin/sdk/sbf/dependencies/platform-tools/rust/lib'
```

Попробуйте следующие решения:

1. Принудительная установка с помощью следующей команды:

```terminal
$ cargo build-sbf --force-tools-install
```

2. Если предыдущее решение не работает, очистите кэш Solana:

```terminal
$ rm -rf ~/.cache/solana/*
```

</Accordion>

<Accordion title="lock file version 4 requires `-Znext-lockfile-bump">
Вы можете исправить это, изменив поле версии в файле `Cargo.lock` с 4 на 3:

```diff title="Cargo.lock"
-version = 4
+version = 3
```

Смотрите [этот issue](https://github.com/coral-xyz/anchor/issues/3392) для
получения дополнительной информации.

</Accordion>

</Accordions>

После применения вышеуказанного решения попробуйте снова запустить
_shell`anchor build`_.

</Callout>

<Callout type="warn">

При запуске _shell`anchor test`_ после создания нового проекта Anchor на Linux
или WSL, вы можете столкнуться со следующими ошибками, если у вас не установлены
Node.js или Yarn:

```
Permission denied (os error 13)
```

```
No such file or directory (os error 2)
```

</Callout>

## Основы Solana CLI

В этом разделе рассматриваются некоторые общие команды Solana CLI, чтобы помочь
вам начать работу.

<Steps>
<Step>

### Конфигурация Solana

Чтобы просмотреть текущую конфигурацию:

```terminal
$ solana config get
```

Вы должны увидеть вывод, подобный следующему:

```
Config File: /Users/test/.config/solana/cli/config.yml
RPC URL: https://api.mainnet-beta.solana.com
WebSocket URL: wss://api.mainnet-beta.solana.com/ (computed)
Keypair Path: /Users/test/.config/solana/id.json
Commitment: confirmed
```

RPC URL и Websocket URL указывают кластер Solana, к которому CLI отправляет
запросы.

Вы можете обновить кластер Solana CLI, используя следующие команды:

```terminal
$ solana config set --url mainnet-beta
$ solana config set --url devnet
$ solana config set --url localhost
$ solana config set --url testnet
```

Вы также можете использовать следующие короткие опции:

```terminal
$ solana config set -um    # For mainnet-beta
$ solana config set -ud    # For devnet
$ solana config set -ul    # For localhost
$ solana config set -ut    # For testnet
```

Путь к keypair указывает на кошелек Solana по умолчанию (keypair), используемый
CLI Solana для оплаты комиссий за транзакции и развертывания программ. По
умолчанию этот файл хранится в `~/.config/solana/id.json`.

</Step>
<Step>

### Создание кошелька

Для отправки транзакций с помощью Solana CLI вам нужен кошелек Solana,
пополненный SOL.

Чтобы сгенерировать keypair по пути по умолчанию, выполните следующую команду:

```terminal
$ solana-keygen new
```

Вы должны увидеть вывод, подобный следующему:

```
Generating a new keypair

For added security, enter a BIP39 passphrase

NOTE! This passphrase improves security of the recovery seed phrase NOT the
keypair file itself, which is stored as insecure plain text

BIP39 Passphrase (empty for none):

Wrote new keypair to /Users/test/.config/solana/id.json
===========================================================================
pubkey: 8dBTPrjnkXyuQK3KDt9wrZBfizEZijmmUQXVHpFbVwGT
===========================================================================
Save this seed phrase and your BIP39 passphrase to recover your new keypair:
cream bleak tortoise ocean nasty game gift forget fancy salon mimic amazing
===========================================================================
```

<Callout type="info">

Если у вас уже есть кошелек в файловой системе, сохраненный в месте по
умолчанию, эта команда не перезапишет его, если вы явно не укажете флаг
`--force`.

</Callout>

Чтобы просмотреть адрес вашего кошелька (публичный ключ), выполните:

```terminal
$ solana address
```

</Step>
<Step>

### Аирдроп SOL

Запросите аирдроп SOL на ваш кошелек для оплаты транзакций и развертывания
программ.

Установите ваш кластер на devnet:

```terminal
$ solana config set -ud
```

Затем запросите аирдроп SOL в devnet:

```terminal
$ solana airdrop 2
```

<Callout>

Аирдропы в devnet ограничены до 5 SOL на запрос. Если вы столкнулись с
ограничениями или ошибками, попробуйте использовать
[Веб-кран](https://faucet.solana.com) вместо этого.

</Callout>

Чтобы проверить баланс SOL вашего кошелька, выполните следующую команду:

```terminal
$ solana balance
```

</Step>
<Step>

### Запустите локальный валидатор

Solana CLI включает встроенный
[тестовый validator](https://docs.anza.xyz/cli/examples/test-validator) для
локальной разработки.

В отдельном терминале выполните следующую команду для запуска локального
валидатора:

```terminal
$ solana-test-validator
```

<Callout>

Не забудьте также обновить ваш CLI для использования localhost перед выполнением
команд Solana CLI:

```terminal
$ solana config set -ul
```

</Callout>

</Step>
</Steps>

## Основы Anchor CLI

В этом разделе рассматриваются некоторые общие команды Anchor CLI для начала
работы. Для получения дополнительной информации об Anchor CLI см.
[документацию Anchor](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Инициализация проекта

Чтобы создать новый проект Anchor, выполните следующую команду:

```terminal
$ anchor init <project-name>
```

Например, чтобы создать проект с названием `my-project`, выполните:

```terminal
$ anchor init my-project
```

Эта команда создает новую директорию с именем проекта и инициализирует новый
проект Anchor с базовой программой на Rust и шаблоном тестов на TypeScript.

Перейдите в директорию проекта:

```terminal
$ cd <project-name>
```

Ознакомьтесь со
[структурой файлов](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
проекта Anchor.

</Step>

<Step>
### Сборка программы

Для сборки проекта выполните следующую команду:

```terminal
$ anchor build
```

Скомпилированную программу можно найти в директории `/target/deploy`.

</Step>
<Step>

### Развертывание программы

Для развертывания проекта выполните следующую команду:

```terminal
$ anchor deploy
```

Эта команда развертывает вашу программу в `cluster`, указанном в файле
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Тестирование программы

Для тестирования проекта выполните следующую команду:

```terminal
$ anchor test
```

<Callout>

Эта команда собирает, развертывает и запускает тесты для вашего проекта.

При использовании `localnet` в качестве `cluster` в `Anchor.toml`, Anchor
автоматически запускает локальный validator, развертывает вашу программу,
запускает тесты и затем останавливает validator.

</Callout>

</Step>
</Steps>
