Этот функциональный шлюз снижает стандартное распределение CU для инструкций
встроенных программ с 200 000 до 3 000 CU для улучшения производительности
validator.

## Влияние

Менее 0,00001% транзакций будут затронуты этим изменением.

**Разработчики** всегда должны использовать лимиты вычислительного бюджета для
явного распределения CU для своих транзакций. Это изменение не повлияет на
транзакции, которые явно распределяют CU.

Следующие встроенные программы будут иметь уменьшенное стандартное распределение
CU до 3 000 CU в результате этого изменения:
