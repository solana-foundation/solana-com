```rs !!request title="Rust"
use anyhow::Result;
use solana_client::{nonblocking::rpc_client::RpcClient, rpc_config::RpcTransactionConfig};
use solana_sdk::{commitment_config::CommitmentConfig, signature::Signature};
use solana_transaction_status_client_types::UiTransactionEncoding;
use std::str::FromStr;

#[tokio::main]
async fn main() -> Result<()> {
    let client = RpcClient::new_with_commitment(
        String::from("https://api.devnet.solana.com"),
        CommitmentConfig::confirmed(),
    );

    // !hover(1:3) signature
    let tx_sig = Signature::from_str(
        "5zSQuTcWsPy2cVAshBXWuJJXLwMD1GbgMpz3iq4xgwiV1s6mxYRbYb7qBiRGZd1xvDcYhQQRBKoNcnW8eKtcyZWg",
    )?;

    // !hover(1:5) 1
    let config = RpcTransactionConfig {
        // !hover commitment
        commitment: CommitmentConfig::finalized().into(),
        // !hover encoding
        encoding: UiTransactionEncoding::Base64.into(),
        // !hover maxSupportedTransactionVersion
        max_supported_transaction_version: Some(0),
    };

    let transaction = client.get_transaction_with_config(&tx_sig, config).await?;

    println!("{:#?}", transaction);

    Ok(())
}
```

### !params

#### !! signature

!type string
!required

Firma de la transacción, como cadena codificada en base-58

#### !! config

!type object

Objeto de configuración que contiene los siguientes campos:

##### !! commitment

!type string
!values confirmed finalized
!default finalized

El commitment describe qué tan finalizado está un bloque en ese momento. Ver
[Configuración del State Commitment](/docs/rpc#configuring-state-commitment).

- `processed` no es compatible.

##### !! maxSupportedTransactionVersion

!type number
!values 0
!default 0

Actualmente, el único valor válido para este parámetro es `0`. Establecerlo en
`0` te permite obtener todas las transacciones, incluyendo tanto las
transacciones versionadas como las heredadas.

Este parámetro determina la versión máxima de transacción que se devolverá en la
respuesta. Si solicitas una transacción con una versión superior a este valor,
se devolverá un error. Si omites este parámetro, solo se devolverán
transacciones heredadas—cualquier transacción versionada resultará en un error.

##### !! encoding

!type string
!values json jsonParsed base64 base58
!default json

Codificación para la Transaction devuelta

- `jsonParsed` intenta usar analizadores de estado específicos del programa para
  devolver datos más legibles y explícitos en la lista
  `transaction.message.instructions`.
- Si se solicita `jsonParsed` pero no se puede encontrar un analizador, la
  instrucción vuelve a la codificación JSON regular (campos `accounts`, `data` y
  `programIdIndex`).

!type object | null

Devuelve `null` si la transacción no se encuentra o no está confirmada, de lo
contrario devuelve un objeto que contiene:

##### !! blockTime

!type i64 | null

Tiempo estimado de producción, como marca de tiempo Unix (segundos desde la
época Unix) de cuando se procesó la transacción. null si no está disponible

##### !! meta

!type object | null

Estado de la transacción
[objeto de metadatos](/docs/rpc/json-structures#transaction-status-metadata) o
`null`.

##### !! slot

!type u64

El slot en el que se procesó esta transacción

##### !! transaction

!type object | [string,encoding]

Objeto [Transaction](/docs/rpc/json-structures#transactions), ya sea en formato
JSON o datos binarios codificados, dependiendo del parámetro de codificación

##### !! version

!type "legacy" | number | undefined

Versión de la transacción. Indefinido si `maxSupportedTransactionVersion` no
está establecido en los parámetros de la solicitud.
