h1: blockSubscribe RPCメソッド
---

新しいブロックが`confirmed`または
`finalized`されるたびに通知を受け取るためのサブスクリプション。

<Callout type="warn" title={"不安定なメソッド"}>
  このサブスクリプションは**不安定**と見なされ、validatorが`--rpc-pubsub-enable-block-subscription`フラグで
  起動された場合にのみ利用可能です。このサブスクリプションの形式は将来変更される可能性があります。
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    // !hover(1:3) 0
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    // !hover(1:6) 1
    {
      // !hover commitment
      "commitment": "confirmed",
      // !hover encoding
      "encoding": "base64",
      // !hover transactionDetails
      "transactionDetails": "full",
      // !hover maxSupportedTransactionVersion
      "maxSupportedTransactionVersion": 0,
      // !hover showRewards
      "showRewards": true
    }
  ]
}
```

### !params

#### !! 0

!type string | object  
!required

アカウントタイプ別に結果を受け取るためのログのフィルター条件。現在サポートされているもの：

- `all` - ブロック内のすべてのトランザクションを含む
- 以下のフィールドを持つJSONオブジェクト：
  - `mentionsAccountOrProgram: <string>` - 指定された公開鍵（base-58エンコード文字列として）に言及するトランザクションのみを返します。特定のブロックに言及がない場合、通知は送信されません。

#### !! 1

!type object

以下のフィールドを含む設定オブジェクト：

##### !! commitment

!type string !values confirmed finalized !default finalized

コミットメントは、その時点でブロックがどの程度確定しているかを示します。
[ステートコミットメントの設定](/docs/rpc#configuring-state-commitment)を参照してください。

- `processed` はサポートされていません。

##### !! encoding

!type string  
!values json jsonParsed base58 base64  
!default json

返される各トランザクションのエンコーディング形式

- `jsonParsed` は、プログラム固有のinstructionsパーサーを使用して、
  `transaction.message.instructions`リストにより人間が読みやすく明示的なデータを返そうとします。
- `jsonParsed`が要求されたがパーサーが見つからない場合、instructionは通常のJSONエンコーディング（`accounts`、`data`、および`programIdIndex`
  フィールド）にフォールバックします。

##### !! transactionDetails

!type string  
!values full accounts signatures none  
!default full

返すトランザクション詳細のレベル

- `accounts`
  がリクエストされた場合、トランザクションの詳細には署名と各トランザクションのアカウントの注釈付きリストのみが含まれます。
- トランザクションのメタデータは以下のみに制限されます：fee、err、pre_balances、post_balances、pre_token_balances、およびpost_token_balances。

##### !! maxSupportedTransactionVersion

!type number !values 0 !default 0

現在、このパラメータの有効な値は `0` のみです。これを `0`
に設定すると、バージョン管理されたトランザクションとレガシートランザクションの両方を含む、すべてのトランザクションを取得できます。

このパラメータは、レスポンスで返されるトランザクションの最大バージョンを決定します。この値より高いバージョンのトランザクションをリクエストすると、エラーが返されます。このパラメータを省略すると、レガシートランザクションのみが返され、バージョン管理されたトランザクションはすべてエラーになります。

##### !! showRewards

!type bool

`rewards`
配列を表示するかどうか。パラメータが提供されない場合、デフォルトでは報酬が含まれます。

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

サブスクリプションID（登録解除に必要）

</APIMethod>

#### 通知フォーマット：

通知は以下のフィールドを持つオブジェクトになります：

- `slot: <u64>` - 対応するslot。
- `err: <object|null>` - 通知の公開中に問題が発生した場合はエラー、それ以外の場合はnull。
- `block: <object|null>` - [getBlock](/docs/rpc/http/getblock) RPC
  HTTPメソッドで見られるブロックオブジェクト。
