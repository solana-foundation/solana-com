Gibt Transaktionsdetails für eine bestätigte Transaktion zurück

<APIMethod>

```jsonc !!request curl
    // !hover(1:5) config
    {
```


```rs !!request title="Rust"
use anyhow::Result;
use solana_client::{nonblocking::rpc_client::RpcClient, rpc_config::RpcTransactionConfig};
use solana_sdk::{commitment_config::CommitmentConfig, signature::Signature};
use solana_transaction_status_client_types::UiTransactionEncoding;
use std::str::FromStr;

#[tokio::main]
async fn main() -> Result<()> {
    let client = RpcClient::new_with_commitment(
        String::from("https://api.devnet.solana.com"),
        CommitmentConfig::confirmed(),
    );

    // !hover(1:3) signature
    let tx_sig = Signature::from_str(
        "5zSQuTcWsPy2cVAshBXWuJJXLwMD1GbgMpz3iq4xgwiV1s6mxYRbYb7qBiRGZd1xvDcYhQQRBKoNcnW8eKtcyZWg",
    )?;

    // !hover(1:5) 1
    let config = RpcTransactionConfig {
        // !hover commitment
        commitment: CommitmentConfig::finalized().into(),
        // !hover encoding
        encoding: UiTransactionEncoding::Base64.into(),
        // !hover maxSupportedTransactionVersion
        max_supported_transaction_version: Some(0),
    };

    let transaction = client.get_transaction_with_config(&tx_sig, config).await?;

    println!("{:#?}", transaction);

    Ok(())
}
```

### !params

#### !! signature

!type string
!required

Transaktionssignatur als Base-58-kodierte Zeichenfolge

#### !! config

!type object

Konfigurationsobjekt mit den folgenden Feldern:

##### !! commitment

!type string
!values confirmed finalized
!default finalized

Das Commitment beschreibt, wie endgültig ein Block zu diesem Zeitpunkt ist.
Siehe
[Konfiguration des State Commitment](/docs/rpc#configuring-state-commitment).

- `processed` wird nicht unterstützt.

##### !! maxSupportedTransactionVersion

!type number
!values 0
!default 0

Derzeit ist der einzige gültige Wert für diesen Parameter `0`. Wenn Sie ihn auf
`0` setzen, können Sie alle Transaktionen abrufen, einschließlich versionierter
und Legacy-Transaktionen.

Dieser Parameter bestimmt die maximale Transaktionsversion, die in der Antwort
zurückgegeben wird. Wenn Sie eine Transaktion mit einer höheren Version als
diesem Wert anfordern, wird ein Fehler zurückgegeben. Wenn Sie diesen Parameter
weglassen, werden nur Legacy-Transaktionen zurückgegeben – jede versionierte
Transaktion führt zu einem Fehler.

##### !! encoding

!type string
!values json jsonParsed base64 base58
!default json

Kodierung für die zurückgegebene Transaktion

- `jsonParsed` Kodierung versucht, programmspezifische State-Parser zu
  verwenden, um menschenlesbarere und explizitere Daten in der
  `transaction.message.instructions` Liste zurückzugeben.
- Wenn `jsonParsed` angefordert wird, aber kein Parser gefunden werden kann,
  fällt die Anweisung auf die reguläre JSON-Kodierung zurück (`accounts`, `data`
  und `programIdIndex` Felder).

!type object | null

Gibt `null` zurück, wenn die Transaktion nicht gefunden oder nicht bestätigt
wurde, andernfalls wird ein Objekt zurückgegeben, das Folgendes enthält:

##### !! blockTime

!type i64 | null

Geschätzte Produktionszeit als Unix-Zeitstempel (Sekunden seit der Unix-Epoche),
wann die Transaktion verarbeitet wurde. null, wenn nicht verfügbar

##### !! meta

!type object | null

Transaktionsstatus
[Metadatenobjekt](/docs/rpc/json-structures#transaction-status-metadata) oder
`null`.

##### !! slot

!type u64

Der slot, in dem diese Transaktion verarbeitet wurde

##### !! transaction

!type object | [string,encoding]

[Transaktions](/docs/rpc/json-structures#transactions)-Objekt, entweder im
JSON-Format oder als kodierte Binärdaten, abhängig vom Kodierungsparameter

##### !! version

!type "legacy" | number | undefined

Transaktionsversion. Undefiniert, wenn `maxSupportedTransactionVersion` nicht in
den Anfrageparametern gesetzt ist.
