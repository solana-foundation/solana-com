h1: blockSubscribe RPC-Methode
---

Abonnieren Sie Benachrichtigungen, wann immer ein neuer Block `confirmed` oder
`finalized` wird.

<Callout type="warn" title={"Instabile Methode"}>
  Dieses Abonnement gilt als **instabil** und ist nur verfügbar, wenn der
  Validator mit dem Flag `--rpc-pubsub-enable-block-subscription` gestartet
  wurde. Das Format dieses Abonnements könnte sich in Zukunft ändern.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    // !hover(1:3) 0
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    // !hover(1:6) 1
    {
      // !hover commitment
      "commitment": "confirmed",
      // !hover encoding
      "encoding": "base64",
      // !hover transactionDetails
      "transactionDetails": "full",
      // !hover maxSupportedTransactionVersion
      "maxSupportedTransactionVersion": 0,
      // !hover showRewards
      "showRewards": true
    }
  ]
}
```

### !params

#### !! 0

!type string | object  
!required

Filterkriterien für die Logs, um Ergebnisse nach Kontotyp zu erhalten; derzeit
unterstützt:

- `all` - alle Transaktionen im Block einschließen
- Ein JSON-Objekt mit dem folgenden Feld:
  - `mentionsAccountOrProgram: <string>` - gibt nur Transaktionen zurück, die
    den angegebenen öffentlichen Schlüssel erwähnen (als base-58 kodierte
    Zeichenfolge). Wenn keine Erwähnungen in einem bestimmten Block vorhanden
    sind, wird keine Benachrichtigung gesendet.

#### !! 1

!type object

Konfigurationsobjekt mit den folgenden Feldern:

##### !! commitment

!type string !values confirmed finalized !default finalized

Das Commitment beschreibt, wie endgültig ein Block zu diesem Zeitpunkt ist.
Siehe
[Konfiguration des State Commitments](/docs/rpc#configuring-state-commitment).

- `processed` wird nicht unterstützt.

##### !! encoding

!type string  
!values json jsonParsed base58 base64  
!default json

Kodierungsformat für jede zurückgegebene Transaktion

- `jsonParsed` versucht, programmspezifische Anweisungsparser zu verwenden, um
  besser lesbare und explizitere Daten in der
  `transaction.message.instructions`Liste zurückzugeben.
- Wenn `jsonParsed` angefordert wird, aber kein Parser gefunden werden kann,
  fällt die Anweisung auf die reguläre JSON-Kodierung zurück (Felder `accounts`,
  `data` und `programIdIndex`).

##### !! transactionDetails

!type string  
!values full accounts signatures none  
!default full

Detailgrad der zurückzugebenden Transaktionen

- Wenn `accounts` angefordert werden, enthalten die Transaktionsdetails nur
  Signaturen und eine kommentierte Liste von Konten in jeder Transaktion.
- Transaktionsmetadaten sind beschränkt auf: Fee, err, pre_balances,
  post_balances, pre_token_balances und post_token_balances.

##### !! maxSupportedTransactionVersion

!type number !values 0 !default 0

Derzeit ist der einzige gültige Wert für diesen Parameter `0`. Wenn Sie ihn auf
`0` setzen, können Sie alle Transaktionen abrufen, einschließlich sowohl
versionierter als auch Legacy- Transaktionen.

Dieser Parameter bestimmt die maximale Transaktionsversion, die in der Antwort
zurückgegeben wird. Wenn Sie eine Transaktion mit einer höheren Version als
diesem Wert anfordern, wird ein Fehler zurückgegeben. Wenn Sie diesen Parameter
weglassen, werden nur Legacy- Transaktionen zurückgegeben – jede versionierte
Transaktion führt zu einem Fehler.

##### !! showRewards

!type bool

ob das `rewards`Array befüllt werden soll. Wenn der Parameter nicht angegeben
wird, werden standardmäßig Rewards einbezogen.

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

Abonnement-ID (wird zum Abbestellen benötigt)

</APIMethod>

#### Benachrichtigungsformat:

Die Benachrichtigung ist ein Objekt mit den folgenden Feldern:

- `slot: <u64>` - Der entsprechende Slot.
- `err: <object|null>` - Fehler, wenn bei der Veröffentlichung der
  Benachrichtigung etwas schief gelaufen ist, andernfalls null.
- `block: <object|null>` - Ein Block-Objekt wie in der
  [getBlock](/docs/rpc/http/getblock) RPC HTTP-Methode.
