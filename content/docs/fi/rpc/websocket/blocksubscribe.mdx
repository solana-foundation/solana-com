h1: blockSubscribe RPC-metodi
---

Tilaa ilmoitukset aina kun uusi lohko on `confirmed` tai `finalized`.

<Callout type="warn" title={"Epävakaa metodi"}>
  Tätä tilausta pidetään **epävakaana** ja se on saatavilla vain, jos validator
  käynnistettiin `--rpc-pubsub-enable-block-subscription` lipulla. Tämän
  tilauksen muoto saattaa muuttua tulevaisuudessa.
</Callout>

<APIMethod>

```jsonc !!request
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "blockSubscribe",
  "params": [
    // !hover(1:3) 0
    {
      "mentionsAccountOrProgram": "LieKvPRE8XeX3Y2xVNHjKlpAScD12lYySBVQ4HqoJ5op"
    },
    // !hover(1:6) 1
    {
      // !hover commitment
      "commitment": "confirmed",
      // !hover encoding
      "encoding": "base64",
      // !hover transactionDetails
      "transactionDetails": "full",
      // !hover maxSupportedTransactionVersion
      "maxSupportedTransactionVersion": 0,
      // !hover showRewards
      "showRewards": true
    }
  ]
}
```

### !params

#### !! 0

!type string | object  
!required

suodatuskriteerit lokeille tulosten saamiseksi tilityypeittäin; tällä hetkellä
tuetut:

- `all` - sisällytä kaikki lohkon transaktiot
- JSON-objekti, jossa on seuraava kenttä:
  - `mentionsAccountOrProgram: <string>` - palauta vain transaktiot, jotka
    mainitsevat annetun julkisen avaimen (base-58-koodattuna merkkijonona). Jos
    mainintoja ei ole tietyssä lohkossa, ilmoitusta ei lähetetä.

#### !! 1

!type object

Määritysobjekti, joka sisältää seuraavat kentät:

##### !! commitment

!type string !values confirmed finalized !default finalized

Commitment kuvaa, kuinka viimeistelty lohko on tietyllä hetkellä. Katso
[Tilan sitoutumisen määrittäminen](/docs/rpc#configuring-state-commitment).

- `processed` ei ole tuettu.

##### !! encoding

!type string  
!values json jsonParsed base58 base64  
!default json

koodausmuoto jokaiselle palautetulle transaktiolle

- `jsonParsed` yrittää käyttää ohjelmakohtaisia käskyjen jäsentimiä
  palauttaakseen helpommin luettavaa ja selkeämpää dataa
  `transaction.message.instructions` listassa.
- Jos `jsonParsed` on pyydetty, mutta jäsennintä ei löydy, käsky palautuu
  tavalliseen JSON-koodaukseen (`accounts`, `data` ja `programIdIndex` kentät).

##### !! transactionDetails

!type string  
!values full accounts signatures none  
!default full

palautettavien transaktion tietojen taso

- Jos `accounts` pyydetään, tapahtuman tiedot sisältävät vain allekirjoitukset
  ja kommentoidun luettelon tileistä jokaisessa tapahtumassa.
- Tapahtuman metatiedot rajoittuvat vain seuraaviin: fee, err, pre_balances,
  post_balances, pre_token_balances ja post_token_balances.

##### !! maxSupportedTransactionVersion

!type number !values 0 !default 0

Tällä hetkellä ainoa kelvollinen arvo tälle parametrille on `0`. Sen asettaminen
arvoon `0` mahdollistaa kaikkien tapahtumien hakemisen, mukaan lukien sekä
versioidut että perinteiset tapahtumat.

Tämä parametri määrittää tapahtuman enimmäisversion, joka palautetaan
vastauksessa. Jos pyydät tapahtumaa, jonka versio on korkeampi kuin tämä arvo,
palautetaan virhe. Jos jätät tämän parametrin pois, vain perinteiset tapahtumat
palautetaan—mikä tahansa versioitu tapahtuma aiheuttaa virheen.

##### !! showRewards

!type bool

määrittää täytetäänkö `rewards` taulukko. Jos parametria ei anneta,
oletusarvoisesti palkkiot sisällytetään.

### !!result

```jsonc !response
{
  "jsonrpc": "2.0",
  // !hover result
  "result": 0,
  "id": 1
}
```

!type integer

tilauksen tunnus (tarvitaan tilauksen peruuttamiseen)

</APIMethod>

#### Ilmoituksen muoto:

Ilmoitus on objekti, jossa on seuraavat kentät:

- `slot: <u64>` - Vastaava slot.
- `err: <object|null>` - Virhe, jos ilmoituksen julkaisemisessa meni jotain
  pieleen, muuten null.
- `block: <object|null>` - Lohko-objekti, kuten nähdään
  [getBlock](/docs/rpc/http/getblock) RPC HTTP -metodissa.
