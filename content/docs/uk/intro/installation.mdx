Цей розділ охоплює кроки для налаштування вашого локального середовища для
розробки на Solana.

## Швидке встановлення

На Mac та Linux запустіть цю єдину команду для встановлення всіх залежностей.

```terminal
$ curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

<Callout type="warn">
  Користувачі Windows: спочатку вам потрібно встановити WSL (див. [Встановлення
  залежностей](#install-dependencies)). Потім запустіть попередню команду в
  терміналі Ubuntu (Linux).
</Callout>

Після встановлення ви повинні побачити вивід, схожий на такий:

```
Installed Versions:
Rust: rustc 1.86.0 (05f9846f8 2025-03-31)
Solana CLI: solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
Anchor CLI: anchor-cli 0.31.1
Node.js: v23.11.0
Yarn: 1.22.1
```

<Callout>

Якщо команда швидкого встановлення не працює, зверніться до розділу
[Встановлення залежностей](#install-dependencies) нижче для інструкцій з
встановлення кожної залежності окремо.

Якщо команда швидкого встановлення виконується успішно, перейдіть до розділів
[Основи Solana CLI](#solana-cli-basics) та
[Основи Anchor CLI](#anchor-cli-basics) нижче.

</Callout>

## Встановлення залежностей

Інструкції нижче проведуть вас через встановлення кожної залежності окремо.

- Користувачі Windows повинні спочатку встановити WSL (підсистему Windows для
  Linux), а потім встановити залежності, вказані в розділі Linux нижче.
- Користувачі Linux повинні спочатку встановити залежності, вказані в розділі
  Linux нижче.
- Користувачі Mac повинні почати з інструкцій з встановлення Rust нижче.

<Accordions>
<Accordion title="Windows Subsystem for Linux (WSL)">

Щоб розробляти програми Solana на Windows **ви повинні використовувати
[WSL](https://learn.microsoft.com/en-us/windows/wsl/install)** (підсистему
Windows для Linux). Встановлюйте всі інші залежності через термінал Linux.

Після встановлення WSL встановіть залежності, вказані в розділі Linux нижче,
перш ніж переходити до встановлення Rust, Solana CLI та Anchor CLI.

Щоб встановити WSL, виконайте наступну команду в Windows PowerShell:

```terminal
$ wsl --install
```

Процес встановлення запропонує вам створити обліковий запис користувача за
замовчуванням.

![Встановлення WSL](/assets/docs/intro/installation/wsl-install.png)

За замовчуванням WSL встановлює Ubuntu. Ви можете відкрити термінал Linux,
шукаючи "Ubuntu" в рядку пошуку.

![WSL Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-search.png)

Якщо ваш термінал Ubuntu виглядає як на зображенні нижче, ви можете зіткнутися з
проблемою, де `ctrl + v` (комбінація клавіш для вставки) не працює в терміналі.

![Термінал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-1.png)

Якщо ви зіткнулися з цією проблемою, відкрийте Windows Terminal, шукаючи
"Terminal" в рядку пошуку.

![Windows Terminal](/assets/docs/intro/installation/wsl-windows-terminal.png)

Далі закрийте Windows Terminal і знову відкрийте термінал Linux, шукаючи Ubuntu
знову. Тепер термінал повинен виглядати як на зображенні нижче, де `ctrl + v`
(комбінація клавіш для вставки) працює.

![Термінал Ubuntu](/assets/docs/intro/installation/wsl-ubuntu-terminal-2.png)

Якщо ви використовуєте VS Code,
[розширення WSL](https://code.visualstudio.com/docs/remote/wsl-tutorial)
дозволяє вам використовувати WSL і VS Code разом.

![Налаштування WSL у VS Code](/assets/docs/intro/installation/wsl-vscode.png)

Тоді ви повинні побачити наступне в рядку стану VS Code:

![WSL: Ubuntu](/assets/docs/intro/installation/wsl-vscode-ubuntu.png)

Після налаштування WSL встановіть усі інші залежності через термінал Ubuntu
(Linux).

Щоб встановити необхідні залежності Solana, спочатку спробуйте запустити команду
[швидкого встановлення](#quick-installation) в терміналі Ubuntu (Linux).

<Callout>

Якщо команда швидкого встановлення не працює, встановіть залежності, вказані в
розділі Linux нижче, перш ніж переходити до окремих кроків для встановлення
Rust, Solana CLI та Anchor CLI.

Якщо команда швидкого встановлення виконується успішно, перейдіть до розділів
[Основи Solana CLI](#solana-cli-basics) та
[Основи Anchor CLI](#anchor-cli-basics) нижче.

</Callout>

### Встановлення Rust

Розробники створюють програми Solana за допомогою
[мови програмування Rust](https://www.rust-lang.org/).

Встановіть Rust за допомогою [rustup](https://www.rust-lang.org/tools/install).

Виконайте наступну команду для встановлення Rust:

```terminal
$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
```

Після завершення встановлення ви повинні побачити таке повідомлення:

<Accordions>
<Accordion title="Повідомлення про успішне встановлення Rust">

```
Rust is installed now. Great!

To get started you may need to restart your current shell.
This would reload your PATH environment variable to include
Cargo's bin directory ($HOME/.cargo/bin).

To configure your current shell, you need to source
the corresponding env file under $HOME/.cargo.

This is usually done by running one of the following (note the leading DOT):
. "$HOME/.cargo/env"            # For sh/bash/zsh/ash/dash/pdksh
source "$HOME/.cargo/env.fish"  # For fish
```

</Accordion>
</Accordions>

Виконайте наступну команду, щоб перезавантажити змінну середовища PATH,
включивши директорію bin Cargo:

```terminal
$ . "$HOME/.cargo/env"
```

Щоб перевірити успішність встановлення, перевірте версію Rust:

```terminal
$ rustc --version
```

Ви повинні побачити результат, схожий на такий:

```
rustc 1.86.0 (05f9846f8 2025-03-31)
```

### Встановлення Solana CLI

Solana CLI надає всі інструменти, необхідні для створення та розгортання програм
Solana.

Встановіть набір інструментів Solana CLI за допомогою офіційної команди
встановлення:

```terminal
$ sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
```

Ви можете замінити `stable` на тег релізу, що відповідає версії програмного
забезпечення вашого бажаного випуску (наприклад, `v2.0.3`), або використовувати
одну з трьох символічних назв каналів: `stable`, `beta` або `edge`.

При першому встановленні Solana CLI ви можете побачити наступне повідомлення, що
пропонує вам додати змінну середовища `PATH`:

```
Close and reopen your terminal to apply the PATH changes or run the following in your existing shell:

export PATH="/Users/test/.local/share/solana/install/active_release/bin:$PATH"
```

<Tabs groupId="language" items={["Linux", "Mac"]}>
<Tab value="Linux">

Якщо ви використовуєте **Linux** або **WSL**, вам потрібно додати бінарний файл
Solana CLI до вашого `PATH`, щоб команда була доступна у вашому терміналі.

Спочатку виконайте наступну команду, щоб перевірити, яку оболонку ви
використовуєте:

```terminal
$ echo $SHELL
```

- Якщо результат містить `/bash`, використовуйте `.bashrc`.
- Якщо результат містить `/zsh`, використовуйте `.zshrc`.

Залежно від вашої оболонки, виконайте відповідну команду.

Для Bash (`bashrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
```

Для Zsh (`zshrc`):

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
```

Потім виконайте наступну команду, щоб оновити сеанс терміналу або перезапустіть
термінал.

```terminal
$ source ~/.bashrc # If using Bash
$ source ~/.zshrc # If using Zsh
```

</Tab>
<Tab value="Mac">

Якщо ви використовуєте Mac з `zsh`, запуск стандартної команди `export PATH`,
яка зазначена в інструкції з встановлення, не зберігається після закриття
терміналу.

Натомість ви можете додати PATH до файлу конфігурації оболонки, виконавши
наступну команду:

```terminal
$ echo 'export PATH="$HOME/.local/share/solana/install/active_release/bin:$PATH"' >> ~/.zshrc
```

Потім виконайте наступну команду, щоб оновити сеанс терміналу або перезапустіть
термінал.

```terminal
$ source ~/.zshrc
```

</Tab>
</Tabs>

Щоб перевірити успішність встановлення, перевірте версію Solana CLI:

```terminal
$ solana --version
```

Ви повинні побачити результат, схожий на такий:

```
solana-cli 2.2.12 (src:0315eb6a; feat:1522022101, client:Agave)
```

Ви можете переглянути всі доступні версії в
[репозиторії Agave на Github](https://github.com/anza-xyz/agave/releases).

<Callout>

Agave — це клієнт validator від [Anza](https://www.anza.xyz/), раніше відомий як
клієнт validator від Solana Labs.

</Callout>

Щоб пізніше оновити Solana CLI до останньої версії, ви можете використати
наступну команду:

```terminal
$ agave-install update
```

</Step>

### Встановлення Anchor CLI

[Anchor](https://www.anchor-lang.com/) — це фреймворк для розробки програм
Solana. Фреймворк Anchor використовує макроси Rust для спрощення процесу
написання програм Solana.

Ви можете встановити Anchor CLI та інструменти двома способами:

1. Anchor Version Manager (AVM) — рекомендований метод встановлення
2. Без AVM — встановлення безпосередньо з GitHub

<Tabs groupId="anchor" items={["AVM", "Without AVM"]}>
<Tab value="AVM">

Менеджер версій Anchor (AVM) дозволяє встановлювати та керувати різними версіями
Anchor у вашій системі та легко оновлювати версії Anchor у майбутньому.

Встановіть AVM за допомогою наступної команди:

```terminal
$ cargo install --git https://github.com/coral-xyz/anchor avm --force
```

Підтвердіть, що AVM встановлено успішно:

```terminal
$ avm --version
```

Встановіть останню версію Anchor CLI за допомогою AVM:

```terminal
$ avm install latest
$ avm use latest
```

Ви можете встановити конкретну версію Anchor CLI, вказавши номер версії:

```terminal
$ avm install 0.30.1
$ avm use 0.30.1
```

<Callout type="info">

Не забудьте виконати команду _shell`avm use`_, щоб вказати версію Anchor CLI,
яка буде використовуватися у вашій системі.

- Якщо ви встановили версію `latest`, виконайте _shell`avm use latest`_.
- Якщо ви встановили версію `0.30.1`, виконайте _shell`avm use 0.30.1`_.

</Callout>

</Tab>

<Tab value="Without AVM">

Встановіть конкретну версію Anchor CLI за допомогою наступної команди:

```terminal
$ cargo install --git https://github.com/coral-xyz/anchor --tag v0.30.1 anchor-cli
```

</Tab>
</Tabs>

Щоб перевірити успішність встановлення, перевірте версію Anchor CLI:

```terminal
$ anchor --version
```

Ви повинні побачити результат, схожий на такий:

```
anchor-cli 0.31.1
```

<Callout type="warn">

При встановленні Anchor CLI на Linux або WSL ви можете зіткнутися з такою
помилкою:

```
error: could not exec the linker cc = note: Permission denied (os error 13)
```

Якщо ви бачите це повідомлення про помилку:

1. Встановіть залежності, перелічені в розділі
   [Linux](/docs/intro/installation#install-dependencies) на початку цієї
   сторінки.
2. Спробуйте встановити Anchor CLI знову.

</Callout>

#### Node.js та Yarn

Типовий тестовий файл проєкту Anchor (TypeScript), створений за допомогою
команди _shell`anchor init`_, потребує Node.js та Yarn. (Шаблон тесту Rust
доступний за допомогою _shell`anchor init --test-template rust`_)

<Accordions>
<Accordion title="Встановлення Node">

Встановіть node за допомогою
[Node Version Manager (nvm)](https://github.com/nvm-sh/nvm).

Встановіть nvm за допомогою наступної команди:

```terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/master/install.sh | bash
```

Перезапустіть термінал і переконайтеся, що команда nvm успішно виконується:

```terminal
$ command -v nvm
```

Далі використайте `nvm` для встановлення node:

```terminal
$ nvm install node
```

Щоб перевірити успішність встановлення, перевірте версію Node:

```terminal
$ node --version
```

Ви повинні побачити такий результат:

```
v23.7.0
```

</Accordion>
<Accordion title="Встановлення Yarn">

Встановіть Yarn:

```terminal
$ npm install --global yarn
```

Щоб перевірити успішність встановлення, перевірте версію Yarn:

```terminal
$ yarn --version
```

Ви повинні побачити такий результат:

```
1.22.1
```

</Accordion>
</Accordions>

<Callout type="warn">

Якщо під час запуску `anchor build` ви зіткнетеся з такими помилками:

<Accordions>
<Accordion title="error: not a directory">


2. Якщо попереднє рішення не працює, очистіть кеш Solana:

</Accordion>

## Основи Solana CLI

Цей розділ розглядає деякі поширені команди Solana CLI, щоб допомогти вам почати
роботу.

<Steps>
<Step>

### Конфігурація Solana

Щоб переглянути поточну конфігурацію:

```terminal
$ solana config get
```

Ви повинні побачити вивід, схожий на такий:

```
Config File: /Users/test/.config/solana/cli/config.yml
RPC URL: https://api.mainnet-beta.solana.com
WebSocket URL: wss://api.mainnet-beta.solana.com/ (computed)
Keypair Path: /Users/test/.config/solana/id.json
Commitment: confirmed
```

RPC URL та Websocket URL визначають кластер Solana, до якого CLI надсилає
запити.

Ви можете оновити кластер Solana CLI за допомогою таких команд:

```terminal
$ solana config set --url mainnet-beta
$ solana config set --url devnet
$ solana config set --url localhost
$ solana config set --url testnet
```

Ви також можете використовувати такі короткі опції:

```terminal
$ solana config set -um    # For mainnet-beta
$ solana config set -ud    # For devnet
$ solana config set -ul    # For localhost
$ solana config set -ut    # For testnet
```

Keypair Path вказує на стандартний гаманець Solana (keypair), який
використовується Solana CLI для оплати комісій за транзакції та розгортання
програм. За замовчуванням цей файл зберігається в `~/.config/solana/id.json`.

</Step>
<Step>

### Створення гаманця

Щоб надсилати транзакції за допомогою Solana CLI, вам потрібен гаманець Solana з
коштами в SOL.

Щоб згенерувати keypair за стандартним шляхом Keypair Path, виконайте таку
команду:

```terminal
$ solana-keygen new
```

Ви повинні побачити вивід, схожий на такий:

```
Generating a new keypair

For added security, enter a BIP39 passphrase

NOTE! This passphrase improves security of the recovery seed phrase NOT the
keypair file itself, which is stored as insecure plain text

BIP39 Passphrase (empty for none):

Wrote new keypair to /Users/test/.config/solana/id.json
===========================================================================
pubkey: 8dBTPrjnkXyuQK3KDt9wrZBfizEZijmmUQXVHpFbVwGT
===========================================================================
Save this seed phrase and your BIP39 passphrase to recover your new keypair:
cream bleak tortoise ocean nasty game gift forget fancy salon mimic amazing
===========================================================================
```

<Callout type="info">

Якщо у вас уже є гаманець файлової системи, збережений у стандартному місці, ця
команда не перезапише його, якщо ви явно не примусите перезаписати за допомогою
прапора `--force`.

</Callout>

Щоб переглянути адресу вашого гаманця (публічний ключ), виконайте:

```terminal
$ solana address
```

</Step>
<Step>

### Аirdrop SOL

Запитайте аirdrop SOL на ваш гаманець для оплати транзакцій та розгортання
програм.

Встановіть ваш кластер на devnet:

```terminal
$ solana config set -ud
```

Потім запитайте аirdrop devnet SOL:

```terminal
$ solana airdrop 2
```

<Callout>

Airdrop у devnet обмежує запити до 5 SOL за один запит. Якщо ви досягли
обмеження швидкості або зіткнулися з помилками, спробуйте натомість використати
[Web Faucet](https://faucet.solana.com).

</Callout>

Щоб перевірити баланс SOL вашого гаманця, виконайте таку команду:

```terminal
$ solana balance
```

</Step>
<Step>

### Запустіть локальний validator

Solana CLI включає вбудований
[тестовий validator](https://docs.anza.xyz/cli/examples/test-validator) для
локальної розробки.

У окремому терміналі виконайте таку команду, щоб запустити локальний validator:

```terminal
$ solana-test-validator
```

<Callout>

Не забудьте також оновити ваш CLI для використання localhost перед виконанням
команд Solana CLI:

```terminal
$ solana config set -ul
```

</Callout>

</Step>
</Steps>

## Основи Anchor CLI

У цьому розділі розглядаються деякі поширені команди Anchor CLI, щоб допомогти
вам почати. Для отримання додаткової інформації про Anchor CLI, дивіться
[документацію Anchor](https://www.anchor-lang.com/docs).

<Steps>
<Step>

### Ініціалізація проєкту

Щоб створити новий проєкт Anchor, виконайте таку команду:

```terminal
$ anchor init <project-name>
```

Наприклад, щоб створити проект під назвою `my-project`, виконайте:

```terminal
$ anchor init my-project
```

Ця команда створює новий каталог з назвою проекту та ініціалізує новий проект
Anchor з базовою програмою на Rust та шаблоном тесту на TypeScript.

Перейдіть до каталогу проекту:

```terminal
$ cd <project-name>
```

Ознайомтеся зі
[структурою файлів](https://www.anchor-lang.com/docs/quickstart/local#project-file-structure)
проекту Anchor.

</Step>

<Step>
### Збірка програми

Щоб зібрати ваш проект, виконайте таку команду:

```terminal
$ anchor build
```

Скомпільовану програму можна знайти в каталозі `/target/deploy`.

</Step>
<Step>

### Розгортання програми

Щоб розгорнути ваш проект, виконайте таку команду:

```terminal
$ anchor deploy
```

Ця команда розгортає вашу програму на `cluster`, вказаному у файлі
[`Anchor.toml`](https://www.anchor-lang.com/docs/references/anchor-toml).

</Step>
<Step>

### Тестування програми

Щоб протестувати ваш проект, виконайте таку команду:

```terminal
$ anchor test
```

<Callout>

Ця команда збирає, розгортає та запускає тести для вашого проекту.

Коли ви використовуєте `localnet` як `cluster` у `Anchor.toml`, Anchor
автоматично запускає локальний validator, розгортає вашу програму, запускає
тести, а потім зупиняє validator.

</Callout>

</Step>
</Steps>
